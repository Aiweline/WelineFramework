本文件用于团队开发拆分。每次实现只领取一个原子任务，读取本文件中的背景和对应任务说明即可，不需要一次性读取所有计划文档。任务状态来自原始索引迁移，后续应以实际代码复核结果为准。

## 2026-05-04 进度复核

本轮继续按强契约计划收敛实现和缺陷，重点处理前端工作台、队列状态、构建任务和 Service 全量测试暴露的问题。

已完成修复如下：

- 修复工作台 runtime/compiled runtime 模板契约漂移，确保 Stage1、Stage2、Build 队列完成状态优先覆盖过期 active error。
- 统一前端需求表单的页面类型自定义状态写入，保留 `page_types_user_customized` 契约字段，避免服务端默认状态覆盖用户本地选择。
- 将目标域名统一为阶段一确认必填门闩，前端按钮禁用逻辑先执行 `globalRunVirtualThemeDisabled || domainLocked`，再叠加队列和失败锁。
- 构建启动不再合成缺失的 Stage2 section spec；缺失已确认任务契约时显式抛出 `Build task missing stage-two section spec`。
- JSON parser 支持修复根对象被截断但末尾已有局部 `}` 的 AI 响应。
- 构建任务入口重启时能识别高 attempt 的遗留 running 任务并允许 reset 后重新进入 pending。
- HTML Blocks 预览在没有 `virtual_theme_id` 但已有 `virtual_pages_by_type` 时可正常加载上下文。
- QuickBuildAggregator 在 `saas` 查询器未注册时返回空订单列表，不再让页面构建相关单测被外部模块注册状态打断。
- 组件并发测试子类覆盖 `generateComponent()` 时不再误走真实 `w_query('ai')` 批处理，避免单元测试阻塞。
- `AiSiteAgentTaskPlanQueueRecoveryService` 恢复为可测试的端口化队列恢复逻辑，覆盖新队列创建、旧队列复用、异常日志和 dispatch 状态传播。
