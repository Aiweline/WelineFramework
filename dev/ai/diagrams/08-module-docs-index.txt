-------------------------------------------------------------------------------
Mandatory Module Reading Protocol
-------------------------------------------------------------------------------

1. Identify the owning module.
   Examples:
   - app/code/Weline/Theme/...      -> Weline_Theme
   - app/code/Weline/Frontend/...   -> Weline_Frontend
   - app/code/WeShop/Product/...    -> WeShop_Product

2. Read the module AI entrypoint first.
   Pattern:
   - app/code/{Vendor}/{Module}/doc/AI-INDEX.md

3. Then read module doc/README.md if it exists.
   The AI-INDEX says whether README exists and lists the task-specific docs
   currently present under that module's doc/ directory.

4. Then read only the docs and source surfaces matched to the task.
   Examples:
   - Controller work -> Controller, route docs, setup:upgrade --route guidance
   - Theme/view work -> Theme AI-INDEX, theme inheritance docs, source template
   - Query/API work -> QueryProvider docs/help, Frontend Weline.Api docs
   - Event/Hook work -> doc/event or doc/hook plus trigger/observer source

5. If AI-INDEX says the module lacks README or task docs, treat it as a code
   structure map only. Read the relevant source and add/update module docs when
   the behavior, contract, or convention is stable.

-------------------------------------------------------------------------------
Regeneration
-------------------------------------------------------------------------------

Run this after adding/removing modules, adding major module docs, or changing
module top-level structure:

php dev/ai/scripts/generate-module-ai-indexes.php

Dry run:

php dev/ai/scripts/generate-module-ai-indexes.php --dry-run

The generator only overwrites files that contain the auto-generated marker:

<!-- weline:module-ai-index:auto-generated -->

If a module needs hand-written AI guidance, remove the marker and maintain that
file manually. The generator will skip it unless --force is passed.

-------------------------------------------------------------------------------
Non-Negotiable Guardrail
-------------------------------------------------------------------------------

Before editing app/code/{Vendor}/{Module}/ source:

1. Read app/code/{Vendor}/{Module}/doc/AI-INDEX.md.
2. Read the module README and task-specific docs listed there.
3. State the matched code surface and source file path.
4. Only then read or edit source.

Skipping this rule is treated as an AI workflow defect because it causes generic
implementation and missed framework conventions.
