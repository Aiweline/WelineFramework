┌─────────────────────────────────────────────────────────────────────────────┐
│ Core Principle: Convention over Configuration (No routes.xml)                │
└─────────────────────────────────────────────────────────────────────────────┘

Auto-discovery: Controller directory structure → URL mapping
After adding Controllers: php bin/w setup:upgrade --route

┌─────────────────────────────────────────────────────────────────────────────┐
│ URL Parsing Rules                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

Frontend URL format:
  /{currency?}/{language?}/{module}/{area}/{controller}/{action}
  (currency/language 可单独出现；组合时支持任意顺序，常用 /CNY/zh_Hans_CN/...)

  Example: /CNY/zh_Hans_CN/catalog/product/view/id/123
           └─┬─┘ └────┬────┘ └──┬──┘ └──┬──┘ └──┬─┘ └──┬──┘ └─┬─┘
          currency lang   module  area  controller action params

Backend URL format:
  /{backendKey}/{currency?}/{language?}/{module}/{area}/{controller}/{action}
  (backendKey 后的 currency/language 可单独出现，组合时支持任意顺序)
  backendKey 不是固定的 `backend` 或 `admin`：必须从运行时
  `Env::getAreaRoutePrefix('backend')` 获取，并且必须作为 URL 第一段。
  `WELINE_AREA_ROUTE` 只能校验当前请求已解析的区域，不能代替 key，
  也不能让无 key URL 作为回退路径。生成后台 URL 必须复用
  `getBackendUrl()` 或等价的统一 URL builder，禁止手写后台首段。

  Example (backendKey 为当前运行时配置值 `EXAMPLE_BACKEND_KEY`):
    /EXAMPLE_BACKEND_KEY/CNY/zh_Hans_CN/catalog/product/edit/id/123
           └─┬──┘ └─┬─┘ └────┬────┘ └──┬──┘ └──┬──┘ └─┬─┘ └──┬──┘ └─┬─┘
          backend currency lang   module  area  ctrl  action params

API URL format:
  /rest/v1/{module}/{action}

  Example: /rest/v1/catalog/products

┌─────────────────────────────────────────────────────────────────────────────┐
│ Controller Directory Mapping                                                  │
└─────────────────────────────────────────────────────────────────────────────┘

Directory structure:
Weline/Catalog/
└── Controller/
    ├── Product/
    │   ├── View.php          → /catalog/product/view
    │   ├── List.php          → /catalog/product/list
    │   └── Search.php        → /catalog/product/search
    ├── Category/
    │   └── View.php          → /catalog/category/view
    └── Backend/              ← Backend controllers
        └── Product/
            ├── Index.php     → /{backendKey}/catalog/product/index
            ├── Edit.php      → /{backendKey}/catalog/product/edit
            └── Save.php      → /{backendKey}/catalog/product/save

Mapping rule:
  Controller/{Area}/{Controller}/{Action}.php
       ↓         ↓         ↓          ↓
  /{module}/{area}/{controller}/{action}

┌─────────────────────────────────────────────────────────────────────────────┐
│ Route Parsing Flow                                                            │
└─────────────────────────────────────────────────────────────────────────────┘

HTTP Request: /{backendKey}/catalog/product/edit/id/123
     │
     ▼
┌──────────────────────────────────────────────────────────────────────────┐
│ Router\Core::parse($url)                                                  │
│   1. Extract and require backendKey (from Env::getAreaRoutePrefix('backend'))│
│   2. Extract currency/language (optional, use defaults)                   │
│   3. Extract module/area/controller/action                                │
│   4. Extract params (id=123)                                              │
└────┬─────────────────────────────────────────────────────────────────────┘
     │
     ▼
┌──────────────────────────────────────────────────────────────────────────┐
│ Router\Handle::dispatch()                                                 │
│   1. Find Controller class:                                               │
│      Weline\Catalog\Controller\Backend\Product\Edit                      │
│   2. Check if class exists                                                │
│   3. Check ACL permission (if #[Acl] attribute exists)                    │
│   4. Instantiate Controller (dependency injection)                        │
└────┬─────────────────────────────────────────────────────────────────────┘
     │
     ▼
┌──────────────────────────────────────────────────────────────────────────┐
│ Controller::execute()                                                     │
│   → Execute business logic                                                │
│   → Return Response or View                                               │
└──────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ Controller Base Classes                                                       │
└─────────────────────────────────────────────────────────────────────────────┘

Frontend Controller:
┌────────────────────────────────────────────────────────────────────────┐
│ namespace Weline\Catalog\Controller\Product;                            │
│                                                                         │
│ use Weline\Framework\App\Controller\AbstractController;                │
│                                                                         │
│ class View extends AbstractController {                                │
│     public function execute() {                                         │
│         $productId = $this->request->getParam('id');                    │
│         // Business logic                                               │
│         return $this->fetch();  // Render template                      │
│     }                                                                   │
│ }                                                                       │
└────────────────────────────────────────────────────────────────────────┘

Backend Controller:
┌────────────────────────────────────────────────────────────────────────┐
│ namespace Weline\Catalog\Controller\Backend\Product;                   │
│                                                                         │
│ use Weline\Admin\Controller\BaseController;                            │
│ use Weline\Framework\Acl\Acl;                                          │
│                                                                         │
│ #[Acl('Weline_Catalog::product_edit')]                                 │
│ class Edit extends BaseController {                                    │
│     public function execute() {                                         │
│         // Business logic                                               │
│         return $this->fetch();                                          │
│     }                                                                   │
│ }                                                                       │
└────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│ URL Generation                                                                │
└─────────────────────────────────────────────────────────────────────────────┘

Frontend URL:
  $this->getUrl('catalog/product/view', ['id' => 123])
  → /CNY/zh_Hans_CN/catalog/product/view/id/123

Backend URL:
  $this->getBackendUrl('catalog/product/edit', ['id' => 123])
  → /{runtimeBackendKey}/CNY/zh_Hans_CN/catalog/product/edit/id/123

API URL:
  $this->getUrl('rest/v1/catalog/products')
  → /rest/v1/catalog/products

┌─────────────────────────────────────────────────────────────────────────────┐
│ Route Configuration (etc/env.php)                                            │
└─────────────────────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────────────────┐
│ return [                                                                │
│     'router' => [                                                       │
│         'Weline_Catalog' => [                                           │
│             'path' => 'catalog',  // URL prefix                         │
│         ],                                                              │
│     ],                                                                  │
│     'backend_router' => [                                               │
│         'Weline_Catalog' => [                                           │
│             'path' => 'catalog',                                        │
│         ],                                                              │
│     ],                                                                  │
│ ];                                                                      │
└────────────────────────────────────────────────────────────────────────┘

说明：模块 `etc/env.php` 中的 `backend_router` 配置值（例如 `admin`）是模块
内部路由名，不是后台入口 key。后台入口 key 由 `router.area_routes.backend.prefix`
运行时配置提供；修改或排查后台地址时，先读取
`Env::getAreaRoutePrefix('backend')`，再把它作为第一段拼接模块路由。
例如 i18n 国家页面的结构是：
`/{backendKey}/i18n/backend/countries`，不能省略 `/{backendKey}`。

┌─────────────────────────────────────────────────────────────────────────────┐
│ Route Cache                                                                   │
└─────────────────────────────────────────────────────────────────────────────┘

Route mapping table cached in: generated/router/routes.php

Structure:
┌────────────────────────────────────────────────────────────────────────┐
│ return [                                                                │
│     'catalog/product/view' => [                                         │
│         'class' => 'Weline\Catalog\Controller\Product\View',           │

│         'acl' => null,                                                  │
│     ],                                                                  │
│     'admin/catalog/product/edit' => [                                   │
│         'class' => 'Weline\Catalog\Controller\Backend\Product\Edit',   │
│         'module' => 'Weline_Catalog',                                   │
│         'acl' => 'Weline_Catalog::product_edit',                        │
│     ],                                                                  │
│ ];                                                                      │
└────────────────────────────────────────────────────────────────────────┘

Update cache: php bin/w setup:upgrade --route

┌─────────────────────────────────────────────────────────────────────────────┐
│ Route Verification (Debug Tool)                                              │
└─────────────────────────────────────────────────────────────────────────────┘

Frontend route:
  php bin/w http:request /catalog/product/view/id/123

Backend route (auto-login):
  php bin/w http:request admin/catalog/product/edit/id/123 -b

API route:
  php bin/w http:request rest/v1/catalog/products -api

Filter output:
  php bin/w http:request admin -b --filter=Warning
  php bin/w http:request admin -b --filter=Fatal

┌─────────────────────────────────────────────────────────────────────────────┐
│ 404 Handling                                                                  │
└─────────────────────────────────────────────────────────────────────────────┘

When route not matched:
  1. Router\Core::parse() returns null
  2. Router\Handle::dispatch() throws 404 exception
  3. Render 404 page (customizable template)

Custom 404 page:
  view/templates/error/404.phtml

┌─────────────────────────────────────────────────────────────────────────────┐
│ Redirect                                                                      │
└─────────────────────────────────────────────────────────────────────────────┘

Redirect within Controller:
  return $this->redirect($this->getUrl('catalog/product/list'));

Redirect with message:
  $this->messageManager->addSuccess(__('Product saved successfully.'));
  return $this->redirect($this->getBackendUrl('catalog/product/index'));

┌─────────────────────────────────────────────────────────────────────────────┐
│ Critical Constraints                                                          │
└─────────────────────────────────────────────────────────────────────────────┘

✗ NEVER create routes.xml (framework doesn't support it)
✗ NEVER manually edit generated/router/routes.php (will be overwritten)
✓ MUST run setup:upgrade --route after adding Controllers
✓ Controller class name MUST match file name
✓ Backend Controllers MUST extend Weline\Admin\Controller\BaseController
✓ Use getUrl()/getBackendUrl() to generate URLs, NO hardcoding

• Framework\App\Controller\*     → Controller base classes
