# 社区模块产品开发执行方案

## 1. 产品定位

社区模块面向 WelineFramework 用户、开发者、站长和插件作者，目标是提供一个可以讨论问题、沉淀技术经验、发布插件、积累贡献、提升用户留存的官方社区。它不是简单论坛，而是“讨论交流 + 技术支持 + 插件贡献 + 成长激励 + W 币消费入口”的综合社区系统。

本文件可拷贝到其它项目作为执行方案。若目标项目不是 WelineFramework，也可以把其中的账号、授权、插件、商城能力映射到目标项目的同类模块。

## 2. 产品目标

- 让框架用户有固定的网站讨论问题、分享经验、反馈建议。
- 通过板块和标签把技术问题、使用交流、AI、新闻、插件发布分流。
- 通过等级、经验、威望、徽章和 W 币提升活跃度和留存。
- 鼓励开发者上传插件、Skill、模板等可下载资源，并用高价值奖励促进生态贡献。
- 丰富个人中心，把账号安全、两步验证、授权应用、贡献记录、资产流水统一呈现。
- 与应用商城、Skill 平台、BBS 等系统通过授权互通账号，而不是重复维护孤立账号。

## 3. 默认板块

第一版默认创建以下板块：

| 板块 | 定位 | 默认权限 |
| --- | --- | --- |
| AI 板块 | AI 使用、AI 插件、提示词、自动化案例 | 登录用户发帖 |
| 框架技术板块 | 安装、模块开发、WLS、主题、部署、错误排查 | 登录用户发帖 |
| 使用交流板块 | 站点建设、商城经验、运营交流、案例展示 | 登录用户发帖 |
| 框架新闻板块 | 版本发布、路线图、官方公告 | 官方和版主发布 |
| 插件与 Skill 发布板块 | 上传插件包、Skill 包、模板包并提供下载 | 开发者认证后发布 |
| 问答板块 | 可设置最佳答案，沉淀 FAQ | 登录用户发帖 |
| 建议与 RFC 板块 | 收集产品建议、投票、路线图讨论 | 登录用户发帖 |

板块需支持后台新增、禁用、排序、设定版主、配置发帖权限和审核策略。

## 4. 用户角色

- 访客：浏览公开帖子、搜索、查看插件详情。
- 注册用户：发帖、回复、点赞、收藏、关注、举报。
- 开发者用户：上传插件、Skill、模板，查看下载数据和审核结果。
- 版主：置顶、加精、锁帖、移动帖子、处理举报。
- 官方运营：发布新闻、配置活动、调整奖励规则。
- 管理员：管理板块、用户、权限、风控、W 币规则和系统配置。

## 5. 核心功能

### 5.1 内容系统

- 主题帖：标题、正文、板块、标签、附件、状态、是否已解决。
- 回复：楼层、引用、@、富文本、代码块、附件。
- 标签：模块名、版本号、错误类型、应用场景。
- 搜索：按关键词、板块、标签、作者、是否已解决、是否含附件过滤。
- 内容状态：普通、置顶、精华、已解决、锁定、隐藏、待审核。
- 举报：垃圾内容、侵权、恶意引导、危险插件、违规广告。

### 5.2 问答与解决机制

- 问题帖可以被作者或版主设置“最佳答案”。
- 最佳答案给回答者发放经验、威望和徽章进度。
- 已解决问题进入 FAQ 候选池。
- 后续可接入 AI，总结问题、推荐相似帖子、生成知识库条目。

### 5.3 插件与 Skill 发布

发布流程：

1. 开发者进入“发布插件/Skill”。
2. 填写名称、简介、版本、兼容框架版本、依赖模块、许可证。
3. 上传插件包、截图、README、变更日志。
4. 系统生成文件校验值和基础风险扫描结果。
5. 提交审核。
6. 审核通过后展示下载页并发放奖励。

下载页展示：

- 插件名称、版本、作者、兼容版本。
- 下载次数、评分、最近更新。
- 文件大小、校验值、许可证。
- 安装说明、变更日志、风险提示。
- 举报和反馈入口。

### 5.4 等级、经验、威望

经验用于等级成长，威望用于表达可信贡献，两者分开计算。

建议规则：

| 行为 | 经验 | 威望 | 说明 |
| --- | --- | --- | --- |
| 每日登录 | 少量 | 无 | 设置日上限 |
| 发帖 | 少量 | 无 | 防止水贴刷分 |
| 回复 | 少量 | 无 | 设置日上限 |
| 回复被点赞 | 少量 | 少量 | 需去重 |
| 答案被采纳 | 中等 | 中等 | 问答核心奖励 |
| 发布插件审核通过 | 大量 | 大量 | 生态贡献核心奖励 |
| 插件下载达到阶梯 | 中等 | 中等 | 设置周期上限 |
| 官方推荐插件 | 大量 | 大量 | 人工审核触发 |
| 举报有效 | 少量 | 少量 | 防滥用 |
| 违规内容被处理 | 扣减 | 扣减 | 支持处罚和申诉 |

等级建议：

- L1 新用户。
- L2 参与者。
- L3 活跃用户。
- L4 技术贡献者。
- L5 资深贡献者。
- L6 社区专家。
- L7 官方认证开发者。

等级权限可以逐步解锁：签名、上传附件大小、发帖免审、插件发布资格、测试版功能申请等。

### 5.5 W 币

W 币是可消费资产，不等同经验和威望。建议第一版只支持站内消费，不支持提现和用户间转账。

来源：

- 插件审核通过奖励。
- 官方活动奖励。
- 优质答案奖励。
- Bug 复现或文档贡献奖励。
- 插件下载里程碑奖励。

用途：

- 应用商城消费抵扣。
- 插件购买或下载抵扣。
- Skill 下载或安装抵扣。
- 活动兑换。

账务规则：

- 必须有 W 币流水表，记录来源、业务单号、数量、状态、过期时间。
- 抵扣需要冻结、确认、取消、退款四种状态。
- 作弊、撤稿、违规插件下架时支持冲正。
- 每个奖励类型必须有日上限或周期上限。

### 5.6 个人中心

个人中心需要从“订单/账号页”升级为“社区身份中心”。

建议导航：

- 概览：等级、经验进度、威望、W 币余额、最近通知。
- 我的帖子：主题帖、回复、收藏、关注。
- 我的贡献：最佳答案、插件、Skill、下载数据、徽章。
- 我的资产：W 币余额、流水、冻结记录、抵扣记录。
- 账号安全：密码、两步验证、恢复码、登录设备。
- 授权应用：论坛、商城、Skill、BBS 等账号互通授权。
- 个人资料：头像、昵称、签名、技术栈、官网、GitHub/Gitee。

两步验证：

- 支持开启、关闭、恢复码、重新绑定。
- 开启插件上传、W 币抵扣、账号授权等高风险操作时，建议提示用户开启。
- 敏感操作可要求临时二次验证。

## 6. 前端与 UI 设计

### 6.1 总体风格

社区是高频使用界面，应采用清晰、克制、信息密度适中的开发者社区风格。避免大面积营销式 Hero、复杂装饰、低效率卡片堆叠。

设计原则：

- 首页第一屏展示真实社区内容，而不是宣传页。
- 主操作明确：发帖、提问、发布插件、搜索。
- 状态清晰：已解决、精华、待审核、已锁定、官方。
- 桌面端优先信息效率，移动端优先阅读和发帖。
- 所有交互元素有 hover、focus、loading、empty、error 状态。

### 6.2 首页

首页模块：

- 顶部搜索框：支持搜索问题、插件、标签。
- 快捷入口：提问、发帖、发布插件。
- 热门板块：AI、框架技术、使用交流、新闻、插件。
- 最新问题：按时间展示，可标记是否已解决。
- 热门插件：展示下载数、评分和兼容版本。
- 官方新闻：展示最近版本、公告、路线图。
- 活跃贡献者：展示头像、等级、贡献类型。

### 6.3 板块页

板块页采用列表视图：

- 左侧或顶部为板块信息、发帖按钮、规则。
- 列表展示标题、标签、作者、回复数、浏览数、最后回复、状态。
- 支持排序：最新、热门、未解决、精华。
- 支持筛选：标签、版本、是否含附件、是否已解决。

### 6.4 帖子详情页

帖子详情页结构：

- 主内容区：标题、标签、作者、正文、附件、操作。
- 回复区：楼层、引用、点赞、采纳、举报。
- 右侧信息区：作者卡、等级、威望、相关问题、同板块热门帖。
- 移动端右侧信息下移，保证正文阅读优先。

### 6.5 插件发布页

采用步骤式表单：

1. 基础信息。
2. 兼容性与依赖。
3. 文件上传。
4. 文档和截图。
5. 奖励与审核说明。
6. 提交确认。

上传区域必须显示：

- 文件名、大小、版本。
- 上传进度。
- 校验值。
- 错误原因。
- 重新上传入口。

### 6.6 后台管理页

后台采用表格、筛选器和详情抽屉：

- 板块管理。
- 内容审核。
- 举报处理。
- 插件审核。
- 积分规则。
- W 币规则。
- 用户处罚和申诉。

## 7. 后端模块建议

### 7.1 社区主模块

建议命名 `Weline_Community`。

职责：

- 板块、主题、回复、标签。
- 内容状态、审核、举报。
- 社区通知。
- 社区首页和前台页面。

### 7.2 成长模块

可命名 `Weline_Reputation`，也可以第一阶段作为社区模块子域。

职责：

- 经验、等级、威望、徽章。
- 积分规则。
- 行为事件转奖励。
- 处罚和回滚。

### 7.3 W 币模块

建议独立为 `Weline_Wallet`，因为应用商城和 Skill 平台也会使用。

职责：

- W 币余额。
- W 币流水。
- 冻结、确认、取消、退款、冲正。
- 抵扣接口。

### 7.4 插件中心模块

可命名 `Weline_PluginHub`，也可以第一阶段作为社区模块子域。

职责：

- 插件包、版本、截图、文档、审核状态。
- 下载记录。
- 评分和反馈。
- 与应用商城或 Skill 平台同步。

### 7.5 账号互通依赖

社区模块不自己实现跨站认证。它依赖统一互通授权模块提供：

- 一键登录。
- 账号自动注册和关联。
- 授权撤销。
- 已授权应用列表。
- 高风险操作两步验证。

## 8. 关键数据模型

### 8.1 社区

- `community_board`：板块。
- `community_topic`：主题帖。
- `community_reply`：回复。
- `community_tag`：标签。
- `community_topic_tag`：主题和标签关系。
- `community_report`：举报。
- `community_moderation_log`：审核日志。
- `community_notification`：通知。

### 8.2 成长

- `reputation_profile`：用户成长档案。
- `reputation_ledger`：经验和威望流水。
- `reputation_rule`：奖励规则。
- `reputation_badge`：徽章。
- `reputation_user_badge`：用户徽章。

### 8.3 W 币

- `wallet_account`：用户钱包。
- `wallet_ledger`：W 币流水。
- `wallet_freeze`：冻结记录。
- `wallet_redeem_order`：抵扣单。

### 8.4 插件

- `plugin_package`：插件主信息。
- `plugin_version`：插件版本。
- `plugin_file`：上传文件。
- `plugin_download_log`：下载日志。
- `plugin_review`：审核记录。
- `plugin_rating`：评分。

## 9. 关键业务事件

社区模块应通过事件或服务接口把行为转换成奖励：

- `community.topic.created`
- `community.reply.created`
- `community.reply.accepted`
- `community.topic.liked`
- `community.plugin.submitted`
- `community.plugin.approved`
- `community.plugin.downloaded`
- `community.report.resolved`
- `community.violation.confirmed`

事件处理必须幂等。每个可奖励行为都要有业务单号，避免重复发放经验、威望或 W 币。

## 10. 风控与审核

第一版必须具备：

- 新用户发帖频率限制。
- 附件上传大小限制。
- 插件上传白名单后缀。
- 插件包基础安全扫描。
- 举报队列。
- 版主处理日志。
- 奖励日上限。
- 可疑下载和刷赞检测。

高风险操作：

- 插件上传。
- W 币抵扣。
- 批量下载。
- 修改邮箱。
- 账号互通绑定。

这些操作建议要求登录态新鲜，必要时触发两步验证。

## 11. 里程碑

### MVP

目标：先让用户能讨论、提问、回复，并形成基础留存。

范围：

- 默认板块。
- 发帖、回复、点赞、收藏。
- 登录用户个人资料。
- 基础经验和等级。
- 后台板块管理和内容审核。
- 首页、板块页、帖子详情页。

验收：

- 用户可注册或登录后发帖。
- 用户可回复并被作者采纳。
- 后台可管理板块和处理举报。
- 发帖和回复产生经验流水。
- 移动端和桌面端核心流程可用。

### V1

目标：建立贡献激励和插件生态入口。

范围：

- 插件与 Skill 发布板块。
- 插件上传、版本、下载页。
- 插件审核。
- 威望和徽章。
- W 币流水。
- 个人中心贡献页和资产页。

验收：

- 开发者可提交插件包。
- 审核通过后生成下载页。
- 插件通过审核后发放经验、威望和 W 币。
- 用户可以查看 W 币流水。
- 管理员可以撤销违规奖励。

### V2

目标：打通应用商城和 Skill 平台消费。

范围：

- W 币商城抵扣。
- Skill 下载抵扣。
- 授权账号互通。
- 统一安全中心。
- 下载数据和作者看板。

验收：

- 社区账号可授权登录商城。
- 商城消费可读取 W 币余额并创建抵扣单。
- 退款或取消后 W 币正确解冻或退回。
- 用户可以撤销商城授权。

### V3

目标：形成知识库和生态增长飞轮。

范围：

- AI 辅助回答。
- 相似问题推荐。
- 已解决问题转 FAQ。
- 插件推荐算法。
- 活动任务系统。

验收：

- 已解决问题可以生成 FAQ 候选。
- 用户提问时能看到相似问题。
- 官方可创建活动任务并发放奖励。

## 12. 验收指标

业务指标：

- 7 日留存。
- 发帖率。
- 回复率。
- 问题解决率。
- 插件上传数。
- 插件审核通过率。
- 插件下载量。
- W 币抵扣转化率。

质量指标：

- 举报处理时长。
- 审核积压数量。
- 奖励重复发放率。
- W 币冲正成功率。
- 授权登录成功率。
- 移动端核心流程完成率。

## 13. 实施优先级

推荐顺序：

1. 先接入统一互通授权模块，保证账号体系可扩展。
2. 开发社区 MVP，让用户可以发帖、回复、解决问题。
3. 加入经验、等级和基础个人中心。
4. 开发插件发布和审核流程。
5. 独立 W 币账本，再接商城抵扣。
6. 最后做 AI、推荐、活动和知识库自动沉淀。

## 14. 关键风险

- 如果没有统一账号互通，论坛、商城、Skill 账号会割裂，后期合并成本很高。
- 如果 W 币没有独立流水和冲正机制，后续接商城会有账务风险。
- 如果插件上传缺少审核和扫描，会给用户站点带来安全风险。
- 如果奖励规则没有日上限，会被刷帖、刷赞、刷下载套利。
- 如果个人中心只展示订单，不展示贡献和安全，会削弱社区留存。

## 15. 最小可执行任务拆分

第一批任务建议拆成：

1. 板块与主题模型。
2. 发帖、回复、帖子详情。
3. 社区首页和板块页 UI。
4. 后台板块管理。
5. 内容审核和举报。
6. 经验等级流水。
7. 个人中心社区概览。
8. 插件发布需求细化。
9. W 币账本需求细化。
10. 账号互通接入方案确认。

每个任务都应包含：目标、涉及页面、涉及接口、数据模型、权限、错误状态、验收命令或验收路径。
