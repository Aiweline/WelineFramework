# Codex Task Journal

这个目录用于保存 Codex 的任务记录，目标是让任务在网络中断、窗口关闭、模型切换后依然可以恢复。

## 规则

1. 每次收到新任务，先创建或更新 `ACTIVE.md`，再创建对应任务文件。
2. 每个任务都要单独落盘到 `tasks/YYYY-MM-DD/` 目录下。
3. 任务进行中要持续追加关键进展，而不是只在结束时补写。
4. 至少记录这些信息：任务目标、当前状态、已完成内容、待办、风险/阻塞、相关文件。
5. 任务结束时补齐结果、验证情况、遗留事项；如果中断，也要写明恢复入口。
6. `plans/` 用于沉淀经过整理的 Codex 计划文档；它不是任务流水账，重点记录结构化方案、阶段划分、关键决策和后续执行依据。

## 目录约定

- `ACTIVE.md`
  当前正在处理的任务摘要，方便下次会话快速恢复。
- `TASK_TEMPLATE.md`
  新任务的记录模板。
- `plans/*.plan.md`
  经过整理后的 Codex 计划文档。
- `tasks/YYYY-MM-DD/*.md`
  按天归档的任务记录，一任务一文件。

## 文件命名建议

推荐格式：

`YYYY-MM-DD-HHMM-short-slug.md`

例如：

`2026-03-21-1510-task-log-policy.md`

slug 尽量使用简短英文，避免路径和编码兼容问题。
