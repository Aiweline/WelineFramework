## 共识技能

所有工程智能体都必须遵守：

- [AI 总则与全局约束](../global-constraints.md)
- [通用工程师-开发规范与代码质量](../skills/通用工程师-开发规范与代码质量/SKILL.md)

所有需求默认按 `产品经理 -> 架构师 -> 高级全栈开发工程师 -> 产品经理验收` 的闭环执行：先拆业务和体验逻辑，再按框架边界拆原子任务并派给合适智能体，开发完成后必须从产品逻辑、审美、异常状态和验证证据复验；不达标则立即进入下一轮拆解、派工、开发和验收。

当前接收用户输入的智能体是需求 Owner 和总验收人。各工程智能体完成任务后只进入“待验收”，必须由需求 Owner 按用户原始输入、Requirement/AC、验证证据和剩余风险逐项复核通过后，才能视为完成。

未完成、未验证、子智能体卡住或证据不足属于需求 Owner 的内部调度状态，不得作为对用户、老板或 `@技术总监` 的交付汇报；Owner 必须继续监工、重拆缺口、改派或追证据，直到完成验收。只有范围、安全、不可逆决策、外部凭据或无法自行裁决的冲突才向人类请求决定。

当发现问题、风险、阻塞、跨角色影响或验证失败时，必须通知 `@Weline-技术主管`，并使用通用技能中的问题上报格式。
