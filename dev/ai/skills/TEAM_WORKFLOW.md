# Team Workflow

本流程按当前团队结构编排，不创建 Technical Director 技能。

## 角色职责

- Technical Director：全局架构把关、风险决策、最终二级验收
- 技术主管：接收任务、拆分任务、分派执行、跟踪进度、执行一级验收
- Specialist Engineers：按技能完成实现、测试、验证、文档和证据输出
- QA测试主管：制定测试策略和门禁，组织质量验收
- QA / Test Specialists：执行单元测试、E2E、HTTP、冒烟、回归
- 文档知识库工程师：维护 README、架构、API、技能索引、变更记录

## 标准执行流

1. Technical Director 下达任务与验收目标
2. 技术主管先读 `AI-ENTRY.md`、图谱、模块文档，完成任务拆分
3. 技术主管按职责派发到框架、业务、前端、运行时、安全、测试、CI、文档角色
4. Specialist Engineers 在各自边界内实现并输出证据
5. QA测试主管定义门禁并组织单测、HTTP、E2E、WLS 验证
6. 技术主管执行一级验收，确认实现与证据一致
7. Technical Director 执行最终二级验收

## 交付要求

- 每个实施角色必须提交可核对的变更与验证证据
- 每个测试角色必须提交可复现的命令、结果和剩余风险
- 每个文档角色必须在相关模块目录中更新 README / API / 架构文档
- WLS 测试必须使用独立实例，且测试后停止实例

## 边界约束

- 技术主管不代替专项工程师实现底层业务代码
- QA 不替代开发角色决定架构方案
- 文档角色不在仓库根目录写详细修复报告
- CI 角色不绕过验证门禁直接放行
