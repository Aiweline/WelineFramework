# WelineFramework 部署文档

## 🚀 一键快速启动

**最简单的方式 - 运行快速启动脚本：**

```bash
# 给脚本执行权限并运行
chmod +x quick-start.sh && ./quick-start.sh
```

脚本会自动检测您的环境并选择最佳部署方式：
- ✅ 自动检测 Docker 环境
- ✅ 自动检测 PHP 8.2+ 环境  
- ✅ 智能选择部署方式
- ✅ 自动显示访问信息

## 依赖环境

- **PHP 8.2+** - 框架运行环境
- **SQLite** - 内置本地数据库，无需额外配置【只是工具性质，如果后期需要大量使用，可以考虑切换数据库即可】
- **Composer** - PHP依赖管理工具
- **Docker** (可选) - 容器化部署，零环境配置

## 快速部署

WelineFramework 提供多种部署方式，选择最适合您的方式：

### 方式一：一键部署脚本 (推荐)

```bash
# 给脚本执行权限
chmod +x deploy.sh

# 运行一键部署脚本
./deploy.sh

# 或者指定部署模式
./deploy.sh --dev    # 开发环境
./deploy.sh --prod   # 生产环境
```

**脚本功能：**
- ✅ 自动检测并安装 PHP 8.2+
- ✅ 自动安装 Composer
- ✅ 智能权限处理
- ✅ 自动设置环境变量
- ✅ 支持多操作系统 (Ubuntu/Debian/CentOS/Arch/macOS)

### 方式二：Docker 部署 (最简单)

```bash
# 使用 Docker Compose 一键部署
docker-compose up -d

# 查看部署状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

**Docker 优势：**
- ✅ 零环境配置，开箱即用
- ✅ 包含 PHP 8.2 + Nginx + SQLite
- ✅ 自动健康检查
- ✅ 数据持久化
- ✅ 支持 Redis/MySQL 扩展

### 方式三：手动部署

```bash
# 1. 安装依赖
php composer.phar install

# 2. 系统升级
php bin/w setup:upgrade

# 3. 安装定时任务
php bin/w cron:install
```

**部署完成！** 🎉

### 方式四：WLS 模式部署（生产可选）

WLS（Weline Server）是框架内置的常驻内存 HTTP 服务器，不依赖 Nginx/PHP-FPM 即可对外提供 Web 服务，也可在 Nginx 前作为后端使用。

**适用场景：**
- 生产环境：用 Nginx 反代到 WLS 端口，或直连 WLS 端口（需放行防火墙）
- 开发/内网：直连 `http://域名或IP:9981` 访问

**快速步骤：**

1. **配置**：在 `app/etc/env.php` 的 `server` 中设置 `host`、`port`（如 `0.0.0.0`、`9981`）。
2. **启动**：`php bin/w server:start`（默认实例）或 `php bin/w server:start -h 0.0.0.0 -p 9981`。
3. **域名接入**：在后台「网站管理」中为对应网站添加域名；外网访问时需 DNS 或 hosts 指向本机，或使用 Nginx 反向代理到 WLS 端口。

**详细说明**（环境配置、多实例、HTTPS、故障排查）见：[Server 模块 - WLS 模式部署指南](app/code/Weline/Server/doc/WLS模式部署指南.md)。

## 访问地址

### 后端管理地址

部署完成后，您可以通过以下地址访问系统：

#### 开发环境
```bash
# 启动开发服务器（默认 WLS 常驻内存模式）
php bin/w server:start

# 访问地址
http://127.0.0.1:9981/{admin_key}/admin/dashboard
```

#### 生产环境（Nginx 或 WLS）
```bash
# Nginx 反代时（用户访问 80/443）
http://your-domain.com/{admin_key}/admin/dashboard

# 直连 WLS 时（需放行端口，如 9981）
http://your-domain.com:9981/{admin_key}/admin/dashboard
```

### 获取管理员密钥

管理员密钥配置在 `app/etc/env.php` 文件中：

```php
// app/etc/env.php
'admin' => 'Be7KAJpVt3bwMMejUZa7VbevRM5GVxjQ',  // 管理员密钥
'api_admin' => '7r1XLapP8oNBJc6grWtUlUqA42e6GZWQ',  // API密钥
```

您可以通过以下方式查看：

```bash
# 方法1：直接查看配置文件
cat app/etc/env.php | grep "'admin'"

# 方法2：使用PHP命令查看
php -r "echo 'Admin Key: ' . include('app/etc/env.php')['admin'] . PHP_EOL;"
```

### 默认访问路径

- **后台首页**: `/{admin_key}/admin/dashboard`
- **系统设置**: `/{admin_key}/admin/system`
- **模块管理**: `/{admin_key}/admin/module`
- **用户管理**: `/{admin_key}/admin/user`

### API 访问地址

#### REST API
```bash
# 开发环境
http://127.0.0.1:9981/{api_key}/rest/v1/

# 生产环境
http://your-domain.com/{api_key}/rest/v1/
```

### 访问示例

使用实际的密钥值（来自 `app/etc/env.php`）：

```bash
# 后台访问
http://127.0.0.1:9981/Be7KAJpVt3bwMMejUZa7VbevRM5GVxjQ/admin/dashboard

# API访问
http://127.0.0.1:9981/7r1XLapP8oNBJc6grWtUlUqA42e6GZWQ/rest/v1/user/list
```

**注意**：请使用您自己 `app/etc/env.php` 文件中的实际密钥值。

## Docker 部署详解

### Docker 环境要求

- **Docker** 20.10+
- **Docker Compose** 2.0+

### Docker 部署步骤

#### 1. 基础部署

```bash
# 克隆项目
git clone <repository-url>
cd WelineFramework

# 一键启动
docker-compose up -d

# 查看状态
docker-compose ps
```

#### 2. 获取访问密钥

```bash
# 获取管理员密钥
docker exec -it welineframework php bin/w admin:key:show

# 获取API密钥
docker exec -it welineframework php bin/w api:key:show
```

#### 3. 访问系统

```bash
# 后台管理
http://localhost/admin/{admin_key}/dashboard

# API接口
http://localhost/api/{api_key}/rest/v1/
```

#### 4. 常用 Docker 命令

```bash
# 查看日志
docker-compose logs -f

# 重启服务
docker-compose restart

# 停止服务
docker-compose down

# 重新构建
docker-compose up -d --build

# 进入容器
docker exec -it welineframework bash

# 执行框架命令
docker exec -it welineframework php bin/w cache:clear
```

### 4. 配置 Nginx (非Docker环境)

使用项目根目录下的 `nginx.conf` 配置文件，根据您的服务器环境调整配置：

```bash
# 复制配置文件到 Nginx 配置目录
cp nginx.conf /etc/nginx/sites-available/welineframework
ln -s /etc/nginx/sites-available/welineframework /etc/nginx/sites-enabled/

# 重启 Nginx
sudo systemctl restart nginx
```

### 5. 设置文件权限

```bash
# 设置必要的文件权限
chmod -R 755 pub/
chmod -R 755 var/
chmod -R 755 generated/
```

## 数据库配置

WelineFramework 默认使用 **SQLite 本地数据库**，无需额外配置：

```php
'db' => [
    'default' => 'sqlite',
    'master' => [
        'type' => 'sqlite',
        'path' => 'app/etc/db.sqlite',  // 数据库文件路径
    ],
],
```

**SQLite 优势：**
- ✅ 无需安装数据库服务器
- ✅ 零配置，开箱即用
- ✅ 文件型数据库，便于备份
- ✅ 性能优秀，适合中小型应用

**可选：切换到 MySQL**
如需使用 MySQL，修改 `app/etc/env.php` 配置即可。

## 常用命令

```bash
# 查看系统状态
php bin/w deploy:mode:show

# 清除缓存
php bin/w cache:clear

# 模块管理
php bin/w module:listing
php bin/w module:enable ModuleName
php bin/w module:disable ModuleName

# 服务器管理（开发环境 / WLS 模式）
php bin/w server:start          # 启动 WLS（默认常驻内存）
php bin/w server:start --cli    # 使用 PHP 内置 CLI 服务器（开发）
php bin/w server:status        # 查看实例与端口状态
php bin/w server:stop           # 停止 WLS
```

## 注意事项

1. **生产环境**：请使用 `deploy:mode:set prod` 设置生产模式
2. **文件权限**：确保 `pub/`、`var/`、`generated/` 目录有写权限
3. **Nginx 配置**：根据实际路径调整 `nginx.conf` 中的 `root` 路径；若使用 WLS，可配置 Nginx 反代到 WLS 端口（见 [WLS 模式部署指南](app/code/Weline/Server/doc/WLS模式部署指南.md)）
4. **数据库**：默认使用 SQLite，无需额外配置；生产环境可切换到 MySQL
5. **WLS 生产部署**：域名需在后台「网站管理」中配置；直连 WLS 时放行对应端口，或使用 Nginx 做 80/443 反代

## 故障排除

### Docker 相关问题

#### Docker 守护进程未运行
```bash
# 使用自动启动脚本
chmod +x docker/docker-start.sh && ./docker/docker-start.sh

# 或手动启动 Docker Desktop (macOS)
open -a Docker

# 或手动启动 Docker 服务 (Linux)
sudo systemctl start docker
```

#### Docker Compose 错误
```bash
# 检查 Docker 状态
docker info

# 重新构建容器
docker-compose down
docker-compose up -d --build

# 查看详细日志
docker-compose logs -f
```

#### 端口冲突
```bash
# 检查端口占用
lsof -i :80
lsof -i :9981

# 修改 docker-compose.yml 中的端口映射
# 例如: "8080:80" 改为 "8081:80"
```

### 权限问题
```bash
chmod -R 755 pub/ var/ generated/
```

### 缓存问题
```bash
php bin/w cache:clear -f
```

### 模块问题
```bash
php bin/w setup:upgrade
```
