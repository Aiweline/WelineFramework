# 事件调试功能使用指南

## 功能概述

WelineFramework 的事件系统现已支持 Observer 调用过程的调试功能，可以帮助开发者了解事件执行情况。

## 功能特性

### 1. CLI 开发模式（自动启用）

当满足以下条件时，自动在控制台输出 Observer 调用信息：
- 运行环境：CLI 模式（命令行）
- 开发模式：`DEV` 常量为 `true`

**输出格式示例：**

```
事件名: module_upgrade_after
--------------------------------------------------------------------------------
类名                                              开始时间          结束时间          耗时
--------------------------------------------------------------------------------
ModuleUpgradeObserver                            10:30:15.123     10:30:15.456     333ms
SetupUpgradeObserver                             10:30:15.460     10:30:15.789     329ms
```

### 2. Web 模式调试（需要参数）

在 Web 模式下，通过添加 URL 参数 `event=debug` 来启用调试：

```
http://your-site.com/some/page?event=debug
```

或通过 POST 参数：

```php
$_POST['event'] = 'debug';
```

### 3. 日志文件记录

以下情况会将事件调试信息写入日志文件 `var/log/event.log`：

- 开发模式启用（`DEV = true`）- 无论 CLI 还是 Web
- Web 模式下传入 `event=debug` 参数

**日志格式示例：**

```
================================================================================
2025-10-15 10:30:15 - 事件名: module_upgrade_after
--------------------------------------------------------------------------------
类名                                              开始时间          结束时间          耗时
--------------------------------------------------------------------------------
ModuleUpgradeObserver                            10:30:15.123     10:30:15.456     333.000ms
SetupUpgradeObserver                             10:30:15.460     10:30:15.789     329.000ms
```

## 使用场景

### 场景 1：CLI 命令开发调试

在开发 CLI 命令时，自动看到事件触发情况：

```bash
php bin/w setup:upgrade
```

如果开发模式开启，所有事件的 Observer 执行情况都会在控制台显示。

### 场景 2：Web 页面性能分析

分析某个页面的事件执行性能：

```
http://localhost/backend/dashboard?event=debug
```

然后查看 `var/log/event.log` 文件，分析各个 Observer 的执行时间。

### 场景 3：事件流程追踪

了解特定操作触发了哪些事件和观察者：

1. 开启开发模式
2. 执行目标操作
3. 查看日志文件，追踪完整的事件调用链

## 输出信息说明

### 字段含义

| 字段 | 说明 |
|------|------|
| 类名 | Observer 类的简短名称（不含命名空间） |
| 开始时间 | Observer 开始执行的时间（精确到毫秒） |
| 结束时间 | Observer 执行完成的时间（精确到毫秒） |
| 耗时 | Observer 执行耗时（单位：毫秒） |

### 错误标记

如果 Observer 执行过程中抛出异常，会在输出中标记：

```
ObserverName                                     10:30:15.123     10:30:15.456     333ms [ERROR: 错误信息]
```

## 性能影响

- **CLI 开发模式**：有轻微性能影响，仅用于开发调试
- **Web 模式**：
  - 无参数：无性能影响
  - 带 `event=debug` 参数：有文件 I/O 开销，仅用于调试
- **生产环境**：关闭 `DEV` 模式且不传 debug 参数时，无任何性能影响

## 注意事项

1. 生产环境请务必关闭 `DEV` 模式
2. Web 模式的 `event=debug` 参数应仅用于开发和调试
3. 日志文件会持续追加，建议定期清理 `var/log/event.log`
4. 如果事件没有注册任何 Observer，不会输出任何调试信息

## 文件位置

- **事件类**：`app/code/Weline/Framework/Event/Event.php`
- **事件管理器**：`app/code/Weline/Framework/Event/EventsManager.php`
- **日志文件**：`var/log/event.log`

## 示例代码

### 手动触发事件并查看调试信息

```php
<?php
define('DEV', true); // 开启开发模式

require __DIR__ . '/app/bootstrap.php';

use Weline\Framework\Manager\ObjectManager;
use Weline\Framework\Event\EventsManager;

$eventManager = ObjectManager::getInstance(EventsManager::class);

// 触发事件
$eventManager->dispatch('your_event_name', [
    'data' => 'some value'
]);

// 查看 var/log/event.log 文件
```

## 技术细节

### 判断逻辑

```php
// CLI 开发模式
$isCliDevMode = (PHP_SAPI === 'cli' && defined('DEV') && DEV);

// Web 调试模式
$isWebDebugMode = (PHP_SAPI !== 'cli' && (
    (isset($_GET['event']) && $_GET['event'] === 'debug') ||
    (isset($_POST['event']) && $_POST['event'] === 'debug') ||
    (defined('DEV') && DEV)
));
```

### 时间精度

使用 `microtime(true)` 获取微秒级精度，显示时保留 3 位小数（毫秒）。
