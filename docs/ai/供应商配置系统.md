# AI供应商配置系统

## 概述

新的AI供应商配置系统使用JSON配置文件来管理供应商信息，替代了之前固化的`SUPPORTED_PROVIDERS`常量。这样可以更灵活地添加和管理新的AI供应商。

## 配置文件位置

供应商配置文件位于：`app/code/Weline/Ai/etc/vendors/`

**路径解析方式**：系统使用相对路径来访问配置文件，相对于`VendorConfigManager.php`文件的位置：
- `VendorConfigManager.php` 位于：`app/code/Weline/Ai/Service/Provider/`
- 通过`dirname(__FILE__)`获取当前文件目录
- 向上两级目录到达模块根目录：`app/code/Weline/Ai/`
- 然后进入`etc/vendors`目录读取配置文件

这种方式确保了无论项目部署在哪个位置，都能正确找到配置文件。

## 配置文件格式

每个供应商对应一个JSON配置文件，文件名格式为：`{供应商代码}.json`

### 配置文件结构

```json
{
    "name": "供应商显示名称",
    "code": "供应商代码",
    "base_url": "API基础URL",
    "models_prefix": ["模型前缀数组"],
    "test_model": "测试用模型名称",
    "api_key_field": "API密钥字段名",
    "base_url_field": "基础URL字段名", 
    "model_field": "模型字段名",
    "description": "供应商描述",
    "website": "官方网站",
    "documentation": "API文档地址",
    "supported_features": {
        "chat": true,
        "completion": true,
        "embedding": false,
        "image": false,
        "audio": false
    },
    "rate_limits": {
        "requests_per_minute": 60,
        "tokens_per_minute": 100000
    }
}
```

### 必需字段

- `name`: 供应商显示名称
- `code`: 供应商代码（必须与文件名一致）
- `base_url`: API基础URL
- `test_model`: 用于连接测试的模型名称
- `api_key_field`: API密钥在配置中的字段名
- `model_field`: 模型名称在API请求中的字段名

### 可选字段

- `models_prefix`: 模型名称前缀数组，用于自动识别供应商
- `base_url_field`: 基础URL字段名
- `description`: 供应商描述
- `website`: 官方网站
- `documentation`: API文档地址
- `supported_features`: 支持的功能特性
- `rate_limits`: 速率限制信息

## 添加新供应商

1. 在`app/code/Weline/Ai/etc/vendors/`目录下创建新的JSON配置文件
2. 按照上述格式填写供应商信息
3. 确保`code`字段与文件名一致
4. 系统会自动加载新配置，无需重启

## 供应商去重

系统会自动检测重复的供应商配置，如果发现重复的供应商代码，会记录警告日志并跳过重复配置。

## 配置验证

系统启动时会自动验证所有供应商配置的完整性，包括：
- 必需字段是否存在
- `code`字段是否与文件名一致
- JSON格式是否正确

## API变更

### 字段名称变更

- `model_id` → `model`: 在API请求中使用`model`字段而不是`model_id`
- 更符合OpenAI API标准

### 动态配置加载

- 移除了固化的`SUPPORTED_PROVIDERS`常量
- 使用`VendorConfigManager`动态加载配置
- 支持配置缓存和重新加载

## 使用示例

```php
use Weline\Ai\Service\Provider\VendorConfigManager;

// 获取所有支持的供应商
$providers = VendorConfigManager::getSupportedProviders();

// 获取指定供应商配置
$deepseekConfig = VendorConfigManager::getProviderConfig('deepseek');

// 根据模型代码推断供应商
$provider = VendorConfigManager::getProviderByModelCode('gpt-3.5-turbo');

// 检查供应商是否支持
$isSupported = VendorConfigManager::isProviderSupported('openai');
```

## 日志记录

供应商配置相关的日志会记录到：`var/log/ai_vendor_config.log`

包括：
- 配置加载成功/失败
- 重复配置警告
- 配置验证错误
- 文件读取错误
