# 测试快速参考

## 🚀 一键启动

### 前端单元测试（Vitest）

```bash
cd tests/unit
npm install
npm start  # Watch 模式，文件变更自动测试
```

### E2E 测试（Playwright）

```bash
cd tests/e2e
npm install
npm start  # 一条命令完成收集和运行
```

### PHP 单元测试（PHPUnit）

```bash
php bin/w p:r YourModule  # 运行特定模块的测试
```

## 📝 测试文件位置

### 前端单元测试

```
app/code/Weline/Theme/view/theme/frontend/assets/js/
├── theme.js              # 源代码
└── theme.test.js         # 测试用例（可选）

# 或统一放在
tests/unit/specs/
└── Weline/Theme/theme.test.js
```

### E2E 测试

```
app/code/Weline/Theme/test/e2e/frontend/
└── theme-override.spec.js
```

### PHP 单元测试

```
app/code/Weline/Theme/test/
└── Unit/Helper/ThemePathResolverTest.php
```

## 🔧 常用命令

### Vitest

```bash
npm start          # Watch 模式
npm run test:ui    # UI 模式
npm run test:run   # 运行一次
npm run test:coverage  # 覆盖率报告
```

### Playwright

```bash
npm start                    # 一键运行
npm start -- --module=Weline_Theme  # 运行特定模块
npm run test:ui             # UI 模式
npm run test:headed         # 有头模式
```

### PHPUnit

```bash
php bin/w p:r                    # 运行所有测试
php bin/w p:r YourModule         # 运行特定模块
php bin/w p:r Module1,Module2    # 运行多个模块
```

## 🎯 测试最佳实践

1. **测试文件命名**: `*.test.js` 或 `*.spec.js`
2. **测试组织**: 按模块组织测试用例
3. **Watch 模式**: 开发时使用 watch 模式
4. **覆盖率**: 关键功能应该有测试覆盖
