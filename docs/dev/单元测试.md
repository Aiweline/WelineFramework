# WelineFramework 单元测试快速配置指南

## 概述

本指南帮助开发者快速配置和运行 WelineFramework 的单元测试。框架使用 PHPUnit 作为测试框架，提供自动化的测试配置和运行机制。

## 快速开始

### 3步开始测试

```bash
# 1. 创建测试目录
mkdir -p app/code/Weline/YourModule/test/

# 2. 编写测试文件
# 创建 app/code/Weline/YourModule/test/YourModuleTest.php

# 3. 运行测试
php bin/w p:r YourModule
```

### 测试文件模板

```php
<?php
declare(strict_types=1);

namespace Weline\YourModule\test;

use Weline\Framework\UnitTest\TestCore;

class YourModuleTest extends TestCore
{
    public function testYourFunction()
    {
        // 测试逻辑
        $this->assertTrue(true);
    }
}
```

## 为什么写测试？

### 核心价值
- **质量保证**: 早期发现bug，确保代码质量
- **开发效率**: 快速反馈，安全重构
- **团队协作**: 接口契约，知识传递
- **业务价值**: 降低维护成本，提高交付速度

### WelineFramework 测试优势
- **自动化配置**: 框架自动处理所有测试配置
- **零配置**: 开发者只需编写测试文件
- **智能识别**: 自动扫描和识别测试文件
- **统一管理**: 集中化的测试运行和报告

## 测试命令使用

### 基本命令

```bash
# 运行所有测试
php bin/w p:r

# 运行特定模块测试
php bin/w p:r Weline_YourModule

# 运行特定测试文件
php bin/w p:r app/code/Weline/YourModule/test/YourModuleTest.php
```

### 支持的选项

```bash
# 运行特定测试套件
php bin/w p:r YourModule

# 运行多个测试套件
php bin/w p:r YourModule,AnotherModule

# 运行所有测试（默认unit套件）
php bin/w p:r
```

**重要说明**: WelineFramework的 `p:r` 命令**不支持**PHPUnit的原生选项（如 `--verbose`、`--coverage-html` 等）。框架会自动生成多种格式的测试报告。

### 自动生成的测试报告

框架会自动在 `dev/phpunit/report/` 目录下生成以下报告：

```bash
# 查看测试报告目录
ls -la dev/phpunit/report/

# 生成的报告文件：
# - index.html          # TestDox HTML报告
# - junit.xml          # JUnit XML报告  
# - testdox.txt        # TestDox文本报告
# - testdox.xml        # TestDox XML报告
# - teamcity.txt       # TeamCity报告
# - logfile.txt        # 文本日志
```

**查看报告**: 测试完成后，框架会自动启动本地服务器 `http://localhost:8080` 来查看HTML报告。

### 开发调试

```bash
# 运行特定模块测试
php bin/w p:r YourModule

# 运行多个模块测试
php bin/w p:r YourModule,Framework

# 查看测试报告
open dev/phpunit/report/index.html
```

**高级调试**: 如需更多调试选项，可直接使用PHPUnit命令：
```bash
# 直接使用PHPUnit进行高级调试
vendor/phpunit/phpunit/phpunit --configuration dev/phpunit/config.xml --verbose --stop-on-failure
```

## 测试配置机制

### 自动化配置

WelineFramework 的测试系统采用**完全自动化**的配置机制：

```bash
# 开发者只需要：
# 1. 创建测试目录
mkdir -p app/code/Weline/YourModule/test/

# 2. 编写测试文件
# 框架自动处理所有配置
```

### 测试文件识别规则

框架自动识别测试文件的规则：

```bash
# 目录位置：app/code/Weline/*/test/
# 文件命名：*Test.php 或 Test*.php  
# 类继承：extends TestCore
# 方法命名：test* 开头

# 示例：
app/code/Weline/User/test/UserTest.php     ✅ 会被识别
app/code/Weline/User/test/TestUser.php     ✅ 会被识别
app/code/Weline/User/test/UserService.php  ❌ 不会被识别
```

### 配置自动生成

```bash
# 执行流程：
php bin/w p:r -> 扫描模块 -> 生成配置 -> 执行测试

# 框架自动：
# - 扫描 app/code/Weline/*/test/ 目录
# - 生成 dev/phpunit/config.xml 配置
# - 为每个模块创建测试套件
# - 执行 PHPUnit 测试
```

## 开发者配置指南

### 目录结构

```bash
# 标准测试目录结构
app/code/Weline/YourModule/
├── test/                    # 测试根目录
│   ├── YourModuleTest.php   # 主测试文件
│   ├── ModelTest.php        # 模型测试
│   ├── ServiceTest.php      # 服务测试
│   └── ApiTest.php          # API测试
```

### 文件命名规范

```bash
# 测试文件命名规则
*Test.php              # 必须以Test.php结尾
Test*.php              # 也可以Test开头

# 示例
UserTest.php           ✅ 正确
UserServiceTest.php    ✅ 正确
User.php               ❌ 错误
```

### 测试类规范

```php
<?php
declare(strict_types=1);

namespace Weline\YourModule\test;

use Weline\Framework\UnitTest\TestCore;

class YourModuleTest extends TestCore
{
    public function testYourFunction()
    {
        // 测试逻辑
        $this->assertTrue(true);
    }
}
```

**关键要点**：
- 📁 目录：`app/code/Weline/YourModule/test/`
- 📝 文件：`*Test.php` 命名
- 🏗️ 继承：`extends TestCore`
- 🔧 方法：`test*` 开头

## 实用测试示例

### 模型测试示例

```php
<?php
// app/code/Weline/User/test/UserTest.php
declare(strict_types=1);

namespace Weline\User\test;

use Weline\Framework\UnitTest\TestCore;
use Weline\Framework\Manager\ObjectManager;
use Weline\User\Model\User;

class UserTest extends TestCore
{
    private User $userModel;
    
    public function setUp(): void
    {
        parent::setUp();
        $this->userModel = ObjectManager::getInstance(User::class);
    }
    
    public function testUserCreation()
    {
        $this->userModel->setName('测试用户');
        $this->userModel->setEmail('test@example.com');
        
        $result = $this->userModel->save();
        $this->assertTrue($result);
        $this->assertNotNull($this->userModel->getId());
    }
    
    public function testUserLoad()
    {
        $this->userModel->load(1);
        $this->assertNotNull($this->userModel->getName());
    }
}
```

### API测试示例

```php
<?php
// app/code/Weline/User/test/UserApiTest.php
declare(strict_types=1);

namespace Weline\User\test;

use Weline\Framework\UnitTest\TestCore;

class UserApiTest extends TestCore
{
    public function testUserApiEndpoint()
    {
        $this->initRequest('/api/user/list');
        
        $response = $this->makeRequest('GET', '/api/user/list');
        $this->assertEquals(200, $response->getStatusCode());
        
        $data = json_decode($response->getBody(), true);
        $this->assertArrayHasKey('data', $data);
    }
}
```

## 检查清单

### 开发者需要检查的内容

```bash

ls -la app/code/Weline/YourModule/test/

# 2. 检查测试文件是否符合命名规范
find app/code/Weline/YourModule/test/ -name "*Test.php"

# 3. 检查测试类是否继承TestCore
grep -r "extends TestCore" app/code/Weline/YourModule/test/

# 4. 检查测试方法是否以test开头
grep -r "public function test" app/code/Weline/YourModule/test/

# 5. 运行测试验证
php bin/w p:r YourModule --verbose
```
