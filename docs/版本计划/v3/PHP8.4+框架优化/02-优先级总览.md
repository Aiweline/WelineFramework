# 02 - 优先级总览（按性能影响排序）

[← 索引](README.md)

实施时从高星到低星执行，优先落地对性能有极大影响的项。

---

## 星级说明

| 星级 | 含义 |
|------|------|
| ⭐⭐⭐⭐⭐ | **极大突破**：启动时间/内存/响应时间显著改善，必须优先落地 |
| ⭐⭐⭐⭐ | **重大提升**：热点路径明显加速，对整体性能贡献大 |
| ⭐⭐⭐ | **明显优化**：可感知的性能与可维护性提升 |
| ⭐⭐ | **以代码质量为主**：与 getter/setter 性能等价，重在可读性与可维护性 |

## PHP 8.4+ 特性优先级（按性能从高到低）

| 序号 | 特性 | 优先级 | 性能影响简述 |
|------|------|--------|--------------|
| 1 | Lazy Objects（延迟对象） | ⭐⭐⭐⭐⭐ | 启动时间 -40%、内存 -30%，按需初始化重型服务 |
| 2 | 新数组函数 | ⭐⭐⭐⭐ | 路由/ACL 等热点路径短路求值，匹配可快数十倍 |
| 3 | 属性注解扩展（路由/缓存等） | ⭐⭐⭐ | 路由树+增量更新，匹配 O(n)→O(log n)，收集与存储更高效 |
| 4 | Asymmetric Visibility | ⭐⭐ | 无直接性能收益，提升封装与类型安全 |
| 5 | Property Hooks | ⭐⭐ | 与 getter/setter 性能等价，主要减少样板代码 |

## 框架核心模块优先级（按性能从高到低）

| 序号 | 模块 | 优先级 | 性能影响简述 |
|------|------|--------|--------------|
| 1 | DI 容器优化（Lazy Objects） | ⭐⭐⭐⭐⭐ | 同上，服务延迟加载 |
| 2 | 路由系统优化（树结构+增量） | ⭐⭐⭐⭐ | 匹配加速、构建时间缩短 |
| 3 | 模板引擎优化（Block 延迟） | ⭐⭐⭐⭐ | 按需渲染 Block，减少无用初始化 |
| 4 | Model 层重构（Property Hooks） | ⭐⭐ | 以代码简化为主，无显著性能提升 |

> **说明**：**实施时请按上表序号 1→4 的优先级执行**，先做 Lazy Objects 与 DI，再做新数组函数与路由/模板，最后做 Property Hooks 与 Model 重构。
