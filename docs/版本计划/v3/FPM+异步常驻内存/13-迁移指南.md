## 0. WeAsync 迁移概述

将现有模块从 FPM 迁移到 **WeAsync**（照搬 Workerman）模式，需要关注以下差异：

| 差异 | FPM | WeAsync |
|------|-----|---------|
| 进程生命周期 | 请求后销毁 | 常驻内存 |
| 静态变量 | 自动清理 | 跨请求保留 |
| 数据库连接 | 每请求新建 | 连接池复用 |
| Session | 原生 PHP Session | Redis/Database |

**好消息**：大多数代码无需修改，框架会自动处理。只有使用了静态变量缓存数据的代码需要调整。

---

## 1. 概述

本指南帮助将现有模块迁移到支持 WeAsync 常驻内存模式。

## 2. 快速检查清单

### 2.1 必须检查

- [ ] 静态变量是否正确重置（或标记 `#[Resettable]`）
- [ ] 是否直接使用超全局变量（建议使用 Request 对象）
- [ ] 是否有单例模式需要处理（标记 `#[InstanceScope(Scope::REQUEST)]`）
- [ ] 数据库连接是否正确管理（使用连接池）
- [ ] 文件句柄是否正确关闭

### 2.2 建议检查

- [ ] 是否使用了全局函数
- [ ] 闭包是否捕获了外部变量
- [ ] 是否有内存密集型操作（可能导致内存泄漏）

## 3. 常见迁移场景

### 3.1 静态变量

```php
// ❌ 旧代码：静态变量跨请求保留
class Cache
{
    private static array $data = [];
    
    public static function get(string $key): mixed
    {
        return self::$data[$key] ?? null;
    }
    
    public static function set(string $key, mixed $value): void
    {
        self::$data[$key] = $value;
    }
}

// ✅ 新代码：标记为可重置
use Weline\Framework\Runtime\Attribute\Resettable;
use Weline\Framework\Runtime\ResettableInterface;

class Cache implements ResettableInterface
{
    #[Resettable(initialValue: [])]
    private static array $data = [];
    
    public static function get(string $key): mixed
    {
        return self::$data[$key] ?? null;
    }
    
    public static function set(string $key, mixed $value): void
    {
        self::$data[$key] = $value;
    }
    
    public function reset(): void
    {
        self::$data = [];
    }
    
    public function getStateInfo(): array
    {
        return ['cache_size' => count(self::$data)];
    }
}
```

### 3.2 超全局变量

```php
// ❌ 旧代码：直接使用 $_GET
class Controller
{
    public function index(): string
    {
        $page = $_GET['page'] ?? 1;
        $keyword = $_GET['keyword'] ?? '';
        return $this->fetch('index', compact('page', 'keyword'));
    }
}

// ✅ 新代码：使用 Request 对象
use Weline\Framework\Http\Request;

class Controller
{
    public function __construct(
        private readonly Request $request,
    ) {}
    
    public function index(): string
    {
        $page = $this->request->getGet('page', 1);
        $keyword = $this->request->getGet('keyword', '');
        return $this->fetch('index', compact('page', 'keyword'));
    }
}
```

### 3.3 单例模式

```php
// ❌ 旧代码：传统单例
class Config
{
    private static ?self $instance = null;
    private array $data = [];
    
    public static function getInstance(): self
    {
        return self::$instance ??= new self();
    }
}

// ✅ 新代码：使用 DI 容器 + 可重置
use Weline\Framework\Manager\Attribute\InstanceScope;
use Weline\Framework\Manager\Scope;
use Weline\Framework\Runtime\ResettableInterface;

#[InstanceScope(Scope::SINGLETON)]
class Config implements ResettableInterface
{
    private array $data = [];
    private bool $loaded = false;
    
    // 不再需要 getInstance，由 ObjectManager 管理
    
    public function reset(): void
    {
        $this->data = [];
        $this->loaded = false;
    }
    
    public function getStateInfo(): array
    {
        return ['loaded' => $this->loaded];
    }
}
```

### 3.4 Session 使用

```php
// ❌ 旧代码：直接使用 $_SESSION
class AuthController
{
    public function login(): void
    {
        session_start();
        $_SESSION['user_id'] = $userId;
    }
    
    public function getCurrentUser(): ?int
    {
        return $_SESSION['user_id'] ?? null;
    }
}

// ✅ 新代码：使用 SessionInterface
use Weline\Framework\Session\SessionInterface;

class AuthController
{
    public function __construct(
        private readonly SessionInterface $session,
    ) {}
    
    public function login(): void
    {
        $this->session->set('user_id', $userId);
        $this->session->save();
    }
    
    public function getCurrentUser(): ?int
    {
        return $this->session->get('user_id');
    }
}
```

### 3.5 数据库连接

```php
// ❌ 旧代码：每次创建连接
class UserRepository
{
    public function find(int $id): ?array
    {
        $pdo = new PDO($dsn, $user, $pass);
        $stmt = $pdo->prepare('SELECT * FROM users WHERE id = ?');
        $stmt->execute([$id]);
        return $stmt->fetch();
    }
}

// ✅ 新代码：使用连接池
use Weline\Framework\Database\Pool\ConnectionPoolManager;

class UserRepository
{
    public function find(int $id): ?array
    {
        $pool = ConnectionPoolManager::getDefaultPool();
        $pdo = $pool->acquire();
        
        try {
            $stmt = $pdo->prepare('SELECT * FROM users WHERE id = ?');
            $stmt->execute([$id]);
            return $stmt->fetch();
        } finally {
            $pool->release($pdo);
        }
    }
}

// ✅✅ 更好：使用框架 Model
use Weline\Framework\Database\AbstractModel;

class User extends AbstractModel
{
    // 框架自动管理连接
}
```

### 3.6 闭包捕获

```php
// ❌ 旧代码：闭包捕获可能泄漏
class Service
{
    private array $largeData = [];
    
    public function process(): void
    {
        // 闭包捕获了 $this，包括 $largeData
        Event::listen('after', function() {
            $this->cleanup();
        });
    }
}

// ✅ 新代码：避免捕获 $this
class Service
{
    private array $largeData = [];
    
    public function process(): void
    {
        $cleanup = $this->cleanup(...);
        
        Event::listen('after', function() use ($cleanup) {
            $cleanup();
        });
    }
}

// ✅✅ 更好：使用观察者类
#[EventListener('after')]
class CleanupObserver
{
    public function execute(Event $event): void
    {
        // 独立的观察者类，不捕获外部状态
    }
}
```

## 5. 测试迁移后的代码

### 5.1 单元测试

```php
<?php
use PHPUnit\Framework\TestCase;
use Weline\Framework\Runtime\StateManager;

class ServiceTest extends TestCase
{
    private StateManager $stateManager;
    
    protected function setUp(): void
    {
        $this->stateManager = new StateManager();
    }
    
    protected function tearDown(): void
    {
        // 每个测试后重置状态
        $this->stateManager->resetAll();
    }
    
    public function testServiceDoesNotLeakState(): void
    {
        $service = new MyService();
        $service->process();
        
        // 重置状态
        $this->stateManager->resetRequestState();
        
        // 验证状态已清理
        $this->assertEmpty(MyService::getState());
    }
}
```

### 5.2 集成测试

```php
<?php
use Weline\Framework\Runtime\RuntimeFactory;
use Weline\Framework\Runtime\RuntimeType;

class PersistentModeTest extends TestCase
{
    public function testMultipleRequestsAreIsolated(): void
    {
        $runtime = RuntimeFactory::create(RuntimeType::REVOLT);
        $runtime->bootstrap();
        
        // 第一个请求
        $request1 = $this->createRequest(['user_id' => 1]);
        $response1 = $runtime->handle($request1);
        $runtime->reset();
        
        // 第二个请求
        $request2 = $this->createRequest(['user_id' => 2]);
        $response2 = $runtime->handle($request2);
        $runtime->reset();
        
        // 验证请求间隔离
        $this->assertNotEquals(
            $response1->getBody(),
            $response2->getBody()
        );
    }
}
```

## 6. 常见问题

### Q: 迁移后性能反而下降了？

检查是否有：
1. 过度的状态检查
2. 不必要的对象创建
3. 连接池配置不当

### Q: 某些模块必须使用静态变量怎么办？

使用 `#[Resettable]` 注解标记，框架会自动在请求间重置。

### Q: 如何确认迁移完成？

1. 运行 `php bin/w runtime:check` 无警告
2. 通过集成测试
3. 在常驻模式下进行压力测试
