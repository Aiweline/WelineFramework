# Weline框架调试函数优化说明

## 优化概述

为了解决 `debug_backtrace` 打印太多导致浏览器卡顿的问题，我们对调试函数进行了以下优化：

## 主要优化内容

### 1. 减少默认追踪层数
- `p()` 函数：从 1 层改为 2 层
- `pp()` 函数：从 2 层改为 2 层
- `d()` 函数：从 2 层改为 2 层
- `dd()` 函数：固定为 2 层
- `w()` 函数：固定为 2 层

### 2. 添加追踪层数限制
所有调试函数都添加了最大追踪层数限制：
```php
$trace_deep = min($trace_deep, 3); // 最大限制3层
```

### 3. 简化追踪信息输出
不再输出完整的 `debug_backtrace` 数组，只显示关键信息：
- 文件路径和行号
- 类名和方法名（如果有）
- 函数名（如果是普通函数）

### 4. 优化 Debug 类
- `Debug::env()` 和 `Debug::target()` 方法中的追踪层数从 3 层减少到 2 层

## 新增调试函数

### 1. p_light() - 轻量级调试
```php
p_light($data, $pass = false, $trace_deep = 1);
```
- 只显示最基本的调用信息
- 默认追踪层数：1层
- 最大追踪层数：2层
- 适合性能敏感的调试场景

### 2. p_fast() - 快速调试
```php
p_fast($data, $pass = false);
```
- 不显示任何追踪信息
- 只输出数据内容
- 最快的调试方式
- 适合只需要查看数据内容的场景

## 使用建议

### 性能敏感场景
```php
// 推荐使用轻量级调试
p_light($data);

// 或者使用快速调试
p_fast($data);
```

### 一般调试场景
```php
// 使用优化后的标准调试函数
p($data);
pp($data);
d($data);
```

### 需要更多追踪信息时
```php
// 可以手动指定更多层数（但不超过3层）
p($data, false, 3);
pp($data, 3);
```

## 性能对比

| 调试函数 | 默认追踪层数 | 最大追踪层数 | 输出复杂度 | 性能影响 |
|---------|-------------|-------------|-----------|---------|
| p() | 2层 | 3层 | 中等 | 低 |
| pp() | 2层 | 3层 | 中等 | 低 |
| d() | 2层 | 3层 | 中等 | 低 |
| dd() | 2层 | 2层 | 中等 | 低 |
| w() | 2层 | 2层 | 中等 | 低 |
| p_light() | 1层 | 2层 | 低 | 很低 |
| p_fast() | 0层 | 0层 | 最低 | 最低 |

## 注意事项

1. **浏览器兼容性**：优化后的调试函数在浏览器中表现更好，不会导致卡顿
2. **CLI 环境**：在命令行环境中，所有调试函数都能正常工作
3. **向后兼容**：所有现有代码无需修改，可以继续使用原有的调试函数
4. **性能提升**：调试信息的生成和处理速度显著提升

## 迁移建议

如果您的代码中有大量使用调试函数的地方，建议：

1. 对于性能敏感的循环或频繁调用的地方，使用 `p_light()` 或 `p_fast()`
2. 对于一般调试场景，继续使用优化后的标准函数
3. 如果确实需要更多追踪信息，可以手动指定层数，但建议不超过3层

## 示例代码

```php
// 在循环中使用轻量级调试
foreach ($items as $item) {
    // 使用轻量级调试，避免性能问题
    p_light($item, true); // 不终止执行
}

// 在关键位置使用快速调试
p_fast($important_data);

// 在一般场景使用标准调试
p($data);
d($data);
```
