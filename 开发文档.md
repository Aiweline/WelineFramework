## 🛠️ 开发工具

### 开发工具面板（仅开发模式）

WelineFramework 提供了一个强大的开发工具面板，帮助开发者快速浏览和测试系统中的所有路由。

**主要特性：**
- 🎯 右侧悬浮面板，不遮挡主要内容
- 🔄 一键展开/收起，状态自动保存
- 📦 按模块分组显示所有路由
- 🔍 实时搜索功能，快速定位路由
- 🔗 点击即可在新标签页中测试路由
- 🎨 支持 Frontend 和 Backend 双模式切换

**使用方法：**

1. 确保处于开发模式：
```bash
php bin/m deploy:mode:set dev
```

2. 升级 DeveloperWorkspace 模块：
```bash
php bin/m module:upgrade Weline_DeveloperWorkspace
```

3. 访问任何前端或后端页面，在页面右侧会看到开发工具触发按钮

4. 点击按钮展开面板，浏览所有可用路由

**详细文档：** 查看 `app/code/Weline/DeveloperWorkspace/doc/开发工具面板使用指南.md`

## 框架概述

WelineFramework 是一个基于PHP 8.1+的现代化Web应用框架，采用MVC架构设计，提供强大的模块化开发能力、ORM数据库操作、主题系统、缓存管理等功能。

### 核心特性

- **MVC架构**：清晰的模型-视图-控制器分离
- **模块化设计**：支持独立模块开发、部署和维护
- **ORM数据库**：自定义ORM链式操作，支持多种数据库
- **主题系统**：灵活的前端主题管理
- **缓存系统**：多层次缓存支持
- **国际化**：完整的多语言支持
- **事件系统**：观察者模式和事件驱动
- **命令行工具**：丰富的CLI命令支持

#### 2.1 路由规则详解

**控制器方法命名规则**:

1. **index方法特殊处理**:
   ```php
   // 控制器方法为index时，路由可以省略index
   class UserController extends BackendController
   {
       function index()  // 路由: /user 或 /user/index
       {
           // 控制器逻辑
       }
   }
   ```

2. **HTTP方法前缀解析**:
   ```php
   // 框架会根据方法名的大写字母拆分，提取HTTP方法前缀
   class UserController extends BackendController
   {
       // 方法名: getData -> 路由: /user/data (仅GET请求)
       function getData()
       {
           // 只接受GET请求
       }
       
       // 方法名: postData -> 路由: /user/data (仅POST请求)
       function postData()
       {
           // 只接受POST请求
       }
       
       // 方法名: deleteData -> 路由: /user/data (仅DELETE请求)
       function deleteData()
       {
           // 只接受DELETE请求
       }
       
       // 方法名: putData -> 路由: /user/data (仅PUT请求)
       function putData()
       {
           // 只接受PUT请求
       }
   }
   ```

3. **方法名解析机制**:
   ```php
   // 解析过程：
   // 1. 按大写字母拆分方法名
   // 2. 检查第一个部分是否为HTTP方法 (GET, POST, PUT, DELETE, UPDATE)
   // 3. 如果是，提取为请求方法限制，剩余部分作为路由路径
   // 4. 如果不是，整个方法名作为路由路径
   
   // 示例：
   'getData' -> ['get', 'Data'] -> 请求方法: GET, 路由: /data
   'postUser' -> ['post', 'User'] -> 请求方法: POST, 路由: /user
   'deleteUser' -> ['delete', 'User'] -> 请求方法: DELETE, 路由: /user
   'updateUser' -> ['update', 'User'] -> 请求方法: UPDATE, 路由: /user
   'getUserData' -> ['get', 'User', 'Data'] -> 请求方法: GET, 路由: /user-data
   'getGetData' -> ['get', 'Get', 'Data'] -> 请求方法: GET, 路由: /get-data
   ```

4. **特殊情况处理**:
   ```php
   class UserController extends BackendController
   {
       // 如果想要路由为 /user/getData，方法名应该是 getGetData
       function getGetData()  // 路由: /user/get-data (仅GET请求)
       {
           // 控制器逻辑
       }
       
       // 如果方法名本身就是HTTP方法，会特殊处理
       function get()  // 路由: /user (仅GET请求)
       {
           // 控制器逻辑
       }
       
       function post()  // 路由: /user (仅POST请求)
       {
           // 控制器逻辑
       }
   }
   ```

5. **路由路径生成规则**:
   ```php
   // 路由路径生成：
   // 1. 控制器名转换为小写
   // 2. 方法名按大写字母拆分后转换为小写，用连字符连接
   // 3. 移除index后缀
   
   // 示例：
   UserController::getData() -> /user/data
   UserController::getUserData() -> /user/user-data
   UserController::postUser() -> /user/user
   UserController::index() -> /user 或 /user/index
   ```

6. **支持的HTTP方法**:
   ```php
   // 框架支持的HTTP方法：
   const METHODS = [
       'GET',     // 获取数据
       'POST',    // 创建数据
       'PUT',     // 更新数据
       'DELETE',  // 删除数据
       'UPDATE',  // 更新数据（自定义方法）
   ];
   ```

7. **路由注册示例**:
   ```php
   // 自动生成的路由规则：
   [
       'user/data' => [
           'class' => 'UserController',
           'method' => 'getData',
           'request_method' => 'GET'
       ],
       'user/data' => [
           'class' => 'UserController', 
           'method' => 'postData',
           'request_method' => 'POST'
       ],
       'user/data' => [
           'class' => 'UserController',
           'method' => 'deleteData', 
           'request_method' => 'DELETE'
       ]
   ]
   ```

8. **最佳实践**:
   ```php
   class UserController extends BackendController
   {
       // ✅ 推荐：清晰的RESTful风格
       function getList()     // GET /user/list
       function postCreate()  // POST /user/create
       function putUpdate()   // PUT /user/update
       function deleteRemove() // DELETE /user/remove
       
       // ✅ 推荐：如果需要完整的方法名作为路由
       function getGetData()  // GET /user/get-data
       function postPostData() // POST /user/post-data
       
       // ✅ 现在支持：getData方法同时支持两种路由
       function getData()     // 同时支持 GET /user/data 和 GET /user/getdata
       function postData()    // 同时支持 POST /user/data 和 POST /user/postdata
       
       // ❌ 避免：容易混淆的命名
       function data()        // 接受所有HTTP方法
   }
   ```

9. **路由兼容性**:
   ```php
   // 修改后的路由规则支持双重路由：
   // 1. 原有路由：提取HTTP方法前缀后的路由
   // 2. 完整路由：保留完整方法名的路由
   
   // 示例：
   function getData() {
       // 支持以下两种路由：
       // GET /user/data     (原有路由)
       // GET /user/getdata  (完整方法名路由)
   }
   
   function postUserData() {
       // 支持以下两种路由：
       // POST /user/user-data    (原有路由)
       // POST /user/postuserdata (完整方法名路由)
   }
   ```

**路由配置**:
- 后端PC路由: `generated/routers/backend_pc.php`
- 后端API路由: `generated/routers/backend_rest_api.php`
- 前端PC路由: `generated/routers/frontend_pc.php`

### 4. 数据库模型 (Model)

**文件位置**: `app/code/Weline/Framework/Database/Model.php`

强大的ORM系统，支持：
- 链式查询
- 模型关联
- 数据验证
- 分页处理
- 缓存集成
- **内置预执行**: 自动处理SQL预处理，防止SQL注入，直接填写值即可

**模型定义**:
```php
class UserModel extends Model
{
    public const table = 'users';
    public const primary_key = 'id';
    public const fields_ID = 'id';
    public const fields_CREATE_TIME = 'create_time';
    public const fields_UPDATE_TIME = 'update_time';
}
```

**查询示例**:
```php
// 基础查询 - ORM自动处理SQL预处理，直接填写值即可
$users = $userModel->where('status', 1)->select()->fetch();

// 条件查询 - 无需手动预处理，框架自动处理
$users = $userModel->where('username', 'john')->where('age', 25, '>')->select()->fetch();

// 分页查询
$users = $userModel->pagination(1, 20)->select()->fetch();

// 关联查询
$users = $userModel->joinModel(ProfileModel::class, 'profile')->select()->fetch();

// 复杂查询 - 所有参数都会自动预处理
$users = $userModel->where('status', 1)
    ->where('create_time', '2023-01-01', '>=')
    ->where('username', '%admin%', 'like')
    ->order('create_time', 'DESC')
    ->select()
    ->fetch();
```

### ORM查询方法详解

**基础查询方法**:
```php
// 查找单条记录
$user = $userModel->where('id', 1)->find()->fetch();

// 查找多条记录
$users = $userModel->where('status', 1)->select()->fetch();

// 获取原始数组数据
$users = $userModel->select()->fetchArray();

// 统计记录数
$count = $userModel->where('status', 1)->total();
```

**条件查询方法**:
```php
// 基础条件
$users = $userModel->where('username', 'john')->select()->fetch();

// 比较条件
$users = $userModel->where('age', 18, '>=')->select()->fetch();

// 模糊查询
$users = $userModel->where('username', '%admin%', 'like')->select()->fetch();

// IN查询
$users = $userModel->where('id', [1,2,3], 'in')->select()->fetch();

// 多条件组合
$users = $userModel->where('status', 1)
    ->where('age', 18, '>=', 'AND')
    ->where('username', '%admin%', 'like', 'OR')
    ->select()
    ->fetch();
```

**排序和分组**:
```php
// 排序
$users = $userModel->order('create_time', 'DESC')->select()->fetch();

// 多字段排序
$users = $userModel->order('status', 'ASC')
    ->order('create_time', 'DESC')
    ->select()
    ->fetch();

// 分组
$users = $userModel->group('status')->select()->fetch();

// 分组后筛选
$users = $userModel->group('status')
    ->having('count(*) > 1')
    ->select()
    ->fetch();
```

**分页查询**:
```php
// 基础分页
$users = $userModel->page(1, 20)->select()->fetch();

// 完整分页（包含总数统计）
$users = $userModel->pagination(1, 20)->select()->fetch();

// 限制查询
$users = $userModel->limit(10, 0)->select()->fetch();
```

**字段操作**:
```php
// 指定查询字段
$users = $userModel->fields('id,username,email')->select()->fetch();

// 字段拼接
$users = $userModel->concat('first_name,last_name', 'full_name')->select()->fetch();

// 字段模糊查询
$users = $userModel->concat_like('first_name,last_name', '%john%')->select()->fetch();

// 分组拼接
$users = $userModel->group('category_id')
    ->group_concat('name', 'category_names')
    ->select()
    ->fetch();
```

**时间范围查询**:
```php
// 时间范围
$users = $userModel->period('today')->select()->fetch();
$users = $userModel->period('current_month')->select()->fetch();
$users = $userModel->period('last_week')->select()->fetch();

// 支持的时间段：all, today, yesterday, current_week, near_week, 
// last_week, near_month, current_month, last_month, quarter, 
// last_quarter, current_year, last_year
```

**数据操作**:
```php
// 插入数据
$result = $userModel->insert(['username' => 'john', 'email' => 'john@example.com'])->fetch();

// 更新数据
$result = $userModel->where('id', 1)->update(['username' => 'john_updated'])->fetch();

// 累加操作
$result = $userModel->where('id', 1)->inc('view_count', 1)->fetch();

// 累减操作
$result = $userModel->where('id', 1)->dec('view_count', 1)->fetch();

// 删除数据

```

**事务操作**:
```php
// 开启事务

try {
    // 执行操作
    $userModel->insert(['username' => 'john'])->fetch();
    $userModel->update(['status' => 1])->fetch();
    
    // 提交事务
    $userModel->commit();
} catch (Exception $e) {
    // 回滚事务
    $userModel->rollBack();
    throw $e;
}
```

**表操作**:
```php

// 备份表

// 自定义SQL查询
```

**查询清理和重置**:
```php

$userModel->clear('orders');  // 清理排序条件
$userModel->clear('limit');   // 清理限制条件
```

## 自定义标签系统 (Taglib)

**文件位置**: `app/code/Weline/Taglib/` 和 `app/code/Weline/*/Taglib/`

WelineFramework提供了强大的自定义标签系统，允许开发者创建可复用的模板标签，实现模块化、组件化的前端开发。

### 标签接口 (TaglibInterface)

**核心接口方法**:
```php
interface TaglibInterface
{
    // 必需方法
    static function name(): string;                    // 标签名称
    static function tag(): bool;                       // 是否支持成对标签
    static function callback(): callable;              // 处理回调函数
    
    // 可选方法
    static function attr(): array;                     // 属性定义
    static function tag_start(): bool;                 // 是否处理标签开始
    static function tag_end(): bool;                   // 是否处理标签结束
    static function tag_self_close(): bool;            // 是否支持自闭合
    static function tag_self_close_with_attrs(): bool; // 自闭合是否支持属性
    static function parent(): ?string;                 // 依赖管理
    static function document(): string;                // 标签文档
}
```

### 创建自定义标签

**基础标签示例**:
```php
<?php
namespace Weline\Taglib;

use Weline\Taglib\TaglibInterface;

class MyTag implements TaglibInterface
{
    /**
     * 标签名称
     */
    public static function name(): string
    {
        return 'my-tag';
    }

    /**
     * 是否支持成对标签 <tag></tag>
     */
    public static function tag(): bool
    {
        return true;
    }

    /**
     * 属性定义
     */
    public static function attr(): array
    {
        return [
            'title' => true,    // 必需属性
            'class' => false,   // 可选属性
            'style' => false    // 可选属性
        ];
    }

    /**
     * 处理回调函数
     */
    public static function callback(): callable
    {
        return function ($tag_key, $config, $tag_data, $attributes) {
            $title = $attributes['title'] ?? '默认标题';
            $class = $attributes['class'] ?? '';
            $style = $attributes['style'] ?? '';
            $content = $tag_data[2] ?? ''; // 标签内容
            
            return <<<HTML
<div class="my-tag {$class}" style="{$style}">
    <h3>{$title}</h3>
    <div class="content">{$content}</div>
</div>
HTML;
        };
    }

    /**
     * 是否处理标签开始
     */
    public static function tag_start(): bool
    {
        return false;
    }

    /**
     * 是否处理标签结束
     */
    public static function tag_end(): bool
    {
        return false;
    }

    /**
     * 是否支持自闭合
     */
    public static function tag_self_close(): bool
    {
        return true;
    }

    /**
     * 自闭合是否支持属性
     */
    public static function tag_self_close_with_attrs(): bool
    {
        return true;
    }

    /**
     * 依赖管理
     */
    public static function parent(): ?string
    {
        return null; // 无依赖
    }

    /**
     * 标签文档
     */
    public static function document(): string
    {
        return '我的自定义标签：<w:my-tag title="标题" class="样式">内容</w:my-tag>';
    }
}
```

**命名规范**:

**文档说明**:

```php
{

```

> 重要规范（2025-10 更新）

- 标签渲染禁止再次输出 PHP 代码。自定义 Taglib 的 callback 必须返回纯 HTML/JS 字符串，不能包含 `<?php ... ?>` 或 `<?= ... ?>`，否则浏览器无法解析。
- 不允许在标签属性中直接写 PHP 表达式。需要的动态值应在模板中先计算为纯文本，再以属性方式传入。
- Block 模板读取参数时使用局部变量（如 `$id/$name/...`），不要在模板里通过 `$this->getData()` 读取。
- 示例（AI 模型选择器标签）：


2. **重新扫描命令** - 扫描所有模块的命令类
3. **更新命令列表** - 重新生成可用的命令列表
4. **验证命令接口** - 检查命令类是否实现正确的接口

**使用场景**：
- 添加新的自定义命令后
- 修改现有命令的配置后
- 命令无法识别时
- 系统升级后需要刷新命令列表

#### `http:request` - HTTP请求测试命令
**功能**：强大的HTTP请求测试工具，使用Guzzle HTTP客户端，支持自动登录、智能Cookie管理、并发请求测试，可以快速测试前端和后端路径，并支持内容过滤和搜索功能

```bash
# 基本用法 - 测试前端路径
php bin/w http:request /
php bin/w http:request index/index

# 测试后端路径（使用 -b 或 -backend 参数，自动登录和管理Cookie）
php bin/w http:request admin -b                              # 自动登录并访问后台首页
php bin/w http:request admin/dashboard -b                    # 自动登录并访问dashboard
php bin/w http:request ai/backend/model -b                   # 自动登录并访问AI模型管理
php bin/w http:request admin/login -b                        # 访问登录页（公共页面，无需登录）

# 自定义登录凭据
php bin/w http:request admin/dashboard -b -u=myuser -p=mypass  # 使用自定义用户名密码

# 测试API后端路径（使用 -api 或 -api-backend 参数，自动添加api_admin密钥并登录）
php bin/w http:request rest -api
php bin/w http:request rest/v1/module/action -api -u=admin -p=admin123456

# 内容搜索和过滤（用于快速定位错误或验证内容）
php bin/w http:request / --filter=welcome              # 搜索包含"welcome"的内容
php bin/w http:request / --filter=welcome -n=5         # 显示匹配行上下5行内容
php bin/w http:request ai/backend/model -b --filter=Warning    # 搜索PHP Warning错误
php bin/w http:request ai/backend/model -b --filter=Fatal      # 搜索PHP Fatal错误
php bin/w http:request admin/login -b --filter=title   # 在后端页面中搜索"title"

# HTTP方法和数据
php bin/w http:request api/data -m=POST -d='{"key":"value"}'
php bin/w http:request api/user --method=PUT --data='{"name":"test"}'

# 自定义请求头
php bin/w http:request / -H="User-Agent: CustomBot"
php bin/w http:request api/data -H="Accept: application/json" -H="Content-Type: application/json"

# 并发请求测试（压力测试）
php bin/w http:request / -C -t=100                     # 并发测试100次
php bin/w http:request admin -b -C -t=50               # 并发压测后台50次（自动复用cookie）

# HTTPS TLS验证
php bin/w http:request https://example.com --tls       # 启用TLS证书验证（默认不验证）
```

**主要功能**：
1. **路径测试** - 测试前端和后端路径的可访问性
2. **自动登录** - 后台和API路径自动登录，无需手动指定`--login`
3. **智能Cookie管理** - 自动保存和复用cookie，过期自动重新登录
4. **内容搜索** - 使用filter参数快速定位响应内容（适合搜索PHP错误）
5. **上下文显示** - 显示匹配内容周围的指定行数（默认3行）
6. **并发测试** - 支持多线程并发请求，用于压力测试
7. **HTTP方法支持** - 支持GET、POST、PUT、DELETE等HTTP方法
8. **自定义请求头** - 可以添加自定义HTTP请求头
9. **TLS验证控制** - 可选择是否验证HTTPS证书
10. **详细的性能统计** - 并发请求时提供详细的性能指标

**参数说明**：
- `path` - 请求路径（必需参数）
- `-b, --backend` - 指定为后端路径（自动添加admin密钥并登录）
- `-api, --api-backend` - 指定为API后端路径（自动添加api_admin密钥并登录）
- `-u, --username=<用户名>` - 登录用户名（默认：admin）
- `-p, --password=<密码>` - 登录密码（默认：admin）
- `-c, --cookie=<文件>` - 指定cookie文件路径（默认：var/http_request_cookies.txt）
- `--filter=<关键词>` - 搜索并提取包含关键词的内容
- `-n=<行数>` - 指定提取的上下文行数（默认3行）
- `-C, --concurrent` - 启用并发请求模式
- `-t, --times=<次数>` - 并发请求次数（默认1次，最大100次）
- `--tls` - 启用HTTPS TLS证书验证（默认不验证）
- `-m, --method=<方法>` - 指定HTTP请求方法（默认GET）
- `-H, --header=<头>` - 添加HTTP请求头
- `-d, --data=<数据>` - 发送POST/PUT数据

**使用场景**：
- 快速测试路由是否正常工作
- 检查页面响应状态码和响应时间
- 搜索响应内容中的特定关键词（如搜索PHP错误、验证内容）
- 测试API接口的可访问性
- 开发调试时快速定位PHP错误（Warning、Fatal、Notice等）
- 查看响应头信息
- 测试后台页面渲染和JS执行情况
- 并发压力测试（测试系统性能和并发能力）
- 验证cookie和会话管理是否正常
- 自动化测试脚本中使用

**自动登录功能**：
- 使用 `-b` 或 `-api` 参数访问后台或API时，**自动登录，无需手动指定 `--login`**
- 默认使用用户名 `admin` 和密码 `admin`，可通过 `-u` 和 `-p` 自定义
- Cookie文件默认保存在 `var/http_request_cookies.txt`
- 如果cookie文件存在且有效，自动复用，避免重复登录
- 如果cookie过期（返回登录页面），自动重新登录
- 登录页面等公共路径无需登录即可访问

**智能Cookie管理流程**：
1. 首次访问后台：自动登录 → 保存cookie到 `var/http_request_cookies.txt`
2. 再次访问后台：读取cookie → 如果有效直接使用 → 如果过期自动重新登录
3. 并发测试：所有并发请求共享同一个cookie文件，避免重复登录

## 模块开发指南

### 1. 创建模块

WelineFramework采用模块化架构，每个模块都是独立的功能单元，类似于`Weline_Acl`、`Weline_FileManager`等。

#### 1.1 模块结构

**标准模块目录结构**:
```
app/code/Weline/YourModule/
├── register.php              # 模块注册文件（必需）
├── composer.json             # Composer配置（可选）
├── doc/                      # 文档目录
│   └── README.md
├── etc/                      # 配置文件目录
│   ├── event.xml            # 事件配置
│   ├── plugin.xml           # 插件配置
│   ├── env.php              # 环境配置
│   └── backend/
│       └── menu.xml         # 后台菜单配置
├── Controller/               # 控制器目录
│   └── Backend/
│       └── YourController.php
├── Model/                    # 模型目录
│   └── YourModel.php
├── Observer/                 # 观察者目录
│   └── YourObserver.php
├── Plugin/                   # 插件目录
│   └── YourPlugin.php
├── Taglib/                   # 自定义标签目录
│   └── YourTag.php
├── Setup/                    # 安装脚本目录
│   └── Install.php
├── Cache/                    # 缓存目录
│   └── YourCache.php
├── view/                     # 视图目录
│   ├── templates/
│   └── statics/
└── Block/                    # 块目录
    └── YourBlock.php
```

#### 1.2 创建模块步骤

**步骤1: 创建模块目录**
```bash
# 创建模块目录
mkdir -p app/code/Weline/YourModule
```

**步骤2: 创建注册文件**
```php
<?php
// app/code/Weline/YourModule/register.php

use Weline\Framework\Register\Register;

Register::register(
    Register::MODULE,           // 注册类型：模块
    'Weline_YourModule',        // 模块名称
    __DIR__,                    // 模块路径
    '1.0.0',                    // 版本号
    '模块描述',                  // 模块描述
    ['Weline_Framework']        // 依赖模块（可选）
);
```

**步骤3: 安装模块**
```bash
# 运行模块安装命令
php bin/w setup:upgrade
```

#### 1.3 模块注册参数详解

```php
Register::register(
    $type,          // 注册类型：Register::MODULE
    $module_name,   // 模块名称：Weline_YourModule
    $param,         // 模块路径：__DIR__
    $version,       // 版本号：1.0.0
    $description,   // 模块描述
    $dependencies   // 依赖模块数组
);
```

**参数说明**:
- `$type`: 注册类型，模块使用`Register::MODULE`
- `$module_name`: 模块名称，格式为`Vendor_ModuleName`
- `$param`: 模块路径，通常使用`__DIR__`
- `$version`: 版本号，遵循语义化版本规范
- `$description`: 模块描述，支持HTML标签
- `$dependencies`: 依赖模块数组，如`['Weline_Framework', 'Weline_Backend']`

#### 1.4 模块配置文件

**事件配置 (etc/event.xml)**:
```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Module::controller_attributes">
        <observer name="Weline_YourModule::controller_attributes" 
                  instance="Weline\YourModule\Observer\ControllerAttributes" 
                  disabled="false" shared="true" sort="0"/>
    </event>
</config>
```

**插件配置 (etc/plugin.xml)**:
```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Plugin/etc/xsd/plugin.xsd"
        xmlns="urn:Weline_Framework::Plugin/etc/xsd/plugin.xsd">
    <plugin name="Weline_YourModule::module_upgrade_plugin" 
            class="Weline\Framework\Setup\Console\Setup\Upgrade">
        <interceptor name="Weline_YourModule::module_upgrade_execute_after_plugin"
                     instance="Weline\YourModule\Plugin\ModuleUpgradeExecuteAfterPlugin"
                     disabled="false" sort="0"/>
    </plugin>
</config>
```

**后台菜单配置 (etc/backend/menu.xml)**:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<menus xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
       xs:noNamespaceSchemaLocation="urn:weline:module:Weline_Backend::etc/xsd/menu.xsd"
       xs:schemaLocation="urn:weline:module:Weline_Backend::etc/xsd/menu.xsd">
    <!-- 主菜单 -->
    <add source="Weline_YourModule::main" 
         name="your_module_main" 
         title="您的模块" 
         action="admin/yourmodule/index" 
         parent="Weline_Backend::dashboard"
         icon="fas fa-cog"
         order="100"/>
    <!-- 子菜单 -->
    <add source="Weline_YourModule::settings" 
         name="your_module_settings" 
         title="模块设置" 
         action="admin/yourmodule/settings" 
         parent="Weline_YourModule::main"
         icon="fas fa-settings"
         order="1"/>
    <add source="Weline_YourModule::list" 
         name="your_module_list" 
         title="数据列表" 
         action="admin/yourmodule/list" 
         parent="Weline_YourModule::main"
         icon="fas fa-list"
         order="2"/>
</menus>
```

**菜单配置要点**:
- **XML声明**: 必须包含 `encoding="UTF-8"`
- **根元素**: 使用 `<menus>` 而不是 `<config>`
- **菜单项**: 使用 `<add>` 元素
- **必需属性**:
  - `source`: ACL权限资源 (格式: `ModuleName::permission`)
  - `name`: 菜单唯一标识符
  - `title`: 显示标题
  - `action`: 路由地址 (格式: `admin/module/controller/action`)
  - `parent`: 父菜单 (使用 `source` 值)
  - `icon`: 图标类名 (推荐使用 `fas fa-*` 或 `mdi mdi-*`)
  - `order`: 排序数字
- **父菜单选择**: 
  - `Weline_Backend::dashboard` - 主仪表板
  - `Weline_Backend::system_menu` - 系统菜单
  - `Weline_Backend::system_configuration` - 系统配置
  - 自定义父菜单的 `source` 值

#### 1.5 模块安装脚本

**安装脚本 (Setup/Install.php)**:
```php
<?php
namespace Weline\YourModule\Setup;

use Weline\Framework\Setup\Data\Context;
use Weline\Framework\Setup\InstallInterface;
use Weline\Framework\Setup\Data\DataInterface;
use Weline\Framework\Manager\ObjectManager;

class Install implements InstallInterface
{
    /**
     * 执行安装 - 必需方法
     */
    public function setup(DataInterface $setup, Context $context): string
    {
        return $this->install($context);
    }

    /**
     * 实际安装逻辑
     */
    public function install(Context $context): string
    {
        try {
            // 安装数据库表
            $model = ObjectManager::getInstance(\Weline\YourModule\Model\YourModel::class);
            $model->install($model->setup(), $context);
            
            // 安装其他数据
            // ...
```

**InstallInterface 实现要点**:

- **返回值**: 必须返回字符串，表示安装结果

- **模型安装**: 通过ObjectManager获取模型实例，调用install方法
- **输出信息**: 使用Context的Output对象输出安装进度和结果

**ModelInterface 实现要点**:
- **必需方法**: 
  - `setup(ModelSetup $setup, Context $context): void` - 模型设置
  - `upgrade(ModelSetup $setup, Context $context): void` - 模型升级
  - `install(ModelSetup $setup, Context $context): void` - 模型安装
- **方法签名**: 必须严格按照接口定义的方法签名实现
- **安装逻辑**: 在install方法中实现数据表创建和初始化数据
- **升级逻辑**: 在upgrade方法中实现版本升级时的数据迁移
- **设置逻辑**: 在setup方法中实现模型初始化设置

```php
public function processModuleSourceFilePath(string $type, string $source): array
{
    $t_f = $type . DS . $source;
    
    // 如果存在跨模块调用的情况（包含::）
    if (is_int(strpos($source, "::"))) {
        $t_f_arr = explode("::", $source);
        if (count($t_f_arr) > 1) {
            $t_f_arr[1] = trim($t_f_arr[1], DS);
            if (is_int(strpos($t_f_arr[1], $type))) {
                $t_f_arr[2] = $t_f_arr[1];
                $t_f_arr[1] = "::";
            } else {
                $t_f_arr[2] = $t_f_arr[1];
                $t_f_arr[1] = "::" . $type . DS;
            }
            $t_f = implode("", $t_f_arr);
        }
    }
    
    if (empty($t_f_arr)) {
        return [$t_f, $this->getRequest()->getModuleName()];
    }
    return [$t_f, array_shift($t_f_arr)];
```

#### 3.2 翻译机制实现

**翻译解析器**: `app/code/Weline/Framework/Phrase/Parser.php`

```php
class Parser
{
    public static function parse(string|array &$words, int|array|string|null $args = null): mixed
    {
        $words = self::processWords($words);
        
        if ($args === null) {
            return $words;
        }
        
        // 参数替换逻辑
        if (is_string($args) || is_numeric($args)) {
            if (str_contains($words, '%{1}')) {
                $words = str_replace('%{1}', '%{}', $words);
            }
            $words = str_replace('%{}', $args, $words);
            return $words;
        }
        
        // 数组参数处理
        if (is_array($args)) {
            foreach ($args as $key => $arg) {
                if (is_numeric($key)) {
                    $key += 1;
                }
                $words = str_replace('%{' . $key . '}', $arg ?? '', $words);
            }
        }
        
        return $words;
    }
}
```

#### 3.3 前端翻译实现

**前端翻译函数**: 在模板头部自动加载

```javascript
// 前端翻译函数
function __(word, arguments = null) {
    return phrase(word, arguments);
}

function phrase(phrase, arguments = null) {
    if (window.site.i18n[phrase] !== undefined) {
        phrase = window.site.i18n[phrase]
    } else {
        window.site.i18n[phrase] = phrase
    }
    
    if (arguments) {
        switch (arguments.constructor) {
            case String:
                phrase = phrase.replace(/%1/g, arguments)
                break;
            case Object:
                for (let argument_key in arguments) {
                    let arg = arguments[argument_key]
                    let regExp = new RegExp(eval('/%' + argument_key + '/g'));
                    phrase = phrase.replace(regExp, arg)
                }
                break;
            case Array:
                for (let argument_key in arguments) {
                    let arg = arguments[argument_key]
                    argument_key = 1 + parseInt(argument_key)
                    let regExp = new RegExp(eval('/%' + argument_key + '/g'));
                    phrase = phrase.replace(regExp, arg)
                }
                break;
        }
    }
    return phrase
}
```

#### 3.5 翻译函数内部实现

**参数处理逻辑**:
```php
// 框架内部处理逻辑
public static function parse(string|array &$words, int|array|string|null $args = null): mixed
{
    $words = self::processWords($words);
    
    if ($args === null) {
        return $words;
    }
    
    // 字符串或数字参数
    if (is_string($args) || is_numeric($args)) {
        // 处理 %{1} 到 %{} 的转换
        if (str_contains($words, '%{1}')) {
            $words = str_replace('%{1}', '%{}', $words);
        }
        $words = str_replace('%{}', $args, $words);
        return $words;
    }
    
    // 数组参数处理
    if (is_array($args)) {
        foreach ($args as $key => $arg) {
            if (is_numeric($key)) {
                $key += 1; // 数字索引从1开始
            }
            $words = str_replace('%{' . $key . '}', $arg ?? '', $words);
        }
    }
    
    return $words;
}
```

```php
<?php
namespace Weline\Framework\Console\Console\Server;

use Weline\Framework\Console\CommandInterface;
use Weline\Framework\Output\Cli\Printing;

class Start implements CommandInterface

    private Printing $printing;

    {
        $this->printing = $printing;
    }

    {

        $host = $args['host'] ?? $args['h'] ?? '127.0.0.1';
        $port = $args['port'] ?? $args['p'] ?? '9981';
        $backend = isset($args['backend']) || isset($args['b']);

        if ($this->isServerRunning($host, $port)) {
            if ($force) {
                $this->printing->note('强制重启服务器...');
                $this->stopServer($host, $port);
            } else {
                $this->printing->success('服务器已在运行中！');
                return;
            }
        }
        
        // 启动服务器
        $this->printing->note("启动服务器: {$host}:{$port}");
        
        if ($backend) {
            $this->startBackendServer($host, $port);
        } else {
            $this->startForegroundServer($host, $port);
        }
    }

    {
        return '启动PHP内置Web服务器。使用 -b 后台运行，-f 强制重启';
    }

        if (is_resource($connection)) {
            fclose($connection);
            return true;
        }
        return false;
    }

    private function startBackendServer(string $host, int $port): void
    {
        // 后台启动逻辑
        $this->printing->success('服务器已在后台启动');
    }

    private function startForegroundServer(string $host, int $port): void
    {
        // 前台启动逻辑

    }
```

### 框架升级机制详解（重要）

WelineFramework 采用基于版本控制的升级机制，确保数据库结构变更的安全性和可追溯性。

#### 升级机制原理

**核心要点**：
1. **版本检测**：框架通过比较模块 `register.php` 中的版本号与已安装版本，判断是否需要升级
2. **升级触发**：只有当模块版本号发生变化时，才会触发升级流程
3. **模型升级**：升级时会调用模型中的 `upgrade()` 方法，执行数据库结构变更

#### 升级流程步骤

**步骤1：修改模型代码**
```php
// 在模型中添加新字段常量
class YourModel extends Model
{
    public const fields_NEW_FIELD = 'new_field';
    
    public function upgrade(ModelSetup $setup, Context $context): void
    {
        // 添加新字段
        if (!$setup->columnExist(self::fields_NEW_FIELD)) {
            $setup->addColumn(
                self::fields_NEW_FIELD, 
                TableInterface::column_type_VARCHAR, 
                255, 
                'not null default ""', 
                '新字段'
            );
        }
    }
}
```

**步骤2：更新模块版本号**
```php
// app/code/YourModule/register.php
Register::register(
    Register::MODULE,
    'Your_Module',
    __DIR__,
    '1.0.1',  // 从 1.0.0 更新到 1.0.1
    '模块描述',
    ['Weline_Framework']
);
```

**步骤3：执行升级命令**
```bash
# 方式1：仅升级模型（推荐）
php bin/w setup:upgrade --model

# 方式2：完整升级（包含路由、事件等）
php bin/w setup:upgrade
```

#### 升级机制的优势

1. **安全性**：版本控制确保升级的可追溯性
2. **幂等性**：多次执行升级命令不会重复执行相同操作
3. **跨环境一致性**：开发、测试、生产环境使用相同的升级流程
4. **回滚支持**：可以基于版本号进行回滚操作

#### 实际应用示例

**场景：为订单表添加支付方式字段**

```php
// 1. 修改 Order 模型

{
    // 添加新字段常量
    public const fields_PAYMENT_METHOD = 'payment_method';
    public const fields_PAYMENT_GATEWAY = 'payment_gateway';
    
    public function upgrade(ModelSetup $setup, Context $context): void
    {
        // 添加支付方式字段

                self::fields_PAYMENT_METHOD,
                TableInterface::column_type_VARCHAR,
                100,
                'not null default ""',
                '支付方式'
            );
        }
        
        // 添加支付网关字段
        if (!$setup->columnExist(self::fields_PAYMENT_GATEWAY)) {
            $setup->addColumn(
                self::fields_PAYMENT_GATEWAY,
                TableInterface::column_type_VARCHAR,
                100,
                'not null default ""',
                '支付网关'
            );
        }
    }
}
```

#### 升级注意事项

1. **版本号规范**：遵循语义化版本规范（主版本.次版本.修订版本）
2. **升级方法实现**：必须在 `upgrade()` 方法中实现具体的数据库变更逻辑
3. **字段检查**：使用 `columnExist()` 等方法检查字段是否已存在，避免重复添加
4. **数据迁移**：如有必要，在升级时进行数据迁移和转换
5. **备份建议**：升级前建议备份数据库

#### 常见升级场景

**添加新字段**：
```php
public function upgrade(ModelSetup $setup, Context $context): void
{
    if (!$setup->columnExist(self::fields_NEW_COLUMN)) {
        $setup->addColumn(
            self::fields_NEW_COLUMN,
            TableInterface::column_type_VARCHAR,
            255,
            'not null default ""',
            '新字段描述'
        );
    }
}
```

**修改字段类型**：
```php
public function upgrade(ModelSetup $setup, Context $context): void
{
    // 注意：某些数据库可能不支持直接修改字段类型
    // 建议先备份数据，然后重建表结构
    if ($setup->columnExist(self::fields_EXISTING_COLUMN)) {
        $setup->alterColumn(
            self::fields_EXISTING_COLUMN,
            TableInterface::column_type_TEXT,
            null,
            'not null',
            '修改后的字段描述'
        );
    }
}
```

**添加索引**：
```php
public function upgrade(ModelSetup $setup, Context $context): void
{
    if (!$setup->indexExist('idx_new_field')) {
        $setup->addIndex('idx_new_field', [self::fields_NEW_FIELD]);
    }
}
```

这个升级机制确保了框架的稳定性和可维护性，是 WelineFramework 的核心特性之一。

### 模型实例化建议

在框架内**不要直接使用 `new` 来实例化模型**，因为对象管理与依赖注入需要由框架的对象管理器统一处理。推荐两种方式：

- 使用全局辅助函数 `w_obj()`：
  ```php
  $catalogModel = w_obj(\Weline\DeveloperWorkspace\Model\Document\Catalog::class);
  ```
  该函数内部会调用 `ObjectManager::getInstance()`，返回共享实例。

- 使用 `ObjectManager`：
  ```php
  $catalogModel = \Weline\Framework\Manager\ObjectManager::make(\Weline\DeveloperWorkspace\Model\Document\Catalog::class);
  ```
  `make` 会创建一个新的实例（非共享），适用于需要独立状态的场景（例如在 observer/批量导入中避免状态污染）。

示例说明：
- 在长运行或批量任务中（Observer、CLI 命令），优先使用 `ObjectManager::make()` 创建模型新实例，避免共享单例导致的数据残留。
- 在普通控制器或调用链中，使用 `w_obj()`（共享实例）可以提升性能并保持统一实例管理。

已把以上建议写入本节以便团队遵循。
  - 每次需要结构变更时务必更新模块版本并执行升级，或在开发环境手动触发 `module:upgrade` 类相关流程。
  - 升级前建议备份数据库（导出或复制 sqlite 文件），以便出问题时回滚。
  - 若只是需要临时测试，可以在 `Setup` 下编写 `Upgrade` 脚本并在升级时由框架调用；但上线前应将变更通过模块版本控制并在 CI/CD 中执行升级命令。

把此节写入模块开发流程文档，提醒团队每次模型字段变更后必须执行升级流程以避免数据库与模型定义不一致的错误（例如缺少 `is_system` 字段导致的查询异常）。

## 22. 路由系统详解

WelineFramework的路由系统分为三种类型：后台路由、PC路由和REST API路由。

### 22.1 路由文件结构

**路由文件位置**:
```php
// 路由文件存储在 generated/routers/ 目录下，这些是框架自动生成的
generated/routers/backend_pc.php        // 后台PC路由
generated/routers/backend_rest_api.php  // 后台REST API路由
generated/routers/frontend_pc.php       // 前端PC路由
generated/routers/frontend_rest_api.php // 前端REST API路由
```

### 22.2 路由组成规则

**路由键格式**:
```php
// PC路由格式
'module/controller/action::METHOD'  // 带HTTP方法限制
'module/controller/action'          // 无HTTP方法限制

// 示例
'yourmodule/backend/item/index::GET'   // 后台项目管理页面
'yourmodule/backend/item/save::POST'   // 后台项目保存
'admin/backend/user/listing'        // 后台用户列表（无方法限制）
```

**路由数据结构**:
```php
'yourmodule/backend/item/index::GET' => [
    'module' => 'YourModule',
    'module_path' => '/path/to/module/',
    'router' => 'yourmodule',                 // 路由别名
    'class' => [
        'area' => 'BackendController',        // 控制器类型
        'name' => 'YourModule\\Controller\\Backend\\Item',
        'method' => 'getIndex',               // 控制器方法
        'controller_name' => 'Backend/Item',  // 控制器路径
        'request_method' => 'GET',            // HTTP方法限制
    ],
]
```

### 22.3 控制器方法解析规则

**方法名解析**:
```php
// 1. HTTP方法前缀解析
getData()     -> /data (GET请求)
postSave()    -> /save (POST请求)
deleteItem()  -> /item (DELETE请求)

// 2. 双重路由支持
getData()     -> 同时支持 /data 和 /getdata (GET请求)
postSave()    -> 同时支持 /save 和 /postsave (POST请求)

// 3. index方法特殊处理
index()       -> / (可省略index后缀)
getIndex()    -> / (GET请求，可省略index后缀)

// 4. 方法名拆分规则
getUserData() -> getUser + Data -> /user-data (GET请求)
postSaveUser() -> postSave + User -> /save-user (POST请求)
```

### 22.4 路由类型说明

**1. 后台PC路由 (BackendController)**:
- **用途**: 后台管理页面
- **特点**: 需要登录验证，返回HTML页面
- **示例**: `admin/backend/user/listing`

**2. 后台REST API路由 (BackendRestController)**:
- **用途**: 后台API接口
- **特点**: 需要登录验证，返回JSON数据
- **示例**: `admin/rest/v1/user/fetch::GET`

**3. 前端PC路由 (FrontendController)**:
- **用途**: 前端页面
- **特点**: 公开访问，返回HTML页面
- **示例**: `frontend/product/detail`

**4. 前端REST API路由 (FrontendRestController)**:
- **用途**: 前端API接口
- **特点**: 公开访问，返回JSON数据
- **示例**: `api/rest/v1/product/list::GET`

### 22.5 路由配置

**模块路由别名设置**:
```php

    'router' => 'yourmodule'  // 设置路由别名，默认为模块名
```

### 22.7 常见路由错误及修复

**错误1：菜单配置中的错误路径**
```xml

<!-- ✅ 正确：使用正确的路由格式 -->
```

**错误2：路由访问地址错误**

**修复步骤**:
1. 检查菜单配置文件 `etc/backend/menu.xml`
2. 确保 `action` 属性使用正确的路由格式
3. 运行 `php bin/w setup:upgrade --route` 更新路由缓存
4. 测试访问地址是否正确

**路由组成验证**:
- `admin_key`: 从 `app/etc/env.php` 获取
- `module_router`: 从模块 `etc/env.php` 的 `router` 配置获取
- `controller_path`: 对应 `Controller/Backend/Order.php` → `backend/order`
- `action`: 对应控制器方法名

## 模块开发常见错误及解决方案

### 错误1: 模块安装脚本方法签名错误

**问题描述：**
```
Fatal error: Declaration of FlashForge\Tool\Setup\Install::setup(Weline\Framework\Setup\Data\DataInterface $setup, Weline\Framework\Setup\Data\Context $context): string must be compatible with Weline\Framework\Setup\InstallInterface::setup(Weline\Framework\Setup\Data\Setup $setup, Weline\Framework\Setup\Data\Context $context): void
```

**错误原因：**
- 使用了错误的接口类型：`DataInterface` 应该是 `Setup`
- 返回类型错误：应该是 `void` 而不是 `string`

**解决方案：**
```php
// ❌ 错误写法
use Weline\Framework\Setup\Data\DataInterface;
public function setup(DataInterface $setup, Context $context): string

// ✅ 正确写法
use Weline\Framework\Setup\Data\Setup;
public function setup(Setup $setup, Context $context): void
```

**完整修复示例：**
```php
<?php
namespace FlashForge\Tool\Setup;

use Weline\Framework\Setup\Data\Context;
use Weline\Framework\Setup\InstallInterface;
use Weline\Framework\Setup\Data\Setup;  // 正确的类型
use Weline\Framework\Manager\ObjectManager;

class Install implements InstallInterface
{
    public function setup(Setup $setup, Context $context): void  // 正确的签名
    {
        $this->install($context);
    }

    public function install(Context $context): void  // 返回类型也改为void
    {
        try {
            // 安装逻辑
            $context->getOutput()->writeln('<info>模块安装完成</info>');
        } catch (\Exception $e) {
            $context->getOutput()->writeln('<error>安装失败: ' . $e->getMessage() . '</error>');
            throw $e;
        }
    }
}
```

## ORM 数据库操作

### 重要说明
- `save()` 方法：直接调用即可，内部已包含数据库执行逻辑，**不需要**链式调用 `fetch()`
- `delete()` 方法：需要链式调用 `fetch()` 来执行删除操作
- `update()` 方法：需要链式调用 `fetch()` 来执行更新操作
- `insert()` 方法：需要链式调用 `fetch()` 来执行插入操作

### 正确用法示例
```php
// 保存数据 - 正确
$model->setData($data);
$result = $model->save(); // 直接调用，不需要 ->fetch()

// 删除数据 - 正确
$model->where('id', $id)->delete()->fetch();

// 更新数据 - 正确  
$model->where('id', $id)->update($data)->fetch();

// 插入数据 - 正确
$model->insert($data)->fetch();
```

## w-delete 组件使用指南

### 概述
`w-delete` 是 WelineFramework 提供的智能删除确认组件，支持自动定位、自适应弹出方向，提供优雅的用户交互体验。

### 基本用法

#### 1. 引入组件
在页面底部引入 `w-delete` 组件：
```html
<js:part name="w-delete"/>
```

#### 2. 基础删除按钮
```html
<a href="/api/delete/item/123" 
   w-delete="true"
   w-msg="确定要删除这个项目吗？"
   class="btn btn-danger">
   删除
</a>
```

#### 3. 使用 w-delete="1"
```html
<button w-delete="1"
        w-url="/api/delete/item/123"
        w-msg="确定要删除这个项目吗？"
        class="btn btn-outline-danger">
   删除项目
</button>
```

### 属性说明

| 属性 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `w-delete` | `"true"` \| `"1"` | ✅ | 启用删除功能 |
| `w-msg` | `string` | ❌ | 自定义确认消息，默认为"确认删除吗？" |
| `w-url` | `string` | ❌ | 删除请求的URL（优先级：w-url > w-ajax > href） |
| `w-ajax` | `string` | ❌ | AJAX请求URL |
| `w-method` | `string` | ❌ | HTTP方法，默认为"DELETE" |
| `w-var-*` | `string` | ❌ | 额外的POST参数 |

### 后端控制器实现

#### 支持 JSON 和表单数据
由于 `w-delete` 组件在 POST 请求时发送 JSON 数据，后端控制器需要使用 `getParams()` 方法同时支持 JSON 和表单数据：

```php
public function postDelete(): string
{
    // 支持 JSON 和表单数据
    $params = $this->request->getParams();
    $id = (int)($params['id'] ?? 0);

    if (!$id) {
        return $this->jsonResponse([
            'success' => false,
            'message' => __('ID不能为空')
        ]);
    }

    try {
        $model = $this->getModel(YourModel::class)->reset()->load($id);
        
        if (!$model->getId()) {
            return $this->jsonResponse([
                'success' => false,
                'message' => __('记录不存在')
            ]);
        }

        $model->delete()->fetch();

        return $this->jsonResponse([
            'success' => true,
            'message' => __('删除成功')
        ]);
    } catch (\Exception $e) {
        return $this->jsonResponse([
            'success' => false,
            'message' => __('删除失败：%{1}', $e->getMessage())
        ]);
    }
}
```

### 注意事项

1. **URL优先级**：`w-url` > `w-ajax` > `href`
2. **事件委托**：组件使用事件委托，支持动态添加的元素
3. **自动移除**：删除成功后会自动移除对应的表格行（如果存在）
4. **错误处理**：网络错误或服务器错误会显示相应的错误消息
5. **国际化**：确认和取消按钮支持 `__()` 函数进行国际化
6. **后端支持**：必须使用 `$this->request->getParams()` 同时支持 JSON 和表单数据
7. **HTTP方法**：POST 请求时组件发送 JSON 数据，需要后端相应处理
