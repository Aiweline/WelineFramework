# Weline_Cdn 模块实现方案（Cloudflare / API Token / Cache Rules）

### 目标

- 默认适配器为 Cloudflare（API Token 认证）。
- 支持规则管理（本地编辑、导入 Cloudflare、默认规则继承）。
- 提供事件 `Weline_Cdn::clear` 触发缓存清理（支持 Cloudflare 全部清理模式）。
- 使用 `Weline\SystemConfig\Model\SystemConfig` 持久化模块配置与域名列表/规则。

### 目录与文件清单

- `app/code/Weline/Cdn/register.php`：注册模块，依赖 `Weline_Framework`, `Weline_SystemConfig`。
- `app/code/Weline/Cdn/etc/event.xml`：声明事件监听，监听 `Weline_Cdn::clear` -> `Weline\Cdn\Observer\Clear`。
- `app/code/Weline/Cdn/etc/default-rules.json`：模块全局默认 Cache Rules；域名默认继承。
- `app/code/Weline/Cdn/Api/AdapterInterface.php`：适配器接口（清理/规则同步/导入）。
- `app/code/Weline/Cdn/Model/Adapter/Cloudflare.php`：Cloudflare 实现（API Token）。
- `app/code/Weline/Cdn/Model/DomainConfig.php`：域名与规则的读取/保存（封装 SystemConfig）。
- `app/code/Weline/Cdn/Service/AdapterResolver.php`：适配器解析与发现（默认、文件、事件、扫描）。
- `app/code/Weline/Cdn/Service/RuleManager.php`：全局/域名规则合并、推送、导入。
- `app/code/Weline/Cdn/Service/CachePurger.php`：统一封装清理模式并调用适配器。
- `app/code/Weline/Cdn/Observer/Clear.php`：事件处理器，解析事件数据并调用 `CachePurger`。
- `app/code/Weline/Cdn/Controller/Backend/Domain/Index.php|Save.php|Delete.php`：后台域名管理（含适配器下拉选择）。
- `app/code/Weline/Cdn/Controller/Backend/Rules/Index.php|Import.php|Sync.php`：后台规则管理（编辑、导入、同步到 Cloudflare）。
- `app/code/Weline/Cdn/Console/Cdn/Domain/Add.php`：添加域名并推送默认规则（支持选择适配器）。
- `app/code/Weline/Cdn/Console/Cdn/Rules/Import.php`：从 Cloudflare 导入域名规则->本地存储。
- `app/code/Weline/Cdn/Console/Cdn/Cache/Clear.php`：命令行触发清理，等价于派发事件。
- `app/code/Weline/Cdn/doc/README.md`：使用说明与事件数据结构定义。

### 配置键（SystemConfig）

- 模块名：`Weline_Cdn`
- 区域：`backend`（账户/敏感信息）、`frontend`（公开只读项）
- 键建议：
  - `auth/api_token`：Cloudflare API Token（backend）。
  - `domains`：JSON 数组，元素：`{domain, zone_id, enabled, adapter, rules_override?}`（backend）。
  - `rules/global`：缓存全局合并结果缓存（backend，可选）。

### 适配器接口与Cloudflare实现

- 接口：`Api/AdapterInterface`
  - `purgeEverything(zoneId)` / `purgeUrls(zoneId, urls)` / `purgeHosts(zoneId, hosts)` / `purgeTags(zoneId, tags)` / `purgeCacheKeys(zoneId, keys)`
  - `getRules(zoneId)` / `putRules(zoneId, rules)` / `ensureZone(domain)`（可选）
- 默认实现：`Model/Adapter/Cloudflare.php`（API Token；`Authorization: Bearer <token>` 调 Cloudflare v4 API）

### 适配器扩展与扫描

- 解析器：`Service/AdapterResolver.php`
  - 内建：`cloudflare` -> `Model\Adapter\Cloudflare`
  - 文件：加载 `etc/cdn_adapters.php` 返回 `[code => class]`
  - 事件：派发 `Weline_Cdn::adapters_collect`，其他模块返回 `[code => class]`
  - 扫描（仅激活模块）：仅扫描“已激活模块”的路径下，按命名 `*\\Cdn\\Adapter\\*` 枚举类并反射校验实现 `AdapterInterface`
  - 冲突：按优先级合并（文件 < 事件 < 扫描）；非接口实现丢弃；结果缓存（内存/可选落盘）
- 使用：`RuleManager`/`CachePurger` 通过 `AdapterResolver->get($code)` 获取；未指定则 `cloudflare`。

### 域名与适配器关联

- 新建域名时：从 `AdapterResolver->list()` 生成下拉框，选择 `adapter` 并与域名一并保存到 `domains`。
- 控制台：`cdn:domain:add --adapter=cloudflare`；缺省使用 `cloudflare`。
- 执行时：`RuleManager`/`CachePurger` 读取域名的 `adapter` 字段，按域名绑定的适配器执行规则同步与缓存清理。

### 规则与继承

- 加载顺序：`等/default-rules.json` -> 模块全局调整（可选）-> 域名 `rules_override`。
- 新增域名：合并规则并 `putRules(zone_id, rules)` 立即推送。
- 导入：`getRules(zone_id)` 保存为域名的 `rules_override`（或单独的 `rules_imported` 字段）。

### 事件与数据契约

- 事件名：`Weline_Cdn::clear`
- 事件数据（数组或对象）：
  - `mode`: `everything|urls|hosts|tags|cache_keys`
  - `domains`: `string[]` 可选，缺省则对全部启用且可用的域名作用
  - `urls|hosts|tags|cache_keys`: 对应数组，按 Cloudflare API 传参
- 处理流程：Observer 校验 -> 归集 zone 与 adapter -> 逐域名调用对应适配器 -> 聚合结果回填事件对象。

### 配置校验与友好错误

- 配置校验：`DomainConfig` 提供 `requireApiToken()`，缺失时抛出 `CdnConfigMissingException`（继承框架异常），消息包含“请在后台配置 Cloudflare API Token（路径：Cdn 配置）或使用命令行设置”。
- 适配器校验：`Cloudflare` 构造或首次调用前校验 Token；缺失时返回带解决建议的错误对象（控制器/控制台统一转换为友好提示）。
- 控制台：捕获配置异常，打印可执行示例：`php bin/w system:config:set Weline_Cdn auth/api_token <token> backend`。
- 后台：`Controller/Backend/*` 在未配置状态显示高亮告警 Banner，并提供跳转到配置页/导入规则入口。
- 事件：`Observer/Clear` 捕获配置异常，设置事件结果 `success=false` 与 `message`，不中断其他域名处理。

### Console 命令（示例）

- `php bin/w cdn:domain:add --domain=example.com --zone=<zone_id> --adapter=cloudflare`：写入 `domains` 并推送默认规则。
- `php bin/w cdn:rules:import --domain=example.com`：拉取 Cloudflare 规则到本地。
- `php bin/w cdn:cache:clear --mode=everything --domain=example.com`：等价派发事件。

### 安全与边界

- API Token 仅存 `backend` 区域；严格不在日志中回显。
- 所有外呼加超时与错误处理；失败不影响其他域名循环执行。

### 关键片段（示例）

- 事件监听（`etc/event.xml`）
```1:9:app/code/Weline/Cdn/etc/event.xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Cdn::clear">
        <observer name="Weline_Cdn::clear"
                  instance="Weline\Cdn\Observer\Clear" shared="true" sort="0"/>
    </event>
</config>
```

- 事件派发（他模块调用）
```php
$eventsManager->dispatch('Weline_Cdn::clear', [
  'mode' => 'urls',
  'domains' => ['example.com'],
  'urls' => ['https://example.com/a.css']
]);
```
