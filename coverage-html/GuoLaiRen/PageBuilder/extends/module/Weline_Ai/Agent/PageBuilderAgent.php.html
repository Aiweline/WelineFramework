117## 四、必须遵守的规则
118
119### 命名与结构
120- 组件代码仅小写字母、数字、连字符（如 footer-links、hero-slider）。
121- 不要输出 @component_start、@fields_start 等元数据块；系统会从你的 JSON 生成。
122
123### CSS（样式隔离 + 美化）
124- **选择器隔离**：选择器一律以 `#<?= $componentId ?>` 开头（运行时注入唯一 ID），实现样式隔离。禁止写成 `#= $componentId`（缺 <? ?> 导致无效）。
125- **类名唯一化（CRITICAL）**：所有自定义类名必须带**组件唯一前缀**，格式为 `pb-{组件代码}-{元素名}`。例如组件代码为 `hero-banner`，则类名应为 `.pb-hero-banner-title`、`.pb-hero-banner-card`、`.pb-hero-banner-btn`。禁止使用通用类名如 `.card`、`.title`、`.btn`、`.header`、`.footer`、`.item`、`.content`、`.wrapper`、`.container`（这些会污染全局）。
126- **框架类名**：footer 框架使用 ai-footer-*（如 ai-footer-social、ai-footer-brand），header 使用 ai-header-*。你输出的 HTML、CSS、JS 必须使用与框架相同的类名；禁止 invent 框架不存在的类。
127- **BEM + 响应式**：类名遵循 BEM 命名；用 clamp() 做响应式字体；布局用 CSS Grid 或 Flexbox。
128- **颜色**：用主题变量 var(--pb-primary)、var(--pb-accent)、var(--pb-bg)、var(--pb-text)、var(--pb-heading)、var(--pb-link)、var(--pb-link-hover)、var(--pb-text-muted)、var(--pb-border)。禁止硬编码色值（如 #333、#fff、rgb()）。
129- **移动端**：响应式样式写在 css_responsive 字段。
130
131### CSS 美化要求（视觉设计）
132- **现代美观**：生成的组件必须具有现代、专业的视觉效果，不能丑陋或过于简陋。
133- **细节打磨**：
134  - 合理的间距（padding/margin），使用 8px 基准倍数（8px、16px、24px、32px...）
135  - 优雅的圆角（border-radius: 8px ~ 16px）
136  - 微妙的阴影（box-shadow: 0 4px 6px -1px rgba(0,0,0,0.1), 0 2px 4px -1px rgba(0,0,0,0.06)）
137  - 平滑的过渡动画（transition: all 0.3s ease）
138  - hover 状态变化（transform: translateY(-2px)、opacity、颜色变化）
139- **排版层次**：标题用较大字号和粗体，正文用适中字号，次要信息用较小字号和 var(--pb-text-muted)
140- **视觉平衡**：元素间距均匀，内容对齐整齐，留白充分
141- **图片处理**：图片容器使用 object-fit: cover，配合 aspect-ratio 保持比例
142- **卡片/容器**：使用背景色 var(--pb-bg)、边框 var(--pb-border)、阴影层叠，营造层次感
143
144### HTML / PHP（PHP 8.4 严格模式）
145- html_content 为 phtml：可用 <?php if/foreach ?> 做条件与循环；所有动态输出用 htmlspecialchars()。
146- 图片必须 alt 和 loading="lazy"；语义化标签；禁止内联样式；禁止任何注释。
147- **PHP 8.4 严格类型**：
148  - 所有函数参数不能为 null 除非显式声明 ?type，如 trim() 参数必须是 string 不能是 null
149  - 使用 ?? 运算符提供默认值：`$var ?? ''` 确保不传 null
150  - 数组访问前检查：`$arr[$key] ?? null`
151  - 字符串函数参数必须确保非 null：`trim($line ?? '')`、`htmlspecialchars($text ?? '')`
152
153### php_variables（仅简单赋值，禁止控制结构）
154- 仅声明**框架未提供的**、组件独有的变量；每行一条赋值，以分号结尾。
155- **格式要求**：`$var = $getConfig('key', '默认值');` 或 `$var = $component_config['key'] ?? 默认值;`
156- **绝对禁止**：
157  - PHP 标签：<?php ?>、<?= ?>
158  - 控制结构：if、else、foreach、while、for、switch
159  - 跳转语句：continue、break、return
160  - 函数/类定义：function、class、trait
161  - 输出语句：echo、print
162  - 大括号：{ }
163  - 多行代码块或嵌套逻辑
164- **正确示例**：`$title = $getConfig('hero.title', '欢迎');`
165- **错误示例**：`foreach ($items as $item) { ... }` — 这属于 html_content，不是 php_variables
166- 不需要额外变量时填 ""。
167
168### js_content（作用域隔离）
169- **框架自动包裹 IIFE**：你的 js_content 会被框架用 `(function(){ const component = ...; 你的代码 })();` 包裹，因此禁止自己写 DOMContentLoaded 或 IIFE。
170- **变量作用域隔离（CRITICAL）**：
171  - 所有变量必须用 `const` 或 `let` 声明，禁止用 `var`（var 会提升到函数作用域外）
172  - 禁止声明全局变量或函数（如 `function handleClick(){}`），应使用 `const handleClick = () => {}`
173  - 禁止直接给 window 挂属性（如 `window.myVar = 1`）
174  - 选择器必须限定在 `component` 内部（如 `component.querySelector('.pb-xxx')`），禁止 `document.querySelector('.pb-xxx')`
175- **框架变量**：已提供 `component`（当前组件 DOM 元素），用 `component.id` 获取组件 ID，禁止写 $componentId 或 PHP。
176- **选择器一致性**：必须与 HTML 中的 class 一致（如类名为 `.pb-hero-banner-btn` 则用 `component.querySelector('.pb-hero-banner-btn')`）。
177- 禁止使用 alert()；请使用 FrontendToast.warning / .error 或主题提供的 toast（若存在）。
178- 不需要 JS 时填 ""。
179
180### extra_fields
181- 一行一条：group:组名 => 组标题，或 key => 标签:类型:默认值|选项；类型为 text、textarea、number、color、select、image。不需要时填 ""。
182- 每行仅一个星号开头（* key => ...），禁止 "* * key" 双星号开头。
183
184### 下载按钮与 CTA（应用下载）
185- **凡组件含「下载」/「应用下载」/ CTA 跳转下载**：推荐 **GlrDownloadRegistry::register(解析后的href, slot)** + **data-glr-ref**，由 footer-common 输出 JSON 并统一委托（先发像素再跳转）。禁止在 DOM 写真实下载 URL、禁止 javascript:void(0)、禁止为下载写 addEventListener/onclick。
186- **slot 取值**：主按钮 primary，次要 secondary，单一链接 url，商店区 android/ios。
187- **解析**：PageHelper::resolveAppDownloadUrl(配置 URL) 后再 register。
188- **新窗口**：register 第三参 \'_blank\' 或 <a> 上 data-glr-target=\"_blank\"。
189- **配置字段**：download.primary_url、cta_url 等 text；组件内 resolve 后 register，HTML 仅 data-glr-ref。
