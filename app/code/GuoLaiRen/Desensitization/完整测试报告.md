## 修复记录

### 1. 框架级 URL 解析问题 ✅
**文件**: `app/code/Weline/Framework/Http/Url.php`  
**问题**: `parse_url()` 在解析失败时返回 `bool`，与声明类型 `array|string` 不兼容  
**修复**: 在第420-421行添加类型检查
```php
$parsed = parse_url($url);
self::$parserUrlCache[$url] = is_array($parsed) ? $parsed : [];
```

### 2. 模板文件引用错误 ✅
**文件**: 6个模板文件  
**问题**: 所有模板引用了不存在的 `bottom.phtml`  
**修复**: 将引用改为 `footer.phtml`
- `view/templates/Backend/Rule/index.phtml`
- `view/templates/Backend/Config/index.phtml`
- `view/templates/Backend/Log/index.phtml`
- `view/templates/Backend/Test/index.phtml`
- `view/templates/Backend/Detect/index.phtml`
- `view/templates/Backend/Rewrite/index.phtml`

### 3. 控制器依赖注入问题 ✅
**文件**: `Controller/Backend/Config.php`  
**问题**: 使用构造函数依赖注入导致框架无法实例化  
**修复**: 移除构造函数，在方法内部使用 `ObjectManager::getInstance()` 动态获取实例

### 4. 服务层数组类型问题 ✅
**文件**: `Service/DesensitizationService.php`  
**问题**: `getRules()` 方法的 `array_map()` 遇到非数组类型  
**修复**: 添加类型检查和对象转换逻辑
