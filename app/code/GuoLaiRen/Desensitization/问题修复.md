# 问题修复记录

## 问题：guolairen_desensitization/backend/rule 报错

### 问题原因
1. **缺少 module.xml 文件** - 模块配置文件缺失
2. **路由别名未配置** - env.php 中缺少路由别名配置

### 修复方案

#### 1. 创建 module.xml 文件
**位置**：`app/code/GuoLaiRen/Desensitization/etc/module.xml`

**内容**：
```xml
<?xml version="1.0"?>
<config xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:noNamespaceSchemaLocation="urn:weline:framework:module:etc/module.xsd">
    <module name="GuoLaiRen_Desensitization" setup_version="1.0.0">
        <sequence>
            <module name="Weline_Framework"/>
            <module name="Weline_Backend"/>
            <module name="Weline_Ai"/>
        </sequence>
    </module>
</config>
```

#### 2. 更新 env.php 配置
在 `app/code/GuoLaiRen/Desensitization/etc/env.php` 中添加路由别名配置：

```php
return [
    // 模块路由别名配置
    'router' => [
        'alias' => 'desensitization',
        'backend_alias' => 'guolairen_desensitization',
    ],
    
    'desensitization' => [
        // ... 其他配置
    ]
];
```

### 为什么不需要 routes.xml？

根据 Weline Framework 的约定：
1. **后台 PC 控制器**会自动映射，不需要在 `routes.xml` 中定义
2. 框架会根据控制器的文件路径自动生成路由：
   - `Controller/Backend/Rule.php` → `/guolairen_desensitization/backend/rule/*`
   - `Controller/Backend/Log.php` → `/guolairen_desensitization/backend/log/*`

3. **路由别名**由 `etc/env.php` 中的 `router` 配置决定
4. **API 路由**才需要在 `routes.xml` 中定义（如果需要）

### 注意事项

1. **模块命名**：确保 `module.xml` 中的模块名与目录名一致
2. **路由别名**：`backend_alias` 要与访问路径一致
3. **依赖关系**：`sequence` 中必须包含所有依赖的模块
4. **版本号**：`setup_version` 用于控制数据库升级

### 如果还有问题

1. 检查模块是否已启用：`php bin/w module:status GuoLaiRen_Desensitization`
2. 检查路由是否注册：查看 `generated/routers/backend_pc.php`
3. 检查控制器是否存在：确认 `Controller/Backend/Rule.php` 存在
4. 检查日志：查看错误日志了解详细信息
