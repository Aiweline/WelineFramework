# 控制器问题修复说明

## 问题原因

所有控制器在构造函数中使用依赖注入，但由于：
1. 移除了属性的声明
2. 但没有移除所有使用这些属性的地方

导致控制器无法正常工作。

## 解决方案

需要在每个使用 `$this->ruleModel` 或 `$this->service` 的地方改为动态获取实例。

## 修复模式

### 原来：
```php
class Rule extends BackendController
{
    private DesensitizationRule $ruleModel;

    public function __construct(DesensitizationRule $ruleModel) {
        $this->ruleModel = $ruleModel;
    }

    public function add() {
        $this->ruleModel->reset(); // ❌ 属性不存在
    }
}
```

### 修复后：
```php
class Rule extends BackendController
{
    // 不需要属性声明

    public function add() {
        $ruleModel = \Weline\Framework\Manager\ObjectManager::getInstance(DesensitizationRule::class);
        $ruleModel->reset(); // ✅ 动态获取
    }
}
```
