## 问题修复记录

### 1. 框架级 URL 解析问题
**问题**: `Weline\Framework\Http\Url::parse_url()` 在解析失败时返回 `bool`，但声明返回 `array|string`

**修复**: 修改 `app/code/Weline/Framework/Http/Url.php`，在第420-421行添加类型检查：
```php
$parsed = parse_url($url);
self::$parserUrlCache[$url] = is_array($parsed) ? $parsed : [];
```

### 2. 模板文件引用错误
**问题**: 所有模板文件引用了不存在的 `Weline_Admin::common/bottom.phtml`

**修复**: 将以下文件中的引用从 `bottom.phtml` 改为 `footer.phtml`：
- `app/code/GuoLaiRen/Desensitization/view/templates/Backend/Rule/index.phtml`
- `app/code/GuoLaiRen/Desensitization/view/templates/Backend/Config/index.phtml`
- `app/code/GuoLaiRen/Desensitization/view/templates/Backend/Log/index.phtml`
- `app/code/GuoLaiRen/Desensitization/view/templates/Backend/Test/index.phtml`
- `app/code/GuoLaiRen/Desensitization/view/templates/Backend/Detect/index.phtml`
- `app/cUIKit/Desensitization/view/templates/Backend/Rewrite/index.phtml`

### 3. 控制器依赖注入问题
**问题**: `Config` 控制器使用了构造函数依赖注入，导致框架无法实例化

**修复**: 移除构造函数，在方法内部使用 `ObjectManager::getInstance()` 动态获取实例
