# 卡片式样式选择器

## 功能概述

页面编辑表单现在使用全新的卡片式样式选择器，提供更直观的可视化体验。用户可以通过卡片预览、搜索过滤等方式快速选择合适的页面样式模板。

## 主要特性

### 1. 📸 智能预览系统

样式卡片支持三种预览模式：

#### 预览图模式（优先）
- 如果样式目录下存在 `preview.webp` 或 `preview.png` 文件
- 直接显示预览图片
- 加载快速，UI响应流畅
- 卡片右下角显示 ✓ 图标

#### Canvas截图预览模式
- 当样式没有预览图时
- 使用iframe加载该页面应用该样式的实时预览（后台隐藏加载）
- 使用html2canvas技术将页面截图并转换为canvas显示
- 自动传递当前页面ID和临时样式代码
- 显示加载动画，截图完成后显示canvas预览
- 截图完成后自动移除iframe，释放资源
- 卡片右下角显示 🖥️ 图标

#### 占位符模式
- 新建页面且样式无预览图时使用
- 显示调色板图标和"无预览"文字
- 避免空白区域影响用户体验

### 2. 🔍 搜索过滤功能

- **实时搜索**：输入时自动过滤，300ms防抖优化性能
- **多字段匹配**：同时搜索样式名称、描述和代码
- **大小写不敏感**：用户体验更友好
- **即时反馈**：无结果时显示友好提示信息

### 3. 🎨 直观的卡片式UI

#### 卡片布局
- **响应式网格**：
  - 手机：1列
  - 平板：2列
  - PC：3列
- **统一高度**：所有卡片高度一致，整齐美观
- **卡片内容**：
  - 顶部：预览区域（高度200px）
  - 中部：样式名称 + 选中徽章
  - 描述：最多显示2行，超出显示省略号
  - 底部：样式代码 + 预览类型图标

#### 交互效果
- **悬停效果**：
  - 边框变为主题色（#667eea）
  - 轻微上浮（2px）
  - 显示阴影
- **选中状态**：
  - 边框高亮（主题色）
  - 背景色变浅蓝（#f8f9ff）
  - 显示"✓ 已选择"徽章
  - 增强阴影效果
- **点击选择**：点击卡片任意位置即可选择该样式

### 4. ⚡ 自动保存与刷新

- **编辑模式**：切换样式时自动保存到服务器
- **页面刷新**：保存成功后自动刷新页面（500ms延迟）
- **即时生效**：刷新后新样式立即应用到整个表单
- **错误处理**：保存失败时显示错误信息并恢复原样式
- **用户反馈**：显示加载状态、成功提示和刷新提示

### 5. 🔄 刷新功能

- **手动刷新**：点击搜索框右侧的刷新按钮
- **重新扫描**：实时扫描样式目录，获取最新样式列表
- **加载动画**：刷新按钮显示旋转图标
- **保持状态**：刷新后保持当前选中的样式

## 技术实现

### 前端架构

```javascript
// 核心函数
loadStyleList()                         // 加载样式列表（AJAX）
renderStyleCards()                      // 渲染样式卡片
selectStyle(styleCode)                  // 选择样式
filterStyles(searchTerm)                // 搜索过滤
autoSaveStyleChange()                   // 自动保存（编辑模式）
captureIframePreview(iframe, canvas)    // 截取iframe内容转canvas（新增）
```

### 后端API

```php
// 控制器方法
Page::styles()                // 获取样式列表（JSON）
Page::stylePreviewImage()     // 获取样式预览图片（二进制流）
Page::styleConfig()           // 获取样式配置
Preview::full()               // 完整页面预览（支持临时样式切换）
```

### 数据流

1. **页面加载** → `DOMContentLoaded` → `loadStyleList()`
2. **AJAX请求** → `/pagebuilder/backend/page/styles`
3. **接收数据** → 保存到 `allStyles` 全局变量
4. **渲染卡片** → `renderStyleCards()` → 动态创建DOM
5. **加载预览**：
   - **有预览图** → 通过 `stylePreviewImage?code=xxx` 获取图片
   - **无预览图** → 创建隐藏iframe → 加载 `preview/full?page_id=xxx&style_code=yyy`
   - **Canvas截图** → iframe加载完成 → `captureIframePreview()` → html2canvas → 显示canvas
6. **用户交互** → 点击卡片 → `selectStyle()` → 更新状态 + 加载配置
7. **自动保存** → `autoSaveStyleChange()` → POST请求 → 更新服务器

### 预览图获取机制

预览图不再直接访问静态文件路径，而是通过后台控制器提供：

**优势**：
- ✅ **权限控制**：需要后台登录权限才能访问
- ✅ **错误处理**：统一处理文件不存在等情况
- ✅ **缓存优化**：支持HTTP缓存（ETag、Last-Modified）
- ✅ **路径安全**：避免直接暴露文件系统路径

**请求示例**：
```
GET /pagebuilder/backend/page/stylePreviewImage?code=marketing-landing
```

**响应头**：
```
Content-Type: image/webp
Content-Length: 12345
Last-Modified: Sun, 19 Oct 2025 10:00:00 GMT
ETag: "abc123def456"
Cache-Control: public, max-age=604800
```

### Canvas截图预览技术

当样式没有预览图时，系统使用以下技术生成实时预览：

#### 工作流程

1. **创建隐藏iframe**
   - 位置：`position: absolute; left: -9999px`
   - 尺寸：1200x800（标准桌面尺寸）
   - URL：`/pagebuilder/backend/preview/full?page_id=xxx&style_code=yyy`

2. **加载完整页面**
   - 使用 `Preview::full()` 方法
   - 支持 `style_code` 参数临时切换样式
   - 应用样式模板的默认配置

3. **html2canvas截图**
   - 等待iframe加载完成（1秒延迟）
   - 使用html2canvas库截取iframe的body
   - 配置：`scale: 0.25`（降低分辨率提高性能）

4. **Canvas渲染**
   - 目标尺寸：400x200（卡片尺寸）
   - 保持宽高比例缩放
   - 居中显示截图内容

5. **清理资源**
   - 显示canvas预览
   - 移除隐藏的iframe
   - 释放内存资源

#### 技术优势

- ✅ **真实预览**：显示页面实际样式效果
- ✅ **自动生成**：无需手动创建预览图
- ✅ **即时更新**：样式修改后自动反映
- ✅ **性能优化**：降低截图分辨率，截图后释放iframe
- ✅ **错误处理**：截图失败时显示友好提示

#### 技术限制

- ⚠️ **加载时间**：首次加载需要1-3秒（包含页面渲染+截图）
- ⚠️ **跨域限制**：iframe内容必须同域（已满足）
- ⚠️ **内存占用**：多个样式同时截图会占用较多内存
- ⚠️ **浏览器兼容**：需要支持html2canvas（现代浏览器均支持）

#### 依赖库

```html
<script src="https://cdn.jsdelivr.net/npm/html2canvas@1.4.1/dist/html2canvas.min.js"></script>
```

### CSS架构

- **BEM命名规范**：`.style-card`, `.style-card-preview`, `.style-card-body`

## 样式预览配置

### 添加预览图

在样式目录下添加预览图文件（二选一）：

```
app/code/GuoLaiRen/PageBuilder/view/templates/style/
├── marketing-landing/
│   ├── header.phtml
│   ├── content.phtml
│   ├── footer.phtml
│   └── preview.webp         ← 推荐（更小的文件）
└── default/
    ├── header.phtml
    ├── content.phtml
    ├── footer.phtml
    └── preview.png          ← 备选格式
```

**预览图要求**：
- **格式**：webp（推荐）或 png
- **尺寸**：建议宽度1200px以上（响应式缩放）
- **宽高比**：建议16:9或4:3
- **文件大小**：建议小于500KB

### 预览图自动检测

系统会自动扫描并检测预览图：
1. 运行 `Style::autoScan()` 方法扫描样式目录
2. 检测 `preview.webp`（优先）
3. 如果不存在，检测 `preview.png`
4. 将路径保存到数据库 `preview_image` 字段

## 使用说明

### 基本操作

1. **查看样式列表**
   - 打开页面编辑表单
   - 样式卡片自动加载并显示

2. **搜索样式**
   - 在搜索框中输入关键词
   - 实时过滤匹配的样式
   - 清空搜索框显示所有样式

3. **选择样式**
   - 点击任意卡片
   - 卡片高亮显示选中状态
   - 编辑模式：自动保存样式到服务器
   - 保存成功后自动刷新页面（500ms延迟）
   - 新建模式：仅更新表单，不会立即保存

4. **刷新样式**
   - 点击搜索框右侧的刷新按钮
   - 系统重新扫描样式目录
   - 更新显示最新样式列表

### 编辑模式特性

当编辑现有页面时：
- 切换样式会**自动保存**到服务器
- 保存成功后显示提示消息
- **自动刷新页面**（500ms延迟）
- 刷新后新样式完全应用
- 样式配置和预览同步更新
- 支持撤销（切换回原样式卡片即可）

### 新建模式特性

当新建页面时：
- 选择样式不会立即保存
- 需要填写完整表单后提交
- 没有iframe实时预览（因为页面还不存在）
- 建议样式添加预览图以提供更好体验

## 故障排除

### 问题：卡片显示空白

**可能原因**：
1. AJAX请求失败
2. 样式数据格式错误

**解决方案**：
1. 打开浏览器开发者工具查看Console日志
2. 检查Network标签中的AJAX请求状态
3. 点击刷新按钮重新加载

### 问题：预览图不显示

**可能原因**：
1. 图片文件不存在
2. 图片路径错误
3. 文件权限问题

**解决方案**：
1. 检查样式目录下是否有 `preview.webp` 或 `preview.png`
2. 运行样式扫描：`php bin/w style:scan`
3. 检查文件权限是否可读

### 问题：iframe预览加载慢

**可能原因**：
1. 页面内容较多
2. 样式配置复杂
3. 网络延迟

**优化建议**：
1. **推荐添加预览图**：图片预览比iframe快得多
2. 优化样式模板性能
3. 减少外部资源依赖

### 问题：搜索无结果

**可能原因**：
1. 关键词拼写错误
2. 样式名称/描述不匹配

**解决方案**：
1. 检查搜索词拼写
2. 尝试更通用的关键词
3. 清空搜索框显示所有样式

## 性能优化

### 已实现的优化

1. **搜索防抖**：300ms延迟，避免频繁渲染
2. **图片优先**：webp格式支持，减小文件大小
3. **按需加载**：iframe仅在需要时创建
4. **样式缓存**：全局 `allStyles` 变量避免重复请求
5. **事件委托**：减少事件监听器数量

### 优化建议

1. **为所有样式添加预览图**：避免使用iframe预览
2. **优化预览图大小**：使用webp格式，压缩图片
3. **限制iframe数量**：考虑懒加载或虚拟滚动
4. **CDN加速**：将预览图放到CDN服务器

## API 文档

### 1. 获取样式列表

**接口**：`GET /pagebuilder/backend/page/styles`

**权限**：`GuoLaiRen_PageBuilder::page_builder_styles`

**响应**：
```json
{
  "success": true,
  "count": 2,
  "data": [
    {
      "id": 1,
      "code": "marketing-landing",
      "name": "营销落地页",
      "description": "适合营销活动的落地页样式",
      "preview_image": "style/marketing-landing/preview.webp",
      "path": "style/marketing-landing",
      "is_active": 1,
      "sort_order": 10
    },
    {
      "id": 2,
      "code": "default",
      "name": "默认样式",
      "description": "简洁的默认页面样式",
      "preview_image": "",
      "path": "style/default",
      "is_active": 1,
      "sort_order": 20
    }
  ]
}
```

### 2. 获取预览图片

**接口**：`GET /pagebuilder/backend/page/stylePreviewImage?code={styleCode}`

**权限**：`GuoLaiRen_PageBuilder::style_preview_image`

**参数**：
- `code` (必需) - 样式代码

**响应**：
- **成功**：返回图片二进制流（image/webp、image/png等）
- **失败**：
  - `400 Bad Request` - 缺少样式代码参数
  - `404 Not Found` - 样式不存在或没有预览图
  - `500 Internal Server Error` - 服务器错误

**缓存支持**：
- `Cache-Control: public, max-age=604800` (7天)
- `ETag` - 文件内容哈希
- `Last-Modified` - 文件修改时间
- 支持 `304 Not Modified` 响应

**示例**：
```
GET /pagebuilder/backend/page/stylePreviewImage?code=marketing-landing
```

### 3. 获取样式配置

**接口**：`GET /pagebuilder/backend/page/styleConfig?style_code={styleCode}&page_id={pageId}&locale={locale}`

**权限**：`GuoLaiRen_PageBuilder::page_builder_config`

**参数**：
- `style_code` (必需) - 样式代码
- `page_id` (可选) - 页面ID
- `locale` (可选) - 语言代码

**响应**：
```json
{
  "success": true,
  "data": {
    "configs": [...],
    "current_values": {...}
  }
}
```

### 4. 页面预览

**接口**：`GET /pagebuilder/backend/page/preview?id={pageId}&locale={locale}&style={styleCode}`

**权限**：`GuoLaiRen_PageBuilder::page_builder_preview`

**参数**：
- `id` (必需) - 页面ID
- `locale` (可选) - 语言代码
- `style` (可选) - 临时应用的样式代码（用于预览不同样式）

**响应**：重定向到前端页面预览URL
