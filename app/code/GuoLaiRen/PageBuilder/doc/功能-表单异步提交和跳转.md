# Marketing Landing 模板 - 表单异步提交和跳转功能

## 功能概述
将营销落地页表单从传统的同步提交改为 AJAX 异步提交，提交成功后自动跳转到页面配置的跳转地址（`redirect_url`），提供更好的用户体验。

## 开发日期
2024-01-XX

## 功能特性

### 1. 异步提交
- ✅ 使用 Fetch API 进行异步提交
- ✅ 无需页面刷新
- ✅ 实时反馈提交状态
- ✅ 优雅的错误处理

### 2. 提交状态
- **提交中**: 按钮显示 "Submitting..."，禁用状态
- **成功**: 显示成功消息，1秒后自动跳转
- **失败**: 显示错误消息，允许重新提交

### 3. 自动跳转
- 提交成功后自动跳转到页面配置的 `redirect_url`
- 如果未配置跳转地址，默认跳转到首页 "/"
- 跳转前显示1秒的成功提示

## 技术实现

### PHP 后端

#### 获取跳转地址
```php
// 提交成功后的跳转地址
$redirectUrl = $page ? $page->getData('redirect_url') : '';
if (empty($redirectUrl)) {
    $redirectUrl = '/'; // 默认跳转到首页
}
```

**字段来源**: `Page` 模型的 `redirect_url` 字段

##### 2. 按钮状态控制
```javascript
function setButtonLoading(loading) {
    if (loading) {
        submitBtn.disabled = true;
        submitBtn.textContent = 'Submitting...';
        submitBtn.style.opacity = '0.7';
        submitBtn.style.cursor = 'not-allowed';
    } else {
        submitBtn.disabled = false;
        submitBtn.textContent = originalBtnText;
        submitBtn.style.opacity = '1';
        submitBtn.style.cursor = 'pointer';
    }
}
```

##### 3. 消息显示
```javascript
function showMessage(message, type) {
    formMessage.textContent = message;
    formMessage.style.display = 'block';
    
    if (type === 'success') {
        formMessage.style.backgroundColor = '#d4edda';
        formMessage.style.color = '#155724';
        formMessage.style.border = '1px solid #c3e6cb';
    } else if (type === 'error') {
        formMessage.style.backgroundColor = '#f8d7da';
        formMessage.style.color = '#721c24';
        formMessage.style.border = '1px solid #f5c6cb';
    } else if (type === 'loading') {
        formMessage.style.backgroundColor = '#d1ecf1';
        formMessage.style.color = '#0c5460';
        formMessage.style.border = '1px solid #bee5eb';
    }
}
```

##### 4. AJAX 提交
```javascript
fetch(form.action, {
    method: 'POST',
    body: formData,
    headers: {
        'X-Requested-With': 'XMLHttpRequest'
    }
})
.then(response => {
    if (!response.ok) {
        throw new Error('Network response was not ok');
    }
    return response.json();
})
.then(data => {
    setButtonLoading(false);
    
    if (data.success || data.status === 'success') {
        showMessage('Success! Redirecting...', 'success');
        
        // 1秒后跳转
        setTimeout(function() {
            window.location.href = redirectUrl;
        }, 1000);
    } else {
        const errorMsg = data.message || data.error || 'Submission failed. Please try again.';
        showMessage(errorMsg, 'error');
    }
})
.catch(error => {
    console.error('Form submission error:', error);
    setButtonLoading(false);
    showMessage('An error occurred. Please try again later.', 'error');
});
```

## 后端API响应格式

### 控制器: `GuoLaiRen\PageBuilder\Controller\Frontend\Form::submit()`

### 成功响应
```json
{
    "success": true,
    "message": "提交成功！感谢您的关注。",
    "data": {
        "submission_id": 123
    }
}
```

### 失败响应
```json
{
    "success": false,
    "message": "邮箱格式不正确"
}
```

### HTTP状态码
- **200**: 请求成功
- **405**: 非 POST 请求
- **500**: 服务器错误

## 用户体验流程

### 正常流程
1. 用户填写表单（邮箱、电话）
2. 用户勾选同意协议
3. 用户点击提交按钮
4. 按钮变为 "Submitting..." 并禁用
5. 显示蓝色提示："Submitting your information..."
6. 提交成功后显示绿色提示："Success! Redirecting..."
7. 1秒后自动跳转到配置的目标页面

### 错误流程
1. 用户填写表单
2. 用户点击提交按钮
3. 如果验证失败或提交失败
4. 显示红色错误提示（具体错误信息）
5. 按钮恢复可用状态
6. 用户可以修改后重新提交

## 配置说明

### 在后台配置跳转地址

1. 登录后台管理系统
2. 进入 **页面管理** → **编辑页面**
3. 找到 **跳转地址 (Redirect URL)** 字段
4. 输入目标URL，例如：
   - `https://example.com/thank-you` - 跳转到感谢页
   - `https://example.com/special-offer` - 跳转到优惠页
   - `/success` - 跳转到网站内的成功页
   - 留空 - 默认跳转到首页

### 跳转地址格式
- **完整URL**: `https://example.com/page`
- **相对路径**: `/page/path`
- **带参数**: `https://example.com/page?utm_source=form&id=123`

## 常见问题

### Q1: 提交后没有跳转？
**A**: 检查以下几点：
1. 页面是否配置了 `redirect_url`
2. 控制器是否返回 `success: true`
3. 浏览器控制台是否有JavaScript错误
4. 检查跳转URL是否正确

### Q2: 提交失败显示什么？
**A**: 显示后端返回的 `message` 字段内容，例如：
- "请输入邮箱地址"
- "邮箱格式不正确"
- "提交失败，请稍后再试"

### Q3: 可以自定义跳转延迟时间吗？
**A**: 可以，修改 JavaScript 中的延迟时间：
```javascript
setTimeout(function() {
    window.location.href = redirectUrl;
}, 1000); // 1000毫秒 = 1秒
```

### Q4: 如何禁用自动跳转？
**A**: 注释或删除跳转代码：
```javascript
if (data.success || data.status === 'success') {
    showMessage('Success! Thank you.', 'success');
    // 不跳转，只显示成功消息
}
```

### Q5: 能否在跳转时传递额外参数？
**A**: 可以，在后台配置跳转URL时带上参数：
```
https://example.com/thank-you?source=landing&campaign=summer
```

或在JavaScript中动态添加：
```javascript
const finalUrl = redirectUrl + '?submission_id=' + data.data.submission_id;
window.location.href = finalUrl;
```
