# 可视化编辑器配置显示 - 测试指南

## 快速测试步骤

### 1. 启动服务器

```bash
cd e:\WelineFramework\DEV-workspace
php bin/w s:sta
```

记录后端地址（从日志 `var/log/server-start.log` 中获取）。

### 2. 登录后台

访问后端地址，使用管理员账号登录（默认 `admin/admin`）。

### 3. 进入页面编辑

导航路径：**PageBuilder > 页面管理 > 编辑页面**

或者直接访问：`后端地址/pagebuilder/backend/page/edit?id=页面ID`

## 常见问题排查

### Q1: 没有显示响应式图标？

**可能原因**：
1. 配置项的 `responsive` 属性为 `false`
2. 模板文件中配置项定义格式不正确

**解决方法**：
1. 检查模板文件（如 `header.phtml`）中的配置定义
2. 确保使用 `responsive` 类型或包含斜杠的默认值（如 `60/80`）
3. 清理缓存：`php bin/w cache:clear -f`

### Q2: 描述信息不显示？

**可能原因**：
1. 配置项的 `description` 属性为空
2. 模板文件中没有定义描述信息

**解决方法**：
1. 在模板文件的配置定义中添加描述：
   ```php
   size.height => 头部高度:responsive:60/80|px[MD]
   ```
   或
   ```php
   size.max_width => 内容最大宽度:responsive:100/1200|%/px[移动端使用百分比，PC端使用像素]
   ```

### Q3: 图标显示为方块？

**可能原因**：
- Material Design Icons 字体库未加载

**解决方法**：
1. 检查浏览器控制台是否有字体加载错误
2. 确认 Material Design Icons CSS 文件已正确引入

### Q4: 样式不生效？

**可能原因**：
- 浏览器缓存

**解决方法**：
1. 强制刷新浏览器：`Ctrl + Shift + R` (Windows) 或 `Cmd + Shift + R` (Mac)
2. 清理后端缓存：`php bin/w cache:clear -f`

## 测试用例

### 测试用例 1：标准响应式配置（带单位）

**输入**：
```php
size.height => 头部高度:responsive:60/80|px[MD]
```

**预期输出**：
- 标签：`头部高度 📱📱💻`
- 描述：`ℹ️ 支持响应式配置（移动端、PC端）`
- 输入框：`┌──────────┬────┐`
           `│ 60/80    │ px │`
           `└──────────┴────┘`
- 单位 `px` 在输入框右侧，浅灰色背景

### 测试用例 2：自定义描述（混合单位）

**输入**：
```php
size.max_width => 内容最大宽度:responsive:100/1200|%/px[移动端使用百分比，PC端使用像素]
```

**预期输出**：
- 标签：`内容最大宽度 📱📱💻`
- 描述：`ℹ️ 移动端使用百分比，PC端使用像素`
- 输入框：`┌──────────┬──────┐`
           `│ 100/1200 │ %/px │`
           `└──────────┴──────┘`
- 单位 `%/px` 清晰显示，说明不同端使用不同单位

### 测试用例 3：非响应式配置

**输入**：
```php
layout.logo_position => Logo位置:select:left|left,center,right
```

**预期输出**：
- 标签：`Logo位置`（无图标徽章）
- 描述：无
- 输入框：下拉选择框

### 测试用例 4：完整三端支持（含帮助信息）

**输入**：
```php
group:size => 尺寸[MTD]:📱响应式配置说明:支持移动端/平板/PC端三种配置格式\n• 单一值: 80 (所有端相同)\n• 双端值: 60/80 (移动端60, PC端80, 推荐)\n• 三端值: 60/70/80 (移动端60, 平板70, PC端80)\n\n断点定义:\nM (Mobile) 移动端: 375px\nT (Tablet) 平板: 768px\nD (Desktop) PC端: 1280px\n\n标记说明: [M]移动端 [MD]移动+PC [MTD]完整三端
size.padding => 内边距:responsive:16/24/40|px[MTD]
```

**预期输出**：
- 分组标题：`尺寸 📱📱💻`
- 帮助信息框：
  - 标题：`ℹ️ 📱响应式配置说明 ▼`
  - 内容：详细的多行说明（默认收起）
  - 点击可折叠/展开
- 配置标签：`内边距 📱📱💻`
- 配置描述：`ℹ️ 支持响应式配置（移动端、平板、PC端）`

### 测试用例 5：帮助信息折叠功能

**操作**：
1. 展开"头部配置 > 尺寸"分组
2. 观察帮助信息框（应该默认收起）
3. 点击帮助标题区域"📱响应式配置说明"
4. 检查内容是否展开
5. 检查折叠图标是否从 `▼` 变为 `▲`
6. 检查内容格式（换行、列表、缩进）
7. 再次点击，检查是否收起
8. 打开浏览器控制台（F12），检查是否有错误

**预期效果**：
- 帮助信息可以顺畅地展开/收起
- 图标正确切换（▼ ↔ ▲）
- 内容格式保持清晰（多行、列表等）
- 浏览器控制台无错误或警告
- 点击响应迅速，无延迟

### 测试用例 6：单位显示完整性

**测试配置项**：
- 头部高度（`responsive` 类型）→ 应显示 `px`
- 内容最大宽度（`responsive` 类型）→ 应显示 `%/px`
- 左右内边距（`responsive` 类型）→ 应显示 `px`
- Logo位置（`select` 类型）→ 不显示单位
- 背景颜色（`color` 类型）→ 不显示单位

**预期效果**：
- 所有响应式配置都显示单位
- 数字配置显示单位
- 选择器和颜色选择器不显示单位
- 单位文字清晰、对齐良好
