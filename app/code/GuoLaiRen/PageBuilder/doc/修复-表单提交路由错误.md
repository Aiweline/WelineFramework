# Marketing Landing 模板 - 表单提交路由错误修复

## 修复日期
2024-01-XX

## 问题描述
表单提交时返回 **404 Not Found** 错误，提交失败。

## 错误原因

### 错误的路由路径
```php
// ❌ 错误 - 缺少 frontend 路径段
$formAction = $this->getUrl('pagebuilder/form/submit', ['page_id' => $page ? $page->getId() : 0]);
```

生成的URL:
```
/pagebuilder/form/submit?page_id=2
```

### 正确的路由路径
根据生成的路由配置 `generated/routers/frontend_pc.php`，正确的路由是：

```php
'pagebuilder/frontend/form/submit' => 
array (
  'module' => 'GuoLaiRen_PageBuilder',
  'module_path' => 'E:\\WelineFramework\\DEV-workspace\\app\\code\\GuoLaiRen\\PageBuilder\\',
  'router' => 'pagebuilder',
  'class' => 
  array (
    'area' => 'FrontendController',
    'name' => 'GuoLaiRen\\PageBuilder\\Controller\\Frontend\\Form',
    'method' => 'submit',
    'controller_name' => 'Frontend/Form',
    'request_method' => '',
  ),
),
```

**关键点**: 路由路径包含 `frontend` 目录层级

## 修复方案

### 修改代码
```php
// ✅ 正确 - 包含完整的路径
$formAction = $this->getUrl('pagebuilder/frontend/form/submit', ['page_id' => $page ? $page->getId() : 0]);
```

生成的URL:
```
/pagebuilder/frontend/form/submit?page_id=2
```

### 测试验证
```bash
# 测试表单提交路由
php bin/w http:request "pagebuilder/frontend/form/submit?page_id=2" -m=POST -d="email=test@example.com&phone=1234567890&marketing_agree=1"
```

**测试结果**：
```
响应状态码: 200
响应内容: {"success":false,"message":"请输入邮箱地址"}
```

虽然返回验证错误（因为缺少 form_key 等），但 **200 状态码** 证明路由已正确工作！

## 路由命名规则

### Weline 框架路由规则
在 Weline 框架中，前端控制器的路由路径遵循以下规则：

```
{router}/{area}/{controller}/{action}
```

### 示例对照

| 控制器位置 | 路由路径 |
|------------|----------|
| `Controller/Frontend/Form.php` | `pagebuilder/frontend/form/submit` |
| `Controller/Frontend/Page.php` | `pagebuilder/frontend/page/view` |
| `Controller/Backend/Page.php` | `pagebuilder/backend/page/index` |

**规律**：
- `Frontend` 目录 → 路由包含 `frontend`
- `Backend` 目录 → 路由包含 `backend`
- 控制器类名 `Form` → 路由包含 `form`
- 方法名 `submit()` → 路由包含 `submit`

## 知识点总结

### Weline 路由系统

#### 路由生成规则
```
{模块router}/{控制器目录}/{控制器类名}/{方法名}
```

#### 实际示例
```php
// 控制器：GuoLaiRen\PageBuilder\Controller\Frontend\Form::submit()
// 
// 路由组成：
// - 模块router: pagebuilder
// - 控制器目录: frontend
// - 控制器类名: form (小写)
// - 方法名: submit
// 
// 最终路由: pagebuilder/frontend/form/submit
```

#### 常见错误
```php
// ❌ 缺少目录层级
'pagebuilder/form/submit'

// ✅ 完整路径
'pagebuilder/frontend/form/submit'
```

### 路由调试技巧

#### 1. 使用 http:request 命令
```bash
# 快速测试路由是否存在
php bin/w http:request "your/route/path"

# 404 = 路由不存在
# 200/405/500 = 路由存在
```

#### 2. 检查生成的路由文件
```bash
# 前端路由
generated/routers/frontend_pc.php

# 后端路由
generated/routers/backend_pc.php

# API路由
generated/routers/frontend_rest_api.php
generated/routers/backend_rest_api.php
```

#### 3. 搜索路由定义
```bash
# 使用 grep 搜索模块路由
grep -n "YourModule" generated/routers/*.php
```

#### 4. 验证控制器路径
确保控制器文件存在于正确位置：
```
Controller/Frontend/Form.php  → frontend/form
Controller/Backend/Page.php   → backend/page
```

## 预防措施

### 1. 使用正确的URL生成方法
```php
// ✅ 推荐：完整路径
$this->getUrl('moduleRouter/area/controller/action')

// ❌ 避免：省略路径
$this->getUrl('moduleRouter/controller/action')
```

### 2. 参考已有路由
查看 `generated/routers/` 中已注册的路由格式，保持一致。

### 3. 开发时测试
每次添加新路由后，立即使用 `http:request` 命令测试。

### 4. 文档记录
在模块文档中明确记录路由路径，避免混淆。
