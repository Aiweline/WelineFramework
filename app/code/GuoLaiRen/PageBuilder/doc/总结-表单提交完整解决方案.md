# PageBuilder表单提交 - 完整解决方案总结

## 问题描述
用户报告表单提交返回404错误。

## 根本原因

### 1. 路由URL错误 ❌
**原代码**:
```php
$formAction = $this->getUrl('pagebuilder/form/submit', ...);
// 生成: /pagebuilder/form/submit  → 404
```

**修复后**:
```php
$formAction = $this->getUrl('pagebuilder/frontend/form/submit', ...);
// 生成: /pagebuilder/frontend/form/submit  → 200 ✅
```

### 2. 页面未发布 ❌
```
页面ID: 2
Status: 0 (未发布) → 前端访问404
```

**已修复**: 设置 status=1

### 3. Style路径未配置 ❌
```
Style: marketing-landing
Path: (空) → 使用默认模板
```

**已修复**: 设置 path=`GuoLaiRen_PageBuilder::style/marketing-landing`

## 可能的问题和解决方案

### 问题1: 页面显示默认模板
**症状**: 看到简单的标题和内容，没有marketing-landing的样式

**原因**:
- Style路径未配置
- 或模板文件不存在

**解决**:
```bash
php dev/fix-style-path-correct.php
```

### 问题2: 表单提交404
**症状**: 点击提交后返回404

**原因**:
- 路由URL错误
- 浏览器缓存既有HTML

**解决**:
1. 确认代码已更新
2. 清除浏览器缓存（Ctrl+Shift+Delete）
3. 硬刷新页面（Ctrl+F5）

### 问题3: 页面访问404
**症状**: 访问页面本身就404

**原因**:
- 页面未发布（status≠1）
- Handle错误
- 使用id而不是handle

**解决**:
```bash
php dev/publish-page.php 2
```

### 问题4: 提交成功但没跳转
**症状**: 表单提交成功，但不跳转

**原因**:
- redirect_url未配置

**解决**:
在后台编辑页面，设置"跳转地址"字段

## Weline路由规则总结

### 路由组成
```
{模块router}/{控制器目录}/{控制器类名}/{方法名}
```

### 实际映射
```
Controller/Frontend/Form.php::submit()
         ↓          ↓     ↓       ↓
    pagebuilder/frontend/form/submit
```

### 常见错误
```
❌ pagebuilder/form/submit          (缺少frontend)
❌ pagebuilder/frontend/form        (缺少方法名)
❌ pagebuilder/submit                (路径不完整)
✅ pagebuilder/frontend/form/submit  (正确)
```
