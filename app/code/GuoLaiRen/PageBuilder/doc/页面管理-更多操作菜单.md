# PageBuilder 页面管理 - "更多操作"下拉菜单

## 功能概述

为 PageBuilder 页面管理界面的**首页类型页面**添加"更多操作"下拉菜单，集中展示重要功能按钮，优化界面布局。

## 功能位置

**路径：** 后台 → PageBuilder → 页面管理

**适用范围：** 仅对**首页类型的顶级页面**（`type = home_page` 且 `parent_id = 0`）显示此菜单

## 下拉菜单内容

### 1. Sitemap 管理（首要）

| 功能 | 图标 | 说明 | 条件 |
|------|------|------|------|
| **查看 Sitemap** | 🌐 `mdi-sitemap` | 在新标签页打开站点 sitemap.xml | 页面已发布 |
| **复制 Sitemap 链接** | 📋 `mdi-content-copy` | 一键复制 Sitemap URL 到剪贴板 | 页面已发布 |

**Sitemap URL 格式：**
```
http://your-site.com/sitemaps/{website_code}/sitemap.xml
```

### 2. 页面管理

| 功能 | 图标 | 说明 |
|------|------|------|
| **新增子页面** | ➕ `mdi-plus-circle` | 创建当前站点的子页面 |
| **SEO Sitemap 管理** | 🔍 `mdi-file-search` | 跳转到 SEO 模块的 Sitemap 管理页面 |

### 3. 高级操作

| 功能 | 图标 | 说明 | 权限 |
|------|------|------|------|
| **导出为静态网站** | 📥 `mdi-download` | 导出整个站点为静态 HTML 文件（包含资源） | `page_builder_export` |
| **站点设置** | ⚙️ `mdi-cog` | 跳转到 Website 模块的站点配置页面 | - |
| **站点页面统计** | 📊 `mdi-chart-bar` | 查看当前站点的所有页面统计 | - |

## 按钮布局对比

### 修改前（所有按钮平铺）

```
[预览] [查看] [拷贝] | [发布] [子页面] [编辑] [导出] [删除站点]
```

**问题：**
- 按钮过多，界面拥挤
- 重要功能（如 Sitemap）不可见
- 不易扩展新功能

### 修改后（分层展示）

```
[预览] [查看] [拷贝] | [发布] [编辑] [更多操作▼] [删除站点]
                                    └── 下拉菜单包含：
                                        - Sitemap 管理
                                        - 页面管理
                                        - 高级操作
```

**优点：**
- 界面简洁，主要按钮突出
- 次要功能收纳在下拉菜单
- 易于添加新功能
- 符合现代UI设计规范

## 重要性排序

按钮重要程度（从高到低）：

1. **预览** - 最常用，查看效果
2. **查看** - 访问已发布页面
3. **拷贝** - 快速分享链接
4. **发布** - 发布管理
5. **编辑** - 编辑页面
6. **更多操作** - 次要功能集合
   - **Sitemap 查看** ⭐ 重要但非频繁
   - **Sitemap 复制** ⭐ 便捷功能
   - 子页面管理
   - SEO 管理
   - 导出功能
   - 站点设置
   - 数据统计
7. **删除站点** - 危险操作，保持可见以警示

## 技术实现

### 1. 前端实现

**文件：** `app/code/GuoLaiRen/PageBuilder/view/templates/Backend/Page/index.phtml`

**关键代码：**

```php
<?php if ($isHomePage && $isTopLevel): ?>
    <!-- 首页类型：更多操作下拉菜单 -->
    <div class="btn-group" role="group">
        <button type="button" 
                class="btn btn-sm btn-secondary dropdown-toggle" 
                data-bs-toggle="dropdown">
            <i class="mdi mdi-dots-horizontal me-1"></i><?= __('更多操作') ?>
        </button>
        <ul class="dropdown-menu dropdown-menu-end">
            <!-- Sitemap 链接 -->
            <li>
                <a class="dropdown-item" href="<?= $sitemapUrl ?>" target="_blank">
                    <i class="mdi mdi-sitemap text-success me-2"></i>
                    <?= __('查看 Sitemap') ?>
                </a>
            </li>
            <!-- 复制 Sitemap 链接 -->
            <li>
                <button class="dropdown-item copy-sitemap-link-btn" 
                        data-url="<?= $sitemapUrl ?>">
                    <i class="mdi mdi-content-copy text-info me-2"></i>
                    <?= __('复制 Sitemap 链接') ?>
                </button>
            </li>
            <!-- 其他菜单项... -->
        </ul>
    </div>
<?php endif; ?>
```

### 2. JavaScript 实现

**复制 Sitemap 链接功能：**

```javascript
// 复制 Sitemap 链接
const copySitemapButtons = document.querySelectorAll('.copy-sitemap-link-btn');

copySitemapButtons.forEach(function(button) {
    button.addEventListener('click', function(e) {
        e.preventDefault();
        const url = this.getAttribute('data-url');
        
        // 使用 Clipboard API
        navigator.clipboard.writeText(url)
            .then(() => {
                showToast('success', '<?= __("Sitemap 链接已复制") ?>', url);
            })
            .catch(err => {
                showToast('error', '<?= __("复制失败") ?>', '<?= __("请手动复制链接") ?>');
            });
    });
});
```

## 使用场景

### 场景 1：SEO 优化

**需求：** 检查站点的 Sitemap 是否正确生成

**步骤：**
1. 进入 PageBuilder 页面管理
2. 找到站点首页
3. 点击"更多操作"
4. 选择"查看 Sitemap"
5. 在新标签页查看 sitemap.xml

### 场景 2：提交搜索引擎

**需求：** 获取 Sitemap URL 提交到 Google Search Console

**步骤：**
1. 进入 PageBuilder 页面管理
2. 找到站点首页
3. 点击"更多操作"
4. 选择"复制 Sitemap 链接"
5. 粘贴到 Google Search Console

### 场景 3：站点管理

**需求：** 管理站点的各项配置和页面

**步骤：**
1. 进入 PageBuilder 页面管理
2. 找到站点首页
3. 点击"更多操作"
4. 根据需要选择：
   - 新增子页面
   - SEO 管理
   - 站点设置
   - 页面统计

## 显示逻辑

### 首页类型页面

**条件：**
- `type = 'home_page'`
- `parent_id = 0` 或 `parent_id IS NULL`
- 页面已发布（`status = 1`）用于显示 Sitemap 链接

**显示：**

## 常见问题

### Q1: 为什么只有首页显示"更多操作"？

**A:** 首页是站点的入口，包含更多站点级别的管理功能（如 Sitemap、站点设置等）。子页面只需要基本的编辑功能。

### Q2: Sitemap 链接打开后显示 502 错误？

**A:** 需要配置 nginx，参见：`doc/Sitemap-Nginx配置修复.md`

### Q3: 复制链接功能不工作？

**A:** 检查浏览器是否支持 Clipboard API，或使用 HTTPS 协议访问后台。

### Q4: 能否自定义菜单项？

**A:** 可以通过修改 `index.phtml` 文件添加新的菜单项，注意保持图标风格和交互一致性。

### Q5: 如何添加权限控制？

**A:** 使用 `<acl source="权限资源ID">` 标签包裹需要权限的菜单项。
