# 样式卡片预览按钮功能

## 功能概述

在样式选择卡片的预览图右上角添加操作按钮，支持放大预览和真实页面预览，方便运营人员快速查看样式效果。

## 主要特性

### 1. 🔍 智能按钮显示

根据样式和页面状态动态显示按钮：

| 场景 | 放大预览按钮 | 真实预览按钮 |
|-----|------------|------------|
| **有预览图 + 编辑模式** | ✅ | ✅ |
| **有预览图 + 新建模式** | ✅ | ❌ |
| **无预览图 + 编辑模式** | ❌ | ✅ |
| **无预览图 + 新建模式** | ❌ | ❌ |

### 2. 🖼️ 放大预览（图片预览）

**功能**：查看完整的样式预览图片

**触发条件**：
- 样式有预览图（preview.webp 或 preview.png）

**交互方式**：
- 点击放大镜图标（`mdi-magnify-plus`）
- 弹出模态框显示完整图片
- 点击背景或按ESC键关闭

**显示效果**：
- 图片自适应模态框大小
- 保持原始宽高比
- 圆角边框美化

### 3. 🖥️ 真实预览（页面预览）

**功能**：在iframe中显示应用该样式的实际页面效果

**触发条件**：
- 编辑模式（有页面ID）

**交互方式**：
- 点击显示器图标（`mdi-monitor`）
- 弹出模态框显示iframe预览
- 自动应用该样式的默认配置
- 点击背景或按ESC键关闭

**显示效果**：
- iframe全屏显示
- 显示加载动画
- 加载完成后自动隐藏动画
- 完整的页面交互体验

## UI设计

### 按钮样式

**位置**：卡片预览区域右上角（绝对定位）

**外观**：
- 尺寸：32x32px
- 背景：白色半透明（rgba(255,255,255,0.95)）
- 边框：1px浅灰色
- 圆角：4px
- 阴影：0 2px 8px rgba(0,0,0,0.1)

**交互效果**：
- 默认隐藏（opacity: 0）
- 鼠标悬停卡片时显示（opacity: 1）
- 悬停按钮时：
  - 背景变为主题色（#667eea）
  - 图标变白色
  - 放大1.1倍
  - 增强阴影效果

### 模态框样式

**尺寸**：
- 宽度：90%（最大1400px）
- 高度：90%
- 圆角：12px

**结构**：
- **头部**：标题 + 图标 + 关闭按钮
- **主体**：预览内容（图片或iframe）
- **背景**：半透明黑色遮罩（rgba(0,0,0,0.8)）

**动画**：
- 背景淡入：0.2s
- 内容上滑：0.3s
- 关闭时淡出

### CSS类


### JavaScript函数

```javascript
// 打开图片预览
openImagePreview(imageUrl, styleName)

// 打开真实页面预览
openRealPreview(styleCode, styleName, pageId)

// 关闭预览模态框
closeStylePreviewModal()
```

```html

    <!-- 放大预览按钮 -->

         onclick="event.stopPropagation(); openImagePreview('...', '...')"

        <i class="mdi mdi-magnify-plus"></i>
    </div>
    
    <!-- 真实预览按钮 -->

         onclick="event.stopPropagation(); openRealPreview('...', '...', ...)" 
         title="真实页面预览">
        <i class="mdi mdi-monitor"></i>
    </div>
</div>

<!-- 预览模态框（页面底部） -->

    <div class="style-preview-modal-content">
        <div class="style-preview-modal-header">...</div>

    </div>
```

## 技术细节

### 事件传播控制

```javascript
onclick="event.stopPropagation(); openImagePreview(...)"
```

**原因**：防止点击预览按钮时触发卡片的选择事件。

### 模态框关闭方式

1. **点击关闭按钮**：`onclick="closeStylePreviewModal()"`
2. **点击背景**：事件委托监听点击模态框自身
3. **ESC键**：监听键盘事件

### iframe加载优化

```javascript
<iframe src="..." onload="this.previousElementSibling.style.display='none'">
```

- 加载时显示loading动画
- 加载完成后自动隐藏动画
- 关闭模态框后延迟300ms清空iframe（释放资源）

### body滚动控制

```javascript
// 打开模态框
document.body.style.overflow = 'hidden';

// 关闭模态框
document.body.style.overflow = '';
```

防止模态框打开时页面背景滚动。

## 优化建议

### 1. 预览图优化

**推荐**：为所有样式添加预览图
- 格式：webp（文件更小）
- 尺寸：1200x675 (16:9)
- 文件大小：< 500KB

**好处**：
- 放大预览直接显示图片（快）
- 真实预览提供实时交互（准确）

### 2. 加载性能

**当前实现**：
- 每次打开真实预览时创建新iframe
- 关闭后延迟300ms销毁iframe
- 避免多个iframe同时存在

**优化方向**：
- 可考虑预加载常用样式的iframe
- 使用虚拟滚动减少同时渲染的卡片数量

### 3. 移动端适配

当前按钮在移动端：
- 触摸设备上按钮始终显示（不需要hover）
- 模态框宽度和高度自适应屏幕
- 可考虑增加按钮尺寸（36x36px）

## 常见问题

### Q1: 为什么有些卡片没有预览按钮？

**A**: 
- 新建页面模式：没有页面ID，无法生成真实预览
- 没有预览图且是新建模式：两种预览都不可用

### Q2: 真实预览显示空白？

**A**: 可能原因：
1. 样式模板有错误
2. 页面数据不完整
3. iframe跨域限制（理论上不应该出现）

**解决方案**：
1. 检查浏览器控制台错误信息
2. 确认样式模板文件存在
3. 检查 `Preview::full()` 方法是否正常

### Q3: 预览按钮不显示？

**A**: 
1. 检查是否鼠标悬停在卡片上
2. 在移动端触摸设备上，按钮应始终显示
3. 检查CSS是否正确加载

### Q4: 关闭模态框后页面无法滚动？

**A**: 这是bug，`body.style.overflow` 没有正确恢复。已在代码中处理：
```javascript
document.body.style.overflow = '';
```
