# PageBuilder 像素统计配置说明

## 概述

PageBuilder 模块已集成 Weline Visitor 像素统计功能，可以在页面模板中配置像素统计，用于记录页面访问数据。像素统计会自动使用页面绑定的网站（`website_id`）进行数据关联。

## 重要说明

### 网站ID自动关联

- **页面已有 `website_id` 字段**：每个页面在创建/编辑时都会绑定一个网站（website_id）
- **无需手动配置**：像素统计会自动从页面对象中读取 `website_id`，无需在可视化配置中额外配置
- **数据关联**：所有像素统计数据都会自动关联到页面绑定的网站，确保数据正确归属

### 配置项说明

在模板的可视化配置中，像素统计配置组包含以下配置项：

1. **启用像素统计** (`pixel.enabled`)
   - 类型：选择框（yes/no）
   - 默认值：yes（默认启用）
   - 说明：控制是否在该页面启用像素统计功能

2. **像素名称** (`pixel.name`)
   - 类型：文本输入
   - 默认值：`{handle}`（自动使用页面的 handle）
   - 说明：用于区分不同页面的统计来源
   - 如果留空，系统会自动使用页面的 handle 作为像素名称

## 使用方法

### 1. 在模板配置中添加像素配置

在模板的 `header.phtml` 文件中，在 `@fields_start` 和 `@fields_end` 之间添加像素配置项：

```php
 * group:pixel => 像素统计:📊访客像素统计配置:配置访客像素统计功能，用于记录页面访问数据。像素统计会自动使用页面绑定的网站（website_id）进行数据关联。
 * pixel.enabled => 启用像素统计:select:yes|yes,no
 * pixel.name => 像素名称:text:{handle}:用于区分不同页面的统计来源，默认使用页面的 handle 作为像素名称。留空将自动使用页面的 handle。
```

### 2. 在模板中输出像素标签

在 `header.phtml` 的 `<head>` 标签内，添加像素输出代码：

```php
<!-- 像素统计配置 -->
<?php
// 读取像素配置（默认启用）
$pixelEnabled = getConfig($styleSettings, 'pixel.enabled', 'yes');
$pixelName = getConfig($styleSettings, 'pixel.name', '');

// 如果像素名称为空，使用页面的 handle
if (empty($pixelName) && $page) {
    $pixelName = $page->getData('handle') ?? '';
}

// 直接从页面对象获取 website_id（页面已有此字段，无需配置）
$websiteId = $page ? (int)($page->getData('website_id') ?? 0) : 0;

// 如果启用了像素统计且页面绑定了网站，输出像素标签
if ($pixelEnabled === 'yes' && $websiteId > 0 && !empty($pixelName) && !$isPreview):
    // 设置 website_id Cookie，供像素代码读取
    \Weline\Framework\Http\Cookie::set('WELINE_WEBSITE_ID', (string)$websiteId, 3600 * 24 * 365);
    
    // 获取网站 URL（用于像素统计）
    $websiteUrl = '';
    try {
        if (class_exists('\Weline\Websites\Model\Website')) {
            $websiteModel = \Weline\Framework\Manager\ObjectManager::getInstance(\Weline\Websites\Model\Website::class);
            $website = clone $websiteModel;
            $website->clear()->load($websiteId);
            if ($website->getId()) {
                $websiteUrl = $website->getData('url') ?? '';
            }
        }
    } catch (\Exception $e) {
        // 如果获取网站信息失败，不影响像素功能
    }
    
    // 设置 website_url Cookie
    if (!empty($websiteUrl)) {
        \Weline\Framework\Http\Cookie::set('WELINE_WEBSITE_URL', $websiteUrl, 3600 * 24 * 365);
    }
?>
    <!-- Weline Visitor Pixel -->
    <pixel name="<?= htmlspecialchars($pixelName) ?>" />
<?php endif; ?>
```

### 3. 在后台配置页面

1. 进入页面编辑界面
2. 打开"可视化配置"
3. 找到"像素统计"配置组
4. 启用"启用像素统计"
5. 设置"像素名称"（可选，留空会自动生成）
6. 保存配置

## 工作原理

### 数据流程

1. **页面加载时**：
   - 检查是否启用了像素统计
   - 从页面对象读取 `website_id`
   - 设置 `WELINE_WEBSITE_ID` Cookie

2. **像素代码执行**：
   - Weline Visitor 的像素标签读取 Cookie 中的 `website_id`
   - 将访问数据发送到后端 API
   - 数据自动关联到对应的网站

3. **数据存储**：
   - 像素数据存储在 `w_pixel` 表中
   - 每条记录都包含 `website_id` 字段
   - 可以通过 `website_id` 查询特定网站的统计数据

### Cookie 设置

像素统计功能会自动设置以下 Cookie：

- `WELINE_WEBSITE_ID`：网站ID，用于标识数据归属的网站
- `WELINE_WEBSITE_URL`：网站URL，用于记录访问来源

这些 Cookie 会在页面加载时自动设置，有效期 365 天。

## 注意事项

1. **网站绑定**：确保页面已正确绑定网站（`website_id` > 0），否则像素统计不会生效

2. **预览模式**：预览模式下不会输出像素标签，避免测试数据污染统计数据

3. **像素名称**：建议使用有意义的像素名称，便于在后台区分不同页面的统计来源

4. **数据查询**：可以通过 Weline Visitor 模块的后台界面查看统计数据，数据会按网站ID自动分组

## 相关模块

- **Weline Visitor**：提供像素统计核心功能
- **Weline Websites**：提供网站管理功能，页面通过 `website_id` 关联网站
