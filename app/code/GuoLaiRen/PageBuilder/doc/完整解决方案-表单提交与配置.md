## 🔧 问题1: URL加密前缀导致404

### 问题描述
表单提交时浏览器返回404错误：
```
URL: /u0Ma5pkolBH3wlDlIh6FV0XCoTTgTE8/CNY/zh_Hans_CN/pagebuilder/frontend/form/submit
Status: 404 Not Found
```

### 根本原因
使用了 `getUrl()` 方法，导致URL包含后台加密前缀。

### 解决方案
**文件**: `app/code/GuoLaiRen/PageBuilder/view/templates/style/marketing-landing/content.phtml`

```php
// ❌ 错误代码
$formAction = $this->getUrl('pagebuilder/frontend/form/submit', ...);

// ✅ 正确代码
$formAction = $this->getFrontendUrl('pagebuilder/frontend/form/submit', ...);
```

### 关键经验
| 方法 | 用途 | URL格式 |
|------|------|---------|
| `getFrontendUrl()` | ✅ 前端 | `/pagebuilder/frontend/form/submit` |
| `getBackendUrl()` | 后台 | `/ENC/admin/page/edit` |
| `getUrl()` | 自动 | ⚠️ 取决于上下文（不推荐）|

## 🔧 问题2: 数据库表字段缺失

### 问题描述
浏览器提交表单时Fatal Error：
```
Fatal error: Undefined constant GuoLaiRen\PageBuilder\Model\FormSubmission::fields_STATUS 
in app/code/GuoLaiRen/PageBuilder/Controller/Frontend/Form.php:90
```

### 根本原因
FormSubmission模型缺少 `fields_STATUS` 常量和状态常量定义，数据库表也缺少 `status` 字段。

### 解决方案

#### 1. 添加模型常量
**文件**: `app/code/GuoLaiRen/PageBuilder/Model/FormSubmission.php`

```php
// 字段常量
public const fields_STATUS = 'status';

// 状态常量
public const STATUS_NEW = 'new';
public const STATUS_PROCESSED = 'processed';
public const STATUS_SPAM = 'spam';
```

#### 2. 修复数据库表结构
**脚本**: `dev/fix-db-direct.php` (已删除)

```bash
php dev/fix-db-direct.php
```

**执行结果**:
- ✅ 添加 `status` 字段 (VARCHAR(20) NOT NULL DEFAULT 'new')
- ✅ 添加 `idx_status` 索引

#### 3. 修复page_id获取逻辑
**文件**: `app/code/GuoLaiRen/PageBuilder/Controller/Frontend/Form.php`

```php
// ❌ 错误：只从POST获取
$pageId = (int)$this->request->getPost('page_id', 0);

// ✅ 正确：支持GET和POST
$pageId = (int)($this->request->getPost('page_id') ?: $this->request->getGet('page_id', 0));
```

### 关键经验
- ✅ 所有数据库字段必须定义对应的 `fields_XXX` 常量
- ✅ 枚举值必须定义常量（如 `STATUS_XXX`）
- ✅ 参数获取要考虑多种来源（GET/POST）

## 🔧 问题3: 后台getFormKey错误

### 问题描述
后台提交记录列表页面出现错误：
```
Uncaught ArgumentCountError: Too few arguments to function 
Weline\Framework\View\Template::getFormKey(), 0 passed in 
app/code/GuoLaiRen/PageBuilder/view/templates/Backend/Submission/index.phtml:172
```

### 根本原因
`getFormKey($url)` 方法需要传入URL参数，但模板中没有传递。

### 解决方案
**文件**: `app/code/GuoLaiRen/PageBuilder/view/templates/Backend/Submission/index.phtml`

```php
// ❌ 错误代码
<input type="hidden" name="form_key" value="<?= $this->getFormKey() ?>">

// ✅ 正确代码
<?php $deleteUrl = $this->getBackendUrl('*/backend/submission/delete'); ?>
<form action="<?= $deleteUrl ?>" method="post">
    <input type="hidden" name="form_key" value="<?= $this->getFormKey($deleteUrl) ?>">
    ...
</form>
```

### 关键经验
- ✅ `getFormKey()` 必须传入目标URL参数
- ✅ 建议先定义URL变量，然后复用
- ✅ CSRF token需要绑定特定URL

## 🎨 新功能: 表单错误信息可配置

### 功能说明
允许在页面配置中自定义表单提交时的所有提示信息，支持多语言。

### 实现方案

#### 1. 添加Schema配置
**文件**: `app/code/GuoLaiRen/PageBuilder/view/templates/style/marketing-landing/content.phtml`

```php
/**
 * @fields_start
 * 
 * group:messages => 表单提示信息配置
 * messages.email_required => 邮箱必填提示:text:请输入邮箱地址
 * messages.email_invalid => 邮箱格式错误提示:text:邮箱格式不正确
 * messages.phone_required => 电话必填提示:text:请输入电话号码
 * messages.phone_invalid => 电话格式错误提示:text:电话号码格式不正确
 * messages.submitting => 提交中提示:text:Submitting your information...
 * messages.success => 提交成功提示:text:Success! Redirecting...
 * messages.error => 提交失败提示:text:Submission failed. Please try again.
 * 
 * @fields_end
 */
```

#### 2. PHP配置读取
```php
// 表单提示信息配置
$formMessages = [
    'email_required' => getContentConfig($styleSettings, 'messages.email_required', '请输入邮箱地址'),
    'email_invalid' => getContentConfig($styleSettings, 'messages.email_invalid', '邮箱格式不正确'),
    'phone_required' => getContentConfig($styleSettings, 'messages.phone_required', '请输入电话号码'),
    'phone_invalid' => getContentConfig($styleSettings, 'messages.phone_invalid', '电话号码格式不正确'),
    'submitting' => getContentConfig($styleSettings, 'messages.submitting', 'Submitting your information...'),
    'success' => getContentConfig($styleSettings, 'messages.success', 'Success! Redirecting...'),
    'error' => getContentConfig($styleSettings, 'messages.error', 'Submission failed. Please try again.')
];
```

#### 3. JavaScript使用配置
```javascript
const messages = <?= json_encode($formMessages) ?>;

// 使用配置的消息
setButtonLoading(true);
showMessage(messages.submitting, 'loading');

// 成功消息
if (data.success) {
    showMessage(messages.success, 'success');
}

// 错误消息（优先使用服务器返回的，否则使用配置的）
const errorMsg = data.message || messages.error;
showMessage(errorMsg, 'error');
```

### 配置项说明

| 配置键 | 说明 | 默认值 |
|--------|------|--------|
| `messages.email_required` | 邮箱必填提示 | 请输入邮箱地址 |
| `messages.email_invalid` | 邮箱格式错误 | 邮箱格式不正确 |
| `messages.phone_required` | 电话必填提示 | 请输入电话号码 |
| `messages.phone_invalid` | 电话格式错误 | 电话号码格式不正确 |
| `messages.submitting` | 提交中提示 | Submitting your information... |
| `messages.success` | 提交成功提示 | Success! Redirecting... |
| `messages.error` | 提交失败提示 | Submission failed. Please try again. |

## 📦 数据库表结构

### 最终表结构
```sql
CREATE TABLE `guolairen_page_builder_form_submission` (
  `submission_id` int(11) NOT NULL AUTO_INCREMENT,
  `page_id` int(11) DEFAULT NULL,
  `email` varchar(255) DEFAULT NULL,
  `phone` varchar(50) DEFAULT NULL,
  `extra_fields` text DEFAULT NULL,
  `ip_address` varchar(45) DEFAULT NULL,
  `user_agent` varchar(255) DEFAULT NULL,
  `referer` varchar(255) DEFAULT NULL,
  `status` varchar(20) NOT NULL DEFAULT 'new',
  `submitted_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`submission_id`),
  KEY `idx_page_id` (`page_id`),
  KEY `idx_email` (`email`),
  KEY `idx_phone` (`phone`),
  KEY `idx_status` (`status`),
  KEY `idx_submitted_at` (`submitted_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
