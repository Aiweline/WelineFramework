# 响应式配置格式使用指南

## 概述

PageBuilder 现在支持统一的响应式配置格式，一个配置项就能定义多端（移动端、平板、PC端）的值，自动生成流畅的响应式CSS。

## 格式规范

### 基本语法

```
配置项 => 标签:类型:值|单位[描述或响应式标记]
```

**参数说明：**
- `标签`：配置项显示名称
- `类型`：输入类型（number、text、select、color等）
- `值`：默认值（支持单一值、双端值、三端值）
- `单位`：可选，如 `px`、`%`、`em` 等（不指定则使用默认px）
- `描述`：可选，方括号内的说明文字或响应式标记

### 值的格式

| 格式 | 示例 | 说明 |
|------|------|------|
| 单一值 | `80` | 所有端使用相同值 |
| 双端值 | `60/80` | 移动端/PC端（推荐） |
| 三端值 | `60/70/80` | 移动端/平板/PC端 |

### 单位

- 支持所有CSS单位：`px`, `%`, `em`, `rem`, `vh`, `vw` 等
- **可选参数**：不配置单位时，默认使用 `px`
- 示例：`60/80` 等同于 `60/80|px`

### 响应式标记与描述

**响应式标记（使用MTD字母）：**
```php
// M = Mobile (移动端), T = Tablet (平板), D = Desktop (PC端)

// 仅移动端
size.height => 头部高度:number:60|px[M]

// 移动端和PC端（推荐）
size.height => 头部高度:number:60/80|px[MD]

// 完整三端支持
size.height => 头部高度:number:60/70/80|px[MTD]
```

**标记说明：**
- `[M]` - 仅移动端，显示 📱 图标
- `[MD]` - 移动端和PC端（推荐），显示 📱💻 图标
- `[MT]` - 移动端和平板，显示 📱📱 图标
- `[MTD]` - 完整三端支持，显示 📱📱💻 图标

**自定义描述：**
```php
// 使用方括号添加自定义描述
size.max_width => 内容最大宽度:number:100/1200|%/px[移动端使用百分比，PC端使用像素]
```

**视觉效果：**
- 响应式配置会根据MTD字母显示对应的Font Awesome图标
- 描述信息会显示在输入框下方
- 鼠标悬停显示完整的设备支持说明
- 帮助用户理解配置项的用途

## 配置示例

### 完整配置示例（Header）

```php
/**
 * 默认样式 - Header
 * @var \Weline\Framework\View\Template $this
 * 
 * @fields_start
 * 
 * ========================================
 * 📱💻 响应式配置说明
 * ========================================
 * 支持三端配置格式：移动端/平板/PC端
 * - 单一值：80 (所有端相同)
 * - 双端值：60/80 (移动端60, PC端80, 平板自动计算)
 * - 三端值：60/70/80 (移动端60, 平板70, PC端80)
 * 
 * 断点定义：
 * M (Mobile)  移动端: 375px
 * T (Tablet)  平板: 768px  
 * D (Desktop) PC端: 1280px
 * 
 * 响应式标记：
 * [M]   - 仅移动端
 * [MD]  - 移动端和PC端（推荐）
 * [MTD] - 移动端、平板、PC端（完整支持）
 * 
 * 配置格式：
 * key => 标签:类型:默认值|单位[MTD=支持三端响应式]
 * ========================================
 * 
 * group:layout => 布局
 * layout.logo_position => Logo位置:select:left|left,center,right
 * 
 * group:style => 样式
 * style.header_background_color => 页头背景色:color:transparent
 * style.border_bottom => 底部边框:select:yes|yes,no
 * style.border_color => 边框颜色:color:#eeeeee
 * 
 * group:size => 尺寸[MD]
 * size.height => 头部高度:number:60/80|px[MD]
 * size.max_width => 内容最大宽度:number:100/1200|%/px[移动端使用百分比，PC端使用像素]
 * size.padding_h => 左右内边距:number:16/40|px[MD]
 * 
 * @fields_end
 */
```

### 配置项说明

**头部配置说明区域：**
- 使用分隔线 `========` 使说明更醒目
- 列出支持的三种配置格式
- 明确断点定义
- 说明配置格式和响应式标记

**响应式配置项：**
- `size.height` - 带响应式标记 `[MD]`（显示📱💻图标）
- `size.max_width` - 带自定义描述说明
- `size.padding_h` - 带响应式标记 `[MD]`（显示📱💻图标）

**普通配置项：**
- `layout.logo_position` - 选择器类型，不支持响应式
- `style.*` - 样式类型，不支持响应式

**图标映射：**
- `[M]` → 📱 (fa-mobile-alt)
- `[MD]` → 📱💻 (fa-mobile-alt + fa-desktop)
- `[MT]` → 📱📱 (fa-mobile-alt + fa-tablet-alt)
- `[MTD]` → 📱📱💻 (fa-mobile-alt + fa-tablet-alt + fa-desktop)

## PHP 使用方法

### 1. 读取配置

```php
// 从样式设置中读取配置值
$heightValue = getConfig($styleSettings, 'size.height', '60/80');
```

### 2. 解析配置

```php
// 解析响应式值
$height = parseResponsiveValue($heightValue . 'px');

// 返回结果：
// [
//     'mobile' => '60',
//     'tablet' => '',        // 空表示自动计算
//     'desktop' => '80',
//     'unit' => 'px'
// ]
```

### 3. 生成CSS

```php
// 生成响应式CSS
$heightResponsive = generateResponsiveCSS($height);

// 返回：
// "clamp(60px, calc(60px + 20 * ((100vw - 375px) / 905)), 80px)"
```

### 4. 应用到模板

```php
<style>
.page-header {
    height: <?= $heightResponsive ?>;
}
</style>
```

## 完整示例

```php
<?php
/**
 * @fields_start
 * group:size => 尺寸
 * size.height => 头部高度:number:60/80|px
 * size.padding => 内边距:number:16/40|px
 * @fields_end
 */

// 获取样式配置
$styleSettings = $this->getData('style_settings') ?: [];

// 读取并解析配置
$heightValue = getConfig($styleSettings, 'size.height', '60/80');
$paddingValue = getConfig($styleSettings, 'size.padding', '16/40');

$height = parseResponsiveValue($heightValue . 'px');
$padding = parseResponsiveValue($paddingValue . 'px');

// 生成响应式CSS
$heightResponsive = generateResponsiveCSS($height);
$paddingResponsive = generateResponsiveCSS($padding);
?>

<style>
.my-component {
    /* 高度从 60px 平滑过渡到 80px */
    height: <?= $heightResponsive ?>;
    
    /* 内边距从 16px 平滑过渡到 40px */
    padding: <?= $paddingResponsive ?>;
}
</style>
```

## 生成的CSS效果

### 输入：`60/80`

```css
height: clamp(60px, calc(60px + 20 * ((100vw - 375px) / 905)), 80px);
```

**效果：**
- 375px 宽度：60px
- 827.5px 宽度：70px（中点自动计算）
- 1280px+ 宽度：80px

### 输入：`16/40`

```css
padding: clamp(16px, calc(16px + 24 * ((100vw - 375px) / 905)), 40px);
```

**效果：**
- 375px 宽度：16px
- 827.5px 宽度：28px（中点自动计算）
- 1280px+ 宽度：40px

## 断点说明

| 断点 | 宽度 | 说明 |
|------|------|------|
| 移动端 | 375px | 最小宽度，使用第一个值 |
| 平板 | 375px - 1280px | 自动计算中间值 |
| PC端 | 1280px | 最大宽度，使用最后一个值 |

## 适用场景

### ✅ 适合使用

- **尺寸类**：高度、宽度、内边距、外边距
- **字体类**：字号、行高、字间距
- **间距类**：上下间距、左右间距
- **边框类**：边框宽度、圆角半径

### ⚠️ 不适用

- **颜色类**：颜色无法用 clamp 计算
- **选择器类**：select 类型不需要响应式
- **文本类**：文本内容不需要响应式

## 最佳实践

### 1. 优先使用双端值

```php
// ✅ 推荐：简洁明了
size.height => 头部高度:number:60/80|px

// ❌ 避免：过度配置
size.height => 头部高度:number:60/65/70/75/80|px
```

### 2. 保持合理的差值

```php
// ✅ 推荐：移动端和PC端差值适中
size.height => 头部高度:number:60/80|px  // 差值 20px

// ⚠️ 注意：差值过大可能导致不协调
size.height => 头部高度:number:40/120|px  // 差值 80px（过大）
```

### 3. 使用一致的单位

```php
// ✅ 推荐：统一使用 px
size.height => 头部高度:number:60/80|px
size.padding => 内边距:number:16/40|px

// ⚠️ 避免：混合单位（除非有特殊需求）
size.height => 头部高度:number:3/5|rem
```

### 4. 设置合理的默认值

```php
// ✅ 推荐：默认值在各端都合理
size.height => 头部高度:number:60/80|px

// ⚠️ 避免：默认值在某些端显示异常
size.height => 头部高度:number:20/200|px
```

## 常见问题

### Q: 为什么我的配置没有生效？

A: 检查以下几点：
1. 配置项格式是否正确（用斜杠分隔）
2. 单位是否添加（如 `|px`）
3. 是否清除了缓存（`php bin/w cache:clear -f`）
4. 是否刷新了样式配置（点击可视化编辑器中的刷新按钮）

### Q: 可以使用三个以上的断点吗？

A: 目前只支持最多3个值（移动端/平板/PC端），更多断点会被忽略。通常两个端点就足够了。

### Q: 可以使用负值吗？

A: 可以，但要确保在所有端点都合理。例如：`-10/10` 表示从 -10px 过渡到 10px。

### Q: 混合单位（如 % 和 px）如何处理？

A: 解析器会检测单位差异，如果不同则使用PC端的值。建议尽量使用相同单位。
