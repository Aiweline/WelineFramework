# 样式配置无法加载问题修复

## 🐛 问题描述

在后台页面编辑界面中，选择"Marketing Landing"样式后，配置项区域提示"该样式没有可配置项"，无法加载样式的配置表单。

### 错误现象
```
选择样式模板: Marketing Landing
配置状态: ❌ 该样式没有可配置项
```

### 影响范围
- 无法配置营销落地页模板的参数
- 无法自定义表单提示信息
- 无法调整页面布局、颜色、字体等样式

---

## 🔍 问题分析

### 问题1: AJAX请求URL路径错误

**错误代码** (`Backend/Page/form.phtml` 第1591行):
```javascript
let configUrl = '<?= $this->getBackendUrl('*/backend/page/styleConfig') ?>...
```

**问题原因**:
- 控制器方法名: `getStyleConfig()`
- Weline框架自动转换: `getStyleConfig` → `style-config`
- 模板使用的URL: `styleConfig`（驼峰式）❌
- 正确的URL应该是: `style-config`（连字符）✅

**结果**: AJAX请求返回404，配置无法加载

---

### 问题2: 缺少自动扫描机制

**原始逻辑**:
- 样式扫描需要手动运行 `php bin/w style:scan`
- 访问编辑页面时不会自动扫描模板
- 模板配置变更后需要手动触发扫描

**问题**:
- 开发时修改模板后忘记扫描
- 配置定义与实际不同步
- 用户体验差

---

## ✅ 解决方案

### 修复1: 更正AJAX请求URL

**文件**: `app/code/GuoLaiRen/PageBuilder/view/templates/Backend/Page/form.phtml`

**修改**:
```javascript
// ❌ 既有代码
let configUrl = '<?= $this->getBackendUrl('*/backend/page/styleConfig') ?>?style_code=' + ...

// ✅ 新代码
let configUrl = '<?= $this->getBackendUrl('*/backend/page/style-config') ?>?style_code=' + ...
```

**说明**: 将驼峰式路径改为连字符路径，匹配Weline框架的URL转换规则。

---

### 修复2: 添加自动扫描功能

**文件**: `app/code/GuoLaiRen/PageBuilder/Controller/Backend/Page.php`

#### 在getCreate方法中添加
```php
#[\Weline\Framework\Acl\Acl('GuoLaiRen_PageBuilder::page_builder_create', '新建页面', 'mdi mdi-plus', '新建页面')]
public function getCreate()
{
    // ✅ 强制扫描样式模板，确保配置是最新的
    Style::forceScan();
    
    $this->assign('page_title', __('新建页面'));
    // ...
}
```

#### 在getEdit方法中添加
```php
#[\Weline\Framework\Acl\Acl('GuoLaiRen_PageBuilder::page_builder_edit', '编辑页面', 'mdi mdi-pencil', '编辑页面')]
public function getEdit()
{
    // ✅ 强制扫描样式模板，确保配置是最新的
    Style::forceScan();
    
    $pageId = $this->request->getGet('id');
    // ...
}
```

**说明**: 每次访问创建或编辑页面时自动扫描模板，确保配置始终是最新的。

## 🎯 Weline框架URL路由规则

### 方法名到URL的转换

| 控制器方法 | URL路径 | 示例 |
|-----------|---------|------|
| `getIndex()` | `index` | `/backend/page/index` |
| `getEdit()` | `edit` | `/backend/page/edit` |
| `getStyleConfig()` | `style-config` | ✅ `/backend/page/style-config` |
| `postCreate()` | `create` | `/backend/page/create` |
| `postDelete()` | `delete` | `/backend/page/delete` |

### 关键规则
1. **驼峰转连字符**: `getStyleConfig` → `style-config`
2. **小写**: 所有URL路径都是小写
3. **连字符分隔**: 多个单词用连字符 `-` 连接
4. **去除get/post前缀**: `getEdit` → `edit`, `postCreate` → `create`

### ❌ 常见错误

```javascript
// ❌ 错误：使用驼峰式
'*/backend/page/styleConfig'
'*/backend/page/getStyleConfig'

// ✅ 正确：使用连字符
'*/backend/page/style-config'
```

## 🔧 样式扫描机制

### 自动扫描触发点

| 触发点 | 方法 | 说明 |
|-------|------|------|
| 访问创建页面 | `getCreate()` | ✅ 新增 |
| 访问编辑页面 | `getEdit()` | ✅ 新增 |
| AJAX加载配置 | `getStyleConfig()` | ✅ 已有 |
| 模型查询 | `Style::autoScan()` | 60秒间隔自动扫描 |
| 手动命令 | `php bin/w style:scan` | 手动触发 |

### 扫描内容
1. 检测 `view/templates/style/` 下的所有模板目录
2. 验证必需文件（header.phtml, footer.phtml, readme.md）
3. 解析配置定义（@fields_start...@fields_end）
4. 更新数据库记录
5. 返回配置结构

### 性能优化
- `forceScan()`: 强制立即扫描，忽略时间间隔
- `autoScan(60)`: 60秒内只扫描一次，避免频繁IO
- 只扫描有变更的文件（比较修改时间）
