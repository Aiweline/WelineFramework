# 前端页面渲染架构

## 概述

PageBuilder 模块的前端页面渲染系统采用**模板化架构**，支持自定义样式模板的灵活渲染。

## 渲染流程

### 1. 控制器处理

**文件**: `Controller/Frontend/Page.php`

控制器根据页面的样式配置决定渲染方式：

```php
if ($styleCode && $style->getId()) {
    // 获取样式模板路径
    $headerTemplate = $style->getHeaderPath();
    $contentTemplate = $style->getContentPath();
    $footerTemplate = $style->getFooterPath();
    
    // 依次渲染三个部分
    echo $this->render($headerTemplate);
    echo $this->render($contentTemplate);
    echo $this->render($footerTemplate);
} else {
    // 使用默认模板 view.phtml
    return $this->fetch();
}
```

### 2. 样式模板结构

每个样式模板包含三个核心文件：

#### header.phtml
- 包含完整的 HTML 文档头部：`<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`
- SEO 元标签（title, description, keywords）
- 追踪代码（GTM, GA4, Facebook Pixel）
- 全局样式重置
- 页面 header 区域

**示例**:
```php
<!DOCTYPE html>
<html lang="<?= htmlspecialchars($locale) ?>">
<head>
    <meta charset="UTF-8">
    <title><?= htmlspecialchars($metaTitle) ?></title>
    <!-- ... 其他 meta 标签和追踪代码 ... -->
</head>
<body>
    <header class="marketing-header">
        <!-- header 内容 -->
    </header>
```

#### content.phtml
- 页面主内容区域
- 表单、文本、图片等可配置元素
- 独立的 CSS 样式定义
- 业务逻辑和交互脚本

**示例**:
```php
<style>
    .marketing-content {
        /* 样式定义 */
    }
</style>

<main class="marketing-content">
    <!-- 主要内容 -->
</main>

<script>
    // 表单提交逻辑
</script>
```

#### footer.phtml
- 页面尾部内容
- 版权声明、链接、免责声明
- HTML 文档结束标签：`</body>`, `</html>`

**示例**:
```php
    <footer class="marketing-footer">
        <!-- footer 内容 -->
    </footer>
</body>
</html>
```

### 3. 默认模板 (view.phtml)

当页面**未指定样式**或**样式不存在**时，使用默认的 `view.phtml` 模板。

**特点**:
- 完整的独立页面（包含完整的 HTML 结构）
- 简洁的设计风格
- 基础的内容展示功能
- 多语言切换器
- 响应式布局

**使用场景**:
- 页面创建初期，尚未选择样式
- 样式模板被删除或禁用
- 快速预览页面内容

## 数据传递

### 可用变量

所有模板文件都可以访问以下变量：

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `$page` | `PageModel` | 页面模型对象 |
| `$content` | `array` | 本地化内容（title, content） |
| `$seo` | `array` | SEO 数据（title, description, keywords） |
| `$style` | `array` | 样式配置数组 |
| `$current_locale` | `string` | 当前语言代码 |
| `$available_locales` | `array` | 可用语言列表 |
| `$has_translation` | `bool` | 是否有当前语言翻译 |
| `$is_locale_supported` | `bool` | 是否支持当前语言 |

### 配置优先级

样式配置的优先级（从高到低）：

1. **翻译的配置** - 特定语言的配置值（最高优先级）
2. **页面保存的配置** - 在后台编辑时保存的值
3. **模板默认配置** - 在 `@fields_start` 中定义的默认值（最低优先级）

## 样式模型 (Style.php)

### 模板路径方法

```php
class Style extends AbstractModel
{
    /**
     * 获取 header.phtml 路径
     */
    public function getHeaderPath(): string
    {
        $path = $this->getData(self::fields_PATH);
        return $path . '/header.phtml';
    }
    
    /**
     * 获取 content.phtml 路径
     */
    public function getContentPath(): string
    {
        $path = $this->getData(self::fields_PATH);
        return $path . '/content.phtml';
    }
    
    /**
     * 获取 footer.phtml 路径
     */
    public function getFooterPath(): string
    {
        $path = $this->getData(self::fields_PATH);
        return $path . '/footer.phtml';
    }
}
```

### 路径格式

`path` 字段存储的是完整的模块相对路径：

```
GuoLaiRen_PageBuilder::style/marketing-landing
GuoLaiRen_PageBuilder::style/default
```

## 创建新样式模板

### 步骤 1: 创建目录结构

```
app/code/GuoLaiRen/PageBuilder/view/templates/style/
└── my-custom-style/
    ├── header.phtml
    ├── content.phtml
    ├── footer.phtml
    └── preview.png (可选)
```

### 步骤 2: 实现 header.phtml

```php
<?php
$page = $this->getData('page');
$styleSettings = $this->getData('style') ?: [];
?>
<!DOCTYPE html>
<html lang="<?= htmlspecialchars($this->getData('locale') ?: 'zh_Hans_CN') ?>">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title><?= htmlspecialchars($page ? $page->getData('meta_title') : '') ?></title>
    
    <style>
        body {
            margin: 0;
            padding: 0;
            font-family: Arial, sans-serif;
        }
    </style>
</head>
<body>
    <header>
        <!-- Header 内容 -->
    </header>
```

### 步骤 3: 实现 content.phtml

使用 `@fields_start` 和 `@fields_end` 定义配置项：

```php
<?php
/**
 * @fields_start
 * 
 * group:layout => 布局设置
 * layout.width => 容器宽度:responsive:100/1280|%/px
 * 
 * group:content => 内容设置
 * content.title => 标题:text:默认标题
 * content.title_color => 标题颜色:color:#333333
 * 
 * @fields_end
 */
$page = $this->getData('page');
$styleSettings = $this->getData('style') ?: [];

function getConfig($settings, $key, $default = '') {
    return $settings[$key] ?? $default;
}

$title = getConfig($styleSettings, 'content.title', '默认标题');
?>

<style>
    /* 样式定义 */
</style>

<main>
    <h1><?= htmlspecialchars($title) ?></h1>
</main>
```

### 步骤 4: 实现 footer.phtml

```php
<?php
$page = $this->getData('page');
?>
    <footer>
        <!-- Footer 内容 -->
    </footer>
</body>
</html>
```

### 步骤 5: 注册样式

运行样式扫描命令：

```bash
php bin/w pagebuilder:style:scan
```

系统会自动扫描并注册新样式到数据库。

## 最佳实践

### 1. 配置项命名

使用清晰的层级结构：

```
group:layout => 布局设置
layout.container_width => 容器宽度
layout.padding_h => 左右内边距

group:style => 样式设置
style.background_color => 背景颜色
style.text_color => 文字颜色
```

### 2. 响应式设计

使用 `responsive` 类型支持三端配置：

```php
// 配置定义
* layout.gap => 元素间距:responsive:20/30/40|px[MTD]

// 解析函数
function parseResponsive($value) {
    $parts = explode('/', trim($value));
    return [
        'mobile' => $parts[0] ?? '',
        'tablet' => $parts[1] ?? $parts[0] ?? '',
        'desktop' => $parts[2] ?? $parts[1] ?? $parts[0] ?? ''
    ];
}
```

### 3. 使用 clamp() 实现流式响应

```php
function genResponsiveCSS($parsed) {
    $mobile = floatval($parsed['mobile']);
    $desktop = floatval($parsed['desktop']);
    $unit = $parsed['unit'];
    
    if ($mobile === $desktop) {
        return $mobile . $unit;
    }
    
    $diff = $desktop - $mobile;
    return "clamp({$mobile}{$unit}, calc({$mobile}{$unit} + {$diff} * ((100vw - 375px) / 905)), {$desktop}{$unit})";
}
```

### 4. SEO 优化

确保在 header.phtml 中正确设置 SEO 标签：

```php
$metaTitle = $page ? $page->getData('meta_title') : '';
$metaDescription = $page ? $page->getData('meta_description') : '';
$metaKeywords = $page ? $page->getData('meta_keywords') : '';
```

### 5. 追踪代码集成

仅在非预览模式下加载追踪代码：

```php
$isPreview = strpos($this->getRequest()->getPathInfo(), '/preview/') !== false;

if (!$isPreview && $page && $page->getData('gtm_id')) {
    // 加载 GTM
}
```

## 故障排查

### 问题 1: 样式配置不生效

**原因**: 缓存未清理或配置解析错误

**解决方案**:
```bash
php bin/w cache:clear -f
```

### 问题 2: 模板路径找不到

**原因**: Style 模型的 `path` 字段为空或格式错误

**解决方案**:
```bash
# 重新扫描样式
php bin/w pagebuilder:style:scan
```

### 问题 3: 页面显示默认模板

**原因**: 页面未选择样式或样式被禁用

**解决方案**:
1. 在后台编辑页面，选择样式模板
2. 确保样式的 `is_active` 字段为 1
