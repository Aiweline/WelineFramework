# PageBuilder 预览生成失败 - 故障排查指南

## 问题描述
后台页面编辑器的预览生成功能失败，可能是 html2canvas 库加载问题。

## 症状
- 点击预览按钮后无响应
- 或者提示 "html2canvas is not defined"
- 浏览器控制台显示 JavaScript 错误

## 可能的原因

### 1. 静态文件路径错误
**检查方法**：
1. 打开后台页面编辑页面
2. 按 F12 打开浏览器开发者工具
3. 查看 Console 标签页
4. 查找调试信息：
   ```
   Static files paths:
   CKEditor: /GuoLaiRen/PageBuilder/view/statics/libs/ckeditor5/ckeditor.js
   html2canvas: /GuoLaiRen/PageBuilder/view/statics/libs/html2canvas/html2canvas.min.js
   html2canvas loaded: true/false
   ClassicEditor loaded: true/false
   ```

5. 查看 Network 标签页，检查这些文件是否返回 404

**解决方案A - 使用 @static 标签（推荐）**：
```php
<!-- 在 form.phtml 中 -->
<script src="@static(GuoLaiRen_PageBuilder::libs/html2canvas/html2canvas.min.js)"></script>
```

**解决方案B - 使用模块静态文件路径**：
```php
<script src="<?= $this->getStaticFile('libs/html2canvas/html2canvas.min.js', 'GuoLaiRen_PageBuilder') ?>"></script>
```

**解决方案C - 使用直接路径**：
```php
<script src="/GuoLaiRen/PageBuilder/view/statics/libs/html2canvas/html2canvas.min.js"></script>
```

### 2. 文件未正确下载
**检查方法**：
```bash
# Windows PowerShell
Get-Item app\code\GuoLaiRen\PageBuilder\view\statics\libs\html2canvas\html2canvas.min.js | Select-Object Length
```

**应该显示**：
- html2canvas.min.js: 约 194 KB (198,687 bytes)

**如果文件不存在或大小错误，重新下载**：
```powershell
Invoke-WebRequest -Uri "https://cdn.jsdelivr.net/npm/html2canvas@1.4.1/dist/html2canvas.min.js" -OutFile "app\code\GuoLaiRen\PageBuilder\view\statics\libs\html2canvas\html2canvas.min.js"
```

### 3. 文件加载顺序问题
html2canvas 必须在预览功能代码之前加载。

**检查**：
```html
<!-- 应该是这个顺序 -->
<script src=".../html2canvas.min.js"></script>
<!-- 然后才是使用 html2canvas 的代码 -->
<script>
  // 使用 html2canvas 的代码
</script>
```

### 4. 跨域问题
如果静态文件服务器与应用服务器不同，可能有跨域问题。

**解决方案**：
在 Nginx 配置中添加：
```nginx
location ~* \.(js|css|png|jpg|jpeg|gif|ico|woff|woff2|ttf|svg)$ {
    add_header Access-Control-Allow-Origin *;
    expires 1y;
}
```

### 5. 浏览器缓存问题
既有版本的文件可能被缓存。

**解决方案**：
1. 清除浏览器缓存
2. 强制刷新（Ctrl + F5 或 Cmd + Shift + R）
3. 或者在 URL 后添加版本号：
```php
<script src="<?= $this->getStaticFile('libs/html2canvas/html2canvas.min.js') ?>?v=<?= time() ?>"></script>
```

## 快速修复方案

### 方案1：使用 @static 标签（推荐）

修改 `app/code/GuoLaiRen/PageBuilder/view/templates/Backend/Page/form.phtml`:

```php
<!-- CKEditor 5 本地文件 -->
<script src="@static(GuoLaiRen_PageBuilder::libs/ckeditor5/ckeditor.js)"></script>
<script src="@static(GuoLaiRen_PageBuilder::libs/ckeditor5/zh-cn.js)"></script>

<!-- html2canvas for screenshot功能 -->
<script src="@static(GuoLaiRen_PageBuilder::libs/html2canvas/html2canvas.min.js)"></script>
```

**@static 标签的优势**：
- ✅ 自动处理模块路径
- ✅ 支持版本控制和缓存
- ✅ 框架推荐的标准方式
- ✅ 更简洁和易维护

### 方案2：添加错误处理

在预览功能代码前添加：

```javascript
// 检查 html2canvas 是否加载
if (typeof html2canvas === 'undefined') {
    console.error('html2canvas 未加载！');
    alert('预览功能需要的库文件未加载，请刷新页面重试。');
    // 尝试动态加载
    const script = document.createElement('script');
    script.src = '/GuoLaiRen/PageBuilder/view/statics/libs/html2canvas/html2canvas.min.js';
    script.onload = function() {
        console.log('html2canvas 已动态加载');
    };
    document.head.appendChild(script);
}
```

### 方案3：回退到 CDN（临时方案）

如果本地文件一直有问题，可以临时回退到 CDN：

```php
<!-- 从 CDN 加载（仅用于调试） -->
<script src="https://cdn.jsdelivr.net/npm/html2canvas@1.4.1/dist/html2canvas.min.js"></script>
```

## 完整的调试步骤

1. **打开浏览器开发者工具**
   - Chrome/Edge: F12
   - Firefox: F12
   - Safari: Cmd + Option + I

2. **查看 Console 标签**
   - 查找红色错误消息
   - 查找 "html2canvas" 相关的错误
   - 查看调试信息输出

3. **查看 Network 标签**
   - 刷新页面
   - 查找 html2canvas.min.js 的请求
   - 检查状态码（应该是 200，不应该是 404）
   - 查看文件大小（应该约 194 KB）

4. **测试 html2canvas 是否可用**
   在 Console 中输入：
   ```javascript
   typeof html2canvas
   ```
   应该显示 "function"，如果显示 "undefined" 则说明未加载

5. **手动测试预览功能**
   在 Console 中输入：
   ```javascript
   html2canvas(document.body).then(canvas => {
       console.log('预览生成成功', canvas);
   });
   ```
