## 📝 命名规范

### 部件名称

- **使用小写字母**：`default`, `custom-header`
- **使用连字符分隔**：`default-header` ✅，`defaultHeader` ❌
- **避免特殊字符**：不能包含空格、下划线（除连字符外）
- **有意义的名称**：使用描述性的名称，如 `product-list`, `user-profile`

### 部件类型

系统预定义的部件类型：

- `header` - 页面头部
- `content` - 内容区域
- `footer` - 页面底部
- `sidebar` - 侧边栏
- `banner` - 横幅广告
- `form` - 表单
- `list` - 列表
- `card` - 卡片
- `general` - 通用部件

**注意**：如需新增类型，请先与团队讨论。

### 文件命名

- **配置文件**：`widget.php`（固定名称）
- **模板文件**：`default.phtml` 或 `{name}.phtml`
- **文档文件**：`doc.md`（可选）
- **Block类**：使用PascalCase，如 `DefaultHeader.php`

## 📁 目录结构规范

### 标准结构

```
app/code/Weline/YourModule/
└── extends/
    └── Weline_Widget/
        └── {ModuleName}/
            └── {type}/
                └── {name}/
                    ├── widget.php          # 必需
                    ├── default.phtml       # 必需（如果未提供block_class）
                    ├── doc.md              # 可选
                    └── assets/             # 可选
                        ├── css/
                        │   └── style.css
                        └── js/
                            └── script.js
```

### 目录命名规则

- **模块名目录**：使用模块的完整名称，如 `Weline_Widget`
- **类型目录**：使用小写字母，如 `header`, `content`
- **部件目录**：使用小写字母和连字符，如 `default`, `custom-header`

### 示例

```
app/code/Weline/Widget/
└── extends/
    └── Weline_Widget/
        └── Weline_Widget/
            ├── header/
            │   ├── default/
            │   │   ├── widget.php
            │   │   └── default.phtml
            │   └── custom/
            │       ├── widget.php
            │       └── custom.phtml
            └── content/
                └── text/
                    ├── widget.php
                    └── text.phtml
```

## 📄 配置文件规范

### widget.php 必需字段

```php
<?php
declare(strict_types=1);
return [
    'name' => '...',           // 必需：部件显示名称
    'description' => '...',    // 必需：部件描述
    'type' => '...',           // 必需：部件类型
    'widget_name' => '...',    // 必需：部件名称（目录名）
    'module' => '...',         // 必需：来源模块
    'version' => '...',        // 必需：版本号
    'template' => '...',       // 必需（如果未提供block_class）：模板路径
    // 或
    'block_class' => '...',   // 必需（如果未提供template）：Block类名
    'params' => [...],        // 可选：参数定义
    'default_injections' => [ // 可选：默认建议注入到哪些布局/slot
        [
            'layout_type' => 'homepage',
            'layout_option' => 'default',
            'slot' => 'widget-hero',
            'area' => 'content',
            'sort_order' => 0,
            'required' => true,
            'reason' => '建议放在首页英雄区，便于运营一键应用',
        ],
    ],
    'doc' => '...'            // 可选：使用说明
];
```

### default_injections 默认注入

- `position` / `page_layouts` / `slot` / `supports` 表示“允许放置范围”。
- `default_injections` 表示“建议默认注入的位置”。
- 只有普通文件 Widget 第一次写入 `widget_registry_entry` 且带有默认注入项时，系统才会派发 `Weline_Widget::widget_install_after` 并自动写入一次；首次自动写入是布局级行为，会覆盖同一主题、同一 `layout_type` / `layout_option` 下已经存在的全部布局身份。旧的 `registry_refresh_after`、`module_install_after`、`module_upgrade_after` source 语义不再兼容默认注入。后续部件注册刷新、打开页面、渲染预览、保存布局都不会自动补回缺失部件。
- 自动写入由同一主题、同一布局身份、同一 `injection_key` 的已处理记录逐身份控制；部件库“应用”tab 只按当前布局身份缺失情况提示，运营删除后可选择“应用当前身份”或“应用全部身份”恢复。
- 运营移动到其他 slot 或调整配置后不会被自动纠正。

模板注释格式：

```php
@widget.default_injections {[{
  "layout_type":"homepage",
  "layout_option":"default",
  "slot":"widget-hero",
  "area":"content",
  "sort_order":0,
  "required":true,
  "reason":"建议放在首页英雄区，便于运营一键应用"
}]}
```

### 版本号规范

使用语义化版本号：`主版本号.次版本号.修订号`

- **主版本号**：不兼容的API修改
- **次版本号**：向下兼容的功能性新增
- **修订号**：向下兼容的问题修正

示例：`1.0.0`, `1.1.0`, `2.0.0`

### 模板路径格式

```
{ModuleName}::{相对路径}
```

示例：
- `Weline_Widget::widgets/header/default.phtml`
- `Weline_Theme::widgets/banner/slider.phtml`

### Block类名格式

完整的命名空间类名：

```
Weline\{Module}\Block\{Type}\{Name}
```

示例：
- `Weline\Widget\Block\Header\Default`
- `Weline\Theme\Block\Banner\Slider`

## 🎨 模板编写规范

### 基本要求

1. **使用PHP短标签**：`<?= ... ?>` 而不是 `<?php echo ... ?>`
2. **转义输出**：所有输出必须使用 `htmlspecialchars()`
3. **语义化HTML**：使用有意义的HTML标签
4. **响应式设计**：确保在不同设备上正常显示
5. **可访问性**：添加适当的ARIA属性

### 模板示例

```html
<!-- ✅ 正确 -->
<header class="widget-header">
    <div class="container">
        <h1 class="site-title">
            <?= htmlspecialchars($title ?? '默认标题') ?>
        </h1>
    </div>
</header>

<!-- ❌ 错误 -->
<header>
    <h1><?= $title ?></h1>  <!-- 未转义，存在XSS风险 -->
</header>
```

### 变量使用

- **检查变量存在**：使用 `??` 或 `isset()` 检查
- **提供默认值**：为所有变量提供合理的默认值
- **类型转换**：确保变量类型正确

```php
<?php
$title = $title ?? '默认标题';
$count = (int)($count ?? 0);
$showSearch = (bool)($show_search ?? true);
?>
```

### 国际化支持

所有用户可见文本必须使用翻译函数：

```php
<?= __('网站标题') ?>
<?= __('欢迎 %{name}', ['name' => $userName]) ?>
```

## 🔧 Block类规范

### 类结构

```php
<?php
declare(strict_types=1);
namespace Weline\Widget\Block\Header;

use Weline\Framework\View\Block;

class Default extends Block
{
    protected string $_template = 'Weline_Widget::widgets/header/default.phtml';
    
    public function __init()
    {
        parent::__init();
        // 初始化逻辑
    }
    
    // 其他方法
}
```

### 方法命名

- **公共方法**：使用camelCase，如 `getData()`, `setTitle()`
- **私有方法**：使用camelCase，前缀 `_` 可选，如 `_processData()`
- **获取器**：前缀 `get`，如 `getTitle()`
- **设置器**：前缀 `set`，如 `setTitle()`

### 数据验证

在 `__init()` 方法中验证和处理数据：

```php
public function __init()
{
    parent::__init();
    
    // 验证必需参数
    $title = $this->getData('title');
    if (empty($title)) {
        $title = '默认标题';
    }
    
    // 类型转换
    $count = (int)$this->getData('count', 0);
    
    // 设置到模板
    $this->assign('title', $title);
    $this->assign('count', $count);
}
```

### 错误处理

```php
try {
    // 可能出错的操作
    $data = $this->fetchData();
} catch (\Exception $e) {
    // 记录错误
    error_log("Widget错误: " . $e->getMessage());
    // 使用默认值
    $data = [];
}
```

## 📋 参数定义规范

### 参数类型

支持的类型：

- `string` - 文本
- `number` - 数字
- `boolean` - 布尔值
- `select` - 选择
- `url` - URL地址
- `email` - 邮箱地址
- `color` - 颜色值

### 参数定义格式

```php
'params' => [
    'title' => [
        'type' => 'string',           // 必需：参数类型
        'label' => '标题',             // 必需：参数标签
        'default' => '默认标题',       // 可选：默认值
        'placeholder' => '请输入标题', // 可选：占位符
        'required' => true,           // 可选：是否必填
        'description' => '页面标题'    // 可选：参数说明
    ],
    'count' => [
        'type' => 'number',
        'label' => '显示数量',
        'default' => 10,
        'min' => 1,                   // 可选：最小值
        'max' => 100                   // 可选：最大值
    ],
    'show_title' => [
        'type' => 'boolean',
        'label' => '显示标题',
        'default' => true
    ],
    'layout' => [
        'type' => 'select',
        'label' => '布局方式',
        'default' => 'grid',
        'options' => [                 // 必需（select类型）
            'grid' => '网格布局',
            'list' => '列表布局'
        ]
    ]
]
```

### 参数命名

- **使用小写字母和下划线**：`show_title`, `logo_url`
- **有意义的名称**：使用描述性的名称
- **避免缩写**：除非是通用缩写（如 `url`, `id`）

## ✅ 代码质量规范

### PHP代码

1. **严格类型声明**：`declare(strict_types=1);`
2. **PSR标准**：遵循PSR-1和PSR-12编码规范
3. **注释**：为复杂逻辑添加注释
4. **错误处理**：适当的异常处理
5. **性能优化**：避免不必要的数据库查询

### JavaScript代码

1. **ES6+语法**：使用现代JavaScript语法
2. **变量声明**：使用 `const` 和 `let`，避免 `var`
3. **函数命名**：使用camelCase
4. **注释**：为复杂逻辑添加注释

### CSS代码

1. **类命名**：使用BEM或类似规范
2. **响应式**：使用媒体查询
3. **浏览器兼容**：考虑主流浏览器兼容性
