## ❓ 使用问题

### Q1: 保存页面时提示"页面标识已存在"？

**原因**：页面标识（handle）在系统中必须唯一。

**解决方案**：
1. 修改页面标识为其他唯一值
2. 如果是要编辑已有页面，确保URL中包含正确的 `page_id` 参数

### Q2: 部件在编辑器中不显示？

**可能原因**：
1. 部件未正确注册
2. 部件配置文件格式错误
3. 部件目录结构不正确

**排查步骤**：
1. 检查部件目录结构是否正确
2. 检查 `widget.php` 文件格式是否正确
3. 查看浏览器控制台是否有错误
4. 访问 **部件管理** 页面查看部件是否被扫描到

### Q3: 部件参数修改不生效？

**可能原因**：
1. 参数格式错误（JSON格式）
2. 参数类型不匹配
3. 未点击应用按钮

**解决方案**：
1. 检查参数是否为有效的JSON格式
2. 确保参数类型与定义一致（文本、数字、布尔值）
3. 修改参数后等待自动更新，或手动刷新预览

### Q4: 预览页面显示空白？

**可能原因**：
1. 页面内容为空
2. 部件渲染出错
3. 模板文件不存在

**排查步骤**：
1. 检查页面内容是否包含 `w:widget` 标签
2. 查看浏览器控制台错误信息
3. 检查部件模板文件是否存在
4. 查看PHP错误日志

### Q5: 如何删除已保存的页面？

**当前限制**：编辑器界面暂不支持删除功能。

**解决方案**：
1. 通过数据库直接删除（需要数据库权限）
2. 通过后台管理功能删除（如果已实现）
3. 联系系统管理员

## 🔧 开发问题

### Q6: 如何创建自定义部件？

**步骤**：
1. 在模块的 `extends/Weline_Widget/` 目录下创建部件目录
2. 创建 `widget.php` 配置文件
3. 创建模板文件或Block类
4. 在编辑器中测试

**详细说明**：参考 [开发指南](./开发指南.md)

### Q7: 部件模板中如何使用参数？

**方法1：直接使用变量**

```php
<?= htmlspecialchars($title ?? '默认标题') ?>
```

**方法2：从params数组获取**

```php
<?php
$title = $params['title'] ?? '默认标题';
?>
<?= htmlspecialchars($title) ?>
```

### Q8: 如何创建需要数据库查询的部件？

**使用Block类**：

```php
<?php
namespace Weline\Widget\Block\List;

use Weline\Framework\View\Block;
use Weline\YourModule\Model\YourModel;

class Article extends Block
{
    protected string $_template = 'Weline_Widget::widgets/list/article.phtml';
    
    public function __init()
    {
        parent::__init();
        
        $model = new YourModel();
        $articles = $model->select()
            ->limit($this->getData('limit', 10))
            ->fetch();
        
        $this->assign('articles', $articles);
    }
}
```

### Q9: 部件如何支持多语言？

**在模板中使用翻译函数**：

```php
<?= __('网站标题') ?>
<?= __('欢迎 %{name}', ['name' => $userName]) ?>
```

**在widget.php中定义参数标签**：

```php
'params' => [
    'title' => [
        'type' => 'string',
        'label' => __('标题'),  // 使用翻译函数
        'default' => __('默认标题')
    ]
]
```

### Q10: 如何添加部件的CSS和JavaScript？

**方法1：在模板中直接引入**

```html
<link rel="stylesheet" href="/static/widgets/header/default.css">
<script src="/static/widgets/header/default.js"></script>
```

**方法2：使用assets目录**

```
widget/
└── default/
    ├── widget.php
    ├── default.phtml
    └── assets/
        ├── css/
        │   └── style.css
        └── js/
            └── script.js
```

## 🐛 故障排查

### Q11: 部件扫描失败？

**检查清单**：
1. ✅ 模块是否已启用
2. ✅ 部件目录结构是否正确
3. ✅ `widget.php` 文件是否存在且格式正确
4. ✅ 文件权限是否正确

**手动测试**：

```php
use Weline\Widget\Service\WidgetScanner;
use Weline\Framework\Manager\ObjectManager;

$scanner = ObjectManager::getInstance(WidgetScanner::class);
$widgets = $scanner->scanAllWidgets();
var_dump($widgets);
```

### Q12: w:widget 标签不渲染？

**检查清单**：
1. ✅ 标签格式是否正确
2. ✅ 部件类型和名称是否存在
3. ✅ 参数JSON格式是否正确
4. ✅ 模板文件是否存在

**调试方法**：

在模板中直接输出：

```php
<?php
// 检查变量
var_dump($type, $name, $params);
?>
```

### Q13: Block类不工作？

**检查清单**：
1. ✅ Block类命名空间是否正确
2. ✅ Block类是否继承自 `Weline\Framework\View\Block`
3. ✅ `_template` 属性是否正确设置
4. ✅ `__init()` 方法是否正确调用 `parent::__init()`

### Q14: 参数传递失败？

**检查JSON格式**：

```javascript
// ✅ 正确
params='{"title":"我的网站"}'

// ❌ 错误 - 引号未转义
params='{"title":"我的网站"}'  // 在HTML中会出错

// ✅ 正确 - 使用单引号包裹
params='{"title":"我的网站"}'
```

## ⚡ 性能优化

### Q15: 部件加载慢？

**优化建议**：
1. **缓存部件配置**：系统已自动缓存，无需手动处理
2. **减少数据库查询**：在Block类中优化查询逻辑
3. **延迟加载**：对于非关键部件，使用延迟加载
4. **压缩资源**：压缩CSS和JavaScript文件

### Q16: 页面渲染慢？

**优化建议**：
1. **减少部件数量**：避免在一个页面中使用过多部件
2. **优化模板**：减少模板中的复杂逻辑
3. **使用缓存**：对静态内容使用缓存
4. **CDN加速**：静态资源使用CDN

### Q17: 编辑器响应慢？

**优化建议**：
1. **减少部件数量**：部件库中部件过多时，使用搜索过滤
2. **浏览器缓存**：确保浏览器缓存正常工作
3. **网络优化**：检查网络连接
