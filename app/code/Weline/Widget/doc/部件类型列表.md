# 部件类型列表

## 📋 概述

本文档列出了 Widget 模块支持的所有部件类型，以及每种类型的用途和推荐场景。

## 🎯 标准部件类型

### header - 页面头部

**用途**：网站或页面的顶部区域

**典型内容**：
- Logo
- 导航菜单
- 搜索框
- 用户登录信息

**示例**：
```html
<w:widget type="header" name="default" params='{"title":"我的网站","logo_url":"/static/logo.png"}' />
```

### content - 内容区域

**用途**：页面的主要内容区域

**典型内容**：
- 文章内容
- 产品介绍
- 文本内容
- 富文本内容

**示例**：
```html
<w:widget type="content" name="text" params='{"text":"这是内容文本"}' />
```

### footer - 页面底部

**用途**：网站或页面的底部区域

**典型内容**：
- 版权信息
- 友情链接
- 联系方式
- 备案信息

**示例**：
```html
<w:widget type="footer" name="default" params='{"copyright":"© 2025 我的网站"}' />
```

### sidebar - 侧边栏

**用途**：页面侧边的内容区域

**典型内容**：
- 导航菜单
- 相关文章
- 广告位
- 标签云

**示例**：
```html
<w:widget type="sidebar" name="navigation" params='{"menu_items":[...]}' />
```

### banner - 横幅广告

**用途**：展示横幅广告或轮播图

**典型内容**：
- 轮播图片
- 广告横幅
- 促销信息

**示例**：
```html
<w:widget type="banner" name="slider" params='{"images":[...]}' />
```

### form - 表单

**用途**：各种表单组件

**典型内容**：
- 联系表单
- 搜索表单
- 登录表单
- 注册表单

**示例**：
```html
<w:widget type="form" name="contact" params='{"action":"/contact/submit"}' />
```

### list - 列表

**用途**：展示列表数据

**典型内容**：
- 文章列表
- 产品列表
- 用户列表
- 新闻列表

**示例**：
```html
<w:widget type="list" name="article" params='{"category":"news","limit":10}' />
```

### card - 卡片

**用途**：卡片式内容展示

**典型内容**：
- 产品卡片
- 用户卡片
- 信息卡片
- 功能卡片

**示例**：
```html
<w:widget type="card" name="product" params='{"product_id":123}' />
```

### general - 通用部件

**用途**：不属于以上类型的通用部件

**典型内容**：
- 自定义组件
- 特殊功能模块
- 第三方集成

**示例**：
```html
<w:widget type="general" name="custom" params='{"config":{...}}' />
```

## 📊 类型选择指南

### 按页面位置选择

| 页面位置 | 推荐类型 |
|---------|---------|
| 页面顶部 | `header` |
| 页面主体 | `content`, `list`, `card` |
| 页面侧边 | `sidebar` |
| 页面底部 | `footer` |
| 页面顶部下方 | `banner` |

### 按功能选择

| 功能需求 | 推荐类型 |
|---------|---------|
| 展示文本内容 | `content` |
| 展示列表数据 | `list` |
| 展示卡片信息 | `card` |
| 用户输入 | `form` |
| 广告展示 | `banner` |
| 导航菜单 | `header`, `sidebar` |

## 🔄 扩展类型

如果需要创建新的部件类型，请遵循以下步骤：

1. **评估需求**：确认新类型是否必要，或可以使用现有类型
2. **团队讨论**：与团队讨论新类型的定义和用途
3. **更新文档**：在本文档中添加新类型的说明
4. **创建示例**：创建该类型的示例部件

## 📝 类型命名规范

- **使用小写字母**：`header`, `content`, `footer`
- **使用单数形式**：`form` 而不是 `forms`
- **有意义的名称**：名称应该清晰表达用途
- **避免缩写**：除非是通用缩写

## ⚠️ 注意事项

1. **类型一致性**：同一功能的部件应使用相同的类型
2. **类型选择**：优先使用标准类型，避免创建过多自定义类型
3. **文档更新**：新增类型时及时更新本文档
