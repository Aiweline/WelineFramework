# Weline_Order 模块 i18n 国际化使用指南

## 概述

`Weline_Order` 模块已完整接入框架的 i18n 国际化系统，所有用户可见文本（后台页面、API 提示、异常消息、Hook/Extends 描述等）均通过 `__()` 翻译函数输出，并由模块内的 `i18n` 目录提供翻译。

## 翻译文件位置

### 目录结构

```text
app/code/Weline/Order/
├── i18n/
│   ├── zh_Hans_CN.csv    # 中文（基础语言）
│   └── en_US.csv        # 英文
└── ...
```

- **`zh_Hans_CN.csv`**：中文翻译文件，源文本与翻译文本通常相同。
- **`en_US.csv`**：英文翻译文件，为中文源文本提供英文翻译。

### CSV 文件格式

```csv
"源文本","翻译文本"
"订单管理","订单管理"
"订单管理","Order Management"
```

- 第一列：代码中 `__()` 的中文原文。
- 第二列：对应语言的翻译文本。
- 使用双引号包裹，逗号分隔。

> 注意：**模块级翻译文件只放在 `app/code/Weline/Order/i18n/` 下，无需再注册语言包。**

## 在订单模块中如何使用翻译

### 1. PHP 控制器中使用

以后台订单控制器为例（`Controller/Backend/Order.php`）：

```php
// 消息提示
$this->getMessageManager()->addError(__('订单ID不能为空'));
$this->getMessageManager()->addSuccess(__('订单取消成功'));

// 视图变量
$this->assign('title', __('订单管理'));
```

后台其它控制器（`Payment.php`、`Shipment.php`、`Refund.php`、`Invoice.php`）同样使用 `__()` 输出所有提示与文案。

### 2. API 控制器中使用

`Controller/Api/Order.php` 中，所有接口返回消息也使用翻译函数：

```php
// 成功返回
return $this->success(__('获取订单列表成功'), [
    'items' => $items,
    'total' => $total,
]);

// 参数校验失败
if (empty($orderId)) {
    return $this->error(__('订单ID不能为空'), '', 400);
}
```

这样前端在切换语言后，API 返回的 `message` 字段会自动显示为对应语言。

### 3. 服务层与异常消息

订单服务与状态机中的异常使用中文为基础文本，同样走翻译系统：

```php
// OrderService.php
if (empty($data['items'])) {
    throw new \Exception(__('订单项不能为空'));
}

// 验证必填字段
if (empty($data[$field])) {
    throw new \Exception(__('%{1}不能为空', [$label]));
}

// OrderStateMachine.php
if (!$this->canTransition($currentStatus, $newStatus)) {
    throw new \Exception(__('订单状态不能从 %{1} 转换到 %{2}', [$currentStatus, $newStatus]));
}
```

在英文环境下，这些异常提示会根据 `en_US.csv` 中的配置显示为英文。

### 4. 后台模板（phtml）中使用

后台订单列表与详情模板（`view/templates/Backend/Order/*.phtml`）全部通过 `__()` 输出界面文案：

```php
<title><?= __('订单管理') ?></title>

<!-- 面包屑 -->
<li class="breadcrumb-item"><a href="@backend-url('admin/index/index')"><?= __('系统') ?></a></li>
<li class="breadcrumb-item active"><?= __('订单管理') ?></li>

<!-- 表头 -->
<th><?= __('订单号') ?></th>
<th><?= __('客户') ?></th>
<th><?= __('订单总额') ?></th>
<th><?= __('订单状态') ?></th>

<!-- 表单标签 -->
<label class="form-label"><?= __('客户姓名') ?></label>
<button type="submit" class="btn btn-primary"><?= __('保存') ?></button>
```

当语言切换为英文时，这些标题、按钮和标签会根据 `en_US.csv` 自动显示英文。

### 5. Hook / Extends 文案

`hook.php` 和 `extends.php` 中的名称与描述也全部使用翻译函数：

```php
// hook.php
return [
    'Weline_Order::backend::order::view::before' => [
        'name' => __('订单详情页之前'),
        'description' => __('在订单详情页面内容之前注入内容，允许其他模块在订单详情页顶部添加自定义内容。'),
    ],
    // ...
];

// extends.php
'Weline_Order::payment_methods' => [
    'name' => __('支付方式扩展'),
    'description' => __('允许其他模块注册新的支付方式。'),
],
```

这保证了在后台扩展配置界面中，这些扩展点的说明也能被翻译。

## 占位符使用规范

订单模块遵循框架的占位符规范，只使用以下三种形式：

- **通用占位符**：`%{}` —— 单个参数  
- **数字占位符**：`%{1}`, `%{2}` —— 多个有顺序的参数  
- **命名占位符**：`%{name}`, `%{count}` —— 推荐，语义清晰

### 示例：订单相关文案

```php
// 数字占位符
throw new \Exception(__('订单状态不能从 %{1} 转换到 %{2}', [$currentStatus, $newStatus]));

// 单参数
throw new \Exception(__('%{1}不能为空', [$label]));
```

对应英文翻译（`en_US.csv`）：

```csv
"订单状态不能从 %{1} 转换到 %{2}","Order status cannot transition from %{1} to %{2}"
"%{1}不能为空","%{1} cannot be empty"
```

## 如何新增翻译文本

### 步骤 1：在代码中使用 `__()`

在任何需要新增文案的地方，直接写中文 + 翻译函数，例如：

```php
// 新增一个错误提示
throw new \Exception(__('订单已锁定，无法修改'));
```

### 步骤 2：更新 `i18n` CSV 文件

在 `i18n/zh_Hans_CN.csv` 中添加一行：

```csv
"订单已锁定，无法修改","订单已锁定，无法修改"
```

在 `i18n/en_US.csv` 中添加对应英文：

```csv
"订单已锁定，无法修改","Order is locked and cannot be modified"
```

### 步骤 3：清理缓存并验证

```bash
php bin/w cache:clear
```

然后切换后台语言为英文，触发该逻辑，确认提示显示为英文。

## 如何新增语言

如果希望为订单模块添加更多语言，只需在 `i18n/` 目录下新增 CSV 文件，例如：

```text
app/code/Weline/Order/i18n/
├── zh_Hans_CN.csv
├── en_US.csv
└── ja_JP.csv        # 日文
```

`ja_JP.csv` 示例：

```csv
"订单管理","注文管理"
"订单列表","注文一覧"
```

> 语言代码需符合框架约定（如 `ja_JP`, `fr_FR` 等），并确保系统已安装对应语言包。

## 与框架 i18n 工具配合

订单模块的翻译词条也可以通过框架命令行工具自动收集和更新：

```bash

php bin/w i18n:collect Weline_Order
```

命令会扫描：

- `app/code/Weline/Order/**/*.phtml`

中的 `__()` 调用，并自动更新 `i18n/zh_Hans_CN.csv` / `en_US.csv`。

## 订单状态翻译

订单模块中的所有状态值（订单状态、支付状态、发货状态）都支持国际化翻译。

### 状态翻译方法

`Order` 模型提供了三个静态方法来获取状态的翻译文本：

```php
use Weline\Order\Model\Order;

// 获取订单状态翻译
$statusLabel = Order::getStatusLabel('pending');  // 返回：'待处理' 或 'Pending'

// 获取支付状态翻译
$paymentLabel = Order::getPaymentStatusLabel('paid');  // 返回：'已支付' 或 'Paid'

// 获取发货状态翻译
$fulfillmentLabel = Order::getFulfillmentStatusLabel('shipped');  // 返回：'已发货' 或 'Shipped'
```

### 在模板中使用

在后台模板中，应使用这些方法来显示状态，而不是直接显示状态值：

```php
<!-- 正确：使用翻译方法 -->
<span class="badge"><?= htmlspecialchars(Order::getStatusLabel($order->getStatus())) ?></span>

<!-- 错误：直接显示状态值 -->
<span class="badge"><?= htmlspecialchars($order->getStatus()) ?></span>
```

### 支持的状态值

#### 订单状态（Order Status）

| 状态值 | 中文 | 英文 |
|--------|------|------|
| `pending` | 待处理 | Pending |
| `processing` | 处理中 | Processing |
| `paid` | 已支付 | Paid |
| `fulfilled` | 已发货 | Fulfilled |
| `completed` | 已完成 | Completed |
| `cancelled` | 已取消 | Cancelled |
| `refunded` | 已退款 | Refunded |

#### 支付状态（Payment Status）

| 状态值 | 中文 | 英文 |
|--------|------|------|
| `pending` | 待支付 | Pending Payment |
| `paid` | 已支付 | Paid |
| `partial` | 部分支付 | Partial Payment |
| `refunded` | 已退款 | Refunded |

#### 发货状态（Fulfillment Status）

| 状态值 | 中文 | 英文 |
|--------|------|------|
| `pending` | 待发货 | Pending Shipment |
| `partial` | 部分发货 | Partial Shipment |
| `shipped` | 已发货 | Shipped |
| `delivered` | 已送达 | Delivered |

### 状态翻译的实现

状态翻译通过 `Order` 模型中的静态方法实现，这些方法内部使用 `__()` 函数：

```php
public static function getStatusLabel(string $status): string
{
    return match($status) {
        self::STATUS_PENDING => __('待处理'),
        self::STATUS_PROCESSING => __('处理中'),
        // ...
        default => $status,
    };
}
```

因此，所有状态文本都会根据当前语言环境自动翻译。

### 在 API 中使用

如果需要在 API 响应中返回翻译后的状态，可以这样使用：

```php
$orderData = $order->getData();
$orderData['status_label'] = Order::getStatusLabel($orderData['status']);
$orderData['payment_status_label'] = Order::getPaymentStatusLabel($orderData['payment_status']);
$orderData['fulfillment_status_label'] = Order::getFulfillmentStatusLabel($orderData['fulfillment_status']);

return $this->success(__('获取订单详情成功'), $orderData);
```

这样前端可以根据语言环境显示对应的状态文本。

## 事件机制扩展状态翻译

订单模块使用事件机制解耦状态翻译逻辑，允许其他模块通过监听事件提供自定义的状态翻译。

### 可用事件

订单模块定义了以下状态相关事件：

1. **查询事件（Query Events）**
   - `Weline_Order::query::get_status_label` - 获取订单状态标签
   - `Weline_Order::query::get_status_class` - 获取订单状态CSS类
   - `Weline_Order::query::get_payment_status_label` - 获取支付状态标签
   - `Weline_Order::query::get_fulfillment_status_label` - 获取发货状态标签

2. **领域事件（Domain Events）**
   - `Weline_Order::domain::resolve_status_info` - 解析完整状态信息（标签、类、图标等）

### 如何监听事件

其他模块可以通过创建观察者来监听这些事件，提供自定义的状态翻译：

```php
namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CustomOrderStatusObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $eventName = $event->getName();
        $data = $event->getData();
        
        if ($eventName === 'Weline_Order::query::get_status_label') {
            $status = $data['status'] ?? '';
            
            // 提供自定义翻译
            if ($status === 'custom_status') {
                $event->setData('label', __('自定义状态'));
            }
        }
    }
}
```

然后在 `etc/event.xml` 中注册观察者：

```xml
<event name="Weline_Order::query::get_status_label">
    <observer name="Your_Module::custom_status_observer" 
              instance="Your\Module\Observer\CustomOrderStatusObserver" 
              disabled="false" 
              shared="true" 
              sort="20"/>
</event>
```

### 事件数据契约

所有状态查询事件都遵循以下数据契约：

**输入参数**：
- `status` (string, required) - 状态代码
- `locale` (string, optional) - 目标语言代码

**输出结果**（由观察者填充）：
- `label` (string) - 状态标签翻译
- `class` (string) - CSS类名（仅用于 `get_status_class` 事件）

### 事件优先级

- `OrderStatusObserver` 的优先级为 10（默认）
- 其他模块的观察者可以设置更高的优先级（如 20、30）来覆盖默认翻译
- 如果多个观察者都提供了翻译，最后一个设置的会生效

### 使用示例

在模板中使用状态方法（内部已使用事件机制）：

```php
// 获取状态标签
$statusLabel = Order::getStatusLabel('pending');

// 获取状态CSS类
$statusClass = Order::getStatusClassByCode('pending');

// 使用辅助类获取完整信息
$statusInfo = StatusHelper::resolveStatusInfo('pending');
// 返回: ['label' => '待处理', 'class' => 'warning', 'icon' => '...', ...]
```

## 使用检查清单（针对 Weline_Order）

为订单模块开发新功能时，请确认：

- **控制器**：所有消息（成功/失败/警告）均使用 `__()`。
- **模板**：所有标题、按钮、表头、占位符文字均使用 `__()`。
- **状态显示**：使用 `Order::getStatusLabel()` 等方法显示状态，不要直接显示状态值。
- **事件机制**：不要直接调用 `OrderStatusService`，使用 `Order` 模型的方法或事件机制。
- **服务/异常**：抛出的业务异常消息使用中文 + `__()`。
- **Hook/Extends**：`name` 和 `description` 均使用 `__()`。
- **CSV 文件**：`zh_Hans_CN.csv` 与 `en_US.csv` 中都存在对应条目。
- **占位符**：严格使用 `%{}` / `%{1}` / `%{name}` 格式，不使用 `%1`。

满足以上条件，即可保证 `Weline_Order` 模块在多语言环境下正常工作，并且支持通过事件机制扩展状态翻译。
