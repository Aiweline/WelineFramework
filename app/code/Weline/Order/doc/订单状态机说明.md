# 订单状态机说明

## 状态定义

订单状态机定义了订单从创建到完成的完整生命周期状态：

```
pending (待处理)
  ↓
processing (处理中)
  ↓
  ├─→ paid (已支付)
  │     ↓
  │   fulfilled (已发货)
  │     ↓
  │   completed (已完成)
  │
  ├─→ cancelled (已取消)
  │
  └─→ refunded (已退款)
```

## 状态转换规则

### 允许的转换

| 当前状态 | 可转换到 | 说明 |
|---------|---------|------|
| pending | processing, cancelled | 订单确认或取消 |
| processing | paid, cancelled | 支付成功或取消 |
| paid | fulfilled, refunded | 发货或退款 |
| fulfilled | completed | 确认收货 |
| processing | cancelled | 处理中可取消 |
| paid | refunded | 已支付可退款 |

### 状态常量

```php
use Weline\Order\Model\Order;

Order::STATUS_PENDING      // 待处理
Order::STATUS_PROCESSING   // 处理中
Order::STATUS_PAID         // 已支付
Order::STATUS_FULFILLED    // 已发货
Order::STATUS_COMPLETED    // 已完成
Order::STATUS_CANCELLED    // 已取消
Order::STATUS_REFUNDED     // 已退款
```

## 使用示例

### 检查状态转换是否允许

```php
use Weline\Framework\Manager\ObjectManager;
use Weline\Order\Service\OrderStateMachine;

$stateMachine = ObjectManager::getInstance(OrderStateMachine::class);

$canTransition = $stateMachine->canTransition('pending', 'processing');
// 返回: true
```

### 执行状态转换

```php
$stateMachine->transition($orderId, 'processing', '订单已确认', false);
```

### 获取可用状态转换

```php
$availableTransitions = $stateMachine->getAvailableTransitions('pending');
// 返回: ['processing', 'cancelled']
```

## 状态说明

### pending (待处理)
- **说明**: 订单刚创建，等待处理
- **可执行操作**: 确认订单、取消订单

### processing (处理中)
- **说明**: 订单已确认，正在处理中
- **可执行操作**: 处理支付、取消订单

### paid (已支付)
- **说明**: 订单已支付，等待发货
- **可执行操作**: 创建发货记录、申请退款

### fulfilled (已发货)
- **说明**: 订单已发货，等待送达
- **可执行操作**: 标记为已送达

### completed (已完成)
- **说明**: 订单已完成，客户已收货
- **可执行操作**: 无（终态）

### cancelled (已取消)
- **说明**: 订单已取消
- **可执行操作**: 无（终态）

### refunded (已退款)
- **说明**: 订单已退款
- **可执行操作**: 无（终态）

## 事件扩展

订单状态机通过事件系统支持动态扩展，允许其他模块监听和扩展状态转换逻辑。

### 状态转换事件

#### 1. 状态转换前事件 (`Weline_Order::order_status_change_before`)

在状态转换执行前触发，允许观察者阻止转换。

**事件数据**:
```php
[
    'order' => Order对象,
    'order_id' => int,
    'old_status' => string,
    'new_status' => string,
    'comment' => string|null,
    'notify_customer' => bool,
    'can_change' => bool, // 观察者可以设置为false来阻止转换
]
```

**示例：阻止已发货订单取消**
```php
<?php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class OrderStatusChangeBeforeObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        if ($data['old_status'] === 'fulfilled' && $data['new_status'] === 'cancelled') {
            $data['can_change'] = false;
            $event->setData($data);
        }
    }
}
```

#### 2. 状态转换后事件 (`Weline_Order::order_status_changed`)

在状态转换执行后触发，用于处理状态变更后的逻辑。

**事件数据**:
```php
[
    'order' => Order对象,
    'order_id' => int,
    'old_status' => string,
    'new_status' => string,
    'comment' => string|null,
    'notify_customer' => bool,
]
```

**默认处理**:
- 记录订单历史（由 `OrderStatusChangedObserver` 处理）
- 发送客户通知（如果需要）

#### 3. 状态转换规则检查事件 (`Weline_Order::order_status_can_transition`)

在检查状态转换规则时触发，允许观察者动态扩展转换规则。

**事件数据**:
```php
[
    'from_status' => string,
    'to_status' => string,
    'can_transition' => bool, // 观察者可以修改此值
    'transitions' => array, // 观察者可以添加转换规则
]
```

**示例：动态添加转换规则**
```php
<?php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class OrderStatusTransitionRulesObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $transitions = $data['transitions'];
        
        // 添加自定义转换规则
        if (!isset($transitions['custom_status'])) {
            $transitions['custom_status'] = ['completed', 'cancelled'];
            $data['transitions'] = $transitions;
        }
        
        // 检查是否允许转换
        if ($data['from_status'] === 'custom_status') {
            $data['can_transition'] = in_array($data['to_status'], $transitions['custom_status']);
        }
        
        $event->setData($data);
    }
}
```

### 状态管理事件

#### 1. 状态保存前事件 (`Weline_Order::order_status_save_before`)

在状态保存前触发，允许观察者进行验证或修改。

#### 2. 状态保存后事件 (`Weline_Order::order_status_saved`)

在状态保存后触发，用于处理保存后的逻辑（如清除缓存）。

#### 3. 状态删除前事件 (`Weline_Order::order_status_delete_before`)

在状态删除前触发，允许观察者阻止删除。

#### 4. 状态删除后事件 (`Weline_Order::order_status_deleted`)

在状态删除后触发，用于处理删除后的逻辑（如清理相关数据）。

### 注册观察者

在 `etc/event.xml` 中注册观察者：

```xml
<event name="Weline_Order::order_status_change_before">
    <observer name="YourModule::order_status_change_before_observer" 
              instance="Weline\YourModule\Observer\OrderStatusChangeBeforeObserver" 
              disabled="false" 
              shared="true" 
              sort="10"/>
</event>
```

## 扩展状态

可以通过以下方式扩展自定义状态：

1. **通过数据库管理界面**：在后台"订单管理 > 订单状态管理"中创建自定义状态
2. **通过Extends系统**：详见 `extends.php` 中的 `OrderStatuses` 扩展点
3. **通过事件系统**：监听 `Weline_Order::order_status_can_transition` 事件，动态添加转换规则
