# 订单状态翻译功能说明

## 功能概述

订单状态翻译功能允许您：
1. **自定义订单状态**：创建新的订单状态，状态代码（code）保持不变
2. **多语言翻译**：为每个状态配置不同语言的显示名称
3. **状态管理**：通过后台界面管理所有订单状态，包括系统状态和自定义状态

## 核心特性

### 1. 状态代码（Code）与显示名称（Name）分离

- **状态代码（Code）**：用于数据库存储和程序逻辑，如 `pending`、`processing`、`paid` 等
- **显示名称（Name）**：用于前端显示，可以翻译成不同语言，如"待处理"、"Pending"等

### 2. 多语言翻译机制

状态名称支持三种翻译方式（按优先级）：

1. **数据库翻译表**：在状态管理界面中配置的翻译（`weline_order_status_translation`）
2. **翻译文件**：在 `i18n/zh_Hans_CN.csv` 和 `i18n/en_US.csv` 中配置的翻译（key格式：`order_status_{code}`）
3. **默认名称**：状态定义中的默认名称（name字段）

### 3. 状态管理功能

- 创建自定义状态
- 编辑状态配置（代码、名称、颜色、图标等）
- 启用/禁用状态
- 删除自定义状态（系统状态不能删除）
- 配置多语言翻译

## 数据库结构

### 订单状态表（weline_order_status）

| 字段 | 类型 | 说明 |
|------|------|------|
| status_id | INT | 主键 |
| code | VARCHAR(50) | 状态代码（唯一） |
| name | VARCHAR(100) | 状态名称（默认语言） |
| description | TEXT | 状态描述 |
| color | VARCHAR(20) | 状态颜色（用于UI显示） |
| icon | VARCHAR(50) | 状态图标 |
| is_system | INT | 是否系统状态（1-是，0-否） |
| is_active | INT | 是否启用（1-是，0-否） |
| sort_order | INT | 排序 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 订单状态翻译表（weline_order_status_translation）

| 字段 | 类型 | 说明 |
|------|------|------|
| translation_id | INT | 主键 |
| status_code | VARCHAR(50) | 状态代码 |
| locale | VARCHAR(20) | 语言代码（如：zh_Hans_CN, en_US） |
| name | VARCHAR(100) | 状态名称（翻译后） |
| description | TEXT | 状态描述（翻译后） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## 使用方法

### 1. 在代码中获取状态翻译

#### 使用Order模型的静态方法

```php
use Weline\Order\Model\Order;

// 获取状态的翻译文本
$statusText = Order::getStatusLabel('pending'); // 返回："待处理" 或 "Pending"

// 在订单对象上使用
$order = new Order();
$order->load($orderId);
$statusText = $order->getStatusText(); // 获取当前订单状态的翻译文本
$statusClass = $order->getStatusClass(); // 获取状态的CSS类名
```

#### 使用OrderStatusService

```php
use Weline\Order\Service\OrderStatusService;
use Weline\Framework\Manager\ObjectManager;

$statusService = ObjectManager::getInstance(OrderStatusService::class);

// 获取状态名称（自动翻译）
$statusName = $statusService->getStatusName('pending'); // 返回："待处理" 或 "Pending"

// 获取指定语言的翻译
$statusName = $statusService->getStatusName('pending', 'en_US'); // 返回："Pending"

// 获取状态的CSS类名
$statusClass = $statusService->getStatusClass('pending'); // 返回："warning"

// 获取状态的图标
$statusIcon = $statusService->getStatusIcon('pending'); // 返回："mdi-clock-outline"
```

### 2. 在模板中显示状态

#### 后端模板

```php
<?php
use Weline\Order\Model\Order;

$status = $order->getData(Order::fields_STATUS);
$statusText = Order::getStatusLabel($status);
$statusClass = $order->getStatusClass();
?>
<span class="badge bg-<?= $statusClass ?>"><?= htmlspecialchars($statusText) ?></span>
```

#### 前端模板

```php
<?php
use Weline\Order\Model\Order;

$statusText = Order::getStatusLabel($order->getStatus());
?>
<div class="order-status"><?= htmlspecialchars($statusText) ?></div>
```

### 3. 创建自定义状态

#### 通过后台界面

1. 访问：订单管理 > 订单状态管理
2. 点击"新建状态"
3. 填写状态信息：
   - 状态代码：只能包含小写字母和下划线，如 `custom_status`
   - 状态名称：默认语言（中文）的名称
   - 状态颜色：选择用于UI显示的颜色
   - 状态图标：Material Design Icons 类名
   - 是否启用：启用或禁用该状态
4. 配置多语言翻译（可选）
5. 保存

#### 通过代码

```php
use Weline\Order\Model\OrderStatus;
use Weline\Framework\Manager\ObjectManager;

$statusModel = ObjectManager::getInstance(OrderStatus::class);
$statusModel->setData(OrderStatus::fields_CODE, 'custom_status')
    ->setData(OrderStatus::fields_NAME, '自定义状态')
    ->setData(OrderStatus::fields_DESCRIPTION, '这是一个自定义状态')
    ->setData(OrderStatus::fields_COLOR, 'info')
    ->setData(OrderStatus::fields_ICON, 'mdi-star')
    ->setData(OrderStatus::fields_IS_SYSTEM, 0)
    ->setData(OrderStatus::fields_IS_ACTIVE, 1)
    ->setData(OrderStatus::fields_SORT_ORDER, 10)
    ->save();
```

### 4. 配置状态翻译

#### 方式一：在翻译文件中配置

编辑 `app/code/Weline/Order/i18n/zh_Hans_CN.csv`：

```csv
order_status_custom_status,自定义状态
```

编辑 `app/code/Weline/Order/i18n/en_US.csv`：

```csv
order_status_custom_status,Custom Status
```

#### 方式二：在数据库中配置

通过后台界面在状态编辑页面配置多语言翻译，或直接操作数据库：

```php
use Weline\Order\Model\OrderStatusTranslation;
use Weline\Framework\Manager\ObjectManager;

$translationModel = ObjectManager::getInstance(OrderStatusTranslation::class);
$translationModel->setData(OrderStatusTranslation::fields_STATUS_CODE, 'custom_status')
    ->setData(OrderStatusTranslation::fields_LOCALE, 'en_US')
    ->setData(OrderStatusTranslation::fields_NAME, 'Custom Status')
    ->setData(OrderStatusTranslation::fields_DESCRIPTION, 'This is a custom status')
    ->save();
```

## 默认状态

系统在安装时会自动创建以下默认状态：

| 代码 | 中文名称 | 英文名称 | 颜色 | 图标 |
|------|---------|---------|------|------|
| pending | 待处理 | Pending | warning | mdi-clock-outline |
| processing | 处理中 | Processing | info | mdi-cog |
| paid | 已支付 | Paid | primary | mdi-check-circle |
| fulfilled | 已发货 | Fulfilled | success | mdi-truck |
| completed | 已完成 | Completed | success | mdi-check-all |
| cancelled | 已取消 | Cancelled | danger | mdi-close-circle |
| refunded | 已退款 | Refunded | secondary | mdi-cash-refund |

## 注意事项

1. **状态代码唯一性**：状态代码（code）必须唯一，不能重复
2. **系统状态保护**：系统状态（is_system=1）不能删除
3. **状态使用检查**：如果状态正在被订单使用，不能删除
4. **代码规范**：状态代码只能包含小写字母和下划线，建议使用下划线分隔单词
5. **翻译优先级**：数据库翻译 > 翻译文件 > 默认名称

## 扩展开发

### 为其他模块添加状态翻译支持

如果其他模块也需要状态翻译功能，可以：

1. 创建类似的状态表和翻译表
2. 创建状态服务类
3. 在模型中使用状态服务获取翻译

参考 `Weline_Order` 模块的实现方式。
