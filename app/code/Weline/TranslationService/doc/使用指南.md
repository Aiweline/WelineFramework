## 翻译服务模块使用指南（Weline_TranslationService）

### 一、模块定位

- **模块代码**: `Weline_TranslationService`  
- **功能**: 为其他模块提供统一的第三方翻译服务调用能力，统一管理多家翻译渠道（Google、百度、DeepL、Microsoft、有道、腾讯等），并记录、统计翻译使用情况。

---

### 二、后台使用说明

#### 1. 入口菜单

- 后台路径：  
  - **系统服务 → 翻译服务 → 渠道配置**  
  - **系统服务 → 翻译服务 → 翻译记录**

#### 2. 渠道配置

1. 进入：**系统服务 → 翻译服务 → 渠道配置**
2. 点击右上角 **「添加渠道」**：
   - **渠道名称**：如 “Google 翻译”、“百度翻译”
   - **渠道代码**：
     - 新建时可手动输入（小写字母/数字/下划线，如 `google` / `baidu`）
     - 也可留空，系统会根据渠道名称自动生成
   - **描述**：对该渠道的简单说明
3. **API 配置**：
   - `API密钥`：从对应服务商控制台获取
   - `API密钥（Secret）`：部分服务商需要额外密钥时填写
   - `API端点URL`：一般保持默认，特殊地区或私有网关时可自定义
4. **高级配置**：
   - `启用渠道`：勾选后才会参与实际翻译
   - `设为默认渠道`：优先选择该渠道（同一时间建议只设一个默认）
   - `优先级`：数值越大优先级越高
   - `速率限制（每分钟）`：限制 QPS，避免触发服务商限流
   - `每日限制`：限制每日调用次数
   - `每字符成本`：单字符费用，用于成本统计（单位自定义，一般为 RMB）
5. **支持的语言**：
   - 在「支持的语言」卡片中勾选当前渠道支持的 ISO 639‑1 语言（如 `zh`、`en`、`ja` 等）
   - 可使用「全选 / 全不选」快速操作
6. 保存后，可在列表中通过 **「测试」** 按钮校验密钥与网络是否配置正确。

#### 3. 翻译记录与统计

1. 进入：**系统服务 → 翻译服务 → 翻译记录**
2. 顶部统计卡片展示：
   - 总记录数 / 成功记录 / 失败记录
   - 总字符数 / 总成本
   - 今日调用次数 / 今日字符数 / 今日成本
3. 筛选条件支持：
   - 渠道、状态（成功/失败/待处理）
   - 源语言 / 目标语言
   - 调用模块（`module_name`）
   - 起止日期
4. 列表支持：
   - 查看单条详情（包括源文本、翻译文本、请求/响应原始数据）
   - 单条删除 / 批量删除
   - 导出统计（JSON 格式）

### 三、在其他模块中调用翻译服务

#### 1. 获取服务实例

```php
use Weline\Framework\Manager\ObjectManager;
use Weline\TranslationService\Api\TranslationServiceInterface;

/** @var TranslationServiceInterface $translationService */
$translationService = ObjectManager::getInstance(TranslationServiceInterface::class);
```

#### 2. 单条文本翻译

```php
$text = 'Hello, World!';

// 目标语言可以是：zh / zh-CN / zh_Hans_CN / zho 等，服务会自动标准化
$translated = $translationService->translate(
    $text,
    'zh',                 // 目标语言
    'auto',               // 源语言，auto = 自动检测
    null,                 // 渠道代码，null = 自动选择（默认渠道 + 语言支持）
    ['module_name' => 'Weline_Cms']  // 调用来源模块（用于统计）
);
```

#### 3. 批量翻译

```php
$texts = ['Title', 'Description', 'Content'];

$results = $translationService->batchTranslate(
    $texts,
    'en',                 // 目标语言
    'zh',                 // 源语言
    null,                 // 可指定渠道代码，如 'google'
    ['module_name' => 'Weline_Product']
);

// $results 为等长数组，对应每条翻译后的文本
```

#### 4. 语言检测

```php
$lang = $translationService->detectLanguage('今天天气不错');
// 返回标准化语言代码，如：'zh'
```

#### 5. 检查语言支持

```php
// 检查是否存在任意「已启用」渠道支持该语言
if ($translationService->supportsLanguage('ja')) {
    // 安全调用翻译
}

// 只检查指定渠道是否支持
$googleSupportJa = $translationService->supportsLanguage('ja', 'google');
```

#### 6. 获取可用渠道列表

```php
$providers = $translationService->getAvailableProviders(); // 如：['google', 'baidu', 'deepl']
```

### 四、语言代码与国际化规范

- 模块内部所有对外接口统一使用 **ISO 639‑1** 核心代码（`zh`、`en`、`ja`、`fr` 等）
- 支持自动解析以下格式并转为标准代码：
  - ISO 639‑1：`zh`、`en`
  - ISO 639‑2：`zho`、`eng`
  - BCP 47：`zh-CN`、`en-US`、`zh_Hans_CN`
- 使用 `LanguageCodeConverter` 进行转换（通常业务层无需直接调用）：

```php
use Weline\TranslationService\Helper\LanguageCodeConverter;

$code = LanguageCodeConverter::normalize('zh_Hans_CN'); // 返回 'zh'
```

### 五、与 I18n 模块配合使用的推荐模式

> **注意**：翻译服务模块主要负责「对接外部机器翻译渠道」，而 I18n 模块负责「站点多语言词典管理」。

典型流程建议：

1. 在业务代码中仍然以中文为基础：`__('商品名称')`
2. 未命中 I18n 词典时，可调用 `TranslationService` 自动翻译一次，
   并将结果写回 I18n 词典或业务表，供后续直接使用。
3. 对于大量词条（如 CMS 内容、商品描述），建议通过后台或 CLI
   批量调用翻译服务，再手工校对。

（具体与 I18n 集成的最佳实践，可参考 `app/code/Weline/I18n/doc/README.md`）

### 六、异常与调试

#### 1. 常见错误消息（已接入 i18n）

- `未找到可用的翻译渠道`
- `未找到渠道适配器：%{1}`
- `文本不能为空`
- 各渠道的「API 密钥未配置 / 响应格式错误 / 语言检测失败」等

#### 2. 调试建议

1. 后台**渠道配置**中先使用「测试」按钮确保配置正确；
2. 若翻译调用失败，可在「翻译记录 → 详情」中查看：
   - 源文本 / 翻译文本
   - 请求数据 / 响应数据（JSON）
   - 错误信息、状态码等
3. 合理设置 `rate_limit`、`daily_limit`，防止服务商限流。

### 七、扩展新的翻译渠道（简要指引）

1. 新建适配器类，实现 `Weline\TranslationService\Api\ProviderInterface`，通常继承 `AbstractProvider`；
2. 在 `ProviderFactory::$adapterMap` 中注册渠道代码与适配器类的映射；
3. 在 `Setup/Install.php` 中追加一条默认渠道记录（可选）；
4. 在后台「渠道配置」中补充 API 密钥、支持语言等信息；
5. 通过「测试」按钮验证连接。

> 若需要更详细的扩展说明，可在 `doc/` 下继续拆分出 `开发指南.md`。
