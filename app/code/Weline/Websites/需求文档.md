# Websites 模块需求文档

## 一、历史需求总结

### 1. 基础功能需求
- **网站管理**：支持创建、编辑、删除网站
- **网站基本信息**：
  - 网站名称（name）
  - 网站代码（code）
  - 网站URL（url）
  - 默认货币（default_currency）
  - 默认语言（default_language）
  - 默认时区（default_timezone）

### 2. 已实现功能
- ✅ 网站CRUD操作（增删改查）
- ✅ 网站列表展示
- ✅ 网站表单编辑
- ✅ 默认货币和默认语言字段（必填）
- ✅ 时区选择功能
- ✅ 语言选择功能（从i18n模块获取已安装的语言）

## 二、当前新增需求

### 1. 多货币支持
- **需求描述**：网站支持设置多个关联货币
- **实现方式**：
  - 创建网站-货币关联表（website_currency）
  - 支持多选货币
  - 如果不设置关联货币，则默认使用数据库中所有启用的货币
  - 在字段旁边添加提示说明："不设置则使用所有启用的货币"

### 2. 多语言支持
- **需求描述**：网站支持设置多个关联语言
- **实现方式**：
  - 创建网站-语言关联表（website_language）
  - 支持多选语言
  - 如果不设置关联语言，则默认使用i18n模块支持的所有语言
  - 在字段旁边添加提示说明："不设置则使用i18n支持的所有语言"

### 3. 默认货币和默认语言字段优化
- **需求描述**：默认货币和默认语言字段改为可选
- **实现方式**：
  - 将数据库字段改为可空（nullable）
  - 如果不设置，系统会根据关联货币/语言自动选择第一个作为默认值
  - 在字段旁边添加提示说明："不设置则从关联货币/语言中选择第一个作为默认值"

### 4. 网站列表功能增强
- **需求描述**：增强网站列表页功能
- **实现内容**：
  - ✅ 添加搜索功能（支持按名称、代码、URL搜索）
  - ✅ 显示默认货币
  - ✅ 显示网站关联的所有货币（以标签形式展示）
  - ✅ 显示网站关联的所有语言（以标签形式展示）

## 三、数据库设计

### 1. 网站表（weline_websites）修改
```sql
-- 修改默认货币和默认语言字段为可空
ALTER TABLE weline_websites 
MODIFY COLUMN default_currency VARCHAR(20) NULL COMMENT '默认货币',
MODIFY COLUMN default_language VARCHAR(20) NULL COMMENT '默认语言';
```

### 2. 网站-货币关联表（weline_website_currency）
```sql
CREATE TABLE weline_website_currency (
    website_currency_id INT(11) PRIMARY KEY AUTO_INCREMENT COMMENT '关联ID',
    website_id INT(11) NOT NULL COMMENT '网站ID',
    currency_code VARCHAR(3) NOT NULL COMMENT '货币代码',
    INDEX idx_website_id (website_id),
    INDEX idx_currency_code (currency_code),
    UNIQUE KEY uk_website_currency (website_id, currency_code)
) COMMENT '网站-货币关联表';
```

### 3. 网站-语言关联表（weline_website_language）
```sql
CREATE TABLE weline_website_language (
    website_language_id INT(11) PRIMARY KEY AUTO_INCREMENT COMMENT '关联ID',
    website_id INT(11) NOT NULL COMMENT '网站ID',
    language_code VARCHAR(20) NOT NULL COMMENT '语言代码',
    INDEX idx_website_id (website_id),
    INDEX idx_language_code (language_code),
    UNIQUE KEY uk_website_language (website_id, language_code)
) COMMENT '网站-语言关联表';
```

## 七、网站检测和验证功能

### 1. 网站数据静态类（WebsiteData）
- **需求描述**：提供静态类向其他模块提供当前请求命中的网站数据
- **实现内容**：
  - ✅ 创建 `Weline\Websites\Data\WebsiteData` 静态类
  - ✅ 存储当前网站实例和完整数据
  - ✅ 提供获取网站基本信息的方法（ID、代码、名称、URL等）
  - ✅ 提供获取关联货币和语言的方法
  - ✅ 提供获取货币详细信息的方法（包含format、symbol等）
  - ✅ 提供货币和语言验证方法

### 2. 货币验证逻辑
- **需求描述**：当访问的网站命中后，验证货币是否被允许
- **验证规则**：
  - 如果网站限定了关联货币，只允许这些货币通过，否则拒绝访问
  - 如果网站没有限定关联货币（关联货币为空），允许所有货币表中的启用货币通过
- **实现方式**：
  - ✅ 创建 `ValidateCurrency` 观察者
  - ✅ 监听 `Framework_Url::detect_currency` 事件
  - ✅ 使用 `WebsiteData::isCurrencyAllowed()` 验证货币
  - ✅ 如果货币不被允许，设置 `result` 为 `false` 并拒绝访问

### 3. 语言验证逻辑
- **需求描述**：当访问的网站命中后，验证语言是否被允许
- **验证规则**：
  - 如果网站限定了关联语言，只允许这些语言通过，否则拒绝访问
  - 如果网站没有限定关联语言（关联语言为空），允许所有i18n支持的激活语言通过
- **实现方式**：
  - ✅ 创建 `ValidateLanguage` 观察者
  - ✅ 监听 `Framework_Url::detect_language` 事件
  - ✅ 使用 `WebsiteData::isLanguageAllowed()` 验证语言
  - ✅ 如果语言不被允许，设置 `result` 为 `false` 并拒绝访问

### 4. 观察者更新
- **DetectWebsite观察者**：
  - ✅ 在 `processSite()` 方法中调用 `WebsiteData::setWebsite()` 设置当前网站
  - ✅ 确保网站数据在货币和语言检测之前设置完成

### 5. 事件配置
- ✅ 在 `event.xml` 中注册货币验证观察者（sort=100，在网站检测之后执行）
- ✅ 在 `event.xml` 中注册语言验证观察者（sort=100，在网站检测之后执行）

## 八、WebsiteData 静态类API

### 基本方法
- `setWebsite(Website $website)`: 设置当前网站
- `getWebsite(): ?Website`: 获取当前网站实例
- `getWebsiteId(): ?int`: 获取网站ID
- `getCode(): ?string`: 获取网站代码
- `getName(): ?string`: 获取网站名称
- `getUrl(): ?string`: 获取网站URL
- `getDefaultCurrency(): ?string`: 获取默认货币
- `getDefaultLanguage(): ?string`: 获取默认语言
- `getDefaultTimezone(): ?string`: 获取默认时区

### 关联数据方法
- `getCurrencyCodes(): array`: 获取关联货币代码列表
- `getLanguageCodes(): array`: 获取关联语言代码列表
- `getCurrencies(): array`: 获取关联货币详细信息（包含format、symbol等）

### 货币详细信息方法（新增）
- `getCurrencyFormat(?string $currencyCode = null): ?string`: 获取货币format格式（如 "1,0"）
- `getCurrency(?string $currencyCode = null): ?array`: 获取指定货币的完整信息
- `getCurrencySymbol(?string $currencyCode = null): ?string`: 获取货币符号（如 "￥"、"$"）
- `getCurrencyPosition(?string $currencyCode = null): ?string`: 获取货币符号位置（"left" 或 "right"）
- `getCurrencyRate(?string $currencyCode = null): ?float`: 获取货币汇率

### 验证方法
- `isCurrencyAllowed(string $currencyCode): bool`: 验证货币是否允许
- `isLanguageAllowed(string $languageCode): bool`: 验证语言是否允许

### 数据方法
- `getData(): ?array`: 获取完整的网站数据数组
- `reset(): void`: 重置所有数据（用于测试或清理）

## 九、使用示例

### 在其他模块中使用WebsiteData

```php
use Weline\Websites\Data\WebsiteData;

// 获取当前网站ID
$websiteId = WebsiteData::getWebsiteId();

// 获取默认货币
$defaultCurrency = WebsiteData::getDefaultCurrency();

// 获取所有关联货币（包含format等信息）
$currencies = WebsiteData::getCurrencies();
foreach ($currencies as $currency) {
    echo $currency['code']; // 货币代码
    echo $currency['format']; // 货币格式，如 "1,0"
    echo $currency['symbol']; // 货币符号
}

// 直接获取货币format格式（推荐方式）
$format = WebsiteData::getCurrencyFormat(); // 获取默认货币的format
$format = WebsiteData::getCurrencyFormat('CNY'); // 获取指定货币的format

// 获取货币符号
$symbol = WebsiteData::getCurrencySymbol('USD'); // 返回 "$"

// 获取货币符号位置
$position = WebsiteData::getCurrencyPosition('EUR'); // 返回 "left" 或 "right"

// 获取货币汇率
$rate = WebsiteData::getCurrencyRate('USD'); // 返回 8.0

// 获取货币完整信息
$currency = WebsiteData::getCurrency('CNY');
// 返回: ['code' => 'CNY', 'name' => '人民币', 'format' => '1,0', 'symbol' => '￥', ...]

// 验证货币是否允许
if (WebsiteData::isCurrencyAllowed('CNY')) {
    // 货币被允许
}

// 获取完整数据
$data = WebsiteData::getData();
```
