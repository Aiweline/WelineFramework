# Weline_Websites 域名管理架构文档

## 概述

域名管理系统为 Weline_Websites 模块提供完整的域名全生命周期管理能力，包括域名商管理、域名购买、DNS/CDN/解析编排和证书管理集成。

## 架构总览

```
域名管理页面（Tab 布局）
├── Tab1: 域名商管理 — 配置 API 服务商（AWS、阿里云、Azure 等）
├── Tab2: 域名购买 — 批量检查可用性 → 统一购买弹窗 → 批量购买
├── Tab3: 证书管理 — 由 Server 模块通过 Hook 注入
└── Tab N: 其他扩展 — 其他模块通过 Hook 注入
```

## 数据模型

| name | VARCHAR(255) | 显示名称 |
| description | VARCHAR(500) | 描述 |
| status | VARCHAR(20) | active / disabled |

| success_count | INT | 成功数 |

### DomainPurchaseItem（购买条目）

| 字段 | 类型 | 说明 |
|------|------|------|

| currency | VARCHAR(10) | 币种 |

### ProvisioningOrder / ProvisioningStep（生命周期编排）

生命周期状态由 `Weline_Saas` 模块中的 `ProvisioningOrder`、`ProvisioningStep` 复用承载：

- `ProvisioningOrder`：根域级流程头信息，保存当前步骤、状态、异常信息
- `ProvisioningStep`：步骤流水，记录 `purchase / dns / resolve / verify / ssl` 等步骤执行结果
- `DomainLifecycleOrchestrationService`：对根域、`@`、`www` 的 DNS、访问、HTTPS 状态做统一编排

## 适配器扩展点

域名商适配器采用 Extends 扩展模式（参照 CDN 适配器架构）。

### 接口

`Weline\Websites\Api\DomainRegistrarInterface`

核心方法：
- `getRegistrarCode()` / `getRegistrarName()` - 标识和名称
- `getConfigFields()` - 动态配置表单字段定义
- `testConnection()` - 测试 API 连通性
- `checkAvailability()` / `batchCheckAvailability()` - 域名可用性检查
- `purchaseDomain()` - 购买域名
- `getDomainList()` / `getDomainDetail()` - 域名列表和详情

### 内置适配器

| 文件 | 代码 | 说明 |
|------|------|------|
| Adapter/AwsRoute53Registrar.php | aws_route53 | AWS Route53 |
| Adapter/AliyunDomainRegistrar.php | aliyun_domain | 阿里云域名 |
| Adapter/AzureDnsRegistrar.php | azure_dns | Azure DNS |
| Adapter/GnameRegistrar.php | gname | GName（已支持 code -1 二次确认） |

### 第三方扩展

其他模块可通过 `extends/module/Weline_Websites/Registrar/{Name}.php` 扩展新的域名商。

## GName 特殊处理

GName 在购买接口中可能返回 `code = -1` 且提示“域名已被注册”，但该域名实际上已经进入当前账号。

当前处理策略：

1. `GnameRegistrar::purchaseDomain()` 收到 `-1` 时先检查响应文案是否属于“已被注册”类歧义结果
2. 调用 `getDomainList()` 二次确认该域名是否已出现在当前账号列表中
3. 若在当前账号下，购买结果按成功处理并返回明确说明
4. 若不在当前账号下，则仍按真实失败返回

## 购买流程

```
用户输入域名列表
    ↓
选择域名商账号
    ↓
调用适配器 batchCheckAvailability()
    ↓
用户选择可用域名 + 绑定站点选项
    ↓
统一购买弹窗（DNS 策略 / CDN 策略 / 是否解析到本机 / 子域名 / 是否启动生命周期跟踪）
    ↓
创建购买订单 (DomainPurchaseOrder)
    ↓
逐个调用适配器 purchaseDomain()
    ↓
购买成功 → 入域名池 (DomainPool) → 可选绑定站点 (WebsiteDomain)
    ↓
启动 DomainLifecycleOrchestrationService（根域级）
    ↓
检查根域购买状态 → DNS/解析 @/www → 校验访问 → 申请 HTTPS → 持续轮询
    ↓
触发 Weline_Websites::domain::purchase_success 事件
```

## 生命周期编排

`DomainLifecycleOrchestrationService` 负责把分散逻辑串起来：

1. 接收根域名和购买选项
2. 复用 `DomainResolveService` 为根域写入解析记录
3. 复用 `DomainPoolResolveService` 检测根域、`@`、`www` 的解析/IP/本机状态
4. 复用 `HealthCheckService` 做访问校验
5. 复用 `DomainProvisioningService::runStepSsl()` 申请 HTTPS
6. 通过 `Weline\Websites\Cron\DomainLifecycleOrchestration` 持续推进未完成订单

页面侧支持：

- `Weline_Websites` 后台购买 Tab
- `GuoLaiRen_PageBuilder` 域名管理页
- `GuoLaiRen_PageBuilder` 快速建站向导

三处购买入口现已统一使用同一套购买弹窗字段与请求参数。
