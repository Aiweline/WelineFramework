# 网站备份

`Weline_Websites` 的网站备份是站点级归档，不是整站代码仓库备份。

## 备份类型

- 完整备份：站点数据 + 关联文件 + manifest + 校验清单。
- 仅站点数据：所有可发现的 `website_id` 关联表记录，输出为 `database/*.json`。
- 仅关联文件：站点记录中引用的安全文件根目录内容，输出为 `files/*`。

## 完整备份内容

完整备份归档位于 `var/backups/websites/*.zip`，包含：

- `manifest.json`：备份类型、站点基础资料、创建时间、创建人、数据表清单、文件数量、缺失文件清单和恢复提示。
- `database/*.json`：所有带 `website_id` 字段的 Model 数据表记录，例如站点、域名、语言、货币、CMS、SEO、主题、访问统计等站点级数据。
- `files/*`：从站点数据中解析到的 `pub/media`、`pub/source`、`pub/sitemaps`、`var/pagebuilder`、`var/export` 下的文件，以及约定目录中的站点文件。
- `checksums.json`：归档内 JSON 和文件的 SHA-256 与大小。

## 安全边界

- `website_id=0` 是系统默认站点，是合法备份目标。
- 下载和删除只允许操作 `var/backups/websites` 下的 `.zip` 文件，文件名必须是 basename，禁止路径穿越。
- 当前页面只提供创建、下载、删除和审计信息展示；恢复操作需要先在测试环境核对 `manifest.json`、数据表清单和 `checksums.json`，不要直接在生产环境自动覆盖数据。
- 文件归档只读取白名单目录，不扫描或打包 `.env`、私钥、数据库连接配置和非站点目录。
