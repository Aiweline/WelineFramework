# 站点首页开始路由配置

## 背景

框架已有全局开始路由配置：

- module: `Weline_Backend`
- key: `start_page_path`
- area: `backend`

`Weline_Frontend::start_page_plugin` 会在前台根路径请求时读取该配置，并把 `/` 映射到配置的前台路由。

## 站点级覆盖

站点后台表单的“首页入口路由”字段复用上述配置，不在 `website` 表新增字段。

- 选择具体路由时，保存为 `system_config` 的站点 scope 配置。
- scope 使用网站代码，由 `SystemConfig::normalizeScope()` 归一化为站点 scope。
- 留空时删除站点 scope 配置，继续继承全局 `start_page_path`。

保存入口：

- `Weline\Websites\Controller\Admin\Website::add()`
- `Weline\Websites\Controller\Admin\Website::edit()`
- `Weline\Websites\Controller\Admin\Website::quickSave()`

读取入口：

- `Weline\Frontend\Plugin\StarPage::beforeGetUrlPath()`

该插件会优先使用当前请求的网站代码读取站点 scope 配置；没有站点覆盖时，由 `SystemConfig` 的 fallback 机制回落到全局配置。

## 路由来源

可选项来自 `generated/routers/frontend_pc.php` 中的前台 PC 路由：

- 无显式方法的路由可作为开始路由。
- `::GET` 路由可作为开始路由。
- 其他方法路由不会出现在选择项中，避免把 POST 等非页面入口设为首页。

## 维护注意

- 修改开始路由机制时，应优先保持 `start_page_path` 配置键兼容。
- 站点级配置不要写入 `website` 模型字段，避免与 SystemConfig scope 机制形成两套来源。
- 如果新增前台路由后选择项未出现，先确认路由已生成到 `generated/routers/frontend_pc.php`。
