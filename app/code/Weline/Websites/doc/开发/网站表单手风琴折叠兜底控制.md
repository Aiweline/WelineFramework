# 网站表单手风琴折叠兜底控制

## 背景

后台网站新增/编辑表单使用 `.website-form-accordion` 渲染基础信息、域名管理、高级选项，以及通过 `Weline_Websites::backend::website::form::sections-after` hook 注入的扩展协议区块。

SEO 与 GEO 模块注入的协议区块依赖 Bootstrap collapse 的 `data-bs-toggle="collapse"` 行为。当后台主题未稳定暴露或初始化 collapse 数据接口时，协议标题可能保持可见但点击后不展开。

## 处理

`app/code/Weline/Websites/view/templates/Admin/Website/form.phtml` 在表单根容器内绑定了局部折叠兜底逻辑：

- 只作用于 `.website-form-accordion` 内部的 `.accordion-button`。
- 优先等待主题原生 Bootstrap 行为处理点击。
- 如果点击后目标 `.accordion-collapse` 和按钮 `aria-expanded` 均未变化，则由表单自身同步 `show`、`collapsed`、`aria-expanded`。
- 展开某一节时，按 `data-bs-parent` 关闭同组已展开区块。

该逻辑不修改 SEO/GEO hook 数据结构，也不影响其他后台页面的折叠控件。
