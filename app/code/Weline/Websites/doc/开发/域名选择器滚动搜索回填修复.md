# 域名选择器滚动搜索回填修复

日期：2026-05-28

## 背景

网站编辑页的域名多选弹层存在三类问题：

- 下拉列表在 OffCanvas / iframe 场景中滚轮滚动不稳定。
- 搜索只过滤首批本地缓存，域名池较多或首批未命中时无法搜索到目标域名。
- 编辑页已有绑定的 `pool_id` 可能被首屏 `limit` 截断，旧 PageBuilder 站点还可能只有 `WebsiteDomain.domain`、没有 `pool_id`，导致选择器不回填。

## 处理

- `Weline\Websites\Taglib\DomainSelect`
  - 下拉列表增加滚动隔离，避免内部滚动触发全局重定位。
  - 搜索改为带 `search=` 参数请求域名池 API。
  - 前端维护已见过的 domain / pool_id 映射，确认选择时不再只依赖当前列表缓存。
  - 有初始 `pool_id` 时自动加载并回填显示。

- `Weline\Websites\Controller\Backend\Api\DomainPool`
  - 当请求包含当前 `pool_ids` 且未搜索时，额外合并这些池记录，避免首屏 `limit` 截断导致编辑页无法回填。

- `GuoLaiRen\PageBuilder\Controller\Backend\WebsiteManagement`
  - 编辑旧站点时，如果 `WebsiteDomain.pool_id` 为空，则尝试通过域名反查 `DomainPool`。
  - 如果旧域名不在域名池中，编辑表单仍显示原域名，并且未重新选择池域名时保存保留原绑定。

## 验证

- `php -l app/code/Weline/Websites/Taglib/DomainSelect.php`
- `php -l app/code/Weline/Websites/Controller/Backend/Api/DomainPool.php`
- `php -l app/code/GuoLaiRen/PageBuilder/Controller/Backend/WebsiteManagement.php`
- `php -l app/code/GuoLaiRen/PageBuilder/view/templates/Backend/WebsiteManagement/form.phtml`
- 浏览器验证 PageBuilder 网站编辑弹层：
  - 旧绑定 `opsflow-onepass-qa6.test` 已在选择器中显示。
  - 搜索 `local-flow` 返回 `local-flow-5c1632.weline.test`。
  - 选择并确认后隐藏值回填为对应 `pool_id`。
