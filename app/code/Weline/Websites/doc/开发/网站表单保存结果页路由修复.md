# 网站表单保存结果页路由修复

## 背景

网站新增/编辑表单在 OffCanvas iframe 内提交。保存成功或失败后，控制器需要重定向到结果页，让 iframe 显示成功/失败状态并通知父页面。

旧路径 `/component/offcanvas/success` 和 `/component/offcanvas/error` 在后台 token 路由下会生成类似：

```text
/{backend-token}/component/offcanvas/error
```

该路径不能稳定命中后台 `Weline\Component\Controller\Backend\Offcanvas`，保存失败时会在 iframe 内显示 `404`。

## 处理

`Weline\Websites\Controller\Admin\Website` 的保存结果重定向已切换为后台 OffCanvas 控制器路径：

- 成功：`component/backend/offcanvas/getSuccess`
- 失败：`component/backend/offcanvas/getError`

保存业务逻辑、字段处理、SEO/GEO 扩展 hook 派发逻辑不变。
