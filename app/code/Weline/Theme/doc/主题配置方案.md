# 主题配置系统完整方案

## 一、方案概述

### 1.1 目标
- 实现主题配置的实时预览
- 确保配置正确保存到主题自己的配置文件中
- 支持布局、色系、部件、变量、组件等各类配置的独立管理
- 提供可靠的配置加载和验证机制

### 1.2 配置存储结构

主题配置存储在数据库 `weline_theme` 表的 `config` 字段（JSON格式），结构如下：

```json
{
    "layouts": {
        "frontend": {
            "account": "profile",
            "homepage": "default",
            "category": "grid",
            "product": "detail",
            "cart": "default",
            "checkout": "one-page"
        },
        "backend": {
            "default": "admin"
        }
    },
    "headers": {
        "frontend": "default",
        "backend": "default"
    },
    "colors": {
        "frontend": "dark",
        "backend": "light"
    },
    "partials": {
        "frontend": {
            "header": "default",
            "footer": "minimal",
            "sidebar": "default"
        },
        "backend": {
            "header": "default",
            "footer": "default",
            "sidebar": "default"
        }
    },
    "variables": {
        "frontend": ["borders", "spacing"],
        "backend": ["borders"]
    },
    "components": {
        "frontend": [],
        "backend": []
    }
}
```

## 二、核心组件设计

### 2.1 配置管理器 (ThemeConfigManager)

**位置**: `app/code/Weline/Theme/Helper/ThemeConfigManager.php`

**职责**:
- 统一管理主题配置的读取、保存、验证
- 处理配置的合并（继承父主题配置）
- 提供配置的默认值

**核心方法**:
```php
class ThemeConfigManager
{
    /**
     * 获取主题配置
     * @param WelineTheme $theme 主题对象
     * @param string $type 配置类型 (layouts|headers|colors|partials|variables|components)
     * @param string $area 区域 (frontend|backend)
     * @return mixed 配置值
     */
    public static function getConfig(WelineTheme $theme, string $type, string $area = 'frontend');
    
    /**
     * 保存主题配置
     * @param WelineTheme $theme 主题对象
     * @param string $type 配置类型
     * @param string $area 区域
     * @param mixed $value 配置值
     * @return bool 是否保存成功
     */
    public static function saveConfig(WelineTheme $theme, string $type, string $area, $value): bool;
    
    /**
     * 获取所有配置
     * @param WelineTheme $theme 主题对象
     * @return array 完整配置数组
     */
    public static function getAllConfig(WelineTheme $theme): array;
    
    /**
     * 验证配置值
     * @param WelineTheme $theme 主题对象
     * @param string $type 配置类型
     * @param string $area 区域
     * @param mixed $value 配置值
     * @return bool 是否有效
     */
    public static function validateConfig(WelineTheme $theme, string $type, string $area, $value): bool;
}
```

### 2.3 配置加载器 (ConfigLoader)

**位置**: `app/code/Weline/Theme/Helper/ConfigLoader.php`

**职责**:
- 在渲染时加载配置（优先预览配置，其次保存的配置，最后默认配置）
- 处理配置的继承关系
- 提供配置的缓存机制

**核心方法**:
```php
class ConfigLoader
{
    /**
     * 加载配置（优先级：预览配置 > 保存配置 > 默认配置）

     * @param string $area 区域

     * @return string 布局选项值

    public static function getLayoutConfig(WelineTheme $theme, string $area, string $layoutType = 'default'): string;
    
    /**
     * 获取色系配置

     * @param string $area 区域

    /**
     * 获取部件配置

    public static function getPartialConfig(WelineTheme $theme, string $area, string $partialType): string;
    
    /**
     * 获取变量配置
     * @param WelineTheme $theme 主题对象
     * @param string $area 区域
     * @return array 变量数组
     */
    public static function getVariablesConfig(WelineTheme $theme, string $area): array;
}
```

## 三、数据流程

### 3.1 配置保存流程

```
用户操作 → 前端收集配置 → AJAX请求 → 后端验证 → 保存到数据库 → 清除缓存 → 返回结果
```

**详细步骤**:
1. 用户在配置界面选择配置项（布局、色系、部件等）
2. 前端JavaScript收集所有配置数据
3. 发送AJAX请求到 `theme/backend/config/layout/postSave`
4. 后端 `Layout::postSave()` 方法：
   - 验证配置值是否有效
   - 获取主题对象
   - 调用 `ThemeConfigManager::saveConfig()` 保存配置
   - 清除主题缓存
   - 返回成功/失败结果

### 3.2 预览流程

```
用户选择配置 → 前端构建预览URL → 后端设置预览配置到Session → 重定向到预览页面 → 渲染时读取预览配置
```

**详细步骤**:
1. 用户在配置界面选择配置项
2. 前端调用 `updatePreview()` 函数
3. 构建预览URL，包含所有配置参数
4. 发送请求到 `theme/backend/config/layout/preview`
5. 后端 `Layout::getPreview()` 方法：
   - 解析配置参数
   - 调用 `PreviewManager::setPreviewConfig()` 设置预览配置到Session
   - 根据布局配置选择预览URL
   - 重定向到预览页面
6. 预览页面渲染时：
   - `ConfigLoader::loadConfig()` 优先读取预览配置
   - 应用配置到页面渲染

### 3.3 配置加载流程

```
页面请求 → 检查预览模式 → 加载预览配置或保存配置 → 应用配置到渲染
```

**详细步骤**:
1. 页面请求到达
2. `ConfigLoader::loadConfig()` 被调用
3. 检查 `PreviewManager::isPreviewMode()`
4. 如果在预览模式，读取Session中的预览配置
5. 如果不在预览模式，读取主题保存的配置
6. 如果配置不存在，使用默认配置
7. 应用配置到页面渲染（布局、色系、部件等）

## 四、实现细节

### 4.1 配置保存实现

**文件**: `app/code/Weline/Theme/Controller/Backend/Config/Layout.php`

```php
public function postSave()
{
    $themeId = $this->request->getParam('theme_id');
    $area = $this->request->getParam('area', 'frontend');
    
    // 获取配置数据
    $layouts = $this->request->getPost('layouts', []);
    $headers = $this->request->getPost('headers', []);
    $colors = $this->request->getPost('colors', []);
    $partials = $this->request->getPost('partials', []);
    $variables = $this->request->getPost('variables', []);
    
    // 加载主题
    $theme = ObjectManager::getInstance(WelineTheme::class);
    $theme->load($themeId);
    
    // 验证并保存各类配置
    if (!empty($layouts[$area])) {
        ThemeConfigManager::saveConfig($theme, 'layouts', $area, $layouts[$area]);
    }
    if (!empty($headers[$area])) {
        ThemeConfigManager::saveConfig($theme, 'headers', $area, $headers[$area]);
    }
    if (!empty($colors[$area])) {
        ThemeConfigManager::saveConfig($theme, 'colors', $area, $colors[$area]);
    }
    if (!empty($partials[$area])) {
        ThemeConfigManager::saveConfig($theme, 'partials', $area, $partials[$area]);
    }
    if (!empty($variables[$area])) {
        ThemeConfigManager::saveConfig($theme, 'variables', $area, $variables[$area]);
    }
    
    // 清除缓存
    $theme->_cache->delete('theme');
    $theme->_cache->delete('theme_parent_' . $theme->getId());
    
    return $this->fetchJson($this->success(__('配置保存成功')));
}
```

### 4.2 预览配置设置实现

**文件**: `app/code/Weline/Theme/Controller/Backend/Config/Layout.php`

```php
public function getPreview()
{
    $themeId = $this->request->getParam('theme_id');
    $area = $this->request->getParam('area', 'frontend');
    
    // 获取预览配置参数
    $previewConfigs = [
        'layouts' => $this->request->getParam('layouts', []),
        'headers' => $this->request->getParam('header', ''),
        'colors' => $this->request->getParam('color', ''),
        'partials' => $this->request->getParam('partials', []),
        'variables' => $this->request->getParam('variables', []),
    ];
    
    // 设置预览配置到Session
    PreviewManager::setPreviewConfig($themeId, $area, $previewConfigs);
    
    // 根据布局配置选择预览URL
    $url = ObjectManager::getInstance(Url::class);
    $previewUrl = $this->getLayoutPreviewUrl($url, $previewConfigs['layouts'], $themeId);
    
    $this->request->getResponse()->redirect($previewUrl);
    return '';
}
```

### 4.3 配置加载实现

**文件**: `app/code/Weline/Theme/Helper/ConfigLoader.php`

```php
public static function loadConfig(WelineTheme $theme, string $type, string $area)
{
    // 1. 优先检查预览配置
    if (PreviewManager::isPreviewMode()) {
        $previewConfig = PreviewManager::getPreviewConfig($type, $area);
        if ($previewConfig !== null) {
            return $previewConfig;
        }
    }
    
    // 2. 读取保存的配置
    $savedConfig = ThemeConfigManager::getConfig($theme, $type, $area);
    if ($savedConfig !== null) {
        return $savedConfig;
    }
    
    // 3. 使用默认配置
    return self::getDefaultConfig($type, $area);
}
```

## 五、配置验证规则

### 5.1 布局配置验证
- 布局类型必须存在于扫描结果中
- 布局选项必须存在于对应布局类型的选项中

### 5.2 色系配置验证
- 色系值必须存在于扫描的色系文件中（如 `_dark.css`, `_light.css`）

### 5.3 部件配置验证
- 部件类型必须有效（header, footer, sidebar等）
- 部件选项必须存在于对应类型的部件文件中

### 5.4 变量配置验证
- 变量值必须是数组
- 每个变量值必须存在于扫描的变量文件中

## 六、缓存策略

### 6.1 配置缓存
- 主题配置缓存键: `theme_config_{themeId}`
- 缓存时间: 7天
- 保存配置时清除缓存

### 6.2 预览配置
- 预览配置存储在Session中，不缓存
- Session过期时间: 30分钟
- 预览完成后可手动清除

## 七、错误处理

### 7.1 配置验证失败
- 返回错误信息，不保存配置
- 前端显示错误提示

### 7.2 预览配置失败
- 回退到保存的配置
- 记录错误日志

### 7.3 配置加载失败
- 使用默认配置
- 记录警告日志
