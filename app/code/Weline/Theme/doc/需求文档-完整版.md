## 2. 总体描述

### 2.1 产品概述

主题编辑器允许用户通过可视化界面配置网站主题的布局和部件。采用类似 Shopify 的 JSON 模板布局概念，但更灵活——支持部件嵌套、可选 Slot、无限层级容器。

### 2.2 用户特征

| 用户角色 | 描述 |
|---------|------|
| 店铺管理员 | 通过后台配置主题外观、布局、部件 |
| 开发人员 | 创建自定义部件、布局模板 |

### 2.3 约束条件

- 必须与现有 Weline Framework 架构兼容
- 支持国际化（中英文）
- 响应式设计

## 3. 功能需求

### 3.1 首页布局系统

#### 3.1.1 简化布局结构

**需求 ID**: REQ-LAYOUT-001  
**优先级**: 高  
**状态**: ✅ 已完成  

首页布局简化为三大区域：Header、Content、Footer。

- Content 区域是一个大型 Slot 容器
- 不预设多个固定用途的 Slot（如 banner-slot、product-slot 等）
- 用户可自由拖入内容类部件或容器

**区域部件规则**:
| 区域 | 可接受 | 拒绝 |
|------|--------|------|
| Header | header 类型部件 | - |
| Content | banner, carousel, product, category, content, container, navigation, search, social, newsletter 等 | header, footer 类型部件 |
| Footer | footer 类型部件 | - |

**验收标准**:
- [x] 首页前端布局 (`layouts/homepage/default.phtml`) 只有一个 `content` slot
- [x] 后台编辑器结构视图 (`ThemeEditor/index.phtml`) 对应简化
- [x] Content 区域通过 `data-wslot-reject="header,footer"` 拒绝 header 和 footer 部件
- [x] 选择内容区域时，右侧部件面板隐藏 header 和 footer 分组 (`filterWidgetsByArea`)
- [x] 拖放验证阻止 header/footer 部件放入内容区域 (`handleSlotDragOver`, `handleSlotDrop`)

#### 3.1.2 基础容器系统

**需求 ID**: REQ-LAYOUT-002  
**优先级**: 高  
**状态**: ✅ 已完成  

行列布局等基础容器没有 accept 限制。

- 行容器（`.weline-row`）可容纳任意部件
- 列容器（`.weline-col`）可容纳任意部件
- 网格容器（`.weline-grid`）可容纳任意部件
- 容器可被任何 Slot 容纳

**验收标准**:
- [x] 基础容器 CSS 类已定义
- [x] 容器部件在 `widget.php` 中定义，type 为 `container`
- [x] WidgetScanner 支持 `container` 类型

### 3.2 部件配置面板

#### 3.2.1 配置项显示

**需求 ID**: REQ-CONFIG-001  
**优先级**: 高  
**状态**: ✅ 已完成  

点击部件时，左侧配置面板应显示该部件的可配置参数。

**问题描述**:
- 原问题: 点击部件后配置面板为空
- 根因: `WidgetScanner` 的 `ALLOWED_TYPES` 缺少 `container` 类型

**解决方案**:
- 添加 `container` 到 `ALLOWED_TYPES` 列表
- 确保 `params` 字段从 registry 正确传递到 API 输出

**验收标准**:
- [x] `WidgetScanner::ALLOWED_TYPES` 包含 `container`
- [x] API `/theme/backend/theme-editor/widgets` 返回部件的 `params` 字段
- [x] 配置面板正确渲染各类型字段（string、number、boolean、select、image、color、url、textarea）

#### 3.2.2 配置项国际化

**需求 ID**: REQ-CONFIG-002  
**优先级**: 高  
**状态**: ✅ 已完成  

配置项的 label、description、placeholder、options 支持多语言翻译。

**解决方案**:
- 在 `ThemeLayoutService::getAvailableWidgets()` 中对 params 进行翻译处理
- 使用 `__()` 函数翻译 label、description、placeholder
- 遍历并翻译 select 类型的 options

**验收标准**:
- [x] `ThemeLayoutService::getAvailableWidgets()` 翻译 params
- [x] 配置面板在中文环境显示中文标签
- [x] 配置面板在英文环境显示英文标签（需添加 i18n 词条）

### 3.3 拖放预览刷新

#### 3.3.1 即时预览更新

**需求 ID**: REQ-PREVIEW-001  
**优先级**: 高  
**状态**: ✅ 已完成  

拖放部件到 Slot 后，预览应即时更新。

**问题描述**:
- 原问题: 保存成功但预览未刷新
- 根因: 新部件在 iframe 中不存在，`updateWidgetPreviewInIframe` 静默失败

**解决方案**:
- 改进 `updateWidgetPreviewInIframe()` 函数
- 添加重试逻辑（最多 3 次，递增延迟）
- 检查 iframe 就绪状态
- 新部件或重试失败时 fallback 到 `loadLayoutPreview()` 完整刷新
- 添加视觉反馈动画（`.widget-updated` 类）

**验收标准**:
- [x] 新部件拖入后预览区域正确显示
- [x] iframe 未就绪时自动重试
- [x] 重试失败时触发完整预览刷新
- [x] 更新成功时部件有视觉反馈（脉冲动画）

#### 3.3.2 配置保存后预览更新

**需求 ID**: REQ-PREVIEW-002  
**优先级**: 中  
**状态**: ✅ 已完成  

修改部件配置并保存后，预览应更新。

**验收标准**:
- [x] 配置保存 API 返回 `preview_html`
- [x] 前端使用 `updateWidgetPreviewInIframe()` 更新对应部件
- [x] 已存在部件直接替换内容，无需完整刷新

#### 3.3.3 预览内链接拦截

**需求 ID**: REQ-PREVIEW-003  
**优先级**: 中  
**状态**: ✅ 已完成  

预览 iframe 内的链接点击应保持在编辑器内，而非跳转离开。

**解决方案**:
- 在 iframe 加载完成后设置链接拦截 (`setupIframeLinkInterception`)
- 内部链接转换为对应的预览 URL（带 `editor_mode=1` 和 `status=draft`）
- 外部链接在新标签页打开
- 根据链接路径自动判断页面类型（home/category/product/cart 等）

**验收标准**:
- [x] 点击 Logo 等内部链接不离开编辑器
- [x] 内部链接转换为预览模式 URL
- [x] 外部链接在新标签页打开
- [x] 页面类型选择器同步更新
