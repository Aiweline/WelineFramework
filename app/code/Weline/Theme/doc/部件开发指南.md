# 部件开发指南

> 版本: 2.1.0 | 更新时间: 2026-02-14

## 概述

部件（Widget）是 Weline 主题系统中的可复用 UI 组件。通过可视化编辑器，用户可以拖拽部件到页面的不同插槽中，实现页面布局的自由定制。

本文档覆盖：部件创建、注册、模板编写、**配置字段使用规则**（@param 与 type）、插槽定义、容器嵌套、以及 accept 规则。

## 快速开始

### 最简部件：3 步完成

**第 1 步：创建模板文件**

路径规范：`view/theme/frontend/widgets/{type}/{code}/default.phtml`

```php
<?php
/**
 * 我的自定义部件
 * 
 * @widget.code {my-widget}
 * @widget.name {我的部件}
 * @widget.description {这是一个示例部件}
 * @widget.type {content}
 * @widget.area {frontend}
 * @widget.position {["content"]}
 * @widget.page_layouts {["*"]}
 * 
 * @param title {default="Hello",type="string",label="标题"}
 * @param color {default="blue",type="select",label="颜色",options={blue:"蓝色",red:"红色",green:"绿色"}}
 */

$title = $this->getData('title') ?? 'Hello';
$color = $this->getData('color') ?? 'blue';
?>
<div class="my-widget" style="color: <?= htmlspecialchars($color) ?>">
    <h2><?= htmlspecialchars($title) ?></h2>
    <p>这是我的自定义部件内容</p>
</div>
```

**第 2 步：注册部件**

在 `extends/module/Weline_Widget/{YourModule}/widget.php` 中添加：

```php
return [
    // 简化格式：只需模板路径，元数据从模板注释自动解析
    'Weline_Theme::theme/frontend/widgets/content/my-widget/default.phtml',
];
```

**第 3 步：升级模块**

```bash
php bin/m s:up
```

完成！部件会自动出现在可视化编辑器的部件面板中。

## 部件定义格式

### 格式 1：简化格式（推荐）

元数据从模板的 `@widget.*` 注释自动解析：

```php
return [
    // 纯模板路径
    'Weline_Theme::theme/frontend/widgets/product/deals-of-day/default.phtml',
    
    // 模板路径 + 覆盖属性
    [
        'template'  => 'Weline_Theme::theme/frontend/widgets/banner/hero-slider/default.phtml',
        'exclusive' => true,  // 覆盖模板中的定义
    ],
];
```

### 格式 2：完整格式（兼容旧版）

在 `widget.php` 中直接定义全部配置：

```php
return [
    [
        'name'         => '今日特价',
        'description'  => '限时特价产品展示，支持倒计时',
        'type'         => 'product',
        'code'         => 'deals-of-day',
        'area'         => 'frontend',
        'position'     => ['content'],       // 允许放置的区域
        'compatible'   => false,
        'exclusive'    => false,
        'module'       => 'Weline_Theme',
        'version'      => '1.0.0',
        'template'     => 'Weline_Theme::theme/frontend/widgets/product/deals-of-day/default.phtml',
        'page_layouts' => ['homepage', 'cms_page'],  // 适用的页面布局
        'params'       => [
            'title' => [
                'type'    => 'string',
                'label'   => '标题',
                'default' => '今日特价',
            ],
            'limit' => [
                'type'    => 'number',
                'label'   => '显示数量',
                'default' => 4,
            ],
        ],
    ],
];
```

## 模板注释元数据

在模板文件顶部的 PHP 注释中使用 `@widget.*` 标签：

| 注释标签 | 必需 | 说明 | 示例 |
|----------|------|------|------|
| `@widget.code` | 是 | 部件唯一标识 | `{deals-of-day}` |
| `@widget.name` | 是 | 显示名称 | `{今日特价}` |
| `@widget.description` | 否 | 部件描述 | `{限时特价产品展示}` |
| `@widget.type` | 是 | 部件类型 | `{product}` |
| `@widget.area` | 否 | 前端/后端 | `{frontend}` |
| `@widget.position` | 是 | 允许放置的区域数组 | `{["content"]}` |
| `@widget.page_layouts` | 否 | 适用的页面布局 | `{["homepage", "cms_page"]}` |
| `@widget.is_container` | 否 | 是否为容器 | `{true}` |
| `@widget.exclusive` | 否 | 是否独占插槽 | `{true}` |

### 参数定义 (@param)

```php
* @param 参数名 {default=默认值,type="类型",label="标签",description="说明",options={key1:"标签1",key2:"标签2"}}
```

**书写规则（必读）**：

- 必须使用 **空格分隔** 格式：`@param 参数名 { ... }`。参数名与左花括号之间有一个空格。
- 主题 partial 可能使用 `@param.参数名 { }`（点号）格式，该格式**不会**被部件配置解析；部件一律用 `@param 参数名 { }`。
- 属性推荐使用双引号包裹含空格或特殊字符的值，如 `label="标题"`、`options={a:"选项A",b:"选项B"}`。
- 若未写 `type` 或写了系统不支持的 type，配置界面会按 **单行文本（text）** 显示；建议为每个参数显式指定 `type`。

**支持的参数类型**（与配置 UI 一一对应）：

| type 值 | 说明 | 配置界面 | 示例 |
|---------|------|----------|------|
| `string` / `text` | 短文本 | 单行输入框 | `type="string",label="标题"` |
| `textarea` / `html` / `richtext` | 长文本或 HTML | 多行/富文本 | `type="textarea",label="内容"` |
| `number` / `int` / `integer` / `float` | 数字 | 数字输入 | `type="number",label="数量"` |
| `bool` / `boolean` | 是/否 | 开关 | `type="bool",label="显示"` |
| `select` / `dropdown` | 枚举 | 下拉框（需配 `options`） | `type="select",label="样式",options={a:"样式A",b:"样式B"}` |
| `color` | 颜色 | 颜色选择器 | `type="color",label="背景色"` |
| `url` / `link` | 链接地址 | 链接输入 | `type="url",label="链接"` |
| `image` / `file` | 图片或文件 | 图片/文件上传 | `type="image",label="Logo"` |
| `array` / `list` | 列表数据 | 数组编辑 | `type="array",label="列表"` |
| `datetime` / `date` / `time` | 日期时间 | 日期时间选择 | `type="datetime",label="时间"` |
| `range` / `slider` | 数值范围 | 滑块 | `type="range",label="透明度"` |
| `icon` | 图标 | 图标选择器 | `type="icon",label="图标"` |

**类型与语义建议**：

- 标题、占位符、按钮文字等短文案 → `string` / `text`
- 描述、嵌入代码、富文本内容 → `textarea` / `html`
- 数量、限制、宽度等数字 → `number` / `int`
- 显示/隐藏、开关类 → `bool` / `boolean`
- 布局、样式、对齐等固定几项 → `select`，并写 `options={key1:"显示1",key2:"显示2"}`
- 颜色 → `color`
- 图片 URL、Logo、封面、二维码 → `image` 或 `file`
- 任何 URL、外链 → `url` 或 `link`
- 无法匹配上述语义或未写 type → 系统按 **text** 处理

**select 类型**：必须写 `options=`（不要写 `option=`）。格式为 `options={值:"显示标签",...}`，例如：

```php
* @param layout {default="full-width",type="select",label="布局",options={full-width:"全宽",with-left:"带左侧栏",with-right:"带右侧栏"}}
```

## 部件类型与区域规则

### 部件类型 (type)

| 类型 | 说明 | 默认可放置区域 |
|------|------|---------------|
| `header` | 头部部件 | header |
| `footer` | 底部部件 | footer |
| `banner` | 横幅/轮播 | banner, content |
| `product` | 产品展示 | content, left_sidebar, right_sidebar |
| `category` | 分类展示 | content, left_sidebar, right_sidebar |
| `sidebar` | 侧栏部件 | left_sidebar, right_sidebar, content |
| `navigation` | 导航部件 | header, left_sidebar |
| `search` | 搜索部件 | header, content |
| `content` | 内容部件 | content, left_sidebar, right_sidebar |
| `container` | 容器部件 | 必须有明确的 position |
| `social` | 社交部件 | footer, left_sidebar, right_sidebar, content |

### 位置 (position) 到区域映射

| position 值 | 对应区域 |
|-------------|---------|
| `header` | header |
| `footer` | footer |
| `sidebar` | left_sidebar, right_sidebar |
| `content` | content, banner |
| `banner` | banner |
| `*` / `all` | 所有区域 |

### 两层过滤体系

```
第 1 层：position + type（部件级）
  → 决定部件在哪些区域的面板中可见
  
第 2 层：accept（插槽级，硬白名单）
  → 如果插槽设置了 accept 列表，部件代码必须在列表中才能放入
  → 设置 accept="*" 表示接受所有部件
```

## 插槽定义

### 在模板中定义插槽

使用 `<w:slot>` 标签或 `data-wslot` 属性：

#### 方式 1：w:slot 标签（容器部件内部推荐）

```html
<w:slot id="widget-hero" 
        name="Hero轮播区域"
        accept="hero-slider,promo-banner"
        exclusive="true"
        position="content"
        wrapper="section"
        class="content-slot slot-hero">
    <div class="slot-placeholder">
        <span>拖放轮播图到此处</span>
    </div>
</w:slot>
```

#### 方式 2：data-wslot 属性（布局模板推荐）

```html
<div class="header-logo" 
     data-wslot="logo" 
     data-wslot-name="Logo"
     data-wslot-accept="logo,brand-logo"
     data-wslot-exclusive="true">
    <a href="/">默认 Logo</a>
</div>
```

### 插槽属性参考

| 属性 | 必需 | 说明 |
|------|------|------|
| `id` / `data-wslot` | 是 | 插槽唯一标识 |
| `name` / `data-wslot-name` | 否 | 编辑器中显示的名称 |
| `accept` / `data-wslot-accept` | 否 | 接受的部件代码列表（逗号分隔），硬白名单 |
| `exclusive` / `data-wslot-exclusive` | 否 | 是否独占（替换内容） |
| `multiple` / `data-wslot-multiple` | 否 | 是否允许多个部件 |
| `wrapper` | 否 | 包裹标签（仅 w:slot） |

**示例：**
```html
<!-- Hero 区域：只接受轮播和横幅部件 -->
<w:slot id="widget-hero" 
        accept="hero-slider,promo-banner"
        exclusive="true">
```

- 设置 `accept` 后，部件面板只显示接受的部件
- 设置 `accept="*"` 表示接受所有部件

## 容器部件开发

容器部件是包含子插槽的部件，支持嵌套。

### 容器部件定义

```php
[
    'name'         => 'Content 容器',
    'type'         => 'container',
    'code'         => 'content-container',
    'is_container' => true,
    'exclusive'    => true,
    'position'     => ['content'],
    'page_layouts' => ['*'],
    'template'     => 'Weline_Theme::theme/frontend/widgets/container/content/default.phtml',
]
```

### 容器模板示例

```php
<?php
/**
 * @widget.code {content-container}
 * @widget.name {Content 容器}
 * @widget.type {container}
 * @widget.is_container {true}
 * @widget.exclusive {true}
 * @widget.position {["content"]}
 * 
 * @param layout {default="full-width",type="select",label="布局类型",options={full-width:"全宽",with-left:"带左侧栏"}}
 */
$layout = $config['layout'] ?? 'full-width';
?>
<main class="widget-content-container layout-<?= htmlspecialchars($layout) ?>">
    <!-- Hero 区域 -->
    <w:slot id="widget-hero" 
            name="Hero轮播区域"
            accept="hero-slider,promo-banner"
            exclusive="true"
            position="content">
        <div class="slot-placeholder">
            <span>Hero 轮播区域</span>
        </div>
    </w:slot>
    
    <!-- 推荐区域 -->
    <w:slot id="widget-featured"
            name="推荐区域"
            accept="featured-products,category-grid,deals-of-day"
            multiple="true"
            position="content">
        <div class="slot-placeholder">
            <span>推荐区域</span>
        </div>
    </w:slot>
    
    <!-- 主内容区 -->
    <w:slot id="widget-main"
            name="主内容区"
            accept="*"
            multiple="true"
            position="content">
        <div class="slot-placeholder">
            <span>主内容区</span>
        </div>
    </w:slot>
</main>
```

### 嵌套渲染机制

后端 `SlotRendererService` 使用**迭代式多遍扫描**：

```
第 1 遍：发现顶级插槽（如 content 区域的 data-wslot）
  → 渲染容器部件（如 content-container）
  → 容器 HTML 中包含新的子插槽（widget-hero, widget-featured 等）

第 2 遍：发现第 1 遍生成的子插槽
  → 渲染子部件（如 hero-slider 放入 widget-hero）
  → 子部件可能再包含更深的插槽

第 N 遍：直到没有新的未处理插槽，或达到最大深度（10 层）
```

## 完整开发示例

### 示例：创建"品牌展示"部件

**1. 创建模板文件**

`view/theme/frontend/widgets/content/brand-showcase/default.phtml`

```php
<?php
/**
 * 品牌展示部件
 * 
 * @widget.code {brand-showcase}
 * @widget.name {品牌展示}
 * @widget.description {展示合作品牌的 Logo 网格}
 * @widget.type {content}
 * @widget.area {frontend}
 * @widget.position {["content"]}
 * @widget.page_layouts {["homepage", "cms_page"]}
 * 
 * @param title {default="合作品牌",type="string",label="标题"}
 * @param columns {default="6",type="select",label="每行列数",options={4:"4列",5:"5列",6:"6列",8:"8列"}}
 * @param show_title {default=true,type="bool",label="显示标题"}
 */

$title = $this->getData('title') ?? __('合作品牌');
$columns = (int)($this->getData('columns') ?? 6);
$showTitle = (bool)($this->getData('show_title') ?? true);
?>
<section class="brand-showcase">
    <?php if ($showTitle): ?>
        <h3 class="brand-showcase-title"><?= htmlspecialchars($title) ?></h3>
    <?php endif; ?>
    <div class="brand-grid" style="grid-template-columns: repeat(<?= $columns ?>, 1fr);">
        <!-- 品牌 Logo 列表 -->
    </div>
</section>

<style>
.brand-showcase { padding: 40px 0; text-align: center; }
.brand-showcase-title { margin-bottom: 30px; font-size: 1.5rem; }
.brand-grid { display: grid; gap: 20px; align-items: center; }
</style>
```

**2. 注册部件**

`extends/module/Weline_Widget/YourModule/widget.php`

```php
<?php
return [
    'Weline_Theme::theme/frontend/widgets/content/brand-showcase/default.phtml',
];
```

**3. 升级并测试**

```bash
# 升级模块
php bin/m s:up

# 如果 WLS 运行中，热重载
php bin/m server:reload
```

## 常见问题

### Q: 部件在面板中不显示？

检查以下几点：
1. `position` 是否匹配当前区域（如 content 区域需要 `position: ["content"]`）
2. `page_layouts` 是否包含当前页面类型（如首页需要包含 `homepage`）
3. 部件类型（type）是否被区域互斥规则排除（如 content 区域排除 header/footer 类型）
4. 运行 `php bin/m s:up` 确保部件已注册

### Q: accept 的作用是什么？

- `accept`：硬白名单。设置后，部件面板只显示接受的部件，不接受的部件不会显示
- `accept="*"`：表示接受所有部件

### Q: 容器部件的子插槽为什么没有渲染？

后端使用迭代式渲染，检查：
1. 子插槽的 `data-wslot` 属性是否正确
2. 子部件的 `slot_id` 是否与子插槽 ID 匹配
3. `SlotRendererService` 的日志，确认迭代是否到达子插槽层级

### Q: 如何让部件在所有区域可用？

设置 `position: ['*']` 即可在所有区域的面板中显示。

### Q: 如何创建跨模块部件？

在你的模块中创建 `extends/module/Weline_Widget/YourModule/widget.php`，引用你的模块的模板路径。部件系统会自动扫描所有模块的 widget 扩展点。
