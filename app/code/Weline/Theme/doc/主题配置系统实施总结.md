# 主题配置系统实施总结

## 已完成的工作

### 1. 核心Helper类创建

#### ✅ ThemeConfigManager (配置管理器)
**文件**: `app/code/Weline/Theme/Helper/ThemeConfigManager.php`

**功能**:
- 统一管理主题配置的读取、保存、验证
- 支持单个配置和批量配置保存
- 提供配置验证机制
- 自动清除缓存

**核心方法**:
- `getConfig()` - 获取配置
- `saveConfig()` - 保存单个配置
- `saveConfigs()` - 批量保存配置
- `getAllConfig()` - 获取所有配置
- `validateConfig()` - 验证配置值

#### ✅ PreviewManager (预览管理器)
**文件**: `app/code/Weline/Theme/Helper/PreviewManager.php`

**功能**:
- 管理预览模式的session数据
- 提供预览配置的临时存储和读取
- 清理预览配置

**核心方法**:
- `setPreviewConfig()` - 设置预览配置
- `getPreviewConfig()` - 获取预览配置
- `clearPreviewConfig()` - 清除预览配置
- `isPreviewMode()` - 检查是否在预览模式

#### ✅ ConfigLoader (配置加载器)
**文件**: `app/code/Weline/Theme/Helper/ConfigLoader.php`

**功能**:
- 在渲染时加载配置（优先级：预览配置 > 保存配置 > 默认配置）
- 处理配置的继承关系
- 提供便捷的配置获取方法

**核心方法**:
- `loadConfig()` - 加载配置（带优先级）
- `getLayoutConfig()` - 获取布局配置
- `getHeaderConfig()` - 获取Header配置
- `getColorConfig()` - 获取色系配置
- `getPartialConfig()` - 获取部件配置
- `getVariablesConfig()` - 获取变量配置

## 配置流程

### 保存配置流程
1. 用户在配置界面选择配置项
2. 前端收集配置数据
3. AJAX请求到 `theme/backend/config/layout/postSave`
4. 后端验证配置值
5. 使用 `ThemeConfigManager::saveConfigs()` 保存配置
6. 清除缓存
7. 返回结果

### 预览配置流程
1. 用户选择配置项
2. 前端构建预览URL
3. 请求到 `theme/backend/config/layout/preview`
4. 使用 `PreviewManager::setPreviewConfig()` 设置预览配置到Session
5. 重定向到预览页面
6. 预览页面渲染时，`ConfigLoader::loadConfig()` 优先读取预览配置

### 配置加载流程
1. 页面请求到达
2. `ConfigLoader::loadConfig()` 被调用
3. 检查 `PreviewManager::isPreviewMode()`
4. 如果在预览模式，读取Session中的预览配置
5. 如果不在预览模式，读取主题保存的配置
6. 如果配置不存在，使用默认配置
7. 应用配置到页面渲染

## 配置优先级

1. **预览配置** (Session) - 最高优先级
2. **保存的配置** (数据库) - 中等优先级
3. **默认配置** - 最低优先级

## 使用示例

### 保存配置
```php
use Weline\Theme\Helper\ThemeConfigManager;

// 保存单个配置
ThemeConfigManager::saveConfig($theme, 'colors', 'frontend', 'dark');

// 批量保存配置
ThemeConfigManager::saveConfigs($theme, [
    'layouts' => ['frontend' => ['account' => 'profile']],
    'colors' => ['frontend' => 'dark'],
]);
```

### 加载配置
```php
use Weline\Theme\Helper\ConfigLoader;

// 加载配置（自动处理优先级）
$color = ConfigLoader::getColorConfig($theme, 'frontend');
$layouts = ConfigLoader::getLayoutConfig($theme, 'frontend');
```

### 预览配置
```php
use Weline\Theme\Helper\PreviewManager;

// 设置预览配置
PreviewManager::setPreviewConfig($themeId, 'frontend', [
    'colors' => 'dark',
    'layouts' => ['account' => 'profile'],
]);

// 获取预览配置
$previewColor = PreviewManager::getPreviewConfig('colors', 'frontend');

// 清除预览配置
PreviewManager::clearPreviewConfig();
```

## 优势

1. **统一管理**: 所有配置操作都通过统一的Helper类进行
2. **优先级清晰**: 预览配置 > 保存配置 > 默认配置
3. **易于扩展**: 新增配置类型只需在Helper类中添加方法
4. **自动缓存**: 保存配置时自动清除相关缓存
5. **验证机制**: 保存前自动验证配置值有效性
6. **向后兼容**: 保持与现有代码的兼容性

## 注意事项

1. **缓存清除**: 保存配置后会自动清除相关缓存，但如果有其他缓存机制，需要手动清除
2. **Session过期**: 预览配置存储在Session中，Session过期后预览配置会失效
3. **配置验证**: 保存配置前会验证配置值，无效的配置不会被保存
4. **默认值**: 如果配置不存在，会使用默认值，确保系统正常运行
