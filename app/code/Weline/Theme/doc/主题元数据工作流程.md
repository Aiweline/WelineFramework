# 主题元数据工作流程

## 概述

主题元数据系统是 WelineFramework 中用于管理主题配置和翻译的核心机制。通过元数据系统，可以实现主题配置的多语言翻译、多范围配置（scope）等功能，支持多网站、多店铺等复杂场景。

## 核心概念

### 1. Meta 元数据

Meta 元数据是主题组件、布局、配置等信息的结构化描述，包括：
- **组件信息**：组件名称、描述、参数定义等
- **配置信息**：主题的各种配置项及其默认值
- **翻译信息**：多语言翻译内容

### 2. Scope（作用域）

Scope 是用于区分不同配置范围的关键机制：

- **默认 Scope**：`default`，用于主题编辑页面的配置
- **自定义 Scope**：如 `homepage`、`product`、`store_1`、`website_2` 等
- **多网站/多店铺**：每个网站或店铺可以使用不同的 scope，实现同一主题的不同配置

### 3. Meta Key 格式

Meta Key 是元数据的唯一标识符，格式如下：

```
@meta::{namespace}.{type}.{identify}.{group}.{field}
```

示例：
```
@meta::theme.component.pagination.info.name
@meta::theme.layout.homepage.config.title
@meta::theme.setting.color.primary
```

### 4. Scope 存储格式

当使用 scope 时，Meta Key 会添加 scope 后缀：

```
@meta::{key}|scope:{scope_value}
```

示例：
```
@meta::theme.component.pagination.info.name|scope:default
@meta::theme.component.pagination.info.name|scope:homepage
@meta::theme.component.pagination.info.name|scope:store_1
```

## 工作流程

### 1. 元数据定义阶段

在组件文件中定义 Meta 信息：

```php
<?php
/**
 * 组件：Pagination
 * 
 * 分页组件，用于数据分页导航
 * 
 * @component Pagination
 * @description 分页组件，用于数据分页导航
 * 
 * @param int currentPage 当前页码 [required] [default=1]
 * @param int totalPages 总页数 [required] [default=1]
 * @param string baseUrl 基础URL [required] [default='']
 * @param string pageParam 页码参数名 [default='page']
 */
// ... 组件代码 ...
?>
```

### 2. 元数据解析阶段

系统会自动解析组件文件中的 Meta 信息：

```php
use Weline\Theme\Helper\ComponentMetaParser;

$meta = ComponentMetaParser::parse($componentFile);
// 返回：
// [
//     'component' => 'Pagination',
//     'description' => '分页组件，用于数据分页导航',
//     'params' => [...]
// ]
```

### 3. 翻译收集阶段

在组件预览或使用时，系统会自动收集翻译词：

```php
use Weline\Framework\Event\EventsManager;

$eventsManager->dispatch('Weline_I18n::collect_translations', [
    'translations' => [
        [
            'word' => '@meta::theme.component.pagination.info.name',
            'translate' => 'Pagination分页组件',
            'module' => 'Weline_Theme'
        ]
    ],
    'module' => 'Weline_Theme'
]);
```

### 4. 翻译存储阶段

翻译值存储在 `i18n_locale_dictionary` 表中：

- **不带 Scope**：`@meta::theme.component.pagination.info.name`
- **带 Scope**：`@meta::theme.component.pagination.info.name|scope:default`

### 5. 翻译读取阶段

使用 `w:meta` 标签读取翻译值：

```html
<!-- 读取默认翻译值 -->
<w:meta>info.name</w:meta>

<!-- 读取指定 scope 的翻译值 -->
<w:meta scope="homepage">info.name</w:meta>
```

## 使用方法

### 1. 在组件文件中定义 Meta

```php
<?php
/**
 * 组件：Alert
 * 
 * 提示信息组件
 * 
 * @component Alert
 * @description 提示信息组件，支持多种类型和样式
 * 
 * @param string type 类型 [default='info']
 * @param string message 提示信息 [required]
 */
// ... 组件代码 ...
?>
```

### 2. 在模板中使用 w:meta 标签

#### 基本使用

```html
<!-- 直接显示翻译值 -->
<w:meta prefix="theme.component.alert">info.name</w:meta>

<!-- 显示翻译按钮（可编辑） -->
<w:meta type="translate" prefix="theme.component.alert">info.name</w:meta>
```

#### 使用 Scope

```html
<!-- 默认 scope（主题编辑页面） -->
<w:meta type="translate" scope="default" prefix="theme.component.alert">info.name</w:meta>

<!-- 自定义 scope（首页） -->
<w:meta type="translate" scope="homepage" prefix="theme.component.alert">info.name</w:meta>

<!-- 使用 PHP 变量作为 scope -->
<w:meta type="translate" scope="<?= $store->getScope() ?>" prefix="theme.component.alert">info.name</w:meta>
<w:meta type="translate" scope="{{store.scope}}" prefix="theme.component.alert">info.name</w:meta>
```

### 3. 在组件预览页面中使用

```php
<?php
// 解析组件 Meta 信息
$meta = ComponentMetaParser::parse($componentFile);
$metaKeyPrefix = 'theme.component.' . strtolower($component);

// 显示可翻译的字段
?>
<div class="component-info-item">
    <span class="component-info-label"><?= __('名称') ?>:</span>
    <w:meta type="translate" prefix="<?= $metaKeyPrefix ?>" scope="default">info.name</w:meta>
</div>
```

## Scope 应用场景

### 1. 主题编辑页面（Default Scope）

在主题编辑页面，所有配置使用 `default` scope：

```html
<w:meta type="translate" scope="default" prefix="theme.component.pagination">info.name</w:meta>
```

### 2. 多网站场景

不同网站使用不同的 scope：

```php
// 网站 1
<w:meta type="translate" scope="website_1" prefix="theme.component.pagination">info.name</w:meta>

// 网站 2
<w:meta type="translate" scope="website_2" prefix="theme.component.pagination">info.name</w:meta>
```

### 3. 多店铺场景

不同店铺使用不同的 scope：

```php
// 店铺 1
<w:meta type="translate" scope="store_1" prefix="theme.component.pagination">info.name</w:meta>

// 店铺 2
<w:meta type="translate" scope="store_2" prefix="theme.component.pagination">info.name</w:meta>
```

### 4. 页面类型场景

不同页面类型使用不同的 scope：

```php
// 首页
<w:meta type="translate" scope="homepage" prefix="theme.component.pagination">info.name</w:meta>

// 产品页
<w:meta type="translate" scope="product" prefix="theme.component.pagination">info.name</w:meta>

// 详情页
<w:meta type="translate" scope="detail" prefix="theme.component.pagination">info.name</w:meta>
```

### 5. 动态 Scope

使用 PHP 变量动态设置 scope：

```php
<?php
// 从主题配置中获取 scope
$scope = $theme->getScope() ?? 'default';

// 从店铺信息中获取 scope
$scope = $store->getScope() ?? 'default';

// 从网站信息中获取 scope
$scope = $website->getScope() ?? 'default';
?>
<w:meta type="translate" scope="<?= $scope ?>" prefix="theme.component.pagination">info.name</w:meta>
```

## 翻译优先级

系统按以下优先级读取翻译值：

1. **带 Scope 的翻译值**：`@meta::key|scope:value`
2. **默认翻译值**：`@meta::key`（不带 scope）
3. **Meta 定义中的默认值**：组件文件中定义的默认值

## 数据存储

### 1. 翻译数据存储

翻译数据存储在 `i18n_locale_dictionary` 表中：

| 字段 | 说明 | 示例 |
|------|------|------|
| `md5` | Meta Key 和 Locale 的 MD5 哈希 | `abc123...` |
| `string` | Meta Key（包含 scope） | `@meta::theme.component.pagination.info.name\|scope:default` |
| `locale` | 语言代码 | `zh_Hans_CN` |
| `translate` | 翻译值 | `分页组件` |

### 2. 主题配置存储

主题配置可以存储在主题表的 JSON 字段中：

```json
{
  "components": {
    "pagination": {
      "default": {
        "info": {
          "name": "分页组件",
          "description": "分页组件，用于数据分页导航"
        }
      },
      "homepage": {
        "info": {
          "name": "首页分页组件",
          "description": "首页专用的分页组件"
        }
      }
    }
  }
}
```

## 最佳实践

### 1. Scope 命名规范

- 使用有意义的名称：`homepage`、`product`、`store_1`
- 避免使用特殊字符
- 保持一致性：同一类型的 scope 使用相同的命名规则

### 2. Meta Key 命名规范

- 使用点号分隔的层级结构
- 保持简洁明了
- 遵循命名空间规范：`theme.{type}.{identify}.{group}.{field}`

### 3. 翻译管理

- 在主题编辑页面统一管理默认翻译（scope=default）
- 为不同 scope 提供独立的翻译管理界面
- 定期同步翻译内容

### 4. 性能优化

- 缓存翻译数据
- 批量加载翻译值
- 使用索引优化查询

## 完整示例

### 示例 1：组件定义和使用

**组件文件** (`pagination.phtml`)：
```php
<?php
/**
 * 组件：Pagination
 * 
 * @component Pagination
 * @description 分页组件，用于数据分页导航
 * 
 * @param int currentPage 当前页码 [default=1]
 */
// ... 组件代码 ...
?>
```

**模板中使用**：
```html
<!-- 显示组件名称（可翻译） -->
<h2><w:meta type="translate" prefix="theme.component.pagination" scope="<?= $scope ?>">info.name</w:meta></h2>

<!-- 显示组件描述（可翻译） -->
<p><w:meta type="translate" prefix="theme.component.pagination" scope="<?= $scope ?>">info.description</w:meta></p>
```

### 示例 2：多店铺场景

```php
<?php
// 获取当前店铺的 scope
$storeScope = $store->getScope() ?? 'default';

// 在模板中使用
?>
<div class="store-header">
    <h1>
        <w:meta type="translate" scope="<?= $storeScope ?>" prefix="theme.setting.store">title</w:meta>
    </h1>
    <p>
        <w:meta type="translate" scope="<?= $storeScope ?>" prefix="theme.setting.store">description</w:meta>
    </p>
</div>
```

### 示例 3：主题配置管理

```php
<?php
// 主题编辑页面（使用 default scope）
$scope = 'default';

// 获取主题配置
$themeConfig = $theme->getConfig($scope);

// 显示配置项
foreach ($themeConfig['components'] as $component => $config) {
    $prefix = "theme.component.{$component}";
    ?>
    <div class="config-item">
        <label>
            <w:meta type="translate" scope="<?= $scope ?>" prefix="<?= $prefix ?>">info.name</w:meta>
        </label>
        <input type="text" 
               value="<w:meta scope="<?= $scope ?>" prefix="<?= $prefix ?>">config.value</w:meta>">
    </div>
    <?php
}
?>
```

## 技术实现细节

### 1. Meta Key 解析

系统会自动解析和补全 Meta Key：

- `info.name` → `@meta::theme.component.pagination.info.name`（需要 prefix）
- `@meta.info.name` → `@meta::theme.component.pagination.info.name`（需要 prefix）
- `@meta::theme.component.pagination.info.name` → 直接使用

### 2. Scope 处理

- 默认值：`default`
- 支持字符串和 PHP 变量
- 存储格式：`@meta::key|scope:value`

### 3. 翻译读取逻辑

```php
// 1. 尝试读取带 scope 的翻译
$keyWithScope = $metaKey . '|scope:' . $scope;
$translation = getTranslation($keyWithScope);

// 2. 如果不存在，读取默认翻译
if (empty($translation)) {
    $translation = getTranslation($metaKey);
}

// 3. 如果仍不存在，使用默认值
if (empty($translation)) {
    $translation = $defaultValue;
}
```

## 总结

主题元数据系统通过 Meta Key 和 Scope 机制，实现了：

1. **统一管理**：所有主题配置和翻译统一管理
2. **多语言支持**：支持多语言翻译
3. **多范围配置**：通过 scope 实现不同范围的独立配置
4. **灵活扩展**：支持多网站、多店铺等复杂场景
5. **易于使用**：通过 `w:meta` 标签简化使用

通过合理使用 scope 机制，可以在同一主题下实现不同网站、店铺、页面的个性化配置，满足复杂的业务需求。
