# Weline Queue 模块 - 事件文档

## 概述

本文档详细说明了 Weline Queue 模块提供的 `Weline_Queue::{action}` 动态事件及其使用方法。该事件在队列执行操作时触发。

## 事件列表

### 1. Weline_Queue::{action} - 队列操作事件（动态事件）

#### 基本信息

- **事件名称**：`Weline_Queue::{action}`（动态事件模式）
- **事件类型**：队列操作事件（动态）
- **触发时机**：在队列执行操作时
- **触发位置**：`app/code/Weline/Queue/Controller/Backend/Queue.php` 第 154 行
- **配置文件**：`app/code/Weline/Queue/etc/event.xml`
- **动态说明**：`{action}` 是动态占位符，可以是 `add`、`edit` 等

#### 功能说明

`Weline_Queue::{action}` 是一个动态事件，其中 `{action}` 是动态占位符。该事件在队列执行操作时触发，允许其他模块监听队列操作。

**动态事件说明**：
- 实际触发的事件名会根据 `action` 的值动态生成
- 例如：`action = 'add'` 时，实际事件名为 `Weline_Queue::add`
- 例如：`action = 'edit'` 时，实际事件名为 `Weline_Queue::edit`

该事件主要用于：
- 监听队列添加操作
- 监听队列编辑操作
- 队列操作日志记录
- 队列操作后处理

#### 触发时机

```php
// app/code/Weline/Queue/Controller/Backend/Queue.php
$data = ['queue' => $this->queue];
$this->getEventManager()->dispatch('Weline_Queue::' . ($edit ? 'edit' : 'add'), $data);
```

#### 使用场景

- **队列添加**：监听队列添加操作（`Weline_Queue::add`）
- **队列编辑**：监听队列编辑操作（`Weline_Queue::edit`）
- **日志记录**：记录队列操作的详细信息
- **后处理**：在队列操作后执行后续处理

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Queue::add">
        <observer name="Your_Module::queue_add"
                  instance="Your\Module\Observer\QueueAddObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
    <event name="Weline_Queue::edit">
        <observer name="Your_Module::queue_edit"
                  instance="Your\Module\Observer\QueueEditObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

#### 事件数据

`Weline_Queue::{action}` 事件传递的数据：

```php
[
    'queue' => object,  // Queue 对象实例
]
```
