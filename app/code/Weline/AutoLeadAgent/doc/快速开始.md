# 快速开始指南

## 📋 前置要求

- PHP 8.1+
- **Chrome 浏览器**（任务执行需在 Chrome 内）
- WelineFramework 已安装
- Weline_Store 模块已安装
- Weline_I18n 模块已安装
- Weline_Backend 模块已安装

## 🚀 安装步骤

### 1. 安装模块

```bash
# 进入项目根目录
cd /path/to/weline-framework

# 运行模块安装
php bin/w setup:upgrade
```

### 2. 安装浏览器扩展（必需）

任务执行依赖 AutoLeadAgent 扩展，提供 MCP 工具（browser_navigate、browser_snapshot、browser_extract 等）：

1. 打开 `chrome://extensions/`，开启「开发者模式」
2. 点击「加载已解压的扩展程序」
3. 选择 `app/code/Weline/AutoLeadAgent/browser-extension/` 目录

### 3. 配置端侧模型（可选）

进入后台「系统 → 自动寻客 → 配置」：

- 选择 Hugging Face 模型并下载
- 支持 Chrome Built-in AI 或 WebLLM 推理

### 4. 编译 WASM（可选）

如需 WASM 决策（agent_brain），使用 Emscripten 编译，参见 [WASM 编译指南](./WASM编译指南.md)。

### 5. 注册 WASM 哈希

如果编译了WASM文件，需要注册哈希值：

```bash
# 自动注册（在安装脚本中已实现）
# 或手动调用 WasmService::registerHash()
```

## ⚙️ 基础配置

### 1. 生成Token

通过API生成Token：

```bash
curl -X POST http://your-domain/api/v1/auto-lead-agent/token \
  -H "Content-Type: application/json" \
  -d '{
    "domain": "your-domain.com",
    "ttl": 3600
  }'
```

### 2. 前端集成

在页面中引入安全外壳：

```html
<script src="/Weline/AutoLeadAgent/view/statics/js/agent-shell.js"></script>
```

安全外壳会自动：
- 检查域名授权
- 验证Token
- 动态加载核心代码
- 加载并验证WASM模块

## 📖 基本使用

### 1. 后台执行任务

1. 登录后台 → 系统 → 自动寻客
2. 确保已安装扩展、配置模型
3. 创建任务并选择来源类型、目标网站
4. 点击启动，任务将按优先级执行：ReAct Agent → AutonomousPipeline → 传统管道

### 2. 创建搜索任务（API）

```bash
curl -X POST http://your-domain/api/v1/auto-lead-agent/search/create \
  -H "Content-Type: application/json" \
  -d '{
    "store_id": 1
  }'
```

### 3. 查看搜索结果

```bash
curl http://your-domain/api/v1/auto-lead-agent/search/1
```

### 4. 后台管理

访问后台管理界面：
```
http://your-domain/admin/auto-lead-agent/backend/index/index
```

## ⚠️ 常见问题

### 1. 模块安装失败

**问题**: 运行 `setup:upgrade` 时报错

**解决**:
- 检查PHP版本是否 >= 8.1
- 检查依赖模块是否已安装
- 查看错误日志

### 2. WASM文件不存在

**问题**: API返回"WASM文件不存在"

**解决**:
- 检查 `view/statics/wasm/agent-core.wasm` 文件是否存在
- 如果不存在，需要编译WASM文件
- 或暂时跳过WASM功能

### 3. Token验证失败

**问题**: 前端显示"Token验证失败"

**解决**:
- 检查Token是否过期
- 检查域名是否匹配
- 重新生成Token
