# 快速开始指南

## 📋 前置要求

- PHP 8.1+
- WelineFramework 已安装
- Weline_Store 模块已安装
- Weline_I18n 模块已安装
- Weline_Backend 模块已安装

## 🚀 安装步骤

### 1. 安装模块

```bash
# 进入项目根目录
cd /path/to/weline-framework

# 运行模块安装
php bin/w setup:upgrade
```

### 2. 编译WASM文件（可选）

如果需要使用WASM核心算法，需要编译WASM文件：

```bash
# 进入WASM源码目录
cd app/code/Weline/AutoLeadAgent/wasm-src

# 按照 README.md 中的说明编译
# 需要安装 Emscripten SDK
```

详细编译说明请参考 [WASM编译指南](./WASM编译指南.md)

### 3. 注册WASM哈希

如果编译了WASM文件，需要注册哈希值：

```bash
# 自动注册（在安装脚本中已实现）
# 或手动调用 WasmService::registerHash()
```

## ⚙️ 基础配置

### 1. 生成Token

通过API生成Token：

```bash
curl -X POST http://your-domain/api/v1/auto-lead-agent/token \
  -H "Content-Type: application/json" \
  -d '{
    "domain": "your-domain.com",
    "ttl": 3600
  }'
```

### 2. 前端集成

在页面中引入安全外壳：

```html
<script src="/Weline/AutoLeadAgent/view/statics/js/agent-shell.js"></script>
```

安全外壳会自动：
- 检查域名授权
- 验证Token
- 动态加载核心代码
- 加载并验证WASM模块

## ⚠️ 常见问题

### 1. 模块安装失败

**问题**: 运行 `setup:upgrade` 时报错

**解决**:
- 检查PHP版本是否 >= 8.1
- 检查依赖模块是否已安装
- 查看错误日志

### 2. WASM文件不存在

**问题**: API返回"WASM文件不存在"

**解决**:
- 检查 `view/statics/wasm/agent-core.wasm` 文件是否存在
- 如果不存在，需要编译WASM文件
- 或暂时跳过WASM功能

### 3. Token验证失败

**问题**: 前端显示"Token验证失败"

**解决**:
- 检查Token是否过期
- 检查域名是否匹配
- 重新生成Token
