# 架构设计

## 📋 概述

Weline_AutoLeadAgent 采用三层动态防御架构，确保核心代码和算法的安全性。

## 🏗️ 系统架构

### 整体架构图

```
┌─────────────────────────────────────────┐
│  安全外壳层 (Shell & Validation)        │
│  - 域名验证                              │
│  - Token 验证                            │
│  - 动态代码加载                          │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  WASM 核心层 (Secure Brain)              │
│  - 客户画像分析算法                      │
│  - 客户评分算法                          │
│  - 数据清洗规则                          │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  LLM 引擎层 (Agent Engine)               │
│  - WebLLM (Phi-3 Mini)                   │
│  - ReAct Agent 循环                      │
│  - 浏览器工具集成                        │
└─────────────────────────────────────────┘
```

## 🔒 安全架构

### 三层动态防御

#### 1. 安全外壳层

**功能**:
- 域名验证：检查 `window.location.hostname`
- Token验证：验证JWT Token的有效性
- 动态代码加载：核心JS通过API动态加载

**实现**:
- `agent-shell.js` - 安全外壳实现
- Token存储在localStorage
- 定时检查Token过期

#### 2. WASM核心层

**功能**:
- 客户评分算法
- 特征提取算法
- 画像匹配算法
- 数据清洗规则

**实现**:
- C/C++源码编译为WASM
- SHA-256哈希验证
- 随机优化参数确保每次编译不同

#### 3. LLM引擎层

**功能**:
- WebLLM模型加载
- ReAct Agent循环
- 浏览器工具调用

**实现**:
- 隔离运行环境
- 工具函数封装
- 状态管理

## 📊 数据流

### 搜索任务流程

```
1. 用户创建搜索任务
   ↓
2. 提取店铺画像
   ↓
3. 分析客户特征
   ↓
4. LLM生成搜索关键词
   ↓
5. 执行网络搜索
   ↓
6. 分析搜索结果
   ↓
7. WASM算法匹配评分
   ↓
8. 保存潜在客户
```

### Token验证流程

```
1. 前端请求Token
   ↓
2. 后端生成JWT Token
   ↓
3. Token包含域名、过期时间、WASM哈希
   ↓
4. 前端存储Token
   ↓
5. 每次请求验证Token
   ↓
6. 检查域名、过期时间
   ↓
7. 验证WASM哈希
```

## 🗄️ 数据库设计

### 表结构

#### 1. weline_auto_lead_agent_token

存储Token信息：

- `token_id` - 主键
- `token` - Token字符串
- `domain` - 授权域名
- `expires_at` - 过期时间
- `wasm_hash` - WASM哈希值

#### 2. weline_auto_lead_agent_wasm_hash

存储WASM哈希记录：

- `hash_id` - 主键
- `wasm_path` - WASM文件路径
- `hash_value` - SHA-256哈希值
- `version` - 版本号

#### 3. weline_auto_lead_agent_lead_candidate

存储潜在客户：

- `candidate_id` - 主键
- `store_id` - 店铺ID
- `profile_data` - 客户画像（JSON）
- `score` - 匹配分数
- `source_url` - 来源URL
- `status` - 状态

#### 4. weline_auto_lead_agent_search_task

存储搜索任务：

- `task_id` - 主键
- `store_id` - 店铺ID
- `status` - 状态
- `progress` - 进度
- `result_data` - 结果数据（JSON）

## 🌐 前端架构

### 模块化设计

```

  ↓
agent-core.js       # 核心逻辑（动态加载）
  ↓

  ↓
```

### 加载顺序

1. 加载安全外壳

4. 加载WASM模块
5. 初始化LLM引擎
6. 启动Agent

## 🔐 安全机制

### 1. 域名锁死

- 前端检查 `window.location.hostname`
- 后端验证Token中的域名
- 跨域请求被拒绝

### 2. 时效性Token

- JWT包含 `exp` 字段
- 前端定时检查过期
- 过期后自动阻止运行

### 3. 动态代码加载

- 核心JS不暴露在静态文件
- 通过API动态加载
- 可以添加代码混淆

### 4. WASM核心

- 关键算法编译为WASM
- 提高反编译难度
- 性能优化

### 5. 内容哈希验证

- SHA-256验证WASM文件
- Token中包含哈希值
- 确保文件完整性

## 🚀 性能优化

### 1. 缓存策略

- Token缓存（localStorage）
- WASM文件缓存
- 搜索结果缓存

### 2. 异步处理

- 搜索任务异步执行
- 不阻塞主线程
- 使用Promise/async-await

### 3. 资源优化

- WASM文件压缩
- JavaScript代码压缩
- CDN加速
