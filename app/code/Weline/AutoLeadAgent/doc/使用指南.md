## 📋 模块概述

Weline_AutoLeadAgent 是一个智能寻客系统，通过分析店铺描述，自动在网上查找具有相同客户画像的潜在客户。

### 核心特性

- **自动寻客**: 基于店铺画像自动搜索匹配客户
- **ReAct Agent**: 模型决策 + MCP 工具调用，完全模型驱动
- **端侧推理**: Chrome AI / WebLLM，支持 Hugging Face 模型
- **MCP 集成**: 浏览器扩展提供 browser_navigate、browser_snapshot、browser_extract 等工具

## 🖥️ 后台管理

### 访问后台

登录后台管理系统，在"系统管理"菜单下找到"自动寻客"。

### 功能模块

#### 1. 搜索任务

查看和管理所有搜索任务：

- **任务列表**: 显示所有搜索任务
- **任务状态**: pending/running/completed/failed/cancelled
- **任务进度**: 0-100%
- **操作**: 查看详情、重新执行、取消任务

#### 2. 潜在客户

查看和管理找到的潜在客户：

- **客户列表**: 显示所有潜在客户
- **匹配分数**: 0-100分
- **客户状态**: pending/verified/rejected
- **操作**: 查看详情、标记验证、导出数据

#### 3. Token管理

管理Agent Token：

- **Token列表**: 显示所有Token
- **Token状态**: 有效/过期
- **操作**: 生成新Token、撤销Token、查看详情

#### 4. WASM管理

管理WASM文件：

- **哈希记录**: 显示所有WASM版本
- **版本管理**: 查看不同版本的哈希值
- **操作**: 注册新版本、查看详情

#### 5. 配置管理

- **端侧模型**: 选择 Hugging Face 模型、启用/禁用、缓存大小
- **网络配置**: 镜像/代理（HuggingFace 下载）
- **Token设置**: Token 默认 TTL 等

## 🔌 API使用

### 认证

所有API请求需要Token认证（除了Token生成接口）。

#### 获取Token

```bash
POST /api/v1/auto-lead-agent/token
Content-Type: application/json

{
  "domain": "your-domain.com",
  "ttl": 3600
}
```

**响应**:
```json
{
  "code": 200,
  "msg": "Token生成成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "domain": "your-domain.com",
    "ttl": 3600
  }
}
```

#### 验证Token

```bash
GET /api/v1/auto-lead-agent/token/validate?token=xxx&domain=your-domain.com
```

### 搜索任务API

#### 创建搜索任务

```bash
POST /api/v1/auto-lead-agent/search/create
Content-Type: application/json
X-Agent-Token: your-token

{
  "store_id": 1
}
```

#### 获取搜索结果

```bash
GET /api/v1/auto-lead-agent/search/{taskId}
X-Agent-Token: your-token
```

### WASM API

#### 获取WASM哈希

```bash
GET /api/v1/auto-lead-agent/wasm/hash
X-Agent-Token: your-token
```

#### 下载WASM文件

```bash
GET /api/v1/auto-lead-agent/wasm/download
X-Agent-Token: your-token
```

### 核心代码API

#### 获取核心代码

```bash
GET /api/v1/auto-lead-agent/core
X-Agent-Token: your-token
```

返回JavaScript代码，用于动态加载。

## 🌐 前端集成

### 基础集成

在页面中引入安全外壳：

```html
<script src="/Weline/AutoLeadAgent/view/statics/js/agent-shell.js"></script>
```

### 初始化流程

1. **域名验证**: 检查当前域名是否在授权列表
2. **Token验证**: 从localStorage读取Token并验证
3. **加载核心代码**: 通过API动态加载核心JS
4. **加载WASM**: 下载并验证WASM模块
5. **初始化Agent**: 启动Agent引擎

### 使用示例

```javascript
// 等待Agent初始化完成
if (window.AgentShell && window.AgentShell.coreLoaded) {
    // Agent已就绪
    const core = window.agentCore;
    
    // 使用WASM算法
    const score = core.calculateCustomerScore(profileData);
    console.log('客户评分:', score);
}
```

### Token管理

```javascript
// 获取Token
fetch('/api/v1/auto-lead-agent/token', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
    },
    body: JSON.stringify({
        domain: window.location.hostname,
        ttl: 3600
    })
})
.then(response => response.json())
.then(data => {
    // 保存Token
    localStorage.setItem('auto_lead_agent_token', data.data.token);
});
```

## ⚙️ 配置说明

### 环境变量

在 `app/etc/env.php` 中配置：

```php
return [
    'auto_lead_agent' => [
        'secret_key' => 'your-secret-key-change-in-production',
        'allowed_domains' => [
            'your-domain.com',
            'www.your-domain.com',
        ],
        'default_token_ttl' => 3600, // 秒
        'wasm_path' => BP . '/app/code/Weline/AutoLeadAgent/view/statics/wasm/agent-core.wasm',
    ],
];
```

### 数据库配置

模块会自动创建以下表：

- `weline_auto_lead_agent_token` - Token存储
- `weline_auto_lead_agent_wasm_hash` - WASM哈希记录
- `weline_auto_lead_agent_lead_candidate` - 潜在客户
- `weline_auto_lead_agent_search_task` - 搜索任务

## 💡 最佳实践

### 1. Token管理

- **定期更新**: Token应定期更新，建议TTL不超过24小时
- **安全存储**: Token存储在localStorage，注意XSS防护
- **域名绑定**: 每个Token绑定特定域名，提高安全性

### 2. 搜索任务

- **批量处理**: 可以创建多个搜索任务并行处理
- **结果缓存**: 搜索结果可以缓存，避免重复搜索
- **定期清理**: 定期清理过期的任务和结果

### 3. 性能优化

- **WASM缓存**: WASM文件可以缓存，减少下载时间
- **异步处理**: 搜索任务应异步处理，避免阻塞
- **分页加载**: 大量数据应分页加载

### 4. 安全建议

- **HTTPS**: 生产环境必须使用HTTPS
- **Token加密**: Token应使用强密钥加密
- **域名白名单**: 严格限制授权域名
- **WASM验证**: 始终验证WASM文件哈希

## 🔍 故障排查

### Token相关问题

**问题**: Token验证失败

**排查步骤**:
1. 检查Token是否过期
2. 检查域名是否匹配
3. 检查Token格式是否正确
4. 查看服务器日志

### WASM相关问题

**问题**: WASM加载失败

**排查步骤**:
1. 检查WASM文件是否存在
2. 检查WASM哈希是否匹配
3. 检查浏览器是否支持WebAssembly
4. 查看浏览器控制台错误

### 搜索任务问题

**问题**: 搜索任务失败

**排查步骤**:
1. 检查店铺数据是否存在
2. 检查网络连接
3. 查看任务日志
4. 检查LLM服务是否可用
