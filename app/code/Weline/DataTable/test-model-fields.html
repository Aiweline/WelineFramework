DataTable 模型字段获取功能测试

强大的容错能力

DataTable 现在具备多种字段获取方式，即使模型类没有实现特定方法，也能智能推断字段信息。

字段获取方法（按优先级）：
  1. getFields() - 直接获取字段数组
  2. getTableFields() - 获取表字段信息
  3. getSchema() - 获取数据库模式
  4. 反射获取属性 - 通过反射获取公共属性
  5. 表名推断 - 根据表名智能推断字段
  6. 类名推断 - 根据模型类名推断字段
  7. 默认字段 - 使用通用默认字段
智能特性：
  • 自动过滤敏感字段（password、token等）
  • 根据字段名智能设置列宽
  • 根据字段类型选择过滤器
  • 状态字段自动生成选项
  • 限制显示字段数量，避免表格过宽
  • 友好的错误日志记录
  • 不会因为字段获取失败而中断

使用示例

1. 最简单的用法（推荐）：
<w:d-table model="WeShop\Store\Model\Store" scope="store-listing"></w:d-table>
2. 带配置的用法：
<w:d-table model="WeShop\Store\Model\Store" scope="store-listing" page-size="50" searchable="true"></w:d-table>
3. 手动配置部分字段（其余自动补全）：
<w:d-table model="WeShop\Store\Model\Store" scope="store-listing"> <w:t-header> <w:field belong="t-header" name="store_id" sortable="true" width="80">ID</w:field> <w:field belong="t-header" name="name" sortable="true" width="200">名称</w:field> </w:t-header> </w:d-table>

支持的模型类型

预定义字段映射：
  • Store - store_id, name, status, created_at, updated_at
  • Product - product_id, name, sku, price, status, created_at
  • User - user_id, username, email, status, created_at
  • Order - order_id, order_number, customer_id, total, status
  • Category - category_id, name, parent_id, sort_order, status
  • Customer - customer_id, name, email, phone, status
  • Admin - admin_id, username, email, role, status
  • Config - config_id, key, value, group, created_at
  • Log - log_id, level, message, context, created_at
  • File - file_id, name, path, size, type, created_at
智能推断规则：
  • 根据模型类名自动匹配字段
  • 根据表名模式匹配字段
  • 支持自定义字段获取方法
  • 自动识别常见字段类型
  • 智能设置列宽和过滤器
  • 自动过滤敏感字段
  • 限制显示字段数量

错误处理

容错机制：
  • 模型不存在 - 使用默认字段，不会中断页面加载
  • 字段方法不存在 - 尝试其他获取方式
  • 字段获取失败 - 记录错误日志，使用默认字段
  • 表名获取失败 - 根据类名推断字段
  • 所有方法都失败 - 使用通用默认字段
建议：

为了获得最佳体验，建议在模型类中实现 getFields() 方法：

public function getFields() { return ['store_id', 'name', 'status', 'created_at', 'updated_at']; }
