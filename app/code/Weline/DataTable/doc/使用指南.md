# Weline DataTable 使用指南

## 概述

Weline DataTable 是一个强大的数据表格模块，提供自动生成和手动配置两种模式，支持完整的增删改查功能。

### 主要特性

- **自动字段生成**: 根据模型自动生成表格字段和表单字段
- **多模型支持**: 支持多个模型的联合查询和JOIN操作
- **实时编辑**: 支持单元格双击编辑，实时保存
- **表单弹窗**: 支持模态框表单进行数据的增删改操作
- **高级筛选**: 支持多种字段类型的筛选功能
- **字段配置**: 支持字段显示/隐藏、排序、宽度调整
- **主题定制**: 支持表格主题和样式的自定义
- **响应式设计**: 支持移动端和桌面端的响应式布局

### 版本信息

- **当前版本**: 2.0.0
- **最低框架版本**: Weline Framework 1.0+
- **PHP版本要求**: PHP 7.4+
- **浏览器支持**: Chrome 60+, Firefox 55+, Safari 12+, Edge 79+

## 快速开始

### 1. 基本用法

最简单的用法，只需传入模型和范围参数：

```phtml
<w:d-table model="Weline\DataTable\Model\TestUser" scope="user-demo"></w:d-table>
```

### 2. 完整配置示例

```phtml
<w:d-table model="Weline\DataTable\Model\TestUser" scope="user-management" 
           page-size="20" show-pagination="true" show-toolbar="true">
    
    <!-- 表单配置 -->
    <w:d-form for="add,edit" title="用户管理" form-mode="modal">
        <w:field name="username" type="text" label="用户名" required="true"></w:field>
        <w:field name="email" type="email" label="邮箱" required="true"></w:field>
        <w:field name="status" type="select" label="状态" options="1:启用,0:禁用"></w:field>
    </w:d-form>
    
    <!-- 过滤器配置 -->
    <w:t-filter>
        <w:field belong="t-filter" name="username" type="text" placeholder="搜索用户名"></w:field>
        <w:field belong="t-filter" name="status" type="select" options="1:启用,0:禁用"></w:field>
    </w:t-filter>
    
    <!-- 表头配置 -->
    <w:t-header>
        <w:field belong="t-header" name="user_id" sortable="true" width="80">ID</w:field>
        <w:field belong="t-header" name="username" sortable="true" width="150">用户名</w:field>
        <w:field belong="t-header" name="email" sortable="true" width="200">邮箱</w:field>
        <w:field belong="t-header" name="status" sortable="true" width="100">状态</w:field>
        <w:field belong="t-header" name="created_at" sortable="true" width="150">创建时间</w:field>
    </w:t-header>
</w:d-table>
```

## 标签属性说明

### d-table 属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| model | string | - | 模型类名（必需） |
| scope | string | - | 表格范围标识（必需） |
| page-size | int | 20 | 每页显示记录数 |
| show-pagination | bool | true | 是否显示分页 |
| show-toolbar | bool | true | 是否显示工具栏 |
| show-config | bool | true | 是否显示配置按钮 |
| height | string | auto | 表格高度 |
| width | string | 100% | 表格宽度 |
| editable | bool | false | 是否可编辑 |
| inline-edit | bool | false | 是否支持行内编辑 |
| modal-edit | bool | true | 是否支持弹窗编辑 |

### d-form 属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| for | string | - | 表单用途（add,edit,view） |
| title | string | - | 表单标题 |
| form-mode | string | modal | 表单模式（modal,inline） |
| form-title | string | - | 弹窗标题 |

### field 属性

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| name | string | - | 字段名（必需） |
| type | string | text | 字段类型 |
| label | string | - | 字段标签 |
| placeholder | string | - | 占位符 |
| required | bool | false | 是否必填 |
| options | string | - | 选项（用于select类型） |
| width | string | auto | 列宽 |
| sortable | bool | false | 是否可排序 |
| searchable | bool | true | 是否可搜索 |

```phtml
    <w:t-header>
        <w:field belong="t-header" name="id" width="80">ID</w:field>

```

支持多模型联合查询：

```phtml

        <w:field belong="t-header" name="u.user_id">用户ID</w:field>
        <w:field belong="t-header" name="u.username">用户名</w:field>
        <w:field belong="t-header" name="o.order_id">订单ID</w:field>
        <w:field belong="t-header" name="o.total_amount">订单金额</w:field>
    </w:t-header>
```

## API 接口

### API 路由格式

DataTable 模块的 API 路由格式为：
```
{模块router}/rest/v1/{控制器名}/{方法名}
```

根据 `etc/env.php` 配置，DataTable 模块的路由别名是 `datatable`。

**实际访问 URL**：
- 前端 API：`/api/rest/v1/datatable/{控制器名}/{方法名}`
- 后端 API：`/{admin_router}/api/rest/v1/datatable/{控制器名}/{方法名}`

| 控制器文件 | 路由路径 | 后端访问 URL | 说明 |
|-----------|---------|-------------|------|
| `Api/Rest/V1/Form.php` | `datatable/rest/v1/form` | `/{admin_router}/api/rest/v1/datatable/form/fields` | 表单相关 API（后端） |
| `Api/Rest/V1/DataTable.php` | `datatable/rest/v1/data-table` | `/{admin_router}/api/rest/v1/datatable/data-table/data` | 数据表格 API（后端） |
| `Api/Rest/V1/Trash.php` | `datatable/rest/v1/trash` | `/{admin_router}/api/rest/v1/datatable/trash/data` | 回收站 API（后端） |

**注意**：
- 类名 `DataTable` 会被自动转换为 kebab-case：`data-table`
- 方法名也会被转换：`postFields()` → `fields` (POST 请求)
- **所有 DataTable 模块的 API 都是后端 API，需要登录验证**
- **后端 JavaScript 使用 `window.api()` 时，路径应该是 `datatable/form/fields`，而不是 `datatable/rest/v1/form/fields`**（因为 `api_host` 已包含 `/api/rest/v1/` 前缀）

### Form 控制器 API（后端，需要登录）

#### 获取表单字段
- **路由路径**: `datatable/rest/v1/form/fields`
- **后端访问 URL**: `/{admin_router}/api/rest/v1/datatable/form/fields`
- **JavaScript 调用**: `window.api('datatable/form/fields')`（后端页面）
- **方法**: `POST` 或 `GET`
- **参数**:
  ```json
  {
    "model": "Weline\\DataTable\\Model\\TestUser",
    "scope": "form",
    "form_id": "user-form",
    "exclude_fields": ["id", "create_time"],
    "include_fields": ["name", "email"],
    "manual_fields": []
  }
  ```

#### 获取表单记录（编辑模式）
- **路由路径**: `datatable/rest/v1/form/record`
- **后端访问 URL**: `/{admin_router}/api/rest/v1/datatable/form/record`
- **JavaScript 调用**: `window.api('datatable/form/record')`（后端页面）
- **方法**: `POST`
- **参数**:
  ```json
  {
    "model": "Weline\\DataTable\\Model\\TestUser",
    "record_id": 1
  }
  ```

### DataTable 控制器 API（后端，需要登录）

#### 获取数据表格数据
- **路由路径**: `datatable/rest/v1/data-table/data`
- **后端访问 URL**: `/{admin_router}/api/rest/v1/datatable/data-table/data`
- **JavaScript 调用**: `window.api('datatable/data-table/data')`（后端页面）
- **方法**: `POST` 或 `GET`
- **参数**:
  ```json
  {
    "model": "Weline\\DataTable\\Model\\TestUser",
    "scope": "table",
    "page": 1,
    "limit": 20,
    "filters": {},
    "sort": {},
    "join": "",
    "model_config": {}
  }
  ```

#### 其他常用 API
- `datatable/rest/v1/data-table/fields` - POST - 获取字段
- `datatable/rest/v1/data-table/save-config` - POST - 保存配置
- `datatable/rest/v1/data-table/create` - POST - 创建记录
- `datatable/rest/v1/data-table` - UPDATE/DELETE - 更新/删除记录
- `datatable/rest/v1/data-table/getConfig` - GET - 获取配置

### Trash 控制器 API（后端，需要登录）

- `datatable/rest/v1/trash/data` - GET - 获取回收站数据
- `datatable/rest/v1/trash/restore` - GET - 恢复记录
- `datatable/rest/v1/trash/force-delete` - GET - 永久删除
- `datatable/rest/v1/trash/empty` - GET - 清空回收站

### API 访问方式

#### 方式1：使用命令行工具

```bash
# 获取表单字段（后端 API，需要登录）
# 注意：命令行工具使用路由路径，不需要 /api/rest/v1/ 前缀
php bin/w http:request -u "datatable/rest/v1/form/fields" -m POST -d '{"model":"Weline\\DataTable\\Model\\TestUser","scope":"form"}' -b --login

# 获取数据表格数据（后端 API，需要登录）
php bin/w http:request -u "datatable/rest/v1/data-table/data" -m POST -d '{"model":"Weline\\DataTable\\Model\\TestUser","scope":"table","page":1,"limit":20}' -b --login
```

**重要说明**：
- **命令行工具**：使用路由路径 `datatable/rest/v1/form/fields`（框架会自动处理）
- **后端 JavaScript**：使用 `window.api('datatable/form/fields')`（`api_host` 已包含 `/api/rest/v1/` 前缀）
- **直接 HTTP 请求**：使用完整 URL `/{admin_router}/api/rest/v1/datatable/form/fields`

**注意**：
- **所有 DataTable 模块的 API 都是后端 API**，继承自 `BackendRestController`，**都需要** `-b --login` 参数
- `Form`、`DataTable` 和 `Trash` 控制器都继承自 `BackendRestController`

#### 方式2：使用浏览器或 HTTP 客户端

**后端 API（Form 控制器）**：
```
POST http://127.0.0.1:9981/{admin_router}/api/rest/v1/datatable/form/fields
Content-Type: application/json
Cookie: {session_cookie}

{
  "model": "Weline\\DataTable\\Model\\TestUser",
  "scope": "form"
}
```

**后端 API（DataTable 控制器）**：
```
POST http://127.0.0.1:9981/{admin_router}/api/rest/v1/datatable/data-table/data
Content-Type: application/json
Cookie: {session_cookie}

{
  "model": "Weline\\DataTable\\Model\\TestUser",
  "scope": "table",
  "page": 1,
  "limit": 20
}
```

### API 常见问题

#### 1. 404 错误
**原因**：路由未注册

**解决方法**：
```bash
php bin/w setup:upgrade -m Weline_DataTable
```

#### 2. 403 错误（后端 API）
**原因**：未登录或未使用正确的后端路由

**解决方法**：
- 确保已登录后台
- 使用 `-b --login` 参数（命令行）
- 或使用正确的后端 URL 格式（浏览器/HTTP 客户端）

#### 3. 路由格式错误
**错误示例**：
```
❌ /rest/v1/datatable/form/fields  # 缺少模块路由前缀
❌ /datatable/api/v1/form/fields   # 使用了错误的路径
```

**正确格式**：
```
✅ 路由路径：datatable/rest/v1/form/fields（用于命令行工具）
✅ 后端 URL：/{admin_router}/api/rest/v1/datatable/form/fields（用于 HTTP 请求）
✅ JavaScript：window.api('datatable/form/fields')（后端页面）
```

### 查看已注册的路由

```bash
php bin/w route:list | Select-String -Pattern "datatable.*rest"
```

## 事件系统

### 可用事件
- `datatable:loaded` - 表格加载完成
- `datatable:refreshed` - 表格刷新完成
- `datatable:row-selected` - 行选择
- `datatable:row-clicked` - 行点击
- `datatable:form-submitted` - 表单提交
- `datatable:data-saved` - 数据保存
- `datatable:data-deleted` - 数据删除

### 事件监听示例

```javascript
// 监听表格加载完成事件
document.addEventListener('datatable:loaded', function(event) {
    console.log('表格加载完成:', event.detail);
});

// 监听行选择事件
document.addEventListener('datatable:row-selected', function(event) {
    console.log('选中行:', event.detail);
});

// 监听表单提交事件
document.addEventListener('datatable:form-submitted', function(event) {
    console.log('表单提交:', event.detail);
});
```

## 自定义渲染

### 自定义单元格渲染

```phtml
<w:field belong="t-header" name="status" renderer="statusRenderer">状态</w:field>
```

```javascript
// 自定义状态渲染器
window.statusRenderer = function(value, row, index) {
    if (value == 1) {
        return '<span class="badge badge-success">启用</span>';
    } else {
        return '<span class="badge badge-danger">禁用</span>';
    }
};
```

### 自定义操作按钮

```phtml
<w:field belong="t-header" name="actions" renderer="actionRenderer" width="150">操作</w:field>
```

```javascript
// 自定义操作按钮渲染器
window.actionRenderer = function(value, row, index) {
    return `
        <button class="btn btn-sm btn-primary" onclick="editRow(${row.id})">编辑</button>
        <button class="btn btn-sm btn-danger" onclick="deleteRow(${row.id})">删除</button>
    `;
};
```

## 常见问题

### Q: 如何隐藏某些字段？
A: 在 `t-header` 中只包含需要显示的字段，或者使用 `display="false"` 属性。

A: 在模型中重写 `getOrderBy` 方法，或者使用自定义的排序字段。
