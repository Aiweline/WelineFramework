# DataTable 模块故障排查文档

## 版本信息
- **文档版本**: 2.0.0
- **最后更新**: 2024年12月

## 一、常见问题

### 1.1 标签不显示

**问题**: DataTable 标签渲染为空或不显示

**可能原因**:
1. 不在后端页面使用
2. 缺少必需属性（model、scope）
3. 模型类不存在

**解决方法**:
1. 确保在后端页面使用标签
2. 检查是否提供了 `model` 和 `scope` 属性
3. 验证模型类是否存在且可访问

**示例**:
```phtml
<!-- 正确用法 -->
<w:d-table model="Weline\DataTable\Model\TestUser" scope="user-list">
</w:d-table>
```

### 1.2 API 返回 404

**问题**: API 请求返回 404 错误

**可能原因**:
1. 路由未注册
2. 路由路径错误
3. 模块未安装

**解决方法**:
```bash
# 重新安装模块
php bin/w setup:upgrade -m Weline_DataTable

# 查看已注册的路由
php bin/w route:list | grep datatable
```

### 1.3 API 返回 403

**问题**: API 请求返回 403 错误

**可能原因**:
1. 未登录后端
2. 权限不足

**解决方法**:
1. 确保已登录后台
2. 检查用户权限配置
3. 使用正确的后端路由访问

### 1.4 数据查询失败

**问题**: 数据查询返回错误

**可能原因**:
1. 模型类不存在
2. 数据库连接失败
3. SQL 语法错误

**解决方法**:
1. 检查模型类是否正确
2. 检查数据库配置
3. 查看错误日志获取详细信息

### 1.5 文件上传失败

**问题**: 文件上传返回错误

**可能原因**:
1. 文件大小超过限制
2. 文件类型不支持
3. 上传目录权限不足

**解决方法**:
1. 检查 PHP 配置（upload_max_filesize, post_max_size）
2. 检查允许的文件类型
3. 检查上传目录的写入权限

### 1.6 导入功能异常

**问题**: 数据导入失败

**可能原因**:
1. PhpSpreadsheet 库未安装
2. 文件格式不支持
3. 字段映射错误

**解决方法**:
```bash
# 安装 PhpSpreadsheet
composer require phpoffice/phpspreadsheet

# 检查文件格式
# 支持: .xlsx, .xls, .csv, .json

# 检查字段映射配置
```

## 二、错误日志

### 2.1 日志位置

错误日志通常记录在以下位置：
- PHP 错误日志（php.ini 配置的 error_log）
- 框架日志文件
- 浏览器控制台（前端错误）

### 2.2 日志格式

```
[DataTable Exception] Context - Code: 1001, Message: 模型类不存在, File: /path/to/file.php, Line: 123
```

### 2.3 查看日志

```bash
# 查看 PHP 错误日志
tail -f /var/log/php_errors.log | grep DataTable

# 查看框架日志
tail -f var/log/system.log | grep DataTable
```

## 三、调试技巧

### 3.1 启用调试模式

在开发环境中启用调试模式，可以获取更详细的错误信息：

```php
define('DEV', true);
```

### 3.2 使用浏览器开发者工具

1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签页的错误信息
3. 查看 Network 标签页的 API 请求和响应

### 3.3 使用 Postman 测试 API

使用 Postman 或其他 API 测试工具直接测试 API 接口：

```
POST http://localhost/api/rest/v1/datatable/data-table/data
Content-Type: application/json
Cookie: session_cookie

{
    "model": "Weline\\DataTable\\Model\\TestUser",
    "scope": "test"
}
```

## 四、性能问题

### 4.1 查询慢

**问题**: 数据查询速度慢

**解决方法**:
1. 添加数据库索引
2. 优化查询条件
3. 使用分页查询
4. 启用查询缓存

### 4.2 渲染慢

**问题**: 表格渲染速度慢

**解决方法**:
1. 减少显示字段数量
2. 使用虚拟滚动（大数据量）
3. 优化前端代码
4. 启用浏览器缓存

### 4.3 内存占用高

**问题**: 内存占用过高

**解决方法**:
1. 使用流式导出（大数据量）
2. 分批处理数据
3. 优化查询，只获取需要的字段
4. 增加 PHP 内存限制（临时方案）
