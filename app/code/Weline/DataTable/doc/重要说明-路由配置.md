# ⚠️ 重要说明：Weline Framework 路由配置

## ❌ 禁止使用 routes.xml

**重要提示**：Weline Framework **禁止创建和使用 `routes.xml` 文件**。

### 错误做法 ❌

```xml
<!-- ❌ 错误：不要创建 routes.xml 文件 -->
<!-- 这是 Magento 框架的方式，不是 Weline Framework 的方式 -->
<routes>
    <route path="/test" method="GET">
        <controller>Test::index</controller>
    </route>
</routes>
```

### 正确做法 ✅

Weline Framework 使用**自动路由注册**机制：

1. **控制器位置决定路由**：
   ```
   Controller/Frontend/Test/Index.php → datatable/frontend/test/index
   Controller/Backend/Test/Index.php  → datatable/backend/test/index
   ```

2. **路由注册命令**：
   ```bash
   php bin/w setup:upgrade
   # 或指定模块
   php bin/w setup:upgrade -m Weline_DataTable
   ```

3. **路由别名配置**（可选）：
   ```php
   // etc/env.php
   return [
       'router' => 'datatable'  // 设置路由别名
   ];
   ```

## 路由规则

### 前端控制器
- **位置**: `Controller/Frontend/{目录}/{控制器名}.php`
- **路由格式**: `{模块router}/frontend/{目录}/{控制器名}/{方法名}`
- **示例**: 
  - `Controller/Frontend/Test/Index.php` → `datatable/frontend/test/index`
  - `Controller/Frontend/User/Profile.php` → `datatable/frontend/user/profile`

### 后端控制器
- **位置**: `Controller/Backend/{目录}/{控制器名}.php`
- **路由格式**: `{模块router}/backend/{目录}/{控制器名}/{方法名}`
- **示例**: 
  - `Controller/Backend/Test/Index.php` → `datatable/backend/test/index`

### API控制器
- **位置**: `Api/Rest/V1/{控制器名}.php`（注意：是 `Api/` 目录，不是 `Controller/Api/`）
- **路由格式**: `{模块router}/rest/v1/{控制器名}/{方法名}`
- **示例**: 
  - `Api/Rest/V1/Form.php` → `datatable/rest/v1/form/{方法名}`
  - `Api/Rest/V1/DataTable.php` → `datatable/rest/v1/data-table/{方法名}`

#### API 控制器基类选择

- **FrontendRestController** - 前端 API（无需登录，无需 `-b` 参数）
  - 适用于：公开访问的 API，不需要登录验证
  
- **BackendRestController** - 后端 API（需要登录，需要 `-b --login` 参数）
  - 适用于：数据管理相关的 API，需要登录验证
  - 示例：`Form`、`DataTable`、`Trash` 控制器
  - 访问：`datatable/rest/v1/form/fields`（需登录）
  - 访问：`datatable/rest/v1/data-table/data`（需登录）

#### API 路由注册流程

1. **创建控制器文件**：`Api/Rest/V1/{控制器名}.php`
2. **继承正确的基类**：
   - `FrontendRestController` - 前端 API（无需登录）
   - `BackendRestController` - 后端 API（需要登录）
3. **运行注册命令**：
   ```bash
   php bin/w setup:upgrade -m Weline_DataTable
   ```
4. **验证路由**：
   ```bash
   php bin/w route:list | Select-String -Pattern "datatable"
   ```

#### API 路由访问示例

**注意**：DataTable 模块的所有 API 都是后端 API，需要登录验证。

**后端 API（Form 控制器）**：
```bash
# 命令行访问（需要 -b --login 参数）
php bin/w http:request -u "datatable/rest/v1/form/fields" -m POST -d '{"model":"Weline\\DataTable\\Model\\TestUser","scope":"form"}' -b --login
```

**后端 API（DataTable 控制器）**：
```bash
# 命令行访问（需要 -b --login 参数）
php bin/w http:request -u "datatable/rest/v1/data-table/data" -m POST -d '{"model":"Weline\\DataTable\\Model\\TestUser","scope":"table"}' -b --login
```

**后端 JavaScript 调用**：
```javascript
// 后端页面中，window.api() 不需要包含 rest/v1，因为 api_host 已包含 /api/rest/v1/ 前缀
window.api('datatable/form/fields')      // 生成：/{admin_router}/api/rest/v1/datatable/form/fields
window.api('datatable/data-table/data')  // 生成：/{admin_router}/api/rest/v1/datatable/data-table/data
```
