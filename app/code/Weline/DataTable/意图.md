# Weline DataTable 模组完整设计意图

## 项目概述
Weline DataTable 是一个功能强大、高度可配置的数据表格模组，专为企业级后台管理系统设计。支持单表和多表联合查询、自动/手动配置、完整的CRUD操作、事务管理、依赖关系处理、文件上传等企业级特性。

## 核心设计理念
- **开发效率优先**：自动生成与手动配置无缝切换，最大化开发效率
- **企业级特性**：支持多表关联、事务管理、依赖关系、级联操作
- **用户体验至上**：直观的界面、清晰的提示、流畅的交互
- **可扩展性**：模块化设计，便于功能扩展和定制

## 主要功能模块

### 1. 核心表格功能
#### 1.1 自动生成表格结构
- 只需传入model和scope参数，即可自动从模型获取字段，生成表头、过滤器、表体、表尾
- 支持多模型（JOIN）配置，自动为字段加别名前缀，避免冲突
- 智能过滤敏感字段（如password、token等），限制字段数量，避免表格过宽
- 自动推断字段类型和验证规则

#### 1.2 手动配置支持
- 支持手动配置t-header、t-filter、t-body、t-footer等子标签
- 精确控制显示字段、顺序、样式、过滤器等
- 手动配置时，自动补全缺失的必要子标签
- 支持模板字段与自动字段的混合使用

#### 1.3 丰富的属性支持
- 基础属性：editable、inline-edit、modal-edit、searchable、sortable
- 显示属性：page-size、show-pagination、show-toolbar、show-config、height、width
- 表单属性：form、form-mode、form-title
- 关联属性：join、dependencies、cascade-delete、transaction

### 2. 多表关联功能
#### 2.1 JOIN查询支持
- 支持多模型联合查询，JOIN条件灵活配置
- 自动生成select字段，处理字段别名冲突
- 支持LEFT JOIN、RIGHT JOIN、INNER JOIN等多种连接方式
- 智能处理关联表的字段显示和筛选

#### 2.2 依赖关系管理
- 支持表间依赖关系配置，默认使用主键ID作为依赖字段
- 允许自定义依赖字段，灵活处理复杂的业务关系
- 自动处理保存时的依赖顺序，确保数据完整性
- 支持一对一、一对多、多对多等关系类型

#### 2.3 多表关联保存
- 支持多表JOIN时的关联保存功能
- 自动处理表间的依赖关系和保存顺序
- 支持事务管理，确保数据一致性
- 智能处理外键关系和数据完整性约束

#### 2.4 表单分组展示
- 为JOIN查询的多表实现fieldset分组表单
- 使用表别名作为分组ID，提供清晰的数据录入界面
- 支持分组的展开/折叠功能
- 自动为每个表生成对应的字段组

### 3. 数据操作功能
#### 3.1 增删改查操作
- 完整的CRUD操作支持
- 支持单表和多表的数据操作
- 实时编辑功能，双击单元格即可编辑
- 批量操作支持（批量删除、批量更新等）

#### 3.2 删除操作增强
- 删除前检查关联表数据
- 显示关联的表和记录数量
- 提供级联删除选项，默认勾选但可取消
- 智能提醒用户可能的影响

#### 3.3 事务管理
- 为多表操作提供完整的事务支持
- 自动回滚机制，确保数据一致性
- 支持嵌套事务和保存点
- 详细的错误日志和异常处理

### 4. 表单系统
#### 4.1 智能表单生成
- 根据模型自动生成表单字段
- 支持多种字段类型和验证规则
- 自动推断字段标签和帮助文本
- 支持字段的显示/隐藏控制

#### 4.2 表单验证
- 前端实时验证
- 后端数据验证
- 自定义验证规则支持
- 友好的错误提示

#### 4.3 文件上传功能
- 支持图片字段类型
- 显示图片文件信息（格式、大小、尺寸）
- 图片预览功能
- 支持多种图片格式（JPG、PNG、GIF、WebP等）
- 文件大小限制和格式验证
- 人性化大小展示（KB、MB、GB）
- 集成 app\code\Weline\FileManager 模组

#### 4.4 表单模式
- 模态框表单
- 内联表单
- 侧边栏表单
- 全屏表单

### 5. 筛选和搜索
#### 5.1 高级筛选
- 多字段组合筛选
- 日期范围筛选
- 数值范围筛选
- 自定义筛选条件

#### 5.2 全文搜索
- 跨字段搜索
- 模糊匹配
- 高亮显示搜索结果
- 搜索历史记录

## 技术架构

### 1. 后端架构
#### 1.1 标签库系统
- **Table.php**：主表格标签，负责整体结构生成和上下文管理
- **TableHeader.php**：表头标签，处理字段显示和排序
- **TableFilter.php**：筛选器标签，处理各种筛选条件
- **TableBody.php**：表体标签，处理数据行显示
- **TableFooter.php**：表尾标签，处理分页和统计信息
- **Field.php**：字段标签，处理各种字段类型的渲染
- **Form.php**：表单标签，处理增删改表单

#### 1.2 助手类系统
- **TableContext.php**：上下文管理，处理标签间的数据传递
- **ModelHelper.php**：模型助手，处理模型字段获取和验证
- **JoinHelper.php**：JOIN助手，处理多表查询逻辑
- **DependencyManager.php**：依赖关系管理器
- **TransactionManager.php**：事务管理器
- **FileUploadHelper.php**：文件上传助手

#### 1.3 API接口层
- **DataTable.php**：REST API接口，处理前后端数据交互
- **MultiTableApi.php**：多表操作API
- **DependencyApi.php**：依赖关系API
- **FileUploadApi.php**：文件上传API
- 完整的错误处理和数据验证

### 2. 前端架构
#### 2.1 核心组件
- **datatable-manager.js**：表格管理器，处理前端交互逻辑
- **datatable-form-manager.js**：表单管理器，处理表单操作
- **dependency-manager.js**：依赖关系管理器
- **transaction-manager.js**：事务管理器
- **file-upload-manager.js**：文件上传管理器

#### 2.2 样式系统
- **datatable.css**：核心样式文件
- **themes/**：主题文件目录
- **responsive.css**：响应式样式
- **animations.css**：动画效果

## 使用示例

### 1. 基础用法
```phtml
<!-- 最简单的用法 -->
<w:d-table model="App\Model\User" scope="user-list"></w:d-table>
```

### 2. 多表关联
```phtml
<w:d-table model="User as u, Order as o"
           join="left u.id = o.user_id"
           dependencies="u.id->o.user_id"
           scope="user-orders">
</w:d-table>
```

### 3. 完整配置
```phtml
<w:d-table model="App\Model\Product"
           scope="product-management"
           editable="true"
           searchable="true"
           sortable="true"
           page-size="20"
           transaction="true"
           cascade-delete="true">

    <w:d-form for="add,edit" title="产品管理">
        <w:field name="name" type="text" label="产品名称" required="true"></w:field>
        <w:field name="price" type="number" label="价格" required="true"></w:field>
        <w:field name="image" type="image" label="产品图片" accept="image/*" max-size="2MB"></w:field>
    </w:d-form>

    <w:t-filter>
        <w:field belong="t-filter" name="name" type="text" placeholder="搜索产品名称"></w:field>
        <w:field belong="t-filter" name="status" type="select" options="1:上架,0:下架"></w:field>
    </w:t-filter>

    <w:t-header>
        <w:field belong="t-header" name="id" sortable="true" width="80">ID</w:field>
        <w:field belong="t-header" name="name" sortable="true" width="200">产品名称</w:field>
        <w:field belong="t-header" name="price" sortable="true" width="100">价格</w:field>
        <w:field belong="t-header" name="image" width="100">图片</w:field>
        <w:field belong="t-header" name="status" sortable="true" width="100">状态</w:field>
    </w:t-header>
</w:d-table>
```

### 4. 多表分组表单
```phtml
<w:d-table model="User as u, Profile as p, Address as a"
           join="left u.id = p.user_id, left u.id = a.user_id"
           dependencies="u.id->p.user_id,u.id->a.user_id"
           scope="user-complete-info">

    <w:d-form for="add,edit" title="用户完整信息">
        <!-- 用户基础信息分组 -->
        <fieldset id="u">
            <legend>用户基础信息</legend>
            <w:field name="u.username" type="text" label="用户名" required="true"></w:field>
            <w:field name="u.email" type="email" label="邮箱" required="true"></w:field>
            <w:field name="u.avatar" type="image" label="头像" accept="image/*" max-size="1MB"></w:field>
        </fieldset>

        <!-- 用户档案分组 -->
        <fieldset id="p">
            <legend>用户档案</legend>
            <w:field name="p.real_name" type="text" label="真实姓名"></w:field>
            <w:field name="p.phone" type="tel" label="电话"></w:field>
        </fieldset>

        <!-- 地址信息分组 -->
        <fieldset id="a">
            <legend>地址信息</legend>
            <w:field name="a.province" type="text" label="省份"></w:field>
            <w:field name="a.city" type="text" label="城市"></w:field>
            <w:field name="a.address" type="textarea" label="详细地址"></w:field>
        </fieldset>
    </w:d-form>
</w:d-table>
```

## 核心特性详解

### 1. 依赖关系处理
- **默认依赖**：表间默认使用主键ID作为依赖字段
- **自定义依赖**：通过`dependencies`属性指定依赖关系，格式：`主表.字段->从表.字段`
- **多重依赖**：支持一个主表对应多个从表的依赖关系
- **保存顺序**：自动计算保存顺序，确保依赖关系正确

### 2. 事务管理
- **自动事务**：多表操作自动开启事务
- **回滚机制**：任何一个表保存失败，自动回滚所有操作
- **嵌套事务**：支持复杂业务场景的嵌套事务
- **错误处理**：详细的错误信息和日志记录

### 3. 级联删除
- **关联检查**：删除前检查所有关联表的数据
- **用户提醒**：显示关联表和记录数量，让用户明确删除影响
- **级联选项**：提供级联删除选项，默认勾选但可取消
- **安全机制**：重要数据的删除需要二次确认

### 4. 表单分组
- **自动分组**：根据表别名自动生成fieldset分组
- **分组标题**：使用表名或自定义标题作为分组标题
- **展开折叠**：支持分组的展开和折叠功能
- **数据提取**：后台API根据分组ID提取对应表的数据

### 5. 图片字段功能
- **文件信息显示**：显示图片格式、大小、尺寸等信息
- **预览功能**：支持图片预览和缩略图显示
- **格式验证**：支持多种图片格式验证
- **大小限制**：可配置文件大小限制
- **人性化提示**：友好的文件上传提示和进度显示
- **FileManager集成**：与Weline\FileManager模组深度集成

## 设计亮点

- **用户体验**：直观的界面、清晰的提示、流畅的交互
- **可扩展性**：模块化设计，便于功能扩展和定制
- **数据安全**：完整的事务支持和错误处理机制
- **文件处理**：完善的文件上传和图片处理功能

添加Model用于支持软删除，回收站等功能，删除数据后可以后悔。然后添加一个定时任务，删除数据后，可以在180天内恢复，180天后自动删除。

数据导出使用流式导出，按照页导出，展示导出进度条，并且显示导出进度页信息。避免一次性大量导出占用服务器资源
