# DataTable 模块开发意图

## 模块定位

DataTable 模块是 WelineFramework 框架的核心数据展示和管理组件，旨在为开发者提供一套完整、易用、高度可配置的数据表格和表单解决方案。通过标签化的方式，实现数据表格的快速构建、数据管理、表单操作等功能，支持单表和多表关联查询，提供完整的CRUD操作能力。

## 核心功能

### 1. 数据表格展示和管理
- **数据列表展示**：支持分页、排序、筛选的数据表格展示
- **实时数据加载**：通过AJAX异步加载数据，支持大数据量处理
- **字段配置**：支持用户自定义显示字段和筛选字段，配置持久化保存
- **工具栏功能**：提供刷新、导出、字段配置、主题切换等工具
- **统计信息**：显示总记录数和当前显示记录数

### 2. 多模型和JOIN查询支持
- **多模型配置**：支持在 `model` 属性中配置多个模型，格式：`Model1 as alias1, Model2 as alias2`
- **JOIN查询**：支持 LEFT JOIN、RIGHT JOIN、INNER JOIN、OUTER JOIN 等关联查询
- **字段别名**：多模型查询时，自动为字段添加表别名前缀，避免字段冲突
- **关联数据展示**：在表格中展示来自多个关联表的数据
- **关联数据保存**：支持多表数据的关联保存，自动处理外键关系

### 3. 文件上传功能
- **文件上传支持**：支持文件类型字段的上传功能
- **图片上传**：专门的图片上传字段，支持图片预览、裁剪、删除等操作
- **文件类型限制**：可配置允许上传的文件类型（如：jpg, png, pdf, doc等）
- **文件大小限制**：可配置最大文件大小限制
- **上传进度显示**：显示文件上传进度条
- **文件信息获取**：获取上传文件的详细信息（大小、类型、尺寸等）
- **文件删除**：支持删除已上传的文件

### 4. 图片字段可视化
- **图片预览**：图片字段在表格中直接显示缩略图
- **图片上传界面**：提供友好的图片上传界面，支持拖拽上传
- **图片预览弹窗**：点击图片可查看大图
- **图片信息展示**：显示图片尺寸、大小、格式等信息
- **图片操作**：支持更换、删除图片等操作
- **多图片支持**：支持一次上传多张图片

### 5. 自动生成功能
- **自动生成表格结构**：当 `d-table` 标签内容为空时，自动从模型获取字段信息生成表格结构
- **自动生成表头**：自动识别模型字段，生成表头列
- **自动生成过滤器**：自动生成前3个主要字段的过滤器
- **智能字段过滤**：自动过滤敏感字段（password、token、secret等）
- **字段数量限制**：自动限制显示字段数量（最多8个），避免表格过宽
- **字段类型推断**：根据字段名自动推断字段类型（如：email、date、status等）
- **自动生成表单字段**：表单中未手动配置的字段，自动从模型获取并生成

### 6. 完整的CRUD操作
- **创建（Create）**：支持单表和多表数据的创建，支持事务处理
- **读取（Read）**：支持分页查询、条件筛选、排序、多表JOIN查询
- **更新（Update）**：支持单条和批量更新，支持行内编辑和弹窗编辑
- **删除（Delete）**：支持软删除和硬删除，支持级联删除，支持批量删除
- **关联数据检查**：删除前检查关联数据，防止误删
- **回收站功能**：支持软删除数据的回收站管理，可恢复和永久删除

### 7. 丰富的字段类型支持
- **基础类型**：text、email、password、number、tel、url、search、textarea
- **选择类型**：select、checkbox、radio
- **日期时间类型**：date、datetime、time、month、week
- **文件类型**：file、image
- **其他类型**：hidden、color、range
- **字段类型自动推断**：根据字段名自动推断合适的字段类型

### 8. 表单管理功能
- **表单模式**：支持模态框（modal）和内联（inline）两种表单模式
- **表单布局**：支持垂直（vertical）和水平（horizontal）两种布局
- **字段自动生成**：支持从模型自动生成表单字段
- **字段排除和包含**：支持排除或只包含指定字段
- **表单验证**：支持前端和后端双重验证
- **多表分组**：多模型表单时，自动按表分组显示字段
- **字段配置保存**：支持用户自定义表单字段配置（显示顺序、是否显示、字段类型等），配置持久化保存
- **配置加载优先级**：用户保存的配置 > 手动配置的字段 > 自动生成的字段

## 标签系统工作原理

### 标签层次结构

```
d-table (表格容器)
├── t-header (表头)
│   └── field (字段定义)
├── t-filter (过滤器)
│   └── field (字段定义)
├── t-body (表体，自动生成)
├── t-footer (表尾，自动生成)
└── d-form (表单，可选)
    └── field (字段定义)
```

### 标签功能说明

#### 1. d-table 标签
**功能**：数据表格的主容器，负责初始化表格、管理上下文、生成表格HTML结构。

**工作原理**：
- 接收 `model` 和 `scope` 必需属性
- 解析多模型配置（如果 `model` 包含多个模型）
- 解析JOIN配置（如果提供了 `join` 属性）
- 创建表格上下文（TableContext），供子标签继承使用
- 检查是否有手动配置的子标签内容
- 如果没有手动配置，自动生成默认的表格结构（t-header、t-filter、t-body、t-footer）
- 生成表格HTML、JavaScript初始化代码、CSS样式引用
- 如果启用了表单（`form="true"`），生成表单HTML和触发按钮

**关键属性**：
- `model`：必需，模型类名，支持多模型格式
- `scope`：必需，数据作用域，用于存储用户配置
- `join`：可选，JOIN查询配置
- `editable`：是否启用编辑功能
- `searchable`：是否启用搜索功能
- `sortable`：是否启用排序功能
- `page-size`：每页显示数量
- `form`：是否启用表单功能

#### 2. t-header 标签
**功能**：定义表格的表头列，控制哪些字段显示在表格中。

**工作原理**：
- 从父标签（d-table）继承 `model` 和 `scope` 属性
- 解析内部的 `field` 标签，获取字段配置
- 生成表头HTML（`<thead>` 和 `<tr>`）
- 记录字段配置到上下文，供前端JavaScript使用
- 支持字段排序、宽度、可见性等配置

#### 3. t-filter 标签
**功能**：定义表格的过滤器，控制哪些字段可以用于筛选数据。

**工作原理**：
- 从父标签（d-table）继承 `model` 和 `scope` 属性
- 解析内部的 `field` 标签，获取过滤器配置
- 生成过滤器HTML（搜索框、下拉选择等）
- 根据字段类型生成对应的过滤器控件（text、select、date等）
- 支持字段的搜索类型配置

#### 4. field 标签
**功能**：定义字段的配置，可以在表头、过滤器、表单中使用。

**工作原理**：
- 必须指定 `belong` 属性，表示属于哪个父标签（t-header、t-filter、d-form）
- 从父标签上下文获取 `model` 和 `scope` 信息
- 根据 `belong` 属性的不同，生成不同的HTML：
  - `belong="t-header"`：生成表头列（`<th>`）
  - `belong="t-filter"`：生成过滤器控件（input、select等）
  - `belong="d-form"`：生成表单字段（input、textarea、select等）
- 支持字段类型、验证规则、选项等配置
- 记录字段配置到上下文，供前端使用

**字段类型处理**：
- 根据 `type` 属性生成对应的HTML控件
- 如果未指定 `type`，根据字段名自动推断类型
- 特殊类型（如 `image`、`file`）会生成专门的上传控件

#### 5. d-form 标签
**功能**：定义表单，用于新增和编辑数据。

**工作原理**：
- 必须指定 `model` 属性，或从父标签（d-table）继承
- 支持两种模式：模态框（modal）和内联（inline）
- 解析内部的 `field` 标签，生成表单字段
- 如果启用了 `auto_fields`，自动从模型获取未配置的字段并生成
- **字段配置加载**：优先加载用户保存的字段配置，如果没有则使用手动配置或自动生成
- 生成表单HTML、验证规则、提交处理代码
- 支持多表分组显示（当 `model` 包含多个模型时）

**字段配置机制**：
- **配置优先级**：
  1. 用户保存的字段配置（最高优先级）
  2. 模板中手动配置的字段（`<w:field>` 标签）
  3. 自动生成的字段（从模型获取）
- **配置保存**：用户可以通过字段配置界面自定义字段的显示顺序、是否显示、字段类型、验证规则等，配置会保存到缓存中
- **配置作用域**：配置基于 `scope` 和 `form_id` 保存，不同表单的配置相互独立
- **配置加载**：表单初始化时，自动从缓存加载用户保存的配置，应用到字段生成中

### 上下文继承机制

**TableContext 上下文管理器**负责管理标签间的属性继承和数据共享：

1. **上下文创建**：`d-table` 标签创建时，将配置信息存储到 TableContext
2. **上下文传递**：子标签（t-header、t-filter、d-form）可以从 TableContext 获取父标签的配置
3. **属性继承**：子标签可以继承父标签的 `model`、`scope`、`join` 等属性
4. **属性覆盖**：子标签显式指定的属性会覆盖继承的属性
5. **上下文栈管理**：支持多层嵌套，使用栈结构管理多个上下文

**继承优先级**：
1. 子标签显式指定的属性（最高优先级）
2. 从父标签上下文继承的属性
3. 默认值（最低优先级）

## 多模型和JOIN查询支持

### 多模型配置

**语法格式**：
```html
<w:d-table 
    model="Model1 as alias1, Model2 as alias2, Model3 as alias3" 
    join="left alias1.field1 = alias2.field2, inner alias2.field3 = alias3.field4"
    scope="multi-model-scope">
</w:d-table>
```

**工作原理**：
1. **模型解析**：解析 `model` 属性，提取模型类和别名
2. **JOIN解析**：解析 `join` 属性，提取JOIN类型和条件
3. **字段前缀**：为每个模型的字段添加别名前缀（如：`alias1.field_name`）
4. **查询构建**：构建多表JOIN查询SQL
5. **数据返回**：返回包含所有表字段的数据，字段名带别名前缀

**字段引用**：
- 在 `field` 标签中，使用 `alias.field_name` 格式引用字段
- 例如：`<w:field belong="t-header" name="admin.username">用户名</w:field>`

### JOIN类型支持

- **LEFT JOIN**：左连接，保留左表所有记录
- **RIGHT JOIN**：右连接，保留右表所有记录
- **INNER JOIN**：内连接，只返回匹配的记录
- **OUTER JOIN**：外连接，返回所有记录

### 多表数据保存

**依赖关系管理**：
- 自动识别表之间的外键关系
- 按照依赖关系顺序保存数据
- 支持事务处理，确保数据一致性

**保存顺序计算**：
- 分析表之间的依赖关系
- 计算保存顺序（拓扑排序）
- 按顺序保存每个表的数据
- 自动处理外键值的传递

## 文件上传功能

### 文件上传API

**接口**：`/datatable/api/upload-image` (POST)

**功能**：
- 验证文件类型和大小
- 保存文件到指定目录
- 返回文件信息（路径、大小、类型等）
- 支持图片文件的详细信息获取（尺寸、EXIF等）

**参数**：
- `field_name`：字段名
- `max_size`：最大文件大小（如：2MB）
- `allowed_types`：允许的文件类型（如：jpg,jpeg,png,gif）
- `upload_path`：上传路径

**返回数据**：
```json
{
    "file_name": "unique_filename.jpg",
    "original_name": "original.jpg",
    "file_path": "/path/to/file.jpg",
    "web_path": "/uploads/file.jpg",
    "file_size": 102400,
    "file_size_formatted": "100 KB",
    "mime_type": "image/jpeg",
    "width": 1920,
    "height": 1080
}
```

### 图片字段可视化

**前端实现**：
- 图片上传控件：支持点击选择和拖拽上传
- 图片预览：上传后立即显示预览图
- 图片信息展示：显示图片尺寸、大小、格式
- 图片操作：支持更换、删除图片
- 多图片支持：支持一次上传多张图片，以网格形式展示

**表格中的图片显示**：
- 图片字段在表格中显示为缩略图
- 点击缩略图可查看大图
- 支持图片懒加载，优化性能

## 自动生成功能详解

### 表格自动生成

**触发条件**：`d-table` 标签内容为空时自动生成

**生成流程**：
1. 实例化模型，获取字段信息
2. 过滤敏感字段（password、token、secret等）
3. 限制字段数量（最多8个）
4. 生成表头字段（前8个字段）
5. 生成过滤器字段（前3个主要字段）
6. 自动生成 t-header、t-filter、t-body、t-footer 标签

**字段选择策略**：
- 优先选择常用字段（id、name、title、status等）
- 排除系统字段（created_at、updated_at等，除非是主要字段）
- 根据字段类型设置合适的宽度和过滤器类型

### 表单自动生成

**触发条件**：`d-form` 标签中 `auto_fields="true"`（默认值）

**生成流程**：
1. **加载用户配置**：首先尝试从缓存加载用户保存的字段配置
2. **应用用户配置**：如果存在用户配置，使用用户配置的字段顺序、显示状态、字段类型等
3. **合并手动配置**：将模板中手动配置的字段（`<w:field>` 标签）合并到配置中
4. **补充自动字段**：如果用户配置和手动配置中都没有的字段，且未在 `exclude_fields` 中，则自动生成
5. **字段过滤**：排除指定排除的字段（`exclude_fields`）
6. **字段包含**：如果指定了包含字段（`include_fields`），只生成包含的字段
7. **生成表单控件**：根据字段类型生成对应的表单控件
8. **设置验证规则**：应用字段验证规则和默认值

**字段生成规则**：
- 主键字段：只读，不显示在新增表单中
- 时间字段：自动设置默认值为当前时间
- 状态字段：自动生成选项（启用/禁用）
- 外键字段：自动生成关联表的选择器

**字段配置保存**：
- 用户可以通过表单字段配置界面自定义字段
- 配置包括：字段显示顺序、是否显示、字段类型、标签文本、占位符、验证规则、选项等
- 配置保存到缓存，键名格式：`datatable_form_fields_{scope}_{form_id}`
- 配置缓存时间：1小时（可配置）
- 下次加载表单时，自动应用保存的配置

## CRUD操作详解

### 创建（Create）

**单表创建**：
- 验证数据格式和必填字段
- 设置默认值（如创建时间）
- 保存到数据库
- 返回新创建记录的ID

**多表创建**：
- 解析模型配置和依赖关系
- 计算保存顺序
- 按顺序保存每个表的数据
- 处理外键值的传递
- 使用事务确保数据一致性

### 读取（Read）

**查询功能**：
- 支持分页查询（page、limit）
- 支持条件筛选（filters）
- 支持排序（sort）
- 支持多表JOIN查询
- 支持字段选择（只查询需要的字段）

**数据返回**：
```json
{
    "data": [...],
    "total": 100,
    "page": 1,
    "limit": 20,
    "pages": 5,
    "has_next": true,
    "has_prev": false
}
```

### 更新（Update）

**更新方式**：
- **行内编辑**：直接在表格中编辑单元格
- **弹窗编辑**：点击编辑按钮，在弹窗中编辑
- **批量更新**：选择多条记录，批量更新

**更新流程**：
1. 加载原数据
2. 验证新数据
3. 更新数据库
4. 返回更新后的数据

### 删除（Delete）

**删除类型**：
- **软删除**：标记为已删除，数据仍保留在数据库中
- **硬删除**：从数据库中永久删除
- **级联删除**：删除主记录时，同时删除关联记录

**删除前检查**：
- 检查是否有关联数据
- 显示关联数据信息
- 确认后执行删除

**批量删除**：

## 核心设计理念

### 1. 标签化开发
通过自定义标签（Taglib）系统，将复杂的数据表格和表单功能封装为简单的标签，开发者只需关注业务逻辑，无需关心底层实现。

### 2. 模型驱动
以数据模型为核心，自动识别模型字段类型、验证规则等信息，自动生成对应的表格列和表单字段。

### 3. 上下文管理
通过 TableContext 上下文管理器，实现标签间的属性继承和数据共享，简化配置复杂度。

**上下文继承机制**：
- 子标签可以自动继承父标签的配置（如 `model`、`scope` 等）
- 支持多层嵌套，子标签可以访问所有父级标签的上下文
- 如果子标签显式指定了属性，则优先使用子标签的属性值
- 如果子标签未指定必需属性，会尝试从父级上下文获取，获取失败则报错

### 4. 渐进式增强
支持从最简单的自动生成到完全自定义的渐进式开发方式，满足不同复杂度的需求。

#### 标签使用规则

**d-form 标签的 model 属性规则**：
- **必需属性**：`d-form` 标签必须指定 `model` 属性，或者确保在 `d-table` 标签内使用
- **获取优先级**：
  1. 优先从标签属性获取：`<w:d-form model="WeShop\Store\Model\Store">`
  2. 如果属性中没有，尝试从父级 `d-table` 标签的上下文获取
  3. 如果都没有，标签将不会渲染（开发环境会显示错误注释，生产环境直接返回空）
- **使用场景**：
  - **独立使用**：必须在标签上显式指定 `model` 属性
    ```html
    <w:d-form model="WeShop\Store\Model\Store">
        <!-- 表单内容 -->
    </w:d-form>
    ```
  - **嵌套使用**：可以继承父级 `d-table` 的 `model` 属性
    ```html
    <w:d-table model="WeShop\Store\Model\Store">
        <w:d-form>
            <!-- 表单内容，会自动继承 table 的 model -->
        </w:d-form>
    </w:d-table>
    ```
- **错误处理**：
  - 如果 `model` 属性缺失且无法从上下文获取，标签会直接报错，不会渲染任何内容
  - 开发环境会在HTML注释中显示详细错误信息
  - 生产环境直接返回空字符串，不渲染标签

### 2. 核心组件
- **TableContext**: 上下文管理器，处理属性继承
- **Taglib类**: 标签处理类，负责标签解析和HTML生成
- **Helper类**: 辅助功能类，提供字段处理、依赖管理等

### 3. 前端集成
- JavaScript管理器：处理表格交互、表单提交等
- CSS样式：提供现代化的UI样式
- 响应式布局：适配不同设备

## 标签验证规则

### d-form 标签验证

**必需属性验证**：
- `model` 属性是 `d-form` 标签的必需属性
- 验证流程：
  1. 检查标签属性中是否指定了 `model`
  2. 如果未指定，检查是否在 `d-table` 标签内，尝试从表格上下文获取
  3. 如果仍然没有，直接返回错误，不渲染标签

**错误处理策略**：
- **开发环境**：在HTML注释中显示详细错误信息，方便调试
- **生产环境**：直接返回空字符串，不渲染标签，避免页面显示错误信息

**示例**：
```html
<!-- 正确：独立使用，显式指定 model -->
<w:d-form model="WeShop\Store\Model\Store">
    <!-- 表单内容 -->
</w:d-form>

<!-- 正确：嵌套使用，继承 table 的 model -->
<w:d-table model="WeShop\Store\Model\Store">
    <w:d-form>
        <!-- 表单内容 -->
    </w:d-form>
</w:d-table>

<!-- 错误：没有指定 model 且不在 table 内，不会渲染 -->
<w:d-form>
    <!-- 这不会渲染，会显示错误信息 -->
</w:d-form>
```

## 安全机制

### 1. 权限控制
- **后端API限制**：所有API控制器继承自 `BackendRestController`，必须登录后台才能访问
- **标签使用限制**：所有DataTable标签只能在后端页面使用，前端请求会自动返回空
- **请求验证**：通过 `isBackend()` 和 `isApiBackend()` 方法验证请求来源

### 2. SQL注入防护
- **参数化查询**：所有数据库查询使用参数化查询，防止SQL注入
- **字段白名单**：字段名验证，只允许模型中的有效字段
- **JOIN条件验证**：JOIN条件解析和验证，防止恶意SQL注入

### 3. XSS防护
- **输出转义**：所有用户输入在输出时进行HTML转义
- **内容过滤**：过滤危险HTML标签和脚本
- **CSP支持**：支持内容安全策略（CSP）

### 4. CSRF防护
- **框架级防护**：依赖WelineFramework框架的CSRF防护机制
- **Token验证**：表单提交时验证CSRF Token

### 5. 文件上传安全
- **文件类型验证**：严格验证文件MIME类型和扩展名
- **文件大小限制**：可配置最大文件大小，防止DoS攻击
- **路径安全**：文件保存路径验证，防止路径遍历攻击
- **文件名安全**：自动生成唯一文件名，防止文件名冲突和注入
- **图片验证**：图片文件使用 `getimagesize()` 验证，确保是真实图片

## 潜在隐患和注意事项

### 1. 多模型配置隐患
- **问题**：多模型配置但未指定JOIN条件，可能导致查询错误
- **处理**：系统会记录警告日志，但不会阻止执行
- **建议**：使用多模型时，务必配置正确的JOIN条件

### 2. 字段获取失败
- **问题**：模型字段获取失败时，可能使用默认字段，导致显示不准确
- **处理**：记录错误日志，使用降级字段列表
- **建议**：确保模型类正确实现字段获取方法

### 3. 上下文丢失
- **问题**：标签解析顺序问题可能导致上下文丢失
- **处理**：使用占位符机制，等待上下文就绪
- **建议**：确保标签嵌套结构正确

### 4. 模型类不存在
- **问题**：模型类不存在时，标签无法正常工作
- **处理**：抛出异常，开发环境显示错误，生产环境返回空
- **建议**：使用前确保模型类存在且可访问

### 5. 文件上传路径安全
- **问题**：文件上传路径配置不当可能导致安全风险
- **处理**：路径验证，防止路径遍历
- **建议**：使用相对路径，限制在指定目录内

### 6. 大数据量导出
- **问题**：大数据量导出可能导致内存溢出
- **处理**：使用流式导出，分批处理
- **建议**：大数据量导出时使用流式导出功能

### 7. 事务嵌套深度
- **问题**：事务嵌套过深可能导致性能问题
- **处理**：使用保存点实现嵌套事务
- **建议**：避免过深的事务嵌套

### 8. 循环依赖
- **问题**：表之间的循环依赖会导致保存失败
- **处理**：依赖管理器会检测并报告循环依赖
- **建议**：设计数据库时避免循环依赖

### 9. 前端请求限制
- **问题**：DataTable标签只能在后端使用，前端请求会返回空
- **处理**：自动检测请求类型，前端请求返回空或注释
- **建议**：确保只在后端页面使用DataTable标签

### 10. 缓存失效
- **问题**：字段配置缓存可能导致配置更新不及时
- **处理**：缓存1小时，支持手动清理
- **建议**：配置更新后手动清理缓存
- **表格和表单配置**：表格字段配置和表单字段配置分别缓存，互不影响

### 11. 标签解析顺序
- **问题**：标签解析顺序可能导致上下文获取失败
- **处理**：使用上下文栈和占位符机制
- **建议**：按照正确的标签嵌套顺序使用

### 12. 敏感字段过滤
- **问题**：自动生成时可能遗漏某些敏感字段
- **处理**：过滤常见敏感字段（password、token等）
- **建议**：手动配置时注意排除敏感字段
