# Deploy 模块使用说明

## 概述

Weline_Deploy 模块提供两套部署方式：

- **`deploy:build`** — 轻量 Git 拉取，仅执行 `git pull`
- **`deploy:release`** — 完整发布流水线：Git 更新 + 后置命令 + 版本戳 + reload

## 快速开始

### 1. 注册模块

首次使用需要更新命令列表：

```bash
php bin/w command:upgrade
```

### 2. 后台配置

进入 `系统管理 > 系统维护 > 部署配置`，填写：

- **项目仓库**：Git 仓库地址、分支/Tag、用户名、Token
- **部署行为**：`pull --ff-only` 或 `reset --hard`；`after_deploy_command` 后置命令
- **Webhook 密钥**（`webhook_secret`）：与 Git 平台 Secret/密码一致；推荐 `php bin/w deploy:webhook:setup` 生成或刷新
- **触发模式**：仅分支 Push / 仅 Tag Push / 都生效；分支过滤、Tag 前缀过滤
- **发布探测 Token**：用于 `GET /deploy/version` 获取详细版本信息

详细说明见 [`doc/backend-config.md`](doc/backend-config.md)。

### 3. 配置 Webhook（可选）

**访问密码**（首次或轮换）：

```bash
# 生成并写入后台
php bin/w deploy:webhook:setup --base-url=https://你的域名

# 刷新访问密码（须同步更新 Git 平台）
php bin/w deploy:webhook:setup --force -y --url=https://你的域名/deploy
```

在 Git 平台添加 Webhook：URL `https://你的域名/deploy`，Secret/密码与后台「Webhook 密钥」相同。

- 密码说明 → [`doc/webhook-secret.md`](doc/webhook-secret.md)
- **Gitee** → [`doc/gitee-webhook.md`](doc/gitee-webhook.md)
- **GitHub** → [`doc/github-webhook.md`](doc/github-webhook.md)

### 4. 执行部署

```bash
# 完整发布（推荐）
php bin/w deploy:release

# 仅 Git 拉取
php bin/w deploy:build
```

## 命令参考

### deploy:build（轻量 Git 拉取）

| 参数 | 说明 | 示例 |
|------|------|------|
| `-b, --branch` | 指定 Git 分支 | `php bin/w deploy:build -b develop` |
| `-f, --force` | 强制拉取（丢弃本地修改） | `php bin/w deploy:build --force` |
| `--no-backup` | 禁用部署前备份 | `php bin/w deploy:build --no-backup` |
| `-h, --help` | 显示帮助信息 | `php bin/w deploy:build --help` |

### deploy:release（完整发布）

| 参数 | 说明 | 示例 |
|------|------|------|
| `-r, --ref` | Git ref（分支或 tag） | `php bin/w deploy:release -r refs/tags/v1.0.0` |
| `--project` | 多项目 key | `php bin/w deploy:release --project=shop` |
| `--no-backup` | 禁用部署前备份 | `php bin/w deploy:release --no-backup` |
| `--env-id` | 覆盖 release_id | `php bin/w deploy:release --env-id=R_xxx` |
| `--worker-build-id` | 覆盖 worker_build_id | `php bin/w deploy:release --worker-build-id=W_xxx` |
| `--skip-git` | 跳过 Git 步骤 | `php bin/w deploy:release --skip-git` |

### deploy:release:status（查看版本）

```bash
php bin/w deploy:release:status
```

输出示例：
```
当前发布版本: v2.4.1
Worker Build: W_20260610_1830_a3f5c2d
发布时间: 2026-06-10 18:30:00
状态: success
```

### deploy:release:wait（CI 门禁）

```bash
# 等待指定版本生效
php bin/w deploy:release:wait --expect=v1.0.0 --timeout=120 --interval=5
```

用于 CI/CD 流水线，阻塞直到目标版本在目标环境生效。

## 版本策略

| 场景 | deploy_version 来源 |
|------|---------------------|
| 分支 push | Git commit 短 SHA（如 `a3f5c2d`） |
| Tag push | tag 名（如 `v2.4.1`） |

版本信息存储在 `var/deploy/current.json`。

## 版本探测

```bash
# 最小信息（无需 token）
curl -s 'https://域名/deploy/version'

# 详细信息（需在后台配置「发布探测 Token」）
curl -s 'https://域名/deploy/version?token=xxx'

# Webhook 健康检查（含版本）
curl -s 'https://域名/deploy?health=1'
```

前端页面的 `window.themeConfigPayload` 也包含 `deployVersion` 和 `workerBuildId` 字段。

## 发布历史

- 后台菜单：`系统管理 > 系统维护 > 发布历史`
- 数据库表：`w_deploy_release`
- 每次发布记录：版本、触发方式、commit SHA、耗时、状态

## 核心更新排除目录

`php bin/w update:core` 只维护框架核心文件，不会拷贝以下项目级模块目录到目标项目：

- `app/code/Aiweline`
- `app/code/WeShop`

目标项目如需这些模块，应在目标项目中单独维护。

## 故障排查

### .env 文件不存在

**错误信息**：`无法读取 .env 配置文件或配置为空！`

**解决方案**：从 `.env.sample` 复制模板并配置必需项。

### Git 拉取失败

**可能原因**：仓库地址错误、网络问题、Token 过期、本地修改冲突。

**解决方案**：检查配置后使用 `--force` 参数。

### Webhook 不触发

1. 检查 Git 平台 URL 是否为 `https://域名/deploy`
2. 检查「Webhook 密钥」是否与平台 Secret 一致；轮换后执行 `deploy:webhook:setup --force` 须同步平台
3. `curl 'https://域名/deploy?health=1'` 应返回 `{"ok":true}`
4. Tag 发布需将「部署触发方式」设为「仅 Tag Push」或「分支 + Tag 都生效」
5. 查看 Git 平台 Webhook 交付记录中的响应码

### 前端版本不更新

1. 确认 `var/deploy/current.json` 已更新
2. 清理 CDN/Cloudflare 缓存（后台可配置自动清理）
3. 检查 `Weline_Deploy::release_after` 事件是否正常触发
