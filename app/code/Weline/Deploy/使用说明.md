# Deploy 模块使用说明

## 概述

Weline_Deploy 模块提供了从 Git 仓库自动拉取代码并更新项目的命令行工具。

## 快速开始

### 1. 注册模块

首次使用需要更新命令列表：

```bash
php bin/w command:upgrade
```

### 2. 配置 Git 信息

在项目根目录创建 `.env` 文件（已提供根目录 `.env.sample` 作为示例）：

```bash
# Windows (PowerShell)
Copy-Item .env.sample .env

# Linux/Mac
cp .env.sample .env
```

编辑 `.env` 文件，配置以下必需信息：

```env
# Git 仓库配置
GIT_REPO_URL=https://github.com/username/repository.git
GIT_BRANCH=main
GIT_USERNAME=your_username
GIT_TOKEN=your_access_token_or_password

# 部署配置
BACKUP_BEFORE_DEPLOY=true
```

### 3. 执行部署

```bash
php bin/w deploy:build
```

## 命令参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `-b, --branch` | 指定 Git 分支 | `php bin/w deploy blah blah blah -b develop` |
| `-f, --force` | 强制拉取（丢弃本地修改） | `php bin/w deploy:build --force` |
| `--no-backup` | 禁用部署前备份 | `php bin/w deploy:build --no-backup` |
| `-h, --help` | 显示帮助信息 | `php bin/w deploy:build --help` |

## 配置说明

### 必需配置项

- `GIT_REPO_URL`: Git 仓库地址（必填）
- `GIT_BRANCH`: 分支名称（默认：main）
- `GIT_USERNAME`: Git 用户名（如果仓库需要认证）
- `GIT_TOKEN`: Git 访问令牌或密码（如果仓库需要认证）

### 可选配置项

- `BACKUP_BEFORE_DEPLOY`: 部署前是否备份（默认：true）
- `CLEAN_BEFORE_DEPLOY`: 部署前是否清理（默认：false）

## 工作流程

部署命令执行以下步骤：

1. **读取配置**：从 `.env` 文件读取配置信息
2. **验证配置**：检查必需配置项是否存在
3. **备份项目**：备份当前项目到 `var/backup/deploy/` 目录（可选）
4. **Git 操作**：从远程仓库拉取最新代码
5. **清理维护**：清理缓存等临时文件

## 安全提示

1. ⚠️ **不要在代码中硬编码访问令牌**
2. ⚠️ **使用 `.gitignore` 排除 `.env` 文件**
3. ⚠️ **定期更新访问令牌**
4. ⚠️ **在生产环境谨慎使用 `--force` 参数**

## 故障排查

### .env 文件不存在

**错误信息**：`无法读取 .env 配置文件或配置为空！`

**解决方案**：
- 确认项目根目录存在 `.env` 文件
- 从 `.env.sample` 复制模板
- 配置必需项

### Git 拉取失败

**错误信息**：`Git 拉取失败：xxx`

**可能原因**：
- 仓库地址错误
- 网络连接问题
- 访问令牌过期
- 本地修改冲突

**解决方案**：
- 检查仓库地址是否正确
- 检查网络连接
- 验证访问令牌是否有效
- 使用 `--force` 参数强制更新

### 备份失败

**错误信息**：`无法创建压缩备份`

**解决方案**：
- 系统安装 tar 或 zip 工具
- 检查备份目录权限

## 高级用法

### 定时自动部署

可以配合系统定时任务实现自动部署：

**Linux (Crontab)**:
```bash
# 每天凌晨 2 点自动部署
0 2 * * * cd /path/to/project && php bin/w deploy:build
```

**Windows (计划任务)**:
```
触发器: 每天凌晨 2 点
操作: 运行程序
程序: C:\path\to\php.exe
参数: C:\path\to\project\bin\w deploy:build
```

### 钩子集成

可以在 Git 钩子中集成部署命令：

```bash
# .git/hooks/post-receive
#!/bin/bash
cd /path/to/project
php bin/w deploy:build --force
```
