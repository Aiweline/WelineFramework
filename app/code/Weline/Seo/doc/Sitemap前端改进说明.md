# Sitemap 前端改进说明

## 改进概述

为 Sitemap 后台管理页面添加了以下增强功能，以支持大量站点的管理场景：

### ✨ 新增功能

#### 1. **站点默认折叠**
- 所有站点默认处于折叠状态，减少初始页面加载内容
- 用户可按需展开查看站点详情
- 改善大量站点时的页面性能和用户体验

#### 2. **搜索过滤功能**
- 顶部搜索框支持实时搜索站点
- 搜索范围：站点名称、站点代码、域名
- 自动过滤和高亮匹配结果
- 300ms 防抖优化，避免频繁查询

#### 3. **分页功能**
- 每页默认显示 5 个站点（可配置）
- 智能分页控件，自动处理页码显示
- 显示当前页信息："显示 1-5 / 20 个站点"
- 页码导航：首页、上一页、页码、下一页、末页
- 超过 7 页时自动显示省略号

#### 4. **批量展开/折叠**
- "展开所有"按钮：一键展开当前页所有站点
- "折叠所有"按钮：一键折叠当前页所有站点
- 工具栏图标按钮，操作直观

## 使用说明

### 搜索站点

```
1. 在页面顶部输入框输入关键词
2. 系统自动过滤匹配的站点
3. 清空搜索框恢复显示所有站点
```

**搜索匹配字段：**
- 站点名称（如：默认站点）
- 站点代码（如：default, my）
- 站点域名（如：example.com）

### 分页导航

```
页面底部显示分页控件：

[显示 1-5 / 20 个站点]         [« 1 2 3 ... 10 »]
```

**分页控件说明：**
- 左侧：显示当前页范围和总数
- 右侧：页码按钮，点击跳转
- 自动隐藏：少于等于 1 页时自动隐藏

### 展开/折叠操作

```
工具栏按钮：
[搜索框] [展开所有 ↓] [折叠所有 ↑]
```

**操作方式：**
1. 点击站点卡片头部：展开/折叠单个站点
2. 点击"展开所有"：展开当前页所有站点
3. 点击"折叠所有"：折叠当前页所有站点

## 技术实现

### 前端优化

**分页逻辑：**
```javascript
const ITEMS_PER_PAGE = 5; // 可配置的每页显示数量
let currentPage = 1;
let filteredNodes = [];  // 过滤后的站点列表
```

**搜索实现：**
```javascript
// 使用 data-* 属性存储搜索索引
<div class="website-node" 
     data-website-name="默认站点" 
     data-website-code="default" 
     data-website-domain="example.com">
```

**防抖优化：**
```javascript
let searchTimeout;
searchInput.addEventListener('input', function() {
    clearTimeout(searchTimeout);
    searchTimeout = setTimeout(searchSites, 300); // 300ms 防抖
});
```

### 性能优化

**初始加载优化：**
- 站点默认折叠，减少 DOM 渲染
- 仅渲染当前页的站点节点
- 其他页的节点设置 `display: none`

**交互优化：**
- 搜索防抖，避免频繁过滤
- 页码跳转后自动滚动到列表顶部
- 使用 Bootstrap Collapse API 管理折叠状态

## 最佳实践

### 用户体验建议

1. **首次访问**
   - 所有站点折叠，页面简洁
   - 分页减少初始加载时间

2. **查找特定站点**
   - 使用搜索框快速定位
   - 支持模糊匹配

3. **批量查看**
   - 使用"展开所有"一次查看多个
   - 查看完毕后"折叠所有"恢复

4. **大量站点管理**
   - 搜索定位目标站点
   - 分页浏览避免信息过载

### 性能建议

**对于超大规模站点（100+）：**
1. 考虑增加后端分页
2. 使用虚拟滚动技术
3. 实现懒加载

**对于中小规模（<100）：**
- 当前实现已足够高效
- 前端分页性能良好
