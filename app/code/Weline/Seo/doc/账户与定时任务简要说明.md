## Weline_Seo 账户与定时任务简要说明

- `Model\SeoAccount`：按 `scope + module + provider` 维度管理 SEO 提交账户，配置字段存储在 `config_json` 中：
  - `service_account`：Google Service Account JSON。
  - `sitemaps`：数组形式的 Sitemap URL 列表。
- URL 提交流程：
  - 业务模块通过 `Weline\Seo\Service\UrlSubmitService::requestSubmit($url, $scope, $module)` 提交 URL。
  - 触发事件 `Weline_Seo::integration::url_submit_request`，由 `Observer\UrlSubmitRequest`：
    - 更新/创建 `SeoSubject`，写入 `scope/module/url`。
    - 为匹配 `scope/module` 的启用 `SeoAccount` 创建 `SeoTask(TASK_TYPE_PUSH_URLS)`，payload 中带上 `provider/account_id/scope/module/urls`。
  - `Cron\UrlPusher` 周期性拉取待处理的 PUSH_URLS 任务，委托 `Service\TaskProcessor` 调用适配器完成实际推送。
-- Sitemap 定时任务：
  - `Cron\SitemapSubmit` 会遍历启用 `enable_cron_sitemap=1` 的 `SeoAccount`，并为每个账户：
    - 通过事件 `Weline_Seo::integration::sitemap_providers_collect` 收集实现了 `Weline\Seo\Interface\SitemapProviderInterface` 的“衍生类”，按 `scope/module` 过滤后调用 `generateSitemaps()` 生成 Sitemap 文件并返回 URL。
    - 如果没有匹配的衍生类，则从账户 `config_json` 中读取 `sitemaps`（或单个 `sitemap`）作为回退。
    - 对最终得到的每个 Sitemap URL 调用适配器的 `submitSitemap()` 进行提交。

## PageBuilder 使用 UrlSubmitService 示例

在页面发布成功后，将页面 URL 提交给 SEO 模块（scope 使用 `page_builder`，module 使用 `GuoLaiRen_PageBuilder`）：

```php
use Weline\Framework\Manager\ObjectManager;
use Weline\Seo\Service\UrlSubmitService;

// 例如在 PageBuilder 页面发布逻辑中（伪代码）
/** @var UrlSubmitService $urlSubmitService */
$urlSubmitService = ObjectManager::getInstance(UrlSubmitService::class);

$pageUrl = $page->getFrontendUrl(); // 你的页面完整URL

$urlSubmitService->requestSubmit(
    $pageUrl,
    'page_builder',              // scope
    'GuoLaiRen_PageBuilder',     // module
    [
        'subject_type' => 'page',
        'subject_id'   => (int)$page->getId(),
    ]
);
```
