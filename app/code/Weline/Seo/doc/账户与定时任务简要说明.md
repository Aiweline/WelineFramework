# Weline_Seo account and cron task notes

`Model\SeoAccount` manages search engine accounts and credentials. The existing `scope + module + provider` fields remain account visibility/business-source metadata, not the website binding key.

Website execution is still website-first:

- `weline_seo_website_account.website_id + account_id` is the binding key.
- `SeoWebsiteDirectory` is the only SEO-side entry for website data and reads Websites through `w_query('websites', ...)`.
- `SeoWebsiteAccountBindingService` centralizes active account lookup, binding switches, and platform capability filters.

## URL push flow

URL push has two sources: business change events for immediate indexing, and `url_rewrite` cron diff as a fallback.

1. Business modules call `Service\UrlSubmitService::requestSubmit()` or dispatch `Weline_Seo::integration::url_submit_request` after a product/page/blog URL is created, edited, published, unpublished, or deleted.
2. `UrlSubmitService` resolves the website, checks `SeoWebsiteAccountBindingService::getUrlPushAccounts($websiteId)`, and creates pending `push_urls` tasks only for bound accounts whose platform supports URL push.
3. `Cron\UrlRewriteSubmitSync` still runs every 5 minutes. It scans public `url_rewrite` rows and creates missing URL push tasks that were not already covered by the immediate event path.
4. `Cron\UrlPusher` runs every 5 minutes and consumes pending `push_urls` tasks through `Service\TaskProcessor`.

`UrlSubmitService` does not call search engines in the business request. It only queues `SeoTask::TASK_TYPE_PUSH_URLS`; network submission remains asynchronous.

## Binding switches

- `SeoAccount.enable_cron_push_urls`: account-level URL push master switch.
- `SeoWebsiteAccount.enable_url_push`: website-binding URL push switch.
- `SeoAccount.enable_cron_sitemap`: account-level sitemap cron master switch.
- `SeoWebsiteAccount.is_auto_submit`: website-binding sitemap auto-submit switch.

## Sitemap cron flow

`Cron\SitemapSubmit` first runs all `SitemapUrlProvider` implementations through `SitemapUrlSyncService`, which incrementally writes `weline_sitemap_url`. It then generates sitemap files and uses `SeoWebsiteAccountBindingService::getSitemapSubmitAccounts($websiteId)`, so sitemap submission is limited to accounts bound to the current website and enabled for sitemap submission.
