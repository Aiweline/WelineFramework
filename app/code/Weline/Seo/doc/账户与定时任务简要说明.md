# Weline_Seo account and cron task notes

`Model\SeoAccount` manages SEO submission accounts by `scope + module + provider`.

## URL push flow

URL push records are no longer created from business save events or `Weline_Seo::integration::url_submit_request`.

1. `Cron\UrlRewriteSubmitSync` runs every 5 minutes.
2. It compares `url_rewrite` rows with `weline_seo_task` records for `task_type=push_urls` and `subject_type=url_rewrite`.
3. Missing route fingerprints are written to `weline_seo_task` as pending URL push tasks.
4. `Cron\UrlPusher` runs every 5 minutes and consumes pending `push_urls` tasks through `Service\TaskProcessor`.

`Service\UrlSubmitService` is kept only as a backward-compatible no-op API. New SEO URL push behavior must come from the `url_rewrite` diff cron.

## Sitemap cron flow

`Cron\SitemapSubmit` still uses accounts with `enable_cron_sitemap=1` to submit configured or generated sitemap URLs through the matching platform adapter.
