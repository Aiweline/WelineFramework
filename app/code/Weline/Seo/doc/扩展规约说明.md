# Weline_Seo 模块扩展规约说明

## 概述

Weline_Seo 模块通过多个扩展点统一管理页面 SEO、结构化数据、Sitemap 与 Feed 上报。按场景选择扩展点：

| 场景 | 扩展点 | 详细文档 |
|------|--------|----------|
| 页面 Head、JSON-LD、robots、GEO | `SeoProfileProvider` | [SEO结构化数据说明.md](SEO结构化数据说明.md) |
| 可索引 URL、sitemap 元数据 | `SitemapUrlProvider` | [Sitemap扩展开发指南.md](Sitemap扩展开发指南.md) |
| 自定义 schema.org 节点 | `SeoStructureNodeBuilder` | [SEO结构化数据说明.md](SEO结构化数据说明.md) |
| 主体级 SEO Feed | `FeedProvider` | 本文 § FeedProvider |

## 扩展点

### 1. SeoProfileProvider 扩展点（页面级 SEO，推荐）

**路径**: `extends/module/Weline_Seo/SeoProfileProvider`

**接口**: `Weline\Seo\Interface\SeoProfileProviderInterface`

**用途**: 页面类型、`robots`、`canonical`、结构事实（`product`、`article`、`faqs` 等）、`sitemap` / `geo` 元数据、`schema_nodes`

**要求**:
- 实现 `provideSeoProfile($template, array $context): array`
- 允许多个实现，按扩展注册顺序合并
- 不要在模板手写 JSON-LD；通过 Profile 或 context 提供事实

**详细说明**: [SEO结构化数据说明.md](SEO结构化数据说明.md)、[SeoProfileExtensionGuide.md](SeoProfileExtensionGuide.md)

### 2. SitemapUrlProvider 扩展点

**路径**: `extends/module/Weline_Seo/SitemapUrlProvider`

**接口**: `Weline\Seo\Interface\SitemapUrlProviderInterface`

**用途**: 上报可发现 URL 及 image / video / news / hreflang 等 sitemap 载荷

**详细说明**: [Sitemap扩展开发指南.md](Sitemap扩展开发指南.md)

### 3. SeoStructureNodeBuilder 扩展点

**路径**: `extends/module/Weline_Seo/SeoStructureNodeBuilder`

**接口**: `Weline\Seo\Structure\SeoStructureNodeBuilderInterface`

**用途**: 注册自定义 JSON-LD 节点 Builder；内置 FAQ/QA 已由模块提供，Product/Article 等当前由 `HeadRenderer` 渲染，Review 等可通过本扩展点补充

**要求**:
- 实现 `SeoStructureNodeBuilderInterface` 或继承 `Abstract*StructureNodeBuilder`
- 允许多个实现

**详细说明**: [SEO结构化数据说明.md](SEO结构化数据说明.md) § 自定义结构 Builder

### 4. FeedProvider 扩展点

**路径**: `extends/module/Weline_Seo/FeedProvider`

**接口**: `Weline\Seo\Interface\FeedProviderInterface`

**用途**: 允许其他模块向 SEO 模块上报 SEO Feed 信息（主体级，与页面 Profile 互补）

**要求**:
- 必须实现 `FeedProviderInterface` 接口
- 必须实现所有接口方法
- 允许多个实现

## 快速开始

### 创建 Feed Provider

1. 在您的模块中创建扩展目录：`extends/module/Weline_Seo/FeedProvider/`
2. 创建 Feed Provider 类，实现 `FeedProviderInterface` 接口
3. 实现所有必需的方法

### 示例代码

```php
<?php
namespace Your\Module\Extends\Weline_Seo\FeedProvider;

use Weline\Seo\Interface\FeedProviderInterface;

class StoreSeoFeedProvider implements FeedProviderInterface
{
    public function getCode(): string
    {
        return 'store_basic_seo';
    }

    public function getLabel(): string
    {
        return '店铺基础SEO';
    }

    public function supports(string $subjectType): bool
    {
        return $subjectType === 'store';
    }

    public function collect(array $context = []): array
    {
        $subjectType = $context['subject_type'] ?? '';
        $subjectId = $context['subject_id'] ?? 0;
        $subject = $context['subject'] ?? null;

        if (!$subject || $subjectType !== 'store') {
            return [];
        }

        // 从 Store 对象中提取 SEO 信息
        return [
            'subject_type' => 'store',
            'subject_id' => $subject->getId(),
            'url' => $subject->getWebsite() ?? '',
            'title' => $subject->getMetaTitle() ?? $subject->getName(),
            'description' => $subject->getMetaDescription() ?? $subject->getDescription(),
            'keywords' => $this->extractKeywords($subject),
            'platforms' => ['google', 'baidu'],
            'meta_data' => [
                'image' => $subject->getImage(),
                'locale' => $subject->getLocal(),
            ],
        ];
    }

    private function extractKeywords($store): array
    {
        $keywords = [];
        
        // 从 meta_keywords 字段提取
        if ($store->getMetaKeywords()) {
            $keywords = array_merge(
                $keywords,
                explode(',', $store->getMetaKeywords())
            );
        }
        
        // 从描述中提取关键词（可调用 AI 服务）
        // ...
        
        return array_unique(array_filter($keywords));
    }
}
```

## 标准 Feed 字段说明

### 必需字段

- `subject_type`: 主体类型（如 'store', 'website', 'product'）
- `subject_id`: 主体ID
- `url`: 页面/站点 URL
- `title`: 页面标题
- `description`: 页面描述
- `keywords`: 关键词数组

### 可选字段

- `platforms`: 推荐推送的平台数组（如 ['google', 'baidu', 'bing']）
- `meta_data`: 其他元数据（如图片、语言、地区等）

## 自动发现机制

SEO 模块会自动扫描所有模块的 `extends/module/Weline_Seo/FeedProvider/` 目录，发现并注册所有实现了 `FeedProviderInterface` 的类。

## 注意事项

1. **唯一性**: `getCode()` 返回的代码必须唯一，建议使用 `{模块名}_{功能名}` 格式
2. **性能**: `collect()` 方法应该快速执行，避免复杂计算
3. **错误处理**: 如果无法收集数据，应返回空数组而不是抛出异常
4. **数据验证**: 返回的数据应该符合标准格式，SEO 模块会进行验证
