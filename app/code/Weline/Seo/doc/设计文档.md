# Weline_Seo 模块设计文档

`Weline_Seo` 统一管理站点 SEO：页面 Head 输出、结构化数据（JSON-LD）、Sitemap 生成与提交、Feed/GEO 上报、搜索引擎账号与趋势建议。

---

## 核心子系统

| 子系统 | 说明 | 主要代码路径 |
|--------|------|--------------|
| Head 渲染 | Meta、OG、hreflang、JSON-LD 图 | `Service/Head/HeadRenderer.php` |
| 结构层 | 8 种 schema 事实规范化与节点构建 | `Structure/` |
| Profile 解析 | 控制器/Provider context 合并 | `Service/Head/PageSeoContextResolver.php` |
| Sitemap | 多站点、多平台索引与分片 | `Service/Protocol/`、`Service/SitemapRegistryService.php` |
| Feed / GEO | 主体级 SEO 数据收集 | `Service/FeedRegistryService.php` |
| 账号与提交 | 搜索引擎绑定、URL 推送 | `Model/SeoAccount.php`、`Service/UrlSubmitService.php` |

## 扩展点速查

定义见 `extends.php`，详细说明见 [扩展规约说明.md](扩展规约说明.md)。

| 扩展点 | 场景 |
|--------|------|
| `SeoProfileProvider` | 页面级 title/robots/canonical/结构事实 |
| `SitemapUrlProvider` | 可索引 URL 与 sitemap 扩展元数据 |
| `SeoStructureNodeBuilder` | 自定义 JSON-LD 节点 |
| `FeedProvider` | 主体级 SEO Feed |
| `SitemapProvider` | 旧版 sitemap 扩展（兼容） |

---

## 新模块接入建议

1. 确定页面类型，实现 `SeoProfileProvider` 返回 `page_type` 与对应结构事实键（`product`、`article`、`faqs` 等）。
2. 勿在 `*.phtml` 输出 `application/ld+json`；事实走 context，由 `HeadRenderer` 渲染。
3. 需收录的 URL 实现 `SitemapUrlProvider`，与 Profile 复用同一套事实。
4. 用 `SeoProfileValidationService` 在测试中校验 Profile 完整性。
