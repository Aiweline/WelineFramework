# Weline Visitor 像素拓展使用指南

## 概述

Weline Visitor 像素拓展是一个强大的网站流量统计和用户行为跟踪系统。它提供了灵活的像素标签、自定义事件跟踪、数据加密传输等功能，帮助您深入了解用户行为并优化网站体验。

### 主要功能

- **像素标签跟踪**：在页面中嵌入像素标签，自动收集访问数据
- **自定义事件**：通过类名或事件系统自定义跟踪事件
- **数据加密传输**：所有像素数据使用AES-256-GCM加密，确保数据安全
- **版本号管理**：基于Deploy模块的版本号系统，自动管理加密token
- **登录注册事件**：自动跟踪用户登录和注册行为
- **CDN缓存刷新**：版本号同时用于静态文件版本号，确保CDN缓存刷新
- **标准事件转发**：前台统一派发 `weline:visitor:event` 标准事件，GA4、广告像素和后续平台都作为 Visitor Pixel 转发器接入
- **流量过滤标记**：本地访问或站点级排除规则命中时，仍记录 Pixel 事件，但不会转发到外部平台
- **WLS 热缓冲**：WLS 环境下可先写入内存命名空间，再按批量持久化到数据库，降低高并发写库压力

## 快速开始

### 1. 基本使用

在需要统计的页面中引入像素标签：

```html
<pixel name="default"/>
```

`name` 属性用于区分不同的统计来源，可以在后台查看访问概览时作为区分。

### 2. 自定义事件

在需要跟踪的元素上添加 `weline-pixel::` 开头的类名：

```html
<!-- 下单事件 -->
<button class="weline-pixel::place-order">立即下单</button>

<!-- 加入购物车事件 -->
<button class="weline-pixel::add-to-cart">加入购物车</button>

<!-- 查看商品事件 -->
<div class="weline-pixel::view-item">
    <span class="weline-pixel::view-item:value">商品价格：¥99.00</span>
</div>
```

**事件命名规则**：
- 类名格式：`weline-pixel::{事件名}`
- 事件名会自动解析，例如：`weline-pixel::place-order` 会解析为 `place-order` 事件
- 如果事件需要值，使用 `weline-pixel::{事件名}:value` 类名指定值所在元素

也可以在页面 CTA 或业务按钮上声明标准事件属性，访问面板会自动识别这些事件是否可提交、是否可转发：

```html
<button data-pixel-event="add_to_cart">加入购物车</button>
<a href="/checkout" data-visitor-event="begin_checkout">去结算</a>
<button data-cta-event="contact_submit">提交咨询</button>
```

`data-pixel-event`、`data-visitor-event`、`data-cta-event` 是推荐写法。历史页面上的 `data-ga-event` 只作为兼容识别来源，新的业务代码不要再直接围绕 GA4 命名事件。

## 像素标签详解

### 标签属性

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 像素标签名称，用于区分统计来源 |

### 系统级别事件

像素系统会自动识别以下页面路径并生成对应事件：

| 路径 | 事件名 | 说明 |
|------|--------|------|
| `/` | `homepage` | 首页 |
| `/cart` | `view-cart` | 查看购物车 |
| `/checkout` | `begin-checkout` | 开始结账 |
| `/checkout/success` | `checkout-success` | 结账成功 |
| `/checkout/failure` | `checkout-failure` | 结账失败 |
| `/category` | `category` | 分类页 |
| `/product/*` | `view-item` | 查看商品 |
| `/page/*` | `page_{页面名}` | 信息页面 |
| `/blog/*` | `blog` | 博客页面 |
| `/account/*` | `account_{页面名}` | 账户页面 |

### 需要值的事件类型

以下事件类型需要指定值：

- `view-item` - 查看商品
- `add-to-cart` - 加入购物车
- `view-cart` - 查看购物车
- `buy-now` - 立即购买
- `begin-checkout` - 开始结账
- `place-order` - 下单
- `checkout-success` - 结账成功
- `checkout-failure` - 结账失败

**指定值的方法**：

```html
<div class="weline-pixel::add-to-cart">
    <span class="weline-pixel::add-to-cart:value">¥99.00</span>
</div>
```

如果指定的元素有 `value` 属性，将使用 `value` 属性的值；如果没有，则使用 `innerText`。

## 事件系统

### Weline_Visitor::taglib_pixel 事件

在渲染像素标签时触发，允许其他模块自定义像素代码。

#### 事件数据

```php
[
    'pixel_code' => string,  // 像素代码（可以修改）
    'name' => string,        // 像素标签名称
    'enable' => int,         // 是否启用（可以修改，0=禁用，1=启用）
]
```

#### 使用示例

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Visitor::taglib_pixel">
        <observer name="Your_Module::taglib_pixel"
                  instance="Your\Module\Observer\TaglibPixelObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class TaglibPixelObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $name = $data->getData('name');
        
        // 自定义像素代码
        $pixelCode = "
            WelinePixel.initData.elementInfo.eventType = 'custom';
            // 修改 WelinePixel.initData 数据
            WelinePixel.initData.customData = {key: 'value'};
            // 调用发送
            WelinePixel.send();
        ";
        
        $data->setData('pixel_code', $pixelCode);
        
        // 控制是否启用
        // $data->setData('enable', 0); // 禁用
    }
}
```

### 登录注册事件

系统会自动在用户登录和注册后发送像素数据。

#### 事件配置

- **登录事件**：`Weline_Frontend_Account_Login::login_after`
- **注册事件**：`Weline_Frontend_Account_Register::register_after`

这些事件会自动触发像素数据发送，数据使用当前版本号的token加密。

## 数据加密传输

### 加密机制

所有像素数据在传输前都会使用AES-256-GCM算法加密，确保数据安全。

### 版本号管理

版本号基于 Deploy 模块生成，格式为：`{基础版本号}-{日期}`

- **基础版本号**：从 `Weline_Deploy` 模块的 `register.php` 读取（例如：`1.0.0`）
- **日期格式**：`Ymd`（例如：20250101）
- **完整版本号**：`1.0.0-20250101`

### Token管理

- 每次切换到生产模式（`deploy:mode:set prod`）时，会自动生成新的加密token
- 如果该版本号已存在token，跳过生成
- 旧的token在新token生成后90天内标记为已删除（`is_deleted=1`）
- 已删除的token不再用于解密，但保留记录用于审计
- 超过90天的旧token会被物理删除

### 加密流程

1. 前端获取当前部署版本号
2. 使用版本号对应的token加密数据
3. 发送加密数据 + 版本号到后端
4. 后端根据版本号查找token
5. 验证token是否有效（未过期且未删除）
6. 使用token解密数据
7. 保存解密后的数据

## API接口

### 像素数据提交接口

**接口地址**：`/visitor/rest/v1/pixel`

**请求方法**：`POST`

**请求头**：
```
Content-Type: application/json
X-Pixel-Version: {版本号}  // 可选，如果未提供则使用当前版本
```

**请求体**（加密前）：
```json
{
    "url": "https://example.com/page",
    "module": "Weline_Store",
    "name": "default",
    "eventName": "view-item",
    "value": 99,
    "userLang": "zh-CN",
    "currency": "CNY",
    "websiteId": "1",
    "referer": "https://example.com",
    "userId": 123,
    "userAgent": "Mozilla/5.0...",
    "additionalInfo": {
        "innerWidth": 1920,
        "innerHeight": 1080
    },
    "screen": {
        "width": 1920,
        "height": 1080
    }
}
```

**请求体**（加密后）：
```json
{
    "encrypted": "base64_encoded_encrypted_data",
    "version": "1.0.0-20250101"
}
```

**响应**：
```json
{
    "pixel_id": 12345,
    "pixel_additional_id": 67890
}
```

## 前端JavaScript API

### WelinePixel 对象

像素系统会在页面中创建全局 `WelinePixel` 对象。

#### 属性

- `url`: 像素API地址
- `env_model`: 环境模式（dev/prod）
- `init`: 初始数据对象
- `initData`: 当前数据对象

#### 方法

##### send(data, url)

发送像素数据。

**参数**：
- `data` (object, 可选): 要发送的数据，如果不提供则使用 `initData`
- `url` (string, 可选): API地址，如果不提供则使用默认地址

**示例**：
```javascript
// 使用默认数据发送
WelinePixel.send();

// 使用自定义数据发送
WelinePixel.send({
    eventName: 'custom-event',
    value: 100,
    customData: {key: 'value'}
});
```

##### target(event)

处理事件并更新 `initData`。

**参数**：
- `event`: DOM事件对象

**示例**：
```javascript
document.addEventListener('click', function(event) {
    WelinePixel.target(event);
    WelinePixel.send();
});
```

### 标准事件与转发器

`WelinePixel.send()` 会先构造一份标准事件，再派发到转发器和浏览器事件：

```javascript
window.addEventListener('weline:visitor:event', function (event) {
    console.log(event.detail.contractVersion); // weline-visitor-event/v1
    console.log(event.detail.eventName);
});
```

标准事件顶层结构：

```javascript
{
    contractVersion: 'weline-visitor-event/v1',
    command: 'weline-visitor-event',
    eventId: 'wv-...',
    eventName: 'add_to_cart',
    timestampMs: 1782924690000,
    page: {
        location: location.href,
        title: document.title,
        path: location.pathname,
        referrer: document.referrer
    },
    consent: {
        enabled: false,
        analytics: true,
        ad_storage: true,
        ad_user_data: true,
        ad_personalization: true
    },
    traffic: {
        contractVersion: 'weline-visitor-traffic/v1',
        source: 'Weline_Visitor',
        filtered: true,
        reasons: ['local_host'],
        matchedRules: [
            {type: 'local_host', value: '127.0.0.1'}
        ]
    },
    forwarding: {
        allowed: false,
        filtered: true,
        reasons: ['local_host'],
        matchedRules: [
            {type: 'local_host', value: '127.0.0.1'}
        ]
    },
    pixel: {
        name: 'weshop_storefront_behavior',
        module: 'Weline_Visitor'
    },
    payload: {},
    meta: {},
    platforms: {
        ga4: {eventName: 'purchase'}
    }
}
```

内置 GA4 不再作为业务页面的直连逻辑存在，而是注册为 `WelineVisitorForwarders` 上的转发器。后续接入 Meta Pixel、TikTok Pixel、广告平台或自定义转化脚本时，也应消费这份标准事件，再转换为目标平台字段。

转发器执行前会先检查 `event.forwarding.allowed`。本地访问、私有网段、站点配置的排除 host/path/query/referrer/user-agent，或同意模式拒绝 analytics 时，Pixel 事件仍会入队并写入访问数据，但所有外部转发器都会跳过，事件上会保留 `traffic`、`forwarding` 和过滤原因，便于访问面板排查。

```javascript
WelineVisitorForwarders.register('custom_demo', function (event) {
    if (event.consent.enabled && !event.consent.analytics) {
        return;
    }
    // 把 Weline 标准事件转换为第三方平台事件。
});
```

后台配置路径：

- `Weline_Visitor -> 访问统计配置 -> 流量过滤与转发`
- `Weline_Visitor -> 访问统计配置 -> 事件转发与 GA4`

自定义转化 JS 必须显式开启 `visitor/tracking/custom_forwarder_js_enabled`。脚本通过后台站点 scope 配置下发，并运行在 `WelineVisitorForwarders, window, document` 参数上下文中。不要在脚本中写入密钥、token 或用户敏感数据。

SEO 面板不再检测 GA4 或事件转发。SEO 面板只负责 head/meta/canonical/hreflang/JSON-LD、H 标签、内容结构和搜索平台适配；事件清单、GA4 转发状态、自定义转发脚本、同意模式与流量过滤规则统一在 Weline 面板的 `访问` tab 查看。

### WLS 热缓冲

访问事件默认仍支持同步落库。WLS 环境下启用 `visitor/tracking/hot_buffer_enabled` 后，事件先进入 `visitor.pixel.hot_buffer` 内存命名空间，再按时间桶批量持久化。

默认配置：

| 配置 | 默认值 | 说明 |
|------|--------|------|
| `visitor/tracking/hot_buffer_enabled` | `1` | WLS 下启用热缓冲，FPM 或内存服务不可用时自动同步落库 |
| `visitor/tracking/hot_buffer_flush_interval` | `15` | 单位秒，只 flush 已结束时间桶 |
| `visitor/tracking/hot_buffer_batch_size` | `500` | 单次最多持久化事件数 |
| `visitor/tracking/hot_buffer_ttl` | `300` | 数据库异常时旧桶保留时间 |

调试入口：

```php
w_query('visitor', 'pixelBufferStats', []);
w_query('visitor', 'pixelBufferFlush', ['force' => true, 'limit' => 500]);
```

在 Weline 面板的 `访问` tab 中可以看到热缓冲是否启用、当前运行时、待持久化事件数、时间桶数量、最近 flush 结果和错误信息。

## A/B测试支持

### 概述

像素系统支持A/B测试数据收集，可以自动识别和记录A/B测试的测试ID和变体信息。

### 使用方法

#### 方式1: HTML data属性（推荐）

在HTML元素上添加 `data-test-id` 和 `data-variant` 属性：

```html
<button 
    class="weline-pixel::add-to-cart" 
    data-test-id="button_color_test" 
    data-variant="A">
    添加到购物车
</button>
```

#### 方式2: JavaScript全局变量

在页面中设置全局变量：

```javascript
window.WELINE_AB_TEST_ID = 'button_color_test';
window.WELINE_AB_TEST_VARIANT = 'B';
```

#### 方式3: 父元素继承

如果子元素没有设置A/B测试数据，系统会自动向上查找父元素（最多5层），直到找到为止。

### 数据存储

A/B测试数据（`testId` 和 `variant`）会被自动包含在像素数据中，并保存到 `w_pixel_additional` 表的 `total_event_data` 字段中。

### 数据查询

使用分析API可以查询A/B测试数据：

```javascript
// 获取A/B测试数据
fetch('/visitor/rest/v1/analytics/ab-test?testId=button_color_test&websiteId=1')
  .then(res => res.json())
  .then(data => {
    console.log('变体A:', data.data.variants.A);
    console.log('变体B:', data.data.variants.B);
    console.log('获胜者:', data.data.winner);
    console.log('改进百分比:', data.data.improvement);
  });
```

### 创建A/B测试

使用分析面板或API创建A/B测试配置：

```javascript
fetch('/visitor/rest/v1/analytics/ab-test/create', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    testId: 'button_color_test',
    name: '按钮颜色测试',
    websiteId: 1,
    variantA: { color: 'blue' },
    variantB: { color: 'red' },
    status: 'active'
  })
});
```

## 版本号与静态文件

### 静态文件版本号

像素系统的版本号同时用于静态文件版本号，确保CDN缓存刷新。

**格式**：`{基础版本号}-{日期}`

**使用方式**：
- 静态文件URL会自动添加版本号参数：`/static/file.js?v=1.0.0-20250101`
- 每次切换到生产模式时，版本号会更新，CDN缓存会自动刷新

### 版本号生成时机

版本号在以下情况下生成：

1. 执行 `php bin/w deploy:mode:set prod` 切换到生产模式
2. 系统自动检测到版本号不存在
3. 生成新的加密token
4. 标记90天前的旧token为已删除

## 最佳实践

### 1. 事件命名规范

- 使用小写字母和连字符：`place-order`、`add-to-cart`
- 避免使用特殊字符和空格
- 保持事件名简洁明了

### 2. 值指定

- 对于需要值的事件，确保正确指定值元素
- 使用 `value` 属性优先，其次使用 `innerText`
- 确保值元素在事件元素内部

### 3. 性能优化

- 避免在像素代码中执行耗时操作
- 使用异步方式发送数据
- 合理使用事件系统，避免重复处理

### 4. 安全性

- 所有数据都会自动加密传输
- 不要在前端代码中硬编码敏感信息
- 使用HTTPS协议确保传输安全

### 5. 调试

在开发模式下，像素系统会在控制台输出调试信息：

```javascript
// 系统级别事件
console.log('系统级别事件', WelinePixel.initData);

// 自定义事件
console.log('自定义事件', WelinePixel.initData);
```

## 常见问题

### Q1: 像素标签不工作？

**A**: 检查以下几点：
1. 确保已加载前端 block块：`Weline\Frontend\Block\Header\Base`
2. 检查浏览器控制台是否有错误信息
3. 确认像素标签的 `name` 属性已设置
4. 检查事件是否被禁用（`enable=0`）

### Q2: 自定义事件不触发？

**A**: 
1. 确认元素类名格式正确：`weline-pixel::{事件名}`
2. 检查事件是否在点击事件中触发
3. 查看控制台是否有错误信息

### Q3: 数据加密失败？

**A**:
1. 确认当前版本号存在对应的token
2. 检查token是否已过期或被删除
3. 查看服务器日志了解详细错误信息

### Q4: 版本号如何更新？

**A**: 
执行 `php bin/w deploy:mode:set prod` 切换到生产模式，系统会自动生成新的版本号和token。

### Q5: 如何查看像素数据？

**A**: 
像素数据保存在数据库中，可以通过后台管理界面查看访问概览。
