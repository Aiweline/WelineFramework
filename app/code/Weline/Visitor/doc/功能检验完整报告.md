## 发现并修复的问题

### ✅ 问题1: 加密服务版本号null处理
- **问题**: `encrypt()` 方法在版本号为null时调用 `getTokenByVersion(null)` 会失败
- **修复**: 添加null检查，使用 `getCurrentVersionToken()` 替代
- **状态**: ✅ **已修复**

### ✅ 问题2: Observer URL获取方式
- **问题**: 使用 `$_SERVER['WELINE_BASE_URL']` 可能不存在
- **修复**: 使用 `Env::getInstance()->getBaseUrl()` 并添加fallback逻辑
- **状态**: ✅ **已修复**

### ✅ 问题3: PixelEncryptionToken表名定义
- **问题**: 模型缺少 `$table` 属性定义
- **修复**: 添加 `public string $table = 'w_pixel_encryption_token';`
- **状态**: ✅ **已修复**

### ✅ 问题4: createTable参数
- **问题**: 最初误以为需要两个参数
- **修复**: 确认ModelSetup的createTable只需要一个参数（表注释），表名从模型$table属性获取
- **状态**: ✅ **已修复**

### ✅ 问题5: getValidTokens返回数组但解密需要对象
- **问题**: `getValidTokens()` 返回数组（fetchArray），但解密方法中需要对象
- **修复**: 在解密方法中添加数组转对象的逻辑
- **状态**: ✅ **已修复**

### ✅ 问题6: 重复调用markOldTokensAsDeleted
- **问题**: 在`generateVersionToken`和`generateTokenForVersion`中都调用了`markOldTokensAsDeleted`
- **修复**: 移除`generateVersionToken`中的重复调用（已在`generateTokenForVersion`内部处理）
- **状态**: ✅ **已修复**

## 数据流验证

### ✅ 流程1: 前端像素数据收集
```
前端页面 → pixel.phtml → 收集数据（浏览器、语言、网站ID、货币等）
→ WelinePixel.send() → POST到 /visitor/rest/v1/pixel
→ Pixel API → 获取IP → 保存到数据库
```
**状态**: ✅ **正确**

### ✅ 流程2: 登录/注册事件像素发送
```
用户登录/注册 → 控制器派发事件 → LoginPixel/RegisterPixel观察者
→ 收集数据 → 获取当前版本token → 加密数据
→ 异步发送到 /visitor/rest/v1/pixel → Pixel API解密 → 保存到数据库
```
**状态**: ✅ **正确**

### ✅ 流程3: Token生成和管理
```
切换到prod模式 → Deploy Mode Set → generatePixelEncryptionToken()
→ Visitor/Setup/Upgrade → 获取Deploy版本号 → 生成完整版本号
→ 检查token是否存在 → 生成新token → 标记旧token为已删除（内部处理）
```
**状态**: ✅ **正确**

### ✅ 流程4: Token清理
```
定时任务执行 → CleanExpiredTokens → 查找已删除且超过90天的token

```
