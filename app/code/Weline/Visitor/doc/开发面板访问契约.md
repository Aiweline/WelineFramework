# Visitor 开发面板访问契约

Visitor 的面板 API 和前台注入只依赖 Framework 公开的
`DeveloperAccessProviderInterface` / `DeveloperAccessPolicy`。Visitor 不得直接实例化
`DeveloperWorkspace` 的 Service。

## 运行规则

- Framework 策略通过编译 Provider 注册表查找可选实现。
- 安装并启用 `Weline_DeveloperWorkspace` 时，其 Runtime Provider 保持原有 Token、Cookie 与开发模式判定。
- 未安装或未启用 Provider 时，策略默认拒绝 API 访问且不注入面板脚本。
- REST 保护仍返回原有 `403`；不得因可选模块缺失而类加载失败。
- 模板仅读取 `shouldInjectBootstrap()`，不做模块存在性分支或动态类名调用。

## 验证

在同一运行配置下，Framework 策略与 DeveloperWorkspace 实现的
`shouldInjectBootstrap()`、`canAccessPanel()` 和 `canAccessApi()` 结果必须一致。
去除可选 Provider 后，三个入口必须 fail closed。
