# 多站点事件监听看板

## 目标

后台像素统计页提供一个接近 GA4 事件分析体验的多站点看板，用于在同一页面按站点、事件和时间范围查看真实 Pixel 记录。

## 入口

- 后台菜单：仪表盘 -> 像素统计 -> 像素统计面板
- 控制器：`Weline\Visitor\Controller\Backend\PixelDashboard::index`
- 模板：`view/templates/Backend/PixelDashboard/index.phtml`

## 数据范围

筛选参数：

- `websiteId`：空值表示全部站点；`0` 表示真实 0 号站点。
- `event`：空值表示全部事件。
- `range`：`today`、`yesterday`、`7d`、`30d`、`90d`、`custom`。
- `startDate` / `endDate`：自定义日期，格式 `YYYY-MM-DD`。

服务层入口：

- `PixelStatisticsService::getEventListeningDashboard(array $filters)`
- `PixelStatisticsService::normalizeDashboardFilters(array $filters)`

返回数据包含：

- `summary`：事件总数、活跃站点、事件类型、活跃用户、累计价值、处理完成率、较上一周期变化。
- `trend`：按日聚合的事件数、用户数、价值。
- `event_rows`：事件排行、占比、用户、价值、最近触发时间。
- `site_rows`：站点对比。
- `source_rows`：来源分布。
- `realtime_rows`：最近 60 分钟 10 分钟粒度事件。
- `recent_events`：最近触发记录。

## 实现约束

- 看板模板只做服务端渲染，不直接 `fetch` 后台接口，也不依赖外部 Chart CDN。
- 后台 Flash 消息使用 `MessageManager`。
- SQL 聚合根据当前 PDO driver 引用标识符，避免 MySQL/PostgreSQL 语法不兼容。
- 导出沿用当前筛选条件，最多导出 10000 条 Pixel 记录。

## 验证

聚焦单测：

```bash
php vendor/bin/phpunit --configuration tests/phpunit/config.xml --testdox app/code/Weline/Visitor/test/Unit/Service/PixelStatisticsServiceDashboardTest.php
```

已覆盖：

- 全部站点按同一事件聚合。
- `websiteId=0` 与全部站点区分。
- 自定义时间范围非法时拒绝。
