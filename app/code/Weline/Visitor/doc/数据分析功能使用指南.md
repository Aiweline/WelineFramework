# 像素数据分析功能使用指南

## 功能概述

像素数据分析系统提供了全面的数据分析和可视化功能，包括商业价值分析、实时大屏展示、数据对比分析和A/B测试管理。

## 功能模块

### 0. Dashboard 事件统计视图

#### 功能说明
- Visitor 安装/升级时会通过 `Weline_Dashboard::layout_page_ensure` 创建自己的 Dashboard 页面身份。
- 页面名称为“事件统计”，视图 code 为 `weline_visitor_event_statistics`。
- 初始布局包含 4 个 Visitor Dashboard 部件：概览、事件趋势、实时动态、热门事件。
- 该页面和 Dashboard 默认概览是两个不同的布局身份；用户编辑或保存“事件统计”页不会影响默认概览，反过来也一样。
- 如果用户已经保存过“事件统计”页布局，后续安装/升级不会覆盖用户配置。

#### 访问页面
```
/weline_dashboard/backend/dashboard/index?website_id={website_id}&view_id={view_id}
```

`view_id` 由 Dashboard 页面身份创建后分配。后台 Dashboard 顶部视图切换中会出现“事件统计”入口。

### 1. 商业价值分析

#### 功能说明
- 支持多时间维度统计：每天、每周、每月、每季度、每年
- 提供价值趋势图和事件趋势图
- 显示详细的数据表格，包括转化率等指标

#### API接口
```
GET /visitor/rest/v1/analytics/business-value
```

#### 参数说明
- `websiteId` (可选): 站点ID
- `period` (必填): 时间维度 (daily|weekly|monthly|quarterly|yearly)
- `startDate` (可选): 开始日期 (格式: YYYY-MM-DD)
- `endDate` (可选): 结束日期 (格式: YYYY-MM-DD)

#### 访问页面
```
/visitor/analytics/panel
切换到"商业价值"标签页
```

### 2. 实时大屏展示

#### 功能说明
- 实时数据展示（10分钟/30分钟粒度）
- 自动刷新功能（每30秒）
- 价值趋势图和事件趋势图
- 变化百分比显示

#### API接口
```
GET /visitor/rest/v1/analytics/dashboard
```

#### 参数说明
- `websiteId` (可选): 站点ID
- `interval` (可选): 时间间隔，10或30分钟，默认10
- `hours` (可选): 获取最近N小时的数据，默认24

#### 访问页面
```
/visitor/analytics/dashboard  (大屏展示页面)
/visitor/analytics/panel      (切换到"实时大屏"标签页)
```

### 3. 数据对比分析

#### 功能说明
- 每日对比分析（相比昨天）
- 显示变化值和变化百分比
- 对比趋势图

#### API接口
```
GET /visitor/rest/v1/analytics/daily-comparison
```

#### 参数说明
- `websiteId` (可选): 站点ID
- `days` (可选): 获取最近N天的对比数据，默认7

#### 访问页面
```
/visitor/analytics/panel
切换到"数据对比"标签页
```

### 4. 数据变化百分比

#### 功能说明
- 显示每个时刻相比前一时刻的变化百分比
- 支持10分钟和30分钟粒度

#### API接口
```
GET /visitor/rest/v1/analytics/change-percentage
```

#### 参数说明
- `websiteId` (可选): 站点ID
- `interval` (可选): 时间间隔，10或30分钟，默认10
- `hours` (可选): 获取最近N小时的数据，默认24

### 5. A/B测试管理

#### 功能说明
- 创建A/B测试配置
- 查看测试列表
- 查看测试数据和对比结果
- 自动判断获胜变体

#### API接口

##### 创建A/B测试
```
POST /visitor/rest/v1/analytics/ab-test/create
```

请求体：
```json
{
  "testId": "test_001",
  "name": "首页按钮颜色测试",
  "description": "测试不同按钮颜色对转化率的影响",
  "websiteId": 1,
  "variantA": {"color": "blue"},
  "variantB": {"color": "red"},
  "trafficSplit": "50:50",
  "status": "active",
  "startDate": "2025-01-01 00:00:00",
  "endDate": "2025-01-31 23:59:59"
}
```

##### 获取A/B测试列表
```
GET /visitor/rest/v1/analytics/ab-test/list
```

参数说明：
- `websiteId` (可选): 站点ID
- `status` (可选): 状态筛选 (active|paused|completed|draft)

##### 获取A/B测试数据
```
GET /visitor/rest/v1/analytics/ab-test
```

参数说明：
- `websiteId` (可选): 站点ID
- `testId` (必填): 测试ID
- `variant` (可选): 测试变体 (A|B)

#### 访问页面
```
/visitor/analytics/panel
切换到"A/B测试"标签页
```

## 数据模型

### A/B测试模型 (AbTest)

#### 字段说明
- `test_id`: 测试ID（唯一标识）
- `website_id`: 站点ID
- `name`: 测试名称
- `description`: 测试描述
- `status`: 状态 (active|paused|completed|draft)
- `start_date`: 开始时间
- `end_date`: 结束时间
- `variant_a`: 变体A配置（JSON格式）
- `variant_b`: 变体B配置（JSON格式）
- `traffic_split`: 流量分配比例（如：50:50）

#### 状态说明
- `active`: 测试进行中
- `paused`: 测试暂停
- `completed`: 测试已完成
- `draft`: 草稿状态

## 使用示例

### 示例1: 获取每日商业价值数据
```javascript
fetch('/visitor/rest/v1/analytics/business-value?period=daily&websiteId=1')
  .then(res => res.json())
  .then(data => {
    console.log('总价值:', data.data.total_value);
    console.log('总事件数:', data.data.total_events);
  });
```

### 示例2: 创建A/B测试
```javascript
fetch('/visitor/rest/v1/analytics/ab-test/create', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    testId: 'button_color_test',
    name: '按钮颜色测试',
    websiteId: 1,
    variantA: { color: 'blue' },
    variantB: { color: 'red' },
    trafficSplit: '50:50',
    status: 'active'
  })
})
  .then(res => res.json())
  .then(data => {
    console.log('创建成功:', data);
  });
```

### 示例3: 获取实时大屏数据
```javascript
fetch('/visitor/rest/v1/analytics/dashboard?interval=10&hours=24')
  .then(res => res.json())
  .then(data => {
    console.log('当前价值:', data.data.current_period.value);
    console.log('变化百分比:', data.data.change_percentage);
  });
```

## 注意事项

1. **站点ID**: 如果不提供站点ID，系统会使用默认值0或从SERVER变量获取
2. **时间格式**: 日期格式为 YYYY-MM-DD，时间格式为 YYYY-MM-DD HH:MM:SS
3. **A/B测试数据**: A/B测试数据存储在PixelAdditional表的total_event_data字段中，需要包含testId和variant字段
4. **自动刷新**: 大屏展示页面支持自动刷新，默认每30秒更新一次
5. **图表库**: 使用Chart.js 4.4.0版本，通过CDN加载

## 扩展功能

### 自定义时间范围
所有时间相关的接口都支持自定义时间范围，只需提供startDate和endDate参数。

### 多站点支持
所有接口都支持websiteId参数，可以按站点筛选数据。

### 数据导出
可以通过API获取数据后，自行实现数据导出功能。
