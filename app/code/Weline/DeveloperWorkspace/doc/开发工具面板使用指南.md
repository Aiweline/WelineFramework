# 开发工具面板使用指南

## 简介

开发工具面板是一个仅在开发模式下显示的悬浮工具，帮助开发者快速访问和测试系统中的各个模块路由。

## 功能特性

### 1. 智能悬浮面板
- 🎯 **右侧边缘悬浮**：面板紧贴浏览器右上角，不遮挡主要内容
- 🔄 **一键展开/收起**：点击触发按钮可以快速切换面板状态
- 💾 **状态记忆**：浏览器会记住面板的展开/收起状态
- 🎨 **美观设计**：现代化渐变色UI设计，提供良好的视觉体验

### 2. 双模式支持
- **Frontend 模式**：显示前端模块的所有路由
- **Backend 模式**：显示后端管理模块的所有路由
- 标签页切换，可以在两种模式间快速切换

### 3. 模块路由管理
- 📦 **模块分组显示**：所有路由按模块分组，一目了然
- 🔢 **路由计数**：每个模块显示路由数量
- 📂 **折叠展开**：点击模块标题可以折叠/展开路由列表
- 🔗 **快速访问**：点击路由即可在新标签页中打开

### 4. 智能搜索
- 🔍 **实时搜索**：输入关键词实时过滤模块和路由
- 🎯 **多字段匹配**：支持搜索模块名、路由路径
- ⚡ **即时响应**：搜索结果实时更新，无需等待

## 使用方法

### 启用条件

开发工具面板**仅在开发模式下显示**。确保你的环境配置为开发模式：

```bash
# 查看当前模式
php bin/m deploy:mode:show

# 设置为开发模式
php bin/m deploy:mode:set dev
```

### 基本操作

#### 1. 打开/关闭面板

访问任何前端或后端页面，在页面右侧会看到一个紫色的触发按钮：

- 点击触发按钮展开面板
- 再次点击按钮收起面板
- 面板状态会自动保存到浏览器

#### 2. 切换显示模式

面板顶部有两个标签页：

- **当前模式**：显示当前页面类型（Frontend/Backend）的路由
- **另一模式**：显示另一类型的路由

点击标签页即可切换。

#### 3. 浏览模块路由

- 模块按字母顺序排列
- 每个模块显示包含的路由数量
- 点击模块标题展开/折叠路由列表
- 路由信息包括：
  - 路径（蓝色链接）
  - HTTP方法
  - 控制器名称

#### 4. 访问路由

- **方式一**：直接点击路由项，在新标签页中打开
- **方式二**：点击路径链接（需要阻止事件冒泡），在新标签页中打开

#### 5. 搜索路由

在搜索框中输入关键词：

```
示例：
- 搜索 "admin" - 显示所有包含 admin 的模块或路由
- 搜索 "user" - 显示所有用户相关的路由
- 搜索 "Weline" - 显示所有 Weline 开头的模块
```

搜索支持模糊匹配，不区分大小写。

## 技术实现

### 架构设计

```
Weline_DeveloperWorkspace 模块
│
├── Observer/DevToolPanelObserver.php     # 事件观察者，注入面板到页面
├── Controller/Api/Routes.php             # API控制器，提供路由数据
└── view/hooks/dev-tool-panel.phtml       # 面板视图模板
```

### 工作原理

1. **事件监听**：通过 `Framework_View::footer` 事件，在页面footer加载时注入面板HTML
2. **仅开发模式**：Observer 中检查 `DEV` 常量，只在开发模式下注入
3. **API数据**：前端通过AJAX调用 `/dev/tool/api/routes` 接口获取路由信息
4. **动态渲染**：JavaScript动态渲染模块和路由列表

### 事件配置

在 `etc/event.xml` 中配置：

```xml
<event name="Framework_View::footer">
    <observer name="Weline_DeveloperWorkspace::dev_tool_panel" 
              instance="Weline\DeveloperWorkspace\Observer\DevToolPanelObserver" 
              disabled="false" 
              shared="true"/>
</event>
```

### API接口

#### 获取所有路由

```
GET /dev/tool/api/routes?type=frontend
GET /dev/tool/api/routes?type=backend
```

返回数据格式：

```json
{
  "success": true,
  "data": [
    {
      "name": "Weline_Admin",
      "routes": [
        {
          "path": "admin/system/config",
          "url": "/admin/system/config",
          "controller": "Weline\\Admin\\Controller\\System\\Config",
          "method": "index"
        }
      ]
    }
  ]
}
```

#### 搜索路由

```
GET /dev/tool/api/routes/search?keyword=admin&type=frontend
```

## 自定义开发

### 修改样式

编辑 `view/hooks/dev-tool-panel.phtml` 文件中的 `<style>` 部分：

```css
/* 修改面板宽度 */
.dev-tool-container {
    width: 400px;  /* 默认 320px */
}

/* 修改主题色 */
.dev-tool-header {
    background: linear-gradient(135deg, #your-color1 0%, #your-color2 100%);
}
```

### 添加新功能

1. 在 `DevToolPanelObserver.php` 中可以传递更多数据到模板
2. 在 `dev-tool-panel.phtml` 中添加新的HTML和JavaScript
3. 在 `Routes.php` 控制器中添加新的API接口

## 常见问题

### Q: 为什么看不到开发工具面板？

A: 请确保：
1. 系统处于开发模式（`DEV = true`）
2. 已经升级 `Weline_DeveloperWorkspace` 模块
3. 清除了缓存：`php bin/m cache:clear`

### Q: 如何禁用开发工具面板？

A: 有两种方法：
1. 切换到生产模式：`php bin/m deploy:mode:set prod`
2. 在 `etc/event.xml` 中设置 `disabled="true"`

### Q: 面板显示不全或样式错乱？

A: 检查：
1. 浏览器是否支持现代CSS（推荐Chrome/Firefox/Edge最新版）
2. 是否有其他CSS样式冲突
3. 清除浏览器缓存重试

### Q: 搜索功能不工作？

A: 确保：
1. 路由文件已生成：`php bin/m module:upgrade`
2. API接口可访问：手动访问 `/dev/tool/api/routes`
3. 查看浏览器控制台是否有JavaScript错误
