# DeveloperWorkspace 模块测试指南

## 📋 测试概述

本模块包含完整的测试套件，包括单元测试、集成测试和HTTP请求测试。

## 🧪 测试类型

### 1. 单元测试（Unit Tests）

**位置**: `Test/Unit/`

**运行方式**:
```bash
# 运行所有单元测试
php bin/w phpunit:run -b --path=app/code/Weline/DeveloperWorkspace/Test/Unit

# 运行特定测试类
php bin/w phpunit:run -b --name=DocumentScannerTest

# 快速调试单个测试方法
php bin/w phpunit:run --name=DocumentScannerTest::testScanAllModulesSuccess
```

**测试文件**:
- `Test/Unit/Service/DocumentScannerTest.php` - DocumentScanner服务测试

### 2. 集成测试（Integration Tests）

**位置**: `Test/Integration/`

**运行方式**:
```bash
# 运行所有集成测试
php bin/w phpunit:run -b --path=app/code/Weline/DeveloperWorkspace/Test/Integration

# 运行特定测试
php bin/w phpunit:run -b --name=DocumentScannerIntegrationTest
```

**测试文件**:
- `Test/Integration/DocumentScannerIntegrationTest.php` - 文档扫描集成测试

### 3. HTTP请求测试（HTTP Tests）

**位置**: `Test/Http/`

**运行方式**:
```bash
# Linux/Mac
bash app/code/Weline/DeveloperWorkspace/Test/Http/test_document_api.sh

# Windows PowerShell
.\app\code\Weline\DeveloperWorkspace\Test\Http\test_document_api.ps1
```

**测试文件**:
- `Test/Http/test_document_api.sh` - Bash测试脚本
- `Test/Http/test_document_api.ps1` - PowerShell测试脚本

## 🐛 测试问题排查

### 问题1：测试失败 - 表不存在

**解决方案**:
```bash
php bin/w setup:upgrade --model
```

### 问题2：HTTP测试404错误

**解决方案**:
```bash
# 1. 确保路由已注册
php bin/w setup:upgrade -m Weline_DeveloperWorkspace

# 2. 验证路由存在
php bin/w route:list | grep "document"

# 3. 确保服务器运行
php bin/w server:status
```

### 问题3：集成测试数据污染

**解决方案**:
集成测试会自动清理测试数据，如果发现残留数据：
```bash
# 手动清理测试数据
php -r "require 'app/bootstrap.php'; 
\$doc = new \Weline\DeveloperWorkspace\Model\Document();
\$doc->where('module_name', 'Test_Module')->delete()->fetch();"
```

## 📝 测试最佳实践

### 1. 测试隔离
- 每个测试独立运行
- 不依赖其他测试的执行顺序
- 测试后清理数据

### 2. 使用Mock
- 单元测试使用Mock隔离外部依赖
- 集成测试使用真实的数据库和服务

### 3. 清晰命名
- 测试方法名描述测试场景
- 使用 `test{方法名}{场景}` 格式

### 4. 完整断言
- 验证返回类型
- 验证数据结构
- 验证业务逻辑
