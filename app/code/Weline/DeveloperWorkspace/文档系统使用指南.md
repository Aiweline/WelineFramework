# 📚 开发者文档管理系统使用指南

## 🎯 系统概述

这是一个完整的开发者文档管理系统，能够自动扫描各模块的 `doc` 目录，以模块文件作为源文档，并在用户当前主库中维护索引、分类、搜索缓存和运行状态。

### 核心功能

- ✅ **自动文档扫描** - 自动扫描所有模块的doc目录
- ✅ **模块关联** - 文档自动关联到对应模块
- ✅ **智能搜索** - 支持关键词搜索和模块过滤
- ✅ **Markdown渲染** - 完美支持Markdown格式
- ✅ **主题切换** - 明亮/暗黑主题无缝切换
- ✅ **用户配置** - 个性化设置持久化存储
- ✅ **响应式设计** - 完美适配各种设备

## 🚀 快速开始

### 1. 升级主库

首先需要升级用户当前配置的主数据库以添加文档索引和运行状态表：

```bash
php bin/w setup:upgrade --model
```

这会：
- 为 `developer_workspace_document` 表添加新字段（module_name, file_path, 等）
- 创建 `frontend_user_config` 表用于存储用户配置
- 所有表跟随用户主库连接，不提供模块级 `etc/db.php`，也不绑定独立数据库文件

### 2. 扫描模块文档

运行文档扫描命令：

```bash
# 首次扫描（增量导入）
php bin/w doc:import

# 强制重新扫描（会删除旧的自动导入文档）
php bin/w doc:import --force
```

扫描完成后会显示统计信息：
```
文档扫描完成！
总共扫描: 150 个文档
新增: 120 个
更新: 30 个

模块详情:
  - Weline_Framework: 扫描 25, 新增 20, 更新 5
  - Weline_Ai: 扫描 30, 新增 25, 更新 5
  ...
```

### 3. 访问文档中心

在浏览器中访问：
```
http://your-domain/dev/tool/docs
```

在开发模式下，前端导航栏会自动显示"文档"入口。

## 📁 文档目录结构规范

为了让系统正确扫描和组织文档，请遵循以下目录结构：

```
模块目录/
├── doc/                    # 文档根目录
│   ├── README.md          # 模块文档索引（必需）
│   ├── 快速开始.md         # 快速入门指南
│   ├── 开发/              # 开发文档分类
│   │   ├── API文档.md
│   │   ├── 开发指南.md
│   │   └── 最佳实践.md
│   ├── 用户/              # 用户文档分类
│   │   ├── 使用手册.md
│   │   └── 常见问题.md
│   ├── 测试/              # 测试文档分类
│   │   ├── 测试指南.md
│   │   └── 测试报告.md
│   ├── API/               # API文档分类
│   │   └── API参考.md
│   └── changes/           # 版本更新记录
│       └── CHANGELOG.md
```

### 支持的文件格式

- `.md` - Markdown文件（推荐）
- `.markdown` - Markdown文件
- `.txt` - 纯文本文件

### 文档命名规范

- 使用描述性的中文名称
- 避免使用特殊字符
- 推荐格式：`功能名称-说明.md`

## 🎨 前端界面功能

### 1. 搜索功能

- **关键词搜索** - 在顶部搜索框输入关键词
  - 搜索范围：标题、摘要、内容
  - 实时搜索（输入后300ms触发）

- **模块过滤** - 通过下拉菜单筛选特定模块
  - 显示每个模块的文档数量
  - 支持"所有模块"选项

### 2. 主题切换

点击右上角的月亮/太阳图标可以切换主题：
- 🌙 **明亮主题** - 适合白天使用
- ☀️ **暗黑主题** - 适合夜间使用

主题设置会自动保存到浏览器，下次访问时自动应用。

### 3. 文档浏览

- **卡片视图** - 文档以卡片形式展示，包含：
  - 标题
  - 模块标签
  - 摘要（前200字符）
  - 文件名
  - 是否自动导入标记

- **详情页面** - 点击卡片查看完整文档
  - Markdown完美渲染
  - 语法高亮
  - 表格支持
  - 代码块支持

### 4. 响应式设计

- **桌面端** - 完整功能，侧边栏导航
- **平板端** - 适配的布局
- **手机端** - 移动优化视图

## 🔧 API接口文档

### 1. 获取模块列表

```http
GET /api/dev/document/modules
```

**响应示例**:
```json
{
  "code": 0,
  "msg": "success",
  "data": [
    {
      "name": "Weline_Framework",
      "display_name": "Weline / Framework",
      "doc_count": 25
    }
  ]
}
```

### 2. 搜索文档

```http
GET /api/dev/document/search?keyword=xxx&module=xxx&page=1&size=20
```

**参数说明**:
- `keyword` - 搜索关键词（可选）
- `module` - 模块名称（可选，"all"表示所有模块）
- `page` - 页码（默认1）
- `size` - 每页数量（默认20，最大100）

**响应示例**:
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "items": [...],
    "pagination": {
      "total": 150,
      "page": 1,
      "pageSize": 20,
      "totalPages": 8
    }
  }
}
```

### 3. 获取文档详情

```http
GET /api/dev/document/detail?id=123
```

**响应示例**:
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "id": 123,
    "title": "API文档",
    "content": "...",
    "module_name": "Weline_Ai",
    "module_display": "Weline / Ai",
    "category": {...}
  }
}
```

### 4. 获取目录树

```http
GET /api/dev/document/catalogs
```

## 💾 数据库结构

这些表属于运行时索引和状态数据，使用用户当前配置的主库连接。模块自动导入文档的正文以模块 `doc/` 文件为事实源，数据库中的 `content` 只作为历史数据或搜索缓存使用。

### developer_workspace_document 表

模块自动导入文档以 `(module_name, file_path)` 作为唯一源文件标识。历史版本如果已经写入重复记录，升级创建唯一索引时会保留最早的 `id`，删除同一源文件标识下的后续重复记录；后续扫描会按模块源文件重新刷新保留记录的标题、摘要和文件状态。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| title | VARCHAR(500) | 标题 |
| summary | VARCHAR(1000) | 摘要 |
| content | LONGTEXT | 内容 |
| category_id | INTEGER | 分类ID |
| author_id | INTEGER | 作者ID |
| **module_name** | VARCHAR(100) | **所属模块** |
| **file_path** | VARCHAR(500) | **文件路径** |
| **file_name** | VARCHAR(200) | **文件名** |
| **is_auto_imported** | INTEGER | **是否自动导入** |
| **sort_order** | INTEGER | **排序** |

### developer_workspace_document_catalog 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| name | VARCHAR(60) | 目录名 |
| description | TEXT | 简介 |
| pid | INTEGER | 父目录ID |
| level | INTEGER | 层级 |
| position | INTEGER | 排序 |
| is_active | INTEGER | 是否激活 |
| **is_system** | INTEGER | **是否系统创建** |

### frontend_user_config 表

| 字段 | 类型 | 说明 |
|------|------|------|
| config_id | INTEGER | 主键 |
| user_id | INTEGER | 用户ID |
| config_key | VARCHAR(100) | 配置键 |
| config_value | TEXT | 配置值 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

## 🎓 最佳实践

### 1. 文档编写建议

- **标题层级** - 使用合理的标题层级（H1-H6）
- **代码示例** - 提供可运行的代码示例
- **图片** - 使用相对路径引用图片
- **链接** - 使用相对路径链接其他文档
- **目录** - 为长文档添加目录导航

### 2. 目录组织建议

- **README.md** - 每个模块必须有README作为入口
- **分类清晰** - 按功能分类存放文档
- **命名规范** - 使用统一的命名格式
- **版本记录** - 在changes目录记录版本更新

### 3. 更新流程

```bash
# 1. 在模块的doc目录添加或修改文档

# 2. 重新扫描文档
php bin/w doc:import

# 3. 在浏览器中验证
# 访问 /dev/tool/docs 查看效果
```

### 4. 自动化建议

可以在模块升级时自动触发文档扫描：

```php
// 在 Observer 中
public function execute(Event $event): void
{
    $scanner = ObjectManager::getInstance(DocumentScanner::class);
    $scanner->scanAllModules();
}
```

## 🐛 故障排查

### 问题1：文档没有被扫描到

**检查清单**:
1. 确认文档在 `doc` 目录下
2. 确认文件扩展名是 `.md` 或 `.txt`
3. 运行 `php bin/w doc:import --force` 强制重新扫描
4. 查看命令行输出的统计信息

### 问题2：中文文档乱码

**解决方案**:
- 确保文档文件使用 UTF-8 编码
- 检查数据库字符集设置

### 问题3：搜索不到文档

**检查清单**:
1. 确认文档已成功导入（查看数据库）
2. 检查搜索关键词是否正确
3. 尝试切换到"所有模块"过滤器
4. 清空搜索框查看所有文档

### 问题4：主题不保存

**解决方案**:
- 清除浏览器缓存
- 检查浏览器是否支持 localStorage
- 查看浏览器控制台是否有错误

## 📚 扩展开发

### 添加新的文档类型

修改 `DocumentScanner::isDocumentFile()`:

```php
private function isDocumentFile(string $fileName): bool
{
    $extension = strtolower(pathinfo($fileName, PATHINFO_EXTENSION));
    return in_array($extension, ['md', 'markdown', 'txt', 'rst', 'asciidoc']);
}
```

修改前端视图的 `renderMarkdown()` 函数：

```javascript

    // 添加自定义渲染逻辑
    marked.setOptions({

            // 使用 highlight.js 或其他库
        }
    });
}
```
