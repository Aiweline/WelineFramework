# Weline_GenerativeEngineOptimization 模块使用指南

## 📖 概述

Weline_GenerativeEngineOptimization（生成式搜索引擎优化）模块专门向AI生成式搜索引擎提供Feed，支持多平台、密钥管理、一键推送和自动更新推送功能。

### 主要功能

- **多平台支持**：支持 Google SGE、Perplexity、Bing Chat、OpenAI、Claude 等主流AI搜索引擎
- **密钥加密存储**：使用 AES-256-CBC 加密算法安全存储API密钥
- **Feed管理**：支持创建和管理多种类型的Feed（内容、产品、文章等）
- **一键推送**：手动、自动、定时三种推送方式
- **推送日志**：完整的推送历史记录和错误追踪

---

## 🚀 快速开始

### 1. 安装模块

```bash
php bin/m module:upgrade Weline_GenerativeEngineOptimization
```

### 2. 配置环境变量（可选但推荐）

在 `app/etc/env.php` 中添加：

```php
return [
    // ... 其他配置
    'geo_secret_key' => 'your-secret-key-here', // 用于加密API密钥的主密钥
];
```

**安全提示**：生产环境必须配置强密钥，建议使用随机生成的32位以上字符串。

## 📋 使用流程

### 第一步：配置平台

#### 1.1 添加平台

1. 进入后台：**生成式搜索引擎优化 > 平台管理**
2. 点击 **"添加平台"** 按钮
3. 填写平台信息：
   - **平台代码**：选择平台（google_sge、perplexity、bing_chat、openai、claude）
   - **平台名称**：显示名称
   - **API端点**：平台的API端点URL（可选，部分平台有默认值）
   - **Feed格式**：选择 Feed 格式（JSON Feed、XML、RSS）
   - **是否启用**：启用该平台
   - **配置（JSON）**：平台特定配置（可选）

4. 点击 **"保存"**

#### 1.2 添加平台账户

1. 在平台列表中，点击 **"账户"** 按钮
2. 点击 **"添加账户"** 按钮
3. 填写账户信息：
   - **账户名称**：用于识别的名称
   - **API密钥**：平台的API密钥（会自动加密存储）
   - **API密钥（可选）**：如果需要第二个密钥
   - **设为默认账户**：是否设为该平台的默认账户
   - **激活**：是否激活该账户

4. 点击 **"保存"**

#### 1.3 测试连接

1. 在账户列表中，点击 **"测试连接"** 按钮
2. 系统会自动测试API密钥是否有效
3. 测试结果会显示在连接状态中

---

### 第二步：创建Feed

#### 2.1 添加Feed

1. 进入：**生成式搜索引擎优化 > Feed管理**
2. 点击 **"添加Feed"** 按钮
3. 填写Feed信息：

   **基本信息**：
   - **Feed名称**：Feed的显示名称
   - **Feed类型**：选择类型（内容、产品、文章、自定义）
   - **数据源类型**：选择数据源（数据库、API、自定义）
   - **数据源配置（JSON）**：配置数据源参数

   **Feed设置**：
   - **Feed URL**：Feed的访问URL（用于外部访问）
   - **更新频率**：选择更新频率（实时、每小时、每天、每周）
   - **自动推送**：是否在内容更新时自动推送
   - **是否启用**：启用该Feed

   **高级配置**：
   - **配置（JSON）**：Feed的额外配置

4. 点击 **"保存"**

#### 2.2 配置数据源

数据源配置使用JSON格式，根据不同的数据源类型有不同的配置：

**数据库数据源示例**：
```json
{
  "table": "cms_page",
  "fields": {
    "title": "title",
    "content": "content",
    "url": "url",
    "published_at": "created_at"
  },
  "where": {
    "status": 1
  },
  "order": "created_at DESC",
  "limit": 100
}
```

**API数据源示例**：
```json
{
  "endpoint": "https://api.example.com/content",
  "method": "GET",
  "headers": {
    "Authorization": "Bearer token"
  },
  "params": {
    "type": "article"
  }
}
```

---

### 第三步：推送Feed

#### 3.1 手动推送（一键推送）

1. 进入：**生成式搜索引擎优化 > 推送管理 > 一键推送**
2. 选择要推送的 **Feed**
3. 选择要推送到的 **平台**（可多选）
4. 点击 **"开始推送"** 按钮
5. 系统会显示推送结果

#### 3.2 自动推送

1. 在创建或编辑Feed时，启用 **"自动推送"** 选项
2. 设置 **"更新频率"** 为 **"实时"**
3. 当内容更新时，系统会自动：
   - 生成Feed条目
   - 推送到所有启用的平台

#### 3.3 定时推送

使用命令行工具定时推送：

```bash
# 推送到指定平台
php bin/m geo:feed:push 1 1

# 推送到所有平台
php bin/m geo:feed:push 1
```

可以结合Cron任务实现定时推送：

```bash
# 每天凌晨2点推送Feed ID为1的内容
0 2 * * * php /path/to/bin/m geo:feed:push 1
```

## 🔧 高级功能

### 生成Feed

#### 使用命令行生成

```bash
# 生成JSON Feed格式
php bin/m geo:feed:generate 1 json_feed

# 生成XML格式
php bin/m geo:feed:generate 1 xml

# 生成RSS格式
php bin/m geo:feed:generate 1 rss
```

#### Feed格式说明

**JSON Feed 1.1**：
- 符合 JSON Feed 标准
- 现代Web应用推荐格式
- 支持结构化数据

**XML/RSS**：
- 传统RSS格式
- 兼容性更好
- 支持RSS 2.0标准

## 🔐 安全说明

### API密钥加密

- 所有API密钥使用 **AES-256-CBC** 加密算法存储
- 加密密钥从环境配置 `geo_secret_key` 获取
- 密钥仅在需要时解密，不会在日志中暴露

### 安全建议

1. **生产环境必须配置强密钥**：
   ```php
   'geo_secret_key' => bin2hex(random_bytes(32)), // 64位十六进制字符串
   ```

2. **定期轮换密钥**（需要重新加密所有密钥）

3. **限制后台访问权限**：使用ACL控制访问

4. **监控推送日志**：及时发现异常推送

## 📊 平台配置说明

### Google SGE

- **平台代码**：`google_sge`
- **API端点**：`https://indexing.googleapis.com/v3/urlNotifications:publish`
- **Feed格式**：推荐 JSON Feed
- **认证方式**：Bearer Token
- **说明**：使用 Google Indexing API，需要逐个推送URL

### Perplexity

- **平台代码**：`perplexity`
- **API端点**：`https://api.perplexity.ai/feeds`
- **Feed格式**：JSON Feed
- **认证方式**：Bearer Token

### Bing Chat

- **平台代码**：`bing_chat`
- **API端点**：`https://ssl.bing.com/webmaster/api.svc/json/SubmitUrlbatch`
- **Feed格式**：JSON Feed
- **认证方式**：API Key
- **说明**：需要配置站点URL

### OpenAI

- **平台代码**：`openai`
- **API端点**：`https://api.openai.com/v1/feeds`
- **Feed格式**：JSON Feed
- **认证方式**：Bearer Token

### Claude

- **平台代码**：`claude`
- **API端点**：`https://api.anthropic.com/v1/feeds`
- **Feed格式**：JSON Feed
- **认证方式**：x-api-key Header

## 🐛 故障排查

### 问题1：推送失败

**可能原因**：
- API密钥无效或过期
- 网络连接问题
- 平台API端点变更
- Feed格式不符合平台要求

**解决方法**：
1. 检查API密钥是否正确
2. 使用"测试连接"功能验证
3. 查看推送日志中的错误信息
4. 检查网络连接和防火墙设置

### 问题2：Feed生成失败

**可能原因**：
- 数据源配置错误
- 数据库连接问题
- API数据源认证失败

**解决方法**：
1. 检查数据源配置JSON格式是否正确
2. 验证数据库连接
3. 测试API数据源是否可访问

### 问题3：自动推送不工作

**可能原因**：
- Observer事件未正确配置
- Feed未启用自动推送
- 更新频率设置不正确

**解决方法**：
1. 检查事件配置（event.xml）
2. 确认Feed已启用自动推送
3. 设置更新频率为"实时"
