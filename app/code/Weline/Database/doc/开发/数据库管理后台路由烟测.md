# Database Admin Backend Route Smoke

## 2026-05-26 Fix Record

- Problem: backend menu actions or template API URLs used a route prefix that did not match `Weline_Database/etc/env.php`, so `setup:upgrade --route` could generate routes that made the menu target return 404.
- Fix: keep `router` and `backend_router` in `Weline_Database/etc/env.php` as `database`; use `database/backend/admin/...` for backend menu actions and template API URLs.
- Smoke target: `database/backend/admin/index` and SQL tab.

## Verification

```bash
php bin/w setup:upgrade --module Weline_Database
php bin/w setup:upgrade --route
php bin/w http:request database/backend/admin/index -b
php bin/w e2e:run --module=Weline_Database --case-id=DATABASE-SMOKE-001 --project=chromium --headless
```
