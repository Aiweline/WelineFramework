# Weline_Database 数据库迁移系统使用手册

## 简介

Weline_Database模块提供了强大的数据库迁移管理功能，支持版本控制、数据备份、安全回滚。即使删除表结构也能完整恢复数据。

## 快速开始

### 1. 模块安装

数据库迁移模块已集成在Weline框架中，无需额外安装。

### 2. 初始化数据库

首次使用前，需要初始化迁移系统：

```bash
# 安装迁移系统数据库表
php bin/w setup:upgrade
```

### 3. 基本使用

#### 查看迁移状态
```bash
php bin/w db:migrate:status --module=Weline_Ai
```

#### 升级迁移
```bash
php bin/w db:migrate:upgrade --module=Weline_Ai --file=create_table__users_20250101-v1.0.0.php
```

#### 回滚迁移
```bash
php bin/w db:migrate:rollback --module=Weline_Ai --file=create_table__users_20250101-v1.0.0.php
```

## 迁移脚本配置

### 1. 创建迁移目录

在您的模块下创建迁移目录：

```
app/code/YourModule/Setup/Db/Migration/
```

### 2. 迁移文件命名

使用语义化命名规范：

```
{action}__{description}_{date}-{version}.php
```

#### 命名示例
- `create_table__users_20250101-v1.0.0.php` - 创建用户表
- `add_column__email_20250102-v1.0.1.php` - 添加邮箱字段
- `drop_column__raw_data_20250103-v1.0.2.php` - 删除原始数据字段
- `modify_column__status_20250104-v1.0.3.php` - 修改状态字段
- `create_index__email_20250105-v1.0.4.php` - 创建邮箱索引

```php

public function install(): bool
```

### 3. 删除字段（带数据备份）

```php
private array $backupData = [];

public function install(): bool
{
    $connection = $this->connectionFactory->getConnection();
    
    // 备份数据
    $this->backupData = $this->backupColumnData('users', 'raw_data');
    
    // 删除字段
    $connection->deleteColumn('users', 'raw_data');
    
    return true;
}

public function uninstall(): bool
{
    $connection = $this->connectionFactory->getConnection();
    
    // 恢复字段
    $connection->addColumn(
        'users',
        'raw_data',
        TableInterface::column_type_TEXT,
        0,
        'null',
        'Raw Data'
    );
    
    // 恢复数据
    $this->restoreColumnData('users', 'raw_data', $this->backupData);
    
    return true;
}

private function backupColumnData(string $table, string $column): array
{
    $connection = $this->connectionFactory->getConnection();
    $query = $connection->select()
        ->from($table, ['id', $column])
        ->where("{$column} IS NOT NULL");
    
    return $connection->fetchAll($query);
}

private function restoreColumnData(string $table, string $column, array $data): void
{
    if (empty($data)) {
        return;
    }
    
    $connection = $this->connectionFactory->getConnection();
    
    foreach ($data as $row) {
        $connection->update(
            $table,
            [$column => $row[$column]],
            ['id = ?' => $row['id']]
        );
    }
}
```

```php
        ['nullable' => true]

    $this->restoreColumnData('users', 'status', $this->backupData);
    
    return true;
}
```

```php
public function validate(): bool
{
    $connection = $this->connectionFactory->getConnection();
    
    // 检查表是否存在
    if (!$connection->isTableExists('users')) {
        return false;
    }
    
    // 检查列是否存在

    return isset($columns['email']);
}
```

## 独立备份与恢复命令

### db:migrate:backup — 手动备份

在执行迁移之前，可以单独创建数据备份：

```bash
# 备份模块下所有 requiresBackup 的 pending 迁移涉及的数据
php bin/w db:migrate:backup --module=Weline_Ai

# 备份指定迁移文件涉及的数据
php bin/w db:migrate:backup --module=Weline_Ai --file=drop_column__raw_data_20250101-v1.0.1.php

# 直接备份指定表（手动模式，无需迁移文件）
php bin/w db:migrate:backup --tables=users,orders
```

命令执行后会输出 `migration_id`，可用于后续恢复。

### db:migrate:restore — 恢复备份

```bash
# 按 backup_id 恢复单条备份记录
php bin/w db:migrate:restore --backup-id=123

# 按 migration_id 恢复该迁移的所有关联备份
php bin/w db:migrate:restore --migration-id=5
```

### AbstractMigration 抽象基类

新迁移脚本推荐继承 `Weline\Database\AbstractMigration`，只需实现 `install()` 和 `uninstall()`，其余方法均有默认实现：

```php
<?php
namespace YourVendor\YourModule\Setup\Db\Migration;

use Weline\Database\AbstractMigration;

class DropColumn__RawData_20250101_v1_0_1 extends AbstractMigration
{
    public function install(): bool
    {
        // 迁移逻辑
        return true;
    }

    public function uninstall(): bool
    {
        // 回滚逻辑
        return true;
    }

    public function getDescription(): string { return '删除 raw_data 字段'; }
    public function getVersion(): string { return '1.0.1'; }
    public function getDate(): string { return '20250101'; }
    public function getType(): string { return 'drop_column'; }
    public function getAffectedTables(): array { return ['my_table']; }
    public function requiresBackup(): bool { return true; }
    public function getBackupStrategy(): array
    {
        return [
            'strategy' => 'column',
            'tables'   => ['my_table'],
            'columns'  => ['raw_data'],
        ];
    }
}
```

执行 `db:migrate:upgrade` 时，若 `requiresBackup()` 返回 `true`，系统会在 `install()` 之前自动按 `getBackupStrategy()` 备份；回滚时自动恢复。
