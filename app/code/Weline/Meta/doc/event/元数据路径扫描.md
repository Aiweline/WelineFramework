# Weline Meta 模块 - 事件文档

## 概述

本文档详细说明了 Weline Meta 模块提供的事件及其使用方法。这些事件允许其他模块与 Meta 模块进行交互，实现跨模块的功能扩展。

## 事件列表

### 1. Weline_Meta::scan_path - 元数据路径扫描事件

#### 基本信息

- **事件名称**：`Weline_Meta::scan_path`
- **事件类型**：元数据扫描
- **触发模块**：`Weline_Meta`
- **服务类**：`Weline\Meta\Service\Scanner`

#### 功能说明

用于触发元数据路径扫描事件，允许其他模块监听并处理扫描任务。扫描器会扫描指定路径下的文件，提取元数据信息（如 `@meta.json` 文件）并存储到数据库中。

#### 使用场景

- 扫描主题文件中的元数据
- 扫描模块视图文件中的元数据
- 批量更新元数据信息
- 自动发现和注册元数据配置

#### 使用方法

##### 基本用法

```php
use Weline\Framework\Event\EventsManager;
use Weline\Framework\Manager\ObjectManager;

// 获取事件管理器实例
/** @var EventsManager $eventsManager */
$eventsManager = ObjectManager::getInstance(EventsManager::class);

// 触发扫描事件
$eventData = [
    'scan_path' => 'Weline_Theme::view/theme',  // 扫描路径，格式：模块名::相对路径
    'namespace' => 'theme',                     // 命名空间，用于分类元数据
    'strict_mode' => true                       // 严格模式，true=只扫描有@meta.json的文件
];
$eventsManager->dispatch('Weline_Meta::scan_path', $eventData);
```

#### 数据格式说明

| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| scan_path | string | 是 | 扫描路径，格式：`模块名::相对路径`，例如：`Weline_Theme::view/theme` | - |
| namespace | string | 否 | 命名空间，用于分类和过滤元数据 | 'theme' |
| strict_mode | bool | 否 | 严格模式，true=只扫描包含@meta.json的文件，false=扫描所有文件 | true |

#### 路径格式说明

扫描路径使用 `模块名::相对路径` 格式：

- **模块名**：使用下划线格式，例如 `Weline_Theme`、`Weline_Admin`
- **相对路径**：相对于模块根目录的路径，使用正斜杠 `/` 分隔
- **示例**：
  - `Weline_Theme::view/theme` - 扫描主题视图目录
  - `Weline_Admin::view/templates` - 扫描管理后台模板目录
  - `Weline_Meta::view/templates` - 扫描Meta模块模板目录

#### 使用示例

##### 示例 1：扫描主题文件

```php
// 在控制器中触发扫描
public function scanTheme(): void
{
    /** @var EventsManager $eventsManager */
    $eventsManager = ObjectManager::getInstance(EventsManager::class);
    
    $eventData = [
        'scan_path' => 'Weline_Theme::view/theme',
        'namespace' => 'theme',
        'strict_mode' => true
    ];
    $eventsManager->dispatch('Weline_Meta::scan_path', $eventData);
    
    $this->getMessageManager()->addSuccess(__('主题扫描任务已提交'));
}
```

##### 示例 2：扫描模块视图

```php
// 扫描指定模块的视图文件
public function scanModuleViews(string $moduleName): void
{
    /** @var EventsManager $eventsManager */
    $eventsManager = ObjectManager::getInstance(EventsManager::class);
    
    $eventData = [
        'scan_path' => $moduleName . '::view/templates',
        'namespace' => 'module',
        'strict_mode' => false  // 扫描所有文件，不限制@meta.json
    ];
    $eventsManager->dispatch('Weline_Meta::scan_path', $eventData);
}
```

##### 示例 3：批量扫描多个路径

```php
// 批量扫描多个路径
public function scanMultiplePaths(): void
{
    /** @var EventsManager $eventsManager */
    $eventsManager = ObjectManager::getInstance(EventsManager::class);
    
    $paths = [
        ['scan_path' => 'Weline_Theme::view/theme', 'namespace' => 'theme'],
        ['scan_path' => 'Weline_Admin::view/templates', 'namespace' => 'admin'],
        ['scan_path' => 'Weline_Meta::view/templates', 'namespace' => 'meta']
    ];
    
    foreach ($paths as $path) {
        $eventData = [
            'scan_path' => $path['scan_path'],
            'namespace' => $path['namespace'],
            'strict_mode' => true
        ];
        $eventsManager->dispatch('Weline_Meta::scan_path', $eventData);
    }
}
```

##### 示例 4：在控制器中使用（完整示例）

```php
<?php

namespace Weline\Meta\Controller\Backend;

use Weline\Framework\App\Controller\BackendController;
use Weline\Framework\Event\EventsManager;
use Weline\Framework\Manager\ObjectManager;

class Scanner extends BackendController
{
    /**
     * 执行扫描（通过事件）
     */
    public function scan()
    {
        $scanPath = $this->request->getPost('scan_path');
        $namespace = $this->request->getPost('namespace', 'theme');
        $strictMode = (bool)$this->request->getPost('strict_mode', true);
        
        if (empty($scanPath)) {
            $this->getMessageManager()->addError(__('请指定扫描路径'));
            $this->redirect('*/index');
            return;
        }
        
        try {
            // 触发扫描事件，让其他模块的观察者处理
            // 注意：dispatch 方法的第二个参数需要按引用传递，所以先创建变量
            $eventData = [
                'scan_path' => $scanPath,
                'namespace' => $namespace,
                'strict_mode' => $strictMode
            ];
            // 使用规范事件名 Weline_Meta::scan_path
            $this->getEventManager()->dispatch('Weline_Meta::scan_path', $eventData);
            
            $this->getMessageManager()->addSuccess(__('扫描任务已提交，请查看扫描结果'));
        } catch (\Exception $e) {
            $this->getMessageManager()->addError(__('扫描失败：%1', $e->getMessage()));
        }
        
        $this->redirect('*/index');
    }
}
```

#### 扫描结果说明

扫描完成后，结果会包含以下信息：

- **success**：成功扫描的文件列表
- **failed**：扫描失败的文件列表（包含错误信息）
- **skipped**：跳过的文件列表（如不符合条件）

#### 系统行为说明

##### 1. 扫描流程

1. 解析扫描路径，获取模块路径和相对路径
2. 遍历指定目录下的所有文件
3. 查找 `@meta.json` 文件（如果启用严格模式）
4. 解析元数据并存储到数据库
5. 返回扫描结果

##### 2. 元数据格式

扫描器会查找 `@meta.json` 文件，格式如下：

```json
{
    "type": "config",
    "identify": "unique-identifier",
    "namespace": "theme",
    "data": {
        "title": "配置标题",
        "description": "配置描述"
    }
}
```

##### 3. 错误处理

- 如果扫描路径不存在，会记录错误但不会中断流程
- 如果文件格式错误，会跳过该文件并记录错误
- 所有错误信息都会包含在返回结果中

#### 注意事项

1. **路径格式**：必须使用 `模块名::相对路径` 格式，否则无法正确解析
2. **模块名格式**：使用下划线格式，例如 `Weline_Theme`，不是 `Weline\Theme`
3. **严格模式**：启用严格模式时，只扫描包含 `@meta.json` 的文件，可以提高扫描效率
4. **命名空间**：建议使用有意义的命名空间，便于后续查询和过滤
5. **性能考虑**：扫描大量文件可能耗时较长，建议在后台任务中执行
6. **事件数据**：事件数据必须按引用传递，不能直接传递数组字面量

#### 故障排查

##### 问题：扫描后没有结果

**可能原因：**
1. 扫描路径格式不正确
2. 模块不存在或路径不存在
3. 严格模式下没有找到 `@meta.json` 文件

**解决方法：**
1. 检查路径格式是否为 `模块名::相对路径`
2. 确认模块已正确安装
3. 检查目标目录是否存在
4. 如果使用严格模式，确认文件包含 `@meta.json`

##### 问题：扫描失败

**可能原因：**
1. 文件权限不足
2. 文件格式错误
3. 数据库连接问题

**解决方法：**
1. 检查文件读取权限
2. 验证 `@meta.json` 文件格式是否正确
3. 检查数据库连接和表结构
4. 查看错误日志获取详细信息

4. **进度回调**：添加进度回调来显示扫描进度
5. **结果通知**：扫描完成后发送通知消息
