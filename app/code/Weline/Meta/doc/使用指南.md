# Weline_Meta 模块使用指南

## 概述

Weline_Meta 模块用于统一管理系统中各种文件的元数据，支持多语言翻译。主要用于主题模板文件的元数据管理。

**重要提示**：在使用 Meta 功能之前，必须先定义 `@meta.json` 规约文件，Meta 模块会自动扫描这些文件来构建元数据结构。

---

## 一、定义 @meta.json 规约文件（必需）

### 1.1 规约文件位置

在使用 Meta 功能之前，必须先定义 `@meta.json` 规约文件。规约文件使用 **Extends 机制** 放置在模块的 `extends` 目录下：

**路径格式**：`app/code/{ModuleName}/extends/Weline_Meta/{ModuleName}/@meta.json`

**示例**：
- Theme 模块：`app/code/Weline/Theme/extends/Weline_Meta/Weline_Theme/@meta.json`
- 其他模块：`app/code/{YourModule}/extends/Weline_Meta/{YourModule}/@meta.json`

### 1.2 规约文件格式

`@meta.json` 文件使用 JSON 格式，定义元数据的层级结构和默认值：

```json
{
  "meta": {
    "base_path": "Weline_Theme::view/theme/",
    "theme": {
      "default": "Weline_Theme",
      "name": "主题命名空间",
      "description": "主题命名空间，用于主题模块的命名空间",
      "frontend": {
        "default": "前端组",
        "name": "前端组",
        "description": "前端组，用于给元素据分组用的，方便维护",
        "layout": {
          "name": "布局命名空间",
          "description": "布局命名空间，用于描述布局的名称",
          "options": {},
          "values": [],
          "default": "default",
          "type": "select",
          "required": true,
          "placeholder": "请选择布局",
          "show": true
        },
        "partials": {
          "name": "部件",
          "description": "部件命名空间，用于描述部件的名称",
          "options": {},
          "values": [],
          "default": "default",
          "type": "select",
          "required": true,
          "placeholder": "请选择部件",
          "show": true
        }
      }
    }
  }
}
```

### 1.3 字段说明

- **`base_path`**：基础路径，用于扫描目录下的文件
- **`default`**：默认值
- **`name`**：显示名称
- **`description`**：描述信息
- **`options`**：选项列表（对象格式，键值对）
- **`values`**：选项列表（数组格式）
- **`type`**：字段类型（`select`、`text`、`textarea` 等）
- **`required`**：是否必填
- **`placeholder`**：占位符文本
- **`show`**：是否显示

### 1.4 层级结构

规约文件支持层级结构，**拥有子元素的节点被认为是组（group）**：

```json
{
  "meta": {
    "theme": {                    // 组：theme
      "frontend": {               // 组：frontend
        "layout": {               // 值：layout
          "account": {            // 值：account（如果目录下有 account 目录，会自动扫描）
            "name": "个人中心布局",
            "description": "个人中心相关布局"
          }
        }
      }
    }
  }
}
```

### 1.5 自动扫描机制

Meta 模块会自动扫描 `base_path` 指定的目录，将目录结构转换为 options，**结果存储在 `meta` 表中，不会回填到 `@meta.json` 文件**：

**目录结构**：
```
view/theme/frontend/layouts/
  ├── account/
  │   ├── auth.phtml
  │   ├── dashboard.phtml
  │   └── profile.phtml
  ├── homepage/
  │   └── default.phtml
  └── product/
      └── list.phtml
```

**自动生成的 options（存储在数据库中）**：
```json
{
  "layout": {
    "options": {
      "account": "个人中心布局",
      "homepage": "首页布局",
      "product": "产品布局"
    }
  }
}
```

**重要说明**：
- `@meta.json` 是只读的规约文件，不会被修改
- 扫描结果存储在 `meta` 表中，不会回填到 `@meta.json` 文件
- 可以通过后端界面查看和管理存储在数据库中的元数据

### 1.6 使用示例

定义规约后，可以在模板文件中使用：

```php
@meta::theme.frontend.layout.account
@meta::theme.frontend.layout.account.name
@meta::theme.frontend.layout.account.description
```

### 1.7 默认选中机制

当 meta 渲染时，如果提供了布局参数，系统会自动选中对应的选项：

```php
// 如果当前布局是 account/auth，会自动选中 account
<w:meta type="select" prefix="theme.frontend.layout" layout="account/auth" />
```

## 二、在模板文件中定义元数据

### 1.1 元数据格式

在模板文件（`.phtml`）的开头，使用以下格式定义元数据：

```php
<?php
/**
 * ============================================
 * 元数据区域 (Metadata Block)
 * ============================================
 * 
 * @meta.namespace theme                    # 命名空间：theme, module, api等
 * @meta.theme.type layout                 # 类型：layout, component, partial等
 * @meta.theme.area frontend               # 区域：frontend, backend（可选）
 * @meta.theme.category account            # 分类：account, product, cart等（可选）
 * @meta.theme.identify auth               # 标识：auth, default, 1280等（文件名或ID）
 * 
 * @meta.theme.info.name 名称：个人中心认证页面布局
 * @meta.theme.info.description 描述：登录/注册等认证页面的专用布局
 * @meta.theme.info.icon 图标：mdi mdi-account-lock
 * @meta.theme.info.version 版本：1.0.0
 * @meta.theme.info.author 作者：Weline Team
 * 
 * @param title 页面标题
 * @param content 认证表单内容（HTML字符串）
 * @param class 额外CSS类
 */
```

### 1.2 必需字段

- `@meta.namespace` - 命名空间（必填）
- `@meta.{namespace}.type` - 类型（必填）
- `@meta.{namespace}.info.name` - 名称（必填）

### 1.3 可选字段

- `@meta.{namespace}.area` - 区域
- `@meta.{namespace}.category` - 分类
- `meta.{namespace}.identify` - 标识
- `@meta.{namespace}.info.description` - 描述
- `@meta.{namespace}.info.icon` - 图标
- `@meta.{namespace}.info.version` - 版本
- `@meta.{namespace}.info.author` - 作者

### 1.4 示例

**布局文件示例** (`app/code/Weline/Theme/view/theme/frontend/layouts/account/auth.phtml`)：

```php
<?php
/**
 * ============================================
 * 元数据区域 (Metadata Block)
 * ============================================
 * 
 * @meta.namespace theme
 * @meta.theme.type layout
 * @meta.theme.area frontend
 * @meta.theme.category account
 * @meta.theme.identify auth
 * 
 * @meta.theme.info.name 名称：个人中心认证页面布局
 * @meta.theme.info.description 描述：登录/注册等认证页面的专用布局
 * @meta.theme.info.icon 图标：mdi mdi-account-lock
 */
```

**组件文件示例** (`app/code/Weline/Theme/view/theme/frontend/components/button.phtml`)：

```php
<?php
/**
 * ============================================
 * 元数据区域 (Metadata Block)
 * ============================================
 * 
 * @meta.namespace theme
 * @meta.theme.type component
 * @meta.theme.area frontend
 * @meta.theme.identify button
 * 
 * @meta.theme.info.name 名称：按钮组件
 * @meta.theme.info.description 描述：通用按钮组件，支持多种样式和尺寸
 * @meta.theme.info.icon 图标：mdi mdi-button-cursor
 */
```

## 三、使用Meta标签读取数据

### 3.1 基本用法

在模板中使用 `@meta{}` 格式读取元数据、配置值或翻译值：

```php
<!-- 读取配置值 -->
@meta{theme.frontend.partials.header.value|default}

<!-- 读取元数据信息 -->
@meta{theme.backend.components.info.name|组件名称}

<!-- 读取翻译值 -->
@meta{theme.layout.account_auth.info.name.lang|个人中心认证页面布局}
```

### 3.2 格式说明

**格式**：`@meta{键名|默认值}`

**说明**：
- **键名**：根据后缀自动识别读取类型（`.value`、`.info`、`.lang` 或完整 metaIdentify）
- **默认值**：直接使用 `|` 后面的内容，**不需要引号包裹**，如果找不到值则显示默认值
- **分隔符**：使用 `|` 分隔键名和默认值

### 3.3 简化格式

键名可以省略 `@meta::` 前缀，系统会自动添加（仅对完整 metaIdentify 格式有效）：

```php
<!-- 完整格式 -->
@meta{@meta::theme.layout.account_auth.info.name|个人中心认证页面布局}

<!-- 简化格式（推荐） -->
@meta{theme.layout.account_auth.info.name|个人中心认证页面布局}
```

### 3.4 默认值

如果找不到值，会显示默认值：

```php
<!-- 默认值不需要引号 -->
@meta{theme.layout.account_auth.info.name|个人中心认证页面布局}

<!-- 默认值可以包含特殊字符 -->
@meta{theme.layout.account_auth.info.name|个人中心认证页面布局（登录/注册）}

<!-- 没有默认值（找不到值时返回空） -->
@meta{theme.layout.account_auth.info.name}
```

**注意**：默认值不需要引号包裹，这样可以避免在外部有引号的情况下出现语法错误。

### 3.5 Meta读取规则（重要）

Meta 读取规则非常简单，**通过后缀自动识别读取类型**，直接返回对应的值。

**自动补全前缀**：如果键名不包含 `theme.` 或 `frontend/backend`，系统会自动补全：
- 自动添加 `theme.` 前缀
- 根据当前请求自动判断前后端（前端或后端），自动添加 `frontend` 或 `backend`

**示例**：
```php
<!-- 简化格式（自动补全） -->
@meta{partials.header.value|default}
// 前端请求时自动补全为：theme.frontend.partials.header.value
// 后端请求时自动补全为：theme.backend.partials.header.value

<!-- 完整格式（推荐） -->
@meta{theme.frontend.partials.header.value|default}
```

| 后缀 | 读取类型 | 数据源 | 返回值 | 使用场景 |
|------|---------|--------|--------|----------|
| `.value` | 配置值 | `MetaConfig` 表 | 字符串 | 读取主题配置、模块配置等 |
| `.info` | 元数据信息 | `Meta` 表 | 字符串 | 读取元数据的名称、描述等信息 |
| `.lang` | 翻译值 | `I18n Dictionary` 表 | 字符串 | 读取多语言翻译文本 |
| 无后缀 | 元数据对象 | `Meta` 表 | `MetaData` 对象 | 需要获取完整的元数据对象 |

#### 3.5.1 `.value` 后缀 - 读取配置值

**格式**：`{key}.value`

**说明**：从 `MetaConfig` 表读取配置值，**直接返回字符串**

**使用场景**：
- 读取主题配置（如头部样式、布局选项等）
- 读取模块配置（如功能开关、参数设置等）

**模板中使用**：
```php
<!-- 完整格式（推荐） -->
@meta{theme.frontend.partials.header.value|default}

<!-- 简化格式（自动补全前缀，根据当前请求判断前后端） -->
@meta{partials.header.value|default}
// 前端请求时自动补全为：theme.frontend.partials.header.value
// 后端请求时自动补全为：theme.backend.partials.header.value

<!-- 读取主题颜色配置 -->
@meta{theme.frontend.colors.primary.value|#007bff}
```

**PHP 代码中使用**：
```php
<?php
use Weline\Meta\Helper\MetaData;

// 直接返回字符串，不需要调用 get() 方法
$headerValue = MetaData::load('theme.frontend.partials.header.value');
// $headerValue 是字符串，如 "minimal" 或 null

// 带默认值处理
$headerValue = MetaData::load('theme.frontend.partials.header.value') ?? 'default';
?>
```

**完整示例**：
```php
<!-- 在模板中动态加载主题头部 -->
<w:theme:template layout="partials.header">
    Weline_Theme::theme/frontend/partials/header/@meta{theme.frontend.partials.header.value|default}.phtml
</w:theme:template>
```

#### 3.5.2 `.info` 后缀 - 读取元数据信息

**格式**：`{metaIdentify}.info.{path}`

**说明**：从 `Meta` 表读取元数据信息，**直接返回字符串**，支持多语言标签

**使用场景**：
- 读取元数据的名称、描述、图标等信息
- 读取组件、布局、部件的元数据信息

**模板中使用**：
```php
<!-- 读取元数据名称 -->
@meta{theme.backend.components.info.name|组件名称}

<!-- 读取元数据描述 -->
@meta{theme.backend.components.info.description|组件描述}

<!-- 读取元数据图标 -->
@meta{theme.backend.components.info.icon|mdi mdi-component}
```

**PHP 代码中使用**：
```php
<?php
use Weline\Meta\Helper\MetaData;

// 直接返回字符串，不需要调用 getLabel() 方法
$name = MetaData::load('theme.backend.components.info.name');
// $name 是字符串，如 "组件名称" 或 null

$description = MetaData::load('theme.backend.components.info.description');
// $description 是字符串，如 "通用组件" 或 null
?>
```

**路径说明**：
- `metaIdentify`：元数据标识，如 `theme.backend.components`
- `path`：信息路径，如 `info.name`、`info.description`、`info.icon` 等

#### 3.5.3 `.lang` 后缀 - 读取翻译值

**格式**：`{translationKey}.lang`

**说明**：从 `I18n Dictionary` 表读取翻译值，**直接返回字符串**，支持多语言回退

**使用场景**：
- 读取已翻译的文本内容
- 需要多语言支持的场景

**模板中使用**：
```php
<!-- 读取翻译值 -->
@meta{theme.layout.account_auth.info.name.lang|个人中心认证页面布局}

<!-- 读取翻译的描述 -->
@meta{theme.layout.account_auth.info.description.lang|登录/注册等认证页面的专用布局}
```

**PHP 代码中使用**：
```php
<?php
use Weline\Meta\Helper\MetaData;

// 直接返回翻译值（字符串），支持多语言回退
$translation = MetaData::load('theme.layout.account_auth.info.name.lang');
// $translation 是字符串，如 "个人中心认证页面布局" 或 null

// 多语言回退机制：
// 1. 优先使用当前语言的翻译
// 2. 如果当前语言没有翻译，回退到默认语言（zh_Hans_CN）
// 3. 如果都没有，返回 null
?>
```

**翻译键格式**：`@meta::{namespace}.{type}.{identify}.{group}.{field}`

**多语言回退机制**：
1. **优先使用当前语言的翻译**：根据用户当前语言环境获取翻译
2. **回退到默认语言**：如果当前语言没有翻译，自动使用 `zh_Hans_CN` 的翻译
3. **返回 null**：如果都没有翻译，返回 `null`

**与 `.info` 的区别**：
- `.info`：从 `Meta` 表读取元数据信息（存储在元数据中）
- `.lang`：从 `I18n Dictionary` 表读取翻译值（存储在翻译字典中）

#### 3.5.4 完整 metaIdentify 格式 - 返回 MetaData 对象

**格式**：`{namespace}.{type}.{identify}` 或 `{namespace}.{type}.{identify}.{group}.{field}`

**说明**：直接使用完整的 metaIdentify，从 `Meta` 表读取元数据，**返回 MetaData 对象**

**使用场景**：
- 需要获取完整的元数据对象
- 需要调用 `get()`、`getLabel()` 等方法
- 需要访问多个字段的值

**模板中使用**：
```php
<!-- 读取元数据标签（自动识别为完整格式） -->
@meta{theme.layout.account_auth.info.name|个人中心认证页面布局}
```

**PHP 代码中使用**：
```php
<?php
use Weline\Meta\Helper\MetaData;

// 返回 MetaData 对象
$metaData = MetaData::load('theme.frontend.layouts.default');

// 获取 setting 中的值
$title = $metaData->get('param.title'); // 或 $metaData->get('title')
$content = $metaData->get('param.content'); // 或 $metaData->get('content')

// 获取翻译后的标签信息
$name = $metaData->getLabel('info.name');
$description = $metaData->getLabel('info.description');

// 检查是否成功加载
if ($metaData->isLoaded()) {
    // 元数据加载成功
}

// 获取所有 setting 数据
$allData = $metaData->getAll();
?>
```

### 3.6 快速参考表

| 格式 | 示例 | 返回值 | 数据源 |
|------|------|--------|--------|
| `{key}.value` | `theme.frontend.partials.header.value` | 字符串 | `MetaConfig` 表 |
| `{metaIdentify}.info.{path}` | `theme.backend.components.info.name` | 字符串 | `Meta` 表 |
| `{translationKey}.lang` | `theme.layout.account_auth.info.name.lang` | 字符串 | `I18n Dictionary` 表 |
| `{metaIdentify}` | `theme.frontend.layouts.default` | `MetaData` 对象 | `Meta` 表 |

### 3.7 使用建议

1. **优先使用后缀格式**：`.value`、`.info`、`.lang` 直接返回字符串，使用更简单
2. **合理使用默认值**：为所有读取操作提供默认值，避免显示空内容
3. **区分使用场景**：
   - 配置值 → 使用 `.value`
   - 元数据信息 → 使用 `.info`
   - 翻译文本 → 使用 `.lang`
   - 完整对象 → 使用完整 metaIdentify

## 四、扫描元数据

### 4.1 自动扫描 @meta.json

Meta 模块在启动时会自动扫描所有模块的 `extends/Weline_Meta/{ModuleName}/@meta.json` 文件，加载规约定义。

### 4.2 扫描目录结构

Meta 模块会根据 `@meta.json` 中的 `base_path` 自动扫描目录结构，生成 options，**结果存储在 `meta` 表中**：

1. **读取规约**：读取 `@meta.json` 文件，获取规约定义
2. **扫描目录**：根据 `base_path` 指定的目录进行扫描
3. **识别子目录**：将子目录名称作为选项键
4. **生成 options**：如果子目录下有 `.phtml` 文件，自动添加到 options
5. **存储到数据库**：将扫描到的 options **存储到 `meta` 表中**，**不会回填到 `@meta.json` 文件**

**注意**：`@meta.json` 文件是只读的规约文件，所有扫描结果都存储在数据库中。

### 4.3 手动触发扫描

可以通过后端界面手动触发扫描：

1. 访问：**数据管理 > 元数据管理 > 扫描元数据**
2. 选择要扫描的模块
3. 点击"扫描"按钮

---

### 3.1 通过后端界面扫描

1. 访问后端菜单：**数据管理 > 元数据管理 > 扫描元数据**
2. 填写扫描路径，格式：`ModuleName::path`
   - 例如：`Weline_Theme::view/theme`
3. 选择命名空间（可选，默认使用文件中的命名空间）
4. 选择是否严格模式（默认开启）
5. 点击"扫描"按钮

### 3.2 扫描路径格式

**格式**：`ModuleName::相对路径`

**示例**：
- `Weline_Theme::view/theme` - 扫描Theme模块的theme目录
- `Weline_Theme::view/theme/frontend/layouts` - 只扫描前端布局目录
- `Weline_ModuleName::view/templates` - 扫描其他模块的模板目录

### 3.3 严格模式

- **开启**：文件必须符合元数据格式要求，不符合会报错
- **关闭**：不符合格式的文件会被跳过，不会报错

## 五、后端管理界面

### 4.1 元数据列表

访问：**数据管理 > 元数据管理 > 元数据列表**

功能：
- 查看所有元数据
- 按命名空间、类型筛选
- 按文件路径搜索
- 编辑元数据信息

1. 在元数据列表中点击"编辑"
2. 修改元数据信息
3. 点击"保存"

## 六、Theme模块集成

### 5.1 Theme模块特殊说明

Theme模块下的文件默认命名空间都是 `theme`，类型根据目录自动识别：
- `layouts/` → `layout`
- `components/` → `component`
- `partials/` → `partial`

### 5.2 Theme模块扫描

Theme模块可以通过以下方式扫描：

**方式1：通过后端界面**
- 扫描路径：`Weline_Theme::view/theme`
- 命名空间：`theme`（可选，文件中有定义）

**方式2：通过事件触发**
Theme模块可以注册观察者监听 `meta_scan_path` 事件，自动扫描。

## 七、常见问题

### Q0: 如何使用 @meta.json 规约文件？

**A**: 
1. 在模块的 `extends/Weline_Meta/{ModuleName}/` 目录下创建 `@meta.json` 文件
2. 定义元数据的层级结构和默认值
3. 设置 `base_path` 指定要扫描的目录
4. Meta 模块会自动扫描并加载规约
5. 在模板文件中使用 `@meta::` 格式引用定义的元数据

### Q0.1: @meta.json 中的 options 如何自动生成？

**A**: 
- Meta 模块会根据 `base_path` 扫描目录结构
- 子目录名称会自动作为 options 的键
- 如果子目录下有 `.phtml` 文件，会自动添加到 options
- 可以通过后端界面手动触发扫描更新 options

### Q0.2: 如何让 meta 渲染时默认选中某个选项？

**A**: 
- 在渲染 meta 标签时，提供 `layout` 参数
- 系统会根据 layout 参数自动匹配并选中对应的选项
- 例如：`<w:meta type="select" prefix="theme.frontend.layout" layout="account/auth" />`

---

### Q1: 元数据格式错误怎么办？

**A**: 检查以下几点：
1. 是否定义了 `@meta.namespace`
2. 是否定义了 `@meta.{namespace}.type`
3. 是否定义了 `@meta.{namespace}.info.name`
4. 命名空间是否正确（Theme模块必须是 `theme`）

### Q2: 翻译不显示怎么办？

**A**: 
1. 确认翻译键格式正确
2. 确认已扫描元数据
3. 确认I18n模块中已有对应语言的翻译
4. 检查默认值是否正确设置
5. 确认使用的是 `@meta{}` 格式，不是其他格式

### Q3: 如何批量扫描多个目录？

**A**: 可以多次扫描，或者通过事件机制让模块自己实现扫描逻辑。

### Q4: 元数据可以自定义字段吗？

**A**: 可以。除了标准的 `name`、`description`、`icon`、`version`、`author` 外，可以添加自定义字段，格式：`@meta.{namespace}.info.{自定义字段名}`

## 八、最佳实践

1. **先定义规约**：在使用 Meta 功能之前，先定义 `@meta.json` 规约文件
2. **只读文件**：`@meta.json` 是只读的规约文件，不会被修改，所有扫描结果存储在 `meta` 表中
3. **合理规划层级**：根据实际需求规划元数据的层级结构
4. **使用 base_path**：设置合理的 `base_path`，让系统自动扫描目录结构
5. **重新扫描**：当目录结构变化时，可以重新扫描，新的 options 会更新到数据库中

1. **统一命名空间**：同一模块使用相同的命名空间
2. **规范类型**：使用标准的类型（layout、component、partial）
3. **完整信息**：尽量填写完整的元数据信息，包括描述、图标等
4. **定期扫描**：修改模板文件后，及时扫描更新元数据
5. **翻译管理**：在后端I18n模块中管理翻译，确保多语言支持
