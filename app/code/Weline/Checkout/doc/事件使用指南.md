# Weline Checkout 事件使用指南

## 概述

Weline_Checkout 模块为支付模块和其他扩展模块提供了23个标准化的事件扩展点，允许在不修改核心代码的情况下扩展结账和支付功能。

**重要说明**：Checkout模块同时提供Hook系统和事件系统，两者用途不同，可以同时使用：
- **Hook系统**：用于视图层（前端布局页面），在主题布局页面中推送模板内容
- **事件系统**：用于服务层（业务逻辑），在业务逻辑中扩展功能

详细说明请参考 [Hook与事件系统的区别](#hook与事件系统的区别) 章节。

## 事件列表

### 结账事件（6个）

| 事件名称 | 触发时机 | 主要用途 |
|---------|---------|---------|
| `Weline_Checkout::checkout::validate::before` | 结账数据验证前 | 修改验证数据、执行预处理 |
| `Weline_Checkout::checkout::validate::after` | 结账数据验证后 | 根据验证结果执行操作 |
| `Weline_Checkout::checkout::calculate_totals::before` | 计算总额前 | 修改计算参数（运费、税费、折扣） |
| `Weline_Checkout::checkout::calculate_totals::after` | 计算总额后 | 修改计算结果、执行后续操作 |
| `Weline_Checkout::checkout::create_order::before` | 创建订单前 | 验证库存、应用优惠券、修改订单数据 |
| `Weline_Checkout::checkout::create_order::after` | 创建订单后 | 扣减库存、发送通知、发起支付 |

### 订单事件（9个）

| 事件名称 | 触发时机 | 主要用途 |
|---------|---------|---------|
| `Weline_Checkout::order::load::before` | 订单加载前 | 执行预处理操作 |
| `Weline_Checkout::order::load::after` | 订单加载后 | 修改订单数据、加载关联数据 |
| `Weline_Checkout::order::status::change::before` | 订单状态变更前 | 验证状态变更是否允许 |
| `Weline_Checkout::order::status::change::after` | 订单状态变更后 | 根据状态变化执行操作 |
| `Weline_Checkout::order::cancel::before` | 订单取消前 | 验证是否可以取消 |
| `Weline_Checkout::order::cancel::after` | 订单取消后 | 恢复库存、处理退款、发送通知 |
| `Weline_Checkout::order::completed` | 订单完成 | 增加积分、发送通知 |
| `Weline_Checkout::order::cancelled` | 订单取消 | 恢复库存、处理退款 |
| `Weline_Checkout::order::refunded` | 订单退款 | 更新账户余额、发送通知 |

### 支付事件（8个）

| 事件名称 | 触发时机 | 主要用途 |
|---------|---------|---------|
| `Weline_Checkout::payment::validate::before` | 支付验证前 | 修改验证数据、执行预处理 |
| `Weline_Checkout::payment::validate::after` | 支付验证后 | 根据验证结果执行操作 |
| `Weline_Checkout::payment::process::before` | 支付处理前 | 准备支付参数、选择支付网关 |
| `Weline_Checkout::payment::process::after` | 支付处理后 | 调用支付网关API、创建支付订单 |
| `Weline_Checkout::payment::callback::before` | 支付回调前 | 验证回调签名 |
| `Weline_Checkout::payment::callback::after` | 支付回调后 | 验证支付结果、更新订单状态 |
| `Weline_Checkout::payment::success` | 支付成功 | 更新订单状态、发送通知、增加积分 |
| `Weline_Checkout::payment::failed` | 支付失败 | 记录日志、发送通知、释放库存 |

## 事件实现方法

### 1. 创建观察者类

在您的模块（如支付模块）的 `Observer/` 目录下创建观察者类。

**文件命名规则**：
- 观察者类名使用描述性名称，如 `CheckoutOrderCreatedObserver`
- 文件扩展名为 `.php`

**示例**：
- 事件名称：`Weline_Checkout::checkout::create_order::after`
- 观察者类名：`CheckoutOrderCreatedObserver`
- 文件路径：`{YourPaymentModule}/Observer/CheckoutOrderCreatedObserver.php`

### 2. 观察者类结构

观察者类必须实现 `ObserverInterface` 接口：

```php
<?php

namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CheckoutOrderCreatedObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     */
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $order = $data['order'] ?? null;
        $orderId = $data['order_id'] ?? null;
        
        // 处理逻辑
        // ...
    }
}
```

### 3. 注册观察者

在您的模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Checkout::checkout::create_order::after">
        <observer name="your_module_order_created" 
                  instance="Weline\YourModule\Observer\CheckoutOrderCreatedObserver" 
                  disabled="false" 
                  shared="true" 
                  sort="10"/>
    </event>
</config>
```

### 配置说明

- **name**: 观察者名称，格式为 `模块名_观察者名`
- **instance**: 观察者类的完整命名空间
- **disabled**: 是否禁用，`false` 表示启用
- **shared**: 是否共享实例，`true` 表示共享
- **sort**: 执行顺序，数字越小越先执行

## 详细事件说明

### 事件1: 创建订单前

**事件名称**: `Weline_Checkout::checkout::create_order::before`

**触发时机**: 在 `CheckoutService::createOrder()` 方法中，数据验证通过后、开始创建订单之前

**可用数据**:
```php
[
    'data' => [
        'customer_id' => 1,
        'items' => [...],
        'shipping_address' => [...],
        'payment_method' => 'alipay',
        // ... 其他订单数据
    ]
]
```

**使用场景**:
- 验证库存
- 应用优惠券
- 修改订单数据
- 验证支付方式

**实现示例**:

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CreateOrderBeforeObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $orderData = &$data['data'];
        
        // 验证库存
        foreach ($orderData['items'] as $item) {
            if (!$this->checkInventory($item['product_id'], $item['quantity'])) {
                throw new \Exception('库存不足');
            }
        }
        
        // 应用优惠券
        if (!empty($orderData['coupon_code'])) {
            $discount = $this->applyCoupon($orderData['coupon_code']);
            $orderData['discount_amount'] = ($orderData['discount_amount'] ?? 0) + $discount;
        }
    }
}
```

详细文档请参考：[创建订单前](./event/checkout/创建订单前.md)

### 事件2: 创建订单后

**事件名称**: `Weline_Checkout::checkout::create_order::after`

**触发时机**: 订单和订单项都创建成功后

**可用数据**:
```php
[
    'order' => Order对象,
    'order_id' => 1
]
```

**使用场景**:
- 扣减库存
- 发送订单创建通知
- 自动发起支付
- 记录订单日志

**实现示例**:

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CreateOrderAfterObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $order = $data['order'];
        $orderId = $data['order_id'];
        
        // 扣减库存
        $items = $order->getItems();
        foreach ($items as $item) {
            $this->deductInventory($item['product_id'], $item['quantity']);
        }
        
        // 发送订单创建通知
        $this->sendOrderCreatedNotification($order);
    }
}
```

详细文档请参考：[创建订单后](./event/checkout/创建订单后.md)

## 事件执行顺序

当创建订单并处理支付时，事件的执行顺序如下：

1. `Weline_Checkout::checkout::validate::before` - 结账数据验证前
2. `Weline_Checkout::checkout::validate::after` - 结账数据验证后
3. `Weline_Checkout::checkout::calculate_totals::before` - 计算总额前
4. `Weline_Checkout::checkout::calculate_totals::after` - 计算总额后
5. `Weline_Checkout::checkout::create_order::before` - 创建订单前
6. 创建订单和订单项
7. `Weline_Checkout::checkout::create_order::after` - 创建订单后
8. `Weline_Checkout::payment::validate::before` - 支付验证前
9. `Weline_Checkout::payment::validate::after` - 支付验证后
10. `Weline_Checkout::payment::process::before` - 支付处理前
11. 创建支付交易记录
12. `Weline_Checkout::payment::process::after` - 支付处理后
13. （支付成功后）`Weline_Checkout::payment::callback::before` - 支付回调前
14. `Weline_Checkout::payment::callback::after` - 支付回调后
15. `Weline_Checkout::payment::success` - 支付成功
16. `Weline_Checkout::order::status::change::before` - 订单状态变更前
17. `Weline_Checkout::order::status::change::after` - 订单状态变更后

## 最佳实践

1. **错误处理**：在观察者中捕获异常，避免影响主流程
2. **数据验证**：在观察者中验证数据，确保数据完整性
3. **日志记录**：记录重要的操作日志，便于排查问题
4. **幂等性**：确保观察者可以安全地多次执行
5. **性能考虑**：避免在观察者中执行耗时操作，必要时使用异步处理
6. **数据修改**：通过引用传递的数据可以直接修改，但要注意数据一致性


## 常见问题

### Q1: 观察者没有被执行？

A: 检查以下几点：
1. 观察者类是否实现了 `ObserverInterface` 接口
2. 是否在 `etc/event.xml` 中正确注册
3. 模块是否已安装并启用
4. 运行 `php bin/w setup:upgrade` 更新模块
5. 运行 `php bin/w event:cache -f` 刷新事件缓存

### Q2: 如何在观察者中访问数据库？

A: 使用ObjectManager获取模型实例：

```php
use Weline\Framework\Manager\ObjectManager;

$orderModel = ObjectManager::getInstance(\Weline\Checkout\Model\Order::class);
```

### Q3: 观察者中可以修改订单数据吗？

A: 在 `checkout::create_order::before` 事件中可以修改 `$data` 数组（通过引用传递），但需要注意：
- 修改后的数据必须通过验证
- 不能破坏数据完整性

### Q4: 多个模块实现同一个事件会冲突吗？

A: 不会。所有模块的观察者都会被调用，按 `sort` 参数指定的顺序执行。

### Q5: 观察者中如何返回数据给调用方？

A: 事件主要用于执行操作，不返回数据。如果需要传递数据，可以使用：
- 在观察者中直接操作数据库
- 通过引用传递的数据可以直接修改
- 使用其他事件系统传递数据

## Hook与事件系统的区别

### Hook系统（视图层）

**用途**：在主题布局页面中注入模板内容

**使用场景**：
- 前端结账布局页面
- 订单列表/详情页面
- 主题布局定制
- 在页面中注入HTML、CSS、JavaScript

**实现方式**：
- 在 `view/hooks/` 目录下创建 `.phtml` 文件
- 文件命名：Hook名称中的 `::` 转换为 `--`
- 例如：`Weline_Checkout--frontend--layouts--checkout--form-before.phtml`

**调用方式**：
- 在模板中使用 `<w:hook>` 标签调用
- 例如：`<w:hook>Weline_Checkout::frontend::layouts::checkout::form-before</w:hook>`

**文件**：
- `hook.php` - 定义Hook规约（名称、描述、文档路径）
- `hook.php` - 定义本模块 Hook 规约；如需 PHP 常量则放在本模块 `Api`

### 事件系统（服务层）

**用途**：在业务逻辑中扩展功能

**使用场景**：
- 订单创建、支付处理、状态变更等业务逻辑
- 数据验证、计算、处理等操作
- 跨模块业务协作

**实现方式**：
- 创建Observer类，实现 `ObserverInterface` 接口
- 在 `etc/event.xml` 中注册观察者

**调用方式**：
- 通过 `EventsManager::dispatch()` 派遣事件

**文件**：
- `event.php` - 定义事件规约（名称、描述、文档路径）
- `etc/event.xml` - 注册观察者

### 两者关系

- **不冲突**：Hook用于视图层，事件用于服务层，两者可以同时使用
- **互补关系**：Hook负责页面展示，事件负责业务逻辑
- **使用建议**：
  - 需要在页面中注入HTML/CSS/JS → 使用Hook
  - 需要在业务逻辑中扩展功能 → 使用事件
