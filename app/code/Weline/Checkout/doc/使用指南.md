## 快速开始

### 安装模块

1. 确保模块已放置在 `app/code/Weline/Checkout/` 目录
2. 运行安装命令：
```bash
php bin/w setup:upgrade
```

3. 安装完成后，访问后台管理系统，在"业务模块"菜单下可以看到"订单管理"菜单项

### 验证安装

访问以下URL验证模块是否正常工作：
- 前端结账页面：`/checkout/frontend/checkout/index`
- 后台订单管理：`/checkout/backend/order/index`

## 基本使用

### 创建订单

#### 使用服务层创建订单

```php
use Weline\Checkout\Service\CheckoutService;
use Weline\Framework\Manager\ObjectManager;

// 获取结账服务实例
$checkoutService = ObjectManager::getInstance(CheckoutService::class);

// 准备订单数据
$orderData = [
    'customer_id' => 1,  // 客户ID
    'items' => [
        [
            'product_id' => 1,
            'product_name' => '商品名称',
            'product_sku' => 'SKU001',
            'quantity' => 2,
            'price' => 100.00,
            'attributes' => [  // 可选：产品属性
                'color' => '红色',
                'size' => 'L'
            ]
        ],
        // 可以添加多个商品
    ],
    'shipping_address' => [
        'name' => '张三',
        'phone' => '13800138000',
        'address' => '北京市朝阳区xxx街道xxx号',
        'postal_code' => '100000'
    ],
    'billing_address' => [  // 可选，如果不提供则使用shipping_address
        'name' => '张三',
        'phone' => '13800138000',
        'address' => '北京市朝阳区xxx街道xxx号'
    ],
    'shipping_method' => 'standard',  // 配送方式
    'shipping_amount' => 10.00,  // 运费
    'tax_amount' => 0.00,  // 税费
    'discount_amount' => 0.00,  // 折扣金额
    'payment_method' => 'alipay',  // 支付方式
    'currency' => 'CNY',  // 货币代码
    'remark' => '订单备注'  // 可选
];

try {
    // 创建订单
    $order = $checkoutService->createOrder($orderData);
    
    // 获取订单信息
    $orderId = $order->getId();
    $orderNumber = $order->getOrderNumber();
    
    echo "订单创建成功，订单号：{$orderNumber}";
} catch (\Exception $e) {
    echo "订单创建失败：" . $e->getMessage();
}
```

#### 订单数据验证

服务会自动验证以下内容：
- 客户ID不能为空
- 订单项不能为空
- 每个订单项必须包含：product_id、quantity、price
- 收货地址不能为空

```javascript
// 提交订单
fetch('/checkout/frontend/checkout/createOrder', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
    },
    body: JSON.stringify({
        items: [
            {
                product_id: 1,
                product_name: '商品名称',
                quantity: 2,
                price: 100.00
            }
        ],
        shipping_address: {
            name: '张三',
```

## 支付集成

### Hook扩展点

结账模块为支付模块提供了6个hook扩展点，支付模块可以通过实现这些hook来扩展支付功能。

#### 1. 创建订单前事件

**事件名称**: `Weline_Checkout::checkout::create_order::before`

**触发时机**: 在订单创建之前

**可用数据**: 
- `data` - 订单数据数组（引用传递，可修改）

**用途**: 数据验证、修改订单数据、添加额外信息

**实现示例**:

在支付模块的 `Observer/` 目录下创建观察者类：

```php
namespace Weline\YourPaymentModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CreateOrderBeforeObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $orderData = &$data['data'];
        
        // 验证支付方式
        if (empty($orderData['payment_method'])) {
            $orderData['payment_method'] = 'alipay';
        }
        
        // 添加支付相关配置
        $orderData['payment_config'] = [
            'gateway' => 'alipay',
            'timeout' => 300
        ];
    }
}
```

然后在 `etc/event.xml` 中注册：

```xml
<event name="Weline_Checkout::checkout::create_order::before">
    <observer name="your_module_create_order_before" 
              instance="Weline\YourPaymentModule\Observer\CreateOrderBeforeObserver" 
              disabled="false" 
              shared="true" 
              sort="10"/>
</event>
```

详细说明请参考 [事件使用指南](./事件使用指南.md)。

### Hook扩展点（视图层）

结账模块为支付模块提供了视图层Hook扩展点，用于在结账页面中注入支付方式选择组件等模板内容。

#### 支付方式选择区域Hook

**Hook名称**: `Weline_Checkout::frontend::layouts::checkout::payment-methods-before`

**触发时机**: 在结账页面渲染支付方式选择区域之前

**用途**: 注入支付方式选择组件、支付说明等

**实现示例**:

在支付模块的 `view/hooks/` 目录下创建文件：
`Weline_Checkout--frontend--layouts--checkout--payment-methods-before.phtml`

```php
<?php
// 注入支付方式选择组件
?>
<div class="payment-methods">
    <h3><?= __('选择支付方式') ?></h3>
    <div class="payment-options">
        <label>
            <input type="radio" name="payment_method" value="alipay" checked>
            <span><?= __('支付宝') ?></span>
        </label>
        <label>
            <input type="radio" name="payment_method" value="wechat">
            <span><?= __('微信支付') ?></span>
        </label>
    </div>
</div>
```

然后在结账页面模板中使用：

```phtml
<w:hook>Weline_Checkout::frontend::layouts::checkout::payment-methods-before</w:hook>
```

详细说明请参考Hook规约文件 `hook.php`。

## 订单状态说明

### 订单状态

- `pending` - 待处理：订单已创建，等待处理
- `processing` - 处理中：订单正在处理中
- `completed` - 已完成：订单已完成
- `cancelled` - 已取消：订单已取消
- `refunded` - 已退款：订单已退款

### 支付状态

- `pending` - 待支付：等待支付
- `paid` - 已支付：支付成功
- `failed` - 支付失败：支付失败
- `refunded` - 已退款：已退款

### 配送状态

- `pending` - 待发货：等待发货
- `shipped` - 已发货：已发货
- `delivered` - 已送达：已送达

## 常见问题

### Q1: 如何集成支付模块？

A: 支付模块需要实现以下hook文件：
1. `Weline_Checkout--payment--process--after.phtml` - 处理支付逻辑
2. `Weline_Checkout--payment--callback.phtml` - 处理支付回调

详细说明请参考 [Hook使用指南](#hook扩展点) 章节。

### Q2: 订单创建失败怎么办？

A: 检查以下几点：
1. 客户ID是否正确
2. 订单项数据是否完整（product_id、quantity、price）
3. 收货地址是否提供
4. 查看错误消息了解具体原因

### Q3: 如何自定义订单号生成规则？

A: 修改 `Model/Order.php` 中的 `generateOrderNumber()` 方法：

```php
public function generateOrderNumber(): string
{
    // 自定义订单号格式
    return 'ORD' . date('YmdHis') . str_pad((string)mt_rand(1, 9999), 4, '0', STR_PAD_LEFT);
}
```

### Q4: 如何添加自定义订单状态？

A: 
1. 在 `Model/Order.php` 中添加状态常量
2. 在 `getStatusText()` 方法中添加状态文本映射
3. 更新翻译文件 `i18n/zh_Hans_CN.csv` 和 `i18n/en_US.csv`

### Q5: 订单取消后如何处理库存？

A: 在 `Weline_Checkout::order::status::change` hook中实现库存恢复逻辑：

```php
// 在库存模块中实现
if ($newStatus === 'cancelled') {
    // 恢复库存
    $this->restoreInventory($order);
}
```

### Q6: 如何实现订单自动取消（超时未支付）？

A: 可以创建一个定时任务（Cron Job），定期检查待支付订单，超过一定时间后自动取消：

```php
// 在Cron任务中
$orders = $orderModel->where('payment_status', 'pending')
    ->where('created_time', date('Y-m-d H:i:s', strtotime('-24 hours')), '<')
    ->select()
    ->fetchArray();

foreach ($orders as $order) {
    $orderService->cancelOrder($order['order_id']);
}
```

## 最佳实践

1. **订单创建使用事务**：确保订单和订单项同时创建成功或同时失败
2. **支付状态更新**：在支付成功后及时更新订单支付状态
3. **订单状态管理**：使用状态机模式管理订单状态流转
4. **错误处理**：捕获并记录所有异常，给用户友好的错误提示
5. **数据验证**：在创建订单前进行完整的数据验证
6. **Hook使用**：通过hook扩展功能，而不是直接修改核心代码
