# Weline_Checkout::payment::process::after - 支付处理后

## 事件说明

在支付处理之后触发，允许其他模块执行实际的支付逻辑，如调用支付网关API、创建支付订单等。

## 触发时机

在 `PaymentService::processPayment()` 方法中，支付交易记录创建后。

## 数据格式

```php
[
    'order_id' => 1,
    'order' => Order对象,
    'transaction_id' => 1,
    'transaction' => PaymentTransaction对象,
    'payment_method' => 'alipay',
    'payment_data' => [...]
]
```

## 可用数据

- `order_id` (int) - 订单ID
- `order` (Order) - 订单对象
- `transaction_id` (int) - 交易ID
- `transaction` (PaymentTransaction) - 交易对象
- `payment_method` (string) - 支付方式
- `payment_data` (array) - 支付数据

## 使用场景

- 调用支付网关API
- 创建支付订单
- 生成支付链接或二维码
- 跳转到支付页面

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class PaymentProcessAfterObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $order = $data['order'];
        $paymentMethod = $data['payment_method'];
        $transaction = $data['transaction'];
        
        // 根据支付方式调用不同的支付网关
        switch ($paymentMethod) {
            case 'alipay':
                $result = $this->callAlipayAPI($order);
                break;
            case 'wechat':
                $result = $this->callWechatAPI($order);
                break;
        }
        
        // 更新交易记录
        if ($result['success']) {
            $transaction->setTransactionNumber($result['trade_no']);
            $transaction->setGatewayResponse(json_encode($result));
            $transaction->save();
        }
    }
}
```

## 注意事项

- 支付交易记录已创建，可以安全地使用交易数据
- 建议在观察者中处理实际的支付逻辑
- 如果支付失败，应该记录日志而不是抛出异常
