# Weline_Checkout::payment::callback::after - 支付回调后

## 事件说明

在处理支付回调之后触发，允许其他模块验证支付结果、更新订单状态、处理退款等。

## 触发时机

在 `PaymentService::handlePaymentCallback()` 方法中，处理回调逻辑之后。

## 数据格式

```php
[
    'callback_data' => [
        'order_id' => 1,
        'transaction_id' => 'TXN123456',
        'status' => 'success',
        'amount' => 100.00,
        // ... 其他回调数据
    ]
]
```

## 可用数据

- `callback_data` (array) - 回调数据数组

## 使用场景

- 验证支付结果
- 更新订单支付状态
- 处理退款
- 发送支付通知

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Manager\ObjectManager;
use Weline\Checkout\Model\Order;

class PaymentCallbackAfterObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $callbackData = $data['callback_data'];
        
        // 获取订单
        $orderId = $callbackData['order_id'];
        $order = ObjectManager::getInstance(Order::class)->load($orderId);
        
        // 验证订单金额
        $callbackAmount = floatval($callbackData['amount'] ?? 0);
        if (abs($callbackAmount - $order->getTotalAmount()) > 0.01) {
            throw new \Exception('订单金额不匹配');
        }
        
        // 更新订单支付状态
        if ($callbackData['status'] === 'success') {
            $order->setPaymentStatus(Order::PAYMENT_STATUS_PAID);
            if ($order->getStatus() === Order::STATUS_PENDING) {
                $order->setStatus(Order::STATUS_PROCESSING);
            }
            $order->save();
        }
    }
}
```

## 注意事项

- 回调数据已处理，可以安全地使用
- 建议在观察者中更新订单状态
- 如果处理失败，应该记录日志而不是抛出异常
