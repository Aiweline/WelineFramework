# Weline_Checkout::payment::failed - 支付失败

## 事件说明

当支付失败时触发，允许其他模块执行失败后的操作，如记录日志、发送通知、释放库存等。

## 触发时机

在 `PaymentService::handlePaymentCallback()` 方法中，当回调状态为 `failed` 或 `fail` 时。

## 数据格式

```php
[
    'callback_data' => [
        'order_id' => 1,
        'transaction_id' => 'TXN123456',
        'status' => 'failed',
        'error_message' => '支付失败原因',
        // ... 其他回调数据
    ],
    'order_id' => 1
]
```

## 可用数据

- `callback_data` (array) - 回调数据数组
- `order_id` (int) - 订单ID

## 使用场景

- 记录支付失败日志
- 发送支付失败通知
- 释放库存
- 更新订单状态

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class PaymentFailedObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $callbackData = $data['callback_data'];
        $orderId = $data['order_id'];
        
        // 记录支付失败日志
        $errorMessage = $callbackData['error_message'] ?? '支付失败';
        error_log("支付失败 - 订单ID: {$orderId}, 原因: {$errorMessage}");
        
        // 发送支付失败通知
        $this->sendPaymentFailedNotification($orderId, $errorMessage);
        
        // 可以在这里释放库存或执行其他清理操作
    }
}
```

## 注意事项

- 支付已失败，可以安全地执行清理操作
- 建议在观察者中进行异步操作，避免阻塞主流程
- 如果操作失败，应该记录日志而不是抛出异常
