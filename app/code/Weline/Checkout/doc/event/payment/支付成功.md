# Weline_Checkout::payment::success - 支付成功

## 事件说明

当支付成功时触发，允许其他模块执行成功后的操作，如更新订单状态、发送通知、增加积分等。

## 触发时机

在 `PaymentService::handlePaymentCallback()` 方法中，当回调状态为 `success` 或 `paid` 时。

## 数据格式

```php
[
    'callback_data' => [
        'order_id' => 1,
        'transaction_id' => 'TXN123456',
        'status' => 'success',
        // ... 其他回调数据
    ],
    'order_id' => 1
]
```

## 可用数据

- `callback_data` (array) - 回调数据数组
- `order_id` (int) - 订单ID

## 使用场景

- 更新订单状态
- 发送支付成功通知
- 增加用户积分
- 触发其他业务流程

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Manager\ObjectManager;
use Weline\Checkout\Service\OrderService;

class PaymentSuccessObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $orderId = $data['order_id'];
        
        // 更新订单状态
        $orderService = ObjectManager::getInstance(OrderService::class);
        $order = $orderService->getOrder($orderId);
        
        if ($order && $order->getStatus() === 'pending') {
            $orderService->updateOrderStatus($orderId, 'processing');
        }
        
        // 发送支付成功通知
        $this->sendPaymentSuccessNotification($order);
        
        // 增加用户积分
        $points = (int)($order->getTotalAmount() * 0.01);
        $this->addUserPoints($order->getCustomerId(), $points);
    }
}
```

## 注意事项

- 支付已成功，可以安全地执行后续操作
- 建议在观察者中进行异步操作，避免阻塞主流程
