# Weline_Checkout::payment::callback::before - 支付回调前

## 事件说明

在处理支付回调之前触发，允许其他模块验证回调签名或执行预处理操作。

## 触发时机

在 `PaymentService::handlePaymentCallback()` 方法中，处理回调逻辑之前。

## 数据格式

```php
[
    'callback_data' => [
        'order_id' => 1,
        'transaction_id' => 'TXN123456',
        'status' => 'success',
        // ... 其他回调数据
    ]
]
```

## 可用数据

- `callback_data` (array, 引用传递) - 回调数据数组，可以在观察者中修改

## 使用场景

- 验证回调签名
- 验证回调数据格式
- 执行预处理操作

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class PaymentCallbackBeforeObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $callbackData = &$data['callback_data'];
        
        // 验证回调签名
        if (!$this->verifyCallbackSignature($callbackData)) {
            throw new \Exception('回调签名验证失败');
        }
        
        // 验证回调数据格式
        if (empty($callbackData['order_id']) || empty($callbackData['status'])) {
            throw new \Exception('回调数据格式错误');
        }
    }
}
```

## 注意事项

- 回调数据通过引用传递，可以直接修改
- 可以抛出异常阻止回调处理
- 建议在观察者中进行完整的签名验证
