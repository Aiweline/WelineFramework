# Weline_Checkout::payment::validate::after - 支付验证后

## 事件说明

在验证支付数据之后触发，允许其他模块根据验证结果执行后续操作。

## 触发时机

在 `PaymentService::validatePayment()` 方法中，执行数据验证之后。

## 数据格式

```php
[
    'data' => [
        'order_id' => 1,
        'payment_method' => 'alipay',
        'amount' => 100.00
    ],
    'result' => [
        'valid' => true,
        'errors' => []
    ]
]
```

## 可用数据

- `data` (array) - 支付数据数组
- `result` (array, 引用传递) - 验证结果，可以在观察者中修改

## 使用场景

- 根据验证结果执行后续操作
- 修改验证结果
- 记录验证日志

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class PaymentValidateAfterObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $result = &$data['result'];
        
        // 记录验证日志
        if (!$result['valid']) {
            error_log('支付验证失败: ' . implode(', ', $result['errors']));
        }
    }
}
```

## 注意事项

- 验证结果通过引用传递，可以直接修改
- 修改验证结果会影响后续流程
