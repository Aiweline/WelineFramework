# Weline_Checkout::payment::validate::before - 支付验证前

## 事件说明

在验证支付数据之前触发，允许其他模块修改验证数据或执行预处理操作。

## 触发时机

在 `PaymentService::validatePayment()` 方法中，执行数据验证之前。

## 数据格式

```php
[
    'data' => [
        'order_id' => 1,
        'payment_method' => 'alipay',
        'amount' => 100.00
    ]
]
```

## 可用数据

- `data` (array, 引用传递) - 支付数据数组，可以在观察者中修改

## 使用场景

- 修改支付数据
- 添加支付配置
- 验证支付限额

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class PaymentValidateBeforeObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $paymentData = &$data['data'];
        
        // 添加支付配置
        $paymentData['payment_config'] = [
            'gateway' => 'alipay',
            'timeout' => 300
        ];
    }
}
```

## 注意事项

- 数据通过引用传递，可以直接修改
- 修改后的数据会影响后续的验证逻辑
