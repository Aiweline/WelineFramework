# Weline_Checkout::checkout::calculate_totals::after - 计算订单总额后

## 事件说明

在计算订单总额之后触发，允许其他模块修改计算结果或执行后续操作。

## 触发时机

在 `CheckoutService::calculateTotals()` 方法中，执行总额计算之后。

## 数据格式

```php
[
    'items' => [...],
    'result' => [
        'subtotal' => 250.00,
        'shipping_amount' => 10.00,
        'tax_amount' => 0.00,
        'discount_amount' => 5.00,
        'total_amount' => 255.00
    ]
]
```

## 可用数据

- `items` (array) - 订单项数组
- `result` (array, 引用传递) - 计算结果，可以在观察者中修改

## 使用场景

- 修改计算结果（如四舍五入规则）
- 应用额外的折扣或费用
- 记录计算日志
- 验证计算结果

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CalculateTotalsAfterObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $result = &$data['result'];
        
        // 应用额外的服务费
        $serviceFee = 2.00;
        $result['total_amount'] += $serviceFee;
        
        // 记录计算日志
        error_log('订单总额计算: ' . $result['total_amount']);
    }
}
```

## 注意事项

- 计算结果通过引用传递，可以直接修改
- 修改后的结果会影响订单创建
- 建议保持计算逻辑的一致性
