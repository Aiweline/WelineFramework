# Weline_Checkout::checkout::validate::after - 结账数据验证后

## 事件说明

在验证结账数据之后触发，允许其他模块根据验证结果执行后续操作。

## 触发时机

在 `CheckoutService::validateCheckout()` 方法中，执行数据验证逻辑之后。

## 数据格式

```php
[
    'data' => [
        'customer_id' => 1,
        'items' => [...],
        // ... 其他订单数据
    ],
    'result' => [
        'valid' => true,
        'errors' => []
    ]
]
```

## 可用数据

- `data` (array) - 结账数据数组
- `result` (array, 引用传递) - 验证结果，可以在观察者中修改

## 使用场景

- 根据验证结果执行后续操作
- 修改验证结果（添加或移除错误）
- 记录验证日志

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CheckoutValidateAfterObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $result = &$data['result'];
        
        // 记录验证日志
        if (!$result['valid']) {
            error_log('结账验证失败: ' . implode(', ', $result['errors']));
        }
        
        // 可以添加额外的验证规则
        // if (某些条件) {
        //     $result['valid'] = false;
        //     $result['errors'][] = '额外的验证错误';
        // }
    }
}
```

## 注意事项

- 验证结果通过引用传递，可以直接修改
- 修改验证结果会影响后续流程
- 建议谨慎修改验证结果，避免破坏业务逻辑
