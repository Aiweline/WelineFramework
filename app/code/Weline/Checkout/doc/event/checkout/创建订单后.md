# Weline_Checkout::checkout::create_order::after - 创建订单后

## 事件说明

在创建订单之后触发，允许其他模块执行后续操作，如扣减库存、发送通知、发起支付等。

## 触发时机

在 `CheckoutService::createOrder()` 方法中，订单和订单项都创建成功后。

## 数据格式

```php
[
    'order' => Order对象,
    'order_id' => 1
]
```

## 可用数据

- `order` (Order) - 订单对象
- `order_id` (int) - 订单ID

## 使用场景

- 扣减库存
- 发送订单创建通知
- 自动发起支付
- 记录订单日志
- 触发其他业务流程

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CreateOrderAfterObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $order = $data['order'];
        $orderId = $data['order_id'];
        
        // 扣减库存
        $items = $order->getItems();
        foreach ($items as $item) {
            $this->deductInventory($item['product_id'], $item['quantity']);
        }
        
        // 发送订单创建通知
        $this->sendOrderCreatedNotification($order);
        
        // 如果订单需要立即支付，自动发起支付
        if ($order->getPaymentMethod() === 'alipay') {
            // 跳转到支付页面或发起支付请求
        }
    }
}
```

## 注意事项

- 订单已保存到数据库，可以安全地使用订单数据
- 建议在观察者中进行异步操作（如发送通知），避免阻塞主流程
- 如果操作失败，应该记录日志而不是抛出异常
