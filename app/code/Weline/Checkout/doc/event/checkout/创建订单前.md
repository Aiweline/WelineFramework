# Weline_Checkout::checkout::create_order::before - 创建订单前

## 事件说明

在创建订单之前触发，允许其他模块修改订单数据、验证库存、应用优惠券等。

## 触发时机

在 `CheckoutService::createOrder()` 方法中，数据验证通过后、开始创建订单之前。

## 数据格式

```php
[
    'data' => [
        'customer_id' => 1,
        'items' => [...],
        'shipping_address' => [...],
        'payment_method' => 'alipay',
        // ... 其他订单数据
    ]
]
```

## 可用数据

- `data` (array, 引用传递) - 订单数据数组，可以在观察者中修改

## 使用场景

- 验证库存
- 应用优惠券
- 修改订单数据
- 添加额外的订单信息
- 验证支付方式

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CreateOrderBeforeObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $orderData = &$data['data'];
        
        // 验证库存
        foreach ($orderData['items'] as $item) {
            if (!$this->checkInventory($item['product_id'], $item['quantity'])) {
                throw new \Exception('库存不足');
            }
        }
        
        // 应用优惠券
        if (!empty($orderData['coupon_code'])) {
            $discount = $this->applyCoupon($orderData['coupon_code']);
            $orderData['discount_amount'] = ($orderData['discount_amount'] ?? 0) + $discount;
        }
        
        // 添加支付配置
        $orderData['payment_config'] = [
            'gateway' => 'alipay',
            'timeout' => 300
        ];
    }
}
```

## 注意事项

- 数据通过引用传递，可以直接修改
- 可以抛出异常阻止订单创建
- 建议在观察者中进行完整的业务验证
