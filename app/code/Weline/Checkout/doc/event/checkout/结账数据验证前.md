# Weline_Checkout::checkout::validate::before - 结账数据验证前

## 事件说明

在验证结账数据之前触发，允许其他模块修改验证数据或执行预处理操作。

## 触发时机

在 `CheckoutService::validateCheckout()` 方法中，执行数据验证逻辑之前。

## 数据格式

```php
[
    'data' => [
        'customer_id' => 1,
        'items' => [...],
        'shipping_address' => [...],
        // ... 其他订单数据
    ]
]
```

## 可用数据

- `data` (array, 引用传递) - 结账数据数组，可以在观察者中修改

## 使用场景

- 修改订单数据（如添加默认值）
- 应用优惠券
- 验证库存
- 添加额外的验证规则

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CheckoutValidateBeforeObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $orderData = &$data['data'];
        
        // 添加默认支付方式
        if (empty($orderData['payment_method'])) {
            $orderData['payment_method'] = 'alipay';
        }
        
        // 应用优惠券
        if (!empty($orderData['coupon_code'])) {
            // 验证并应用优惠券
        }
    }
}
```

## 注意事项

- 数据通过引用传递，可以直接修改
- 修改后的数据会影响后续的验证逻辑
- 建议在观察者中进行数据验证，避免破坏数据完整性
