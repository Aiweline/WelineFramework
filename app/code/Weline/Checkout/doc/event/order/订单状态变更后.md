# Weline_Checkout::order::status::change::after - 订单状态变更后

## 事件说明

在订单状态变更之后触发，允许其他模块根据状态变化执行相应操作，如发送通知、更新库存等。

## 触发时机

在 `OrderService::updateOrderStatus()` 方法中，状态更新成功后。

## 数据格式

```php
[
    'order_id' => 1,
    'old_status' => 'pending',
    'new_status' => 'processing',
    'order' => Order对象
]
```

## 可用数据

- `order_id` (int) - 订单ID
- `old_status` (string) - 旧状态
- `new_status` (string) - 新状态
- `order` (Order) - 订单对象

## 使用场景

- 根据状态变化发送通知
- 更新库存
- 触发其他业务流程
- 记录状态变更日志

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class OrderStatusChangeAfterObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $oldStatus = $data['old_status'];
        $newStatus = $data['new_status'];
        $order = $data['order'];
        
        // 订单处理中时发送通知
        if ($newStatus === 'processing') {
            $this->sendOrderProcessingNotification($order);
        }
        
        // 订单取消时恢复库存
        if ($newStatus === 'cancelled' && $oldStatus !== 'cancelled') {
            $this->restoreInventory($order);
        }
    }
}
```

## 注意事项

- 状态已更新到数据库
- 建议在观察者中进行异步操作，避免阻塞主流程
- 如果操作失败，应该记录日志而不是抛出异常
