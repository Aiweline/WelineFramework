# Weline_Checkout::order::cancel::before - 订单取消前

## 事件说明

在取消订单之前触发，允许其他模块验证是否可以取消或执行预处理操作。

## 触发时机

在 `OrderService::cancelOrder()` 方法中，执行取消操作之前。

## 数据格式

```php
[
    'order_id' => 1,
    'order' => Order对象
]
```

## 可用数据

- `order_id` (int) - 订单ID
- `order` (Order) - 订单对象

## 使用场景

- 验证是否可以取消订单
- 执行取消前的业务逻辑
- 记录取消日志

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class OrderCancelBeforeObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $order = $data['order'];
        
        // 验证是否可以取消
        if ($order->isPaid() && !$this->canRefund($order)) {
            throw new \Exception('已支付订单无法取消');
        }
        
        // 记录取消日志
        error_log("准备取消订单: {$order->getOrderNumber()}");
    }
}
```

## 注意事项

- 可以抛出异常阻止订单取消
- 建议在观察者中进行完整的业务验证
