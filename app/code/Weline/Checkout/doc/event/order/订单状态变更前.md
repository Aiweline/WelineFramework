# Weline_Checkout::order::status::change::before - 订单状态变更前

## 事件说明

在订单状态变更之前触发，允许其他模块验证状态变更是否允许或执行预处理操作。

## 触发时机

在 `OrderService::updateOrderStatus()` 方法中，执行状态更新之前。

## 数据格式

```php
[
    'order_id' => 1,
    'old_status' => 'pending',
    'new_status' => 'processing',
    'order' => Order对象
]
```

## 可用数据

- `order_id` (int) - 订单ID
- `old_status` (string) - 旧状态
- `new_status` (string) - 新状态
- `order` (Order) - 订单对象

## 使用场景

- 验证状态变更是否允许
- 执行状态变更前的业务逻辑
- 记录状态变更日志

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class OrderStatusChangeBeforeObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $oldStatus = $data['old_status'];
        $newStatus = $data['new_status'];
        $order = $data['order'];
        
        // 验证状态变更是否允许
        if ($oldStatus === 'completed' && $newStatus === 'pending') {
            throw new \Exception('已完成订单不能变更为待处理状态');
        }
        
        // 记录状态变更日志
        error_log("订单状态变更: {$oldStatus} -> {$newStatus}");
    }
}
```

## 注意事项

- 可以抛出异常阻止状态变更
- 建议在观察者中进行完整的业务验证
