# Weline_Checkout::order::load::after - 订单加载后

## 事件说明

在加载订单数据之后触发，允许其他模块修改订单数据或执行后续操作。

## 触发时机

在 `OrderService::getOrder()` 方法中，订单加载成功后。

## 数据格式

```php
[
    'order_id' => 1,
    'order' => Order对象
]
```

## 可用数据

- `order_id` (int) - 订单ID
- `order` (Order, 引用传递) - 订单对象，可以在观察者中修改

## 使用场景

- 修改订单数据（如添加额外信息）
- 加载关联数据
- 格式化订单数据
- 记录访问日志

## 使用示例

```php
namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class OrderLoadAfterObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $order = &$data['order'];
        
        if ($order) {
            // 加载关联数据
            $order->setData('customer_name', $this->getCustomerName($order->getCustomerId()));
            
            // 格式化地址
            $shippingAddress = $order->getShippingAddressArray();
            $order->setData('formatted_address', $this->formatAddress($shippingAddress));
        }
    }
}
```

## 注意事项

- 订单对象通过引用传递，可以直接修改
- 修改后的订单数据会影响调用方
- 如果订单不存在，`order` 为 null
