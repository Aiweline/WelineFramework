# Weline Maintenance 模块 - 事件文档

## 概述

本文档详细说明了 Weline Maintenance 模块的维护模式实现机制及其提供的 `Weline_Maintenance::maintenance` 事件。

## 维护模式监听时机

### 监听事件

维护模式监听 `Weline_Framework::App::run_before` 事件，这是**框架中最早的事件时机**。

#### 为什么选择这个时机？

| 优势 | 说明 |
|------|------|
| ✅ 无数据库连接 | URL 解析之前触发，不会连接数据库 |
| ✅ 无第三方请求 | 不会触发任何第三方 API 调用 |
| ✅ 最小资源消耗 | 仅加载系统初始化和环境配置 |
| ✅ 快速响应 | 直接返回维护页面，响应速度最快 |

#### 事件触发顺序对比

```
1. Weline_Framework::App::run_before  ← 维护模式监听（最早）
   ↓
2. URL 解析（可能涉及数据库查询）
   ↓
3. Weline_Framework::App::url_parsed_after
   ↓
4. Weline_Framework_Router::before_start（旧的监听点）
   ↓
5. 路由处理、控制器执行...
```

### 配置文件

```xml
<!-- app/code/Weline/Maintenance/etc/event.xml -->
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <!-- 
        维护模式监听：使用最早的事件时机
        - Weline_Framework::App::run_before 在 URL 解析之前触发
        - 此时没有数据库连接、没有第三方服务请求
        - sort=-999 确保优先于其他观察者执行
    -->
    <event name="Weline_Framework::App::run_before">
        <observer name="Weline_Maintenance::app_run_before" 
                  instance="Weline\Maintenance\Observer\BeforeStart" 
                  disabled="false" 
                  shared="true" 
                  sort="-999"/>
    </event>
</config>
```

## 事件列表

### 1. Weline_Maintenance::maintenance - 白名单扩展事件

#### 基本信息

- **事件名称**：`Weline_Maintenance::maintenance`
- **事件类型**：系统维护事件
- **触发时机**：在系统检测到维护模式开启且请求不在默认白名单中时
- **触发位置**：`app/code/Weline/Maintenance/Observer/MaintenanceInterceptor.php`

#### 功能说明

`Weline_Maintenance::maintenance` 事件允许其他模块：
- 添加维护模式白名单 URL
- 自定义维护模式响应
- 标记请求为已处理（跳过默认维护页面）

#### 事件数据

```php
[
    'white_urls' => array,   // 白名单 URL 数组（可修改）
    'uri'        => string,  // 当前请求的 URI
    'handled'    => bool,    // 是否已被处理（设为 true 跳过默认响应）
]
```

#### 使用方法

##### 添加白名单 URL

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Maintenance::maintenance">
        <observer name="Your_Module::maintenance"
                  instance="Your\Module\Observer\MaintenanceObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\DataObject\DataObject;
use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class MaintenanceObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        /** @var DataObject $data */
        $data = $event->getData('data');
        $whiteUrls = $data->getData('white_urls') ?? [];
        
        // 添加白名单 URL
        $whiteUrls[] = '/api/health';       // 健康检查接口
        $whiteUrls[] = '/admin/login';      // 管理员登录
        $whiteUrls[] = '/maintenance/info'; // 维护信息页面
        
        $data->setData('white_urls', $whiteUrls);
    }
}
```

##### 自定义维护响应

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\DataObject\DataObject;
use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CustomMaintenanceResponseObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        /** @var DataObject $data */
        $data = $event->getData('data');
        $uri = $data->getData('uri');
        
        // 自定义 API 响应
        if (str_contains($uri, '/api/')) {
            http_response_code(503);
            header('Content-Type: application/json');
            echo json_encode([
                'error' => 'SERVICE_UNAVAILABLE',
                'message' => '服务维护中',
            ]);
            
            // 标记为已处理，跳过默认响应
            $data->setData('handled', true);
            exit;
        }
    }
}
```

## 默认白名单

维护模式默认允许以下资源访问：

```php
// 静态资源扩展名
'.css', '.js', '.png', '.jpg', '.jpeg', '.gif', '.svg', '.ico', 
'.woff', '.woff2', '.ttf', '.eot'

// 静态资源目录
'/pub/static/', '/pub/media/'
```

## 响应类型

维护模式会根据请求类型自动返回不同格式的响应：

### API 请求（JSON）

```json
{
    "success": false,
    "code": "maintenance",
    "message": "系统正在升级维护中，请稍后再试。",
    "data": {
        "retry_after": 60
    }
}
```

HTTP 头：
- `HTTP/1.1 503 Service Unavailable`
- `Content-Type: application/json; charset=utf-8`
- `Retry-After: 60`

### 普通请求（HTML）

返回美观的维护页面，包含：
- 渐变背景
- 动画效果
- 进度条指示
- 响应式设计

## 配置选项

在 `app/etc/env.php` 中配置：

```php
return [
    // 开启维护模式
    'maintenance' => true,
    
    // Retry-After 响应头的秒数（可选，默认 60）
    'maintenance_retry_after' => 120,
];
```

## 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **CLI 模式**：命令行模式下不检查维护模式，可正常执行命令
3. **性能考虑**：该事件在每次请求时都会触发，应避免执行耗时操作
4. **安全性**：谨慎添加白名单 URL，避免在维护模式下暴露敏感功能
