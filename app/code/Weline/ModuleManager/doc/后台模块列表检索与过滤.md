# 后台模块列表检索与过滤

## 页面入口

- 后台路由：`/module-manager/backend/listing`
- 控制器：`app/code/Weline/ModuleManager/Controller/Backend/Listing.php`
- 模板源文件：`app/code/Weline/ModuleManager/view/templates/Backend/Listing/index.phtml`
- 模板编译产物 `view/tpl/**` 禁止直接修改。

## 查询参数

页面使用普通 GET 表单，筛选条件会进入 URL，刷新、复制链接和翻页时均可保持。

| 参数 | 允许值 | 含义 |
|---|---|---|
| `search` | 文本 | 同时匹配模块名、描述、命名空间、基础路径、路由和版本字段。 |
| `status` | 空、`1`、`0` | 全部、已启用、已禁用。 |
| `position` | 空、`system`、`system_config`、`app`、`composer` | 模块来源/类型。 |
| `schema_status` | 空、`consistent`、`drifted`、`unverified`、`unknown` | 代码、数据库和 Schema 一致性状态。 |
| `pageSize` | `10`、`25`、`50`、`100` | 每页数量；其他值回退为 25。 |
| `page` | 正整数 | 当前页；筛选或每页数量变化时重新从第一页开始。 |

## 服务端行为

- 所有搜索、筛选、排序和分页都由控制器查询完成，不在浏览器中加载全部模块后再过滤。
- 搜索用 ORM `concat_like()` 组合允许字段；筛选值先经过白名单归一化，再进入查询。
- 默认按模块名升序排列。
- 控制器向模板提供总数、可见范围、当前页数量和已启用筛选数，空结果显示可恢复的清除筛选入口。
- 分页链接必须保留当前 `search/status/position/schema_status/pageSize`。

## 界面与安全边界

- 宽表压缩为模块、状态、类型、版本、Schema、路由、更新时间和操作八列；模块描述、命名空间和路径合并到身份列。
- 状态切换与版本回滚继续使用既有业务链路；本次筛选不改变其权限、确认和回滚语义。
- 筛选是幂等 GET 导航，不新增 Ajax/XHR/fetch/axios。
- 用户可见文本使用 `__()` 并同步 `i18n/zh_Hans_CN.csv`、`i18n/en_US.csv`。
- 任何后台筛选变更都要用登录态 Browser 验证默认列表、搜索命中、组合筛选、空结果、重置、分页和控制台错误。

## 索引同步

MCP 对该控制器、模板、翻译和本文档建立同项目内容索引。MCP 自身应用修改时同步定点重索引；其他本地编辑由 `PostToolUse` 后台刷新，并由下一次索引读取的新鲜度检查兜底。AI 应先一次确定所有相关路径，再用一次 `get_indexed_files(paths[])` 批量读取内容。
