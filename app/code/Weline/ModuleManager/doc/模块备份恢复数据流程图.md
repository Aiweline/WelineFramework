# 模块备份与恢复数据流程图

## 卸载备份流程（module:remove）

```mermaid
flowchart TD
    A[执行 module:remove] --> B[确认卸载模块列表]
    B --> C{是否存在 ModuleBackupService?}
    C -->|否| E[跳过数据库备份]
    C -->|是| D[调用 backupModuleTables]

    D --> D1[读取 Module\\Table 注册表]
    D1 --> D2[遍历模块所有表]
    D2 --> D3{表是否存在?}
    D3 -->|否| D4[记录跳过并继续]
    D3 -->|是| D5[统计记录数]
    D5 --> D6[ALTER TABLE 原表 RENAME TO 备份表]
    D6 --> D7[写入 Backup 记录表]

    D7 --> E[打印备份结果]

    E --> F[通过事件调用 UninstallService::uninstall]
    F --> G[备份模块文件]
    G --> H[触发 before/after 卸载事件]
    H --> I[模块卸载完成]
```

## 恢复流程（module:restore）

```mermaid
flowchart TD
    R0[执行 module:restore] --> R1[解析模块名和备份时间戳]
    R1 --> R2[确认危险操作(删除当前表)]
    R2 --> R3[调用 ModuleBackupService::restoreModuleTables]

    R3 --> R4[根据模块名与时间戳查询 Backup 记录]
    R4 --> R5{是否找到备份?}
    R5 -->|否| R6[返回失败信息并结束]
    R5 -->|是| R7[遍历备份记录中的每个表]

    R7 --> R8[DROP TABLE IF EXISTS 原表名]
    R8 --> R9[ALTER TABLE 备份表 RENAME TO 原表名]
    R9 --> R10[打印恢复结果]

    R10 --> R11[更新 Backup 状态为 restored]
    R11 --> R12[返回成功]
```

## 重装并恢复流程（module:reinstall --restore）

```mermaid
flowchart TD
    X0[执行 module:reinstall] --> X1[检查 deploy 模式]
    X1 --> X2[解析模块列表和 --restore 参数]
    X2 --> X3[确认危险操作]

    X3 --> X4[为每个模块执行 dropModuleTables]
    X4 --> X5[清理 modules.php 和 module_dependencies.php]
    X5 --> X6[重新注册模块并执行 Setup/Install + Model install]

    X6 --> X7{是否指定 --restore?}
    X7 -->|否| X11[结束重装流程]
    X7 -->|是| X8[调用 ModuleBackupService::restoreModuleTables]

    X8 --> X9[若有卸载备份: 覆盖当前表]
    X9 --> X10[打印恢复结果]
    X10 --> X11[结束重装流程]
```

## 状态转换图（备份记录）

```mermaid
stateDiagram-v2
    [*] --> Active: 生成卸载备份
    Active --> Restored: module:restore 成功恢复
    Active --> Deleted: 管理命令/逻辑标记删除
    Restored --> Deleted: 后续清理
```

## 异常与回滚流程

### 1. 备份失败

- 某张表不存在或重命名失败：
  - 在 `ModuleBackupService` 内记录 warning 日志。
  - 跳过该表，继续备份其他表。
- 所有表均未成功备份：
  - 返回 `success = false`。
  - `module:remove` 检测到失败后取消卸载，并输出错误信息。

### 2. 恢复失败

- 某张备份表不存在：
  - 打印 warning，跳过该表。
- RENAME 或 DROP 失败：
  - 记录 error，但继续处理其他表。
- 最终调用者（`module:restore` 或 `module:reinstall --restore`）负责汇总结果并输出。
