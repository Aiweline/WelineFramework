# Weline_Marketing 模块扩展开发文档

## 概述

Weline_Marketing 模块提供了强大的扩展能力，允许其他模块通过扩展点来自定义条件和动作，实现更灵活的营销规则。

## 扩展点

### 1. 条件扩展点（Condition）

**路径**: `extends/module/Weline_Marketing/Condition`

**接口**: `Weline\Marketing\Interface\Rule\ConditionInterface`

**用途**: 扩展营销规则的条件判断功能

**要求**:
- 必须实现 `ConditionInterface` 接口
- 必须实现所有接口方法
- 允许多个实现

### 2. 动作扩展点（Action）

**路径**: `extends/module/Weline_Marketing/Action`

**接口**: `Weline\Marketing\Interface\Rule\ActionInterface`

**用途**: 扩展营销规则的动作执行功能

**要求**:
- 必须实现 `ActionInterface` 接口
- 必须实现所有接口方法
- 允许多个实现

## 快速开始

### 创建自定义条件

1. 在您的模块中创建扩展目录：`extends/module/Weline_Marketing/Condition/`
2. 创建条件类，实现 `ConditionInterface` 接口
3. 实现所有必需的方法

**示例代码**:

```php
<?php
namespace Vendor\Module\Extends\Weline_Marketing\Condition;

use Weline\Marketing\Interface\Rule\ConditionInterface;
use Weline\Marketing\Model\Rule\Condition\AbstractCondition;

class CustomCondition extends AbstractCondition
{
    public function getCode(): string
    {
        return 'custom_condition';
    }
    
    public function getName(): string
    {
        return '自定义条件';
    }
    
    public function getDescription(): string
    {
        return '这是一个自定义条件示例';
    }
    
    public function validate(array $condition, array $context): bool
    {
        // 自定义验证逻辑
        $value = $condition['value'] ?? null;
        $customer = $context['customer'] ?? null;
        
        // 实现您的验证逻辑
        return true; // 或 false
    }
    
    public function getFormFields(): array
    {
        return [
            [
                'name' => 'value',
                'label' => '条件值',
                'type' => 'text',
                'required' => true,
            ],
        ];
    }
}
```

### 创建自定义动作

1. 在您的模块中创建扩展目录：`extends/module/Weline_Marketing/Action/`
2. 创建动作类，实现 `ActionInterface` 接口
3. 实现所有必需的方法

**示例代码**:

```php
<?php
namespace Vendor\Module\Extends\Weline_Marketing\Action;

use Weline\Marketing\Interface\Rule\ActionInterface;
use Weline\Marketing\Model\Rule\Action\AbstractAction;

class CustomAction extends AbstractAction
{
    public function getCode(): string
    {
        return 'custom_action';
    }
    
    public function getName(): string
    {
        return '自定义动作';
    }
    
    public function getDescription(): string
    {
        return '这是一个自定义动作示例';
    }
    
    public function execute(array $action, array $context): array
    {
        // 自定义执行逻辑
        $value = $action['value'] ?? 0;
        
        return [
            'discount_amount' => $value,
            'messages' => ['执行了自定义动作'],
        ];
    }
    
    public function getFormFields(): array
    {
        return [
            [
                'name' => 'value',
                'label' => '动作值',
                'type' => 'number',
                'required' => true,
            ],
        ];
    }
}
```

## 返回值格式

### 条件验证返回值

条件验证返回 `bool` 类型：
- `true`: 条件满足
- `false`: 条件不满足

### 动作执行返回值

动作执行返回 `array` 类型，格式如下：

```php
[
    'discount_amount' => 10.00,      // 折扣金额
    'shipping_discount' => 5.00,     // 运费折扣
    'free_shipping' => false,        // 是否免运费
    'gifts' => [                     // 赠品列表
        [
            'product_id' => 1,
            'qty' => 1,
            'type' => 'product',
        ],
    ],
    'messages' => [                  // 提示消息
        '享受优惠 10 元',
    ],
]
```

## 最佳实践

1. **继承抽象基类**: 建议继承 `AbstractCondition` 或 `AbstractAction`，可以复用通用功能
2. **错误处理**: 在验证和执行过程中，应该妥善处理异常情况
3. **性能优化**: 避免在条件验证和动作执行中进行耗时的操作
4. **数据验证**: 在执行前验证输入数据的有效性
5. **文档说明**: 为自定义条件和动作提供清晰的文档说明

## 示例场景

### 场景1: 自定义会员等级条件

```php
class MemberLevelCondition extends AbstractCondition
{
    public function getCode(): string
    {
        return 'member_level';
    }
    
    public function validate(array $condition, array $context): bool
    {
        $customer = $context['customer'] ?? null;
        if (!$customer) {
            return false;
        }
        
        $level = $customer['member_level'] ?? 0;
        $requiredLevel = (int)($condition['value'] ?? 0);
        
        return $level >= $requiredLevel;
    }
}
```

```php
    public function execute(array $action, array $context): array
    {
        $points = (int)($action['points'] ?? 0);
        $customerId = $context['customer']['id'] ?? null;
        
        if ($customerId && $points > 0) {
            // 添加积分逻辑
            // ...
        }
        
        return [
            'messages' => [sprintf('获得 %d 积分', $points)],
        ];
    }
}
```

## 注意事项

1. 扩展类必须放在正确的目录结构中
2. 类名和文件名必须遵循 PSR-4 规范
3. 扩展类会被自动加载，无需手动注册
4. 确保扩展类不会与内置类冲突（使用唯一的代码标识）
