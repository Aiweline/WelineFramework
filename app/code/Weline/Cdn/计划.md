# Weline_Cdn 模块实现计划

## 模块概述

Weline_Cdn 是一个多适配器CDN管理模块，提供缓存清理、规则管理、预热等功能。默认支持Cloudflare，同时允许其他模块通过适配器模式贡献其他CDN提供商。

## 功能需求

### 核心功能

1. **多适配器支持**
   - 默认适配器：Cloudflare（API Token认证）
   - 扩展机制：扫描已激活模块中的 `*\Cdn\Adapter\*` 类，实现 `AdapterInterface` 接口
   - 创建域名时可选择关联适配器

2. **账户管理**
   - 统一的账户管理界面
   - 每个适配器可在模块配置中设置默认账户
   - 域名可继承默认账户或使用自定义账户
   - 支持查看账户关联的域名列表
   - 标记默认账户

3. **域名管理**
   - 从 `Weline_Websites` 模块选择网站添加CDN域名
   - 一个网站对应一个域名（从网站配置中自动获取主域名）
   - 域名配置：适配器、Zone ID、账户关联、规则覆盖
   - 支持启用/禁用域名

4. **缓存清理**
   - 支持所有Cloudflare清理模式：
     - 清理所有缓存（purge_everything）
     - 按URL清理（purge_urls）
     - 按Host清理（purge_hosts）
     - 按Tag清理（purge_tags）
     - 按Cache Key清理（purge_cache_keys）
   - 提供事件 `Weline_Cdn::clear` 供其他模块调用
   - 支持命令行和HTTP API接口

5. **规则管理**
   - 全局默认规则：存储在 `etc/default-rules.json`
   - 规则合并：默认规则 + 域名覆盖规则
   - 规则导入：从Cloudflare导入现有规则
   - 规则推送：将合并后的规则推送到CDN
   - 添加域名时自动推送默认规则

6. **缓存预热**
   - 事件驱动：`Weline_Cdn::send_warmup` 事件收集预热URL
   - Provider机制：扫描模块的 `Cdn/WarmupProvider.php` 自动收集URL
   - 定时执行：每5分钟执行一次预热任务
   - URL解析：根据URL最长匹配解析对应的站点和域名
   - 站点间隔控制：每个域名的预热间隔可配置
   - 特殊Header：使用 `CF-Prewarm: t0k3n-xxxxx` 标记预热请求
   - 统计信息：URL计数、处理次数、成功/失败次数

7. **HTTP API**
   - 对外提供缓存刷新接口（需要认证）

### 事件系统

#### 1. Weline_Cdn::clear

**用途**：触发CDN缓存清理

**事件数据**：
```php
[
    'domain' => 'example.com', // 域名ID或名称（必需）
    'mode' => 'everything',     // 清理模式：everything|urls|hosts|tags|cache_keys
    'data' => []                // 模式相关的数据（如urls、hosts等）
]
```

**返回数据**：
```php
[
    'success' => true|false,
    'message' => '清理结果消息',
    'result' => [] // 详细结果
]
```

**使用示例**：
```php
$event = new Event('Weline_Cdn::clear', [
    'domain' => 'example.com',
    'mode' => 'everything'
]);
EventsManager::getInstance()->dispatch('Weline_Cdn::clear', $event);
```

#### 2. Weline_Cdn::send_warmup

**用途**：提交预热URL

**事件数据**：
```php
[
    'module' => 'Weline_Cms',        // 模块名（必需）
    'urls' => [                      // URL数组（必需）
        'https://example.com/page1',
        'https://example.com/page2',
        // 或
        [
            'url' => 'https://example.com/page3',
            'site_id' => 1,           // 可选
            'domain_id' => 1          // 可选
        ]
    ],
    'dedupe' => true                 // 是否去重（默认true）
]
```

**返回数据**：
```php
[
    'success' => true|false,
    'message' => '处理结果消息',
    'inserted_count' => 10,
    'updated_count' => 5
]
```

**使用示例**：
```php
$event = new Event('Weline_Cdn::send_warmup', [
    'module' => 'Weline_Cms',
    'urls' => ['https://example.com/page1', 'https://example.com/page2']
]);
EventsManager::getInstance()->dispatch('Weline_Cdn::send_warmup', $event);
```

### 适配器接口

所有CDN适配器必须实现 `Weline\Cdn\Api\AdapterInterface` 接口：

```php
interface AdapterInterface
{
    // 基本信息
    public function getAdapterCode(): string;
    public function getAdapterName(): string;
    
    // 缓存清理方法
    public function purgeEverything(string $zoneId, array $credentials): array;
    public function purgeUrls(string $zoneId, array $urls, array $credentials): array;
    public function purgeHosts(string $zoneId, array $hosts, array $credentials): array;
    public function purgeTags(string $zoneId, array $tags, array $credentials): array;
    public function purgeCacheKeys(string $zoneId, array $keys, array $credentials): array;
    
    // 规则管理方法
    public function getRules(string $zoneId, array $credentials): array;
    public function putRules(string $zoneId, array $rules, array $credentials): array;
    
    // Zone管理方法
    public function ensureZone(string $domain, array $credentials): array;
}
```

### 适配器扫描机制

1. 扫描已激活的模块
2. 查找 `{ModulePath}/Cdn/Adapter/*.php` 文件
3. 加载类并验证是否实现 `AdapterInterface`
4. 调用 `getAdapterCode()` 获取适配器代码
5. 注册到适配器列表

### 预热Provider机制

其他模块可以提供预热URL的Provider：

1. 创建文件：`{ModulePath}/Cdn/WarmupProvider.php`
2. 实现静态方法 `execute()`，返回URL数组：
```php
class WarmupProvider
{
    public static function execute(): array
    {
        return [
            'https://example.com/page1',
            'https://example.com/page2',
            // 或
            [
                'url' => 'https://example.com/page3',
                'site_id' => 1
            ]
        ];
    }
}
```

3. 预热任务会自动扫描并调用该方法收集URL

## 命令行工具设计

### 1. cdn:domain:add

添加CDN域名（从网站创建）

```bash
php bin/w cdn:domain:add --site=main [--adapter=cloudflare] [--zone-id=xxx]
```

**参数说明**：
- `--site`：网站代码（必需）
- `--adapter`：适配器代码（可选，默认cloudflare）
- `--zone-id`：Zone ID（可选，会自动查询）

### 2. cdn:rules:import

导入规则

```bash
php bin/w cdn:rules:import --domain=example.com
```

**参数说明**：
- `--domain`：域名ID或名称（必需）

### 3. cdn:cache:clear

清理缓存

```bash
php bin/w cdn:cache:clear --domain=example.com --mode=everything [--urls=url1,url2] [--hosts=host1,host2] [--tags=tag1,tag2]
```

**参数说明**：
- `--domain`：域名ID或名称（必需）
- `--mode`：清理模式（必需）
- `--urls`：URL列表（mode=urls时必需，逗号分隔）
- `--hosts`：Host列表（mode=hosts时必需，逗号分隔）
- `--tags`：Tag列表（mode=tags时必需，逗号分隔）
- `--keys`：Cache Key列表（mode=cache_keys时必需，逗号分隔）

## 后台界面设计

### 1. 账户管理

- 列表页：显示所有账户，标记默认账户
- 创建/编辑页：配置适配器、名称、凭据、默认标记
- 关联域名：显示使用该账户的域名列表

### 2. 域名管理

- 列表页：显示所有域名，支持筛选
- 创建页：从网站选择、配置适配器、Zone ID、账户关联
- 编辑页：修改配置、导入/导出规则、手动清理缓存

### 3. 规则管理

- 全局规则：编辑默认规则（etc/default-rules.json）
- 域名规则：查看/编辑域名规则覆盖
- 规则导入：从CDN导入现有规则
- 规则推送：将规则推送到CDN

### 4. 预热管理

- 列表页：显示所有预热URL，支持按模块筛选
- 统计信息：按模块统计URL数量、处理次数、成功率
- 手动触发：手动执行预热任务

## 默认规则配置

`etc/default-rules.json` 包含框架相关的默认缓存规则：

- 静态资源缓存（/static/, /media/）
- 后台路径不缓存（/admin/, /backend/）

添加域名时会自动推送这些规则到Cloudflare。

## 扩展适配器示例

其他模块可以贡献CDN适配器：

```php
// app/code/Vendor/Module/Cdn/Adapter/MyCdn.php
namespace Vendor\Module\Cdn\Adapter;

use Weline\Cdn\Api\AdapterInterface;

class MyCdn implements AdapterInterface
{
    public function getAdapterCode(): string
    {
        return 'mycdn';
    }

    public function getAdapterName(): string
    {
        return 'My CDN';
    }

    // 实现其他接口方法...
}
```

模块激活后，该适配器会自动被发现并可用。

## 注意事项

1. **账户凭据安全**：凭据以JSON格式存储在数据库中，建议在生产环境加密存储
2. **API调用频率**：注意Cloudflare API的调用频率限制，预热任务应遵守站点间隔
3. **预热Header**：使用特殊Header `CF-Prewarm` 标记预热请求，需要在Cloudflare配置Skip规则避免统计和日志
4. **错误处理**：所有API调用都应有友好的错误提示，特别是账户未配置的情况
5. **扩展性**：保持接口和服务的扩展性，方便后续添加新的CDN提供商
