# 项目配置文件使用指南

## 简介

除了在后台管理界面配置同步映射外，Weline_Async 模块还支持在项目根目录（BP）下创建 `weline-async.json` 配置文件。这是**默认的项目库同步监控配置**，只要该文件存在，就会自动运行，不受后台配置影响。

## 配置文件位置

在项目根目录（BP）下创建 `weline-async.json` 文件：

```
项目根目录/
├── weline-async.json  ← 配置文件位置
├── app/
├── bin/
└── ...
```

## 配置文件格式

`weline-async.json` 文件采用 JSON 格式，包含以下字段：

```json
{
  "host": {
    "host": "192.168.1.100",
    "port": 22,
    "user": "root",
    "password": "your_password",
    "key_path": "/path/to/id_rsa"
  },
  "mapping": {
    "local_path": "/path/to/local/directory",
    "remote_path": "/path/to/remote/directory",
    "exclude_patterns": [
      "node_modules",
      ".git",
      "*.log",
      ".idea"
    ]
  }
}
```

### 字段说明

#### host（主机配置）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| host | string | 是 | 远程服务器地址（IP或域名） |
| port | integer | 否 | SSH端口，默认22 |
| user | string | 是 | SSH用户名 |
| password | string | 否* | SSH密码（与key_path二选一） |
| key_path | string | 否* | SSH私钥文件路径（与password二选一） |

*注意：`password` 和 `key_path` 至少需要提供一个。

#### mapping（目录映射配置）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| local_path | string | 否 | 本地目录的完整路径。如果未指定或为 `./`，默认使用项目根目录（BP） |
| remote_path | string | 是 | 远程服务器上的目录路径 |
| include_paths | array | 否 | 要同步的目录列表（相对于 local_path）。如果指定，只同步这些目录；如果未指定，同步整个 local_path |
| exclude_patterns | array | 否 | 排除模式列表（支持通配符）。用于排除不需要同步的文件/目录 |

## 配置示例

### 示例1：使用密码认证

```json
{
  "host": {
    "host": "example.com",
    "port": 22,
    "user": "deploy",
    "password": "your_secure_password"
  },
  "mapping": {
    "local_path": "/var/www/project",
    "remote_path": "/home/deploy/project",
    "exclude_patterns": [
      "node_modules",
      ".git",
      "*.log",
      "vendor"
    ]
  }
}
```

### 示例2：使用SSH密钥认证

```json
{
  "host": {
    "host": "192.168.1.100",
    "port": 22,
    "user": "root",
    "key_path": "/home/user/.ssh/id_rsa"
  },
  "mapping": {
    "local_path": "/home/user/projects/myapp",
    "remote_path": "/var/www/myapp",
    "exclude_patterns": [
      "node_modules",
      ".git",
      ".env",
      "*.cache"
    ]
  }
}
```

### 示例3：最小配置（使用项目根目录）

```json
{
  "host": {
    "host": "192.168.1.100",
    "user": "root",
    "password": "password"
  },
  "mapping": {
    "remote_path": "/var/www/html/project"
  }
}
```

**注意**：如果未指定 `local_path`，默认使用项目根目录（BP）。

### 示例4：指定要同步的目录（include_paths）

```json
{
  "host": {
    "host": "192.168.1.100",
    "user": "root",
    "password": "password"
  },
  "mapping": {
    "local_path": "./",
    "remote_path": "/var/www/html/project",
    "include_paths": [
      "app/code/Weline/Async",
      "view/templates",
      "pub/static"
    ],
    "exclude_patterns": [
      "node_modules",
      ".git",
      "*.log"
    ]
  }
}
```

**说明**：只同步 `include_paths` 中指定的目录，其他目录不会被同步。

## 使用方法

### 1. 创建配置文件

在项目根目录创建 `weline-async.json` 文件，填写配置信息。

### 2. 启动同步

运行以下命令启动同步：

```bash
php bin/w async:start
```

系统会自动检测 `weline-async.json` 文件，如果存在则启动项目配置的同步。

### 3. 查看状态

```bash
php bin/w async:status
```

状态输出中会显示项目配置的同步状态，标识为 `mapping_id: project`。

### 4. 停止同步

```bash
php bin/w async:stop
```

会同时停止项目配置和后台配置的所有同步。

## 特性说明

### 1. 优先级

- **项目配置文件优先**：如果 `weline-async.json` 存在，会**始终运行**，不受后台配置影响
- **独立运行**：项目配置的同步独立于后台配置的映射，两者可以同时运行
- **自动启动**：使用 `async:start` 命令时，会自动检测并启动项目配置的同步

### 2. 根目录同步特殊规则

- **默认根目录**：如果 `local_path` 未指定或为 `./`，默认使用项目根目录（BP）
- **include_paths 的作用**：
  - 如果指定了 `include_paths`，**只同步这些目录**，其他目录不会被同步
  - 如果未指定 `include_paths`，同步整个 `local_path`（或整个项目根目录）
- **exclude_patterns 的作用**：
  - 用于排除不需要同步的文件/目录（如 `node_modules`、`.git` 等）
  - 即使指定了 `include_paths`，`exclude_patterns` 仍然生效

### 3. 与后台配置的关系

- 项目配置和后台配置**互不影响**，可以同时存在
- 项目配置主要用于**项目级别的默认同步**（如开发环境同步到测试环境）
- 后台配置用于**多主机、多映射的复杂场景**

### 4. 守护进程支持

守护进程模式会自动监控项目配置的同步状态，如果检测到未运行会自动重启：

```bash
php bin/w async:daemon
```

## 安全注意事项

### 1. 密码安全

⚠️ **重要**：`weline-async.json` 文件包含敏感信息（SSH密码），请确保：

- 文件已添加到 `.gitignore`，不会被提交到版本控制系统
- 文件权限设置为 `600`（仅所有者可读写）：
  ```bash
  chmod 600 weline-async.json
  ```
- 不要将配置文件分享给他人

### 2. 密钥路径

如果使用 SSH 密钥认证，确保：

- 密钥文件路径正确
- 密钥文件权限正确（通常为 `600`）
- 密钥文件不会被意外删除

## 故障排除

### 配置文件格式错误

如果配置文件格式不正确，会显示错误信息：

```
错误: weline-async.json 配置文件格式错误: ...
```

检查 JSON 格式是否正确，可以使用在线 JSON 验证工具。

### 同步未启动

1. 检查配置文件是否存在
2. 检查配置文件格式是否正确
3. 检查 SSH 连接信息是否正确
4. 查看日志文件：`var/async/logs/mapping_project.log`

### 文件未同步

1. 检查本地路径是否存在
2. 检查远程路径是否存在
3. 检查是否指定了 `include_paths`，该文件是否在包含列表中
4. 检查排除模式是否匹配了该文件
5. 检查 SSH 连接是否正常

## 最佳实践

1. **使用密钥认证**：推荐使用 SSH 密钥而不是密码，更安全
2. **合理设置 include_paths**：如果只需要同步部分目录，使用 `include_paths` 指定，避免同步不必要的文件
3. **合理设置排除模式**：排除不需要同步的文件（如 `node_modules`、`.git` 等）
4. **定期检查日志**：查看 `var/async/logs/mapping_project.log` 了解同步状态
5. **使用守护进程**：在生产环境使用守护进程模式，确保同步持续运行

## 相关命令

- `php bin/w async:start` - 启动所有同步（包括项目配置）
- `php bin/w async:stop` - 停止所有同步（包括项目配置）
- `php bin/w async:restart` - 重启所有同步（包括项目配置）
- `php bin/w async:status` - 查看所有同步状态（包括项目配置）
- `php bin/w async:daemon` - 守护进程模式（自动监控和重启）
