# Geo定位通道选择指南

## 📋 概述

Geo定位模块支持多个免费和付费的定位服务通道。本文档详细说明各通道的特点、限制和适用场景，帮助您选择合适的通道配置。

| 通道 | 免费 | API Key | 配额限制 | HTTPS | 优先级 | 推荐度 | 官网 |
|------|------|---------|----------|-------|--------|--------|------|
| **ip-api.com** | ✅ | ❌ | 45/分钟 | ❌ | 1 | ⭐⭐⭐⭐⭐ | [ip-api.com](http://ip-api.com) |
| **geojs.io** | ✅ | ❌ | 无限制 | ✅ | 2 | ⭐⭐⭐⭐⭐ | [geojs.io](https://www.geojs.io) |
| **ipwhois.app** | ✅ | ❌ | 有限制 | ✅ | 3 | ⭐⭐⭐⭐ | [ipwhois.app](https://ipwhois.app) |
| **ipinfo.io** | ✅ | ✅ | 50,000/月 | ✅ | 4 | ⭐⭐⭐⭐ | [ipinfo.io](https://ipinfo.io) |
| **ipapi.co** | ✅ | ✅ | 1,000/天 | ✅ | 5 | ⭐⭐⭐ | [ipapi.co](https://ipapi.co) |

## 📖 详细通道说明

### 1. ip-api.com（推荐，默认启用）

**基本信息**：
- **官网**: [http://ip-api.com](http://ip-api.com)
- **API文档**: [http://ip-api.com/docs/api:json](http://ip-api.com/docs/api:json)
- **状态**: 完全免费，无需注册
- **优先级**: 1（最高）

**特点**：
- ✅ 完全免费，无需注册和API Key
- ✅ 稳定性高，响应速度快
- ✅ 提供详细的地理位置信息
  - 国家、国家代码
  - 地区、城市
  - 经纬度坐标
  - 时区信息
  - ISP信息
- ⚠️ 速率限制：45请求/分钟（免费版）
- ⚠️ 仅支持HTTP（不支持HTTPS）

**适用场景**：
- 中小型应用
- 请求量不大的场景（< 45请求/分钟）
- 不需要HTTPS的场景

**配置示例**：
```php
'ip-api.com' => [
    'enabled' => true,
    'priority' => 1,
    'timeout' => 5
]
```

### 2. geojs.io（推荐，默认启用）

**基本信息**：
- **官网**: [https://www.geojs.io](https://www.geojs.io)
- **API文档**: [https://www.geojs.io/docs/v1/endpoints/geo/](https://www.geojs.io/docs/v1/endpoints/geo/)
- **状态**: 完全免费，无需注册
- **优先级**: 2

**特点**：
- ✅ 完全免费，无需注册和API Key
- ✅ **无配额限制**（重要优势）
- ✅ 支持HTTPS
- ✅ 响应速度快
- ✅ 提供基本的地理位置信息
- ⚠️ 数据精度相对较低

**适用场景**：
- 需要高可用性的应用
- 请求量较大的场景
- 需要HTTPS支持的应用
- 对精度要求不高的场景

**配置示例**：
```php

    'enabled' => true,
    'priority' => 2,
    'timeout' => 5

```

### 3. ipwhois.app（默认启用）

**基本信息**：
- **官网**: [https://ipwhois.app](https://ipwhois.app)
- **API文档**: [https://ipwhois.io/documentation](https://ipwhois.io/documentation)
- **状态**: 免费，无需注册
- **优先级**: 3

**特点**：
- ✅ 免费，无需API Key
- ✅ 支持HTTPS
- ✅ 提供IP地址的详细信息
  - 地理位置信息
  - ISP信息
  - 组织信息
- ⚠️ 有配额限制（具体限制需查看官网）
- ⚠️ 响应速度中等

**适用场景**：
- 作为备用通道
- 提供额外的fallback选项
- 需要详细IP信息的场景

**配置示例**：
```php

    'enabled' => true,
    'priority' => 3,
    'timeout' => 5

```

### 4. ipinfo.io（可选，需要API Key）

**基本信息**：
- **官网**: [https://ipinfo.io](https://ipinfo.io)
- **API文档**: [https://ipinfo.io/developers](https://ipinfo.io/developers)
- **注册**: [https://ipinfo.io/signup](https://ipinfo.io/signup)
- **状态**: 免费版可用，需要注册
- **优先级**: 4

**特点**：
- ✅ 免费版：**50,000请求/月**
- ✅ 稳定性高，数据准确
- ✅ 支持HTTPS
- ✅ 提供详细的IP信息
  - 地理位置信息
  - ISP信息
  - 组织信息
  - ASN信息
  - 公司信息
- ⚠️ 需要注册账号获取API Key
- ⚠️ 免费版有配额限制

**适用场景**：
- 需要更高配额的应用（50,000请求/月）
- 需要更准确数据的应用
- 需要详细IP信息的场景

**配置步骤**：
1. 访问 [https://ipinfo.io/signup](https://ipinfo.io/signup) 注册账号
2. 获取API Key
3. 在 `etc/env.php` 中配置：
```php
'ipinfo.io' => [
    'enabled' => true,
    'priority' => 4,
    'api_key' => 'your-api-key-here',
    'timeout' => 5
]
```

## 🎯 选择建议

### 小型应用（< 1,000请求/天）

**推荐配置**：使用默认的3个免费通道
- ip-api.com（优先级1）
- geojs.io（优先级2）
- ipwhois.app（优先级3）

**优势**：
- 无需注册和配置
- 完全免费
- 自动fallback，高可用性

### 中型应用（1,000 - 50,000请求/月）

**推荐配置**：启用ipinfo.io
- ip-api.com（优先级1）
- geojs.io（优先级2）
- ipwhois.app（优先级3）
- **ipinfo.io（优先级4）** - 需要注册获取API Key

**优势**：
- 更高的配额（50,000请求/月）
- 更准确的数据
- 保持fallback机制

### 大型应用（> 50,000请求/月）

**推荐方案**：
1. **使用付费版本**：升级ipinfo.io或ipapi.co到付费版本
2. **自建定位服务**：使用本地数据库或缓存
3. **混合方案**：付费通道 + 免费通道fallback

## 🔄 Fallback机制

系统会自动按优先级顺序尝试各个通道：

1. **尝试优先级1的通道**（ip-api.com）
2. **如果失败，自动尝试优先级2的通道**（geojs.io）
3. **如果还失败，尝试优先级3的通道**（ipwhois.app）
4. **继续尝试后续启用的通道**
5. **如果所有通道都失败，返回错误信息**

**优势**：
- 自动故障转移，提高可用性
- 无需手动干预
- 透明的用户体验

## ❓ 常见问题

### Q: 我应该使用哪个通道？

**A**: 对于大多数应用，使用默认的3个免费通道即可。如果请求量较大（> 1,000请求/天），建议启用ipinfo.io。

### Q: 需要注册API Key吗？

**A**: 默认的3个通道（ip-api.com, geojs.io, ipwhois.app）无需注册。只有ipinfo.io和ipapi.co需要注册获取API Key。

### Q: 如何查看使用的通道？

**A**: 定位结果中的 `source` 字段会显示使用的通道名称。

### Q: 可以只使用一个通道吗？

**A**: 可以，但不推荐。建议至少启用2-3个通道，确保fallback机制正常工作。
