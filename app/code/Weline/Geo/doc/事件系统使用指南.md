# Geo 事件系统使用指南

## 📖 概述

Geo 模块提供了完整的事件系统，允许其他模块通过触发事件来添加、更新或删除Feed条目，实现模块间的解耦。

## 🎯 设计理念

- **解耦设计**：其他模块不需要直接调用Feed模块的代码
- **事件驱动**：通过事件系统实现模块间通信
- **灵活扩展**：可以轻松添加新的事件监听器
- **自动处理**：事件触发后自动创建/更新Feed条目

---

## 📋 可用事件

### 1. feed_item_add - 添加Feed条目

**事件名称**：`Weline_Geo::feed_item_add`

**触发时机**：需要向Feed添加新条目时

**事件数据格式**：
```php
[
    'feed_id' => 1,              // 必需：Feed ID
    'item_type' => 'article',    // 必需：条目类型（article, product, page等）
    'item_id' => 123,            // 必需：条目ID（源数据ID）
    'title' => '文章标题',       // 可选：标题
    'content' => '文章内容',     // 可选：内容
    'url' => 'https://...',      // 可选：URL
    'metadata' => [...],          // 可选：元数据（数组）
    'is_published' => 1,         // 可选：是否发布（默认1）
    'published_at' => time(),    // 可选：发布时间（默认当前时间）
]
```

---

### 2. feed_item_update - 更新Feed条目

**事件名称**：`Weline_Geo::feed_item_update`

**触发时机**：需要更新Feed条目时

**事件数据格式**：与 `feed_item_add` 相同

**说明**：如果条目不存在，会自动创建

---

### 3. feed_item_delete - 删除Feed条目

**事件名称**：`Weline_Geo::feed_item_delete`

**触发时机**：需要删除Feed条目时

**事件数据格式**：
```php
[
    'feed_id' => 1,              // 必需：Feed ID
    'item_type' => 'article',    // 必需：条目类型
    'item_id' => 123,            // 必需：条目ID
]
```

## 🚀 使用方法

### 方法1：使用FeedEventDispatcher服务（推荐）

```php
use Weline\Geo\Service\FeedEventDispatcher;
use Weline\Framework\Manager\ObjectManager;

// 获取事件分发服务
/** @var FeedEventDispatcher $dispatcher */
$dispatcher = ObjectManager::getInstance(FeedEventDispatcher::class);

// 添加Feed条目
$dispatcher->dispatchFeedItemAdd(
    feedId: 1,
    itemType: 'article',
    itemId: 123,
    itemData: [
        'title' => '文章标题',
        'content' => '文章内容',
        'url' => 'https://example.com/article/123',
        'metadata' => [
            'author' => '作者名',
            'tags' => ['标签1', '标签2'],
        ],
    ]
);

// 更新Feed条目
$dispatcher->dispatchFeedItemUpdate(
    feedId: 1,
    itemType: 'article',
    itemId: 123,
    itemData: [
        'title' => '更新后的标题',
        'content' => '更新后的内容',
    ]
);

// 删除Feed条目
$dispatcher->dispatchFeedItemDelete(
    feedId: 1,
    itemType: 'article',
    itemId: 123
);

// 批量添加到多个Feed
$dispatcher->dispatchFeedItemAddToFeeds(
    feedIds: [1, 2, 3],
    itemType: 'article',
    itemId: 123,
    itemData: [
        'title' => '文章标题',
        'content' => '文章内容',
        'url' => 'https://example.com/article/123',
    ]
);
```

### 方法2：直接使用EventsManager

```php
use Weline\Framework\Event\EventsManager;
use Weline\Framework\Manager\ObjectManager;

// 获取事件管理器
/** @var EventsManager $eventsManager */
$eventsManager = ObjectManager::getInstance(EventsManager::class);

// 添加Feed条目
```

## 💡 实际应用示例

### 示例1：CMS模块 - 文章发布后添加到Feed

```php
// app/code/Weline/Cms/Controller/Backend/Article.php

namespace Weline\Cms\Controller\Backend;

use Weline\Geo\Service\FeedEventDispatcher;
use Weline\Framework\Manager\ObjectManager;

class Article extends BackendController
{
    public function save(): string
    {
        // ... 保存文章逻辑 ...
        
        // 文章保存成功后，触发Feed事件
        if ($article->isPublished()) {
            /** @var FeedEventDispatcher $dispatcher */
            $dispatcher = ObjectManager::getInstance(FeedEventDispatcher::class);
            
            // 获取所有内容类型的Feed ID
            $feedIds = $this->getContentFeedIds();
            
            // 添加到Feed
            $dispatcher->dispatchFeedItemAddToFeeds(
                feedIds: $feedIds,
                itemType: 'article',
                itemId: $article->getId(),
                itemData: [
                    'title' => $article->getTitle(),
                    'content' => $article->getContent(),
                    'url' => $article->getUrl(),
                    'metadata' => [
                        'author' => $article->getAuthor(),
                        'category' => $article->getCategory(),
                        'tags' => $article->getTags(),
                    ],
                    'published_at' => $article->getPublishedAt(),
                ]
            );
        }
        
        return $this->jsonResponse(true, '保存成功');
    }
    
    protected function getContentFeedIds(): array
    {
        // 查询所有内容类型的Feed ID
        // 这里可以根据业务逻辑查询
        return [1, 2]; // 示例
    }
}
```

### 示例2：产品模块 - 产品更新后更新Feed

```php
// app/code/Weline/Product/Model/Product.php

namespace Weline\Product\Model;

use Weline\Geo\Service\FeedEventDispatcher;
use Weline\Framework\Manager\ObjectManager;

class Product extends Model
{
    public function save_after(): void
    {
        // 产品保存后触发Feed更新事件
        if ($this->isEnabled()) {
            /** @var FeedEventDispatcher $dispatcher */
            $dispatcher = ObjectManager::getInstance(FeedEventDispatcher::class);
            
            // 获取所有产品类型的Feed ID
            $feedIds = $this->getProductFeedIds();
            
            // 更新Feed条目
            $dispatcher->dispatchFeedItemUpdateToFeeds(
                feedIds: $feedIds,
                itemType: 'product',
                itemId: $this->getId(),
                itemData: [
                    'title' => $this->getName(),
                    'content' => $this->getDescription(),
                    'url' => $this->getUrl(),
                    'metadata' => [
                        'price' => $this->getPrice(),
                        'sku' => $this->getSku(),
                        'category' => $this->getCategory(),
                    ],
                ]
            );
        }
    }
}
```

### 示例3：监听模型保存事件自动添加Feed

```php
// app/code/Weline/YourModule/etc/event.xml

<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    
    <!-- 监听文章模型保存后事件 -->
    <event name="Weline_Cms_Model_Article_model_save_after">
        <observer name="Weline_YourModule::article_save_add_to_feed" 
                  instance="Weline\YourModule\Observer\ArticleSaveAfter" />
    </event>
</config>

// app/code/Weline/YourModule/Observer/ArticleSaveAfter.php

namespace Weline\YourModule\Observer;

use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Event\EventInterface;
use Weline\Geo\Service\FeedEventDispatcher;
use Weline\Framework\Manager\ObjectManager;

class ArticleSaveAfter implements ObserverInterface
{
    public function execute(EventInterface $event): void
    {
        $data = $event->getData();
        $article = $data['model'] ?? null;
        
        if (!$article || !$article->isPublished()) {
            return;
        }
        
        /** @var FeedEventDispatcher $dispatcher */
        $dispatcher = ObjectManager::getInstance(FeedEventDispatcher::class);
        
        // 添加到Feed ID为1的内容Feed
        $dispatcher->dispatchFeedItemAdd(
            feedId: 1,
            itemType: 'article',
            itemId: $article->getId(),
            itemData: [
                'title' => $article->getTitle(),
                'content' => $article->getContent(),
                'url' => $article->getUrl(),
            ]
        );
    }
}
```

## 🔧 事件数据字段说明

### 必需字段

- **feed_id** (int)：Feed ID，指定要添加到的Feed
- **item_type** (string)：条目类型，用于分类（如：article, product, page等）
- **item_id** (int)：条目ID，源数据的ID

### 可选字段

- **title** (string)：标题
- **content** (string)：内容（纯文本）
- **url** (string)：URL地址
- **metadata** (array)：元数据，可包含：
  - `content_html`：HTML内容
  - `author`：作者
  - `tags`：标签数组
  - `category`：分类
  - 其他自定义字段
- **is_published** (int)：是否发布（1=发布，0=未发布，默认1）
- **published_at** (int)：发布时间戳（默认当前时间）

---

## ⚡ 自动推送机制

如果Feed配置了以下条件，事件触发后会自动推送：

1. **Feed已启用**（`is_enabled = 1`）
2. **启用自动推送**（`is_auto_push = 1`）
3. **更新频率为实时**（`update_frequency = 'realtime'`）

满足以上条件时，添加或更新Feed条目后会自动推送到所有启用的平台。

## 📝 最佳实践

### 1. 使用FeedEventDispatcher服务

推荐使用 `FeedEventDispatcher` 服务，它提供了类型安全的方法和更好的代码可读性。

### 2. 批量操作

如果需要添加到多个Feed，使用 `dispatchFeedItemAddToFeeds` 方法，避免循环调用。

### 3. 错误处理

事件系统会自动记录错误日志，但建议在关键业务逻辑中添加try-catch。

### 4. 性能考虑

- 批量操作时使用批量方法
- 避免在循环中频繁触发事件
- 对于大量数据，考虑使用队列异步处理

```bash

```

```bash
tail -f var/log/php_error.log
```
