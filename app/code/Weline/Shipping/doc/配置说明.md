## 快递公司配置

### 手动类型配置

**适用场景**：没有API接口的快递公司

**配置步骤**：

1. 进入后台：**配送管理 > 配送系统**，切换至「快递公司」Tab
2. 点击"新增"
3. 填写信息：
   - **快递公司代码**：唯一标识，建议使用大写字母（如：SF、YTO、ZTO）
   - **快递公司名称**：显示名称（如：顺丰速运、圆通速递）
   - **类型**：选择 `manual`
   - **追踪URL模板**：必填，格式：
     ```
     https://www.sf-express.com/track/{tracking_number}
     ```
     或
     ```
     https://www.yto.net.cn/track/{tracking_number}
     ```
   - **追踪支持状态**：选择 `not_supported`
   - **是否启用**：勾选

**URL模板变量**：
- `{tracking_number}` - 会被实际物流单号替换

### API类型配置

**适用场景**：有API接口的快递公司

**配置步骤**：

1. 进入后台：**配送管理 > 配送系统**，切换至「快递公司」Tab
2. 点击"新增"
3. 填写信息：
   - **快递公司代码**：唯一标识
   - **快递公司名称**：显示名称
   - **类型**：选择 `api`
   - **追踪URL模板**：必填，即使有API也要配置
   - **追踪API端点**：API接口地址
   - **API请求方法**：GET 或 POST
   - **API配置**：JSON格式，包含认证信息
   - **追踪支持状态**：选择 `supported`
   - **是否启用**：勾选

**API配置示例**：

```json
{
  "api_key": "your_api_key",
  "api_secret": "your_api_secret",
  "customer": "your_customer_code",
  "request_format": "json",
  "response_format": "json"
}
```

**注意事项**：
- API配置中的敏感信息（如密钥）请妥善保管
- 建议使用环境变量或配置中心管理API密钥
- API端点需要支持HTTPS

## 费用模板配置

### 按重量计算

**适用场景**：按重量计费的快递

**配置参数**：
- **计算类型**：选择 `weight`
- **基础费用**：起步价（如：10.00）
- **重量单位**：kg 或 lb
- **重量费率**：每单位重量的费用（如：5.00）

**计算示例**：
- 订单重量：2.5kg
- 基础费用：10元
- 重量费率：5元/kg
- **总费用**：10 + 2.5 × 5 = 22.5元

### 按体积计算

**适用场景**：按体积计费的快递（如大件物品）

**配置参数**：
- **计算类型**：选择 `volume`
- **基础费用**：起步价
- **体积单位**：m³ 或 ft³
- **体积费率**：每单位体积的费用

**计算示例**：
- 订单体积：0.5m³
- 基础费用：20元
- 体积费率：30元/m³
- **总费用**：20 + 0.5 × 30 = 35元

### 按件数计算

**适用场景**：按件数计费的快递

**配置参数**：
- **计算类型**：选择 `quantity`
- **基础费用**：起步价
- **件数费率**：每件的费用

**计算示例**：
- 订单件数：5件
- 基础费用：8元
- 件数费率：2元/件
- **总费用**：8 + 5 × 2 = 18元

### 固定费用

**适用场景**：固定配送费用

**配置参数**：
- **计算类型**：选择 `fixed`
- **基础费用**：固定费用

**计算示例**：
- 基础费用：15元
- **总费用**：15元（无论重量、体积、件数）

### 混合模式

**适用场景**：需要同时考虑多种因素的复杂计费

**配置参数**：
- **计算类型**：选择 `mixed`
- **基础费用**：起步价
- **混合模式配置**：JSON格式

**配置示例**：

```json
{
  "base_fee": 10.00,
  "weight": {
    "enabled": true,
    "rate": 5.00,
    "unit": "kg"
  },
  "volume": {
    "enabled": true,
    "rate": 20.00,
    "unit": "m3"
  },
  "quantity": {
    "enabled": false
  }
}
```

**计算逻辑**：
- 如果同时启用重量和体积，取两者中较大的值
- 或根据业务需求自定义计算逻辑

## 免邮规则配置

### 订单金额免邮

**配置示例**：

- **规则名称**：订单满100元免邮
- **规则代码**：FREE_SHIP_100
- **条件类型**：订单金额
- **最小订单金额**：100.00
- **优先级**：10

### 会员等级免邮

**配置示例**：

- **规则名称**：VIP会员免邮
- **规则代码**：VIP_FREE_SHIP
- **条件类型**：会员等级
- **会员等级ID列表**：`[1, 2, 3]`（JSON数组）
- **优先级**：20

### 地区免邮

**配置示例**：

- **规则名称**：本地区免邮
- **规则代码**：LOCAL_FREE_SHIP
- **条件类型**：地区
- **地区ID列表**：`[10, 20, 30]`（JSON数组）
- **优先级**：15

### 优惠券免邮

**配置示例**：

- **规则名称**：免邮优惠券
- **规则代码**：COUPON_FREE_SHIP
- **条件类型**：优惠券
- **优惠券代码列表**：`["FREE_SHIP", "SHIP50"]`（JSON数组）
- **优先级**：25

### 混合条件免邮

**配置示例**：

- **规则名称**：VIP会员且订单满50元免邮
- **规则代码**：VIP_50_FREE_SHIP
- **条件类型**：混合条件
- **混合条件配置**：

```json
{
  "logic": "AND",
  "conditions": [
    {
      "type": "order_amount",
      "operator": ">=",
      "value": 50.00
    },
    {
      "type": "member_level",
      "operator": "IN",
      "value": [1, 2]
    }
  ]
}
```

**逻辑说明**：
- `logic: "AND"` - 所有条件都满足
- `logic: "OR"` - 任一条件满足

**操作符说明**：
- `>=` - 大于等于
- `<=` - 小于等于
- `>` - 大于
- `<` - 小于
- `=` 或 `==` - 等于
- `IN` - 在列表中

## 常见配置问题

### Q1: 如何配置多个快递公司？

A: 在后台逐个添加快递公司，每个快递公司使用唯一的代码标识。

### Q2: 如何实现阶梯计费？

A: 使用混合模式，配置多个条件，或创建多个费用模板，在配送服务中根据订单金额选择不同的模板。

### Q3: 如何配置国际配送？

A: 
1. 添加国际地区数据
2. 创建国际配送区域
3. 配置国际快递公司
4. 创建国际配送服务

### Q4: 如何实现按距离计费？

A: 目前不支持直接按距离计费，可以通过以下方式实现：
1. 创建多个配送区域（按距离划分）
2. 为每个区域配置不同的费用模板
3. 根据地址匹配对应的区域和费用

### Q5: 如何配置特殊时段配送？

A: 可以创建多个配送服务，在服务名称中标注时段，用户选择时可以看到不同的服务选项。
