## 模块概述

Shipping模块是WelineFramework的配送管理系统，提供完整的配送服务配置、费用计算、免邮规则和物流跟踪功能。

### 核心功能

- **全球地区管理**：支持国家、省/州、市、区县层级结构
- **配送区域配置**：灵活配置配送覆盖区域
- **快递公司管理**：支持手动和API两种类型的快递公司
- **费用计算**：支持重量、体积、件数、固定费用和混合模式
- **免邮规则**：支持订单金额、会员等级、地区、优惠券等多种条件
- **物流跟踪**：统一的物流跟踪接口，支持API和手动类型

## 功能特性

### 1. 地区管理

- 支持层级结构（国家 → 省/州 → 市 → 区县）
- 关联i18n国家数据
- 支持地区导入和同步
- 支持邮政编码格式配置

### 2. 配送区域

- 可配置多个地区组合
- 支持精确匹配和模糊匹配
- 自动地址匹配功能

### 3. 快递公司

- **手动类型**：配置追踪URL模板，用户跳转到快递公司官网查询
- **API类型**：对接第三方物流API，实时查询物流信息
- **强制追踪支持**：所有快递公司必须配置追踪功能

### 4. 费用计算

支持多种计算模式：

- **按重量**：`费用 = 基础费用 + 重量 × 重量费率`
- **按体积**：`费用 = 基础费用 + 体积 × 体积费率`
- **按件数**：`费用 = 基础费用 + 件数 × 件数费率`
- **固定费用**：`费用 = 基础费用`
- **混合模式**：可同时启用多种计算方式

### 5. 免邮规则

支持多种免邮条件：

- **订单金额**：订单金额达到指定金额免邮
- **会员等级**：指定会员等级免邮
- **地区**：指定地区免邮
- **优惠券**：使用指定优惠券免邮
- **混合条件**：支持AND/OR逻辑组合

### 6. 物流跟踪

- **统一接口**：所有快递公司使用统一的查询接口
- **API类型**：实时查询物流信息，自动更新状态
- **手动类型**：返回追踪链接，引导用户到官网查询
- **不支持追踪**：返回友好的提示信息

## 快速开始

### 1. 安装模块

```bash
# 运行模块升级命令
php bin/w module:upgrade

# 或指定模块
php bin/w module:upgrade --module Weline_Shipping
```

### 2. 初始化地区数据

进入后台：**配送系统 > 地区管理**

1. 点击"从i18n同步"按钮，导入国家数据
2. 手动添加省/州、市、区县数据
3. 或使用API批量导入地区数据

### 3. 配置快递公司

进入后台：**配送系统 > 快递公司**

1. 点击"新增"按钮
2. 填写快递公司信息：
   - **快递公司代码**：唯一标识（如：SF、YTO）
   - **快递公司名称**：显示名称（如：顺丰速运）
   - **类型**：选择manual或api
   - **追踪URL模板**：必填，格式如：`https://www.sf-express.com/track/{tracking_number}`
   - **API配置**：API类型需要配置API端点和认证信息

### 4. 创建费用模板

进入后台：**配送系统 > 费用模板**

1. 点击"新增"按钮
2. 选择计算类型（重量/体积/件数/固定/混合）
3. 配置费率参数
4. 保存模板

**示例：按重量计算**

- 基础费用：10元
- 重量单位：kg
- 重量费率：5元/kg

计算：订单重量2kg，费用 = 10 + 2 × 5 = 20元

### 5. 配置免邮规则

进入后台：**配送系统 > 免邮规则**

1. 点击"新增"按钮
2. 选择条件类型
3. 配置条件参数
4. 设置优先级（数字越大优先级越高）

**示例：订单金额满100元免邮**

- 条件类型：订单金额
- 最小订单金额：100

### 6. 创建配送区域

进入后台：**配送系统 > 配送区域**

1. 点击"新增"按钮
2. 填写区域名称和代码
3. 选择包含的地区
4. 保存区域

### 7. 配置配送服务

进入后台：**配送系统 > 配送服务**

1. 点击"新增"按钮
2. 选择快递公司
3. 选择配送区域
4. 选择费用模板（可选）
5. 选择免邮规则（可选）
6. 配置预计配送天数
7. 保存服务

## 地区管理

### 地区层级结构

```
国家 (country)
  └── 省/州 (province)
      └── 市 (city)
          └── 区县 (district)
```

### 添加地区

**后台操作**：

1. 进入 **配送系统 > 地区管理**
2. 选择父级地区（如要添加省，选择国家）
3. 点击"新增"按钮
4. 填写地区信息：
   - 国家代码（ISO 3166-1 alpha-2，如：CN）
   - 地区代码（唯一标识）
   - 地区名称
   - 地区类型
   - 邮政编码格式（可选，正则表达式）

### 从i18n同步

系统支持从i18n模块同步国家数据：

1. 进入 **配送系统 > 地区管理**
2. 点击"从i18n同步"按钮
3. 系统自动导入已安装的国家数据

## 配送区域配置

### 创建配送区域

1. 进入 **配送系统 > 配送区域**
2. 点击"新增"按钮
3. 填写区域信息：
   - 区域名称（如：华东地区）
   - 区域代码（唯一标识，如：EAST_CHINA）
   - 描述（可选）
4. 选择包含的地区：
   - 可以多选国家、省、市、区县
   - 支持树形选择
5. 保存区域

### 地址匹配规则

系统按以下优先级匹配配送区域：

1. **精确匹配**：区县 → 市 → 省 → 国家
2. **模糊匹配**：如果精确匹配失败，向上级匹配

**示例**：

- 收货地址：中国 > 北京 > 朝阳区
- 配送区域A：包含"朝阳区" → 匹配成功
- 配送区域B：包含"北京市" → 如果区域A不存在，匹配成功
- 配送区域C：包含"中国" → 如果区域A和B都不存在，匹配成功

## 快递公司管理

### 快递公司类型

#### 手动类型（manual）

适用于没有API接口的快递公司：

1. **配置追踪URL模板**：
   ```
   https://www.sf-express.com/track/{tracking_number}
   ```
   `{tracking_number}` 会被实际物流单号替换

2. **追踪支持状态**：设置为 `not_supported`
3. 用户查询时，系统返回追踪链接，引导用户到官网查询

#### API类型（api）

适用于有API接口的快递公司：

1. **配置API端点**：
   ```
   https://api.example.com/tracking
   ```

2. **配置API方法**：GET 或 POST

3. **配置API参数**（JSON格式）：
   ```json
   {
     "api_key": "your_api_key",
     "api_secret": "your_api_secret",
     "request_format": "json"
   }
   ```

4. **追踪支持状态**：设置为 `supported`
5. 系统自动调用API查询物流信息

### 强制追踪要求

**重要**：所有快递公司必须配置 `tracking_url_template`，即使不支持在线追踪，也必须提供追踪链接模板。

## 费用模板配置

### 计算类型说明

#### 1. 按重量计算（weight）

适用于按重量计费的场景：

- **基础费用**：起步价
- **重量单位**：kg 或 lb
- **重量费率**：每单位重量的费用

**计算公式**：
```
费用 = 基础费用 + (重量 × 重量费率)
```

#### 2. 按体积计算（volume）

适用于按体积计费的场景：

- **基础费用**：起步价
- **体积单位**：m³ 或 ft³
- **体积费率**：每单位体积的费用

**计算公式**：
```
费用 = 基础费用 + (体积 × 体积费率)
```

#### 3. 按件数计算（quantity）

适用于按件数计费的场景：

- **基础费用**：起步价
- **件数费率**：每件的费用

**计算公式**：
```
费用 = 基础费用 + (件数 × 件数费率)
```

#### 4. 固定费用（fixed）

适用于固定配送费用的场景：

- **基础费用**：固定费用

**计算公式**：
```
费用 = 基础费用
```

#### 5. 混合模式（mixed）

支持同时使用多种计算方式：

**配置示例**（JSON格式）：
```json
{
  "base_fee": 10.00,
  "weight": {
    "enabled": true,
    "rate": 5.00,
    "unit": "kg"
  },
  "volume": {
    "enabled": false
  },
  "quantity": {
    "enabled": true,
    "rate": 2.00
  }
}
```

**计算公式**：
```
费用 = 基础费用 + 重量费用 + 件数费用
```

## 免邮规则配置

### 条件类型

#### 1. 订单金额（order_amount）

订单金额达到指定金额免邮：

- **最小订单金额**：如100元

#### 2. 会员等级（member_level）

指定会员等级免邮：

- **会员等级ID列表**：如 [1, 2, 3]

#### 3. 地区（region）

指定地区免邮：

- **地区ID列表**：如 [10, 20, 30]

#### 4. 优惠券（coupon）

使用指定优惠券免邮：

- **优惠券代码列表**：如 ["FREE_SHIP", "SHIP50"]

#### 5. 混合条件（mixed）

支持AND/OR逻辑组合：

**配置示例**（JSON格式）：
```json
{
  "logic": "AND",
  "conditions": [
    {
      "type": "order_amount",
      "operator": ">=",
      "value": 100.00
    },
    {
      "type": "member_level",
      "operator": "IN",
      "value": [1, 2, 3]
    }
  ]
}
```

**逻辑说明**：
- `AND`：所有条件都满足才免邮
- `OR`：任一条件满足就免邮

**操作符说明**：
- `>=`：大于等于
- `<=`：小于等于
- `>`：大于
- `<`：小于
- `=` 或 `==`：等于
- `IN`：在列表中

### 优先级

免邮规则支持优先级设置：

- **数字越大优先级越高**
- 系统按优先级从高到低检查规则
- 找到第一个满足条件的规则即生效

## 配送服务配置

配送服务是核心配置，关联了快递公司、配送区域、费用模板和免邮规则。

### 创建配送服务

1. 进入 **配送系统 > 配送服务**
2. 点击"新增"按钮
3. 填写服务信息：
   - **服务名称**：如"顺丰快递-华东地区"
   - **服务代码**：唯一标识，如：SF_EAST_CHINA
   - **快递公司**：选择已配置的快递公司
   - **配送区域**：选择配送覆盖区域
   - **费用模板**：选择费用计算模板（可选）
   - **免邮规则**：选择免邮规则（可选）
   - **预计配送天数**：最小和最大天数
   - **是否免邮**：如果整个服务都免邮，勾选此项

### 服务匹配流程

当用户选择收货地址时，系统会：

1. **匹配配送区域**：根据收货地址匹配配送区域
2. **获取可用服务**：获取该区域下的所有启用服务
3. **计算费用**：
   - 如果服务配置为免邮，费用为0
   - 如果配置了免邮规则，检查是否满足条件
   - 如果都不满足，使用费用模板计算
4. **返回服务列表**：按排序顺序返回

## 物流跟踪

### 查询物流信息

#### 前端API

```javascript
// 查询物流跟踪
fetch('/shipping/frontend/tracking/query', {
    method: 'POST',
    body: JSON.stringify({
        tracking_number: '1234567890',
        carrier_id: 1,
        force_refresh: false  // 是否强制刷新
    })
})
.then(response => response.json())
.then(data => {
    if (data.success) {
        // 支持追踪
        console.log('状态：', data.status);
        console.log('当前位置：', data.current_location);
        console.log('跟踪节点：', data.nodes);
    } else {
        // 不支持追踪
        console.log('提示：', data.message);
        console.log('追踪链接：', data.tracking_url);
    }
});
```

#### 返回格式

**支持追踪**：
```json
{
  "success": true,
  "tracking_number": "1234567890",
  "carrier": {
    "code": "SF",
    "name": "顺丰速运"
  },
  "status": "in_transit",
  "current_location": "北京分拨中心",
  "estimated_delivery_date": "2024-01-15 18:00:00",
  "nodes": [
    {
      "time": "2024-01-10 10:00:00",
      "location": "北京分拨中心",
      "status": "已发货",
      "description": "快件已从北京分拨中心发出",
      "type": "pickup"
    }
  ],
  "tracking_url": "https://www.sf-express.com/track/1234567890"
}
```

**不支持追踪**：
```json
{
  "success": false,
  "tracking_number": "1234567890",
  "carrier": {
    "code": "CUSTOM",
    "name": "自定义快递"
  },
  "status": "not_supported",
  "message": "该快递公司暂不支持在线追踪，请联系客服查询",
  "tracking_url": "https://example.com/track/1234567890",
  "support_contact": "客服电话：400-xxx-xxxx"
}
```

### 物流状态

- `pending` - 待发货
- `in_transit` - 运输中
- `delivered` - 已送达
- `exception` - 异常（如退回、丢失等）
- `not_supported` - 不支持追踪

### 跟踪节点类型

- `pickup` - 取件
- `transit` - 运输中
- `arrival` - 到达
- `delivery` - 派送
- `exception` - 异常

## 前端API使用

### 获取可用配送服务

```javascript
fetch('/shipping/frontend/shippingservice/getAvailableServices', {
    method: 'POST',
    body: JSON.stringify({
        country_code: 'CN',
        province: '北京',
        city: '北京市',
        district: '朝阳区'
    })
})
.then(response => response.json())
.then(data => {
    if (data.success) {
        data.data.forEach(service => {
            console.log('服务名称：', service.service_name);
            console.log('预计天数：', service.estimated_days_min + '-' + service.estimated_days_max + '天');
        });
    }
});
```

### 计算配送费用

```javascript
fetch('/shipping/frontend/shippingservice/calculateFee', {
    method: 'POST',
    body: JSON.stringify({
        service_id: 1,
        order_amount: 150.00,
        weight: 2.5,
        volume: 0.1,
        quantity: 3,
        member_level_id: 2,
        region_id: 10,
        coupon_code: 'FREE_SHIP'
    })
})
.then(response => response.json())
.then(data => {
    if (data.success) {
        if (data.data.is_free) {
            console.log('免邮！原因：', data.data.reason);
        } else {
            console.log('配送费用：', data.data.fee, '元');
        }
    }
});
```

## 常见问题

### Q1: 如何配置新的快递公司？

A: 进入后台 **配送系统 > 快递公司**，点击"新增"，填写快递公司信息。**注意**：`tracking_url_template` 是必填项，即使不支持在线追踪也必须配置。

### Q2: 如何实现免邮？

A: 有两种方式：
1. **服务级别免邮**：在配送服务中勾选"是否免邮"
2. **规则免邮**：配置免邮规则，系统自动判断是否满足条件

### Q3: 费用计算不准确？

A: 检查以下几点：
1. 费用模板的计算类型是否正确
2. 费率参数是否正确配置
3. 混合模式配置是否正确
4. 货币单位是否一致

### Q4: 地址匹配不到配送区域？

A: 检查以下几点：
1. 地区数据是否完整（国家、省、市、区县）
2. 配送区域是否包含该地址的地区
3. 地区名称是否匹配（区分大小写）

### Q5: 物流跟踪查询失败？

A: 检查以下几点：
1. 快递公司是否配置了追踪URL模板
2. API类型快递公司的API配置是否正确
3. 物流单号格式是否正确
4. 网络连接是否正常（API类型）

### Q6: 如何对接第三方物流API？

A: 
1. 创建快递公司，类型选择"api"
2. 配置API端点和请求方法
3. 在API配置中填写认证信息
4. 系统会自动调用API并解析返回数据

**注意**：目前系统提供适配器接口，可以扩展支持不同的API格式。
