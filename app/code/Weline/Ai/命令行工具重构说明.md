# AI模块命令行工具重构说明

重构日期：2025年10月9日

---

## 📊 重构概述

将AI模块的命令行工具从扁平化结构重构为层次化结构，提高代码组织性和可维护性。

---

## 🔄 目录结构变化

### 重构前（扁平化结构）
```
Console/
├── ModelCollectCommand.php
├── AdapterScanCommand.php
└── DefaultModelCommand.php
```

### 重构后（层次化结构）
```
Console/
└── Ai/
    ├── Model/
    │   ├── Collect.php
    │   └── DefaultModel.php
    └── Adapter/
        └── Scan.php
```

## ⚡ 命令调用方式

### 旧命令（已删除）
```bash
# 这些命令已不再可用
php bin/m model-collect-command
php bin/m adapter-scan-command
php bin/m default-model-command
```

### 新命令（推荐使用）
```bash
# 模型收集
php bin/m ai:model:collect
php bin/m ai:model:collect -v  # 详细模式

# 适配器扫描
php bin/m ai:adapter:scan
php bin/m ai:adapter:scan --clean  # 清理无效适配器
php bin/m ai:adapter:scan -v  # 详细模式

# 默认模型管理
php bin/m ai:model:default-model
php bin/m ai:model:default-model action=list
php bin/m ai:model:default-model action=init
php bin/m ai:model:default-model action=validate
php bin/m ai:model:default-model action=clear-cache
```

## ✨ 功能改进

### 1. 增强的帮助信息

每个命令都增加了完整的help实现：
```bash
php bin/m ai:model:collect --help
php bin/m ai:adapter:scan --help
php bin/m ai:model:default-model --help
```

输出包含：
- 命令描述
- 可用选项
- 使用示例
- 注意事项

### 2. 详细输出模式

支持 `-v` 或 `--verbose` 选项显示详细信息：

```bash
# 普通模式
php bin/m ai:model:collect

  - GPT-3.5 Turbo (gpt-3.5-turbo) - openai

# 详细模式
php bin/m ai:model:collect -v
成功收集 2 个模型：
  ✓ GPT-4 (gpt-4) - openai
    输入价格: $0.030000/1K tokens | 输出价格: $0.060000/1K tokens
  ✓ GPT-3.5 Turbo (gpt-3.5-turbo) - openai
    输入价格: $0.002000/1K tokens | 输出价格: $0.002000/1K tokens
```

### 3. 更好的视觉反馈

使用Unicode字符增强输出：
- ✓ 成功标记
- ✗ 失败标记
- • 列表项
- 🔒 保护标记
- ℹ 信息提示

### 4. 优化的错误处理

```php
} catch (\Exception $e) {
    echo "✗ 收集模型失败：" . $e->getMessage() . "\n";
    if ($verbose) {
        echo "错误堆栈：\n" . $e->getTraceAsString() . "\n";
```

## 🎯 重构优势

### 1. 更好的代码组织
- **层次化**：按功能模块组织（Model、Adapter）
- **清晰度**：文件名直接反映功能（Collect、Scan、DefaultModel）
- **可扩展**：易于添加新的命令类别

### 2. 符合命名规范
- **命名空间**：遵循PSR-4标准
- **类名简洁**：去除冗余的"Command"后缀
- **目录结构**：与命令分组一致

### 3. 易于维护
- **模块化**：每个命令独立，互不影响
- **可测试**：清晰的依赖注入
- **文档化**：完整的PHPDoc注释

### 4. 用户友好
- **命令分组**：`ai:model:*` 和 `ai:adapter:*`
- **直观命令**：命令名称清晰表达功能
- **完整帮助**：每个命令都有详细帮助信息

## 🎓 最佳实践

### 命令组织原则

1. **按功能分组**：相关命令放在同一目录
2. **简洁命名**：类名直接反映功能
3. **完整帮助**：提供详细的使用说明
4. **选项支持**：支持常用选项（-v, --help等）
5. **友好输出**：使用视觉符号增强可读性

### 命令命名规范

```
ai:category:action
│  │        │
│  │        └─ 动作（collect, scan, default等）
│  └────────── 类别（model, adapter等）
└──────────── 模块（ai）
```
