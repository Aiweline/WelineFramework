## Service层测试示例

```php
<?php

declare(strict_types=1);

namespace Weline\Ai\Test\Unit\Service;

use PHPUnit\Framework\TestCase;
use Weline\Ai\Service\AiApiKeyService;
use Weline\Ai\Model\AiApiKey;

class test_api_key_service extends TestCase
{
    private AiApiKeyService $service;
    private AiApiKey $apiKeyModel;

    protected function setUp(): void
    {
        parent::setUp();
        
        // 创建Mock对象
        $this->apiKeyModel = $this->createMock(AiApiKey::class);
        
        // 创建服务实例
        $this->service = new AiApiKeyService($this->apiKeyModel);
    }

    /**
     * 测试：创建API密钥
     */
    public function testCreateApiKey()
    {
        // 设置Mock期望
        $this->apiKeyModel->expects($this->once())
            ->method('setData');

        $this->apiKeyModel->expects($this->once())
            ->method('save');

        // 执行测试
        $result = $this->service->createApiKey('Test Key', 1, 1);

        // 断言
        $this->assertInstanceOf(AiApiKey::class, $result);
    }
}
```

## 测试最佳实践

### 1. 命名规范

- 测试文件: `test_*.php` (小写+下划线)
- 测试类: `test_*` (小写+下划线)
- 测试方法: `test*` (驼峰命名)

### 2. 断言使用

```php
// 基本断言
$this->assertTrue($condition);
$this->assertFalse($condition);
$this->assertEquals($expected, $actual);
$this->assertNull($value);
$this->assertNotNull($value);

// 类型断言
$this->assertInstanceOf(ClassName::class, $object);
$this->assertIsArray($value);
$this->assertIsString($value);
$this->assertIsInt($value);

// 数组断言
$this->assertArrayHasKey('key', $array);
$this->assertCount(5, $array);
$this->assertContains($value, $array);

// 异常断言
$this->expectException(Exception::class);
$this->expectExceptionMessage('Error message');
```

### 3. Mock使用

```php
// 创建Mock
$mock = $this->createMock(ClassName::class);

// 设置方法返回值
$mock->method('methodName')->willReturn($value);

// 设置方法期望调用次数
$mock->expects($this->once())->method('methodName');
$mock->expects($this->exactly(3))->method('methodName');
$mock->expects($this->never())->method('methodName');

// 设置方法参数期望
$mock->expects($this->once())
    ->method('methodName')
    ->with($this->equalTo($expectedParam));

// 使用回调验证参数
$mock->expects($this->once())
    ->method('methodName')
    ->with($this->callback(function($param) {
        return $param > 0;
    }));
```

### 4. 数据提供者

```php
/**
 * @dataProvider validDataProvider
 */
public function testWithDataProvider($input, $expected)
{
    $result = $this->service->process($input);
    $this->assertEquals($expected, $result);
}

public function validDataProvider(): array
{
    return [
        'case1' => ['input1', 'expected1'],
        'case2' => ['input2', 'expected2'],
        'case3' => ['input3', 'expected3'],
    ];
}
```

### 5. 测试覆盖率

- **目标覆盖率**: ≥80%
- **关键代码覆盖**: 100%
- **边界条件**: 必须测试

### 6. 测试隔离

- 每个测试方法应该独立
- 使用`setUp()`和`tearDown()`初始化和清理
- 不要依赖测试执行顺序

### 7. 避免的做法

❌ **不要**:
- 测试框架代码
- 测试第三方库
- 在测试中访问真实数据库（使用Mock）
- 在测试中进行网络请求
- 编写依赖其他测试的测试

✅ **应该**:
- 测试业务逻辑
- 测试边界条件
- 测试异常情况
- 使用Mock隔离依赖
- 保持测试简单明了

## 调试技巧

### 1. 输出调试信息

```php
public function testDebug()
{
    $result = $this->service->process();
    
    // 输出调试信息
    fwrite(STDERR, print_r($result, true));
    
    $this->assertTrue($result);
}
```

### 2. 使用PHPUnit调试参数

```bash
# 详细输出
php bin/w phpunit:run --module=Weline_Ai -b --verbose

# 只运行失败的测试
php bin/w phpunit:run --module=Weline_Ai -b --filter=testFailedMethod
```

### 3. 跳过或标记测试

```php
// 跳过测试
$this->markTestSkipped('此功能暂未实现');

// 标记为不完整
$this->markTestIncomplete('需要进一步实现');

// 标记为风险测试
$this->markTestRisky('此测试可能不稳定');
```
