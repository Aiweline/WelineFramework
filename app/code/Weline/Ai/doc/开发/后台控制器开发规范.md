# Weline框架后台控制器开发规范

## 控制器构造函数规范

### ❌ 错误做法
```php
public function __construct(AiModel $aiModel, ModelCollector $modelCollector) {
    $this->aiModel = $aiModel;
    $this->modelCollector = $modelCollector;
    parent::__construct(); // ❌ 错误！父类没有__construct()方法
}
```

### ✅ 正确做法
```php
public function __construct(AiModel $aiModel, ModelCollector $modelCollector) {
    $this->aiModel = $aiModel;
    $this->modelCollector = $modelCollector;
    // ✅ 不需要调用parent::__construct()
}
```

## 关键要点

1. **Weline框架的继承链**：
   ```
   YourController → BackendPageController → BackendController → PcController → Core
   ```

2. **父类都没有 `__construct()` 方法**：
   - `BackendPageController` 没有 `__construct()`
   - `BackendController` 没有 `__construct()`
   - `PcController` 没有 `__construct()`
   - `Core` 没有 `__construct()`

3. **初始化机制**：
   - `__construct()` → 仅用于依赖注入
   - `__init()` → 用于实际的初始化工作

## 依赖注入失败处理

### 问题现象
```
Fatal error: Call to a member function select() on false
```

### 解决方案
```php
/**
 * 确保模型对象有效
 */
private function getAiModel(): AiModel {
    if (!$this->aiModel || $this->aiModel === false) {
        $this->aiModel = ObjectManager::getInstance(AiModel::class);
    }
    return $this->aiModel;
}

// 使用方式
public function index(): string {
    $models = $this->getAiModel()->reset()
        ->pagination($page, $pageSize)
        ->select()
        ->fetch();
}
```

## 后台模板布局规范

功能模块通过 Framework 的模块中立控制器请求后台布局，禁止继承 `Weline\Admin\Controller\BaseController` 或在模板中复制 Admin 的完整页面骨架：

```php
use Weline\Framework\App\Controller\BackendPageController;

class YourController extends BackendPageController
{
    public function index(): string
    {
        $this->assign('title', __('页面标题'));
        return $this->fetch();
    }
}
```

页面模板只输出本模块内容，`BackendPageController` 通过 `BackendLayoutProviderInterface` 统一包装当前后台布局：

```html
<div class="container-fluid">
    <!-- 本模块内容 -->
</div>
```

## 常见错误总结

1. **调用不存在的父类构造函数** → 导致报错
2. **依赖注入失败** → 使用ObjectManager作为备用方案
3. **功能模块继承 Admin 内部控制器或复制完整布局** → 形成模块耦合和重复页面骨架

## ORM方法规范

### ❌ 错误的方法名
```php
// 错误：使用了不存在的orderBy方法
$models = $this->aiModel->reset()
    ->orderBy(AiModel::fields_CREATED_TIME, 'DESC') // ❌ 错误！
    ->select()
    ->fetch();
```

### ✅ 正确的方法名
```php
// 正确：使用order方法
$models = $this->aiModel->reset()
    ->order(AiModel::fields_CREATED_TIME, 'DESC') // ✅ 正确！
    ->select()
    ->fetch();

// 多字段排序
$models = $this->aiModel->reset()
    ->order(AiModel::fields_SUPPLIER, 'ASC')
    ->order(AiModel::fields_NAME, 'ASC')
    ->select()
    ->fetch();
```

### 框架ORM方法参考
```php
// 基本查询方法
->where('field', 'value')           // 条件查询
->where('field', 'value', 'like')   // LIKE查询
->order('field', 'DESC')            // 排序（DESC/ASC）
->pagination($page, $pageSize)      // 分页
->select()                          // 执行查询
->fetch()                          // 获取结果
->find()                           // 查找单条记录
->load($id)                        // 根据ID加载
->save()                           // 保存数据
->delete()                         // 删除数据
```

## 最佳实践

1. **构造函数**：只进行依赖注入，不调用 `parent::__construct()`
2. **模型获取**：添加备用的ObjectManager获取机制
3. **模板布局**：继承 `BackendPageController`，模板只维护本模块内容
4. **错误处理**：在控制器方法中添加try-catch错误处理
5. **ORM方法**：使用正确的方法名，如 `order()` 而不是 `orderBy()`
