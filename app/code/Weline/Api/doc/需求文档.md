# Weline API 模块需求文档

## 一、项目概述

### 1.1 项目背景
创建 `Weline\Api` 模块，专门管理API集成功能，包括API用户管理、令牌管理、API文档生成、API调试等功能。

### 1.2 核心目标
- 统一管理API用户和权限
- 提供完整的API文档生成和浏览功能
- 支持API在线调试（沙盒环境）
- 实现安全的令牌交换机制

## 二、功能需求

### 2.1 API文档界面改造

#### 2.1.1 URL路由调整
- **当前地址**: `http://127.0.0.1:9981/dev/docs?catalog=api`
- **目标地址**: `http://127.0.0.1:9981/dev/docs/api`
- **实现方式**: 在 `DeveloperWorkspace` 模块中添加新的路由和控制器方法

#### 2.1.2 API文档生成
- **数据来源**: 从API控制器的Acl注解中读取权限信息
- **文档格式**: 参照Swagger格式
- **分组方式**: 使用控制器类名作为分组，类的描述作为分组描述
- **API信息**: 
  - 类名称（分组）
  - 方法名称（API端点）
  - 请求方法（GET/POST/PUT/DELETE等）
  - 请求参数
  - 响应格式
  - 权限要求（从Acl注解读取）

#### 2.1.3 API文档界面功能
- 显示所有API分组（按控制器类分组）
- 每个分组下显示所有API方法
- 显示API的详细信息（参数、响应、权限等）
- 支持搜索和过滤
- 兼容暗色/亮色主题

### 2.2 API用户管理

#### 2.2.1 API用户模型
- **位置**: `app/code/Weline/Api/Model/ApiUser.php`
- **说明**: API用户模型，基于 `BackendUser` 模型扩展
- **字段**:
  - `user_id`: 用户ID（关联BackendUser）
  - `api_key`: API密钥（用于令牌交换）
  - `api_secret`: API密钥（用于令牌交换）
  - `token_expire_time`: 令牌有效期（秒）
  - `is_enabled`: 是否启用
  - `created_at`: 创建时间
  - `updated_at`: 更新时间

#### 2.2.2 API用户管理界面
- **位置**: `app/code/Weline/Admin/Controller/Api/User.php`
- **功能**:
  - API用户列表（显示、搜索、分页）
  - 创建API用户
  - 编辑API用户
  - 删除API用户
  - 启用/禁用API用户
  - 查看API用户详情
  - **重要**: 界面必须集成到管理员布局中，不能是独立界面

#### 2.2.3 API用户角色管理
- **角色授权**: 使用 `Weline\Acl` 模块的权限资源
- **角色绑定**: API用户可以绑定多个角色
- **权限选择**: 使用jstree组件选择权限
  - jstree必须兼容暗色/亮色主题
  - 支持树形结构展示权限
  - 支持多选
  - 保存选中的权限到角色

### 2.3 API认证和授权

#### 2.3.1 API登录功能迁移
- **源位置**: `app/code/Weline/Backend/Controller/Api/Auth.php`
- **目标位置**: `app/code/Weline/Api/Controller/Auth.php`
- **功能**:
  - 登录并获取token
  - 刷新token
  - 撤销token
  - 获取当前用户信息
  - 获取token信息

#### 2.3.2 API校验Observer迁移
- **源位置**: `app/code/Weline/Backend/Observer/ApiControllerInitBefore.php`
- **目标位置**: `app/code/Weline/Api/Observer/ApiControllerInitBefore.php`
- **功能**:
  - 校验API请求的token
  - 区分后台API（`api_admin`）和前台API（`api`）的area
  - 检查用户状态
  - 返回401/403错误

#### 2.3.3 Area区分逻辑
- **后台API**: `WELINE_AREA=api_admin` 或 `isApiBackend()`
- **前台API**: `WELINE_AREA=api` 或 `isApiFrontend()`
- **校验规则**:
  - 后台API: 使用 `BackendApiSession`，校验后台用户
  - 前台API: 使用 `FrontendApiSession`（需创建），校验前台用户

### 2.4 令牌交换机制

#### 2.4.1 令牌类型
1. **访问令牌（Access Token）**: 用于API请求认证
2. **刷新令牌（Refresh Token）**: 用于刷新访问令牌
3. **通行令牌（Pass Token）**: API用户创建时生成的长期令牌，用于令牌交换

#### 2.4.2 令牌交换流程
1. **创建API用户时**:
   - 生成 `api_key` 和 `api_secret`
   - 设置令牌有效期（`token_expire_time`）

2. **首次获取令牌**:
   - 使用 `api_key` 和 `api_secret` 请求获取访问令牌
   - 返回访问令牌和刷新令牌

3. **刷新令牌**:
   - 在令牌有效期内，使用访问令牌和通行令牌（`api_key`）请求刷新
   - 返回新的访问令牌和刷新令牌
   - 旧的令牌失效

4. **令牌交换接口**:
   - `POST /api/auth/exchange`: 使用api_key和api_secret交换访问令牌
   - `POST /api/auth/refresh`: 使用访问令牌和通行令牌刷新令牌

#### 2.4.3 令牌安全
- 访问令牌有效期：由API用户配置的 `token_expire_time` 决定
- 刷新令牌有效期：通常比访问令牌长
- 通行令牌：长期有效，但可以手动撤销
- 令牌存储：使用 `BackendUserToken` 表存储

### 2.5 API文档界面登录功能

#### 2.5.1 登录界面
- 在API文档界面添加登录功能
- 支持API用户登录
- 登录成功后存储令牌到本地存储（localStorage）

#### 2.5.2 令牌管理
- 显示当前登录状态
- 显示令牌信息（有效期等）
- 支持手动刷新令牌
- 支持登出（清除令牌）

#### 2.5.3 API请求测试
- 在API文档界面可以直接测试API
- 自动携带令牌（从localStorage读取）
- 显示请求和响应信息
- 支持多种请求方法（GET/POST/PUT/DELETE等）

### 2.6 API调试功能

#### 2.6.1 沙盒数据库
- **配置**: 使用 `app/etc/env.php` 中的 `sandbox_db` 配置
- **默认**: 如果未配置，使用SQLite数据库 `app/etc/sandbox_db.sqlite`
- **用途**: API调试时的所有数据操作都进入沙盒数据库，不影响正式数据库

#### 2.6.2 调试模式识别
- 在API文档界面发起的请求自动进入调试模式
- 调试模式下，所有数据库操作使用沙盒数据库连接
- 通过请求头或参数标识调试请求

#### 2.6.3 调试数据隔离
- 调试数据完全隔离，不影响正式数据
- 支持清空沙盒数据库
- 支持查看沙盒数据库数据

## 三、技术实现

### 3.1 模块结构

```
app/code/Weline/Api/
├── Controller/
│   ├── Auth.php              # API认证控制器（迁移自Backend）
│   └── ...
├── Model/
│   ├── ApiUser.php           # API用户模型
│   └── ...
├── Observer/
│   └── ApiControllerInitBefore.php  # API校验Observer（迁移自Backend）
├── Session/
│   ├── BackendApiSession.php # 后台API Session（可能需要扩展）
│   └── FrontendApiSession.php # 前台API Session（新建）
├── Service/
│   ├── TokenService.php      # 令牌服务
│   ├── ApiDocService.php     # API文档生成服务
│   └── SandboxService.php    # 沙盒数据库服务
├── doc/
│   └── 需求文档.md           # 本文档
├── etc/
│   ├── module.xml            # 模块配置
│   └── event.xml             # 事件配置
└── register.php              # 模块注册文件
```

### 3.2 数据库表设计

#### 3.2.1 API用户表（扩展BackendUser）
- 使用 `BackendUser` 表作为基础
- 新增 `w_api_user` 表存储API用户扩展信息：
  - `user_id`: 用户ID（关联BackendUser）
  - `api_key`: API密钥
  - `api_secret`: API密钥（加密存储）
  - `token_expire_time`: 令牌有效期（秒，默认7天）
  - `is_enabled`: 是否启用
  - `created_at`: 创建时间
  - `updated_at`: 更新时间

#### 3.2.2 令牌表（使用现有BackendUserToken）
- 使用 `BackendUserToken` 表
- 扩展字段支持：
  - `token`: 令牌值
  - `type`: 令牌类型（`access_token`, `refresh_token`, `pass_token`）
  - `token_expire_time`: 过期时间
  - `user_id`: 用户ID
