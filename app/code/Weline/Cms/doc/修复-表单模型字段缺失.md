# 表单提交模型字段缺失问题修复

## 问题描述

浏览器提交表单时遇到 **Fatal Error**：

```
Fatal error: Uncaught Error: Undefined constant GuoLaiRen\PageBuilder\Model\FormSubmission::fields_STATUS 
in E:\WelineFramework\DEV-workspace\app\code\GuoLaiRen\PageBuilder\Controller\Frontend\Form.php:90
```

## 错误分析

### 问题原因
FormSubmission模型缺少 `fields_STATUS` 常量定义和相关的状态常量。

### 错误位置
**文件**: `app/code/GuoLaiRen/PageBuilder/Controller/Frontend/Form.php` 第90行

**代码**:
```php
->setData(FormSubmission::fields_STATUS, FormSubmission::STATUS_NEW) // ❌ fields_STATUS 未定义
```

## 解决方案

### 1. 添加字段常量

**文件**: `app/code/GuoLaiRen/PageBuilder/Model/FormSubmission.php`

#### 添加fields_STATUS常量
```php
// 字段定义
public const fields_ID = 'submission_id';
public const fields_PAGE_ID = 'page_id';
public const fields_EMAIL = 'email';
public const fields_PHONE = 'phone';
public const fields_EXTRA_FIELDS = 'extra_fields';
public const fields_IP_ADDRESS = 'ip_address';
public const fields_USER_AGENT = 'user_agent';
public const fields_REFERER = 'referer';
public const fields_STATUS = 'status';              // ✅ 新增
public const fields_SUBMITTED_AT = 'submitted_at';
public const fields_CREATE_TIME = 'create_time';

// 状态常量
public const STATUS_NEW = 'new';                    // ✅ 新增
public const STATUS_PROCESSED = 'processed';        // ✅ 新增
public const STATUS_SPAM = 'spam';                  // ✅ 新增
```

### 2. 更新数据库表结构

#### 在install()方法中添加status字段
```php
->addColumn(
    self::fields_STATUS,
    TableInterface::column_type_VARCHAR,
    20,
    "not null default 'new'",
    '状态'
)
```

#### 添加status字段索引
```php
->addIndex(TableInterface::index_type_KEY, 'idx_status', [self::fields_STATUS], '状态索引')
```

#### 实现upgrade()方法
```php
public function upgrade(ModelSetup $setup, Context $context): void
{
    // 添加status字段（如果不存在）
    if ($setup->tableExist() && !$setup->getConn()->hasField($setup->getTableName(), self::fields_STATUS)) {
        $setup->getConn()->addColumn(
            $setup->getTableName(),
            self::fields_STATUS,
            TableInterface::column_type_VARCHAR,
            20,
            "not null default 'new'",
            '状态'
        );
        
        // 添加索引
        $setup->getConn()->addIndex(
            $setup->getTableName(),
            TableInterface::index_type_KEY,
            'idx_status',
            [self::fields_STATUS]
        );
    }
}
```

### 3. 修复page_id获取逻辑

**问题**: 控制器只从POST数据中读取page_id，但表单是通过URL参数传递的。

**文件**: `app/code/GuoLaiRen/PageBuilder/Controller/Frontend/Form.php`

#### 修改前 ❌
```php
$pageId = (int)$this->request->getPost('page_id', 0);
```

#### 修改后 ✅
```php
$pageId = (int)($this->request->getPost('page_id') ?: $this->request->getGet('page_id', 0));
```

**说明**: 优先从POST获取，如果为空则从GET参数获取。

## 状态常量说明

### 可用状态

| 常量 | 值 | 说明 |
|------|-----|------|
| `STATUS_NEW` | `'new'` | 新提交，待处理 |
| `STATUS_PROCESSED` | `'processed'` | 已处理 |
| `STATUS_SPAM` | `'spam'` | 垃圾信息 |

### 使用示例

#### 创建新提交记录
```php
$submission = clone $this->formSubmissionModel;
$submission->clearData()
    ->setData(FormSubmission::fields_PAGE_ID, $pageId)
    ->setData(FormSubmission::fields_EMAIL, $email)
    ->setData(FormSubmission::fields_PHONE, $phone)
    ->setData(FormSubmission::fields_STATUS, FormSubmission::STATUS_NEW) // 默认新状态
    ->save();
```

#### 更新状态为已处理
```php
$submission->setData(FormSubmission::fields_STATUS, FormSubmission::STATUS_PROCESSED)
    ->save();
```

#### 标记为垃圾信息
```php
$submission->setData(FormSubmission::fields_STATUS, FormSubmission::STATUS_SPAM)
    ->save();
```

#### 查询特定状态的提交
```php
$newSubmissions = $model
    ->where(FormSubmission::fields_STATUS, FormSubmission::STATUS_NEW)
    ->select()
    ->fetch()
    ->getItems();
```

## 关键经验总结

### 1. 模型常量定义规范
- 所有数据库字段必须定义对应的 `fields_XXX` 常量
- 状态、类型等枚举值必须定义相应的常量（如 `STATUS_XXX`）
- 避免在代码中直接使用字符串字面量

### 2. 数据库表升级
- 新字段必须在 `install()` 方法中定义
- 已有表的新字段需要在 `upgrade()` 方法中添加
- 使用 `hasField()` 检查字段是否存在，避免重复添加

### 3. 参数获取优先级
```php
// ✅ 灵活获取，支持多种来源
$value = $this->request->getPost('key') ?: $this->request->getGet('key', 'default');

// ❌ 只支持单一来源
$value = $this->request->getPost('key', 'default');
```

### 4. 错误排查顺序
1. **Fatal Error** → 检查常量/类定义
2. **404 Not Found** → 检查路由配置
3. **200 业务错误** → 检查业务逻辑
4. **数据库错误** → 检查表结构和字段

### 5. 测试层次
1. **命令行测试** → 验证路由和基本逻辑
2. **浏览器测试** → 验证完整流程和UI
3. **数据库验证** → 确认数据正确保存
