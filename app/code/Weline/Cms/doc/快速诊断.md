# CMS模块快速诊断指南

## 快速检查清单

### 1. 模块状态检查

```bash
# 检查模块是否已注册
php bin/w module:list | findstr Cms

# 应该显示：Weline_Cms # 开启
```

### 2. 强制重新安装

如果模块显示为禁用或有问题，执行：

```bash
# 1. 清理所有缓存
php bin/w cache:clear -f

# 2. 重新安装模块
php bin/w setup:upgrade Weline_Cms

# 3. 再次清理缓存
php bin/w cache:clear -f
```

### 3. 检查菜单访问

完成安装后，访问后台：

1. 登录后台管理系统
2. 查看左侧菜单栏
3. 应该看到 **"页面管理"** 菜单项
4. 展开后应该看到：
   - CMS内容管理系统
   - 表单提交管理
   - 模板管理

### 4. 直接访问URL测试

如果菜单不显示，可以尝试直接访问：

```
http://your-domain/admin/cms/backend/page/index
```

### 5. 常见问题快速修复

#### 问题：菜单不显示

**解决方案**：
```bash
php bin/w setup:upgrade Weline_Cms
php bin/w cache:clear -f
```

#### 问题：404错误

**解决方案**：
1. 检查路由配置：`app/code/Weline/Cms/etc/env.php` 应该包含 `'router' => 'cms'`
2. 清理路由缓存：
```bash
rm -rf generated/routes.php
php bin/w setup:upgrade Weline_Cms
```

#### 问题：权限错误

**解决方案**：
1. 确认当前登录用户是管理员（角色ID=1）
2. 或者为当前角色分配 `Weline_Cms::page_builder` 权限

### 6. 验证数据库表

使用以下命令检查表是否存在（根据数据库类型调整）：

**MySQL**:
```bash
php bin/w db:query "SHOW TABLES LIKE 'weline_cms%'"
```

**PostgreSQL**:
```bash
php bin/w db:query "SELECT tablename FROM pg_tables WHERE tablename LIKE 'weline_cms%'"
```

应该看到：
- `weline_cms_page`
- `weline_cms_style`
- `weline_cms_form_submission`
- `cms_page_local_description`

### 7. 完整重置（最后手段）

如果以上都不行，执行完整重置：

```bash
# 1. 清理所有缓存和生成文件
rm -rf generated/*
rm -rf var/cache/*
rm -rf var/page_cache/*

# 2. 重新安装所有模块
php bin/w setup:upgrade

# 3. 清理缓存
php bin/w cache:clear -f

# 4. 重启服务器
php bin/w s:sta
```
