# Weline Cms 使用手册

## 一、模块概览

`Weline_Cms` 模块提供一套「可视化页面 + 多语言 + 表单收集」的完整解决方案，典型场景包括：

- 着陆页 / 活动页 / 品牌页
- 隐私政策、服务条款等政策页面
- 带表单的营销收集页（如留资页、预约页）

> 安装与基础信息请参考根目录的 `README.md`，本文只关注**后台如何使用**。

---

## 二、后台入口与权限

- 后台菜单路径：**内容管理 → CMS内容管理系统**
- 主要子菜单：
  - **CMS内容管理系统**：页面列表与编辑（核心入口）
  - **表单提交管理**：查看前端用户提交记录
  - **模板管理**：管理可选的页面样式模板（Header/Content/Footer + 配色）

后台所有按钮/文本均已接入 i18n，开发时使用 `__('中文文案')` 即可。

---

## 三、快速创建一个页面

1. 进入 **内容管理 → CMS内容管理系统**
2. 点击右上角 **「新建页面」**
3. 在「基本信息」区域填写：
   - **页面句柄（handle）**
     - 用于 URL，如：`about-us`、`privacy-policy`
     - 只能包含小写字母、数字和 `-`
   - **页面类型**
     - 预置类型：主页、关于我们、隐私政策、配送政策、退款政策等
   - **页面名称 / 页面标题**
4. （可选）设置：
   - **父页面**：用于形成「主页 → 政策页」等层级关系
   - **品牌资源**：Logo、网站图标等
5. 点击 **「创建页面」** 保存，跳转到编辑页。

> 句柄是所有路由和 SEO 的基础，请务必唯一、稳定，不建议频繁修改。

---

## 四、选择样式模板与可视化配置

在页面编辑表单中，找到 **「样式与配置」** 区域：

1. 从「样式模板」下拉框中选择一个样式（如：`marketing-landing`、`market-mastery` 等）
2. 选择后系统会：
   - 自动扫描该模板下的 `header.phtml / content.phtml / footer.phtml`
   - 解析其中的 `@fields_start ... @fields_end` 配置块
   - 在后台生成可视化配置表单（颜色、图片、文案、布局等）
3. 你可以：
   - 修改文案、颜色、图片等配置
   - 通过「可视化配置」区域的「预览」按钮，实时预览效果
4. 支持：
   - **自动保存**：部分配置支持自动保存（详见 `doc/功能-样式切换自动保存.md`）
   - **重置为默认**：按组或按字段恢复默认值

> 各内置模板的详细说明参见：  
> - `marketing-landing-快速启动.md` / `marketing-landing-模板说明.md`  
> - `style/xxx/readme.md`（每个模板目录下都有自己的说明）

## 五、多语言内容编辑（页面正文 + SEO）

### 1. 默认语言与支持语言

在「多语言设置」区域：

- 选择 **默认语言**（如：`zh_Hans_CN`）
- 在「支持的语言翻译」中勾选需要的语言（如：`en_US`）
- 默认语言的内容存储在主表 `weline_cms_page` 中
- 其他语言存储在 `cms_page_local_description` 中

### 2. 编辑多语言内容

1. 在「内容配置」中填写默认语言的正文（可使用可视化编辑器）
2. 在下方「多语言内容翻译」区域：
   - 切换到对应语言的 Tab（如：英文）
   - 填写：名称 / 标题 / 正文 / SEO 标题 / 描述 / 关键词等
3. 保存后，前端会根据当前语言自动优先展示翻译内容，找不到时回退到默认语言。

> 多语言字段和 LocalModel 设计说明参见：  
> - `PageBuilder-多语言配置系统.md`  
> - `i18n-translation-guide.md`  
> - `local-tag-translation-guide.md` / `local-tag-usage-guide.md`

## 六、表单提交与数据查看

部分模板（如 `marketing-landing`、`stockcircle-email`）内置了表单：

- 前端用户提交的表单数据会写入 `weline_cms_form_submission` 表
- 可在 **「表单提交管理」** 中：
  - 按页面、邮箱、电话过滤
  - 查看详情、导出 CSV

表单完整方案与排错说明参见：

## 七、前端访问与路由规则

### 1. 友好 URL 自动路由

`Weline\Cms\Controller\Router` 会拦截前端请求：

- 规则示例：
  - `/about-us` → `/cms/frontend/page/view?handle=about-us`
  - `/privacy-policy` → `/cms/frontend/page/view?handle=privacy-policy`
- 支持「预览模式」：`?preview=1` 可在未发布状态下预览页面

### 2. 显式访问地址

- 通用路由：`/cms/frontend/page/view?handle={handle}`
- 模板说明中常见示例：  
  例如 `stockcircle-email/readme.md` 中：
  - `/cms/page/view?handle=your-handle`

> Router 的实现和回退逻辑可参考：`Controller/Router.php` 与 `Helper/PageHelper.php`。

## 八、i18n 国际化约定

- 模块翻译文件：`i18n/zh_Hans_CN.csv`、`i18n/en_US.csv`
- 所有用户可见文本在代码中必须使用 `__()`：
  - 示例：`__('CMS内容管理系统')`、`__('表单提交管理')`
- 翻译规范与占位符规则参考：
  - `doc/i18n-translation-guide.md`
  - `Weline/I18n/doc/README.md`
