# CMS模块后台管理界面故障排查指南

## 问题描述

后台管理界面无法正常工作，可能表现为：
- 菜单不显示
- 点击菜单后404错误
- 权限错误
- 页面空白

## 排查步骤

### 1. 确认模块已正确安装

```bash
# 检查模块是否已注册
php bin/w module:list | findstr Cms

# 应该显示：Weline_Cms # 开启
```

如果模块未显示或显示为禁用，执行：

```bash
# 重新安装模块
php bin/w setup:upgrade Weline_Cms

# 清理缓存
php bin/w cache:clear -f
```

### 2. 检查数据库表是否已创建

```bash
# 检查数据库表是否存在
php bin/w db:query "SHOW TABLES LIKE 'weline_cms%'"
```

应该看到以下表：
- `weline_cms_page` - 页面主表
- `weline_cms_style` - 样式表
- `weline_cms_form_submission` - 表单提交表
- `cms_page_local_description` - 多语言翻译表

如果表不存在，执行：

```bash
# 强制重新创建表
php bin/w setup:upgrade Weline_Cms
```

### 3. 检查菜单是否已注册

```bash
# 检查菜单是否在数据库中
php bin/w db:query "SELECT * FROM backend_menu WHERE module='Weline_Cms'"
```

应该看到以下菜单项：
- `Weline_Cms::menu_page_management` - 页面管理（顶级菜单）
- `Weline_Cms::page_builder` - CMS内容管理系统
- `Weline_Cms::form_submissions` - 表单提交管理
- `Weline_Cms::template_management` - 模板管理

如果菜单不存在，执行：

```bash
# 重新加载菜单配置
php bin/w setup:upgrade Weline_Cms
php bin/w cache:clear -f
```

### 4. 检查权限配置

```bash
# 检查ACL权限是否已创建
php bin/w db:query "SELECT * FROM acl WHERE source_id LIKE 'Weline_Cms::%'"
```

应该看到以下权限：
- `Weline_Cms::page_builder` - 访问模块
- `Weline_Cms::page_builder_index` - 查看列表
- `Weline_Cms::page_builder_create` - 新建页面
- `Weline_Cms::page_builder_edit` - 编辑页面
- `Weline_Cms::page_builder_delete` - 删除页面
- `Weline_Cms::page_builder_translate` - 翻译页面
- `Weline_Cms::form_submissions` - 表单提交管理
- `Weline_Cms::template_management` - 模板管理

如果权限不存在，执行：

```bash
# 重新注册权限
php bin/w setup:upgrade Weline_Cms
```

### 5. 检查路由配置

确认路由文件 `etc/env.php` 内容：

```php
<?php
return [
    'router' => 'cms',
];
```

检查路由是否已注册：

```bash
# 查看路由文件
cat generated/routes.php | findstr cms
```

### 6. 检查控制器文件

确认以下控制器文件存在且命名空间正确：

- `app/code/Weline/Cms/Controller/Backend/Page.php`
- `app/code/Weline/Cms/Controller/Backend/Submission.php`
- `app/code/Weline/Cms/Controller/Backend/Template.php`
- `app/code/Weline/Cms/Controller/Backend/Preview.php`

### 7. 检查日志文件

查看错误日志：

```bash
# 查看PHP错误日志
tail -f var/log/php_error.log

# 查看系统日志
tail -f var/log/system.log
```

### 8. 常见问题及解决方案

#### 问题1：菜单不显示

**原因**：菜单未正确注册到数据库

**解决方案**：
```bash
php bin/w setup:upgrade Weline_Cms
php bin/w cache:clear -f
```

#### 问题2：点击菜单后404错误

**原因**：路由未正确配置或控制器路径错误

**解决方案**：
1. 检查 `etc/env.php` 中的路由配置
2. 确认控制器文件存在
3. 清理路由缓存：
```bash
php bin/w cache:clear -f
```

#### 问题3：权限错误

**原因**：用户角色没有相应权限

**解决方案**：
1. 检查用户角色权限设置
2. 确认ACL权限已创建
3. 如果是管理员角色（ID=1），应该自动拥有所有权限

#### 问题4：页面空白

**原因**：PHP语法错误或依赖问题

**解决方案**：
1. 检查PHP错误日志
2. 确认所有依赖模块已安装：
   - Weline_Framework
   - Weline_Admin
   - Weline_Backend
   - Weline_I18n

### 9. 完整重置步骤

如果以上步骤都无法解决问题，执行完整重置：

```bash
# 1. 清理所有缓存
php bin/w cache:clear -f

# 2. 重新安装模块
php bin/w setup:upgrade Weline_Cms

# 3. 清理生成的文件
rm -rf generated/*
rm -rf var/cache/*
rm -rf var/page_cache/*

# 4. 重新生成
php bin/w setup:upgrade

# 5. 重启服务器（如果正在运行）
php bin/w s:sta
```

### 10. 验证步骤

完成以上步骤后，验证功能：

1. **访问后台菜单**：
   - 登录后台管理系统
   - 查看左侧菜单，应该看到"页面管理"菜单
   - 展开后应该看到"CMS内容管理系统"、"表单提交管理"、"模板管理"

2. **访问页面列表**：
   - 点击"CMS内容管理系统"
   - 应该显示页面列表（可能为空）

3. **创建测试页面**：
   - 点击"新建页面"
   - 填写基本信息并保存
   - 应该能成功创建
