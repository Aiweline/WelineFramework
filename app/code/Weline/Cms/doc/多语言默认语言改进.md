# PageBuilder 多语言默认语言改进

## 改进概述

本次改进优化了 PageBuilder 模块的多语言功能，特别是默认语言的处理逻辑，确保默认语言与支持的语言列表之间的一致性。

## 改进内容

### 1. 默认语言自动初始化

**问题**：之前在新建或编辑页面时，如果没有设置默认语言，系统没有明确的默认值。

**解决方案**：
- 新建页面时，如果没有设置默认语言，自动使用框架当前语言（通过 `Cookie::getLang()` 获取）
- 编辑页面时，如果页面的默认语言为空，也自动使用框架当前语言

**涉及文件**：
- `view/templates/Backend/Page/form.phtml` - 第9-32行
- `Controller/Backend/Page.php` - `postCreate()` 和 `postEdit()` 方法

### 2. 默认语言自动加入支持的语言列表

**问题**：之前可能出现默认语言不在支持的语言列表中的情况。

**解决方案**：
- 在模板渲染时，自动将默认语言添加到 `selectedLocales` 数组中
- 在保存页面时（创建和编辑），后端也会验证并确保默认语言在支持的语言列表中
- 当用户切换默认语言时，前端 JavaScript 会自动将新的默认语言添加到支持的语言列表

**涉及文件**：
- `view/templates/Backend/Page/form.phtml` - 第24-32行（PHP部分），第985-1030行（JavaScript `selectDefaultLocale` 函数）
- `Controller/Backend/Page.php` - 第128-136行（创建），第308-316行（编辑）

### 3. 默认语言不可删除

**问题**：用户可能会误删除默认语言，导致数据不一致。

**解决方案**：
- 当用户尝试删除默认语言时，弹出提示："默认语言不能从支持的语言列表中删除！如需更改，请先切换默认语言。"
- 默认语言的删除按钮视觉上被禁用（透明度降低，鼠标样式变为 `not-allowed`）
- 默认语言标签添加左侧黄色边框作为视觉标识

**涉及文件**：
- `view/templates/Backend/Page/form.phtml` - 第1139-1170行（`removeLocaleTag` 函数），第1198-1229行（`updateLocaleTagsDeleteButton` 函数）

### 4. 删除按钮状态自动更新

**问题**：当切换默认语言时，删除按钮状态需要相应更新。

**解决方案**：
- 新增 `updateLocaleTagsDeleteButton()` 函数，用于更新所有语言标签的删除按钮状态
- 在以下场景自动调用：
  - 页面加载时（DOMContentLoaded）
  - 添加语言标签后（`addLocaleTag` 函数）
  - 切换默认语言后（`selectDefaultLocale` 函数）

**涉及文件**：
- `view/templates/Backend/Page/form.phtml` - 第1198-1229行（函数定义），第1020行（切换默认语言时调用），第1130行（添加标签时调用），第1374行（页面加载时调用）

## 视觉标识

### 默认语言标签
- 左侧有 **3px 黄色边框**（`#ffc107`）
- 删除按钮透明度降低至 **0.3**
- 删除按钮鼠标样式为 `not-allowed`
- 鼠标悬停时显示提示："默认语言不能删除"

### 普通语言标签
- 无特殊边框
- 删除按钮正常显示
- 删除按钮可正常点击

## 使用流程

### 新建页面
1. 打开新建页面表单
2. 系统自动将框架当前语言设置为默认语言
3. 默认语言自动出现在支持的语言列表中，带有黄色左边框
4. 用户可以选择其他语言作为默认语言，系统会自动添加到支持的语言列表

### 编辑页面
1. 打开编辑页面表单
2. 如果页面已有默认语言，则使用页面的默认语言；否则使用框架当前语言
3. 默认语言标签带有黄色左边框，删除按钮被禁用
4. 用户尝试删除默认语言时会弹出提示
5. 用户切换默认语言后，新的默认语言会自动添加到支持的语言列表，并更新视觉标识

## 技术实现

### 前端（JavaScript）
- `selectDefaultLocale(localeCode, localeName)` - 选择默认语言
- `addLocaleTag(localeCode, localeName)` - 添加语言标签
- `removeLocaleTag(localeCode)` - 移除语言标签（带默认语言检查）
- `updateLocaleTagsDeleteButton()` - 更新删除按钮状态

### 后端（PHP）
- 在 `postCreate()` 和 `postEdit()` 方法中验证默认语言
- 确保默认语言在支持的语言列表中
- 如果没有默认语言，使用 `Cookie::getLang()` 作为默认值

## 注意事项

1. **框架语言获取**：使用 `\Weline\Framework\Http\Cookie::getLang()` 获取当前框架语言
2. **跨浏览器兼容**：使用标准的 DOM API 和 CSS 属性，确保跨浏览器兼容
3. **用户体验**：通过视觉标识（黄色边框、禁用状态）明确告知用户哪个是默认语言
4. **数据一致性**：前后端双重验证，确保默认语言始终在支持的语言列表中
