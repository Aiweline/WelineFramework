# 后台字段翻译功能说明

## 🎯 功能概述

在后台编辑页面中，使用 `<local>` 标签为字段添加翻译按钮，实现**单字段快速翻译**功能。

---

## ✨ 核心特点

### 1. 单字段翻译按钮
- 每个字段旁边都有独立的 **🌍 点击翻译** 按钮
- 点击按钮打开翻译弹窗
- 弹窗中可以为所有语言填写该字段的翻译
- 保存后立即生效

### 2. 支持翻译的字段
- ✅ **页面名称** (name)
- ✅ **页面标题** (title)
- ✅ **SEO标题** (meta_title)
- ✅ **SEO描述** (meta_description)
- ✅ **SEO关键词** (meta_keywords)

### 3. 自动回退显示
- 如果字段有翻译 → 显示翻译内容
- 如果字段无翻译 → 显示默认语言内容

---

## 🚀 使用流程

### 步骤 1：创建/编辑页面
```
1. 进入后台 → 页面管理 → 页面构建器
2. 点击"新建页面"或编辑现有页面
3. 填写基本信息并保存
```

### 步骤 2：选择支持的语言
```
1. 在右侧"多语言设置"卡片中
2. 选择"默认语言"
3. 勾选需要翻译的"支持的语言翻译"
4. 保存页面
```

### 步骤 3：翻译字段
```
1. 在编辑页面中找到需要翻译的字段
2. 点击字段标签旁边的 "🌍 点击翻译" 按钮
3. 在弹窗中填写各语言的翻译内容
4. 点击"保存"
```

### 步骤 4：验证翻译
```
1. 刷新编辑页面
2. 翻译按钮会变成 "✅ 已翻译" 状态
3. 前端访问页面会自动显示对应语言版本
```

## 💡 `<local>` 标签工作原理

### 标签结构

```html
<local model="GuoLaiRen\PageBuilder\Model\Page\LocalDescription" 
       field="name" 
       id="<?= $page->getId() ?>" 
       name="page-name-field">
    <span class="badge bg-info ms-2">
        <i class="mdi mdi-translate"></i> 点击翻译
    </span>
</local>
```

### 属性说明

| 属性 | 说明 | 示例 |
|------|------|------|
| `model` | 翻译模型类 | `GuoLaiRen\PageBuilder\Model\Page\LocalDescription` |
| `field` | 要翻译的字段名 | `name`, `title`, `meta_title` 等 |
| `id` | 记录ID | `$page->getId()` |
| `name` | 唯一标识符 | `page-name-field` |

### 显示逻辑

```php
// 框架自动处理显示逻辑：
if (有当前语言的翻译) {
    显示翻译内容
} else {
    显示默认语言内容
}
```

## 🔄 完整的多语言工作流

### 第一阶段：创建页面（默认语言）

```
1. 创建新页面
2. 填写默认语言的所有信息
   - 页面名称
   - 页面标题
   - 页面内容（Editor.js）
   - SEO信息
3. 保存页面
```

### 第二阶段：配置支持的语言

```
1. 在"多语言设置"中选择默认语言
2. 勾选需要翻译的语言
3. 保存
```

### 第三阶段：翻译字段

```
1. 点击字段旁边的 "🌍 点击翻译" 按钮
2. 在弹窗中填写各语言的翻译
3. 逐个翻译所有字段：
   ☑ 页面名称
   ☑ 页面标题
   ☑ SEO标题
   ☑ SEO描述
   ☑ SEO关键词
```

### 第四阶段：翻译内容

```
1. 前端访问页面（后台登录状态）
2. 点击页面内容的 <local> 标签翻译按钮
3. 翻译页面内容（Editor.js JSON）
```

### 第五阶段：发布

```
1. 将页面状态改为"已发布"
2. 前端自动显示对应语言版本
```

## 🎨 最佳实践

### 1. 翻译顺序建议

```
优先级 1：页面结构字段（编辑页面的 <local> 标签）
  - 页面名称
  - 页面标题
  - SEO信息

优先级 2：页面内容（前端页面的 <local> 标签）
  - 页面主体内容
  - Editor.js 数据

优先级 3：优化与调整
  - 根据前端显示效果微调翻译
```

### 2. 翻译质量检查

```
☑ 所有必填字段都已翻译
☑ 翻译内容符合目标语言习惯
☑ SEO标题长度适中（50-60字符）
☑ SEO描述清晰准确（150-160字符）
☑ 关键词准确且用逗号分隔
☑ 前端显示效果良好
```

### 3. 字段翻译技巧

**页面名称翻译**：
- 简短、准确
- 方便管理员识别
- 示例：`关于我们` → `About Us` → `私たちについて`

**页面标题翻译**：
- 包含关键信息
- 适合浏览器标签显示
- 示例：`关于我们 - 公司介绍` → `About Us - Company` → `会社概要`

**SEO翻译**：
- 标题包含核心关键词
- 描述吸引点击
- 关键词符合搜索习惯
- 示例：
  ```
  中文：关于我们 - 创新科技公司 | 科技服务
  英文：About Us - Innovative Tech Company | Tech Services
  日文：会社概要 - 革新的なテクノロジー企業 | 技術サービス
  ```

## 🔧 技术实现

### 数据库存储

**表名**：`page_local_description`

```sql
CREATE TABLE `page_local_description` (
  `page_id` INT NOT NULL,              -- 页面ID
  `local_code` VARCHAR(10) NOT NULL,   -- 语言代码（如 en_US）
  `name` VARCHAR(255),                 -- 翻译后的名称
  `title` VARCHAR(255),                -- 翻译后的标题
  `content` TEXT,                      -- 翻译后的内容
  `meta_title` VARCHAR(255),           -- 翻译后的SEO标题
  `meta_description` TEXT,             -- 翻译后的SEO描述
  `meta_keywords` VARCHAR(255),        -- 翻译后的SEO关键词
  PRIMARY KEY (`page_id`, `local_code`)
);
```

### 翻译数据流程

```
1. 用户点击 🌍 点击翻译
   ↓
2. 框架加载翻译弹窗
   - 读取 page_local_description 表
   - 显示现有翻译
   ↓
3. 用户填写/修改翻译
   ↓
4. 点击保存
   ↓
5. 框架保存到数据库
   - UPSERT 到 page_local_description
   ↓
6. 前端访问时自动显示对应语言
```

### 前端显示逻辑

```php
// 框架自动处理
$currentLocale = '当前用户语言代码';
$pageId = $page->getId();
$field = 'name';

// 查询翻译
$translation = LocalDescription::where('page_id', $pageId)
    ->where('local_code', $currentLocale)
    ->where('field', $field)
    ->first();

// 显示内容
echo $translation ? $translation->value : $page->getName();
```

## 🐛 故障排除

### 1. 翻译按钮不显示

**原因**：页面未保存或缓存问题

**解决**：
```bash
# 清理缓存
php bin/w cache:clear -f

# 刷新页面
```

### 2. 点击翻译按钮没反应

**原因**：JavaScript未加载或模型未正确配置

**解决**：
1. 检查浏览器控制台是否有错误
2. 确认 `LocalDescription` 模型继承自 `Weline\I18n\LocalModel`
3. 确认 `Page` 模型的 `LocalDescription` 关联正确

### 3. 翻译保存失败

**原因**：数据库表不存在或权限问题

**解决**：
```bash
# 运行模块升级
php bin/w setup:upgrade -m GuoLaiRen_PageBuilder

# 清理缓存
php bin/w cache:clear -f
```

### 4. 前端不显示翻译

**原因**：
- 语言未选中
- 翻译未保存
- 缓存问题

**解决**：
1. 确认在"多语言设置"中勾选了该语言
2. 确认翻译已保存（按钮显示为 ✅）
3. 清理缓存并刷新前端页面
