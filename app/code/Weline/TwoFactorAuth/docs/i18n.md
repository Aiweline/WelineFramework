# i18n国际化使用指南

## 📝 概述

所有用户可见文本都应使用框架的翻译系统，翻译文件位于`i18n/`目录。

---

## 📁 翻译文件

```
i18n/
├── zh_CN.csv    # 中文
└── en_US.csv    # 英文
```

CSV格式：
```csv
"源文本","翻译文本"
"双因素认证器","双因素认证器"
```

---

## 🔧 使用方法

### PHP
```php
// 简单翻译
__('双因素认证器')

// 带参数
__('用户 %{name} 有 %{count} 条消息', ['name' => $user, 'count' => 5])
```

### JavaScript
```javascript
// 简单翻译
__('双因素认证器')

// 带参数
__('✓ 导入成功，共 %{count} 个账户', {count: accounts.length})
```

### 模板
```php
<h1><?= __('双因素认证器') ?></h1>
<button><?= __('添加') ?></button>
```

---

## 📊 占位符

| 格式 | 说明 | 示例 |
|------|------|------|
| `%{}` | 单个参数 | `__('欢迎 %{}!', 'John')` |
| `%{1}, %{2}` | 数组参数 | `__('用户 %{1} 在 %{2} 登录', ['admin', '2025-10-26'])` |
| `%{name}` | 命名参数（推荐） | `__('用户 %{name}', ['name' => 'admin'])` |

---

## ✅ 最佳实践

1. **保持文本完整性**：`__('用户 %{name} 有 %{count} 条消息')` ✅
2. **使用命名占位符**：参数多时更易读 ✅
3. **同步更新CSV**：新增文本时更新zh_CN.csv和en_US.csv ✅
4. **不硬编码字典**：使用框架的`__()`函数 ✅
