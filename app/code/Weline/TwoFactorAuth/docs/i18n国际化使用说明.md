## 🔧 使用方法

### 1. PHP中使用翻译

#### 基本用法

```php
// 简单翻译
echo __('双因素认证器');

// 带参数的翻译 - 通用占位符

// 带参数的翻译 - 数字占位符

// 带参数的翻译 - 命名占位符（推荐）
echo __('用户 %{name} 有 %{count} 条消息', [

    'count' => $messageCount
]);
```

### 2. JavaScript中使用翻译

#### 框架自动注入

Weline Framework会自动将`__()`函数注入到页面中，可直接使用：

```javascript
// 简单翻译
console.log(__('双因素认证器'));

// 带参数的翻译 - 单个参数
alert(__('欢迎 %{}!', userName));

// 带参数的翻译 - 数组参数
console.log(__('用户 %{1} 有 %{2} 条消息', [userName, messageCount]));

// 带参数的翻译 - 对象参数（推荐）
showMessage(__('用户 %{name} 有 %{count} 条消息', {
    name: userName,
    count: messageCount
}));
```

#### PWA应用中使用

```javascript
// app/code/Weline/TwoFactorAuth/view/statics/twofa-app/app.js

// Toast提示
function showToast(message) {
    // message已经是翻译后的文本
    const toast = document.createElement('div');
    toast.textContent = message;
    document.body.appendChild(toast);
}

// 使用翻译
showToast(__('✓ 账户保存成功'));
showToast(__('❌ 无效的密钥格式'));
showToast(__('✓ 导入成功，共 %{count} 个账户', {count: accounts.length}));

// 确认对话框
if (confirm(__('确定要删除这个账户吗？'))) {
    deleteAccount(id);
}

// 动态内容
function generateAccountHTML(account) {
    return `
        <div class="account-card">
            <h3>${account.issuer}</h3>
            <p>${account.account}</p>
            <button onclick="deleteAccount(${account.id})" 
                    title="${__('删除')}">
                ${__('删除')}
            </button>
        </div>
    `;
}
```

#### Fallback机制

如果框架未注入`__()`函数（例如在独立HTML文件中），可以提供fallback：

```javascript
// 检查并提供fallback
if (typeof window.__ === 'undefined') {
    window.__ = function(text, args = null) {
        // 简单的参数替换fallback
        if (args !== null) {
            if (typeof args === 'object' && !Array.isArray(args)) {
                // 命名参数：%{name}, %{count}
                Object.keys(args).forEach(key => {
                    text = text.replace(new RegExp(`%\\{${key}\\}`, 'g'), args[key]);
                });
            } else if (Array.isArray(args)) {
                // 数组参数：%{1}, %{2}
                args.forEach((value, index) => {
                    text = text.replace(`%{${index + 1}}`, value);
                });
            } else {
                // 单个参数：%{}
                text = text.replace('%{}', args);
            }
        }
        return text;
    };
}
```

## 📊 占位符语法

### 支持的占位符格式

#### 1. 通用占位符 `%{}`

**适用场景**：单个参数

```javascript
// PHP
__('欢迎 %{}!', 'John')
// 输出：欢迎 John!

// JavaScript
__('您有 %{} 条新消息', 5)
// 输出：您有 5 条新消息
```

#### 2. 数字占位符 `%{1}`, `%{2}`, ...

**适用场景**：多个位置固定的参数

```javascript
// PHP
__('用户 %{1} 在 %{2} 登录', ['admin', '2025-10-26 10:30'])
// 输出：用户 admin 在 2025-10-26 10:30 登录

// JavaScript
__('%{1} + %{2} = %{3}', [1, 2, 3])
// 输出：1 + 2 = 3
```

#### 3. 命名占位符 `%{name}`, `%{count}`, ...（推荐）

**适用场景**：复杂场景、可读性要求高

```javascript
// PHP
__('订单 %{order_id} 已发货，预计 %{days} 天内到达，收货人：%{receiver}', [
    'order_id' => 'ORD20250001',
    'days' => 3,
    'receiver' => '张三'
])
// 输出：订单 ORD20250001 已发货，预计 3 天内到达，收货人：张三

// JavaScript
__('用户 %{name} 有 %{count} 条未读消息', {
    name: 'admin',
    count: 15
})
// 输出：用户 admin 有 15 条未读消息
```

## ✅ 最佳实践

### 1. 保持文本完整性

✅ **好的做法**：
```javascript
__('用户 %{name} 有 %{count} 条新消息')
```

❌ **不好的做法**：
```javascript
__('用户') + ' ' + name + ' ' + __('有') + ' ' + count + ' ' + __('条新消息')
```

**原因**：
- 不同语言的语序可能不同
- 翻译人员需要看到完整的句子才能正确翻译

---

### 2. 使用命名占位符

✅ **推荐**：
```javascript
__('订单 %{order_id} 共 %{amount} 元，%{status}', {
    order_id: 'ORD001',
    amount: 299.00,
    status: '已支付'
})
```

❌ **不推荐**：
```javascript
__('订单 %{1} 共 %{2} 元，%{3}', ['ORD001', 299.00, '已支付'])
```

**原因**：
- 命名占位符更易读
- 参数顺序可能因语言而异
- 维护更方便

---

### 3. 添加翻译时同步更新CSV

每次添加新的翻译文本时，都应更新两个CSV文件：

**中文**（`zh_CN.csv`）：
```csv
"新增的文本","新增的文本"
```

**英文**（`en_US.csv`）：
```csv
"新增的文本","New Text"
```

---

### 4. 特殊字符处理

CSV文件中的特殊字符需要转义：

```csv
"包含"引号"的文本","Text with ""quotes"""
"包含,逗号的文本","Text with , comma"
```

## 🔄 框架自动加载机制

### 翻译文件加载流程

1. **框架启动时**：
   - 扫描所有模块的`i18n`目录
   - 加载当前语言的CSV文件
   - 构建翻译字典并缓存

2. **用户请求时**：
   - 根据用户语言偏好选择翻译
   - Cookie: `lang`
   - 浏览器语言：`Accept-Language`

3. **JavaScript中**：
   - 框架自动将`__()`函数注入到页面
   - 函数内部查询已加载的翻译字典

## ⚠️ 注意事项

### 1. 不要在JavaScript中硬编码翻译字典

❌ **错误做法**：
```javascript
const translations = {
    'zh_CN': {'hello': '你好'},
    'en_US': {'hello': 'Hello'}
};
```

✅ **正确做法**：
```javascript
// 直接使用框架提供的 __() 函数
__('hello')  // 框架自动查找翻译
```

### 2. 翻译文件只需在模块i18n目录

```
app/code/Weline/TwoFactorAuth/
└── i18n/          ← 只需要在这里
    ├── zh_CN.csv
    └── en_US.csv
```

**不需要**在其他地方配置翻译。

### 3. PWA应用的特殊处理

PWA应用是静态HTML，框架的`__()`函数可能不会自动注入。

**解决方案**：
- 提供JavaScript fallback
- 或通过后台控制器输出HTML（注入`__()`函数）
