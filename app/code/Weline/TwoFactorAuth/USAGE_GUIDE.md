# 使用指南

## 🎯 快速上手

### 第一步：安装模块

模块已经创建在 `app/code/Weline/TwoFactorAuth/` 目录下。

运行安装命令：

```bash
# 在项目根目录执行
php bin/weline setup:upgrade
```

安装完成后，数据库会自动创建以下表：
- `weline_two_factor_user_two_factor` - 用户2FA配置表

### 第二步：访问管理后台

访问地址：`http://your-domain/backend/TwoFactorAuth/Index`

您会看到2FA管理主页面，显示当前状态。

### 第三步：启用2FA

1. 点击「立即启用」按钮
2. 您会看到一个二维码和密钥
3. 选择以下任一方式：

#### 方式A：使用我们自己的验证器（推荐）

1. 点击页面上的「打开验证器APP」链接
2. 或直接访问：`http://your-domain/twofa-app/`
3. 在验证器APP中点击「+」按钮
4. 选择「扫描二维码」或「手动输入」
5. 扫描管理页面上的二维码，或复制密钥手动输入

#### 方式B：使用第三方验证器

1. 在手机上打开 Google Authenticator 或 Microsoft Authenticator
2. 点击「添加账户」
3. 扫描管理页面上的二维码
4. 验证器会自动添加账户

### 第四步：验证并完成

1. 在验证器应用中找到刚添加的账户
2. 复制显示的6位数字验证码
3. 在管理页面的输入框中输入验证码
4. 点击「确认启用」

### 第五步：保存备份码 ⚠️ 重要！

启用成功后，页面会显示10个备份码，例如：
```
1234-5678
8765-4321
...
```

**请务必保存这些备份码！** 建议：
- 截图保存
- 打印出来
- 存储到密码管理器

当您无法获取验证器时（如手机丢失），可以使用备份码登录。

## 📱 使用验证器APP

### 打开APP

访问：`http://your-domain/twofa-app/`

### 📸 快速上手：截图扫描二维码（推荐） 🆕

这是网页端最便捷的添加方式，特别适合在电脑上使用：

**步骤：**
1. 在管理后台启用2FA时，会显示二维码
2. 用手机对二维码截图（或右键保存二维码图片）
3. 打开Weline验证器APP
4. 点击右下角「+」按钮
5. 选择「扫描二维码」标签
6. 点击「📁 选择二维码图片」
7. 选择刚才的截图
8. ✅ 系统自动识别并填充账户信息
9. 确认信息无误后点击「添加」

**优势：**
- ✅ 无需摄像头权限
- ✅ 支持任意设备（电脑、平板、手机）
- ✅ 可以保存二维码图片作为备份
- ✅ 识别准确率高
- ✅ 操作简单快捷

### 添加账户

1. 点击右下角的「+」按钮
2. 选择添加方式：

#### 方式1：手动输入
1. 选择「手动输入」标签
2. 填写：
   - **发行者**：如 "Weline Framework"
   - **账户名**：如 "admin@example.com"
   - **密钥**：从管理页面复制的密钥
   - **备注**（可选）：如 "公司主账号-生产环境" 🆕
3. 点击「添加」

> 💡 **新功能提示**：添加备注可以方便区分相同服务的不同账户！

#### 方式1.5：扫描二维码 🆕（推荐）
1. 选择「扫描二维码」标签
2. 两种扫描方式：

**A. 上传截图（最简单）**：
1. 先用手机对二维码截图
2. 点击「选择二维码图片」
3. 选择刚才的截图
4. ✨ **自动识别并填充**：字段变绿色，提示可编辑 🆕
5. ✏️ **编辑信息**（新增）：
   - 可以修改账户名（如：`admin` → `admin-prod`）
   - 可以添加备注（如：`生产环境管理员账号`）
   - 备注字段会自动聚焦提示您
6. 确认信息后点击「添加」

**B. 摄像头实时扫描**（手机端）：
1. 点击「启动摄像头扫描」
2. 允许浏览器访问摄像头
3. 📷 提示："摄像头已启动，请将二维码对准镜头" 🆕
4. 将二维码对准摄像头
5. ✨ **自动识别并填充**：字段绿色高亮 🆕
6. ✏️ **编辑信息**：修改名称或添加备注 🆕
7. 点击「添加」

> 🌟 **新体验**：扫描成功后，已填充的字段会显示绿色高亮3秒，备注字段会自动聚焦，提示您可以编辑所有信息！

#### 方式2：导入备份 🆕
1. 选择「导入备份」标签
2. 两种导入方式：
   - **选择文件**：点击「选择文件」，选择备份文件（JSON或TXT）
   - **粘贴内容**：直接粘贴备份文件内容到文本框
3. 系统会自动解析并预览账户
4. 点击「导入」完成

### 查看验证码

- 账户添加后会立即显示6位验证码
- 验证码每30秒自动更新
- 右侧有倒计时显示
- 点击验证码可快速复制

### 安装到手机桌面（PWA）

#### iOS设备：

1. 使用Safari浏览器打开验证器APP
2. 点击底部的「分享」按钮 📤
3. 滑动找到「添加到主屏幕」
4. 点击「添加」

#### Android设备：

1. 使用Chrome浏览器打开验证器APP
2. 点击右上角的菜单 ⋮
3. 选择「添加到主屏幕」或「安装应用」
4. 点击「安装」

安装后，APP会像原生应用一样显示在手机桌面，并支持离线使用！

### 导出备份 🆕

为了数据安全，建议定期导出备份：

1. 点击右侧的「💾」导出按钮
2. **选择导出格式**（新增5种格式）：
   - 📱 **Weline验证器** - 本应用标准格式（推荐日常备份）
   - 🛡️ **Aegis Authenticator** - 可直接导入到Aegis（Android最佳）
   - 🤖 **andOTP** - 开源Android验证器格式
   - 🔐 **2FAS Authenticator** - 跨平台验证器格式
   - 📄 **URI列表（通用）** - 兼容所有验证器（兼容性最好）
3. 系统自动下载对应格式的备份文件
4. 文件名格式：`{格式}-backup-{时间戳}.{扩展名}`
5. 妥善保存该文件

**注意：** 备份文件包含所有账户的密钥，请安全存储！

**选择建议**：
- 💾 日常备份 → **Weline格式**
- 📱 迁移到Android → **Aegis格式**（最简单）
- 🌐 最大兼容性 → **URI列表**
- 🎯 特定验证器 → 选择对应格式

详细说明请参考：[导出格式指南](docs/EXPORT_FORMATS_GUIDE.md)

### 从其他验证器迁移 🆕

支持从以下验证器导入账户：

#### 支持的格式

| 格式 | 来源 | 说明 |
|------|------|------|
| **JSON格式** | 通用 | 标准JSON格式，包含issuer、account、secret等字段 |
| **URI列表** | 通用 | 每行一个otpauth://链接 |
| **Aegis导出** | Aegis Authenticator | Aegis应用的完整JSON导出 |
| **Weline备份** | 本应用 | 使用导出功能生成的备份文件 |

#### 导入步骤示例

**从Aegis Authenticator导入：**
1. 在Aegis中打开设置 → 导出
2. 选择「导出为JSON（明文）」
3. 保存文件
4. 在Weline验证器中选择「导入备份」
5. 选择刚才保存的文件

**从其他验证器（无导出功能）：**
1. 如果验证器不支持导出，需要重新添加账户
2. 在原验证器中查看密钥（通常在账户详情中）
3. 在Weline验证器中手动添加

**使用URI列表格式：**

创建一个文本文件，每行一个账户：
```
otpauth://totp/GitHub:user1?secret=JBSWY3DPEHPK3PXP&issuer=GitHub
otpauth://totp/Google:user2@gmail.com?secret=ABCDEFGHIJK&issuer=Google
otpauth://totp/AWS:admin?secret=MNOPQRSTUVW&issuer=AWS
```

然后导入该文件。

#### JSON格式示例

```json
[
  {
    "issuer": "GitHub",
    "account": "user@example.com",
    "secret": "JBSWY3DPEHPK3PXP",
    "digits": 6,
    "period": 30,
    "type": "totp"
  },
  {
    "issuer": "Google",
    "account": "user@gmail.com",
    "secret": "ABCDEFGHIJKLMNOP",
    "digits": 6,
    "period": 30,
    "type": "totp"
  }
]
```

## 🔑 使用备份码

### 什么时候需要备份码？

- 手机丢失或损坏
- 验证器应用被删除
- 换新手机但未迁移验证器
- 验证器时间不同步导致验证码错误

### 如何使用

在登录时，输入框中输入备份码（格式：1234-5678）而不是6位验证码。

**注意：** 每个备份码只能使用一次！

### 重新生成备份码

如果备份码用完或丢失：

1. 访问 2FA 管理页面
2. 点击「重新生成备份码」
3. 输入当前验证码确认身份
4. 保存新的备份码

## 🔒 安全建议

### ✅ 应该做的

- ✅ 立即保存备份码
- ✅ 定期检查2FA状态
- ✅ 使用独立的验证器应用
- ✅ 保护好密钥和备份码
- ✅ 在更换手机前先禁用2FA，换手机后重新启用

### ❌ 不应该做的

- ❌ 将密钥和备份码存储在邮箱中
- ❌ 截图后上传到云盘
- ❌ 与他人分享密钥
- ❌ 在多个账户使用相同的密钥
- ❌ 忽略保存备份码

## 🐛 常见问题

### Q1: 验证码总是错误怎么办？

**A:** 可能是时间不同步的问题。

**解决方案：**
1. 确保服务器时间正确
2. 确保手机时间设置为「自动」
3. 尝试使用备份码登录

### Q2: 二维码无法显示？

**A:** 可能是网络问题。

**解决方案：**
1. 使用「手动输入密钥」方式
2. 复制密钥到验证器应用中手动添加

### Q3: 手机丢了怎么办？

**A:** 使用备份码。

**步骤：**
1. 使用备份码登录账户
2. 进入2FA管理页面
3. 禁用旧的2FA
4. 在新手机上重新启用2FA

### Q4: 忘记保存备份码了？

**A:** 如果还能登录：

1. 登录后台
2. 访问2FA管理页面
3. 点击「重新生成备份码」
4. 这次一定要保存！

**如果无法登录：** 需要联系系统管理员重置2FA。

### Q5: 可以同时在多个设备上使用吗？

**A:** 可以！

同一个密钥可以添加到多个验证器应用中：
- 手机上的验证器
- 电脑上的浏览器（Weline验证器）
- 备用手机上的验证器

所有设备会生成相同的验证码。

### Q6: 验证器APP可以离线使用吗？

**A:** 可以！

验证码生成是基于时间和密钥的算法，不需要网络连接。只要有密钥，就可以离线生成验证码。

### Q7: 为什么要用自己开发的验证器？

**A:** 优势：

1. **品牌定制** - 完全自己的品牌
2. **无需安装** - PWA技术，直接浏览器使用
3. **跨平台** - 任何设备的浏览器都能用
4. **完全掌控** - 所有代码都是自己的
5. **标准兼容** - 可以和其他验证器互换使用
6. **导入导出** 🆕 - 支持多种格式的备份导入导出

### Q8: 支持从哪些验证器导入？

**A:** 支持多种格式：

1. **直接支持**：
   - Aegis Authenticator（JSON导出）
   - 标准JSON格式
   - URI列表格式
   - 本应用的备份文件

2. **需要转换**：
   - Google Authenticator（需要使用"账户转移"功能）
   - Microsoft Authenticator（可导出部分账户）

3. **无法导入**：
   - 不支持导出的验证器需要重新添加账户

### Q9: 导入的账户安全吗？

**A:** 完全安全！

- 所有数据存储在本地LocalStorage
- 不上传到任何服务器
- 使用与原验证器相同的密钥
- 生成的验证码完全相同

### Q10: 可以同时使用多个验证器吗？

**A:** 可以！

同一个账户可以同时在多个验证器中使用：
1. 导出Weline验证器的备份
2. 导入到其他支持的验证器
3. 或者重新扫描二维码添加到新验证器

所有验证器会生成相同的验证码。

### Q11: 截图扫描识别不了怎么办？ 🆕

**A:** 尝试以下方法：

1. **提高图片质量**：
   - 确保截图清晰，二维码完整
   - 避免模糊、反光、变形
   - 尽量使用高分辨率截图

2. **调整图片**：
   - 裁剪掉多余部分，只保留二维码
   - 确保二维码占据图片主要区域
   - 避免二维码过小

3. **使用其他方式**：
   - 尝试摄像头实时扫描
   - 手动输入密钥（最可靠）
   - 复制otpauth链接粘贴

4. **检查图片格式**：
   - 支持：JPG, PNG, WebP, GIF
   - 文件大小：建议小于5MB

### Q12: 摄像头扫描需要什么条件？ 🆕

**A:** 需要满足：

1. **浏览器支持**：
   - Chrome 53+
   - Firefox 36+
   - Safari 11+
   - Edge 12+

2. **安全上下文**：
   - HTTPS访问（必需）
   - 或localhost访问

3. **权限**：
   - 允许浏览器访问摄像头
   - 某些浏览器需要用户明确授权

4. **设备**：
   - 有摄像头的设备
   - 摄像头驱动正常

**如果无法使用摄像头**，建议使用截图上传方式（更简单）。

## 🔧 开发者指南

### 在登录流程中集成2FA

```php
use Weline\TwoFactorAuth\Service\TwoFactorAuthService;

// 在控制器中注入服务
private TwoFactorAuthService $twoFactorAuthService;

// 登录逻辑
public function login()
{
    $userId = /* 通过用户名密码验证后获得的用户ID */;
    
    // 检查是否启用2FA
    if ($this->twoFactorAuthService->isEnabled($userId)) {
        // 如果启用，要求输入验证码
        $code = $this->request->getPost('2fa_code');
        
        if (!$code) {
            return $this->json([
                'require_2fa' => true,
                'message' => '请输入验证码'
            ]);
        }
        
        // 验证验证码
        if (!$this->twoFactorAuthService->verify($userId, $code)) {
            return $this->json([
                'success' => false,
                'message' => '验证码错误'
            ]);
        }
    }
    
    // 登录成功
    return $this->loginSuccess($userId);
}
```

### 通过API使用

```bash
# 初始化2FA
curl -X POST http://your-domain/api/2fa/initialize \
  -d "user_id=1&account=admin@example.com"

# 启用2FA
curl -X POST http://your-domain/api/2fa/enable \
  -d "user_id=1&secret=JBSWY3DPEHPK3PXP&code=123456"

# 验证验证码
curl -X POST http://your-domain/api/2fa/verify \
  -d "user_id=1&code=123456"
```
