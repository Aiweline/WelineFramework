# 语言选择器弹层层级修复

## 问题

后台网站编辑等表单中使用 `<w:i18n:language:select>` 时，下拉弹层原本作为控件内部的绝对定位元素渲染。页面外层如果存在 `overflow: hidden`、滚动容器或新的层叠上下文，即使提高弹层 `z-index`，也可能被父容器裁剪或遮住。

## 修复

- `Weline\I18n\Taglib\LanguageSelect` 默认将下拉弹层打开时临时挂载到 `document.body`。
- 弹层改用 `position: fixed` 根据触发按钮的视口位置计算宽度、上下方向和最大高度。
- 保留 `inline-dropdown="true"` 作为内联渲染退路，兼容确实需要跟随局部容器的特殊场景。

## 验证

- `php -l app/code/Weline/I18n/Taglib/LanguageSelect.php`
