# Web Components 架构建议

## 您的方案评估

### ✅ 优点

1. **统一管理**：将语言、货币、地区切换统一到一个 `switcher` 组件下，便于维护
2. **代码复用**：避免重复代码，提高开发效率
3. **一致性**：所有切换器使用相同的交互模式和样式

### ⚠️ 需要考虑的问题

1. **组件粒度**：如果所有切换器都放在一个组件中，可能会导致组件过于复杂
2. **灵活性**：不同切换器可能有不同的需求（如地区切换可能需要地图选择）

## 推荐的架构方案

### 方案一：统一 Switcher 组件（推荐）✅

**优点**：
- 代码统一，易于维护
- 交互一致，用户体验好
- 符合 DRY 原则

**实现**：
- 使用 `type` 属性区分不同类型
- 通过配置和事件处理不同逻辑
- 保持组件简洁，只负责 UI 和交互

**适用场景**：
- 语言、货币、地区切换功能相似
- 都是下拉选择模式
- 不需要特殊交互

### 方案二：基础组件 + 扩展组件

**优点**：
- 更灵活，可以针对不同场景定制
- 组件职责更清晰

**实现**：
```html
<!-- 基础组件 -->
<weline-switcher-base type="language">...</weline-switcher-base>

<!-- 扩展组件 -->
<weline-language-switcher>...</weline-language-switcher>
<weline-currency-switcher>...</weline-currency-switcher>
<weline-region-switcher>...</weline-region-switcher>
```

**适用场景**：
- 不同切换器有显著不同的交互需求
- 需要高度定制化

## SEO 友好性设计原则

### 1. 不使用 Shadow DOM（或谨慎使用）

**问题**：
- Shadow DOM 中的内容对搜索引擎不可见
- 即使使用 Declarative Shadow DOM，支持也有限

**解决方案**：
- ✅ 使用 Light DOM（推荐）
- ✅ 或使用 Declarative Shadow DOM + polyfill
- ✅ 在 Shadow DOM 外保留 SEO 内容

### 2. 服务端渲染关键内容

**原则**：
- 所有选项链接必须在初始 HTML 中
- 使用 PHP 模板渲染，而不是 JavaScript 生成
- 确保禁用 JS 时基础功能仍然可用

**实现**：
```php
<!-- ✅ 正确：内容在服务端渲染 -->
<weline-switcher>
    <a href="/zh_Hans_CN/page" data-value="zh_Hans_CN">中文</a>
    <a href="/en_US/page" data-value="en_US">English</a>
</weline-switcher>

<!-- ❌ 错误：内容由 JS 生成 -->
<weline-switcher id="switcher"></weline-switcher>
<script>
    // 搜索引擎无法索引这些内容
    document.getElementById('switcher').innerHTML = '...';
</script>
```

### 3. 渐进增强

**原则**：
- 基础功能是链接，不依赖 JavaScript
- JavaScript 只负责交互增强（下拉菜单、动画等）
- 确保无 JS 时仍可使用

**实现**：
```html
<!-- 基础功能：链接 -->
<a href="/zh_Hans_CN/page">中文</a>
<a href="/en_US/page">English</a>

<!-- JS 增强：下拉菜单 -->
<weline-switcher>
    <a href="/zh_Hans_CN/page" data-value="zh_Hans_CN">中文</a>
    <a href="/en_US/page" data-value="en_US">English</a>
</weline-switcher>
```

### 4. 语义化 HTML

**原则**：
- 使用合适的 HTML 标签（`<nav>`、`<a>`、`<ul>`、`<li>`）
- 添加 ARIA 属性提升可访问性
- 使用有意义的文本标签

**实现**：
```html
<weline-switcher 
    type="language" 
    role="combobox"
    aria-label="选择语言">
    <a href="/zh_Hans_CN/page" data-value="zh_Hans_CN" aria-current="true">中文</a>
    <a href="/en_US/page" data-value="en_US">English</a>
</weline-switcher>
```

### 5. 隐藏 SEO 内容

**原则**：
- 保留原始链接结构用于 SEO
- 使用 CSS 隐藏但保持可访问
- 使用 `aria-hidden="true"` 标记

**实现**：
```html
<weline-switcher>
    <!-- 交互UI -->
    <button>中文 ▼</button>
    <ul>...</ul>
    
    <!-- SEO内容（隐藏但可索引） -->
    <div aria-hidden="true" style="position: absolute; clip: rect(0,0,0,0);">
        <a href="/zh_Hans_CN/page">中文</a>
        <a href="/en_US/page">English</a>
    </div>
</weline-switcher>
```

## 最终推荐方案

### 架构设计

```
weline-switcher (统一组件)
├── type="language" → 语言切换
├── type="currency" → 货币切换
├── type="region" → 地区切换
└── type="locale" → 区域设置切换
```

### 实现特点

1. **SEO 友好**：
   - ✅ 使用 Light DOM
   - ✅ 服务端渲染内容
   - ✅ 渐进增强
   - ✅ 语义化 HTML

2. **可维护性**：
   - ✅ 统一组件，代码复用
   - ✅ 通过配置区分不同类型
   - ✅ 清晰的 API 和事件

3. **用户体验**：
   - ✅ 一致的交互模式
   - ✅ 键盘导航支持
   - ✅ 屏幕阅读器支持
   - ✅ 响应式设计

4. **扩展性**：
   - ✅ 易于添加新类型
   - ✅ 支持自定义样式
   - ✅ 支持事件监听

## 使用建议

### 1. 在 Header 中使用

```php
<!-- header.phtml -->
<div class="header-actions">
    <!-- 语言切换 -->
    <w:template>Weline_Frontend::components/switcher.phtml</w:template>
    <?php
    $this->setData('type', 'language');
    $this->setData('current', $_SERVER['WELINE_USER_LANG'] ?? 'zh_Hans_CN');
    ?>
    
    <!-- 货币切换 -->
    <w:template>Weline_Frontend::components/switcher.phtml</w:template>
    <?php
    $this->setData('type', 'currency');
    $this->setData('current', $_SERVER['WELINE_USER_CURRENCY'] ?? 'CNY');
    ?>
</div>
```

### 2. 在 Footer 中使用

```php
<!-- footer.phtml -->
<footer>
    <div class="footer-locale">
        <w:template>Weline_Frontend::components/switcher.phtml</w:template>
        <?php
        $this->setData('type', 'language');
        $this->setData('current', $_SERVER['WELINE_USER_LANG'] ?? 'zh_Hans_CN');
        ?>
    </div>
</footer>
```

### 3. 在设置页面中使用

```php
<!-- account/settings.phtml -->
<div class="settings-section">
    <h3><?= __('语言设置') ?></h3>
    <w:template>Weline_Frontend::components/switcher.phtml</w:template>
    <?php
    $this->setData('type', 'language');
    $this->setData('current', $_SERVER['WELINE_USER_LANG'] ?? 'zh_Hans_CN');
    ?>
</div>
```

3. ✅ **用户体验一致**：所有切换器交互相同
