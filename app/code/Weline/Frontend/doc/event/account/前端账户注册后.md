# Weline Frontend 模块 - 事件文档

## 概述

本文档详细说明了 Weline Frontend 模块提供的 `Weline_Weline_Frontend_Account_Register::register_after` 事件及其使用方法。该事件在前端用户注册成功后触发，允许其他模块监听注册事件并执行相应操作。

## 事件列表

### 1. Weline_Weline_Frontend_Account_Register::register_after - 前端账户注册后事件

#### 基本信息

- **事件名称**：`Weline_Weline_Frontend_Account_Register::register_after`
- **事件类型**：用户认证事件
- **触发时机**：在前端用户注册成功后
- **触发位置**：`app/code/Weline/Frontend/Controller/Account/Register.php` 第 127 行
- **配置文件**：`app/code/Weline/Frontend/etc/event.xml`

#### 功能说明

`Weline_Weline_Frontend_Account_Register::register_after` 事件在前端用户注册成功后触发，此时：
- 新用户已成功创建
- 用户信息已保存到数据库
- 用户会话已创建
- 可以执行注册后的操作

该事件主要用于：
- 发送欢迎邮件
- 记录注册日志
- 更新统计信息
- 执行注册后的业务逻辑
- 访客像素追踪
- 新用户奖励

#### 触发时机

```php
// app/code/Weline/Frontend/Controller/Account/Register.php
// 用户注册成功后
$eventManager = ObjectManager::getInstance(EventsManager::class);
$eventData = new \Weline\Framework\DataObject\DataObject([
    'user' => $newUser,
    'request' => $this->request,
    'session' => $this->session
]);
$eventManager->dispatch('Weline_Weline_Frontend_Account_Register::register_after', $eventData);
```

#### 使用场景

- **欢迎邮件**：发送欢迎邮件给新注册用户
- **注册日志**：记录用户注册日志，包括注册时间、IP地址等
- **统计信息**：更新用户注册统计信息
- **业务逻辑**：执行注册后的业务逻辑，如赠送新人优惠券、初始化用户数据等
- **访客追踪**：记录访客像素数据，用于分析和追踪
- **用户激活**：执行用户激活相关操作

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Weline_Frontend_Account_Register::register_after">
        <observer name="Your_Module::register_after"
                  instance="Your\Module\Observer\RegisterAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class RegisterAfterObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $user = $data->getData('user');           // 新注册的用户对象
        $request = $data->getData('request');      // 请求对象
        $session = $data->getData('session');      // 会话对象
        
        // 执行注册后操作
        // 例如：发送欢迎邮件、记录日志等
    }
}
```

#### 事件数据

`Weline_Weline_Frontend_Account_Register::register_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| user | object | 新注册的用户对象 |
| request | object | HTTP请求对象 |
| session | object | 会话对象 |

**获取数据**：

```php
$data = $event->getData();
$user = $data->getData('user');
$request = $data->getData('request');
$session = $data->getData('session');
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **异常处理**：如果观察者抛出异常，可能会影响注册流程
3. **性能考虑**：该事件在每次用户注册时都会触发，应避免执行耗时操作
4. **错误处理**：建议在观察者中添加异常处理，避免影响其他观察者执行
5. **用户信息**：用户对象包含完整的用户信息，可以直接使用
6. **异步处理**：对于耗时操作（如发送邮件），建议使用队列异步处理
