# Weline Frontend 模块 - 事件文档

## 概述

本文档详细说明了 Weline Frontend 模块提供的 `Weline_Weline_Frontend_Account_Login::login_after` 事件及其使用方法。该事件在前端用户登录成功后触发，允许其他模块监听登录事件并执行相应操作。

## 事件列表

### 1. Weline_Weline_Frontend_Account_Login::login_after - 前端账户登录后事件

#### 基本信息

- **事件名称**：`Weline_Weline_Frontend_Account_Login::login_after`
- **事件类型**：用户认证事件
- **触发时机**：在前端用户登录成功后
- **触发位置**：`app/code/Weline/Frontend/Controller/Account/Login.php` 第 169 行
- **配置文件**：`app/code/Weline/Frontend/etc/event.xml`

#### 功能说明

`Weline_Weline_Frontend_Account_Login::login_after` 事件在前端用户登录成功后触发，此时：
- 用户已成功通过身份验证
- 用户会话已创建
- 用户信息已加载
- 可以执行登录后的操作

该事件主要用于：
- 记录登录日志
- 更新用户最后登录时间
- 发送登录通知
- 统计登录信息
- 执行登录后的业务逻辑
- 访客像素追踪

#### 触发时机

```php
// app/code/Weline/Frontend/Controller/Account/Login.php
// 用户登录成功后
$eventManager = ObjectManager::getInstance(EventsManager::class);
$eventData = new \Weline\Framework\DataObject\DataObject([
    'user' => $user,
    'request' => $this->request,
    'session' => $this->session
]);
$eventManager->dispatch('Weline_Weline_Frontend_Account_Login::login_after', $eventData);
```

#### 使用场景

- **登录日志**：记录用户登录日志，包括登录时间、IP地址等
- **最后登录时间**：更新用户的最后登录时间
- **登录通知**：发送登录成功通知给用户
- **统计信息**：统计用户登录次数、活跃度等
- **业务逻辑**：执行登录后的业务逻辑，如加载用户数据、初始化购物车等
- **访客追踪**：记录访客像素数据，用于分析和追踪

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Weline_Frontend_Account_Login::login_after">
        <observer name="Your_Module::login_after"
                  instance="Your\Module\Observer\LoginAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class LoginAfterObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $user = $data->getData('user');           // 用户对象
        $request = $data->getData('request');      // 请求对象
        $session = $data->getData('session');      // 会话对象
        
        // 执行登录后操作
        // 例如：记录日志、更新统计等
    }
}
```

#### 事件数据

`Weline_Weline_Frontend_Account_Login::login_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| user | object | 登录成功的用户对象 |
| request | object | HTTP请求对象 |
| session | object | 会话对象 |

**获取数据**：

```php
$data = $event->getData();
$user = $data->getData('user');
$request = $data->getData('request');
$session = $data->getData('session');
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **异常处理**：如果观察者抛出异常，可能会影响登录流程
3. **性能考虑**：该事件在每次用户登录时都会触发，应避免执行耗时操作
4. **错误处理**：建议在观察者中添加异常处理，避免影响其他观察者执行
5. **用户信息**：用户对象包含完整的用户信息，可以直接使用
