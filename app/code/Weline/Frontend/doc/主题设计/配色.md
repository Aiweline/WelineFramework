## 设计理念

### 核心原则
1. **统一性**：所有模块使用统一的配色系统，确保视觉一致性
2. **可维护性**：通过CSS变量集中管理颜色，便于全局调整
3. **可扩展性**：支持多主题切换，易于扩展新主题
4. **语义化**：使用语义化的变量名，而非具体颜色值
5. **响应式**：支持暗色/亮色主题自动切换

### 设计风格
- **参考风格**：Amazon简约风格
- **设计理念**：实用、清晰、专业
- **色彩倾向**：中性色为主，品牌色为辅

- **成功** (Success): `#28a745`
- **警告** (Warning): `#ffc107`
- **错误** (Error): `#dc3545`

## CSS变量系统

### 命名规范

#### 1. 变量命名规则
- 使用 `--` 前缀（CSS自定义属性标准）
- 使用小写字母和连字符
- 语义化命名，而非具体颜色值
- 分组命名：`--color-*`, `--spacing-*`, `--font-*` 等

#### 2. 命名示例

```css
/* ✅ 正确 - 语义化命名 */
--color-primary
--color-text-primary
--color-bg-secondary
--spacing-md
--font-size-base

/* ❌ 错误 - 具体颜色值命名 */
--color-yellow
--color-gray-500
--spacing-16px
```

### 使用方式

#### 1. 在CSS中使用

```css
.button {
    background-color: var(--color-primary);
    color: var(--color-text-primary);
    padding: var(--spacing-sm) var(--spacing-md);
    border-radius: var(--border-radius-md);
    box-shadow: var(--shadow-md);
}

.button:hover {
    background-color: var(--color-primary-light);
}

.button:focus {
    outline: 2px solid var(--color-border-focus);
    box-shadow: var(--shadow-focus);
}
```

#### 2. 在HTML中使用（内联样式）

```html
<div style="background-color: var(--color-bg-primary); color: var(--color-text-primary);">
    内容
</div>
```

#### 3. 在JavaScript中使用

```javascript
// 获取变量值
const primaryColor = getComputedStyle(document.documentElement)
    .getPropertyValue('--color-primary');

// 设置变量值
document.documentElement.style.setProperty('--color-primary', '#ff0000');
```

## 使用规范

### 1. 颜色使用规范

#### ✅ 推荐做法

```css
/* 使用语义化变量 */
.text-primary {
    color: var(--color-text-primary);
}

.bg-primary {
    background-color: var(--color-bg-primary);
}

.border-default {
    border-color: var(--color-border-default);
}
```

#### ❌ 避免做法

```css
/* 直接使用颜色值 */
.text-primary {
    color: #111;
}

.bg-primary {
    background-color: #fff;
}
```

### 2. 组件样式规范

```css
/**
 * 组件：按钮
 * 使用统一的变量系统
 */
.btn {
    /* 颜色 */
    background-color: var(--color-primary);
    color: var(--color-text-primary);
    border: var(--border-width-thin) solid var(--color-primary-border);
    
    /* 间距 */
    padding: var(--spacing-sm) var(--spacing-md);
    
    /* 字体 */
    font-family: var(--font-family-base);
    font-size: var(--font-size-base);
    font-weight: var(--font-weight-normal);
    line-height: var(--line-height-normal);
    
    /* 边框 */
    border-radius: var(--border-radius-md);
    
    /* 阴影 */
    box-shadow: var(--shadow-sm);
    
    /* 过渡 */
    transition: all 0.2s ease;
}

.btn:hover {
    background-color: var(--color-primary-light);
    box-shadow: var(--shadow-md);
}

.btn:focus {
    outline: 2px solid var(--color-border-focus);
    box-shadow: var(--shadow-focus);
}

.btn:active {
    background-color: var(--color-primary-dark);
}
```

### 3. 模块扩展规范

其他模块应遵循以下规范：

```css
/**
 * 模块：YourModule
 * 路径：app/code/Weline/YourModule/view/statics/css/module.css
 */

/* 1. 导入核心变量（如果需要） */
@import url('/static/Weline_Frontend/theme/assets/css/theme.css');

/* 2. 使用核心变量 */
.your-module-component {
    background-color: var(--color-bg-primary);
    color: var(--color-text-primary);
    padding: var(--spacing-md);
    border: var(--border-width-thin) solid var(--color-border-default);
    border-radius: var(--border-radius-md);
}

/* 3. 如需扩展变量，使用模块前缀 */
:root {
    --your-module-color-accent: #ff0000;
}
```

## 主题切换机制

### 1. HTML属性切换

```html
<!-- 亮色主题（默认） -->
<html data-theme="light">

<!-- 暗色主题 -->
<html data-theme="dark">
```

### 2. JavaScript切换

```javascript
// 切换主题
function switchTheme(theme) {
    document.documentElement.setAttribute('data-theme', theme);
    localStorage.setItem('weline-theme', theme);
}

// 初始化主题
function initTheme() {
    const savedTheme = localStorage.getItem('weline-theme') || 'light';
    switchTheme(savedTheme);
}

// 监听系统主题变化
if (window.matchMedia) {
    const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)');
    mediaQuery.addEventListener('change', (e) => {
        switchTheme(e.matches ? 'dark' : 'light');
    });
}
```

### 3. 在theme.js中集成

```javascript
// 在 theme.js 中添加主题管理
Weline.Theme = {
    current: 'light',
    
    switch: function(theme) {
        document.documentElement.setAttribute('data-theme', theme);
        localStorage.setItem('weline-theme', theme);
        this.current = theme;
    },
    
    init: function() {
        const saved = localStorage.getItem('weline-theme');
        const system = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
        const theme = saved || system;
        this.switch(theme);
    }
};

// 页面加载时初始化
Weline.Theme.init();
```

## 最佳实践

### 1. 颜色选择原则

- **对比度**：确保文本与背景的对比度符合WCAG AA标准（至少4.5:1）
- **语义化**：使用语义化的颜色（success, error等）而非具体颜色值
- **一致性**：相同功能的元素使用相同的颜色变量

### 2. 变量管理原则

- **集中管理**：所有颜色变量集中在 `_colors.css` 中定义
- **避免重复**：不要在不同文件中重复定义相同的颜色值
- **版本控制**：颜色变更应记录在版本更新日志中

### 3. 组件开发原则

- **使用变量**：组件样式必须使用CSS变量，不得硬编码颜色值
- **响应主题**：组件应自动适配亮色/暗色主题
- **文档化**：复杂组件应提供样式使用文档

### 4. 性能优化

- **CSS变量**：CSS变量性能优秀，可放心使用
- **避免过度嵌套**：变量引用不要超过3层
- **按需加载**：主题文件可按需加载

## 扩展指南

### 1. 添加新颜色

1. 在 `view/theme/variables/_colors.css` 中添加变量定义
2. 在亮色和暗色主题中分别定义值
3. 更新本文档的配色表
4. 在组件中使用新变量

### 2. 创建新颜色主题

1. 创建 `view/theme/colors/_your-theme.css` 文件
2. 覆盖需要修改的颜色变量
3. 在 `theme.css` 中导入新颜色主题
4. 更新主题切换逻辑

### 3. 模块集成

其他模块应：
1. 导入核心变量文件
2. 使用统一的变量命名规范
3. 遵循组件样式规范
4. 支持主题切换
