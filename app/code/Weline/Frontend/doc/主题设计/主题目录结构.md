# Weline Frontend 主题抽象目录结构

## 概述

`view/theme/` 目录是 Weline Frontend 模块的主题抽象层，提供统一的设计系统、组件模板和样式规范，供 Frontend 模块内部和其他模块引用使用。

## 目录结构

```
app/code/Weline/Frontend/view/
├── theme/                          # 主题抽象目录（核心）
│   ├── variables/                  # CSS变量定义
│   │   ├── _colors.css            # 颜色变量（核心）
│   │   ├── _spacing.css           # 间距变量
│   │   ├── _typography.css        # 字体变量
│   │   ├── _shadows.css           # 阴影变量
│   │   └── _borders.css           # 边框变量
│   ├── colors/                     # 颜色主题定义
│   │   ├── _light.css             # 亮色主题颜色
│   │   ├── _dark.css              # 暗色主题颜色
│   │   └── _amazon.css            # Amazon风格主题颜色（默认）
│   ├── components/                 # 可复用组件模板
│   │   ├── button.phtml           # 按钮组件
│   │   ├── input.phtml            # 输入框组件
│   │   ├── card.phtml             # 卡片组件
│   │   ├── modal.phtml            # 模态框组件
│   │   ├── alert.phtml            # 提示框组件
│   │   └── form-group.phtml       # 表单组组件
│   ├── layouts/                    # 布局模板
│   │   ├── default.phtml          # 默认布局
│   │   ├── auth.phtml             # 认证页面布局（登录/注册）
│   │   ├── dashboard.phtml        # 仪表盘布局
│   │   └── minimal.phtml          # 极简布局
│   ├── partials/                   # 部分模板片段
│   │   ├── header.phtml           # 头部片段
│   │   ├── footer.phtml           # 底部片段
│   │   ├── sidebar.phtml          # 侧边栏片段
│   │   └── breadcrumb.phtml       # 面包屑片段
│   ├── assets/                     # 主题静态资源
│   │   ├── css/
│   │   │   ├── theme.css          # 主题主样式（导入所有变量）
│   │   │   ├── components.css     # 组件样式
│   │   │   └── utilities.css      # 工具类样式
│   │   ├── js/
│   │   │   └── theme.js           # 主题JS（主题切换、工具函数）
│   │   └── images/
│   │       └── theme/              # 主题相关图片
│   │           ├── logo.svg
│   │           └── placeholder.png
│   └── config/                     # 主题配置
│       └── theme.json              # 主题配置文件（元数据）
├── blocks/                         # 现有区块（保持不变）
├── frontend/                       # 现有前端页面（保持不变）
├── statics/                        # 现有静态资源（保持不变）
└── templates/                      # 现有模板（保持不变）
```

## 目录说明

### 1. `variables/` - CSS变量定义

**作用**：定义所有设计系统的核心变量

**文件说明**：
- `_colors.css` - 颜色变量（必读，参考 `配色.md`）
- `_spacing.css` - 间距变量（xs, sm, md, lg, xl等）
- `_typography.css` - 字体变量（字体族、大小、粗细、行高）
- `_shadows.css` - 阴影变量（sm, md, lg, xl, focus等）
- `_borders.css` - 边框变量（宽度、圆角等）

**使用方式**：
```css
/* 在 theme/assets/css/theme.css 中导入 */
@import '../variables/_colors.css';
@import '../variables/_spacing.css';
/* ... */
```

### 2. `colors/` - 颜色主题定义

**作用**：定义不同颜色主题的变量覆盖值

**文件说明**：
- `_light.css` - 亮色主题颜色（默认）
- `_dark.css` - 暗色主题颜色
- `_amazon.css` - Amazon风格主题颜色（可扩展）

**使用方式**：
```css
/* 在 theme/assets/css/theme.css 中按需导入 */
@import '../colors/_light.css';
@import '../colors/_dark.css';
```

### 3. `components/` - 可复用组件模板

**作用**：提供可复用的UI组件模板

**组件列表**：
- `button.phtml` - 按钮组件
  - 参数：`text`, `type` (primary/secondary), `size`, `class`, `icon`
- `input.phtml` - 输入框组件
  - 参数：`name`, `type`, `placeholder`, `label`, `required`, `error`
- `card.phtml` - 卡片组件
  - 参数：`title`, `content`, `footer`, `class`
- `modal.phtml` - 模态框组件
  - 参数：`id`, `title`, `content`, `footer`, `size`
- `alert.phtml` - 提示框组件
  - 参数：`type` (success/error/warning/info), `message`, `dismissible`
- `form-group.phtml` - 表单组组件
  - 参数：`label`, `input`, `help`, `error`

**使用方式**：
```php
<?php
// 在模板中使用组件
echo $this->fetch('Weline_Frontend::theme/components/button.phtml', [
    'text' => __('登录'),
    'type' => 'primary',
    'size' => 'md',
    'class' => 'btn-login'
]);
?>
```

### 4. `layouts/` - 布局模板

**作用**：提供页面布局模板

**布局列表**：
- `default.phtml` - 默认布局（包含header和footer）
- `auth.phtml` - 认证页面布局（登录/注册页面）
- `dashboard.phtml` - 仪表盘布局（带侧边栏）
- `minimal.phtml` - 极简布局（无header/footer）

**使用方式**：
```php
<?php
// 在控制器中使用布局
$this->assign('layout', 'Weline_Frontend::theme/layouts/auth.phtml');
return $this->fetch('your-page');
?>
```

### 5. `partials/` - 部分模板片段

**作用**：提供可复用的页面片段

**片段列表**：
- `header.phtml` - 头部片段（可被layouts引用）
- `footer.phtml` - 底部片段（可被layouts引用）
- `sidebar.phtml` - 侧边栏片段
- `breadcrumb.phtml` - 面包屑导航片段

**使用方式**：
```php
<?php
// 在布局或页面中使用片段
echo $this->fetch('Weline_Frontend::theme/partials/header.phtml');
?>
```

### 6. `assets/` - 主题静态资源

**作用**：存放主题相关的CSS、JS和图片资源

**文件说明**：
- `css/theme.css` - 主样式文件（导入所有变量和主题）
- `css/components.css` - 组件样式
- `css/utilities.css` - 工具类样式
- `js/theme.js` - 主题JS（主题切换、工具函数）
- `images/theme/` - 主题相关图片

**使用方式**：
```php
<!-- 在 head.phtml 中加载 -->
<link rel="stylesheet" href="@static(Weline_Frontend::theme/assets/css/theme.css)">
<script src="@static(Weline_Frontend::theme/assets/js/theme.js)"></script>
```

### 7. `config/` - 主题配置

**作用**：存放主题的元数据和配置信息

**文件说明**：
- `theme.json` - 主题配置文件
  ```json
  {
    "name": "Weline Default Theme",
    "version": "1.0.0",
    "description": "Weline Frontend 默认主题",
    "author": "Weline Team",
    "variables": {
      "primary-color": "#f0c14b",
      "font-family": "-apple-system, BlinkMacSystemFont, 'Segoe UI'"
    },
    "themes": ["light", "dark", "amazon"]
  }
  ```

## 加载机制

### 1. CSS加载顺序

```php
<!-- 在 head.phtml 中 -->
<!-- 1. 先加载主题变量和基础样式 -->
<link rel="stylesheet" href="@static(Weline_Frontend::theme/assets/css/theme.css)">

<!-- 2. 根据主题模式加载特定颜色主题 -->
<?php
$themeMode = $themeConfig->getThemeModel() ?: 'light';
if ($themeMode && $themeMode !== 'light'): ?>
<link rel="stylesheet" href="@static(Weline_Frontend::theme/colors/_<?= $themeMode ?>.css)">
<?php endif; ?>
```

### 2. 模板引用路径

```php
// 引用主题组件
$this->fetch('Weline_Frontend::theme/components/button.phtml', $data);

// 引用主题布局
$this->fetch('Weline_Frontend::theme/layouts/auth.phtml', $data);

// 引用主题片段
$this->fetch('Weline_Frontend::theme/partials/header.phtml', $data);
```

### 3. 静态资源路径

```php
// CSS文件
@static(Weline_Frontend::theme/assets/css/theme.css)

// JS文件
@static(Weline_Frontend::theme/assets/js/theme.js)

// 图片文件
@static(Weline_Frontend::theme/assets/images/theme/logo.svg)
```

## 与其他目录的关系

### 1. 与 `templates/` 的关系

- `templates/` - 公共模板（head.phtml, header.phtml, footer.phtml）
- `theme/partials/` - 主题片段（可被templates引用）
- `theme/layouts/` - 主题布局（使用templates中的公共模板）

**关系图**：
```
templates/public/head.phtml
    ↓ 引用
theme/assets/css/theme.css
    ↓ 引用
theme/variables/_colors.css
```

### 2. 与 `statics/` 的关系

- `statics/` - 通用静态资源（第三方库、通用图片等）
- `theme/assets/` - 主题特定资源（主题CSS、JS、图片）

**关系图**：
```
statics/libs/bootstrap/  (第三方库)
    ↓
theme/assets/css/theme.css  (主题样式，可能使用Bootstrap)
```

### 3. 与 `blocks/` 的关系

- `blocks/` - PHP Block类对应的模板
- `theme/components/` - 纯模板组件（无PHP逻辑）

**关系图**：
```
blocks/header/base.phtml  (PHP Block)
    ↓ 可能引用
theme/components/button.phtml  (纯模板组件)
```

## 其他模块使用规范

### 1. 引用主题变量

```css
/* 在其他模块的CSS中 */
@import url('/static/Weline_Frontend/theme/assets/css/theme.css');

.my-module-component {
    background-color: var(--color-bg-primary);
    color: var(--color-text-primary);
    padding: var(--spacing-md);
}
```

### 2. 使用主题组件

```php
<?php
// 在其他模块的模板中
echo $this->fetch('Weline_Frontend::theme/components/button.phtml', [
    'text' => __('提交'),
    'type' => 'primary'
]);
?>
```

### 3. 扩展主题变量

```css
/* 在其他模块中扩展变量 */
:root {
    /* 使用基础变量 */
    --my-module-primary: var(--color-primary);
    
    /* 扩展新变量（使用模块前缀） */
    --my-module-accent: #ff0000;
}
```

## 开发规范

### 1. 文件命名规范

- CSS变量文件：使用 `_` 前缀（`_colors.css`）
- 主题文件：使用 `_` 前缀（`_light.css`）
- 组件文件：使用小写和连字符（`button.phtml`）
- 布局文件：使用小写和连字符（`auth.phtml`）

### 2. 变量命名规范

- 使用语义化命名：`--color-primary` 而非 `--color-yellow`
- 使用分组前缀：`--color-*`, `--spacing-*`, `--font-*`
- 使用连字符分隔：`--color-text-primary`

### 3. 组件开发规范

- 组件应使用主题变量，不得硬编码颜色值
- 组件应支持主题切换（自动适配亮色/暗色）
- 组件应提供清晰的参数文档

### 4. 版本管理

- 主题变量变更应更新 `配色.md` 文档
- 新增组件应更新本文档
- 重大变更应记录版本历史

## 最佳实践

### 1. 变量使用

✅ **推荐**：
```css
.button {
    background-color: var(--color-primary);
    padding: var(--spacing-sm) var(--spacing-md);
}
```

❌ **避免**：
```css
.button {
    background-color: #f0c14b;
    padding: 8px 16px;
}
```

### 2. 组件复用

✅ **推荐**：
```php
<?= $this->fetch('Weline_Frontend::theme/components/button.phtml', $data) ?>
```

❌ **避免**：
```html
<button style="background: #f0c14b; padding: 8px 16px;">按钮</button>
```

### 3. 主题扩展

✅ **推荐**：
```css
/* 在模块中扩展，使用模块前缀 */
:root {
    --my-module-color: var(--color-primary);
}
```

❌ **避免**：
```css
/* 直接修改基础变量 */
:root {
    --color-primary: #ff0000; /* 不要这样做 */
}
```

## 迁移指南

### 从现有代码迁移到主题系统

1. **识别硬编码的颜色值**
   ```css
   /* 旧代码 */
   .button { background: #f0c14b; }
   
   /* 新代码 */
   .button { background: var(--color-primary); }
   ```

2. **提取重复的组件为模板**
   ```php
   <!-- 旧代码：重复的按钮HTML -->
   <button class="btn btn-primary">登录</button>
   
   <!-- 新代码：使用组件 -->
   <?= $this->fetch('Weline_Frontend::theme/components/button.phtml', [
       'text' => __('登录'),
       'type' => 'primary'
   ]) ?>
   ```

3. **使用布局模板**
   ```php
   // 旧代码：每个页面重复header/footer
   // 新代码：使用布局模板
   $this->assign('layout', 'Weline_Frontend::theme/layouts/default.phtml');
   ```
