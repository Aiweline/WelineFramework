# variables/ 与 colors/ 目录区别说明

## 核心区别

### `variables/_colors.css` - 颜色变量定义（基础层）

**作用**：定义所有颜色变量的**名称**和**默认值**

**内容**：定义变量的默认值（通常使用亮色主题作为默认值）

**示例**：
```css
/**
 * variables/_colors.css
 * 定义颜色变量的默认值（亮色主题作为默认）
 */
:root {
    /* 品牌色 */
    --color-primary: #f0c14b;
    --color-primary-light: #f4d078;
    --color-primary-dark: #e7b92e;
    
    /* 文本色 */
    --color-text-primary: #111;
    --color-text-secondary: #767676;
    
    /* 背景色 */
    --color-bg-primary: #fff;
    --color-bg-secondary: #f8f9fa;
    
    /* 功能色 */
    --color-success: #28a745;
    --color-error: #dc3545;
}
```

**特点**：
- 定义变量的**完整列表**
- 提供**默认值**（通常是亮色主题的值）
- 所有变量都在这里**声明**

---

### `colors/` - 颜色主题覆盖（主题层）

**作用**：根据不同主题**覆盖**颜色变量的值

**内容**：只覆盖需要改变的颜色变量值

**示例**：
```css
/**
 * colors/_light.css
 * 亮色主题（默认，通常为空或与variables中的默认值一致）
 */
[data-theme="light"],
:root {
    /* 如果与默认值一致，可以省略 */
    /* 或者明确指定，便于理解 */
    --color-text-primary: #111;
    --color-bg-primary: #fff;
}

/**
 * colors/_dark.css
 * 暗色主题 - 覆盖需要改变的颜色变量
 */
[data-theme="dark"] {
    /* 只覆盖需要改变的值 */
    --color-text-primary: #e5e5e5;  /* 覆盖默认的 #111 */
    --color-text-secondary: #a6a6a6; /* 覆盖默认的 #767676 */
    --color-bg-primary: #1a1a1a;     /* 覆盖默认的 #fff */
    --color-bg-secondary: #2d2d2d;   /* 覆盖默认的 #f8f9fa */
    
    /* 品牌色通常不变，所以不需要覆盖 */
    /* --color-primary: #f0c14b; */  /* 保持不变 */
}
```

**特点**：
- 只覆盖**需要改变**的变量
- 使用 `[data-theme="xxx"]` 选择器
- 不改变的值可以省略

## 工作流程

### 1. 加载顺序

```css
/* theme.css 中的加载顺序 */

/* 第一步：加载变量定义（默认值） */
@import '../variables/_colors.css';
/* 此时所有变量都有默认值 */

/* 第二步：加载主题覆盖 */
@import '../colors/_light.css';  /* 亮色主题（默认） */
@import '../colors/_dark.css';   /* 暗色主题覆盖 */
```

### 2. 变量解析过程

```css
/* 1. variables/_colors.css 定义默认值 */
:root {
    --color-text-primary: #111;  /* 默认值 */
}

/* 2. colors/_dark.css 覆盖值 */
[data-theme="dark"] {
    --color-text-primary: #e5e5e5;  /* 覆盖为暗色值 */
}

/* 3. 使用 */
.text {
    color: var(--color-text-primary);
    /* 亮色主题：使用 #111 */
    /* 暗色主题：使用 #e5e5e5 */
}
```

## 为什么需要两个目录？

### 原因1：职责分离

- **`variables/`** - 定义**所有变量**和**默认值**（基础层）
- **`colors/`** - 定义**主题覆盖**（主题层）

### 原因2：可维护性

- 所有变量集中在一个地方定义（`variables/_colors.css`）
- 主题覆盖独立管理（`colors/`）
- 新增变量只需在 `variables/` 中定义
- 新增主题只需在 `colors/` 中添加覆盖文件

### 原因3：灵活性

- 可以定义多个主题（light, dark, amazon等）
- 每个主题只覆盖需要的变量
- 未覆盖的变量使用默认值

## 实际示例

### 场景：添加新颜色变量

**步骤1**：在 `variables/_colors.css` 中定义
```css
:root {
    --color-accent: #ff6b6b;  /* 新增强调色 */
}
```

**步骤2**：在 `colors/_dark.css` 中覆盖（如果需要）
```css
[data-theme="dark"] {
    --color-accent: #ff8787;  /* 暗色主题下的强调色 */
}
```

**步骤3**：在组件中使用
```css
.highlight {
    color: var(--color-accent);
    /* 亮色主题：使用 #ff6b6b */
    /* 暗色主题：使用 #ff8787 */
}
```

## 优化建议

### 方案A：保持当前结构（推荐）

**优点**：
- 职责清晰：变量定义 vs 主题覆盖
- 易于维护：所有变量集中定义
- 灵活扩展：新增主题只需添加覆盖文件

**结构**：
```
variables/_colors.css  → 定义所有变量和默认值
colors/_light.css      → 亮色主题（可选，默认值）
colors/_dark.css       → 暗色主题覆盖
```

### 方案B：合并到 variables/（简化）

**如果觉得两个目录太复杂，可以合并**：

```
variables/
├── _colors.css        # 颜色变量定义（默认值）
├── _colors-light.css  # 亮色主题
├── _colors-dark.css   # 暗色主题
├── _spacing.css
└── _typography.css
```

**优点**：
- 结构更简单
- 所有变量相关文件在一个目录

**缺点**：
- 变量定义和主题覆盖混在一起
- 不够清晰

---

## 推荐方案

**推荐保持当前结构**（`variables/` + `colors/`），因为：

1. **语义清晰**：
   - `variables/` = 变量定义（基础）
   - `colors/` = 主题覆盖（应用）

2. **易于理解**：
   - 开发者一看就知道：变量在哪里定义，主题在哪里覆盖

3. **便于扩展**：
   - 未来可能还有其他类型的主题覆盖（如 `spacing/`, `typography/` 等）
   - 保持结构一致性

---

## 总结

| 目录 | 作用 | 内容 | 示例 |
|------|------|------|------|
| `variables/_colors.css` | 定义所有颜色变量和默认值 | 完整的变量列表和默认值 | `--color-primary: #f0c14b;` |
| `colors/_dark.css` | 覆盖特定主题的变量值 | 只覆盖需要改变的值 | `[data-theme="dark"] { --color-text-primary: #e5e5e5; }` |

**简单记忆**：
- `variables/` = **定义**变量（基础层）
- `colors/` = **覆盖**变量（主题层）
