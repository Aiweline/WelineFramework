# 前端主题规划文档

## 📋 概述

本文档记录了前端主题系统的规划、设计和实现方案，包括主题结构重组、配置系统、模板标签系统等核心内容。

---

## 🎯 核心设计理念

### 1. 模块化与可配置性
- **主题组件化**：将页面拆分为可复用的组件和片段（partials）
- **配置驱动**：通过后台配置系统选择不同的主题选项
- **灵活扩展**：支持主题继承和多级配置

### 2. 前后端分离
- **独立主题系统**：前端和后端使用独立的主题目录结构
- **区域隔离**：`frontend/` 和 `backend/` 目录完全分离
- **独立配置**：前后端可以配置不同的 partials 选项

### 3. 标准化与规范化
- **统一命名规范**：遵循框架的命名约定
- **标准化目录结构**：统一的主题目录组织方式
- **模板标签系统**：使用标准化的模板加载标签

## 🔧 Partials 系统重组

### 设计目标

将原本扁平化的 `partials/` 目录重组为按类型分类的结构，支持每个类型有多种可选方案。

### 重组方案

#### 1. 目录结构重组

**原结构**（扁平化）：
```
partials/
├── header.phtml
├── footer.phtml
└── sidebar.phtml
```

**新结构**（类型分类）：
```
partials/
├── header/
│   ├── default.phtml
│   ├── minimal.phtml
│   └── centered.phtml
├── footer/
│   ├── default.phtml
│   └── minimal.phtml
└── sidebar/
    └── default.phtml
```

#### 2. 优势

- **可扩展性**：每个类型可以有多个选项
- **组织清晰**：按功能类型分类，便于管理
- **配置灵活**：后台可以选择不同的 partials 组合

#### 3. 实现方式

- **PartialsScanner 助手类**：扫描可用的 partials 选项
- **主题配置存储**：在 `WelineTheme` 模型的 `config` 字段中存储配置（JSON格式）
- **配置继承**：子主题可以继承父主题的配置

## ⚙️ 主题配置系统

### 配置存储

#### 数据库字段

在 `WelineTheme` 模型中添加 `config` 字段：

```php
public const fields_CONFIG = 'config';
```

字段类型：`TEXT`（存储 JSON 格式的配置数据）

#### 配置结构

```json
{
  "frontend": {
    "header": "default",
    "footer": "minimal",
    "sidebar": "default"
  },
  "backend": {
    "header": "default",
    "footer": "default",
    "sidebar": "default"
  }
}
```

### 配置管理

#### 1. 后台配置界面

- **访问路径**：`/theme/backend/config/partials?theme_id={主题ID}`
- **功能**：
  - 显示可用的 partials 选项
  - 支持为前端和后端分别配置
  - 实时预览配置效果
  - 保存配置到数据库

#### 2. 配置读取

使用 `Weline\Theme\Helper\PartialsScanner` 类：

```php
// 获取可用的 partials 选项
$availablePartials = PartialsScanner::getAvailablePartials($theme, 'frontend');

// 获取配置的 partials
$configuredPartials = PartialsScanner::getPartialsConfig($theme, 'frontend');

// 获取 partials 路径
$headerPath = PartialsScanner::getPartialsPath($theme, 'frontend', 'header', 'default');
```

#### 3. 配置继承

- 子主题可以继承父主题的配置
- 子主题可以覆盖父主题的配置
- 配置合并策略：子主题配置优先

## 🏷️ 模板标签系统

### 标签分类

#### 1. `<w:template>` 标签

**用途**：加载 `view/templates/` 目录下的模板文件

**语法**：
```html
<w:template>Weline_Module::path/to/template.phtml</w:template>
```

**示例**：
```html
<w:template>Weline_Admin::common/head.phtml</w:template>
<w:template>Weline_Admin::common/left-sidebar.phtml</w:template>
```

#### 2. `<w:theme:template>` 标签（新增）

**用途**：专门加载 `view/theme/` 目录下的主题模板文件

**语法**：
```html
<w:theme:template>Weline_Theme::frontend/components/loading.phtml</w:theme:template>
<w:theme:template>Weline_Theme::backend/components/message.phtml</w:theme:template>
```

**特点**：
- 自动识别 `dir_type_THEME` 类型
- 路径中不需要包含 `theme/` 前缀
- 支持前端和后端区域自动识别

**实现位置**：
- `app/code/Weline/Framework/View/Taglib.php`：标签定义
- `app/code/Weline/Framework/View/TraitTemplate.php`：路径解析

### 标签使用规范

#### ✅ 正确使用

```html
<!-- 加载 templates 目录下的文件 -->
<w:template>Weline_Admin::common/head.phtml</w:template>

<!-- 加载 theme 目录下的文件 -->
<w:theme:template>Weline_Theme::backend/components/message.phtml</w:theme:template>
```

#### ❌ 错误使用

```html
<!-- 错误：不应该用 w:template 加载 theme 文件 -->
<w:template>Weline_Theme::theme/backend/components/message.phtml</w:template>

<!-- 错误：不应该用 w:theme:template 加载 templates 文件 -->
<w:theme:template>Weline_Admin::common/head.phtml</w:theme:template>
```

## 🚀 主题 JS 优化

### 优化目标

1. **减少文件大小**：移除冗余代码
2. **提高性能**：异步加载，不阻塞主线程
3. **按需加载**：替换自动预加载为按需加载
4. **配置驱动**：从 `window.WelineConfig` 读取配置

### 优化内容

#### 1. 移除默认配置

**优化前**：
```javascript
const defaultConfig = {
    baseUrl: '/',
    // ... 大量默认配置
};
```

**优化后**：
```javascript
// 从 window.WelineConfig 读取配置
const config = window.WelineConfig || {};
```

#### 2. 移除自动预加载

**优化前**：
```javascript
// 自动预加载所有模块
Weline.preLoad(['module1', 'module2', ...]);
```

**优化后**：
```javascript
// 按需加载
Weline.load('module1').then(() => {
    // 模块加载完成
});
```

#### 3. 异步操作优化

- 所有 API 调用使用异步方式
- 使用 Promise 和 async/await
- 避免阻塞主线程

#### 4. 移除兼容代码

- 移除 `window.WelineApi` 兼容代码
- 移除 `window.WelineTheme` 兼容代码
- 移除 `window.WelineApiConfig` 兼容代码
- 统一使用 `Weline.*` 命名空间

## 🔄 主题继承机制

### 继承规则

1. **模块级主题继承**：模块可以创建自己的 `view/theme` 目录，继承 Theme 模块的主题文件
2. **数据库主题继承**：子主题可以继承父主题的所有文件（通过数据库 parent_id 字段）
3. **配置继承**：子主题配置可以继承父主题配置
4. **覆盖机制**：优先级从高到低：
   - 模块自己的 theme 目录（最高优先级）
   - 当前主题的文件
   - 父主题的文件
   - Theme 模块的基础文件（最低优先级）

### 模块级主题继承

#### 设计目的

- **配置分离**：前端有很多信息要配置（购物车、搜索、用户菜单等），不适合直接在 Theme 模块配置
- **模块定制**：每个模块可以定制自己的主题文件，只覆盖需要的部分
- **灵活扩展**：其他主题开发也可以继承，形成多级继承链

#### 继承顺序

当查找主题文件时，系统按以下顺序查找：

1. **当前请求模块的 theme 目录**
   ```
   app/code/Weline/{CurrentModule}/view/theme/{area}/partials/{type}/{option}.phtml
   ```

2. **默认模块的 theme 目录**（前端：Weline_Frontend，后端：Weline_Backend）
   ```
   app/code/Weline/Frontend/view/theme/frontend/partials/{type}/{option}.phtml
   ```

3. **当前主题的文件**（数据库主题）
   ```
   app/design/{ThemeName}/view/theme/{area}/partials/{type}/{option}.phtml
   ```

4. **父主题的文件**（递归查找）

5. **Theme 模块的基础文件**（最后备选）
   ```
   app/code/Weline/Theme/view/theme/{area}/partials/{type}/{option}.phtml
   ```

#### 使用示例

**Frontend 模块继承 Theme 模块的 Header**：

```php
// Frontend 模块创建自己的 header
// app/code/Weline/Frontend/view/theme/frontend/partials/header/default.phtml

<?php
/**
 * Frontend 模块 Header 片段
 * 继承 Theme 模块的默认 Header，添加前端特定功能
 */
// 添加购物车、搜索、用户菜单等前端特定功能
?>
```

**其他模块也可以继承**：

```php
// 其他模块也可以创建自己的 theme 目录
// app/code/Weline/YourModule/view/theme/frontend/partials/header/default.phtml
```

### 实现方式

```php
// 获取 partials 路径（支持模块级和数据库主题继承）
$path = PartialsScanner::getPartialsPath($theme, 'frontend', 'header', 'default');
// 查找顺序：
// 1. 当前模块的 theme 目录
// 2. Weline_Frontend 模块的 theme 目录
// 3. 当前主题的文件
// 4. 父主题的文件
// 5. Theme 模块的基础文件
```

## 📊 使用示例

### 1. 在布局中使用 Partials

```php
<?php
use Weline\Theme\Block\Partials;

/** @var Partials $partialsBlock */
$partialsBlock = ObjectManager::getInstance(Partials::class);
?>

<!DOCTYPE html>
<html>
<head>
    <title><?= $title ?></title>
</head>
<body>
    <!-- 动态加载配置的 header -->
    <?= $this->fetch($partialsBlock->getPartialsPath('frontend', 'header')) ?>
    
    <main>
        <?= $content ?>
    </main>
    
    <!-- 动态加载配置的 footer -->
    <?= $this->fetch($partialsBlock->getPartialsPath('frontend', 'footer')) ?>
</body>
</html>
```

### 2. 使用模板标签

```html
<!-- 加载主题组件 -->
<w:theme:template>Weline_Theme::backend/components/loading.phtml</w:theme:template>

<!-- 加载模板文件 -->
<w:template>Weline_Admin::common/head.phtml</w:template>
```

### 3. 后台配置 Partials

访问 `/theme/backend/config/partials?theme_id=1`，选择：
- 前端 Header：`default` / `minimal` / `centered`
- 前端 Footer：`default` / `minimal`
- 后端 Header：`default`
- 后端 Footer：`default`
- 后端 Sidebar：`default`
