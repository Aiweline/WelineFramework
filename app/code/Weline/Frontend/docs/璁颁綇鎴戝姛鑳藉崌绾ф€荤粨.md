## 🔄 工作流程对比

### 之前（简单checkbox）
```
登录 → 勾选"记住我" → 设置30天Cookie → 完成
```
**缺点**：
- ❌ 时长固定，不够灵活
- ❌ 没有数据库记录
- ❌ 无法追溯和管理
- ❌ 无法强制失效

### 现在（灵活选择）
```
登录
  ↓
选择时长（6h/1d/1w/1m）
  ↓
生成64位随机Token
  ↓
保存到数据库（user_id, token, expire_time）
  ↓
设置Cookie（token）
  ↓
完成

下次访问
  ↓
AutoLogin Observer触发
  ↓
验证Cookie中的Token
  ↓
查询数据库
  ↓
检查过期时间
  ↓
自动登录 or 清除无效Token
```

**优点**：
- ✅ 时长灵活可选
- ✅ 数据库存储可追溯
- ✅ 支持强制登出
- ✅ 自动清理过期Token
- ✅ 记录使用时间

## 📊 数据库设计

### frontend_user_token 表

| 字段 | 类型 | 说明 | 索引 |
|------|------|------|------|
| token_id | INT | 主键 | PRIMARY |
| user_id | INT | 用户ID | INDEX |
| token | VARCHAR(64) | Token字符串 | UNIQUE INDEX |
| type | VARCHAR(32) | Token类型 | - |
| token_expire_time | INT | 过期时间戳 | INDEX |
| created_at | TIMESTAMP | 创建时间 | - |
| last_used_at | TIMESTAMP | 最后使用时间 | - |

**索引策略**：
- `idx_user_id`: 快速查找用户的Token
- `idx_token`: 快速验证Token
- `idx_expire`: 快速清理过期Token

## 🔍 代码示例

### 检查用户的Token
```php
use Weline\Frontend\Model\FrontendUserToken;

$userToken = ObjectManager::getInstance(FrontendUserToken::class);
$tokens = $userToken->builder()
    ->where('user_id', 123)
    ->where('type', 'remember_me')
    ->select();

foreach ($tokens as $token) {
    echo "Token: {$token['token']}\n";
    echo "过期: " . date('Y-m-d H:i:s', $token['token_expire_time']) . "\n";
    echo "最后使用: {$token['last_used_at']}\n";
}
```

### 添加自定义时长
```html
<!-- 在 login.phtml 中添加 -->

```

### 强制用户重新登录
```php
// 删除所有Token
$userToken = ObjectManager::getInstance(FrontendUserToken::class);
$userToken->builder()
    ->where('user_id', $userId)
    ->where('type', 'remember_me')
    ->delete();
```
