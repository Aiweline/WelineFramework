# 官方凭据与一键授权指南

面向后台「融媒体管理 → 账户管理」中的**官方凭据**区。后台 UI 会按所选平台动态展示对应步骤；本文档为完整参考。

## 通用准备

1. **范围**：顶部先选好「范围类型 → 站点范围 → 二级范围」。
2. **平台应用（App/Client）**：支持一键 OAuth 的平台必须先在统一配置中心配置应用凭据。
3. **账户级 Token**：一键授权成功后自动写入；也可人工粘贴。
4. **OAuth 回调**（仅 OAuth 平台）：固定为 `/weline_social/backend/oauth/callback`，需在开发者后台加入白名单（含后台域名与端口）。

### 打开系统配置

账户管理页「打开系统配置」会带 URL 定位参数：

- `module=Weline_Social`
- `guide_key=social/platform/{code}/...`（按平台定位字段）
- `guide_return` 回到融媒体管理页

手动路径：`系统管理 → 系统配置 → 统一配置中心 → Weline_Social → 融媒体平台应用`

### 连接账户（推荐流程）

1. 顶部选择当前配置范围。
2. 「平台库」点「添加到当前范围」，或切到「账户管理」手动选平台。
3. 点「查看官方凭据指南」查看当前平台步骤。
4. 需要 App/Client 的平台：点「打开系统配置」填写后返回。
5. OAuth 平台点「一键授权」；其余平台按字段人工填写。
6. 点「保存账户」→ 账户列表「检测」→ 通过后勾选「启用发布能力」。
7. 在「范围社媒」确认账户已绑定当前范围默认列表。

## 各平台凭据指南

### Facebook

| 项目 | 说明 |
|------|------|
| 授权方式 | 一键授权（OAuth） |
| 系统配置 | App ID、App Secret、Graph API 版本 |
| 账户字段 | Page Access Token、Page ID |
| 发布 | 文本、链接、图片 URL |
| 官方文档 | [Graph API](https://developers.facebook.com/docs/graph-api/) · [Pages API](https://developers.facebook.com/docs/pages-api/) |

**开发者后台**：Meta 开发者后台创建应用，启用 Facebook Login，申请 `pages_manage_posts`、`pages_read_engagement` 等 Page 权限；OAuth 回调设为 `/weline_social/backend/oauth/callback`。

**本页步骤**：系统配置填 App ID/Secret → 一键授权（Page 管理员）→ 自动写入 Token → 保存 → 检测 → 启用发布。

## 一键授权 vs 人工粘贴

- **一键授权**：OAuth2 平台；依赖「融媒体平台应用」App/Client；Token 加密存 Social 账户。
- **人工粘贴**：API Key / Webhook / Bot Token 等；编辑时凭据留空保留旧值。
- **仅展示**：授权方式选「仅展示」时可不配发布凭据。

## 常见报错

| 提示 | 处理 |
|------|------|
| Operation is not exposed to frontend worker API | 执行 `php bin/w framework:compile` 并 `server:reload`。 |
| 该平台暂不可一键授权 | 检查统一配置中心是否已填 App/Client。 |
| 该平台不支持一键授权 | 正常；按指南人工填写。 |
| 授权状态无效或已过期 | 重新点「一键授权」。 |
| 请先选择范围类型和站点范围 | 顶部范围未选全。 |

## 发布前检查清单

- [ ] 统一配置中心已配置对应平台应用（OAuth 平台）
- [ ] 账户已保存且「检测」通过
- [ ] `publish_enabled=1`
- [ ] 已加入当前范围默认社媒
- [ ] 草稿存在，一键发布勾选了可发布账户
