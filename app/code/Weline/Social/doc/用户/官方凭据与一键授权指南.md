# 官方凭据与一键授权指南

面向后台「融媒体管理 → 账户管理」中的**官方凭据**区。

## 你需要准备什么

1. **范围**：顶部先选好「范围类型 → 站点范围 → 二级范围」。
2. **平台应用（App/Client）**：一键授权平台必须先在统一配置中心配置应用凭据。
3. **账户级 Token**：一键授权成功后自动写入；也可人工粘贴。

## 配置应用 App/Client（必做）

路径：

`系统管理 → 系统配置 → 统一配置中心 → Weline_Social → 融媒体平台应用`

按平台填写（示例）：

| 平台 | 关键字段 |
|------|----------|
| Facebook / Instagram | App ID、App Secret（Instagram 可留空复用 Facebook） |
| YouTube | OAuth Client ID / Secret |
| TikTok | Client Key / Secret |
| X | Client ID / Secret |
| LinkedIn / Pinterest / Snapchat | 各自 Client/App ID 与 Secret |
| WhatsApp / 微信 | 默认应用参数；账户仍需人工填 Token |

保存后回到融媒体管理页。OAuth 回调地址固定为：

`/weline_social/backend/oauth/callback`

请在各平台开发者后台把该回调（含你的后台域名与端口）加入白名单。

## 连接账户（推荐流程）

1. 顶部选择当前配置范围（例如：站点 → 某站点 → 站点默认）。
2. 「平台库」点「添加到当前范围」，或切到「账户管理」手动选平台。
3. 账户名称会随平台与范围自动联动（新建时）。
4. 在「官方凭据」：
   - **支持一键授权的平台**：点「一键授权」，完成平台登录后自动回写 Token。
   - **不支持一键授权**（如 WhatsApp / 微信等）：按字段人工粘贴 Access Token 等。
5. 点「保存账户」。
6. 在账户列表点「检测」；通过后才可勾选「启用发布能力」并进入一键发布。
7. 「范围社媒」里确认该账户已绑定当前范围默认列表（从平台库添加时会自动绑定）。

## 一键授权 vs 人工粘贴

- **一键授权**：适合 OAuth2 平台。依赖「融媒体平台应用」里的 App/Client；账户 Token 加密保存在 Social 账户，不写进 SystemConfig。
- **人工粘贴**：适合 API Key / 人工 Token 平台，或本地联调。编辑时凭据字段留空表示保留已加密旧值。
- **仅展示**：授权方式选「仅展示」时，可不配发布凭据，只用于前台社媒部件。

## 常见报错

| 提示 | 处理 |
|------|------|
| Operation is not exposed to frontend worker API | 模块 Query 未对后台 worker 开放对应操作；升级/重载后应消失。仍出现则执行 `php bin/w framework:compile` 并 `server:reload`。 |
| 该平台暂不可一键授权 | 检查统一配置中心是否已填该平台 App/Client。 |
| 该平台不支持一键授权 | 正常；请改用人工填写官方凭据。 |
| 授权状态无效或已过期 | 重新点「一键授权」。 |
| 请先选择范围类型和站点范围 | 顶部范围未选全；选完后再添加平台。 |

## 发布前检查清单

- [ ] 统一配置中心已配置对应平台应用
- [ ] 账户已保存且「检测」通过
- [ ] `publish_enabled=1`
- [ ] 已加入当前范围默认社媒（范围社媒页）
- [ ] 草稿存在，一键发布勾选了可发布账户
