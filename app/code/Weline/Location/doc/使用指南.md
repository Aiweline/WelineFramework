## 🚀 快速开始

### 1. 模块自动加载

Geo定位模块已通过hook机制自动注入到前端布局中，无需手动引入。模块会在页面加载时自动声明，支持按需加载。

### 2. 后端通道配置

模块默认启用了3个免费高可用的定位通道：
- **ip-api.com** - 完全免费，无需API Key
- **geojs.io** - 完全免费，无需API Key
- **ipwhois.app** - 免费，无需API Key

系统会自动按优先级顺序尝试各个通道，如果某个通道失败，会自动fallback到下一个通道。

配置位置：`app/code/Weline/Geo/etc/env.php`

### 3. 检查模块是否可用

```javascript
// 检查WelineLocation是否已加载
if (typeof WelineLocation !== 'undefined') {
    console.log('Geo模块已加载');
} else {
    // 手动加载模块
    await Weline.load('geo');
}
```

### 4. 最简单的使用示例

```javascript
// 获取当前位置（自动选择最佳定位方式）
try {
    const position = await WelineLocation.getLocation();
    console.log('当前位置:', position.latitude, position.longitude);
    console.log('定位来源:', position.source); // 显示使用的通道
} catch (error) {
    console.error('定位失败:', error.message);
}
```

## 📚 基础使用

### 1. 浏览器定位

浏览器定位提供最精确的位置信息，但需要用户授权。

#### 基本用法

```javascript
// 获取当前位置
const position = await WelineLocation.getCurrentPosition();
console.log('纬度:', position.latitude);
console.log('经度:', position.longitude);
console.log('精度:', position.accuracy, '米');
```

#### 使用选项

```javascript
// 高精度定位
const position = await WelineLocation.getCurrentPosition({
    enableHighAccuracy: true,  // 启用高精度
    timeout: 10000,            // 10秒超时
    maximumAge: 60000         // 缓存60秒
});
```

#### 错误处理

```javascript
try {
    const position = await WelineLocation.getCurrentPosition();
} catch (error) {
    switch (error.message) {
        case '用户拒绝了定位权限':
            alert('请允许浏览器定位权限以使用此功能');
            break;
        case '位置信息不可用':
            // 降级到IP定位
            const ipPosition = await WelineLocation.getLocationByIP();
            break;
        case '定位请求超时':
            // 增加超时时间重试
            const position = await WelineLocation.getCurrentPosition({
                timeout: 30000
            });
            break;
        default:
            console.error('定位失败:', error.message);
    }
}
```

### 2. IP定位

IP定位不需要用户授权，但精度较低，通常只能定位到城市级别。

#### 基本用法

```javascript
// 通过IP获取位置
const position = await WelineLocation.getLocationByIP();
console.log('国家:', position.country);
console.log('城市:', position.city);
console.log('纬度:', position.latitude);
console.log('经度:', position.longitude);
```

#### IP定位返回的数据结构

```javascript
{
    latitude: 39.9042,        // 纬度
    longitude: 116.4074,      // 经度
    accuracy: null,           // 精度（IP定位通常为null）
    country: '中国',          // 国家
    countryCode: 'CN',        // 国家代码
    region: '北京',           // 地区/省份
    city: '北京市',           // 城市
    timezone: 'Asia/Shanghai', // 时区
    timestamp: 1234567890,     // 时间戳
    source: 'ip'              // 定位来源
}
```

### 3. 智能定位（推荐）

智能定位会自动选择最佳定位方式：优先使用浏览器定位，失败则自动降级到IP定位。

```javascript
// 智能定位（推荐使用）
try {
    const position = await WelineLocation.getLocation({
        enableHighAccuracy: true,
        timeout: 5000
    });
    
    if (position.source === 'browser') {
        console.log('使用浏览器定位，精度:', position.accuracy, '米');
    } else {
        console.log('使用IP定位，城市:', position.city);
    }
} catch (error) {
    console.error('所有定位方式都失败:', error.message);
}
```

模块内置了位置缓存机制，默认缓存时长为5分钟。

#### 清除缓存

```javascript
// 清除所有位置缓存，强制获取最新位置

```

#### 设置缓存时长

```javascript
// 设置缓存时长为10分钟
WelineLocation.setCacheDuration(10 * 60 * 1000);

// 禁用缓存（每次获取最新位置）
WelineLocation.setCacheDuration(0);
```

## ⚙️ 多通道配置

### 通道选择指南

Geo定位模块支持多个免费和付费的定位服务通道。以下是各通道的详细说明，帮助您选择合适的通道：

#### 1. ip-api.com（推荐，默认启用）

- **官网**: [http://ip-api.com](http://ip-api.com)
- **API文档**: [http://ip-api.com/docs/api:json](http://ip-api.com/docs/api:json)
- **特点**:
  - ✅ 完全免费，无需注册和API Key
  - ✅ 稳定性高，响应速度快
  - ✅ 提供详细的地理位置信息（国家、城市、经纬度、时区等）
  - ⚠️ 速率限制：45请求/分钟（免费版）
  - ⚠️ 仅支持HTTP（不支持HTTPS）
- **适用场景**: 中小型应用，请求量不大的场景
- **优先级**: 1（最高）

#### 2. geojs.io（推荐，默认启用）

- **官网**: [https://www.geojs.io](https://www.geojs.io)
- **API文档**: [https://www.geojs.io/docs/v1/endpoints/geo/](https://www.geojs.io/docs/v1/endpoints/geo/)
- **特点**:
  - ✅ 完全免费，无需注册和API Key
  - ✅ 无配额限制
  - ✅ 支持HTTPS
  - ✅ 响应速度快
  - ⚠️ 数据精度相对较低
- **适用场景**: 需要高可用性的应用，请求量较大的场景
- **优先级**: 2

#### 3. ipwhois.app（默认启用）

- **官网**: [https://ipwhois.app](https://ipwhois.app)
- **API文档**: [https://ipwhois.io/documentation](https://ipwhois.io/documentation)
- **特点**:
  - ✅ 免费，无需API Key
  - ✅ 提供IP地址的详细信息
  - ⚠️ 有配额限制（具体限制需查看官网）
  - ⚠️ 响应速度中等
- **适用场景**: 作为备用通道，提供额外的fallback选项
- **优先级**: 3

#### 4. ipinfo.io（可选，需要API Key）

- **官网**: [https://ipinfo.io](https://ipinfo.io)
- **API文档**: [https://ipinfo.io/developers](https://ipinfo.io/developers)
- **注册**: [https://ipinfo.io/signup](https://ipinfo.io/signup)
- **特点**:
  - ✅ 免费版：50,000请求/月
  - ✅ 稳定性高，数据准确
  - ✅ 支持HTTPS
  - ✅ 提供详细的IP信息（ISP、组织、ASN等）
  - ⚠️ 需要注册账号获取API Key
  - ⚠️ 免费版有配额限制
- **适用场景**: 需要更高配额或更准确数据的应用
- **优先级**: 4

#### 5. ipapi.co（可选，需要API Key）

- **官网**: [https://ipapi.co](https://ipapi.co)
- **API文档**: [https://ipapi.co/documentation/](https://ipapi.co/documentation/)
- **注册**: [https://ipapi.co/signup/](https://ipapi.co/signup/)
- **特点**:
  - ✅ 免费版：1,000请求/天
  - ✅ 稳定性高
  - ✅ 支持HTTPS
  - ✅ 提供详细的地理位置和网络信息
  - ⚠️ 需要注册账号获取API Key
  - ⚠️ 免费版配额较小
- **适用场景**: 低请求量的应用，或作为备用通道
- **优先级**: 5

### 选择建议

- **小型应用（< 1000请求/天）**: 使用默认的3个免费通道即可，无需配置
- **中型应用（1000-50000请求/月）**: 建议注册ipinfo.io获取更高配额
- **大型应用（> 50000请求/月）**: 考虑使用付费版本或自建定位服务

### 配置通道

编辑 `app/code/Weline/Geo/etc/env.php` 文件：

```php
return [
    'location' => [
        'providers' => [
            'ip-api.com' => [
                'enabled' => true,   // 启用/禁用
                'priority' => 1,     // 优先级（数字越小优先级越高）
                'timeout' => 5       // 超时时间（秒）
            ],
            'geojs.io' => [
                'enabled' => true,
                'priority' => 2,
                'timeout' => 5
            ],
            'ipwhois.app' => [
                'enabled' => true,
                'priority' => 3,
                'timeout' => 5
            ],
            // 付费通道（需要API Key）
            'ipinfo.io' => [
                'enabled' => false,  // 默认禁用
                'priority' => 4,
                'api_key' => '',     // 配置API Key后启用
                'timeout' => 5
            ],
            'ipapi.co' => [
                'enabled' => false,  // 默认禁用
                'priority' => 5,
                'api_key' => '',     // 配置API Key后启用
                'timeout' => 5
            ]
        ],
        'timeout' => 5,  // 全局默认超时时间（秒）
        'retry' => 1     // 每个通道重试次数
    ]
];
```

### Fallback机制

系统会自动按优先级顺序尝试各个通道：

1. 首先尝试优先级1的通道（ip-api.com）
2. 如果失败，自动尝试优先级2的通道（geojs.io）
3. 如果还失败，尝试优先级3的通道（ipwhois.app）
4. 如果所有通道都失败，返回错误信息

### 查看使用的通道

定位结果中会包含 `source` 字段，显示使用的通道：

```javascript
const position = await WelineLocation.getLocationByIP();
console.log('使用的通道:', position.source); // 'ip-api.com', 'geojs.io', 等
```

## ❓ 常见问题

### Q1: 为什么定位失败？

**可能的原因：**

1. **用户拒绝了定位权限**
   - 解决：提示用户允许定位权限
   ```javascript
   try {
       const position = await WelineLocation.getCurrentPosition();
   } catch (error) {
       if (error.message === '用户拒绝了定位权限') {
           alert('请允许浏览器定位权限以使用此功能');
       }
   }
   ```

2. **浏览器不支持Geolocation API**
   - 解决：自动降级到IP定位
   ```javascript
   const position = await WelineLocation.getLocation(); // 自动选择最佳方式
   ```

3. **定位请求超时**
   - 解决：增加超时时间
   ```javascript
   const position = await WelineLocation.getCurrentPosition({
       timeout: 30000  // 30秒
   });
   ```

### Q2: 如何提高定位精度？

使用高精度模式：

```javascript
const position = await WelineLocation.getCurrentPosition({
    enableHighAccuracy: true,  // 启用高精度
    timeout: 15000             // 增加超时时间
});
```

### Q3: 定位信息多久更新一次？

模块内置了5分钟的缓存机制。如果需要获取最新位置：

```javascript
// 清除缓存后获取
WelineLocation.clearCache();
const position = await WelineLocation.getCurrentPosition();
```

### Q4: 如何在HTTP环境下使用定位？

某些浏览器在HTTP环境下可能限制定位功能。建议：

1. 使用HTTPS
2. 降级到IP定位
3. 提示用户切换到HTTPS

```javascript
if (location.protocol === 'http:') {
    console.warn('HTTP环境下定位功能可能受限，建议使用HTTPS');
    // 使用IP定位
    const position = await WelineLocation.getLocationByIP();
}
```

### Q5: IP定位精度如何？

IP定位精度较低，通常只能定位到城市级别，误差可能在几公里到几十公里。适合用于：

- 默认地址选择
- 内容本地化
- 粗略位置判断

不适合用于：

- 精确导航
- 附近商家查找（需要高精度）
- 实时追踪
