# 统一卸载服务使用文档

## 概述

`UninstallService` 是 Weline Framework 提供的统一卸载服务，位于 `Weline\Framework\Uninstall` 命名空间下。该服务用于管理模块、主题、翻译包的卸载、备份和回滚操作。

## 功能特性

- ✅ 统一的卸载接口，支持模块、主题、翻译包
- ✅ 自动备份功能，卸载前自动备份
- ✅ 权限检查，确保备份目录权限正确
- ✅ 回滚功能，支持从备份恢复
- ✅ 事件系统支持，其他模块可以通过事件通知卸载服务
- ✅ 灵活的备份目录配置
- ✅ 统一的卸载返回数据格式，包含详细的卸载步骤

## 目录结构

```
app/code/Weline/Framework/Uninstall/
├── UninstallService.php          # 卸载服务主类
└── Observer/
    └── UninstallObserver.php     # 卸载事件观察者
```

## 配置

### 备份目录配置

在 `app/etc/env.php` 中配置备份目录：

```php
return [
    // ... 其他配置
    'uninstall' => [
        // 如果为 null，则使用项目根目录上级的 storage 目录
        // 例如：项目在 E:\WelineFramework\DEV-workspace
        // 备份目录将使用 E:\WelineFramework\storage
        'backup_dir' => null,
        
        // 或者指定自定义路径
        // 'backup_dir' => '/path/to/backup/storage',
    ],
    // ... 其他配置
];
```

### 用户权限配置

确保 `app/etc/env.php` 中配置了正确的运行用户：

```php
return [
    // ... 其他配置
    'user' => 'www', // 网站运行用户
    // ... 其他配置
];
```

**重要提示**：
- 如果配置了 `user` 键，系统会检查当前运行用户是否与配置的用户一致
- 外部备份目录（项目根目录上级的 storage 目录）必须对配置的用户有读写权限
- 如果权限检查失败，系统会提示错误信息

## 使用方法

### 方法一：通过事件通知（推荐）

这是推荐的使用方式，其他模块通过触发事件来通知卸载服务执行卸载操作：

```php
use Weline\Framework\Event\EventsManager;
use Weline\Framework\Manager\ObjectManager;
use Weline\Framework\Uninstall\UninstallService;

// 获取事件管理器
$eventManager = ObjectManager::getInstance(EventsManager::class);

// 准备事件数据
$eventData = [
    'type' => UninstallService::TYPE_MODULE,  // 卸载类型：module/theme/i18n
    'name' => 'Weline_Demo',                   // 模块名/主题名/语言代码
    'auto_backup' => true,                     // 是否自动备份，默认 true
];

// 触发卸载事件
$eventManager->dispatch('Framework_UninstallService::uninstall', $eventData);

// 获取卸载结果
$uninstallResult = $eventData['uninstall_result'] ?? null;

if ($uninstallResult && $uninstallResult['success']) {
    echo "卸载成功！\n";
    echo "备份路径：" . $uninstallResult['backup_path'] . "\n";
    
    // 查看卸载步骤
    foreach ($uninstallResult['steps'] as $step) {
        echo "步骤：" . $step['step'] . " - " . $step['message'] . "\n";
    }
} else {
    echo "卸载失败：" . ($uninstallResult['message'] ?? '未知错误') . "\n";
}
```

### 方法二：直接调用服务

也可以直接调用卸载服务：

```php
use Weline\Framework\Manager\ObjectManager;
use Weline\Framework\Uninstall\UninstallService;

// 获取服务实例
$uninstallService = ObjectManager::getInstance(UninstallService::class);

// 卸载模块（自动备份）
$result = $uninstallService->uninstall(
    UninstallService::TYPE_MODULE,
    'Weline_Demo',
    true  // 自动备份
);

if ($result['success']) {
    echo "卸载成功！\n";
    echo "备份路径：" . $result['backup_path'] . "\n";
} else {
    echo "卸载失败：" . $result['message'] . "\n";
}
```

## 卸载类型

服务支持三种卸载类型：

- `UninstallService::TYPE_MODULE` - 模块
- `UninstallService::TYPE_THEME` - 主题
- `UninstallService::TYPE_I18N` - 翻译包

## 统一返回数据格式

卸载操作返回统一的数据格式：

```php
[
    'success' => true,                    // 是否成功
    'type' => 'module',                   // 卸载类型
    'name' => 'Weline_Demo',              // 名称
    'backup_path' => '/path/to/backup',   // 备份路径
    'message' => '卸载完成',               // 消息
    'steps' => [                          // 卸载步骤
        [
            'step' => 'start',            // 步骤名称
            'message' => '开始卸载...',    // 步骤消息
            'time' => '2024-01-01 12:00:00', // 时间
            'success' => true,            // 是否成功
        ],
        // ... 更多步骤
    ],
    'start_time' => '2024-01-01 12:00:00', // 开始时间
    'end_time' => '2024-01-01 12:00:01',   // 结束时间
]
```

## 使用示例

### 示例 1：卸载模块

```php
use Weline\Framework\Event\EventsManager;
use Weline\Framework\Manager\ObjectManager;
use Weline\Framework\Uninstall\UninstallService;

$eventManager = ObjectManager::getInstance(EventsManager::class);
$eventData = [
    'type' => UninstallService::TYPE_MODULE,
    'name' => 'Weline_Demo',
    'auto_backup' => true,
];

$eventManager->dispatch('Framework_UninstallService::uninstall', $eventData);

$result = $eventData['uninstall_result'];
if ($result['success']) {
    // 执行实际的卸载逻辑（数据库操作等）
    // ...
}
```

### 示例 2：卸载主题

```php
$eventData = [
    'type' => UninstallService::TYPE_THEME,
    'name' => 'MyTheme',
    'auto_backup' => true,
];

$eventManager->dispatch('Framework_UninstallService::uninstall', $eventData);
```

### 示例 3：卸载翻译包

```php
$eventData = [
    'type' => UninstallService::TYPE_I18N,
    'name' => 'en_US',
    'auto_backup' => true,
];

$eventManager->dispatch('Framework_UninstallService::uninstall', $eventData);
```

### 示例 4：手动备份

```php
use Weline\Framework\Manager\ObjectManager;
use Weline\Framework\Uninstall\UninstallService;

$uninstallService = ObjectManager::getInstance(UninstallService::class);

// 备份模块
$result = $uninstallService->backupModule('Weline_Demo');

// 备份主题
$result = $uninstallService->backupTheme('MyTheme');

// 备份翻译包
$result = $uninstallService->backupI18n('en_US');
```

### 示例 5：回滚操作

```php
// 回滚模块（使用最新备份）
$result = $uninstallService->rollbackModule('Weline_Demo');

// 回滚模块（使用指定备份）
$result = $uninstallService->rollbackModule('Weline_Demo', '/path/to/backup/2024-01-01_12-00-00');

// 回滚主题
$result = $uninstallService->rollbackTheme('MyTheme');

// 回滚翻译包
$result = $uninstallService->rollbackI18n('en_US');
```

### 示例 6：获取备份列表

```php
// 获取模块备份列表
$backups = $uninstallService->getBackupList(UninstallService::TYPE_MODULE, 'Weline_Demo');

foreach ($backups as $backup) {
    echo "备份时间：" . $backup['time'] . "\n";
    echo "备份路径：" . $backup['path'] . "\n";
}
```

## 事件系统

### 卸载通知事件

**事件名称**：`Framework_UninstallService::uninstall`

**事件数据**：
```php
[
    'type' => 'module',        // 卸载类型：module/theme/i18n
    'name' => 'Weline_Demo',   // 模块名/主题名/语言代码
    'auto_backup' => true,     // 是否自动备份
]
```

**返回数据**：
卸载结果会返回到事件数据的 `uninstall_result` 字段中。

### 其他相关事件

卸载服务还会触发以下事件，其他模块可以监听这些事件来扩展功能：

#### 模块相关事件
- `Framework_UninstallService::before_uninstall_module` - 模块卸载前
- `Framework_UninstallService::after_uninstall_module` - 模块卸载后
- `Framework_UninstallService::before_rollback_module` - 模块回滚前
- `Framework_UninstallService::after_rollback_module` - 模块回滚后

#### 主题相关事件
- `Framework_UninstallService::before_uninstall_theme` - 主题卸载前
- `Framework_UninstallService::after_uninstall_theme` - 主题卸载后
- `Framework_UninstallService::before_rollback_theme` - 主题回滚前
- `Framework_UninstallService::after_rollback_theme` - 主题回滚后

#### 翻译包相关事件
- `Framework_UninstallService::before_uninstall_i18n` - 翻译包卸载前
- `Framework_UninstallService::after_uninstall_i18n` - 翻译包卸载后
- `Framework_UninstallService::before_rollback_i18n` - 翻译包回滚前
- `Framework_UninstallService::after_rollback_i18n` - 翻译包回滚后

### 监听事件示例

在模块的 `etc/event.xml` 中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <!-- 模块卸载前 -->
    <event name="Framework_UninstallService::before_uninstall_module">
        <observer name="Your_Module::before_uninstall_module"
                  instance="Your\Module\Observer\BeforeUninstallModule"
                  disabled="false"
                  shared="true"
                  sort="10"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class BeforeUninstallModule implements ObserverInterface
{
    public function execute(Event $event)
    {
        $data = $event->getData();
        $moduleName = $data['module_name'] ?? '';
        $backupPath = $data['backup_path'] ?? '';
        
        // 执行自定义逻辑
        // 例如：清理缓存、记录日志等
    }
}
```

## 备份目录结构

备份目录结构如下：

```
storage/
├── module/
│   └── Weline_Demo/
│       └── 2024-01-01_12-00-00/
│           ├── backup_info.json
│           └── [模块文件...]
├── theme/
│   └── MyTheme/
│       └── 2024-01-01_12-00-00/
│           ├── backup_info.json
│           └── [主题文件...]
└── i18n/
    └── en_US/
        └── 2024-01-01_12-00-00/
            ├── backup_info.json
            └── [翻译包文件...]
```

### 备份信息文件格式

每个备份目录都包含一个 `backup_info.json` 文件，格式如下：

```json
{
    "type": "module",
    "name": "Weline_Demo",
    "backup_time": "2024-01-01 12:00:00",
    "backup_path": "/path/to/backup/2024-01-01_12-00-00",
    "module_path": "/path/to/module",
    "module_info": {
        "name": "Weline_Demo",
        "version": "1.0.0",
        ...
    }
}
```

## 已对接的卸载操作

以下位置的卸载操作已经对接了统一卸载服务：

1. **模块卸载**：`Weline\Framework\Module\Console\Module\Remove`
   - 在卸载前通过事件通知卸载服务进行备份

2. **主题卸载**：`Weline\Theme\Console\Theme\Create::uninstallTheme()`
   - 在卸载前通过事件通知卸载服务进行备份

## 注意事项

1. **权限要求**：
   - 备份目录必须对配置的运行用户有读写权限
   - 如果使用外部目录（项目根目录上级），确保该目录存在且可访问

2. **备份大小**：
   - 备份会复制完整的文件目录，确保有足够的磁盘空间
   - 建议定期清理旧的备份文件

3. **卸载操作**：
   - 卸载服务主要负责备份和文件操作
   - 实际的卸载逻辑（如数据库操作）需要由调用方处理
   - 建议在卸载前先通过事件通知卸载服务进行备份

4. **回滚操作**：
   - 回滚会覆盖目标目录的文件
   - 回滚前建议先备份当前状态

5. **事件顺序**：
   - 卸载前事件在备份完成后、实际卸载前触发
   - 卸载后事件在实际卸载完成后触发
   - 回滚前事件在恢复文件前触发
   - 回滚后事件在恢复文件后触发

6. **卸载步骤**：
   - 所有卸载操作都会记录详细的步骤信息
   - 步骤信息包含在返回数据的 `steps` 字段中
   - 每个步骤包含：步骤名称、消息、时间、是否成功

## 故障排除

### 权限检查失败

**问题**：提示权限检查失败

**解决方案**：
1. 检查 `app/etc/env.php` 中的 `user` 配置是否正确
2. 确保备份目录对配置的用户有读写权限
3. 如果使用外部目录，确保该目录存在且可访问

### 备份目录创建失败

**问题**：无法创建备份目录

**解决方案**：
1. 检查父目录是否存在
2. 检查父目录的权限
3. 尝试手动创建目录并设置正确的权限

### 备份文件过大

**问题**：备份时间过长或磁盘空间不足

**解决方案**：
1. 检查磁盘空间是否充足
2. 考虑排除不必要的文件（需要修改服务代码）
3. 定期清理旧的备份文件

### 事件未触发

**问题**：触发事件后没有执行卸载

**解决方案**：
1. 检查事件名称是否正确：`Framework_UninstallService::uninstall`
2. 检查事件数据是否包含 `type` 和 `name` 字段
3. 检查事件观察者是否已注册（运行 `php bin/w module:upgrade`）
4. 检查事件配置文件中是否有观察者注册

## 最佳实践

1. **使用事件通知**：推荐使用事件通知方式，而不是直接调用服务
2. **检查返回结果**：始终检查卸载结果，确保操作成功
3. **处理错误情况**：妥善处理备份失败的情况
4. **记录日志**：在卸载前后记录日志，便于追踪问题
5. **定期清理**：定期清理旧的备份文件，释放磁盘空间
