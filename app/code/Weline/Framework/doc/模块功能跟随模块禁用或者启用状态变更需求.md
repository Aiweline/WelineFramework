# 模块功能跟随模块禁用或者启用状态变更需求

## 需求概述

确保所有模块相关的功能（extends、事件、观察者、Hook、插件等）在收集和运行时都遵循模块的激活状态。只有激活的模块才能：
1. 在收集阶段被收集到注册表中
2. 在运行时被执行

如果模块被禁用，其相关的功能项应该：
1. 在收集阶段被标记为禁用状态（如果已收集）
2. 在运行时被跳过执行

## 注意事项

1. **向后兼容性**: 确保旧版本的注册表文件可以正常读取，如果缺少status字段，默认视为激活状态

2. **性能考虑**: 
   - 模块状态检查应该使用缓存，避免重复查询
   - 在收集阶段就过滤掉禁用的模块，减少运行时检查

3. **错误处理**: 
   - 如果模块状态检查失败，应该记录日志但不中断流程
   - 在开发模式下可以抛出异常，生产模式下应该容错

4. **模块依赖**: 
   - 如果模块A依赖模块B，禁用模块B时，模块A的相关功能也应该被禁用
   - 这需要在模块依赖检查中处理

## 总结

本需求确保所有模块相关的功能（Hook、事件、观察者、插件、Extends）在收集和运行时都遵循模块的激活状态。主要修改包括：

4. **统一服务**: 创建 `RegistryUpdateService` 服务，统一管理注册表更新
