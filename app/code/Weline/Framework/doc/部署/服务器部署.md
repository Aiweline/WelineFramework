# 服务器部署说明

## 功能概述

Weline框架提供内置的PHP Web服务器，用于开发环境快速启动和调试。支持前台运行和后台运行两种模式。

## 运行模式

### 后台运行模式（默认）
- 默认模式，服务器在后台运行
- 不显示实时日志，不阻塞终端
- **重要特性**：程序会进入后台运行状态，保持进程监控
- 自动检测服务器进程状态，如果进程停止会自动退出
- 需要使用 `php bin/w server:stop` 停止
- 适合日常开发和长期运行

### 前台运行模式
- 使用 `-f` 或 `--foreground` 参数启动
- 服务器在前台运行，直接执行PHP内置服务器命令
- 显示实时日志输出和访问信息
- **重要特性**：关闭终端时进程会自动终止，不会产生残留进程
- 按 `Ctrl+C` 停止服务器
- 适合调试和查看实时日志

## 使用方法

### 启动服务器

```bash
# 默认后台运行模式
php bin/w server:start

# 前台运行模式
php bin/w server:start -f
# 或
php bin/w server:start --foreground
```

### 查看服务器状态

```bash
php bin/w server:status
```

### 停止服务器

```bash
php bin/w server:stop
```

### 清理残留进程

```bash
php bin/w server:cleanup
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 无参数 | 后台运行模式 | 后台运行 |
| `-f` 或 `--foreground` | 前台运行模式 | - |
| `-r` 或 `--force` | 强制重启 | - |
| `-h` 或 `--host` | 指定主机地址 | 127.0.0.1 |
| `-p` 或 `--port` | 指定端口 | 9981 |

## 注意事项

### 后台运行模式（默认）

**重要变更**：现在默认后台运行：
- 服务器在后台运行，不阻塞终端
- **重要特性**：程序会进入后台运行状态，保持进程监控
- 自动检测服务器进程状态，如果进程停止会自动退出
- 需要使用 `php bin/w server:stop` 停止服务器
- 适合日常开发和长期运行

### 前台运行模式

- 使用 `-f` 或 `--foreground` 参数启动
- 服务器直接在前台运行，不创建分离的子进程
- 关闭终端时进程会自动终止，不会产生残留
- 按 `Ctrl+C` 可以正常停止服务器
- 适合调试和查看实时日志

### 端口冲突

如果端口 9981 被占用，服务器将无法启动。可以使用以下命令检查端口占用：

```bash
# Windows
netstat -ano | findstr :9981

# Linux/Mac
lsof -i :9981
```

### 开发环境专用

此服务器仅适用于开发环境，请勿用于生产环境。生产环境建议使用 Nginx + PHP-FPM 或 Apache + mod_php。

## 故障排除

### 常见问题

1. **"服务器已在运行中！"**
   - 检查是否有残留进程：`php bin/w server:status`
   - 清理残留进程：`php bin/w server:cleanup`
   - 或直接停止：`php bin/w server:stop`

2. **"端口被占用"**
   - 检查端口占用情况
   - 停止占用端口的进程
   - 或修改服务器端口配置

3. **"进程ID显示不正确"**
   - 运行清理命令：`php bin/w server:cleanup`
   - 重新启动服务器

4. **"配置信息不完整"**
   - 自动修复：系统会自动检测并更新配置
   - 手动清理：`php bin/w server:cleanup`

### 推荐使用流程

1. **开发时**：使用实时运行模式 `php bin/w server:start`
2. **长期运行**：使用后台模式 `php bin/w server:start -b`
3. **遇到问题时**：先运行 `php bin/w server:cleanup` 清理
4. **切换项目时**：记得停止当前服务器 `php bin/w server:stop`

### 日志文件

服务器日志位于：
- 启动日志：`var/log/server-start.log`
- 错误日志：`var/log/error.log`
- 调试日志：`var/log/debug.log`
