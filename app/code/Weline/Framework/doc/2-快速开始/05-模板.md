# 模板

## 摘要

本文档介绍 WelineFramework 的模板系统，包括为什么建议使用原生 PHP 模板、模板的优势和使用方法。

## 什么是模板

模板（Template）是 MVC 架构中的视图层组件，负责渲染 HTML 页面。WelineFramework 采用原生 PHP 模板引擎，使用 `.phtml` 或 `.html` 文件作为模板文件。

## 为什么建议使用原生 PHP 模板

WelineFramework 强烈建议使用原生 PHP 模板，原因如下：

### 1. 性能优势

原生 PHP 模板直接编译为 PHP 代码，无需额外的模板引擎解析，性能最优。原生才是性能天花板，没有中间层转换，执行效率最高。

### 2. SEO 友好

原生 PHP 模板生成的 HTML 代码清晰，搜索引擎可以更好地理解和索引页面内容，有利于 SEO 优化。

### 3. CDN 友好

原生模板生成的静态资源路径清晰，便于 CDN 缓存和加速。模板中的静态资源可以直接被 CDN 识别和缓存。

### 4. 开发效率

- **无需学习新语法**：直接使用 PHP 语法，降低学习成本
- **IDE 支持完善**：PHP IDE 对原生 PHP 代码支持最好，代码提示和调试更方便
- **调试简单**：可以直接在模板中调试 PHP 代码

### 5. 兼容性好

原生 PHP 模板兼容所有 PHP 特性，可以使用最新的 PHP 语法和特性，不受模板引擎限制。

## 目录文件结构

模板文件应放在设计目录下：

```
app/design/
├── frontend/                    # 前端模板目录
│   └── default/                 # 默认主题
│       └── Weline_YourModule/   # 模块模板目录
│           └── templates/       # 模板文件目录
│               └── index/
│                   └── index.phtml
└── backend/                     # 后端模板目录
    └── default/
        └── Weline_YourModule/
            └── templates/
                └── index/
                    └── index.phtml
```

### 目录说明

- **app/design/**: 设计目录，存放所有模板文件
- **frontend/backend**: 前端和后端模板目录
- **default**: 默认主题目录
- **Weline_YourModule**: 模块模板目录，对应模块名
- **templates/**: 模板文件目录
- **index/index.phtml**: 控制器对应的模板文件

## 模板文件命名规则

模板文件路径对应控制器路径：

- 控制器：`Controller/Frontend/Index.php`，方法：`index()`
- 模板文件：`app/design/frontend/default/Weline_YourModule/templates/frontend/index/index.phtml`

## 模板语法

### 1. 输出变量

```php
<?= htmlspecialchars($title ?? '') ?>
<?= $this->getData('title') ?>
```

### 2. 条件判断

```php
<?php if ($condition): ?>
    <p>条件为真</p>
<?php else: ?>
    <p>条件为假</p>
<?php endif; ?>
```

### 3. 循环

```php
<?php foreach ($items as $item): ?>
    <div><?= htmlspecialchars($item['name'] ?? '') ?></div>
<?php endforeach; ?>
```

### 4. 包含其他模板

```php
<?= $this->getChildHtml('header') ?>
```

### 5. 使用框架标签

```php
<?= $this->getTag('w:block', ['name' => 'header']) ?>
```

## 在控制器中传递数据

```php
public function index()
{
    $this->assign('title', '页面标题');
    $this->assign('items', $items);
    return $this->fetch();
}
```

## 模板示例

### 基础模板

```php
<!DOCTYPE html>
<html>
<head>
    <title><?= htmlspecialchars($title ?? '') ?></title>
</head>
<body>
    <h1><?= htmlspecialchars($title ?? '') ?></h1>
    
    <?php if (!empty($items)): ?>
        <ul>
            <?php foreach ($items as $item): ?>
                <li><?= htmlspecialchars($item['name'] ?? '') ?></li>
            <?php endforeach; ?>
        </ul>
    <?php endif; ?>
</body>
</html>
```

## 后续处理

### 1. 创建模板文件

在对应的设计目录下创建模板文件，确保路径正确。

### 2. 配置主题

如果需要使用自定义主题，在系统配置中设置主题名称。

## 验证

### 验证模板是否正常渲染

1. 访问控制器 URL，检查页面是否正常显示
2. 检查浏览器源代码，确认 HTML 结构正确
3. 检查变量是否正确输出

### 常见问题

1. **模板未找到**：检查模板文件路径是否正确，确保文件存在
2. **变量未输出**：检查控制器中是否正确传递了变量
3. **语法错误**：检查 PHP 语法是否正确，注意 PHP 8.2+ 对 null 参数的限制
