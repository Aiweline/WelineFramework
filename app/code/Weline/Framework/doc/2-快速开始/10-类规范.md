# 类规范

## 摘要

本文档介绍 WelineFramework 框架中类的规范，包括 `__init()` 和 `__construct()` 方法的使用规范。

## 什么是类规范

类规范是 WelineFramework 框架中定义类时必须遵循的规范，主要包括初始化方法的使用规范。框架提供了两种初始化方法：`__init()` 和 `__construct()`，它们有不同的使用场景和限制。

## 为什么需要类规范

类规范确保了框架的依赖注入机制正常工作，提供了以下优势：
- **依赖注入**：通过 ObjectManager 自动解析和注入依赖
- **类型安全**：确保类的正确初始化
- **统一管理**：统一类的创建和管理方式

## 初始化方法

### __init() 方法

`__init()` 是框架提供的类初始化函数，等同于 `__construct()` 函数，但**必须使用 ObjectManager::getInstance() 容器才能识别**。

#### 使用场景

- 使用 ObjectManager 创建实例时
- 需要框架自动依赖注入时

#### 示例

```php
<?php

declare(strict_types=1);

namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class YourObserver implements ObserverInterface
{
    private $dependency;

    /**
     * 使用 __init() 方法
     * 必须通过 ObjectManager::getInstance() 创建实例
     */
    public function __init($dependency)
    {
        $this->dependency = $dependency;
    }

    public function execute(Event $event)
    {
        // 处理逻辑
    }
}
```

### __construct() 方法

`__construct()` 是 PHP 标准的构造函数，可以在其中初始化类。**必须使用 ObjectManager::getInstance() 容器才能识别依赖注入**。

#### 使用场景

- 标准的 PHP 构造函数
- 需要依赖注入时

#### 示例

```php
<?php

declare(strict_types=1);

namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\YourModule\Model\YourModel;

class YourObserver implements ObserverInterface
{
    private YourModel $model;
    private $otherDependency;

    /**
     * 使用 __construct() 方法
     * 必须通过 ObjectManager::getInstance() 创建实例才能识别依赖注入
     */
    public function __construct(
        YourModel $model,
        $otherDependency
    ) {
        $this->model = $model;
        $this->otherDependency = $otherDependency;
    }

    public function execute(Event $event)
    {
        // 使用注入的依赖
        $this->model->load(1);
    }
}
```

## 依赖注入

### 使用 ObjectManager 创建实例

框架通过 `ObjectManager::getInstance()` 自动解析和注入依赖：

```php
use Weline\Framework\Manager\ObjectManager;

// 自动注入依赖
$observer = ObjectManager::getInstance(YourObserver::class);
```

### 依赖注入规则

1. **类型提示**：构造函数参数使用类型提示，框架会自动解析
2. **接口注入**：可以注入接口，框架会解析对应的实现类
3. **必需依赖**：所有依赖都会自动注入，无需手动传递

## 完整示例

### 事件观察者示例

```php
<?php

declare(strict_types=1);

namespace Weline\Backend\Observer;

use Weline\Backend\Config\MenuXmlReader;
use Weline\Backend\Model\Menu;
use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class UpgradeMenu implements ObserverInterface
{
    private Menu $menu;
    private MenuXmlReader $menuReader;

    /**
     * 使用 __construct() 进行依赖注入
     * 必须使用 ObjectManager::getInstance() 容器才能识别
     */
    public function __construct(
        Menu $menu,
        MenuXmlReader $menuReader
    ) {
        $this->menu = $menu;
        $this->menuReader = $menuReader;
    }

    /**
     * 执行观察者逻辑
     */
    public function execute(Event $event)
    {
        $modules_xml_menus = $this->menuReader->read();
        // 处理逻辑
    }
}
```

## 重要提示

1. **必须使用 ObjectManager**：无论是 `__init()` 还是 `__construct()`，都必须使用 `ObjectManager::getInstance()` 创建实例，否则依赖注入不会生效。

2. **类型提示**：构造函数参数应使用类型提示，框架会自动解析依赖。

3. **可选参数**：如果参数有默认值，框架会使用默认值，不会强制注入。

## 后续处理

### 1. 创建类时遵循规范

确保所有类都遵循框架的类规范，使用正确的初始化方法。

### 2. 使用 ObjectManager 创建实例

始终使用 `ObjectManager::getInstance()` 创建类的实例，确保依赖注入正常工作。

## 验证

### 验证依赖注入是否正常

1. 创建带有依赖的类
2. 使用 ObjectManager 创建实例
3. 检查依赖是否正确注入

### 常见问题

1. **依赖未注入**：检查是否使用了 ObjectManager 创建实例
2. **类型错误**：检查类型提示是否正确
3. **循环依赖**：避免类之间的循环依赖
