## Model相关错误

### 错误1: Model接口方法签名不兼容

**错误信息**:
```
Fatal error: Declaration of Vendor\Module\Model\Entity::setup() must be 
compatible with Weline\Framework\Database\ModelInterface::setup(
Weline\Framework\Setup\Db\ModelSetup $setup, 
Weline\Framework\Setup\Data\Context $context): void
```

**原因**: 方法参数类型或返回类型不匹配

**错误代码**:
```php
use Weline\Framework\Setup\Setup;  // ❌ 错误的类
use Weline\Framework\Setup\Model\Context;  // ❌ 错误的命名空间

public function setup(Setup $setup, Context $context) // ❌ 缺少返回类型
{
}
```

**正确代码**:
```php
use Weline\Framework\Setup\Db\ModelSetup;  // ✅ 正确
use Weline\Framework\Setup\Data\Context;   // ✅ 正确

public function setup(ModelSetup $setup, Context $context): void  // ✅ 正确
{
    // 表创建在 Setup/Install.php 中处理
}

public function upgrade(ModelSetup $setup, Context $context): void {}
public function install(ModelSetup $setup, Context $context): void {}
```

**检查清单**:
- [ ] 使用 `Weline\Framework\Setup\Db\ModelSetup`
- [ ] 使用 `Weline\Framework\Setup\Data\Context`
- [ ] 添加 `: void` 返回类型
- [ ] 三个方法都必须实现

---

### 错误2: Model缺少抽象方法

**错误信息**:
```
Fatal error: Class Vendor\Module\Model\Entity contains 3 abstract methods 
and must therefore be declared abstract or implement the remaining methods 
(ModelInterface::setup, ModelInterface::upgrade, ModelInterface::install)
```

**原因**: 没有实现ModelInterface的必需方法

**解决方案**:
```php
class Entity extends Model
{
    public function _init(): void
    {
        $this->_table = 'your_table';
        $this->_id_field_name = 'id';
    }
    
    // ✅ 必须添加这三个方法
    public function setup(ModelSetup $setup, Context $context): void {}
    public function upgrade(ModelSetup $setup, Context $context): void {}
    public function install(ModelSetup $setup, Context $context): void {}
}
```

---

### 错误3: 表名未指定

**错误信息**:
```
没有指定table表名！
```

**原因**: 忘记实现 `_init()` 方法

**解决方案**:
```php
class Entity extends Model
{
    // ✅ 必须实现_init()
    public function _init(): void
    {
        $this->_table = 'your_table_name';  // 表名
        $this->_id_field_name = 'id';        // 主键名
    }
}
```

## Controller相关错误

### 错误4: success()方法访问级别冲突

**错误信息**:
```
Fatal error: Access level to Vendor\Module\Controller\Api\Entity::success() 
must be protected (as in class Weline\Framework\Controller\Core) or weaker
```

**原因**: 重写了父类的protected方法为private

**错误代码**:
```php
class Entity extends FrontendRestController
{
    // ❌ 不要自定义这些方法
    private function success(array $data): array
    {
        return ['success' => true, 'data' => $data];
    }
    
    private function error(string $message, int $code): array
    {
        return ['success' => false, 'error' => $message];
    }
}
```

**正确代码**:
```php
class Entity extends FrontendRestController
{
    public function get(): array
    {
        try {
            // 业务逻辑
            
            // ✅ 直接使用父类方法
            return $this->success('请求成功', $data);
        } catch (\Exception $e) {
            // ✅ 直接使用父类方法
            return $this->error($e->getMessage(), 500);
        }
    }
    
    // ✅ 不要重写success()和error()方法
}
```

### 错误5: Controller基类未找到

**错误信息**:
```
Error: Class "Weline\Framework\App\Controller\Rest" not found
```

**原因**: 使用了不存在的类名

**错误代码**:
```php
use Weline\Framework\App\Controller\Rest;  // ❌ 这个类不存在

class Entity extends Rest {}
```

**正确代码**:
```php
// ✅ REST API控制器
use Weline\Framework\App\Controller\FrontendRestController;
class Entity extends FrontendRestController {}

// ✅ 或后台REST API
use Weline\Framework\App\Controller\BackendRestController;
class Entity extends BackendRestController {}
```

## 数据库相关错误

### 错误6: 表未创建

**症状**: Setup/Install.php执行了但数据库表不存在

**可能原因**:
1. `createTable()` 调用方式不正确
2. 表已存在导致跳过
3. SQL语法错误被静默忽略

**调试方法**:
```php
// 在 Setup/Install.php 中添加调试输出
public function setup(Setup $setup, Context $context)
{
    try {
        echo "Creating table: your_table\n";
        
        $setup->getConnection()->createTable(
            'your_table',
            function ($table) {
                $table->addColumn('id', 'int', 10, 'PRIMARY KEY AUTO_INCREMENT')
                      ->addColumn('name', 'varchar', 255, 'NOT NULL');
            },
            '表描述'
        );
        
        echo "Table created successfully\n";
    } catch (\Exception $e) {
        echo "Error creating table: " . $e->getMessage() . "\n";
        throw $e;
    }
}
```

**验证表是否存在**:
```bash
# 查看所有表
php bin/w db:show-tables

# 或编写测试脚本
php -r "
require 'app/bootstrap.php';
\$conn = \Weline\Framework\Manager\ObjectManager::getInstance(
    \Weline\Framework\Database\ConnectionFactory::class
)->getConnection();
\$tables = \$conn->query('SELECT name FROM sqlite_master WHERE type=\"table\"')->fetchAll();
print_r(\$tables);
"
```

### 错误7: 数据库连接方法不存在

**错误信息**:
```
Call to undefined method Connector::fetchOne()
```

**原因**: 方法名不正确或不存在

**错误代码**:
```php
// ❌ fetchOne() 不存在
$result = $connection->fetchOne("SELECT * FROM table");
```

**正确代码**:
```php
// ✅ 使用select()构建查询
$select = $connection->select()
    ->from('table_name')
    ->where('field = ?', $value);
    
$result = $connection->fetch($select);  // 返回数组
$row = $result[0] ?? null;

// ✅ 或使用Model查询
$model = ObjectManager::getInstance(YourModel::class);
$model->load($id);
```

## 路由相关错误

### 错误8: API端点404

**症状**: 访问API返回404

**检查清单**:

1. **确认routes.xml存在**:
```bash
ls app/code/Vendor/ModuleName/etc/routes.xml
```

2. **确认路由格式正确**:
```xml
<?xml version="1.0"?>
<config xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
        xsi:noNamespaceSchemaLocation="urn:weline:framework:router:etc/routes.xsd">
    <router area="frontend_rest_api">
        <route path="/api/v1/entity/:id" method="GET">
            <module>Vendor_ModuleName</module>
            <controller>Api\Entity</controller>
            <action>get</action>
        </route>
    </router>
</config>
```

3. **重新安装模块注册路由**:
```bash
php bin/w setup:upgrade
```

4. **检查路由是否注册**:
```bash
grep -r "api/v1/entity" generated/routers/
```

5. **测试端点**:
```bash
php bin/w http:request GET /api/v1/entity/1
```

---

### 错误9: 路由参数获取失败

**症状**: 路径参数为null

**错误代码**:
```php
// 路由: /api/v1/entity/:id
public function get(): array
{
    $id = $this->request->getParam('entity_id');  // ❌ 参数名不匹配
}
```

**正确代码**:
```php
// 路由: /api/v1/entity/:id
public function get(): array
{
    $id = $this->request->getParam('id');  // ✅ 参数名匹配路由定义
    $id = (int) $id;  // 转换类型
}
```

## 模块安装错误

### 错误10: 模块未显示

**症状**: `php bin/w module:list` 看不到模块

**检查清单**:

1. **检查register.php**:
```php
<?php
// ✅ 正确格式
\Weline\Framework\Register\Register::register(
    __DIR__, 
    \Weline\Framework\Register\Register::MODULE
);
```

2. **检查module.xml**:
```xml
<?xml version="1.0"?>
<config xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
        xsi:noNamespaceSchemaLocation="urn:weline:framework:module:etc/module.xsd">
    <module name="Vendor_ModuleName" setup_version="1.0.0">
        <sequence>
            <module name="Weline_Framework"/>
        </sequence>
    </module>
</config>
```

3. **运行安装**:
```bash
php bin/w setup:upgrade
```

4. **清除缓存**:
```bash
rm -rf var/cache/*
rm -rf generated/code/*
```

### 错误11: 模块命名不规范

**错误**: 模块无法加载

**错误命名**:
```
- vendor_modulename  (❌ 全小写)
- Vendor-ModuleName  (❌ 使用连字符)
- vendor.modulename  (❌ 使用点号)
```

**正确命名**:
```
✅ Vendor_ModuleName  (驼峰命名 + 下划线)
```

## 依赖注入错误

### 错误12: 类不存在

**错误信息**:
```
Class does not exist
```

**原因**: 命名空间或类名错误

**错误代码**:
```php
public function __construct(
    private readonly EntityService $service  // ❌ 缺少命名空间
) {}
```

**正确代码**:
```php
use Vendor\ModuleName\Service\EntityService;  // ✅ 导入类

public function __construct(
    private readonly EntityService $service
) {}

// ✅ 或使用完整命名空间
public function __construct(
    private readonly \Vendor\ModuleName\Service\EntityService $service
) {}
```
