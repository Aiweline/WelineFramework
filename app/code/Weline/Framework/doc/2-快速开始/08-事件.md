# 事件

## 摘要

本文档介绍 WelineFramework 的事件系统，包括事件的说明、作用、如何发放事件、观察事件以及如何收集事件。

## 什么是事件

事件（Event）是 WelineFramework 框架中的观察者模式实现，允许在代码执行过程中触发监听器。当代码运行到特定位置时，会触发事件，将数据传递给事件处理器，如果该事件有观察者监听，则执行相应的处理逻辑。

## 为什么需要事件

事件系统提供了以下优势：
- **解耦代码**：在不修改源码的情况下扩展功能
- **灵活扩展**：通过观察者模式实现功能的动态扩展
- **模块化**：各模块可以独立监听和处理事件
- **可维护性**：降低代码耦合度，提高可维护性

## 事件系统组成

### 1. 事件发放（Dispatch）

在代码中触发事件，传递数据给观察者。

### 2. 事件观察（Observer）

监听特定事件，当事件触发时执行处理逻辑。

### 3. 事件收集

框架自动收集所有模块中注册的事件观察者。

## 目录文件结构

事件观察者应放在模组的 `Observer` 目录下，事件配置在 `etc/event.xml` 文件中：

```
app/code/Weline/YourModule/
├── Observer/
│   └── YourObserver.php
└── etc/
    └── event.xml
```

### 目录说明

- **Observer/**: 观察者目录，存放事件观察者类
- **etc/event.xml**: 事件配置文件，注册事件观察者

## 发放事件

在代码中使用 `EventsManager` 触发事件：

```php
<?php

use Weline\Framework\Event\EventsManager;
use Weline\Framework\Manager\ObjectManager;

// 获取事件管理器
$eventManager = ObjectManager::getInstance(EventsManager::class);

// 触发事件
$eventManager->dispatch('event_name', $data);
```

### 事件数据传递

事件可以传递任意数据：

```php
$eventManager->dispatch('user_created', [
    'user_id' => 123,
    'username' => 'test',
    'email' => 'test@example.com'
]);
```

## 观察事件

### 1. 创建观察者类

观察者类必须实现 `ObserverInterface` 接口：

```php
<?php

declare(strict_types=1);

namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class YourObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     */
    public function execute(Event $event)
    {
        $data = $event->getData();
        // 处理逻辑
    }
}
```

### 2. 注册观察者

在 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework:Event/etc/xsd/event.xsd">
    <event name="event_name">
        <observer name="your_module_observer" 
                  instance="Weline\YourModule\Observer\YourObserver" 
                  disabled="false" 
                  shared="true" 
                  sort="10"/>
    </event>
</config>
```

### 配置说明

- **name**: 观察者名称，格式为 `模块名_观察者名`
- **instance**: 观察者类的完整命名空间
- **disabled**: 是否禁用，`false` 表示启用
- **shared**: 是否共享实例，`true` 表示共享
- **sort**: 执行顺序，数字越小越先执行

## 事件收集

框架会自动扫描所有模块的 `etc/event.xml` 文件，收集事件观察者。确保：

1. 观察者类实现了 `ObserverInterface` 接口
2. 在 `etc/event.xml` 中正确注册
3. 运行了 `module:upgrade` 命令

## 事件示例

### 示例：用户创建后发送邮件

#### 1. 触发事件

```php
// 在用户创建后触发事件
$eventManager->dispatch('user_created', [
    'user_id' => $user->getId(),
    'email' => $user->getEmail()
]);
```

#### 2. 创建观察者

```php
<?php

namespace Weline\YourModule\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class SendWelcomeEmail implements ObserverInterface
{
    public function execute(Event $event)
    {
        $data = $event->getData();
        $email = $data['email'] ?? '';
        
        // 发送欢迎邮件
        // mail($email, 'Welcome', 'Welcome to our site!');
    }
}
```

#### 3. 注册观察者

```xml
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework:Event/etc/xsd/event.xsd">
    <event name="user_created">
        <observer name="your_module_send_welcome_email" 
                  instance="Weline\YourModule\Observer\SendWelcomeEmail" 
                  disabled="false" 
                  shared="true" 
                  sort="10"/>
    </event>
</config>
```

## 后续处理

### 1. 运行模块升级

创建观察者后，运行模块升级命令：

```bash
php bin/w module:upgrade
```

### 2. 测试事件

触发事件，检查观察者是否正常执行。

## 验证

### 验证事件是否正常触发

1. 在代码中触发事件
2. 检查观察者是否执行
3. 检查事件数据是否正确传递

### 常见问题

1. **观察者未执行**：检查 `event.xml` 配置是否正确，确保运行了 `module:upgrade` 命令
2. **事件未触发**：检查事件名称是否正确，确保事件已正确触发
3. **数据传递错误**：检查事件数据格式是否正确
