# 模组管理

## 摘要

本文档介绍 WelineFramework 中模组的安装、升级和卸载管理，包括如何在模组安装、升级、卸载时处理数据。

## 什么是模组管理

模组管理是 WelineFramework 框架提供的模组生命周期管理功能，包括模组的安装、升级和卸载。在模组的生命周期中，可以执行相应的数据初始化、迁移和清理操作。

## 为什么需要模组管理

模组管理提供了以下功能：
- **数据初始化**：在模组安装时初始化数据库表和数据
- **数据迁移**：在模组升级时迁移和更新数据
- **数据清理**：在模组卸载时清理相关数据
- **版本控制**：管理模组的版本和依赖关系

## 目录文件结构

模组管理文件应放在模组的 `Setup` 目录下：

```
app/code/Weline/YourModule/
└── Setup/
    ├── Install.php      # 安装脚本
    ├── Upgrade.php      # 升级脚本
    └── Remove.php       # 卸载脚本
```

### 目录说明

- **Setup/**: 安装脚本目录，存放模组生命周期管理文件
- **Install.php**: 安装脚本，首次安装时执行
- **Upgrade.php**: 升级脚本，模组升级时执行
- **Remove.php**: 卸载脚本，模组卸载时执行

## 安装脚本（Install.php）

安装脚本在模组首次安装时执行，用于初始化数据库表和数据。

### 接口定义

```php
<?php

namespace Weline\Framework\Setup;

interface InstallInterface
{
    public function setup(Setup $setup, Context $context): void;
}
```

### 示例

```php
<?php

declare(strict_types=1);

namespace Weline\YourModule\Setup;

use Weline\Framework\Setup\Data\Context;
use Weline\Framework\Setup\Db\Setup;
use Weline\Framework\Setup\InstallInterface;

class Install implements InstallInterface
{
    public function setup(Setup $setup, Context $context): void
    {
        // 初始化数据库表
        // 初始化默认数据
    }
}
```

## 在模型中处理数据

除了使用 Setup 脚本，还可以在模型中直接处理数据：

### Model 中的方法

- **install()**: 首次安装时执行
- **setup()**: 开发模式下每次执行 `module:upgrade` 都会触发（非开发模式下不触发）
- **upgrade()**: 模组升级时执行

### 示例

```php
<?php

namespace Weline\YourModule\Model;

use Weline\Framework\Database\Model;
use Weline\Framework\Setup\Data\Context;
use Weline\Framework\Setup\Db\ModelSetup;

class YourModel extends Model
{
    /**
     * 安装数据表
     */
    public function install(ModelSetup $setup, Context $context): void
    {
        if (!$setup->tableExist()) {
            $setup->createTable('表名')
                ->addColumn(self::fields_ID, TableInterface::column_type_INTEGER, 0, 'primary key auto_increment', 'ID')
                ->addColumn(self::fields_NAME, TableInterface::column_type_VARCHAR, 255, 'not null', '名称')
                ->create();
        }
    }

    /**
     * 开发模式下的设置（每次 module:upgrade 都会触发）
     */
    public function setup(ModelSetup $setup, Context $context): void
    {
        // 开发模式下可以删除重建表进行测试
        // $setup->dropTable();
        // $this->install($setup, $context);
    }

    /**
     * 升级数据表
     */
    public function upgrade(ModelSetup $setup, Context $context): void
    {
        // 升级逻辑
        // 添加字段、修改字段等
    }
}
```

## 执行时机

### Install

- **执行时机**：首次安装模组时执行
- **执行次数**：仅执行一次
- **用途**：初始化数据库表和数据

### Setup

- **执行时机**：开发模式下每次执行 `php bin/w module:upgrade` 都会触发
- **执行次数**：开发模式下每次都会执行
- **用途**：开发测试，可以删除重建表

### Upgrade

- **执行时机**：模组升级版本时执行
- **执行次数**：每次升级都会执行
- **用途**：数据迁移和表结构更新

### Remove

- **执行时机**：模组卸载时执行
- **执行次数**：卸载时执行一次
- **用途**：清理数据和配置

## 后续处理

### 1. 运行模组安装

```bash
php bin/w module:upgrade
```

### 2. 运行模组升级

```bash
php bin/w module:upgrade Weline_YourModule
```

### 3. 运行模组卸载

```bash
php bin/w module:remove Weline_YourModule
```

## 验证

### 验证安装是否成功

1. 检查数据库表是否创建
2. 检查默认数据是否初始化
3. 检查模组是否在模组列表中

### 验证升级是否成功

1. 检查数据是否迁移
2. 检查表结构是否更新
3. 检查版本号是否更新

### 常见问题

1. **安装失败**：检查 Install.php 是否正确实现接口
2. **升级失败**：检查 Upgrade.php 逻辑是否正确
3. **数据未初始化**：检查 install() 方法是否正确执行
