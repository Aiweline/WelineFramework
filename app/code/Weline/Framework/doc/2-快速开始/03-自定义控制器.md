# 自定义控制器

## 摘要

本文档介绍如何在 WelineFramework 中创建自定义控制器，包括前端控制器、后端控制器、REST API 控制器的区别和使用方法。

## 什么是控制器

控制器（Controller）是 MVC 架构中的核心组件，负责处理用户请求、调用业务逻辑、返回响应。在 WelineFramework 中，控制器根据使用场景分为前端控制器、后端控制器和 REST API 控制器。

## 为什么需要不同类型的控制器

不同类型的控制器适用于不同的场景：
- **前端控制器**：用于渲染 HTML 页面，适合用户界面展示
- **后端控制器**：用于后台管理界面，通常需要权限验证
- **REST API 控制器**：用于提供 JSON 格式的 API 接口，适合前后端分离架构

## 目录文件结构

控制器文件应放在模组的 `Controller` 目录下，目录结构对应 URL 路径：

```
app/code/Weline/YourModule/
├── Controller/
│   ├── Index.php                    # 前端控制器
│   ├── Frontend/
│   │   └── Index.php                # 前端控制器（显式指定）
│   ├── Backend/
│   │   ├── Index.php                # 后端控制器
│   │   └── Api/
│   │       └── Index.php            # 后端 REST API 控制器
│   └── Api/
│       └── Index.php                # 前端 REST API 控制器
└── etc/
    └── env.php                      # 路由配置文件
```

### 目录说明

- **Controller/**: 控制器目录，子目录结构对应 URL 路径
- **etc/env.php**: 路由配置文件，定义模组的路由前缀（router）

## 控制器类型

### 1. FrontendController - 前端控制器

用于前端页面展示，返回 HTML 模板。

```php
<?php

declare(strict_types=1);

namespace Weline\YourModule\Controller;

use Weline\Framework\App\Controller\FrontendController;

class Index extends FrontendController
{
    public function index()
    {
        $this->assign('title', __('首页'));
        return $this->fetch();
    }
}
```

### 2. BackendController - 后端控制器

用于后台管理界面，通常需要权限验证。**通过后端单独入口 area 访问**。

```php
<?php

declare(strict_types=1);

namespace Weline\YourModule\Controller\Backend;

use Weline\Framework\App\Controller\BackendController;

class Index extends BackendController
{
    public function index()
    {
        $this->assign('title', __('后台首页'));
        return $this->fetch();
    }
}
```

**访问地址**：`http://your-domain/{admin_key}/{your-module|env.router}/backend/index/index` 或 `http://your-domain/{admin_key}/{your-module|env.router}/backend/index` 或 `http://your-domain/{admin_key}/{your-module|env.router}/backend`

**注意**：
- 后端控制器通过框架的后端单独入口 area（admin）访问
- `{admin_key}` 从 `app/etc/env.php` 中获取的 `admin` 配置值
- `{your-module|env.router}` 从模块的 `etc/env.php` 中获取，如果配置了 `router` 则使用该值（转为小写），否则使用模块名转小写（下划线保留）

### 3. FrontendRestController - 前端 REST API 控制器

用于前端 API 接口，返回 JSON 格式数据。**无需登录认证**。

```php
<?php

declare(strict_types=1);

namespace Weline\YourModule\Controller\Api;

use Weline\Framework\App\Controller\FrontendRestController;

class Index extends FrontendRestController
{
    public function index()
    {
        return $this->fetchJson([
            'code' => 200,
            'msg' => 'success',
            'data' => ['message' => 'Hello World']
        ]);
    }
}
```

**访问地址**：`http://your-domain/api/rest/v1/{your-module|env.router}/index/index` 或 `http://your-domain/api/rest/v1/{your-module|env.router}/index` 或 `http://your-domain/api/rest/v1/{your-module|env.router}`

### 4. BackendRestController - 后端 REST API 控制器

用于后端 API 接口，返回 JSON 格式数据。**需要管理员登录认证**。

```php
<?php

declare(strict_types=1);

namespace Weline\YourModule\Controller\Backend\Api;

use Weline\Framework\App\Controller\BackendRestController;

class Index extends BackendRestController
{
    public function index()
    {
        return $this->fetchJson([
            'code' => 200,
            'msg' => 'success',
            'data' => ['message' => 'Hello World']
        ]);
    }
}
```

**访问地址**：`http://your-domain/{api_admin}/rest/v1/{your-module|env.router}/index/index` 或 `http://your-domain/{api_admin}/rest/v1/{your-module|env.router}/index` 或 `http://your-domain/{api_admin}/rest/v1/{your-module|env.router}`

**注意**：
- 后端 API 会自动检查登录状态，未登录会返回 401 错误
- `{api_admin}` 从 `app/etc/env.php` 中获取的 `api_admin` 配置值
- `{your-module|env.router}` 从模块的 `etc/env.php` 中获取，如果配置了 `router` 则使用该值（转为小写），否则使用模块名转小写（下划线保留）

## 路由配置

在 `etc/env.php` 文件中配置路由前缀：

```php
<?php

return [
    'router' => 'your-module',  // 路由前缀（如果未配置，则使用模块名转小写）
    'dependencies' => []
];
```

**重要说明**：
- 如果 `env.php` 中配置了 `router`，则使用该值（转为小写）作为模块入口识别
- 如果未配置 `router`，则使用模块名转小写作为入口识别，**模块名中的下划线会保留**
- 例如：
  - 模块 `Weline_YourModule`，未配置 router 时，入口为 `weline_yourmodule`（下划线保留）
  - 模块 `Weline_YourModule`，配置了 `router => 'your-module'` 时，入口为 `your-module`

## URL 路径规则

控制器的 URL 路径遵循以下规则，根据不同的访问区域（area）有不同的入口：

**重要提示**：
- 当控制器方法名为 `index()` 时，URL 末尾连续的 `index` 可以省略。例如：
  - `index/index` 可以简化为 `index` 或直接省略
  - `{your-module|env.router}/index/index` 可以简化为 `{your-module|env.router}/index` 或 `{your-module|env.router}`
  - 框架会自动识别并匹配到对应的 `index()` 方法
- 路由前缀说明：`{your-module|env.router}` 表示如果 `etc/env.php` 中配置了 `router`，则使用该值（转为小写），否则使用模块名转小写（下划线保留）。例如：`Weline_YourModule` → `weline_yourmodule`

### 前端访问（Frontend Area）

- **路径格式**：`/{router}/{controller_path}/{action}`
- **router**: `{your-module|env.router}` - 如果 `etc/env.php` 中配置了 `router`，则使用该值（转为小写），否则使用模块名转小写（下划线保留）
- **示例**：
  - 文件：`Controller/Index.php`，方法：`index()`
  - URL：`http://your-domain/{your-module|env.router}/index/index` 或 `http://your-domain/{your-module|env.router}/index` 或 `http://your-domain/{your-module|env.router}`
  - 文件：`Controller/Frontend/Index.php`，方法：`index()`
  - URL：`http://your-domain/{your-module|env.router}/frontend/index/index` 或 `http://your-domain/{your-module|env.router}/frontend/index` 或 `http://your-domain/{your-module|env.router}/frontend`

### 后端访问（Backend Area）

框架后端有单独的入口 area，通过 `admin` 标识访问：

- **路径格式**：`/{admin_key}/{router}/{controller_path}/{action}`
- **admin_key**: 从 `app/etc/env.php` 中获取的 `admin` 配置值
- **router**: `{your-module|env.router}` - 如果 `etc/env.php` 中配置了 `router`，则使用该值（转为小写），否则使用模块名转小写（下划线保留）
- **示例**：
  - 文件：`Controller/Backend/Index.php`，方法：`index()`
  - URL：`http://your-domain/{admin_key}/{your-module|env.router}/backend/index/index` 或 `http://your-domain/{admin_key}/{your-module|env.router}/backend/index` 或 `http://your-domain/{admin_key}/{your-module|env.router}/backend`

### 前端 API 访问（Frontend API Area）

- **路径格式**：`/api/rest/v1/{router}/{controller_path}/{action}`
- **router**: `{your-module|env.router}` - 如果 `etc/env.php` 中配置了 `router`，则使用该值（转为小写），否则使用模块名转小写（下划线保留）
- **示例**：
  - 文件：`Controller/Api/Index.php`，方法：`index()`
  - URL：`http://your-domain/api/rest/v1/{your-module|env.router}/index/index` 或 `http://your-domain/api/rest/v1/{your-module|env.router}/index` 或 `http://your-domain/api/rest/v1/{your-module|env.router}`

### 后端 API 访问（Backend API Area）

- **路径格式**：`/{api_admin}/rest/v1/{router}/{controller_path}/{action}`
- **api_admin**: 从 `app/etc/env.php` 中获取的 `api_admin` 配置值
- **router**: `{your-module|env.router}` - 如果 `etc/env.php` 中配置了 `router`，则使用该值（转为小写），否则使用模块名转小写（下划线保留）
- **示例**：
  - 文件：`Controller/Backend/Api/Index.php`，方法：`index()`
  - URL：`http://your-domain/{api_admin}/rest/v1/{your-module|env.router}/index/index` 或 `http://your-domain/{api_admin}/rest/v1/{your-module|env.router}/index` 或 `http://your-domain/{api_admin}/rest/v1/{your-module|env.router}`

## HTTP 方法限制

可以通过方法名前缀限制 HTTP 请求方法：

- `index()`: 允许所有 HTTP 方法
- `getIndex()`: 仅允许 GET 请求
- `postIndex()`: 仅允许 POST 请求
- `putIndex()`: 仅允许 PUT 请求
- `deleteIndex()`: 仅允许 DELETE 请求

```php
public function postIndex()
{
    // 仅处理 POST 请求
    $data = $this->_request->getPost();
    // 处理逻辑...
    return $this->fetchJson(['code' => 200]);
}
```

## 高级用法：BaseController

当多个控制器有共同的初始化逻辑时，可以创建 BaseController：

```php
<?php

declare(strict_types=1);

namespace Weline\YourModule\Controller;

use Weline\Framework\App\Controller\FrontendController;

class BaseController extends FrontendController
{
    public function __init()
    {
        parent::__init();
        // 公共初始化逻辑
        $this->assign('module_name', 'YourModule');
    }
}
```

子控制器继承 BaseController：

```php
<?php

declare(strict_types=1);

namespace Weline\YourModule\Controller;

class Index extends BaseController
{
    public function index()
    {
        // 可以直接使用父类初始化的变量
        return $this->fetch();
    }
}
```

## 后续处理

### 1. 注册路由

创建、改名或删除控制器后，需要重建路由：

```bash
# 全量重建所有启用模块的路由
php bin/w setup:upgrade --route

# 只重建指定模块，保留其他模块路由
php bin/w setup:upgrade --route --module=Vendor_ModuleName
```

- 全量模式以当前启用模块和控制器为唯一事实源，会从空快照重建 `backend_pc`、`frontend_pc`、`backend_rest_api` 和 `frontend_rest_api`，因此会清除已删除、改名或 area 变更后的陈旧路由。
- `--module` 模式只清除并重建指定模块的路由，不得覆盖其他模块。
- `generated/routers/*.php` 是生成产物，不要手工修改。

### 2. 创建模板文件

对于前端控制器，需要创建对应的模板文件：

```
app/design/frontend/default/Weline_YourModule/templates/
└── index/
    └── index.phtml
```

## 验证

### 验证控制器是否可用

1. **前端控制器验证**：
   - 访问 URL：`http://your-domain/{your-module|env.router}/index/index` 或 `http://your-domain/{your-module|env.router}/index` 或 `http://your-domain/{your-module|env.router}`
   - 应该能看到渲染的页面
   - **注意**：当方法名为 `index()` 时，末尾的 `index` 可以省略

2. **后端控制器验证**：
   - 访问 URL：`http://your-domain/{admin_key}/{your-module|env.router}/backend/index/index` 或 `http://your-domain/{admin_key}/{your-module|env.router}/backend/index` 或 `http://your-domain/{admin_key}/{your-module|env.router}/backend`
   - 应该能看到后台管理页面
   - **注意**：必须通过后端入口 area 访问，不能直接使用前端路径

3. **前端 REST API 控制器验证**：
   - 访问 URL：`http://your-domain/api/rest/v1/{your-module|env.router}/index/index` 或 `http://your-domain/api/rest/v1/{your-module|env.router}/index` 或 `http://your-domain/api/rest/v1/{your-module|env.router}`
   - 应该返回 JSON 格式数据

4. **后端 REST API 控制器验证**：
   - 访问 URL：`http://your-domain/{api_admin}/rest/v1/{your-module|env.router}/index/index` 或 `http://your-domain/{api_admin}/rest/v1/{your-module|env.router}/index` 或 `http://your-domain/{api_admin}/rest/v1/{your-module|env.router}`
   - 应该返回 JSON 格式数据
   - **注意**：需要先登录，未登录会返回 401 错误

5. **HTTP 方法限制验证**：
   - 使用 GET 访问 `postIndex()` 方法应该返回 405 错误
   - 使用 POST 访问应该正常响应

### 常见问题

1. **404 错误**：
   - 检查路由配置是否正确，确保运行了 `php bin/w setup:upgrade --route`
   - 检查 `env.php` 中的 `router` 配置是否正确
   - 如果未配置 `router`，确保使用模块名（小写）作为入口

2. **后端控制器无法访问**：
   - 确保通过后端入口 area 访问：`/{admin_key}/{your-module|env.router}/backend/...`
   - 不要使用前端路径访问后端控制器
   - 检查 `app/etc/env.php` 中的 `admin` 配置值
   - 检查模块 `etc/env.php` 中的 `router` 配置，或确认模块名（小写）是否正确

3. **后端 API 返回 401 错误**：
   - 需要先登录后台管理系统
   - 检查登录状态和会话是否有效

4. **模板未找到**：检查模板文件路径是否正确，确保模板文件存在

5. **权限错误**：后端控制器可能需要权限验证，检查 ACL 配置
