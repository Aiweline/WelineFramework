## 关键发现和纠正

### 1. Model接口签名错误

**旧文档**: 没有明确说明接口方法的参数类型

**新文档**: 
```php
// ❌ 错误
use Weline\Framework\Setup\Setup;
public function setup(Setup $setup, Context $context) {}

// ✅ 正确
use Weline\Framework\Setup\Db\ModelSetup;
use Weline\Framework\Setup\Data\Context;
public function setup(ModelSetup $setup, Context $context): void {}
```

### 2. Controller方法重写问题

**旧文档**: 未提及不能重写父类方法

**新文档**: 
```php
// ❌ 错误 - 导致访问级别冲突
private function success() {}
private function error() {}

// ✅ 正确 - 直接使用父类方法
return $this->success('消息', $data);
return $this->error('错误', $code);
```

### 3. 数据库操作方法

**旧文档**: 未明确数据库连接的具体方法

**新文档**: 
- ❌ `$connection->fetchOne()` - 不存在
- ✅ `$connection->select()->from()->fetch()` - 正确方法
- ✅ 建议使用Model的ORM方法

### 4. 路由配置细节

**旧文档**: 缺少area属性说明

**新文档**: 
- 明确area可选值：frontend_rest_api, backend_rest_api等
- 路径参数格式：`:paramName`
- controller路径格式：`Api\Entity`（使用反斜杠）

### 5. 测试框架配置

**旧文档**: 缺少完整的测试配置

**新文档**: 
- phpunit.xml完整配置
- TestCase基类示例
- TDD原则和markTestIncomplete使用
