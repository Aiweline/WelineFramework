# 服务器部署说明

## 功能概述

Weline 框架提供三种 Web 服务器方式：

- **FPM 模式**（常用生产部署）：Nginx/Apache + PHP-FPM，将请求转发到 `pub/index.php`，由 PHP-FPM 进程处理。配置 Web 服务器虚拟主机即可，无需执行 `server:start`。
- **WLS 模式（推荐）**：常驻内存 HTTP 服务器，多进程、高性能，**默认监听 80/443 直连省去 Nginx**；可 `-p 9981` 等自定义端口。执行 `php bin/w server:start` 启动。详见 [Server 模块 - WLS 模式部署指南](../../../Server/doc/WLS模式部署指南.md)。
- **CLI 服务器**：使用 `php bin/w server:start --cli` 启动 PHP 内置 CLI 服务器，仅适用于开发环境快速调试。

以下说明适用于 WLS 与 CLI 的通用操作；FPM 仅需配置 Web 服务器指向项目；WLS 专属配置、多实例、域名接入与 HTTPS 见上述链接。

## 运行模式

### WLS 模式（默认）

- 执行 `php bin/w server:start` 启动，以守护进程方式运行多 Worker，不阻塞终端。
- 使用 `php bin/w server:stop` 停止；`php bin/w server:status` 查看状态。
- 无“前台/后台”参数：WLS 始终以守护进程运行。详见 [WLS 模式部署指南](../../../Server/doc/WLS模式部署指南.md)。

### CLI 模式（仅开发）

- 执行 `php bin/w server:start --cli` 启动 PHP 内置 CLI 服务器。
- **后台运行**（默认）：不阻塞终端，需用 `server:stop` 停止。
- **前台运行**：加 `-f` 或 `--foreground`，即 `php bin/w server:start --cli -f`，实时看日志，关闭终端或 `Ctrl+C` 即停。

## 使用方法

### 启动服务器

```bash
# WLS 模式（默认，守护进程）
php bin/w server:start

# CLI 模式：后台
php bin/w server:start --cli

# CLI 模式：前台（实时日志，Ctrl+C 停止）
php bin/w server:start --cli -f
```

### 查看服务器状态

```bash
php bin/w server:status
```

### 停止服务器

```bash
php bin/w server:stop
```

### 清理残留进程

```bash
php bin/w server:cleanup
```

### 参数说明

| 参数 | 说明 | 适用 |
|------|------|------|
| 无参数 | 启动 WLS（守护进程，默认 80/443） | 默认 |
| `--cli` | 使用 PHP 内置 CLI 服务器 | 仅开发 |
| `-f` 或 `--foreground` | 前台运行（实时日志） | 仅与 `--cli` 同用 |
| `--host` | 指定主机地址（默认 127.0.0.1；直连外网用 --host 0.0.0.0；-h 保留给帮助） | WLS/CLI |
| `-p` 或 `--port` | 指定端口（默认 80/443，HTTPS 时 443；可 -p 9981 等） | WLS/CLI |
| `-c` 或 `--count` | Worker 进程数（默认 auto） | WLS |
| （Master 进程默认启用，无需参数） | 监控并自动重启异常 Worker；HTTPS 时自动启动 HTTP 重定向 | WLS |
| `-r` 或 `--restart` | 平滑重启 | 仅 WLS |
| `-f`（与 `-r` 同用） | 直接切换、不等待 | 仅 WLS |
| `--ssl-cert` / `--ssl-key` | 指定 SSL 证书与私钥路径 | WLS |

## 注意事项

### WLS 与 CLI 区别

- **WLS**（`server:start` 默认）：常驻多进程，无 `-f` 前台模式，始终守护进程运行；停止用 `server:stop`。
- **CLI**（`server:start --cli`）：单进程 PHP 内置服务器；可加 `-f` 前台运行以便看日志，关闭终端即停。

### 默认端口与改端口

- **默认**：WLS 默认监听 **80（HTTP）**，启用 HTTPS 时自动使用 **443**，直连省去 Nginx。
- **改端口**：例如使用 9981：`php bin/w server:start -p 9981`，或配置 `env.server.port`。

### 端口冲突

若默认 80/443 被占用，可改用其它端口（如 `-p 9981`）或检查占用：

```bash
# Windows（检查 443）
netstat -ano | findstr :443

# Linux/Mac
lsof -i :443
```

### 80/443 标准端口（WLS 自我处理）

WLS 支持直接监听 **80（HTTP）** 和 **443（HTTPS）** 端口：

```bash
# 监听 80 端口（单 Worker，标准 HTTP）
php bin/w server:start -p 80 -c 1

# 监听 443 端口（单 Worker，HTTPS，需证书）
php bin/w server:start -p 443 -c 1
```

- **Linux/Mac**：80/443 为特权端口，需 **root** 或 **setcap** 才能绑定：
  - 以 root 运行：`sudo php bin/w server:start -p 80 -c 1`
  - 或授权能力：`sudo setcap cap_net_bind_service=+ep $(which php)`
  - 或使用 **Nginx 反代**：Nginx 监听 80/443，`proxy_pass` 到本机高端口（如 9981）
- **Windows**：无特权端口限制，可直接 `-p 80` 或 `-p 443`。
- 80/443 通常单端口监听，建议 `-c 1`；多 Worker 会占用 `port`～`port+count-1`。

### 多 Worker 端口分发说明

WLS 默认每个 Worker 监听不同端口（`port, port+1, port+2...`），外部只知道主端口时，只有第一个 Worker 能响应。

**解决方案：**

- **单 Worker**：对于标准 HTTP/HTTPS 端口（80/443），建议 `-c 1` 单 Worker 监听，避免端口分发问题。
- **Nginx 负载均衡**：多 Worker 时，可用 Nginx `upstream` 均衡分发到多个 Worker 端口：

```nginx
upstream wls_backend {
    server 127.0.0.1:443;
    server 127.0.0.1:444;
    server 127.0.0.1:445;
    server 127.0.0.1:446;
}
server {
    listen 443 ssl;
    location / {
        proxy_pass https://wls_backend;
        proxy_set_header Host $host;
    }
}
```

- **多 Master 共用 HTTP 重定向**：多个 WLS 实例（多 Master）时，HTTP 80 端口的重定向进程只需一个（不加载框架，仅做 301 重定向，负担极低），框架会自动共用已有的 HTTP 重定向进程。

### HTTP 重定向到 HTTPS（HTTPS 时）

**Master 进程默认启用**。启用 **HTTPS** 时，Master 会**自动启动一个独立的 HTTP 进程**（不计入 Worker 数），监听 **80** 端口，将 HTTP 请求 **301 重定向到 HTTPS**，避免漏掉 HTTP 流量。若未启用 HTTPS，则不启动该进程。

- 启动：`php bin/w server:start`（HTTPS 默认启用时即会启动 HTTP 重定向）。
- `server:status` 会显示 HTTP 重定向状态；`server:stop` 会一并停止该进程。

### 开发与生产

- **FPM 模式**（Nginx/Apache + PHP-FPM）：常用生产部署，配置 Web 服务器将请求转发到 `pub/index.php` 即可，无需执行 `server:start`。
- **CLI 模式**（`server:start --cli`）：仅适用于开发环境。
- **WLS 模式**（`server:start` 默认）：可用于开发和生产；生产环境可直连 WLS 端口或使用 Nginx 反向代理到 WLS。详见 [WLS 模式部署指南](../../../Server/doc/WLS模式部署指南.md)。

## 故障排除

### 常见问题

1. **"服务器已在运行中！"**
   - 检查是否有残留进程：`php bin/w server:status`
   - 清理残留进程：`php bin/w server:cleanup`
   - 或直接停止：`php bin/w server:stop`

2. **"端口被占用"**
   - 检查端口占用情况
   - 停止占用端口的进程
   - 或修改服务器端口配置

3. **"进程ID显示不正确"**
   - 运行清理命令：`php bin/w server:cleanup`
   - 重新启动服务器

4. **"配置信息不完整"**
   - 自动修复：系统会自动检测并更新配置
   - 手动清理：`php bin/w server:cleanup`

### Master 进程（默认启用）

WLS 启动后**始终由 Master 进程**监控 Worker，Worker 异常退出时自动重启。

```bash
php bin/w server:start
```

- **重载**：重载信号（如 CLI 执行完成、配置变更、热重载）应**发给 Master**（SIGHUP），由 Master 统一写重载标记文件，Worker 轮询后重载；与 Workerman 的「Master 收 SIGHUP 再协调 Worker 重载」一致。
- **定期巡检**：Master 主循环每约 5 秒做健康检查（端口是否在监听），发现 Worker 掉线则尝试重启，并限制重启频率。
- **性能**：Master 为独立进程，仅做 sleep + 少量端口检测与写文件，**不会拉低 Worker 性能**。
- `server:status` 会显示 Master PID、Worker 与 HTTP 重定向状态。
- 启用 HTTPS 时，Master 会额外启动 HTTP 重定向进程（默认 80）；停止时 `server:stop` 会先停 Master，再停 Worker 与 HTTP 重定向进程。

### 推荐使用流程

1. **日常开发**：`php bin/w server:start`（WLS 默认，监听 80/443）
2. **改端口**：`php bin/w server:start -p 9981`
3. **需看实时日志**：`php bin/w server:start --cli -f`（CLI 前台）
4. **生产/长运行**：`php bin/w server:start`（Master 默认启用，监控 Worker）
5. **遇到问题时**：先运行 `php bin/w server:cleanup` 清理
6. **切换项目时**：记得停止 `php bin/w server:stop`

### 日志文件

服务器日志位于：
- 启动日志：`var/log/server-start.log`
- Master 进程：`var/log/master.log`
- Worker 日志：`var/log/worker-{端口}.log`
- HTTP 重定向日志：`var/log/http_redirect-{端口}.log`（HTTPS 时）
- 错误日志：`var/log/error.log`
- 调试日志：`var/log/debug.log`
