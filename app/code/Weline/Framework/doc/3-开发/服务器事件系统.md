# 服务器事件系统

## 概述

Weline框架为服务器管理提供了完整的事件系统，允许开发者在服务器启动和停止时执行自定义逻辑。

## 可用事件

### 1. 服务器启动事件

**事件名称**: `Weline_Framework_Server::start_after`

**触发时机**: 服务器成功启动后

**事件数据**:
```php
[
    'pid' => int,           // 进程ID
    'host' => string,       // 主机地址
    'port' => int,          // 端口号
    'backend' => bool,      // 是否后台运行
    'start_time' => int,    // 启动时间戳
    'force' => bool         // 是否强制启动
]
```

### 2. 服务器停止事件

**事件名称**: `Weline_Framework_Server::stop_after`

**触发时机**: 服务器停止操作完成后

**事件数据**:
```php
[
    'host' => string,           // 主机地址
    'port' => int,              // 端口号
    'force' => bool,            // 是否强制停止
    'stopped_pids' => array,    // 停止的进程ID列表
    'stop_time' => int          // 停止时间戳
]
```

## 创建观察者

### 1. 创建观察者类

观察者类必须实现 `Weline\Framework\Event\ObserverInterface` 接口：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Event\Event;

class ServerStartObserver implements ObserverInterface
{
    public function execute(Event $event): void
    {
        $data = $event->getData();
        
        // 你的自定义逻辑
        $this->logServerStart($data);
        $this->sendNotification($data);
        $this->updateDatabase($data);
    }
    
    private function logServerStart(array $data): void
    {
        // 记录日志
    }
    
    private function sendNotification(array $data): void
    {
        // 发送通知
    }
    
    private function updateDatabase(array $data): void
    {
        // 更新数据库
    }
}
```

### 2. 注册观察者

在模块的 `etc/events.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="urn:magento:framework:Event/etc/events.xsd">
    <!-- 服务器启动事件 -->
    <event name="Weline_Framework_Server::start_after">
        <observer name="your_module_server_start" instance="Your\Module\Observer\ServerStartObserver" sort="10"/>
    </event>
    
    <!-- 服务器停止事件 -->
    <event name="Weline_Framework_Server::stop_after">
        <observer name="your_module_server_stop" instance="Your\Module\Observer\ServerStopObserver" sort="10"/>
    </event>
</config>
```

## 使用示例

### 1. 监控和日志记录

```php
class ServerMonitorObserver implements ObserverInterface
{
    public function execute(Event $event): void
    {
        $data = $event->getData();
        
        // 记录到监控系统
        $this->logToMonitoringSystem($data);
        
        // 发送邮件通知
        if ($data['backend']) {
            $this->sendEmailNotification('服务器已在后台启动', $data);
        }
    }
}
```

### 2. 数据库记录

```php
class ServerDatabaseObserver implements ObserverInterface
{
    public function execute(Event $event): void
    {
        $data = $event->getData();
        
        // 记录到数据库
        $this->insertServerRecord($data);
    }
    
    private function insertServerRecord(array $data): void
    {
        // 插入服务器记录到数据库
    }
}
```

### 3. 资源清理

```php
class ServerCleanupObserver implements ObserverInterface
{
    public function execute(Event $event): void
    {
        $data = $event->getData();
        
        // 清理临时文件
        $this->cleanupTempFiles();
        
        // 清理缓存
        $this->clearCache();
        
        // 重置状态
        $this->resetStatus();
    }
}
```

## 事件执行顺序

观察者按照 `sort` 属性定义的顺序执行，数值越小优先级越高：

```xml
<observer name="high_priority" instance="Your\Observer\HighPriorityObserver" sort="1"/>
<observer name="normal_priority" instance="Your\Observer\NormalPriorityObserver" sort="10"/>
<observer name="low_priority" instance="Your\Observer\LowPriorityObserver" sort="100"/>
```

## 注意事项

1. **异常处理**: 观察者中的异常不会影响服务器启动/停止流程
2. **性能考虑**: 观察者应该快速执行，避免阻塞主流程
3. **数据安全**: 事件数据是引用传递，修改会影响后续观察者
4. **日志记录**: 建议在观察者中添加适当的日志记录
5. **错误处理**: 观察者应该处理可能的错误情况

## 调试和测试

### 1. 查看事件日志

事件日志会写入到 `var/log/server-events.log` 文件中。

### 2. 测试观察者

```php
// 手动触发事件进行测试
$eventManager = ObjectManager::getInstance(EventsManager::class);
$testData = [
    'pid' => 12345,
    'host' => '127.0.0.1',
    'port' => 9981,
    'backend' => true,
    'start_time' => time(),
    'force' => false
];
$eventManager->dispatch('Weline_Framework_Server::start_after', $testData);
```

### 3. 禁用观察者

可以通过注释掉 `events.xml` 中的观察者来临时禁用：

```xml
<!-- <observer name="your_observer" instance="Your\Observer\Class" sort="10"/> -->
```

## 扩展功能

### 1. 自定义事件数据

可以在观察者中向事件数据添加额外信息：

```php
public function execute(Event $event): void
{
    $data = $event->getData();
    $data['custom_field'] = 'custom_value';
    $event->setData($data);
}
```

### 2. 条件执行

根据事件数据条件执行不同的逻辑：

```php
public function execute(Event $event): void
{
    $data = $event->getData();
    
    if ($data['force']) {
        $this->handleForceStart($data);
    } else {
        $this->handleNormalStart($data);
    }
}
```

这个事件系统为服务器管理提供了强大的扩展能力，开发者可以根据需要添加各种自定义功能。
