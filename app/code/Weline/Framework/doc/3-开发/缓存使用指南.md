# 缓存使用指南

## 契约（必读）

`w_cache($identity)` 返回的池遵守：

1. **普通路径**（`get` / `set` / `remember` / `delete` / `has`）：框架**自动**注入当前请求的 `website_code` + `lang` + `currency`，并固定带上 `area`（frontend/backend）。业务只写逻辑键，不要手拼站点/语言/货币。
2. **特殊路径**（`getCustom` / `setCustom` / `rememberCustom` / `deleteCustom` / `hasCustom`）：维度以 **bool 形参**控制，**默认 `false` = 逃逸（不注入）**；传 `true` 才挂上对应维。

```php
$cache = w_cache('product');

// 常规：自动全维，开发者无需关心维度
$cache->set('price:' . $id, $payload, 1800);
$price = $cache->get('price:' . $id);
$price = $cache->remember('price:' . $id, 1800, function () { /* rebuild */ });

// 全局词典等：与当前请求站点环境无关 → Custom 全逃逸
$phrase = w_cache('phrase');
$phrase->setCustom('phrase_locale_words_' . $lang, $words);
$words = $phrase->getCustom('phrase_locale_words_' . $lang);

// 只要语言维（仍不挂站点/货币）
$cache->getCustom('menu:header', lang: true);
$cache->setCustom('menu:header', $html, 600, lang: true);
```

## 何时用 Custom

| 场景 | 用法 |
|------|------|
| 前台展示、价格、区块、与站点/语言/货币相关 | 普通 `get/set/remember` |
| 语言词典、插件/事件元数据、模块结构、bootstrap 全局表 | `*Custom` 全逃逸（或不传维） |
| 明确只要某几维 | `*Custom` 且对应 bool=`true` |

禁止用 `getCustom(..., website: true, lang: true, currency: true)` 代替普通 `get`。

## 键与防击穿

- 存储键由 `KeyBuilder::applyDimensionFlags` + `KeyBuilder::build` 生成。
- `website` 只认规范化 `website_code`（零号站 = `default`），不把 `website_id` 写入键。
- `remember` / `rememberCustom` 的 single-flight **锁与最终存储键同源**。

## 迁移说明

升级后旧裸键会冷 miss（自然重建即可）。原先依赖「全局共享、不带站点维」的写入必须改为 `*Custom`，否则会被默认全维错误分片。

Framework 内已迁移示例：`Weline\Framework\Phrase\Parser`、Plugin/Event 配置缓存。
