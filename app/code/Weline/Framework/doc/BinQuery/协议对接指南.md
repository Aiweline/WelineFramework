# BinQuery 协议对接指南

当语言没有官方 SDK 时，可以直接实现 WQB1 二进制协议并请求 `/bin/query`。

## Endpoint 推导

开发者只需要填写域名：

```text
example.com
```

客户端推导 endpoint：

```text
https://example.com/bin/query
```

## apiKey 与 Header

`apiKey` 填 `Weline_Api` 第三方应用的 `access_token`。它是临时访问令牌，默认有效期 `3600` 秒；过期后需要用 `refresh_token` 调用 `POST /api/rest/v1/apps/refresh` 换新 token。

```http
POST /bin/query HTTP/1.1
Authorization: Bearer {apiKey}
X-Weline-BinQuery-Protocol: binquery-v1
Content-Type: application/x-weline-query-bin
Accept: application/x-weline-query-bin
```

也可以用以下 Header 传入同一个 access token：

```http
X-Weline-BinQuery-Key: {apiKey}
X-Weline-Api-Key: {apiKey}
X-API-Token: {apiKey}
```

推荐仍然使用标准 `Authorization: Bearer {apiKey}`。

## 后台创建与授权流程

1. 创建第三方应用：后台 API 集成创建，或调用 `POST /api/rest/v1/apps/create`。返回 `client_id` 和 `client_secret`。
2. 授权 BinQuery scope：至少包含 `Weline_Framework::binquery` 或 `Weline_Framework::binquery::post`。授权返回一次性 `code`。
3. 换取 token：调用 `POST /api/rest/v1/apps/token`，传 `client_id`、`client_secret`、`code`、`redirect_uri`，返回 `access_token`、`refresh_token`、`expires_in`。
4. 请求 BinQuery：把 `access_token` 放到 `Authorization: Bearer`，按 WQB1 协议发送 payload。

生命周期：

- `client_id/client_secret`：长期应用凭据；应用被禁用、删除或重置后失效。
- `code`：一次性临时授权码。
- `access_token`：请求 `/bin/query` 的临时凭证，默认 `3600` 秒。
- `refresh_token`：刷新凭证，默认 `2592000` 秒；刷新成功后旧 refresh token 会被撤销。

## Binary Packet

包头：

```text
magic: WQB1
version: 1 byte, 当前为 1
payload: Weline binary value
```

值类型：

```text
0x00 null
0x01 false
0x02 true
0x03 int: sign byte + varuint magnitude
0x04 float64: big-endian IEEE 754
0x05 string: varuint byte length + UTF-8 bytes
0x06 bytes: varuint byte length + bytes
0x07 list: varuint count + values
0x08 map: varuint count + repeated key/value
```

map key 使用 UTF-8 string，长度以 varuint 表示。

### WQB1 限制

- 单个完整包（含 `WQB1` magic 和 version）最大 4MB（4,194,304 bytes）。
- string、bytes 和 map key 单值最大 2MB（2,097,152 bytes）。
- 值嵌套深度最大 32，list 最多 200 项，map 最多 100 个 key。
- 整数范围保持 JavaScript safe integer：`-9007199254740991` 到 `9007199254740991`。

这些值是 `binquery-v1` 的现有兼容边界；本轮只统一服务端事实源和错误文案，没有修改任何类型 tag、varuint、字节序或包头。

## connect

```json
{
  "type": "connect",
  "area": "frontend"
}
```

响应：

```json
{
  "ok": true,
  "data": {
    "protocol": "binquery-v1",
    "default_area": "frontend",
    "capabilities": ["connect", "query", "call", "graph", "help", "docs", "exists"]
  },
  "error": null,
  "request_id": "..."
}
```

## query/help/docs/exists

```json
{"type": "query", "what": "providers"}
{"type": "query", "what": "provider", "provider": "theme"}
{"type": "query", "what": "operations", "provider": "theme"}
{"type": "query", "what": "docs", "provider": "theme", "operation": "list"}
{"type": "query", "what": "exists", "provider": "theme", "operation": "list"}
```

```json
{
  "type": "call",
  "provider": "theme",
  "operation": "list",
  "params": {
    "page": 1,
    "page_size": 20
  }
}
```

## graph

```json
{
  "type": "graph",
  "graph": [
    {
      "as": "themes",
      "provider": "theme",
      "operation": "list",
      "params": {"page": 1, "page_size": 20}
    }
  ]
}
```

Graph 只允许 `mode=read` 且 `graph=true` 的外部 operation。

## 错误结构

错误仍然是 WQB1 二进制响应，解码后结构固定：

```json
{
  "ok": false,
  "data": null,
  "error": {
    "code": "validation_error",
    "message": "Provider and operation are required."
  },
  "request_id": "..."
}
```

常见错误码：

- `protocol_error`
- `auth_error`
- `scope_denied`
- `validation_error`
- `capability_denied`
- `not_found`
- `business_error`

未预期服务端异常会保持 HTTP 500 和 `business_error`，但客户端只会收到通用错误文案。`request_id` 仍会在响应中返回，用于与服务端结构化日志关联；异常类、内部消息、文件位置和堆栈不再返回给客户端。

## CDN cache marker

当 operation docs 中存在：

```json
{
  "cache": {
    "cdn": true,
    "ttl": "15m",
    "visibility": "public",
    "key_params": ["page", "page_size"],
    "vary": ["area", "locale", "currency"]
  }
}
```

客户端可以追加：

```text
?__wq_cache=wq1.frontend.theme.list.{hash}
```

hash 计算输入：

```json
{
  "area": "frontend",
  "provider": "theme",
  "operation": "list",
  "params": {"page": 1, "page_size": 20},
  "vary": {"area": "frontend", "locale": null, "currency": null}
}
```

使用 JSON UTF-8 字符串做 SHA-256，取前 24 位十六进制。服务端会重新计算 marker；不匹配时强制 `Cache-Control: no-store`。
