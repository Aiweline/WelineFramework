# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Resource::compiler` 事件及其使用方法。该事件在编译资源文件时触发。

## 事件列表

### 1. Framework_Resource::compiler - 资源编译器事件

#### 基本信息

- **事件名称**：`Framework_Resource::compiler`
- **事件类型**：资源编译事件
- **触发时机**：在编译资源文件时
- **触发位置**：`app/code/Weline/Framework/Resource/Compiler.php` 第 50 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_Resource::compiler` 事件在编译资源文件时触发，允许其他模块监听并处理资源编译过程。事件数据包含区域、类型、资源列表等信息。

该事件主要用于：
- 监听资源编译过程
- 处理资源文件
- 资源编译日志记录
- 资源编译后处理

#### 触发时机

```php
// app/code/Weline/Framework/Resource/Compiler.php
$data = new DataObject([
    'area' => $area,
    'type' => $this->reader->getSourceType(),
    'resources' => $config_resource
]);
$this->getEventManager()->dispatch('Framework_Resource::compiler', $data);
```

#### 使用场景

- **资源处理**：在资源编译时处理资源文件
- **日志记录**：记录资源编译的详细信息
- **资源优化**：对资源进行优化处理
- **资源验证**：验证资源文件的完整性

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Resource::compiler">
        <observer name="Your_Module::resource_compiler"
                  instance="Your\Module\Observer\ResourceCompilerObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ResourceCompilerObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $area = $data->getData('area');
        $type = $data->getData('type');
        $resources = $data->getData('resources');
        
        // 处理资源编译逻辑
        // 例如：记录日志、优化资源、验证资源等
    }
}
```

#### 事件数据

`Framework_Resource::compiler` 事件传递的数据：

```php
[
    'area' => string,        // 区域（frontend/backend）
    'type' => string,        // 资源类型（css/js等）
    'resources' => array,    // 资源文件列表
]
```

**数据说明**：
- `area`：资源所属区域，通常是 `frontend` 或 `backend`
- `type`：资源类型，如 `css`、`js` 等
- `resources`：资源文件列表数组

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **性能考虑**：该事件在资源编译时触发，应避免执行耗时操作
3. **资源修改**：可以修改事件数据中的资源列表来影响编译结果
