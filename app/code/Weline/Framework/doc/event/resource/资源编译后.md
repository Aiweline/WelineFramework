# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Resource::compiler_after` 事件及其使用方法。该事件在所有资源文件编译完成后触发。

## 事件列表

### 1. Framework_Resource::compiler_after - 资源编译后事件

#### 基本信息

- **事件名称**：`Framework_Resource::compiler_after`
- **事件类型**：资源编译事件
- **触发时机**：在所有资源文件编译完成后
- **触发位置**：`app/code/Weline/Framework/Resource/Compiler.php` 第 81 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_Resource::compiler_after` 事件在所有资源文件编译完成后触发，允许其他模块执行编译后的处理操作。事件数据包含资源类型和所有区域的资源内容。

该事件主要用于：
- 在所有资源编译完成后执行统一处理
- 提取资源中的翻译词
- 注入额外的代码到编译后的资源
- 资源编译后的验证和优化

#### 触发时机

```php
// app/code/Weline/Framework/Resource/Compiler.php
$afterData = new DataObject([
    'type' => $this->reader->getSourceType(),
    'resources' => $config_resources
]);
$this->getEventManager()->dispatch('Framework_Resource::compiler_after', $afterData);
```

事件在所有区域的资源编译完成后触发一次，传递所有区域的资源内容。

#### 使用场景

- **提取翻译词**：从编译后的 JS 资源中提取翻译词，自动生成翻译文件
- **代码注入**：向编译后的资源注入额外的 JavaScript 代码
- **资源优化**：对编译后的资源进行优化处理
- **内容验证**：验证编译后资源的完整性

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Resource::compiler_after">
        <observer name="Your_Module::resource_compiler_after"
                  instance="Your\Module\Observer\ResourceCompilerAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="50"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ResourceCompilerAfterObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $eventData = $event->getEvenData();
        $type = $eventData->getData('type');
        $resources = $eventData->getData('resources');
        
        // resources 是按 area 分组的数组
        // 格式：['frontend' => '内容', 'backend' => '内容']
        foreach ($resources as $area => $areaResources) {
            // 处理每个区域的资源
            // 例如：提取翻译词、注入代码等
        }
        
        // 可以修改 resources 来更新事件数据
        $eventData->setData('resources', $resources);
    }
}
```

#### 事件数据

`Framework_Resource::compiler_after` 事件传递的数据：

```php
[
    'type' => string,        // 资源类型（如 'weline.modules.js'）
    'resources' => array,    // 按区域分组的资源内容
                            // 格式：['frontend' => '内容', 'backend' => '内容']
]
```

**数据说明**：
- `type`：资源类型，如 `weline.modules.js`、`css`、`js` 等
- `resources`：按区域分组的资源内容数组，键为区域名（如 `frontend`、`backend`），值为该区域的资源内容字符串

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **性能考虑**：该事件在所有资源编译完成后触发，可以执行相对耗时的操作
3. **资源修改**：可以修改事件数据中的资源内容来影响最终的编译结果
4. **数据格式**：`resources` 是按区域分组的数组，不是单个字符串
