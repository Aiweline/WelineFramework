# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Framework_Router::route_after` 事件及其使用方法。该事件在路由处理完成后触发，允许其他模块修改路由结果。

## 事件列表

### 1. Weline_Framework_Router::route_after - 路由处理后事件

#### 基本信息

- **事件名称**：`Weline_Framework_Router::route_after`
- **事件类型**：路由处理事件
- **触发时机**：在路由处理完成后
- **触发位置**：`app/code/Weline/Framework/Router/Core.php` 第 610 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Weline_Framework_Router::route_after` 事件在路由处理完成后触发，此时：
- 控制器已执行
- 结果已生成
- 可以修改路由结果
- 可以执行路由后处理

该事件主要用于：
- 修改路由结果
- 添加响应头
- 执行路由后处理
- 记录路由日志

#### 触发时机

```php
// app/code/Weline/Framework/Router/Core.php
$result = call_user_func([$dispatch, $method]);
# ----------事件：处理url之前 开始------------
$resultData = new DataObject(['result' => $result, 'route' => $this]);
$eventData = ['data' => $resultData];
$eventManager->dispatch('Weline_Framework_Router::route_after', $eventData);
```

#### 使用场景

- **结果修改**：修改控制器返回的结果
- **响应头设置**：设置自定义响应头
- **路由后处理**：执行路由后处理逻辑
- **日志记录**：记录路由处理日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Framework_Router::route_after">
        <observer name="Your_Module::modify_route_result"
                  instance="Your\Module\Observer\ModifyRouteResultObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ModifyRouteResultObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $resultData = $event->getData('data');
        $result = $resultData->getData('result');
        
        // 修改结果
        $modifiedResult = $this->modifyResult($result);
        
        $resultData->setData('result', $modifiedResult);
    }
    
    private function modifyResult($result)
    {
        // 结果修改逻辑
        return $result;
    }
}
```

#### 事件数据

`Weline_Framework_Router::route_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| data | DataObject | 包含 result 和 route 的数据对象 |

**获取和修改数据**：

```php
$resultData = $event->getData('data');
$result = $resultData->getData('result');
$route = $resultData->getData('route');

// 修改结果
$resultData->setData('result', $modifiedResult);
```

#### 使用示例

##### 示例 1：修改路由结果

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ModifyRouteResultObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $resultData = $event->getData('data');
        $result = $resultData->getData('result');
        
        // 如果是字符串结果，添加水印
        if (is_string($result)) {
            $result = $this->addWatermark($result);
            $resultData->setData('result', $result);
        }
    }
    
    private function addWatermark(string $content): string
    {
        // 添加水印逻辑
        return $content;
    }
}
```

##### 示例 2：设置响应头

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class SetResponseHeadersObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        // 设置自定义响应头
        header('X-Custom-Header: value');
        header('X-Response-Time: ' . (microtime(true) - $_SERVER['REQUEST_TIME_FLOAT']));
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **结果修改**：必须通过 `$resultData->setData('result', $modifiedResult)` 来更新结果
3. **性能考虑**：该事件在每次请求时都会触发，应避免执行耗时操作
4. **响应头**：设置响应头应在输出内容之前

#### 系统行为说明

1. **默认观察者**：暂无默认观察者

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 后执行的观察者可以覆盖先执行的观察者的修改

## 扩展开发

如果需要扩展路由处理后的功能，可以：

1. **创建新的观察者**：实现 `ObserverInterface` 接口
2. **注册观察者**：在模块的 `etc/event.xml` 中注册
3. **设置执行顺序**：通过 `sort` 属性控制执行顺序
4. **处理异常**：确保观察者中的异常被正确处理
