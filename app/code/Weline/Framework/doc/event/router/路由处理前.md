# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Router::route_before` 事件及其使用方法。该事件在路由处理之前触发，允许其他模块在路由前执行操作。

## 事件列表

### 1. Framework_Router::route_before - 路由处理前事件

#### 基本信息

- **事件名称**：`Framework_Router::route_before`
- **事件类型**：路由处理事件
- **触发时机**：在路由处理之前
- **触发位置**：`app/code/Weline/Framework/Router/Core.php` 第 593 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_Router::route_before` 事件在路由处理之前触发，此时：
- 路由对象已创建
- 控制器尚未执行
- 可以修改路由信息
- 可以执行路由前检查

该事件主要用于：
- 路由权限检查
- 路由信息修改
- 路由前处理
- 路由日志记录

#### 触发时机

```php
// app/code/Weline/Framework/Router/Core.php
/**@var EventsManager $eventManager */
$eventManager = ObjectManager::getInstance(EventsManager::class);
$eventData = ['route' => $this];
$eventManager->dispatch('Framework_Router::route_before', $eventData);
```

#### 使用场景

- **权限检查**：检查路由访问权限
- **路由修改**：修改路由信息
- **路由前处理**：执行路由前处理逻辑
- **日志记录**：记录路由访问日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Router::route_before">
        <observer name="Your_Module::check_route_permission"
                  instance="Your\Module\Observer\CheckRoutePermissionObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CheckRoutePermissionObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $route = $event->getData('route');
        
        // 检查路由权限
        if (!$this->hasPermission($route)) {
            throw new \Exception('没有访问权限');
        }
    }
    
    private function hasPermission($route): bool
    {
        // 权限检查逻辑
        return true;
    }
}
```

#### 事件数据

`Framework_Router::route_before` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| route | Core | 路由对象 |

**获取数据**：

```php
$route = $event->getData('route');
```

#### 使用示例

##### 示例 1：路由权限检查

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Acl\Acl;
use Weline\Framework\Manager\ObjectManager;

class CheckRoutePermissionObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $route = $event->getData('route');
        $uri = $route->getRequest()->getUri();
        
        // 检查权限
        /** @var Acl $acl */
        $acl = ObjectManager::getInstance(Acl::class);
        
        if (!$acl->hasPermission($uri)) {
            // 重定向到登录页
            header('Location: /admin/login');
            exit;
        }
    }
}
```

##### 示例 2：路由日志记录

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class LogRouteAccessObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $route = $event->getData('route');
        $request = $route->getRequest();
        
        $logData = [
            'time' => date('Y-m-d H:i:s'),
            'uri' => $request->getUri(),
            'method' => $request->getMethod(),
            'ip' => $_SERVER['REMOTE_ADDR'] ?? '',
        ];
        
        // 记录日志
        error_log(json_encode($logData, JSON_UNESCAPED_UNICODE));
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **异常处理**：如果观察者抛出异常，可能会阻止路由处理
3. **性能考虑**：该事件在每次请求时都会触发，应避免执行耗时操作
4. **路由对象**：路由对象是只读的，不应直接修改

#### 系统行为说明

1. **默认观察者**：暂无默认观察者

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 建议将权限检查放在前面（较小的 sort 值）

## 扩展开发

如果需要扩展路由处理前的功能，可以：

1. **创建新的观察者**：实现 `ObserverInterface` 接口
2. **注册观察者**：在模块的 `etc/event.xml` 中注册
3. **设置执行顺序**：通过 `sort` 属性控制执行顺序
4. **处理异常**：确保观察者中的异常被正确处理
