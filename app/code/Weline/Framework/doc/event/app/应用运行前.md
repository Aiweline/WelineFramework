# 应用运行前

事件：`Weline_Framework::App::run_before`

该事件由 FPM/WLS 共用的 `RequestPipeline` 在 URL 已完成规范化、且 early response/FPC 未命中后触发。它位于 lazy Session 与 Router/Controller 之前，适合执行只依赖规范化请求上下文的轻量初始化或可选集成。

固定位置：

```text
pre_route_gate
→ URL/apply
→ early response/FPC miss
→ run_before
→ lazy Session
→ Router/Controller
```

约束：

- 必须早于缓存执行的维护/安全规则注册到 `Weline_Framework::App::pre_route_gate`，不能放在本事件。
- 不得再次解析原始 HTTP 请求，不得派发 Router/Controller，不得使用 `exit/die`。
- early response 命中时本事件不会执行；所有出口的清理由 Runtime `finally` 与 `RequestResetterInterface` 保证。
- 观察者应保持无阻塞、可重复，并只引用 Framework 或已声明模块的 `Api` 契约。

清理与响应不变量：

- FPM 必须在当前 `Context` 仍然有效时执行 `RequestContext::cleanup()`，然后才能 `Context::leave()`；重复的 `terminate()` 不得再次执行回调。
- WLS 必须先快照待发送的 Header/Cookie，再执行请求态重置和 Context 退出。
- early `Response` 被吸收到当前请求时，同名多值 Header 的第一个值覆盖旧值，后续值按原顺序追加；正文、状态码和 Cookie 不得丢失。

注册示例：

```xml
<event name="Weline_Framework::App::run_before">
    <observer name="Vendor_Module::request_context"
              instance="Vendor\Module\Observer\RequestContextObserver"
              disabled="false"
              shared="true"
              sort="100"/>
</event>
```
