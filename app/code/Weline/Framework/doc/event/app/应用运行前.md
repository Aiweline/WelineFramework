# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `App::run_before` 事件及其使用方法。该事件在应用运行前触发，允许其他模块在 URL 解析和路由处理之前执行必要的初始化或检查操作。

## 事件列表

### 1. App::run_before - 应用运行前事件

#### 基本信息

- **事件名称**：`App::run_before`
- **事件类型**：应用生命周期事件
- **触发时机**：在 `App::run()` 方法中，`App::init()` 之后，URL 解析之前
- **触发位置**：`app/code/Weline/Framework/App.php` 第 275 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`App::run_before` 事件在应用运行前触发，此时：
- 系统初始化已完成（`App::init()` 已执行）
- 环境变量和配置已加载
- URL 尚未解析
- 路由尚未处理
- 请求对象尚未创建

该事件主要用于：
- 执行应用运行前的初始化操作
- 检查全页缓存
- 处理媒体资源请求
- 执行权限检查
- 设置全局变量或环境配置
- 其他需要在路由处理前执行的操作

#### 触发时机

```php
// app/code/Weline/Framework/App.php
public static function run(): string
{
    self::init();  // 系统初始化
    
    $_SERVER['WELINE_PARSER_URL'] = true;
    $_SERVER['WELINE_IS_MEDIA'] = false;
    
    /**@var EventsManager $eventManager */
    $eventManager = ObjectManager::getInstance(EventsManager::class);
    $eventManager->dispatch('App::run_before');  // ← 事件在此处触发
    
    // URL 解析和路由处理...
}
```

#### 使用场景

- **全页缓存检查**：在 URL 解析前检查是否存在全页缓存，如果存在则直接返回缓存内容
- **媒体资源处理**：处理静态资源请求，如 CSS、JS、图片等
- **权限预检查**：在路由处理前进行权限验证
- **环境配置**：根据请求信息设置环境变量或配置
- **请求预处理**：修改或验证请求参数
- **日志记录**：记录请求开始时间或请求信息
- **安全检查**：执行安全相关的检查，如 IP 白名单、请求频率限制等

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="App::run_before">
        <observer name="Your_Module::your_observer_name"
                  instance="Your\Module\Observer\YourObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class YourObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        // 在这里执行你的逻辑
        // 例如：检查缓存、处理媒体资源、权限验证等
        
        // 可以通过 $_SERVER 获取请求信息
        $requestUri = $_SERVER['REQUEST_URI'] ?? '';
        $requestMethod = $_SERVER['REQUEST_METHOD'] ?? 'GET';
        
        // 可以设置全局变量
        $_SERVER['YOUR_CUSTOM_FLAG'] = true;
        
        // 如果需要阻止后续处理，可以设置标志
        // $_SERVER['WELINE_PARSER_URL'] = false;
    }
}
```

#### 事件数据

`App::run_before` 事件不传递额外的数据，观察者可以通过以下方式获取信息：

1. **通过 `$_SERVER` 超全局变量**：
   - `$_SERVER['REQUEST_URI']` - 请求 URI
   - `$_SERVER['REQUEST_METHOD']` - 请求方法
   - `$_SERVER['HTTP_HOST']` - 主机名
   - `$_SERVER['WELINE_PARSER_URL']` - 是否解析 URL（可修改）
   - `$_SERVER['WELINE_IS_MEDIA']` - 是否媒体资源（可修改）

2. **通过 `$_GET`、`$_POST`、`$_COOKIE` 等超全局变量**获取请求参数

3. **通过 `App::Env()` 方法**获取环境配置

#### 使用示例

##### 示例 1：检查全页缓存

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Cache\CacheInterface;
use Weline\Framework\Manager\ObjectManager;

class CheckFullPageCacheObserver implements ObserverInterface
{
    private CacheInterface $cache;

    public function __construct(CacheInterface $cache)
    {
        $this->cache = $cache->create();
    }

    public function execute(Event &$event): void
    {
        // 生成缓存键
        $cacheKey = 'full_page_' . md5($_SERVER['REQUEST_URI'] ?? '');
        
        // 检查缓存
        $cachedContent = $this->cache->get($cacheKey);
        
        if ($cachedContent !== false) {
            // 如果存在缓存，直接输出并退出
            echo $cachedContent;
            exit;
        }
    }
}
```

##### 示例 2：处理媒体资源

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class MediaResourceObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $requestUri = $_SERVER['REQUEST_URI'] ?? '';
        
        // 检查是否是媒体资源请求
        if (preg_match('/\.(css|js|jpg|jpeg|png|gif|svg|woff|woff2|ttf|eot)$/i', $requestUri)) {
            $_SERVER['WELINE_IS_MEDIA'] = true;
            $_SERVER['WELINE_PARSER_URL'] = false; // 跳过 URL 解析
            
            // 处理媒体资源...
            $this->serveMediaResource($requestUri);
            exit;
        }
    }
    
    private function serveMediaResource(string $path): void
    {
        // 处理媒体资源的逻辑
    }
}
```

##### 示例 3：权限预检查

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Manager\ObjectManager;
use Weline\Acl\Model\Acl;

class PermissionCheckObserver implements ObserverInterface
{
    private Acl $acl;

    public function __construct(Acl $acl)
    {
        $this->acl = $acl;
    }

    public function execute(Event &$event): void
    {
        $requestUri = $_SERVER['REQUEST_URI'] ?? '';
        
        // 检查是否是后端请求
        if (str_starts_with($requestUri, '/admin/')) {
            // 执行权限检查
            if (!$this->acl->hasPermission($requestUri)) {
                // 如果没有权限，重定向到登录页
                header('Location: /admin/login');
                exit;
            }
        }
    }
}
```

##### 示例 4：设置环境变量

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\App;

class EnvironmentSetupObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        // 根据请求信息设置环境变量
        $host = $_SERVER['HTTP_HOST'] ?? '';
        
        if (str_contains($host, 'admin.')) {
            App::Env('area', 'admin');
            $_SERVER['WELINE_AREA'] = 'admin';
        } elseif (str_contains($host, 'api.')) {
            App::Env('area', 'api');
            $_SERVER['WELINE_AREA'] = 'api';
        } else {
            App::Env('area', 'frontend');
            $_SERVER['WELINE_AREA'] = 'frontend';
        }
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **性能考虑**：该事件在每次请求时都会触发，应避免执行耗时操作
3. **退出控制**：如果需要阻止后续处理，可以使用 `exit` 或设置 `$_SERVER['WELINE_PARSER_URL'] = false`
4. **全局变量**：可以修改 `$_SERVER` 中的变量来影响后续处理流程
5. **错误处理**：观察者中的异常不会被框架捕获，应自行处理错误
6. **缓存检查**：如果检查到缓存并直接输出，记得使用 `exit` 退出，避免继续执行后续逻辑

#### 系统行为说明

1. **默认观察者**：
   - `Weline_Framework_Router::check_full_page_cache` - 检查全页缓存（sort: 100）
   - `Weline_MediaManager::router_run_before` - 处理媒体资源（sort: 0）

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 如果某个观察者使用 `exit` 退出，后续观察者不会执行

3. **URL 解析控制**：
   - 如果设置 `$_SERVER['WELINE_PARSER_URL'] = false`，将跳过 URL 解析
   - 如果设置 `$_SERVER['WELINE_IS_MEDIA'] = true`，标记为媒体资源请求

#### 故障排查

##### 问题：观察者未执行

**可能原因：**
1. 观察者未正确注册到 `etc/event.xml`
2. 观察者类不存在或路径错误
3. 观察者被禁用（`disabled="true"`）
4. 事件缓存未更新

**解决方法：**
1. 检查 `etc/event.xml` 中的观察者配置
2. 确认观察者类路径和命名空间正确
3. 检查 `disabled` 属性是否为 `false`
4. 运行 `php bin/w event:cache -f` 清除事件缓存

##### 问题：观察者执行顺序不正确

**可能原因：**
1. `sort` 属性设置不当
2. 多个观察者使用相同的 `sort` 值

**解决方法：**
1. 调整 `sort` 属性值，确保执行顺序正确
2. 避免使用相同的 `sort` 值，或使用不同的值来明确顺序

##### 问题：观察者执行后影响后续处理

**可能原因：**
1. 观察者中使用了 `exit` 但没有正确处理
2. 修改了 `$_SERVER` 变量导致后续处理异常

**解决方法：**
1. 确保 `exit` 的使用是预期的
2. 谨慎修改 `$_SERVER` 变量，了解每个变量的作用

## 扩展开发

如果需要扩展应用运行前的功能，可以：

1. **创建新的观察者**：实现 `ObserverInterface` 接口
2. **注册观察者**：在模块的 `etc/event.xml` 中注册
3. **设置执行顺序**：通过 `sort` 属性控制执行顺序
4. **处理异常**：确保观察者中的异常被正确处理
