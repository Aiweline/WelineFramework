# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `App::run_after` 事件及其使用方法。该事件在应用运行完成后触发，允许其他模块在页面输出前对结果进行修改或注入内容。

## 事件列表

### 1. App::run_after - 应用运行后事件

#### 基本信息

- **事件名称**：`App::run_after`
- **事件类型**：应用生命周期事件
- **触发时机**：在 `App::run()` 方法中，路由处理完成后，页面输出前
- **触发位置**：`app/code/Weline/Framework/App.php` 第 342 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`App::run_after` 事件在应用运行完成后触发，此时：
- 路由处理已完成
- 控制器已执行
- 页面内容已生成（存储在 `result` 中）
- 页面尚未输出到浏览器
- 可以修改输出结果

该事件主要用于：
- 修改页面输出内容
- 注入调试工具或开发面板
- 添加统计代码或追踪脚本
- 修改响应头
- 执行日志记录
- 其他需要在输出前执行的操作

#### 触发时机

```php
// app/code/Weline/Framework/App.php
public static function run(): string
{
    // ... 路由处理 ...
    $result = ObjectManager::getInstance(\Weline\Framework\Router\Core::class)->start();
    
    // 创建事件数据对象
    $data = new DataObject(['result' => $result]);
    
    // 触发事件，允许观察者修改 result
    $eventManager->dispatch('App::run_after', $data);
    
    // 获取修改后的结果
    $result = $data->getData('result');
    
    // 输出结果
    if (!CLI) {
        echo($result);
        exit(0);
    }
    return $result;
}
```

#### 使用场景

- **开发工具注入**：在开发模式下注入调试面板、性能分析工具等
- **统计代码注入**：添加 Google Analytics、百度统计等追踪代码
- **内容修改**：修改页面内容，如添加水印、修改链接等
- **响应头设置**：设置自定义响应头
- **日志记录**：记录页面访问日志、性能日志等
- **内容压缩**：压缩 HTML、CSS、JS 输出
- **SEO 优化**：自动添加 SEO 标签或结构化数据

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="App::run_after">
        <observer name="Your_Module::your_observer_name"
                  instance="Your\Module\Observer\YourObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class YourObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        // 获取页面输出结果
        $result = $event->getData('result');
        
        if (empty($result) || !is_string($result)) {
            return;
        }
        
        // 修改结果
        $modifiedResult = $this->modifyResult($result);
        
        // 更新结果
        $event->setData('result', $modifiedResult);
    }
    
    private function modifyResult(string $result): string
    {
        // 在这里修改结果
        return $result;
    }
}
```

#### 事件数据

`App::run_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| result | string | 页面输出内容，可以修改此值来改变最终输出 |

**获取和修改数据**：

```php
// 获取结果
$result = $event->getData('result');

// 修改结果
$event->setData('result', $modifiedResult);
```

#### 使用示例

##### 示例 1：注入开发工具面板

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class DevToolPanelObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        // 只在开发模式下显示
        if (!DEV) {
            return;
        }
        
        $result = $event->getData('result');
        
        if (empty($result) || !is_string($result)) {
            return;
        }
        
        // 检查是否是 HTML 响应
        if (!$this->isHtmlResponse($result)) {
            return;
        }
        
        // 渲染开发工具面板
        $panelHtml = $this->renderPanel();
        
        if (empty($panelHtml)) {
            return;
        }
        
        // 在最后一个 </body> 前注入面板
        $bodyClosePos = strripos($result, '</body>');
        
        if ($bodyClosePos !== false) {
            $before = substr($result, 0, $bodyClosePos);
            $after = substr($result, $bodyClosePos);
            $result = $before . $panelHtml . $after;
        } else {
            // 没找到 </body>，直接追加到末尾
            $result = $result . $panelHtml;
        }
        
        // 更新结果
        $event->setData('result', $result);
    }
    
    private function isHtmlResponse(string $output): bool
    {
        return (stripos($output, '<html') !== false || 
                stripos($output, '<!doctype') !== false ||
                stripos($output, '<body') !== false);
    }
    
    private function renderPanel(): string
    {
        // 渲染开发工具面板 HTML
        return '<div id="dev-tool-panel">...</div>';
    }
}
```

<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class AnalyticsObserver implements ObserverInterface

##### 示例 3：压缩 HTML 输出

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class HtmlCompressObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $result = $event->getData('result');
        
        if (empty($result) || !is_string($result)) {
            return;
        }
        
        // 只在生产环境压缩
        if (!PROD) {
            return;
        }
        
        // 检查是否是 HTML 响应
        if (!$this->isHtmlResponse($result)) {
            return;
        }
        
        // 压缩 HTML
        $compressed = $this->compressHtml($result);
        
        // 更新结果
        $event->setData('result', $compressed);
    }
    
    private function compressHtml(string $html): string
    {
        // 移除 HTML 注释
        $html = preg_replace('/<!--(?!\s*(?:\[if [^\]]+]|<!|>))(?:(?!-->).)*-->/s', '', $html);
        
        // 移除多余空白
        $html = preg_replace('/\s+/', ' ', $html);
        $html = preg_replace('/>\s+</', '><', $html);
        
        // 移除标签间的空白
        $html = preg_replace('/\s+>/', '>', $html);
        $html = preg_replace('/<\s+/', '<', $html);
        
        return trim($html);
    }
    
    private function isHtmlResponse(string $output): bool
    {
        return (stripos($output, '<html') !== false || 
                stripos($output, '<!doctype') !== false);
    }
}
```

##### 示例 4：记录访问日志

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Http\Request;
use Weline\Framework\Manager\ObjectManager;

class AccessLogObserver implements ObserverInterface
{
    private Request $request;

    public function __construct(Request $request)
    {
        $this->request = $request;
    }

    public function execute(Event &$event): void
    {
        $result = $event->getData('result');
        
        // 记录访问日志
        $this->logAccess($result);
    }
    
    private function logAccess(string $result): void
    {
        $logData = [
            'time' => date('Y-m-d H:i:s'),
            'uri' => $_SERVER['REQUEST_URI'] ?? '',
            'method' => $_SERVER['REQUEST_METHOD'] ?? 'GET',
            'ip' => $_SERVER['REMOTE_ADDR'] ?? '',
            'user_agent' => $_SERVER['HTTP_USER_AGENT'] ?? '',
            'response_size' => strlen($result),
            'status_code' => http_response_code() ?: 200,
        ];
        
        // 写入日志文件
        $logFile = BP . 'var/log/access.log';
        $logDir = dirname($logFile);
        
        if (!is_dir($logDir)) {
            mkdir($logDir, 0755, true);
        }
        
        file_put_contents(
            $logFile,
            json_encode($logData, JSON_UNESCAPED_UNICODE) . PHP_EOL,
            FILE_APPEND | LOCK_EX
        );
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **性能考虑**：该事件在每次请求时都会触发，应避免执行耗时操作
3. **结果修改**：必须通过 `$event->setData('result', $modifiedResult)` 来更新结果
4. **HTML 检查**：在修改 HTML 内容前，应先检查是否是 HTML 响应
5. **CLI 模式**：CLI 模式下也会触发此事件，但结果不会输出到浏览器
6. **错误处理**：观察者中的异常不会被框架捕获，应自行处理错误
7. **内存使用**：修改大量内容时注意内存使用，避免内存溢出

#### 系统行为说明

1. **默认观察者**：
   - `Weline_DeveloperWorkspace::dev_tool_panel` - 注入开发工具面板（sort: 0）

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 后执行的观察者可以覆盖先执行的观察者的修改

3. **结果输出**：
   - Web 模式下：结果会通过 `echo` 输出，然后 `exit(0)` 退出
   - CLI 模式下：结果会作为返回值返回

#### 故障排查

##### 问题：观察者未执行

**可能原因：**
1. 观察者未正确注册到 `etc/event.xml`
2. 观察者类不存在或路径错误
3. 观察者被禁用（`disabled="true"`）
4. 事件缓存未更新

**解决方法：**
1. 检查 `etc/event.xml` 中的观察者配置
2. 确认观察者类路径和命名空间正确
3. 检查 `disabled` 属性是否为 `false`
4. 运行 `php bin/w event:cache -f` 清除事件缓存

##### 问题：修改结果未生效

**可能原因：**
1. 没有使用 `$event->setData('result', $modifiedResult)` 更新结果
2. 结果被后续观察者覆盖
3. `sort` 值设置不当，导致执行顺序错误

**解决方法：**
1. 确保使用 `setData` 方法更新结果
2. 检查其他观察者的 `sort` 值，确保执行顺序正确
3. 调整 `sort` 值，确保在正确的时机执行

##### 问题：HTML 注入位置不正确

**可能原因：**
1. 没有正确查找 HTML 标签位置
2. HTML 内容不标准，缺少必要的标签

**解决方法：**
1. 使用 `strripos()` 查找最后一个标签位置
2. 检查 HTML 内容是否包含必要的标签（如 `</body>`、`</head>`）
3. 如果找不到标签，可以考虑追加到末尾
