# 请求前置门禁

事件：`Weline_Framework::App::pre_route_gate`

该事件由 FPM/WLS 共用的 `RequestPipeline` 在 URL 解析和应用层 FPC 之前触发。它只用于必须早于缓存执行的强制门禁和静态资源短路，例如维护模式、受控静态文件发布；观察者不得启动 Session、派发业务 Router/Controller、访问可选业务服务或修改 FPC 正文。

固定顺序：

```text
Context / Worker mandatory policy
→ pre_route_gate
→ URL 解析与规范化
→ early response / FPC
→ run_before
→ lazy Session
→ Router / Controller（一次）
→ run_after
→ Runtime finally / RequestResetter
```

early response 和异常不触发 `run_after`，但 Runtime 的 `finally` 与模块注册的 `RequestResetterInterface` 在所有出口执行，负责请求态、Session、事务和连接清理。
