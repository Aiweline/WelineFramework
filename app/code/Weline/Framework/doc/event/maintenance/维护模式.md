# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Maintenance::maintenance` 事件及其使用方法。该事件在系统进入维护模式时触发，允许其他模块自定义维护模式的响应方式。

## 事件列表

### 1. Weline_Maintenance::maintenance - 维护模式事件

#### 基本信息

- **事件名称**：`Weline_Maintenance::maintenance`
- **事件类型**：系统维护事件
- **触发时机**：在系统进入维护模式时
- **触发位置**：`app/code/Weline/Maintenance/Plugin/Init.php` 第 36 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Weline_Maintenance::maintenance` 事件在系统进入维护模式时触发，允许其他模块添加维护模式白名单URL。可以通过修改事件数据中的 `white_urls` 字段来添加白名单URL，这些URL在维护模式下仍然可以访问。

该事件主要用于：
- 添加维护模式白名单URL
- 自定义维护模式行为
- 维护模式下的特殊处理
- 维护模式日志记录
- API请求统一JSON响应

#### 触发时机

```php
// app/code/Weline/Maintenance/Plugin/Init.php
if (Env::getInstance()->getConfig('maintenance', false)) {
    $event = ObjectManager::getInstance(EventsManager::class);
    $data = new DataObject(['white_urls' => []]);
    $event->dispatch('Weline_Maintenance::maintenance', $data);
    $white_urls = $data->getData('white_urls');
    // 检查当前URL是否在白名单中
}
```

#### 使用场景

- **白名单URL**：添加维护模式下仍可访问的URL
- **API访问**：允许某些API在维护模式下继续工作
- **管理后台**：允许管理员在维护模式下访问后台
- **特殊页面**：允许访问特定的维护说明页面
- **JSON响应**：API请求统一返回JSON格式的维护提示

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Maintenance::maintenance">
        <observer name="Your_Module::maintenance"
                  instance="Your\Module\Observer\MaintenanceObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class MaintenanceObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $whiteUrls = $data->getData('white_urls') ?? [];
        
        // 添加白名单URL
        $whiteUrls[] = '/api/health';
        $whiteUrls[] = '/admin/login';
        $whiteUrls[] = '/maintenance';
        
        $data->setData('white_urls', $whiteUrls);
    }
}
```

#### 事件数据

`Weline_Maintenance::maintenance` 事件传递的数据：

```php
[
    'white_urls' => array,  // 白名单URL数组（可以修改）
]
```

**数据说明**：
- `white_urls`：维护模式下仍可访问的URL数组，观察者可以向此数组添加URL

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **URL格式**：白名单URL应该是路径格式，如 `/admin/login` 或 `/api/health`
3. **性能考虑**：该事件在每次请求时都会触发（如果维护模式开启），应避免执行耗时操作
4. **安全性**：谨慎添加白名单URL，避免在维护模式下暴露敏感功能
5. **API响应**：API请求可以通过观察者统一返回JSON格式的维护提示
