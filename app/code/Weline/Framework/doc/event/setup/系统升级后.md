# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Setup::upgrade_after` 事件及其使用方法。该事件在系统升级完成后触发，允许其他模块执行升级后的操作。

## 事件列表

### 1. Framework_Setup::upgrade_after - 系统升级后事件

#### 基本信息

- **事件名称**：`Framework_Setup::upgrade_after`
- **事件类型**：系统生命周期事件
- **触发时机**：在系统升级命令执行完成后
- **触发位置**：`app/code/Weline/Framework/Setup/Console/Setup/Upgrade.php` 第 56 行和第 253 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_Setup::upgrade_after` 事件在系统升级完成后触发，此时：
- 所有模块的升级脚本已执行完成
- 数据库结构已更新
- 系统配置已更新
- 可以执行升级后的清理、缓存刷新等操作

该事件主要用于：
- 刷新系统缓存
- 重建索引
- 执行数据迁移
- 更新配置文件
- 清理临时文件
- 发送升级通知

#### 触发时机

```php
// app/code/Weline/Framework/Setup/Console/Setup/Upgrade.php
public function execute(array $args = [], array $data = [])
{
    // ... 执行模块升级 ...
    $moduleUpdate = ObjectManager::getInstance(\Weline\Framework\Module\Console\Module\Upgrade::class);
    $moduleUpdate->execute($args, $data);
    
    # 触发系统升级后事件
    /**@var EventsManager $eventsManager */
    $eventsManager = ObjectManager::getInstance(EventsManager::class);
    $eventsManager->dispatch('Framework_Setup::upgrade_after');
}
```

#### 使用场景

- **缓存刷新**：清除所有缓存，确保新代码生效
- **索引重建**：重建数据库索引，优化查询性能
- **数据迁移**：执行数据格式转换或迁移
- **配置更新**：更新系统配置文件
- **通知发送**：发送升级完成通知给管理员
- **日志记录**：记录升级完成日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Setup::upgrade_after">
        <observer name="Your_Module::clear_cache_after_upgrade"
                  instance="Your\Module\Observer\ClearCacheAfterUpgradeObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Cache\CacheInterface;
use Weline\Framework\Manager\ObjectManager;

class ClearCacheAfterUpgradeObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        // 清除所有缓存
        $this->clearAllCache();
    }
    
    private function clearAllCache(): void
    {
        // 清除各种缓存
        $cacheTypes = ['config', 'block_html', 'full_page', 'translate'];
        
        foreach ($cacheTypes as $cacheType) {
            try {
                /** @var CacheInterface $cache */
                $cache = ObjectManager::getInstance(CacheInterface::class . 'Factory');
                $cacheInstance = $cache->create($cacheType);
                $cacheInstance->clean();
            } catch (\Exception $e) {
                // 忽略缓存清除错误
            }
        }
    }
}
```

#### 事件数据

`Framework_Setup::upgrade_after` 事件不传递额外的数据。

#### 使用示例

##### 示例 1：清除所有缓存

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Manager\ObjectManager;

class ClearCacheObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        // 清除配置缓存
        $configCache = ObjectManager::getInstance(\Weline\Framework\Cache\ConfigCache::class);
        $configCache->clean();
        
        // 清除视图缓存
        $viewCache = ObjectManager::getInstance(\Weline\Framework\View\Cache\ViewCache::class);
        $viewCache->clean();
        
        // 清除翻译缓存
        $phraseCache = ObjectManager::getInstance(\Weline\Framework\Phrase\Cache\PhraseCache::class);
        $phraseCache->clean();
    }
}
```

##### 示例 2：重建数据库索引

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Manager\ObjectManager;

class ReindexAfterUpgradeObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        // 重建数据库索引
        $indexer = ObjectManager::getInstance(\Weline\Framework\Database\Console\Index\Reindex::class);
        $indexer->execute([], []);
    }
}
```

##### 示例 3：发送升级通知

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Event\EventsManager;
use Weline\Framework\Manager\ObjectManager;

class SendUpgradeNotificationObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        /** @var EventsManager $eventsManager */
        $eventsManager = ObjectManager::getInstance(EventsManager::class);
        
        // 发送系统消息通知
        $eventsManager->dispatch('Weline_Framework::msg', [
            'data' => [
                'title' => __('系统升级完成'),
                'content' => __('系统已成功升级到最新版本，请检查系统功能是否正常。'),
                'severity' => 'info',
            ]
        ]);
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **性能考虑**：该事件在系统升级时触发，应避免执行耗时操作
3. **错误处理**：观察者中的异常不会被框架捕获，应自行处理错误
4. **幂等性**：确保观察者的操作是幂等的，可以安全地重复执行
5. **依赖关系**：注意观察者之间的依赖关系，合理设置 `sort` 值

#### 系统行为说明

1. **默认观察者**：
   - `Weline_Framework_Database::setup_upgrade_adapter_scan` - 扫描数据库适配器（sort: 100）
   - `Weline_Framework_Extends::setup_upgrade_after` - 扩展升级后处理（sort: 200）

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 建议将缓存清除操作放在前面（较小的 sort 值）

#### 故障排查

##### 问题：观察者未执行

**可能原因：**
1. 观察者未正确注册到 `etc/event.xml`
2. 观察者类不存在或路径错误
3. 观察者被禁用（`disabled="true"`）
4. 事件缓存未更新

**解决方法：**
1. 检查 `etc/event.xml` 中的观察者配置
2. 确认观察者类路径和命名空间正确
3. 检查 `disabled` 属性是否为 `false`
4. 运行 `php bin/w event:cache -f` 清除事件缓存

##### 问题：升级后缓存未清除

**可能原因：**
1. 观察者执行失败
2. 缓存类型不正确
3. 缓存清除逻辑有误

**解决方法：**
1. 检查观察者是否正常执行
2. 确认缓存类型名称正确
3. 检查缓存清除逻辑

## 扩展开发

如果需要扩展系统升级后的功能，可以：

1. **创建新的观察者**：实现 `ObserverInterface` 接口
2. **注册观察者**：在模块的 `etc/event.xml` 中注册
3. **设置执行顺序**：通过 `sort` 属性控制执行顺序
4. **处理异常**：确保观察者中的异常被正确处理
5. **性能优化**：避免在观察者中执行耗时操作
