# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Deploy_Mode_Set::prod_after` 事件及其使用方法。该事件在部署模式切换到生产环境后触发，允许其他模块执行生产环境相关的操作。

## 事件列表

### 1. Framework_Deploy_Mode_Set::prod_after - 部署模式切换到生产环境后事件

#### 基本信息

- **事件名称**：`Framework_Deploy_Mode_Set::prod_after`
- **事件类型**：部署模式事件
- **触发时机**：在部署模式切换到生产环境后
- **触发位置**：`app/code/Weline/Framework/Console/Console/Deploy/Mode/Set.php` 第 146 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_Deploy_Mode_Set::prod_after` 事件在部署模式切换到生产环境后触发，此时：
- 部署模式已设置为 `prod`（生产环境）
- 部署版本信息已更新
- 可以执行生产环境相关的操作

该事件主要用于：
- 清理开发环境文件
- 优化生产环境配置
- 压缩资源文件
- 更新缓存策略
- 设置生产环境标志
- 执行生产环境初始化

#### 触发时机

```php
// app/code/Weline/Framework/Console/Console/Deploy/Mode/Set.php
case 'prod':
    // ... 生产环境设置逻辑 ...
    $eventManager = ObjectManager::getInstance(EventsManager::class);
    $eventData = new \Weline\Framework\DataObject\DataObject([
        'mode' => $type,
        'deploy_version' => $this->getDeployModuleVersion(),
        'printer' => $this->printer
    ]);
    $eventManager->dispatch('Framework_Deploy_Mode_Set::prod_after', $eventData);
    break;
```

#### 使用场景

- **资源优化**：压缩和优化CSS、JS等资源文件
- **缓存清理**：清理开发环境的缓存文件
- **配置更新**：更新生产环境的配置文件
- **性能优化**：执行生产环境的性能优化操作
- **安全设置**：设置生产环境的安全配置
- **监控设置**：设置生产环境的监控和日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Deploy_Mode_Set::prod_after">
        <observer name="Your_Module::deploy_prod_after"
                  instance="Your\Module\Observer\DeployModeSetProdAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class DeployModeSetProdAfterObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $mode = $data->getData('mode');              // 'prod'
        $deployVersion = $data->getData('deploy_version'); // 部署版本
        $printer = $data->getData('printer');        // 控制台输出对象
        
        // 执行生产环境相关操作
        // 例如：清理开发文件、优化资源等
    }
}
```

#### 事件数据

`Framework_Deploy_Mode_Set::prod_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| mode | string | 部署模式，值为 `prod` |
| deploy_version | string | 部署版本号 |
| printer | object | 控制台输出对象，用于输出信息 |

**获取数据**：

```php
$data = $event->getData();
$mode = $data->getData('mode');
$deployVersion = $data->getData('deploy_version');
$printer = $data->getData('printer');
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **异常处理**：如果观察者抛出异常，可能会影响部署模式切换流程
3. **性能考虑**：该事件在部署模式切换时触发，可以执行一些耗时操作（如资源压缩）
4. **错误处理**：建议在观察者中添加异常处理，避免影响其他观察者执行
5. **控制台输出**：可以使用 `printer` 对象输出操作进度和结果
