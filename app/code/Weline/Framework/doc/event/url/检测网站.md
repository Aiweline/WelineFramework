# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Framework_Url::detect_website` 事件及其使用方法。该事件在URL解析时检测网站时触发，允许其他模块自定义网站检测逻辑。

## 事件列表

### 1. Weline_Framework_Url::detect_website - 检测网站事件

#### 基本信息

- **事件名称**：`Weline_Framework_Url::detect_website`
- **事件类型**：URL解析事件
- **触发时机**：在URL解析时检测网站时
- **触发位置**：`app/code/Weline/Framework/Http/Url.php` 第 518 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Weline_Framework_Url::detect_website` 事件在URL解析时检测网站时触发，允许其他模块自定义网站检测逻辑。通过设置 `result` 为 `true` 表示检测成功，系统会使用该网站信息。

该事件主要用于：
- 自定义网站检测逻辑
- 支持多站点URL
- 网站代码验证
- 网站映射

#### 触发时机

```php
// app/code/Weline/Framework/Http/Url.php
$detect_website_data = new DataObject([
    'result' => false,
    'uri' => $uri,
    'code' => $code,
]);
$eventManager->dispatch('Weline_Framework_Url::detect_website', $detect_website_data);
```

#### 使用场景

- **自定义网站检测**：根据URL、域名、Header等自定义网站检测
- **网站代码映射**：将URL中的网站代码映射到系统网站代码
- **网站验证**：验证网站代码是否有效
- **多站点支持**：支持多站点URL结构

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Framework_Url::detect_website">
        <observer name="Your_Module::custom_website_detect"
                  instance="Your\Module\Observer\CustomWebsiteDetectObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CustomWebsiteDetectObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $code = $event->getData('code');
        
        // 自定义网站检测逻辑
        if ($this->isValidWebsite($code)) {
            $event->setData('result', true);
        }
    }
    
    private function isValidWebsite(string $code): bool
    {
        // 验证网站代码
        $validWebsites = ['main', 'shop', 'blog'];
        return in_array($code, $validWebsites);
    }
}
```

#### 事件数据

`Weline_Framework_Url::detect_website` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| result | bool | 检测结果，设置为 true 表示检测成功 |
| uri | string | URI字符串（可以修改） |
| code | string | 网站代码 |

**获取和修改数据**：

```php
// 获取数据
$code = $event->getData('code');
$uri = $event->getData('uri');

// 设置检测结果
$event->setData('result', true);

// 修改URI（如果需要）
$event->setData('uri', $modifiedUri);
```

#### 使用示例

##### 示例 1：根据域名检测网站

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class DomainWebsiteDetectObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $host = $_SERVER['HTTP_HOST'] ?? '';
        
        // 根据域名检测网站
        $domainMap = [
            'shop.example.com' => 'shop',
            'blog.example.com' => 'blog',
            'www.example.com' => 'main',
        ];
        
        if (isset($domainMap[$host])) {
            $event->setData('code', $domainMap[$host]);
            $event->setData('result', true);
        }
    }
}
```

##### 示例 2：网站代码映射

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class WebsiteCodeMappingObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $code = $event->getData('code');
        
        // 网站代码映射
        $websiteMap = [
            'store' => 'shop',
            'news' => 'blog',
        ];
        
        if (isset($websiteMap[$code])) {
            $event->setData('code', $websiteMap[$code]);
            $event->setData('result', true);
        }
    }
}
```

#### 注意事项

1. **result设置**：必须设置 `result` 为 `true` 才能表示检测成功
2. **URI修改**：可以修改URI来移除网站代码前缀
3. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
4. **性能考虑**：该事件在每次URL解析时都可能触发，应避免执行耗时操作

#### 系统行为说明

1. **默认行为**：
   - 默认 `result` 为 `false`
   - 如果所有观察者都未设置 `result` 为 `true`，网站检测失败

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 第一个设置 `result` 为 `true` 的观察者会生效
