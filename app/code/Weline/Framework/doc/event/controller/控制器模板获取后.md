# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Framework::Controller::fetch_file_after` 事件及其使用方法。该事件在控制器获取模板文件后触发，允许其他模块修改渲染后的模板内容。

## 事件列表

### 1. Weline_Framework::Controller::fetch_file_after - 控制器模板获取后事件

#### 基本信息

- **事件名称**：`Weline_Framework::Controller::fetch_file_after`
- **事件类型**：控制器模板渲染事件
- **触发时机**：在控制器获取模板文件后，返回渲染内容之前
- **触发位置**：`app/code/Weline/Framework/Controller/PcController.php` 第 292 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Weline_Framework::Controller::fetch_file_after` 事件在控制器获取模板文件后触发，允许其他模块修改渲染后的模板内容。通过修改事件数据中的 `content` 字段，可以改变最终返回的模板内容。

该事件主要用于：
- 模板内容后处理
- 内容注入和修改
- 模板内容验证
- 性能优化

#### 触发时机

```php
// app/code/Weline/Framework/Controller/PcController.php
$content = $this->getTemplate()->fetch($fileName);
// 触发Weline_Framework::Controller::fetch_file_after事件
$eventData->setData('content', $content);
$this->getEventManager()->dispatch('Weline_Framework::Controller::fetch_file_after', $eventData);
return $eventData->getData('content');
```

#### 使用场景

- **内容注入**：向渲染后的模板内容注入额外的 HTML、CSS 或 JavaScript
- **内容修改**：修改渲染后的模板内容
- **内容验证**：验证模板内容的合法性
- **性能优化**：对模板内容进行优化处理（如压缩、缓存等）

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Framework::Controller::fetch_file_after">
        <observer name="Your_Module::controller_fetch_content"
                  instance="Your\Module\Observer\ControllerFetchFileAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\DataObject\DataObject;

class ControllerFetchFileAfterObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        /** @var DataObject $eventData */
        $eventData = $event->getData('data');
        if (!$eventData instanceof DataObject) {
            return;
        }

        $content = $eventData->getData('content');
        $fileName = $eventData->getData('fileName');
        
        // 处理模板内容
        // 例如：注入额外的内容
        $content = $this->injectContent($content);
        
        // 更新内容
        $eventData->setData('content', $content);
    }
    
    private function injectContent(string $content): string
    {
        // 实现内容注入逻辑
        return $content;
    }
}
```

#### 事件数据

`Weline_Framework::Controller::fetch_file_after` 事件传递的数据：

```php
[
    'fileName' => string,        // 模板文件路径
    'content' => string,         // 渲染后的模板内容（可以修改）
    'controller' => object,      // 控制器实例（PcController）
    'layoutType' => string|null, // 布局类型（从控制器的 $layoutType 属性获取）
]
```

**数据说明**：
- `fileName`：实际使用的模板文件路径
- `content`：渲染后的模板内容，观察者可以修改此值来改变最终返回的内容
- `controller`：控制器实例，可以用于访问控制器的属性和方法
- `layoutType`：布局类型，从控制器的 `$layoutType` 属性获取

#### 使用示例

##### 示例 1：注入额外的 JavaScript 代码

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\DataObject\DataObject;

class InjectJavaScriptObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        /** @var DataObject $eventData */
        $eventData = $event->getData('data');
        $content = $eventData->getData('content');
        $fileName = $eventData->getData('fileName');
        
        // 只在特定页面注入
        if (str_contains($fileName, 'templates/Index/index.phtml')) {
            $jsCode = '<script>
                console.log("Page loaded: ' . htmlspecialchars($fileName) . '");
                // 其他 JavaScript 代码
            </script>';
            
            // 在 </body> 标签前注入
            if (preg_match('/(<\/body>)/i', $content)) {
                $content = preg_replace('/(<\/body>)/i', $jsCode . "\n$1", $content, 1);
            } else {
                $content .= $jsCode;
            }
            
            $eventData->setData('content', $content);
        }
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **内容修改**：修改 `content` 后必须调用 `setData('content', $content)` 才能生效
3. **性能考虑**：该事件在每次渲染模板时都会触发，应避免执行耗时操作
4. **内容格式**：修改内容时，应确保保持 HTML 结构的完整性
5. **内存使用**：处理大量内容时，注意内存使用情况

#### 系统行为说明

1. **默认行为**：
   - 控制器直接返回渲染后的模板内容
   - 观察者可以修改返回的内容

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 后执行的观察者可以修改先执行的观察者的结果

3. **内容更新**：
   - 只有调用 `setData('content', $content)` 才能更新最终返回的内容
   - 多个观察者可以依次修改内容
