# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Framework::Controller::fetch_file_before` 事件及其使用方法。该事件在控制器获取模板文件前触发，允许其他模块修改模板文件路径或执行预处理操作。

## 事件列表

### 1. Weline_Framework::Controller::fetch_file_before - 控制器模板获取前事件

#### 基本信息

- **事件名称**：`Weline_Framework::Controller::fetch_file_before`
- **事件类型**：控制器模板渲染事件
- **触发时机**：在控制器获取模板文件前，渲染模板内容之前
- **触发位置**：`app/code/Weline/Framework/Controller/PcController.php` 第 286 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Weline_Framework::Controller::fetch_file_before` 事件在控制器获取模板文件前触发，允许其他模块修改模板文件路径或执行预处理操作。通过修改事件数据中的 `fileName` 字段，可以改变要加载的模板文件。

该事件主要用于：
- 根据布局类型自动加载主题布局模板
- 模板文件路径动态替换
- 模板文件预处理
- 多主题支持

#### 触发时机

```php
// app/code/Weline/Framework/Controller/PcController.php
$eventData = new DataObject([
    'fileName' => $fileName, 
    'content' => '',
    'controller' => $this,
    'layoutType' => $this->layoutType ?? null
]);
$this->getEventManager()->dispatch('Weline_Framework::Controller::fetch_file_before', $eventData);
$fileName = $eventData->getData('fileName');
```

#### 使用场景

- **主题布局自动加载**：根据控制器的 `layoutType` 属性，自动加载对应的主题布局模板
- **模板文件替换**：在特定条件下替换模板文件
- **模板文件预处理**：在加载模板文件前进行预处理
- **多主题支持**：根据主题动态切换模板文件

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Framework::Controller::fetch_file_before">
        <observer name="Your_Module::controller_fetch_layout"
                  instance="Your\Module\Observer\ControllerFetchFileBeforeObserver"
                  disabled="false"
                  shared="true"
                  sort="10"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\DataObject\DataObject;

class ControllerFetchFileBeforeObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        /** @var DataObject $eventData */
        $eventData = $event->getData('data');
        if (!$eventData instanceof DataObject) {
            return;
        }

        $controller = $eventData->getData('controller');
        $layoutType = $eventData->getData('layoutType');
        $fileName = $eventData->getData('fileName');
        
        // 如果没有指定 layoutType，不处理
        if (empty($layoutType)) {
            return;
        }
        
        // 根据 layoutType 加载对应的布局模板
        // ...
        
        // 更新模板文件路径
        $eventData->setData('fileName', $newFileName);
    }
}
```

#### 事件数据

`Weline_Framework::Controller::fetch_file_before` 事件传递的数据：

```php
[
    'fileName' => string,        // 模板文件路径（可以修改）
    'content' => string,         // 模板内容（初始为空字符串）
    'controller' => object,      // 控制器实例（PcController）
    'layoutType' => string|null, // 布局类型（从控制器的 $layoutType 属性获取）
]
```

**数据说明**：
- `fileName`：模板文件的路径，观察者可以修改此值来改变要加载的模板文件
- `content`：模板内容，初始为空字符串，在 `fetch_file_after` 事件中才会被填充
- `controller`：控制器实例，可以用于访问控制器的属性和方法
- `layoutType`：布局类型，从控制器的 `$layoutType` 属性获取，如果控制器未设置则为 `null`

#### 使用示例

##### 示例 1：根据布局类型加载主题布局

```php
<?php

namespace Weline\Theme\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\DataObject\DataObject;
use Weline\Theme\Helper\ConfigLoader;
use Weline\Theme\Model\WelineTheme;

class ControllerFetchFileBefore implements ObserverInterface
{
    private WelineTheme $welineTheme;

    public function __construct(WelineTheme $welineTheme)
    {
        $this->welineTheme = $welineTheme;
    }

    public function execute(Event &$event): void
    {
        /** @var DataObject $eventData */
        $eventData = $event->getData('data');
        $layoutType = $eventData->getData('layoutType');
        
        if (empty($layoutType)) {
            return;
        }

        // 获取当前主题
        $theme = $this->welineTheme->getActiveTheme();
        $area = 'frontend'; // 根据实际情况判断
        
        // 从主题配置中获取布局选项
        $layoutConfig = ConfigLoader::getLayoutConfig($theme, $area);
        $layoutOption = $layoutConfig[$layoutType] ?? 'default';
        
        // 构建布局模板路径
        $layoutPath = "theme/{$area}/layouts/{$layoutType}/{$layoutOption}.phtml";
        
        // 将原模板路径保存为变量
        $eventData->setData('contentTemplate', $eventData->getData('fileName'));
        $eventData->setData('fileName', $layoutPath);
    }
}
```

##### 示例 2：条件性模板替换

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\DataObject\DataObject;

class ConditionalTemplateObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        /** @var DataObject $eventData */
        $eventData = $event->getData('data');
        $fileName = $eventData->getData('fileName');
        $controller = $eventData->getData('controller');
        
        // 根据条件替换模板
        if (str_contains($fileName, 'templates/Index/index.phtml')) {
            // 检查是否启用新模板
            if ($this->isNewTemplateEnabled()) {
                $newFileName = str_replace('index.phtml', 'index_v2.phtml', $fileName);
                if (file_exists($newFileName)) {
                    $eventData->setData('fileName', $newFileName);
                }
            }
        }
    }
    
    private function isNewTemplateEnabled(): bool
    {
        // 实现检查逻辑
        return true;
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **文件存在性**：修改 `fileName` 后，应确保新文件存在，否则可能导致模板加载失败
3. **性能考虑**：该事件在每次获取模板文件时都会触发，应避免执行耗时操作
4. **路径格式**：修改 `fileName` 时，应使用模块路径格式（如 `Weline_Module::path/to/file.phtml`）或相对路径
5. **控制器属性**：可以通过 `controller` 访问控制器的 `$layoutType` 等属性

#### 系统行为说明

1. **默认行为**：
   - 控制器直接使用指定的模板文件路径
   - 观察者可以修改模板文件路径

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 后执行的观察者可以修改先执行的观察者的结果

3. **路径更新**：
   - 只有调用 `setData('fileName', $newFileName)` 才能更新最终使用的模板文件路径
   - 多个观察者可以依次修改路径
