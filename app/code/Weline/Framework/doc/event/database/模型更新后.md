# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Database::model_update_after` 事件及其使用方法。该事件在数据库模型更新后触发，允许其他模块在更新后执行操作。

## 事件列表

### 1. Framework_Database::model_update_after - 模型更新后事件

#### 基本信息

- **事件名称**：`Framework_Database::model_update_after`
- **事件类型**：数据库事件
- **触发时机**：在数据库模型更新后
- **触发位置**：`app/code/Weline/Framework/Database/Model/ModelManager.php` 第 63 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_Database::model_update_after` 事件在数据库模型更新后触发，允许其他模块在更新后执行操作。

该事件主要用于：
- 更新后处理
- 缓存刷新
- 更新后验证
- 日志记录

#### 触发时机

```php
// app/code/Weline/Framework/Database/Model/ModelManager.php
$data = new DataObject(['model' => $model, 'type' => $type, 'object' => $this, 'module' => $module]);
$this->getEvenManager()->dispatch('Framework_Database::model_update_before', $data);
$this->setupModel($model, $type, $context);
$this->getEvenManager()->dispatch('Framework_Database::model_update_after', $data);
```

#### 使用场景

- **更新后处理**：执行模型更新后的处理逻辑
- **缓存刷新**：刷新模型相关缓存
- **更新后验证**：验证模型更新结果
- **日志记录**：记录模型更新日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Database::model_update_after">
        <observer name="Your_Module::model_update_after_handler"
                  instance="Your\Module\Observer\ModelUpdateAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ModelUpdateAfterObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $model = $event->getData('model');
        $type = $event->getData('type');
        $module = $event->getData('module');
        
        // 执行更新后操作
        // ...
    }
}
```

#### 事件数据

`Framework_Database::model_update_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| model | AbstractModel | 模型实例 |
| type | string | 更新类型 |
| object | ModelManager | ModelManager实例 |
| module | string | 模块名 |

**获取数据**：

```php
$model = $event->getData('model');
$type = $event->getData('type');
$object = $event->getData('object');
$module = $event->getData('module');
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **性能考虑**：该事件在模型更新时触发，应避免执行耗时操作
