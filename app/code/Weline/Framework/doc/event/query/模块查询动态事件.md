# {Module}::query - 模块查询动态事件

## 事件说明

动态事件：任意 `Module_Name::query` 触发时，由 `ModuleQueryDispatcher` 自动路由到 `FrameworkQueryService::execute()`，由 `QueryProviderRegistry` 中已注册的查询器处理。

**各模块通过 extends 注册 QueryProviderInterface 实现即可**，无需监听任何事件。

## 约定

- 事件名格式：`{Module}::query`，例如 `Weline_Widget::query`、`Weline_Order::query`
- provider 转换规则：取模块名最后一段并转小写
  - `Weline_Widget` -> `widget`
  - `Weline_Order` -> `order`
  - `Vendor_ModuleName` -> `modulename`

## 调用方（dispatch）

```php
$eventData = [
    'data' => [
        'operation' => 'getAvailableList',
        'params' => ['page_type' => 'homepage', 'filter_options' => null],
        'area' => 'backend',
    ],
];
$this->getEventsManager()->dispatch('Weline_Widget::query', $eventData);
$result = $eventData['data']['result'] ?? null;
```

## 模块实现方

在模块目录下 `extends/module/Weline_Framework/Query/` 放置实现 `QueryProviderInterface` 的类：

```php
class WidgetQueryProvider implements QueryProviderInterface
{
    public function getProviderName(): string { return 'widget'; }
    public function execute(string $operation, array $params = []): mixed { /* ... */ }
    public function getDescriptor(): array { /* ... */ }
}
```

执行 `php bin/w extends:rebuild` 使框架扫描到新查询器。

## 数据格式

传入 `data`：

- `operation` (必填)：操作名
- `params` (可选)：参数数组
- `area` (可选)：frontend|backend，默认 backend

返回（写入 event data）：

- `result`：执行结果
- `error`：错误信息（异常时）

## 使用说明查询

```php
$result = $queryService->execute('framework', 'introspect', ['what' => 'providers']);
```
