# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `{table_name}_model_load_before` 动态事件及其使用方法。该事件在模型加载数据前触发，允许其他模块在加载前执行操作。

## 事件列表

### 1. {table_name}_model_load_before - 模型加载前事件（动态事件）

#### 基本信息

- **事件名称**：`{table_name}_model_load_before`（动态事件模式）
- **事件类型**：模型生命周期事件（动态）
- **触发时机**：在模型加载数据前
- **触发位置**：`app/code/Weline/Framework/Database/AbstractModel.php` 第 507 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`
- **动态说明**：`{table_name}` 是动态占位符，表示数据库表名

#### 功能说明

`{table_name}_model_load_before` 是一个动态事件，其中 `{table_name}` 是动态占位符，表示数据库表名。该事件在模型加载数据前触发，允许其他模块在加载前执行操作。

**动态事件说明**：
- 实际触发的事件名会根据表名动态生成
- 例如：表名为 `user` 时，实际事件名为 `user_model_load_before`
- 例如：表名为 `admin_user` 时，实际事件名为 `admin_user_model_load_before`

该事件主要用于：
- 加载前数据验证
- 加载前权限检查
- 加载前数据预处理
- 加载前日志记录

#### 触发时机

```php
// app/code/Weline/Framework/Database/AbstractModel.php
$eventData = ['model' => $this, 'field_or_pk_value' => $field_or_pk_value, 'value' => $value];
$this->getEvenManager()->dispatch($this->getOriginTableName() . '_model_load_before', $eventData);
```

#### 使用场景

- **数据验证**：在加载前验证数据有效性
- **权限检查**：检查用户是否有权限加载该数据
- **数据预处理**：对加载条件进行预处理
- **日志记录**：记录数据加载日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <!-- 监听特定表的加载前事件 -->
    <event name="user_model_load_before">
        <observer name="Your_Module::user_load_before_handler"
                  instance="Your\Module\Observer\UserLoadBeforeObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class UserLoadBeforeObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $model = $event->getData('model');
        $fieldOrPkValue = $event->getData('field_or_pk_value');
        $value = $event->getData('value');
        
        // 执行加载前操作
        // 例如：权限检查、数据验证等
    }
}
```

#### 事件数据

`{table_name}_model_load_before` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| model | AbstractModel | 模型实例 |
| field_or_pk_value | string | 字段名或主键值 |
| value | mixed | 字段值（如果指定了字段名） |

**获取数据**：

```php
$model = $event->getData('model');
$fieldOrPkValue = $event->getData('field_or_pk_value');
$value = $event->getData('value');
```

#### 注意事项

1. **动态事件匹配**：系统会自动将实际事件名（如 `user_model_load_before`）与动态事件模式（`{table_name}_model_load_before`）进行匹配
2. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
3. **异常处理**：如果观察者抛出异常，可能会阻止模型加载
4. **性能考虑**：该事件在每次模型加载时都会触发，应避免执行耗时操作

#### 系统行为说明

1. **动态事件匹配**：
   - 当触发 `user_model_load_before` 时，系统会匹配模式 `{table_name}_model_load_before`
   - 匹配成功后，使用该模式的规约和文档信息

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 如果某个观察者抛出异常，后续观察者不会执行
