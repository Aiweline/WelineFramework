# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `{table_name}_model_delete_before` 动态事件及其使用方法。该事件在模型删除数据前触发，允许其他模块在删除前执行操作。

## 事件列表

### 1. {table_name}_model_delete_before - 模型删除前事件（动态事件）

#### 基本信息

- **事件名称**：`{table_name}_model_delete_before`（动态事件模式）
- **事件类型**：模型生命周期事件（动态）
- **触发时机**：在模型删除数据前
- **触发位置**：`app/code/Weline/Framework/Database/AbstractModel.php` 第 879 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`
- **动态说明**：`{table_name}` 是动态占位符，表示数据库表名

#### 功能说明

`{table_name}_model_delete_before` 是一个动态事件，其中 `{table_name}` 是动态占位符，表示数据库表名。该事件在模型删除数据前触发，允许其他模块在删除前执行操作。

**动态事件说明**：
- 实际触发的事件名会根据表名动态生成
- 例如：表名为 `user` 时，实际事件名为 `user_model_delete_before`

该事件主要用于：
- 删除前权限检查
- 删除前数据备份
- 删除前关联数据检查
- 删除前日志记录

#### 触发时机

```php
// app/code/Weline/Framework/Database/AbstractModel.php
$eventData = new DataObject(['model' => &$this]);
$this->getEvenManager()->dispatch($this->processTable() . '_model_delete_before', $eventData);
```

#### 使用场景

- **权限检查**：检查用户是否有权限删除该数据
- **数据备份**：在删除前备份数据
- **关联检查**：检查是否有关联数据需要处理
- **日志记录**：记录数据删除日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <!-- 监听特定表的删除前事件 -->
    <event name="user_model_delete_before">
        <observer name="Your_Module::user_delete_before_handler"
                  instance="Your\Module\Observer\UserDeleteBeforeObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class UserDeleteBeforeObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $model = $event->getData('model');
        
        // 执行删除前操作
        // 例如：权限检查、数据备份等
    }
}
```

#### 事件数据

`{table_name}_model_delete_before` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| model | AbstractModel | 模型实例（引用传递，可以修改） |

#### 注意事项

1. **动态事件匹配**：系统会自动将实际事件名与动态事件模式进行匹配
2. **执行顺序**：观察者按照 `sort` 属性排序执行
3. **异常处理**：如果观察者抛出异常，可能会阻止模型删除
4. **性能考虑**：该事件在每次模型删除时都会触发，应避免执行耗时操作
