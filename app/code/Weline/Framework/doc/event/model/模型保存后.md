# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `{model_class}_model_save_after` 动态事件及其使用方法。该事件在模型保存数据后触发，允许其他模块在保存后执行操作。

## 事件列表

### 1. {model_class}_model_save_after - 模型保存后事件（动态事件）

#### 基本信息

- **事件名称**：`{model_class}_model_save_after`（动态事件模式）
- **事件类型**：模型生命周期事件（动态）
- **触发时机**：在模型保存数据后
- **触发位置**：`app/code/Weline/Framework/Database/AbstractModel.php` 第 689 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`
- **动态说明**：`{model_class}` 是动态占位符，表示模型类名（下划线格式）

#### 功能说明

`{model_class}_model_save_after` 是一个动态事件，其中 `{model_class}` 是动态占位符，表示模型类名（命名空间中的反斜杠替换为下划线）。该事件在模型保存数据后触发，允许其他模块在保存后执行操作。

**动态事件说明**：
- 实际触发的事件名会根据模型类名动态生成
- 例如：类名为 `Weline\Admin\Model\User` 时，实际事件名为 `Weline_Admin_Model_User_model_save_after`

该事件主要用于：
- 保存后数据后处理
- 保存后缓存更新
- 保存后通知发送
- 保存后日志记录

#### 触发时机

```php
// app/code/Weline/Framework/Database/AbstractModel.php
$this->getEvenManager()->dispatch($model_event_name . '_model_save_after', $evenData);
```

#### 使用场景

- **数据后处理**：对保存的数据进行后处理
- **缓存更新**：更新相关缓存
- **通知发送**：发送保存成功通知
- **日志记录**：记录数据保存日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <!-- 监听特定模型的保存后事件 -->
    <event name="Weline_Admin_Model_User_model_save_after">
        <observer name="Your_Module::user_save_after_handler"
                  instance="Your\Module\Observer\UserSaveAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class UserSaveAfterObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $model = $event->getData('model');
        
        // 执行保存后操作
        // 例如：缓存更新、通知发送等
    }
}
```

#### 事件数据

`{model_class}_model_save_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| model | AbstractModel | 模型实例（已保存数据） |

#### 注意事项

1. **动态事件匹配**：系统会自动将实际事件名与动态事件模式进行匹配
2. **执行顺序**：观察者按照 `sort` 属性排序执行
3. **性能考虑**：该事件在每次模型保存时都会触发，应避免执行耗时操作
