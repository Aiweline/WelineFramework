# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `{table_name}_model_load_after` 动态事件及其使用方法。该事件在模型加载数据后触发，允许其他模块在加载后执行操作。

## 事件列表

### 1. {table_name}_model_load_after - 模型加载后事件（动态事件）

#### 基本信息

- **事件名称**：`{table_name}_model_load_after`（动态事件模式）
- **事件类型**：模型生命周期事件（动态）
- **触发时机**：在模型加载数据后
- **触发位置**：`app/code/Weline/Framework/Database/AbstractModel.php` 第 519 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`
- **动态说明**：`{table_name}` 是动态占位符，表示数据库表名

#### 功能说明

`{table_name}_model_load_after` 是一个动态事件，其中 `{table_name}` 是动态占位符，表示数据库表名。该事件在模型加载数据后触发，允许其他模块在加载后执行操作。

**动态事件说明**：
- 实际触发的事件名会根据表名动态生成
- 例如：表名为 `user` 时，实际事件名为 `user_model_load_after`
- 例如：表名为 `admin_user` 时，实际事件名为 `admin_user_model_load_after`

该事件主要用于：
- 加载后数据后处理
- 加载后数据转换
- 加载后缓存处理
- 加载后日志记录

#### 触发时机

```php
// app/code/Weline/Framework/Database/AbstractModel.php
$this->getEvenManager()->dispatch($this->getOriginTableName() . '_model_load_after', $eventData);
```

#### 使用场景

- **数据后处理**：对加载的数据进行后处理
- **数据转换**：转换数据格式
- **缓存处理**：更新缓存
- **日志记录**：记录数据加载日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <!-- 监听特定表的加载后事件 -->
    <event name="user_model_load_after">
        <observer name="Your_Module::user_load_after_handler"
                  instance="Your\Module\Observer\UserLoadAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class UserLoadAfterObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $model = $event->getData('model');
        
        // 执行加载后操作
        // 例如：数据转换、缓存更新等
    }
}
```

#### 事件数据

`{table_name}_model_load_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| model | AbstractModel | 模型实例（已加载数据） |
| field_or_pk_value | string | 字段名或主键值 |
| value | mixed | 字段值（如果指定了字段名） |

#### 注意事项

1. **动态事件匹配**：系统会自动将实际事件名与动态事件模式进行匹配
2. **执行顺序**：观察者按照 `sort` 属性排序执行
3. **性能考虑**：该事件在每次模型加载时都会触发，应避免执行耗时操作
