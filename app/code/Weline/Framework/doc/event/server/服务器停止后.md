# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Server::stop_after` 事件及其使用方法。该事件在服务器停止后触发，允许其他模块执行停止后的清理操作。

## 事件列表

### 1. Framework_Server::stop_after - 服务器停止后事件

#### 基本信息

- **事件名称**：`Framework_Server::stop_after`
- **事件类型**：服务器生命周期事件
- **触发时机**：在服务器停止后
- **触发位置**：`app/code/Weline/Framework/Console/Console/Server/Stop.php` 第 131 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_Server::stop_after` 事件在服务器停止后触发，此时：
- 服务器进程已停止
- 服务器已释放资源
- 可以执行停止后的清理操作

该事件主要用于：
- 记录服务器停止日志
- 清理临时文件
- 释放资源
- 保存状态
- 发送停止通知

#### 触发时机

```php
// app/code/Weline/Framework/Console/Console/Server/Stop.php
$eventData = [
    'pid' => $pid,
    'stop_time' => time(),
    'force' => $force
];
$eventManager->dispatch('Framework_Server::stop_after', $eventData);
```

#### 使用场景

- **停止日志**：记录服务器停止日志
- **清理临时文件**：清理服务器运行产生的临时文件
- **释放资源**：释放服务器占用的资源
- **保存状态**：保存服务器状态信息
- **通知发送**：发送服务器停止通知

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Server::stop_after">
        <observer name="Your_Module::log_server_stop"
                  instance="Your\Module\Observer\LogServerStopObserver"
                  disabled="false"
                  shared="true"
                  sort="10"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class LogServerStopObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $pid = $event->getData('pid');
        $stopTime = date('Y-m-d H:i:s', $event->getData('stop_time'));
        
        // 记录停止日志
        error_log("[{$stopTime}] Server stopped - PID: {$pid}");
    }
}
```

#### 事件数据

`Framework_Server::stop_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| pid | int | 服务器进程 ID |
| stop_time | int | 停止时间戳 |
| force | bool | 是否强制停止 |

**获取数据**：

```php
$pid = $event->getData('pid');
$stopTime = $event->getData('stop_time');
$force = $event->getData('force');
```

#### 使用示例

##### 示例 1：记录停止日志

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class LogServerStopObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $pid = $event->getData('pid');
        $stopTime = date('Y-m-d H:i:s', $event->getData('stop_time'));
        $force = $event->getData('force') ? ' (forced)' : '';
        
        $logMessage = sprintf(
            "[%s] Server stopped - PID: %d%s",
            $stopTime,
            $pid,
            $force
        );
        
        error_log($logMessage);
    }
}
```

##### 示例 2：清理临时文件

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CleanupTempFilesObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        // 清理临时文件
        $tempDir = BP . 'var/tmp/';
        
        if (is_dir($tempDir)) {
            $files = glob($tempDir . '*');
            foreach ($files as $file) {
                if (is_file($file)) {
                    // 删除超过 24 小时的临时文件
                    if (time() - filemtime($file) > 86400) {
                        unlink($file);
                    }
                }
            }
        }
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **性能考虑**：该事件在服务器停止时触发，应避免执行耗时操作
3. **错误处理**：观察者中的异常不会被框架捕获，应自行处理错误
4. **资源释放**：确保及时释放占用的资源

#### 系统行为说明

1. **默认观察者**：
   - `Weline_Framework_Server::stop_logger` - 记录停止日志（sort: 10）

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 建议将日志记录放在前面（较小的 sort 值）

4. **处理异常**：确保观察者中的异常被正确处理
