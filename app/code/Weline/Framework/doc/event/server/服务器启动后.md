# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Server::start_after` 事件及其使用方法。该事件在服务器启动完成后触发，允许其他模块执行启动后的操作。

## 事件列表

### 1. Framework_Server::start_after - 服务器启动后事件

#### 基本信息

- **事件名称**：`Framework_Server::start_after`
- **事件类型**：服务器生命周期事件
- **触发时机**：在服务器启动完成后
- **触发位置**：`app/code/Weline/Framework/Console/Console/Server/Start.php` 第 242 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_Server::start_after` 事件在服务器启动完成后触发，此时：
- 服务器进程已启动
- 服务器已监听指定端口
- 可以执行启动后的初始化操作

该事件主要用于：
- 记录服务器启动日志
- 初始化服务
- 预热缓存
- 健康检查
- 发送启动通知

#### 触发时机

```php
// app/code/Weline/Framework/Console/Console/Server/Start.php
$eventData = [
    'pid' => $pid,
    'host' => $host,
    'port' => $port,
    'backend' => $backend,
    'start_time' => time(),
    'force' => $force
];
$eventManager->dispatch('Framework_Server::start_after', $eventData);
```

#### 使用场景

- **启动日志**：记录服务器启动日志
- **服务初始化**：初始化各种服务
- **缓存预热**：预热常用缓存
- **健康检查**：执行健康检查
- **通知发送**：发送服务器启动通知

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Server::start_after">
        <observer name="Your_Module::log_server_start"
                  instance="Your\Module\Observer\LogServerStartObserver"
                  disabled="false"
                  shared="true"
                  sort="10"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class LogServerStartObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $pid = $event->getData('pid');
        $host = $event->getData('host');
        $port = $event->getData('port');
        
        // 记录启动日志
        error_log("Server started: PID={$pid}, Host={$host}, Port={$port}");
    }
}
```

#### 事件数据

`Framework_Server::start_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| pid | int | 服务器进程 ID |
| host | string | 服务器监听地址 |
| port | int | 服务器监听端口 |
| backend | bool | 是否后端模式 |
| start_time | int | 启动时间戳 |
| force | bool | 是否强制启动 |

**获取数据**：

```php
$pid = $event->getData('pid');
$host = $event->getData('host');
$port = $event->getData('port');
$backend = $event->getData('backend');
$startTime = $event->getData('start_time');
$force = $event->getData('force');
```

#### 使用示例

##### 示例 1：记录启动日志

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class LogServerStartObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $pid = $event->getData('pid');
        $host = $event->getData('host');
        $port = $event->getData('port');
        $backend = $event->getData('backend') ? 'backend' : 'frontend';
        $startTime = date('Y-m-d H:i:s', $event->getData('start_time'));
        
        $logMessage = sprintf(
            "[%s] Server started - PID: %d, Host: %s, Port: %d, Mode: %s",
            $startTime,
            $pid,
            $host,
            $port,
            $backend
        );
        
        error_log($logMessage);
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **性能考虑**：该事件在服务器启动时触发，应避免执行耗时操作
3. **错误处理**：观察者中的异常不会被框架捕获，应自行处理错误
4. **异步操作**：耗时操作建议使用异步方式执行

#### 系统行为说明

1. **默认观察者**：
   - `Weline_Framework_Server::start_logger` - 记录启动日志（sort: 10）

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 建议将日志记录放在前面（较小的 sort 值）
