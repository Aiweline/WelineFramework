# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Framework::msg` 事件及其使用方法。该事件用于跨模块发送系统消息通知，由 Weline_Admin 模块监听并写入消息中心。

## 事件列表

### 1. Weline_Framework::msg - 系统消息通知事件

#### 基本信息

- **事件名称**：`Weline_Framework::msg`
- **事件类型**：系统消息通知
- **观察者类**：`Weline\Admin\Observer\SystemNotificationObserver`
- **数据模型**：`Weline\Admin\Model\System\SystemNotification`
- **配置文件**：`app/code/Weline/Admin/etc/event.xml`

#### 功能说明

用于跨模块发送系统消息通知，由 Weline_Admin 模块监听并写入消息中心。任何模块都可以触发此事件来发送系统消息给管理员。

#### 使用场景

- 订单状态变更通知
- 系统警告和错误提示
- 用户操作通知
- 系统维护通知
- 数据同步完成通知
- 审核流程通知
- 库存预警通知

#### 使用方法

##### 基本用法

```php
use Weline\Framework\Event\EventsManager;
use Weline\Framework\Manager\ObjectManager;

// 获取事件管理器实例
/** @var EventsManager $eventsManager */
$eventsManager = ObjectManager::getInstance(EventsManager::class);

// 发送系统消息
$eventsManager->dispatch('Weline_Framework::msg', [
    'data' => [
        'title'   => __('系统通知'),              // 必填：消息标题
        'content' => __('这是一条系统消息内容'),   // 必填：消息内容
        'is_read' => false,                       // 可选：是否已读，默认 false
        'is_icon' => 1,                          // 可选：使用图标头像，1=使用，0=不使用
        'is_img'  => 0,                          // 可选：使用图片头像，1=使用，0=不使用
        'avatar'  => 'ri-notification-line',     // 可选：头像内容（图标名或图片路径）
    ]
]);
```

#### 数据格式说明

| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| title | string | 是 | 消息标题，最大长度120字符，建议使用 __() 进行国际化 | - |
| content | string | 是 | 消息内容，支持多行文本，建议使用 __() 进行国际化 | - |
| is_read | bool | 否 | 是否已读，false=未读，true=已读 | false |
| is_icon | int | 否 | 是否使用图标头像，1=使用，0=不使用 | 1 |
| is_img | int | 否 | 是否使用图片头像，1=使用，0=不使用 | 0 |
| avatar | string | 否 | 头像内容，图标名称（如 'ri-notification-line'）或图片路径 | 'ri-notification-line' |

#### 使用示例

##### 示例 1：使用图标头像

```php
$eventsManager->dispatch('Weline_Framework::msg', [
    'data' => [
        'title' => __('订单通知'),
        'content' => __('您有新的订单需要处理'),
        'is_icon' => 1,
        'is_img' => 0,
        'avatar' => 'ri-shopping-cart-line'  // RemixIcon 图标名称
    ]
]);
```

##### 示例 2：使用图片头像

```php
$eventsManager->dispatch('Weline_Framework::msg', [
    'data' => [
        'title' => __('用户消息'),
        'content' => __('用户提交了新的反馈'),
        'is_icon' => 0,
        'is_img' => 1,
        'avatar' => 'assets/images/users/avatar-1.jpg'  // 图片路径
    ]

```

##### 示例 3：在控制器中使用

```php
<?php

namespace Your\Module\Controller;

use Weline\Framework\Event\EventsManager;
use Weline\Framework\Manager\ObjectManager;

class YourController
{
    /**
     * 发送系统消息示例
     */
    public function sendNotification(): void
    {
        /** @var EventsManager $eventsManager */
        $eventsManager = ObjectManager::getInstance(EventsManager::class);
        
        // 发送订单通知
        $eventsManager->dispatch('Weline_Framework::msg', [
            'data' => [
                'title' => __('新订单提醒'),
                'content' => __('订单编号：ORD-20241219-001 已创建，请及时处理。'),
                'is_read' => false,
                'is_icon' => 1,
                'avatar' => 'ri-shopping-bag-line'
            ]
        ]);
    }
}
```

#### 系统行为说明

##### 1. 数据验证

- 如果数据为空或不是数组，观察者会自动写入一条"错误"系统消息
- 如果缺少必填字段（title 或 content），也会写入错误消息
- 错误消息包含发送者信息，便于排查问题

##### 2. 去重机制

- 如果存在相同 `title` + `content` 的消息且为已读状态
- 系统会将该消息重置为"未读"状态，不会重复创建新记录
- 这样可以避免重复消息，同时确保重要消息不会被忽略

##### 3. 默认行为

- 如果不指定头像类型，默认使用图标头像
- 默认图标为 `ri-notification-line`
- 如果不指定 `is_read`，默认为 `false`（未读）

##### 4. 国际化支持

- 所有可见字符串（title、content）建议使用 `__()` 函数包装
- 这样可以支持多语言切换
- 框架会自动根据当前语言环境翻译消息内容

#### 注意事项

1. **必需字段**：`title` 和 `content` 是必需字段，缺少这些字段消息将不会被保存
2. **头像类型**：`is_icon` 和 `is_img` 不能同时为 1，如果同时设置，优先使用 `is_icon`
3. **默认行为**：如果不指定头像类型，默认使用图标头像（`is_icon=1`）
4. **错误处理**：消息保存失败不会影响主流程，错误会被静默处理（开发模式下会记录到错误日志）
5. **事件名称**：必须使用 `Weline_Framework::msg` 作为事件名称
6. **数据包装**：事件数据必须包装在 `data` 键中
7. **字符长度**：标题最大长度为 120 字符，超出部分会被截断
8. **性能考虑**：频繁发送消息可能影响性能，建议合理控制消息发送频率
9. **循环检测**：事件系统会自动检测循环调用，防止在消息处理中再次触发消息事件
