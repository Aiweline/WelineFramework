# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Framework_UninstallService::uninstall` 事件及其使用方法。该事件在卸载服务执行卸载操作时触发，允许其他模块监听卸载过程并执行相应操作。

## 事件列表

### 1. Weline_Framework_UninstallService::uninstall - 卸载服务事件

#### 基本信息

- **事件名称**：`Weline_Framework_UninstallService::uninstall`
- **事件类型**：卸载服务事件
- **触发时机**：在执行卸载操作时
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Weline_Framework_UninstallService::uninstall` 事件在卸载服务执行卸载操作时触发，允许其他模块监听卸载过程并执行相应操作。

该事件主要用于：
- 监听模块卸载
- 监听主题卸载
- 监听翻译包卸载
- 执行卸载前的清理操作
- 执行卸载后的清理操作
- 记录卸载日志

#### 触发时机

该事件在 `UninstallService` 执行卸载操作时触发，通常在以下场景中：
- 模块卸载
- 主题卸载
- 翻译包卸载

#### 使用场景

- **卸载监听**：监听模块、主题或翻译包的卸载过程
- **清理操作**：执行卸载相关的清理操作
- **日志记录**：记录卸载操作的日志
- **数据备份**：在卸载前备份相关数据
- **依赖检查**：检查卸载对象的依赖关系

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Framework_UninstallService::uninstall">
        <observer name="Your_Module::uninstall_observer"
                  instance="Your\Module\Observer\UninstallObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class UninstallObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $type = $data->getData('type');        // module/theme/i18n
        $name = $data->getData('name');        // 模块名/主题名/语言代码
        $autoBackup = $data->getData('auto_backup'); // 是否自动备份
        
        // 执行卸载相关操作
        // 例如：清理缓存、记录日志等
    }
}
```

#### 事件数据

`Weline_Framework_UninstallService::uninstall` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 卸载类型：`module`（模块）、`theme`（主题）、`i18n`（翻译包） |
| name | string | 卸载对象的名称（模块名/主题名/语言代码） |
| auto_backup | bool | 是否自动备份 |

**获取数据**：

```php
$data = $event->getData();
$type = $data->getData('type');
$name = $data->getData('name');
$autoBackup = $data->getData('auto_backup');
```

#### 卸载结果

卸载结果会返回到事件数据的 `uninstall_result` 字段中：

```php
$data = $event->getData();
$result = $data->getData('uninstall_result');
// $result 包含卸载操作的结果信息
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **异常处理**：如果观察者抛出异常，可能会影响卸载流程
3. **性能考虑**：该事件在卸载操作时触发，应避免执行耗时操作
4. **错误处理**：建议在观察者中添加异常处理，避免影响其他观察者执行
5. **数据修改**：可以通过修改事件数据来影响卸载流程
