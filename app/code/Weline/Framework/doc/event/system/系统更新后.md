# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_System::system_update_after` 事件及其使用方法。该事件在系统更新完成后触发，允许其他模块执行更新后的操作。

## 事件列表

### 1. Framework_System::system_update_after - 系统更新后事件

#### 基本信息

- **事件名称**：`Framework_System::system_update_after`
- **事件类型**：系统生命周期事件
- **触发时机**：在系统更新完成后
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_System::system_update_after` 事件在系统更新完成后触发，此时：
- 系统更新操作已执行完成
- 系统配置已更新
- 可以执行更新后的清理、缓存刷新等操作

该事件主要用于：
- 刷新系统缓存
- 重建索引
- 执行数据迁移
- 更新配置文件
- 清理临时文件
- 发送更新通知
- 自动检测和更新系统信息（如国家信息等）

#### 触发时机

该事件在系统更新命令执行完成后触发，通常在以下场景中：
- 系统版本更新
- 系统配置更新
- 系统文件更新

#### 使用场景

- **缓存刷新**：清除所有缓存，确保新代码生效
- **索引重建**：重建数据库索引，优化查询性能
- **数据迁移**：执行数据格式转换或迁移
- **配置更新**：更新系统配置文件
- **通知发送**：发送更新完成通知给管理员
- **信息同步**：同步系统信息（如国家、语言等）

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_System::system_update_after">
        <observer name="Your_Module::system_update_after"
                  instance="Your\Module\Observer\SystemUpdateAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class SystemUpdateAfterObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        // 执行系统更新后的操作
        // 例如：刷新缓存、重建索引等
    }
}
```

#### 事件数据

`Framework_System::system_update_after` 事件可能传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| 无 | - | 该事件通常不传递数据，或传递空数据对象 |

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **异常处理**：如果观察者抛出异常，可能会影响系统更新流程
3. **性能考虑**：该事件在系统更新时触发，应避免执行耗时操作
4. **错误处理**：建议在观察者中添加异常处理，避免影响其他观察者执行
