# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Framework_Template::after_tags_config` 事件及其使用方法。该事件在模板标签配置完成后触发，允许其他模块修改标签配置。

## 事件列表

### 1. Weline_Framework_Template::after_tags_config - 标签配置后事件

#### 基本信息

- **事件名称**：`Weline_Framework_Template::after_tags_config`
- **事件类型**：模板配置事件
- **触发时机**：在模板标签配置完成后
- **触发位置**：`app/code/Weline/Framework/View/Taglib.php` 第 937 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Weline_Framework_Template::after_tags_config` 事件在模板标签配置完成后触发，允许其他模块修改标签配置。可以添加、修改或删除标签配置。

该事件主要用于：
- 添加自定义标签
- 修改标签配置
- 扩展标签功能
- 标签配置验证

#### 触发时机

```php
// app/code/Weline/Framework/View/Taglib.php
$event->dispatch('Weline_Framework_Template::after_tags_config', $data);
```

#### 使用场景

- **添加自定义标签**：添加模块特定的模板标签
- **修改标签配置**：修改现有标签的配置
- **扩展标签功能**：扩展标签的功能
- **标签验证**：验证标签配置

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Framework_Template::after_tags_config">
        <observer name="Your_Module::add_custom_tags"
                  instance="Your\Module\Observer\AddCustomTagsObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class AddCustomTagsObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        
        // 添加自定义标签配置
        $data['custom_tag'] = [
            'class' => 'Your\Module\Tag\CustomTag',
            'method' => 'render',
        ];
        
        $event->setData($data);
    }
}
```

#### 事件数据

`Weline_Framework_Template::after_tags_config` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| data | array | 标签配置数组，可以修改此数组来改变标签配置 |

**获取和修改数据**：

```php
// 获取标签配置
$data = $event->getData();

// 添加标签配置
$data['tag_name'] = ['class' => 'Class\Name', 'method' => 'methodName'];

// 更新标签配置
$event->setData($data);
```

#### 使用示例

##### 示例 1：添加自定义标签

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class AddCustomTagsObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        
        // 添加自定义标签
        $data['product'] = [
            'class' => 'Your\Module\Tag\ProductTag',
            'method' => 'render',
        ];
        
        $data['category'] = [
            'class' => 'Your\Module\Tag\CategoryTag',
            'method' => 'render',
        ];
        
        $event->setData($data);
    }
}
```

##### 示例 2：修改标签配置

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ModifyTagsObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        
        // 修改现有标签配置
        if (isset($data['block'])) {
            $data['block']['class'] = 'Your\Module\Tag\CustomBlockTag';
        }
        
        $event->setData($data);
    }
}
```

#### 注意事项

1. **数据格式**：标签配置必须是有效的数组格式
2. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
3. **覆盖行为**：后执行的观察者可以覆盖先执行的观察者的设置
4. **性能考虑**：该事件在模板初始化时触发，应避免执行耗时操作

#### 系统行为说明

1. **默认行为**：
   - 默认使用系统标签配置
   - 观察者可以修改标签配置数组

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 后执行的观察者可以覆盖先执行的观察者的设置

3. **设置执行顺序**：通过 `sort` 属性控制执行顺序
4. **处理异常**：确保观察者中的异常被正确处理
