# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Framework_Template::after_compile` 事件及其使用方法。该事件在模板编译完成后触发，允许其他模块处理编译后的模板内容。

## 事件列表

### 1. Weline_Framework_Template::after_compile - 模板编译后事件

#### 基本信息

- **事件名称**：`Weline_Framework_Template::after_compile`
- **事件类型**：模板编译事件
- **触发时机**：在模板编译完成后，写入编译文件之前
- **触发位置**：`app/code/Weline/Framework/View/Template.php` 第 325 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Weline_Framework_Template::after_compile` 事件在模板编译完成后触发，允许其他模块处理编译后的模板内容。可以修改模板内容、提取信息、注入代码等。

该事件主要用于：
- 提取模板中的 JS 模块声明
- 提取模板中的翻译词
- 注入额外的代码到编译后的模板
- 修改模板内容
- 模板内容验证

#### 触发时机

```php
// app/code/Weline/Framework/View/Template.php
$eventsManager->dispatch('Weline_Framework_Template::after_compile', $eventData);
```

事件只在以下条件满足时触发：
- 开发模式（DEV = true），或
- 编译文件不存在，或
- 编译文件比模板文件旧（需要重新编译）

#### 使用场景

- **提取 JS 模块**：从模板中提取 JS 模块声明，用于模块化加载
- **提取翻译词**：从模板中提取需要翻译的文本，自动生成翻译文件
- **代码注入**：向编译后的模板注入额外的 PHP 代码或 JavaScript 代码
- **内容验证**：验证模板内容的合法性
- **性能优化**：对模板内容进行优化处理

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Framework_Template::after_compile">
        <observer name="Your_Module::template_compile_handler"
                  instance="Your\Module\Observer\TemplateCompileObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class TemplateCompileObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        // 获取模板内容
        $content = $event->getData('content');
        $comFileName = $event->getData('comFileName');
        $tplFile = $event->getData('tplFile');
        
        // 处理模板内容
        // ...
        
        // 更新内容
        $event->setData('content', $content);
    }
}
```

#### 事件数据

`Weline_Framework_Template::after_compile` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| content | string | 编译后的模板内容，可以修改此内容来改变最终编译结果 |
| comFileName | string | 编译后的文件路径 |
| tplFile | string | 原始模板文件路径 |
| template | Template | Template 实例对象 |

**获取和修改数据**：

```php
// 获取模板内容
$content = $event->getData('content');
$comFileName = $event->getData('comFileName');
$tplFile = $event->getData('tplFile');

// 修改模板内容
$content = str_replace('old', 'new', $content);

// 更新内容
$event->setData('content', $content);
```

#### 使用示例

##### 示例 1：提取 JS 模块声明

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ExtractJsModulesObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $content = $event->getData('content');
        
        // 提取 JS 模块声明
        preg_match_all('/data-module="([^"]+)"/', $content, $matches);
        $modules = $matches[1] ?? [];
        
        // 处理模块信息
        // ...
        
        // 注入模块信息到模板
        $modulesJson = json_encode($modules);
        $injectCode = "<?php\n\$__js_modules = {$modulesJson};\n?>";
        $content = str_replace('</body>', $injectCode . "\n</body>", $content);
        
        $event->setData('content', $content);
    }
}
```

```php
<?php
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **内容修改**：修改 `content` 后必须调用 `setData('content', $content)` 才能生效
3. **性能考虑**：该事件在每次模板编译时触发，应避免执行耗时操作
4. **触发条件**：事件只在需要重新编译时触发，已编译的模板不会触发
5. **错误处理**：观察者中的异常应被正确处理，避免影响模板编译

#### 系统行为说明

1. **默认行为**：
   - 模板编译后直接写入编译文件
   - 观察者可以修改编译后的内容

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 后执行的观察者可以修改先执行的观察者的结果

3. **内容更新**：
   - 只有调用 `setData('content', $content)` 才能更新最终编译结果
   - 多个观察者可以依次修改内容
