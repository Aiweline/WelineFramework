# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_View::{position}` 动态事件及其使用方法。该事件在渲染视图指定位置时触发，允许其他模块注入该位置的内容。

## 事件列表

### 1. Framework_View::{position} - 视图位置事件（动态事件）

#### 基本信息

- **事件名称**：`Framework_View::{position}`（动态事件模式）
- **事件类型**：视图渲染事件（动态）
- **触发时机**：在渲染视图指定位置时
- **触发位置**：`app/code/Weline/Framework/View/TraitTemplate.php` 第 62 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`
- **动态说明**：`{position}` 是动态占位符，可以是 `head`、`footer`、`fetch_file` 等

#### 功能说明

`Framework_View::{position}` 是一个动态事件，其中 `{position}` 是动态占位符。该事件在渲染视图指定位置时触发，允许其他模块注入该位置的内容。通过设置 `class` 字段来指定该位置的渲染类。

**动态事件说明**：
- 实际触发的事件名会根据 `position` 的值动态生成
- 例如：`position = 'head'` 时，实际事件名为 `Framework_View::head`
- 例如：`position = 'footer'` 时，实际事件名为 `Framework_View::footer`
- 例如：`position = 'fetch_file'` 时，实际事件名为 `Weline_Framework_View::fetch_file`

该事件主要用于：
- 注入视图位置内容
- 自定义视图位置渲染
- 添加CSS、JS等资源
- 设置页面元信息

#### 触发时机

```php
// app/code/Weline/Framework/View/TraitTemplate.php
$eventData = ['position' => $position, 'is_backend' => $is_backend, 'class' => ''];
$this->eventsManager->dispatch("Framework_View::{$position}", $eventData);
$class = $this->eventsManager->getEventData("Framework_View::{$position}")->getData('class');
```

#### 使用场景

- **注入头部内容**：在页面头部注入CSS、JS、Meta标签等（`position = 'head'`）
- **注入底部内容**：在页面底部注入JS脚本、统计代码等（`position = 'footer'`）
- **文件获取**：自定义视图文件获取逻辑（`position = 'fetch_file'`）
- **自定义位置**：为其他自定义位置注入内容

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <!-- 监听 head 位置 -->
    <event name="Framework_View::head">
        <observer name="Your_Module::inject_head_content"
                  instance="Your\Module\Observer\InjectHeadContentObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
    
    <!-- 监听 footer 位置 -->
    <event name="Framework_View::footer">
        <observer name="Your_Module::inject_footer_content"
                  instance="Your\Module\Observer\InjectFooterContentObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class InjectHeadContentObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        // 设置头部渲染类
        $event->setData('class', 'Your\Module\Block\Head');
    }
}
```

#### 事件数据

`Framework_View::{position}` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| position | string | 位置名称（head、footer、fetch_file 等） |
| is_backend | bool | 是否是后端 |
| class | string | 位置渲染类名，可以设置此值来指定渲染类 |

**获取和修改数据**：

```php
// 获取数据
$position = $event->getData('position');
$isBackend = $event->getData('is_backend');

// 设置位置渲染类
$event->setData('class', 'Your\Module\Block\Head');
```

#### 使用示例

##### 示例 1：注入头部CSS和JS

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class InjectHeadResourcesObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $position = $event->getData('position');
        
        // 只在 head 位置注入
        if ($position === 'head') {
            $event->setData('class', 'Your\Module\Block\Head');
        }
    }
}
```

然后在 `Your\Module\Block\Head` 类中：

```php
<?php

namespace Your\Module\Block;

class Head
{
    public function render(): string
    {
        $html = '<link rel="stylesheet" href="/static/css/custom.css">';
        $html .= '<script src="/static/js/custom.js"></script>';
        return $html;
    }
}
```

##### 示例 2：注入底部统计代码

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class InjectFooterAnalyticsObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $position = $event->getData('position');
        $isBackend = $event->getData('is_backend');
        
        // 只在 footer 位置且非后端模式下注入
        if ($position === 'footer' && !$isBackend) {
            $event->setData('class', 'Your\Module\Block\Footer');
        }
    }
}
```

#### 注意事项

1. **动态事件匹配**：系统会自动将实际事件名（如 `Framework_View::head`）与动态事件模式（`Framework_View::{position}`）进行匹配
2. **class设置**：必须设置 `class` 字段来指定位置渲染类
3. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
4. **覆盖行为**：后执行的观察者可以覆盖先执行的观察者的设置
5. **性能考虑**：该事件在每次渲染视图时都会触发，应避免执行耗时操作

#### 系统行为说明

1. **动态事件匹配**：
   - 当触发 `Framework_View::head` 时，系统会匹配模式 `Framework_View::{position}`
   - 匹配成功后，使用该模式的规约和文档信息

2. **默认行为**：
   - 默认 `class` 为空
   - 如果未设置 `class`，使用默认位置渲染

3. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 后执行的观察者可以覆盖先执行的观察者的设置

## 扩展开发

如果需要扩展视图位置功能，可以：

1. **创建新的观察者**：实现 `ObserverInterface` 接口
2. **注册观察者**：在模块的 `etc/event.xml` 中注册，使用实际事件名（如 `Framework_View::head`）
3. **设置执行顺序**：通过 `sort` 属性控制执行顺序
4. **处理异常**：确保观察者中的异常被正确处理
