# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_View::fetch_file` 事件及其使用方法。该事件在获取视图文件时触发，允许其他模块自定义视图文件路径。

## 事件列表

### 1. Framework_View::fetch_file - 视图文件获取事件

#### 基本信息

- **事件名称**：`Framework_View::fetch_file`
- **事件类型**：视图渲染事件
- **触发时机**：在获取视图文件时
- **触发位置**：`app/code/Weline/Framework/View/TraitTemplate.php` 第 444 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_View::fetch_file` 事件在获取视图文件时触发，允许其他模块自定义视图文件路径。通过修改事件数据中的 `filename` 字段，可以改变要加载的视图文件。

该事件主要用于：
- 自定义视图文件路径
- 视图文件继承和覆盖
- 多主题支持
- 视图文件动态替换

#### 触发时机

```php
// app/code/Weline/Framework/View/TraitTemplate.php
$fileData = new DataObject(['filename' => $filename, 'type' => 'compile', 'object' => $this]);
$this->eventsManager->dispatch('Framework_View::fetch_file', $fileData);
$event_filename = $fileData->getData('filename');
```

#### 使用场景

- **视图文件继承**：允许子模块覆盖父模块的视图文件
- **多主题支持**：根据主题动态切换视图文件
- **视图文件替换**：在特定条件下替换视图文件
- **视图文件预处理**：在加载视图文件前进行预处理

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_View::fetch_file">
        <observer name="Your_Module::custom_view_file"
                  instance="Your\Module\Observer\CustomViewFileObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class CustomViewFileObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $fileData = $event->getData();
        $filename = $fileData->getData('filename');
        $type = $fileData->getData('type');
        $object = $fileData->getData('object');
        
        // 自定义视图文件路径逻辑
        // 例如：根据条件替换视图文件
        if ($type === 'compile' && str_contains($filename, 'templates/Backend')) {
            // 替换为自定义视图文件
            $customFilename = str_replace('templates/Backend', 'templates/Custom', $filename);
            if (file_exists($customFilename)) {
                $fileData->setData('filename', $customFilename);
            }
        }
    }
}
```

#### 事件数据

`Framework_View::fetch_file` 事件传递的数据：

```php
[
    'filename' => string,  // 视图文件路径（可以修改）
    'type' => string,      // 文件类型，通常是 'compile'
    'object' => object,    // Template 对象实例
]
```

**数据说明**：
- `filename`：视图文件的完整路径，观察者可以修改此值来改变要加载的视图文件
- `type`：文件类型，通常为 `'compile'`，表示编译类型
- `object`：`Template` 对象的实例，可以用于访问模板相关的属性和方法

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **文件存在性**：修改 `filename` 后，应确保新文件存在，否则可能导致视图加载失败
3. **性能考虑**：该事件在每次获取视图文件时都会触发，应避免执行耗时操作
4. **文件路径**：修改 `filename` 时，应使用绝对路径或相对于项目根目录的路径
