# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Framework_Module::module_install_after` 事件及其使用方法。该事件在模块安装完成后触发，允许其他模块执行安装后的操作。

## 事件列表

### 1. Weline_Framework_Module::module_install_after - 模块安装后事件

#### 基本信息

- **事件名称**：`Weline_Framework_Module::module_install_after`
- **事件类型**：模块生命周期事件
- **触发时机**：在模块安装完成后
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Weline_Framework_Module::module_install_after` 事件在模块安装完成后触发，此时：
- 模块已成功安装
- 模块的数据库表已创建
- 模块的配置文件已初始化
- 可以执行安装后的初始化操作

该事件主要用于：
- 初始化模块数据
- 创建默认配置
- 执行数据迁移
- 注册模块服务
- 刷新缓存
- 发送安装通知

#### 触发时机

该事件在模块安装命令执行完成后触发，通常在以下场景中：
- 新模块安装
- 模块重新安装
- 模块依赖安装

#### 使用场景

- **数据初始化**：创建模块所需的默认数据
- **配置初始化**：设置模块的默认配置
- **服务注册**：注册模块提供的服务
- **缓存刷新**：清除相关缓存，确保新模块生效
- **通知发送**：发送模块安装完成通知
- **信息同步**：同步模块相关信息（如国家、语言等）

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Framework_Module::module_install_after">
        <observer name="Your_Module::module_install_after"
                  instance="Your\Module\Observer\ModuleInstallAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ModuleInstallAfterObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $data = $event->getData();
        $moduleName = $data->getData('module') ?? '';
        
        // 执行模块安装后的操作
        // 例如：初始化数据、创建配置等
    }
}
```

#### 事件数据

`Weline_Framework_Module::module_install_after` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| module | string | 已安装的模块名称 |

**获取数据**：

```php
$data = $event->getData();
$moduleName = $data->getData('module');
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **异常处理**：如果观察者抛出异常，可能会影响模块安装流程
3. **性能考虑**：该事件在模块安装时触发，应避免执行耗时操作
4. **错误处理**：建议在观察者中添加异常处理，避免影响其他观察者执行
5. **模块检查**：建议在观察者中检查模块名称，只处理相关模块
