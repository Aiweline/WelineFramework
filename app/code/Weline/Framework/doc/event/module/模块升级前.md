# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Module::module_upgrade_before` 事件及其使用方法。该事件在模块升级前触发，允许其他模块在升级前执行必要的操作。

## 事件列表

### 1. Framework_Module::module_upgrade_before - 模块升级前事件

#### 基本信息

- **事件名称**：`Framework_Module::module_upgrade_before`
- **事件类型**：模块生命周期事件
- **触发时机**：在模块升级命令执行前
- **触发位置**：`app/code/Weline/Framework/Module/Console/Module/Upgrade.php` 第 68 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_Module::module_upgrade_before` 事件在模块升级前触发，此时：
- 模块升级命令已开始执行
- 模块信息已加载
- 升级脚本尚未执行
- 可以执行升级前的检查、备份等操作

该事件主要用于：
- 检查升级前置条件
- 备份模块数据
- 验证模块依赖
- 执行升级前检查
- 记录升级日志

#### 触发时机

```php
// app/code/Weline/Framework/Module/Console/Module/Upgrade.php
public function execute(array $args = [], array $data = [])
{
    /**@var EventsManager $eventsManager */
    $eventsManager = ObjectManager::getInstance(EventsManager::class);
    $eventsManager->dispatch('Framework_Module::module_upgrade_before');
    
    // ... 执行模块升级 ...
}
```

#### 使用场景

- **前置条件检查**：检查模块升级的前置条件是否满足
- **数据备份**：备份模块相关数据，防止升级失败
- **依赖验证**：验证模块依赖是否满足升级要求
- **升级前检查**：执行升级前的各种检查
- **日志记录**：记录升级开始日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Module::module_upgrade_before">
        <observer name="Your_Module::backup_before_upgrade"
                  instance="Your\Module\Observer\BackupBeforeUpgradeObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class BackupBeforeUpgradeObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        // 执行升级前备份
        $this->backupModuleData();
    }
    
    private function backupModuleData(): void
    {
        // 备份模块数据
        // ...
    }
}
```

#### 事件数据

`Framework_Module::module_upgrade_before` 事件不传递额外的数据。

#### 使用示例

##### 示例 1：检查升级前置条件

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\App\Exception;

class CheckUpgradePrerequisitesObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        // 检查 PHP 版本
        if (version_compare(PHP_VERSION, '8.1.0', '<')) {
            throw new Exception(__('模块升级需要 PHP 8.1.0 或更高版本'));
        }
        
        // 检查扩展
        if (!extension_loaded('pdo_mysql')) {
            throw new Exception(__('模块升级需要 PDO MySQL 扩展'));
        }
        
        // 检查磁盘空间
        $freeSpace = disk_free_space(BP);
        if ($freeSpace < 100 * 1024 * 1024) { // 100MB
            throw new Exception(__('磁盘空间不足，至少需要 100MB 可用空间'));
        }
    }
}
```

<?php

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **异常处理**：如果观察者抛出异常，可能会阻止模块升级
3. **性能考虑**：该事件在模块升级时触发，应避免执行耗时操作
4. **幂等性**：确保观察者的操作是幂等的，可以安全地重复执行

#### 系统行为说明

1. **默认观察者**：暂无默认观察者

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行
   - 建议将检查类操作放在前面（较小的 sort 值）
