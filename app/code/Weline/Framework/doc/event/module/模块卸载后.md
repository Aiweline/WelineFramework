# Weline Framework 模块 - 模块卸载后事件

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Module::module_uninstall_after` 事件及其使用方法。  
该事件在模块卸载流程完成后触发，用于通知其他模块执行卸载后的清理和收尾操作。

## 事件列表

### 1. Framework_Module::module_uninstall_after - 模块卸载后事件

#### 基本信息

- **事件名称**：`Framework_Module::module_uninstall_after`
- **事件类型**：模块生命周期事件
- **触发时机**：在模块卸载流程完成后
- **典型配置文件**：各模块的 `etc/event.xml`

#### 功能说明

`Framework_Module::module_uninstall_after` 事件用于在模块被卸载完成后，通知其他模块执行与该模块相关的后续处理，例如：

- 清理与该模块相关的缓存、临时数据
- 清理与该模块相关的扩展配置、索引、运行数据
- 记录卸载审计日志
- 做一些与卸载结果相关的收尾操作

#### 使用场景

- **数据库清理**：在模块卸载后，执行额外的数据迁移或结构清理
- **缓存与索引刷新**：移除与该模块相关的缓存、索引数据
- **审计与监控**：记录模块卸载行为，用于审计或监控
- **跨模块联动**：其他依赖该模块的模块在此事件中完成依赖解绑或降级处理

#### 使用方法

##### 在模块中注册观察者

在目标模块的 `etc/event.xml` 文件中注册观察者，例如：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Module::module_uninstall_after">
        <observer name="Your_Module::module_uninstall_after"
                  instance="Your\Module\Observer\ModuleUninstallAfterObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ModuleUninstallAfterObserver implements ObserverInterface
{
    /**
     * @param Event $event
     */
    public function execute(Event &$event): void
    {
        // 兼容两种传参方式：直接数组或包裹在 data 中
        $data = $event->getData('data') ?? $event->getData();

        // 根据实际约定从数据中读取被卸载模块名等信息（若有）
        $module = $data['module'] ?? ($data['module_name'] ?? null);

        if (!$module) {
            return;
        }

        // 在这里执行与模块卸载相关的清理或后续操作
    }
}
```

#### 事件数据

`Framework_Module::module_uninstall_after` 事件的数据结构由触发方约定，通常至少会包含被卸载模块的名称，例如：

| 字段        | 类型   | 说明                 |
|-------------|--------|----------------------|
| module      | string | 被卸载的模块名称（可选） |
| module_name | string | 被卸载的模块名称（可选，兼容字段） |

**获取数据示例**：

```php
$data = $event->getData('data') ?? $event->getData();
$module = $data['module'] ?? ($data['module_name'] ?? null);
```

> 提示：由于该事件主要作为卸载完成后的通知钩子使用，实际字段可能会根据触发方实现进行扩展，观察者应做好空值与字段存在性的判断。

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行。
2. **异常处理**：如果观察者抛出异常，可能会影响卸载流程或其他观察者的执行，建议在内部捕获异常并记录日志。
3. **性能考虑**：卸载流程通常是一次性操作，但仍应避免过于耗时的处理，必要时可将重任务拆分为异步任务。
4. **幂等性**：建议观察者实现尽量具备幂等性，避免重复执行带来副作用。
