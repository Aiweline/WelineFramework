# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Weline_Framework_Module::module_upgrade` 事件及其使用方法。该事件在模块升级时触发，允许其他模块监听模块升级过程。

## 事件列表

### 1. Weline_Framework_Module::module_upgrade - 模块升级事件

#### 基本信息

- **事件名称**：`Weline_Framework_Module::module_upgrade`
- **事件类型**：模块生命周期事件
- **触发时机**：在模块升级过程中
- **触发位置**：`app/code/Weline/Framework/Module/Console/Module/Upgrade.php` 第 445 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Weline_Framework_Module::module_upgrade` 事件在模块升级时触发，此时：
- 模块升级脚本正在执行
- 数据库结构正在更新
- 可以监听升级过程并执行相关操作

该事件主要用于：
- 监听模块升级过程
- 执行升级相关的操作
- 更新模块依赖
- 刷新模块缓存
- 记录升级日志

#### 触发时机

```php
// app/code/Weline/Framework/Module/Console/Module/Upgrade.php
public function execute(array $args = [], array $data = [])
{
    // ... 执行模块升级 ...
    
    /**@var EventsManager $eventsManager */
    $eventsManager = ObjectManager::getInstance(EventsManager::class);
    $eventsManager->dispatch('Weline_Framework_Module::module_upgrade');
}
```

#### 使用场景

- **监听升级过程**：监听模块升级过程，执行相关操作
- **更新依赖**：更新模块依赖关系
- **刷新缓存**：刷新模块相关缓存
- **日志记录**：记录模块升级日志

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Weline_Framework_Module::module_upgrade">
        <observer name="Your_Module::scan_adapter_after_upgrade"
                  instance="Your\Module\Observer\ScanAdapterAfterUpgradeObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ScanAdapterAfterUpgradeObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        // 扫描数据库适配器
        $this->scanAdapters();
    }
    
    private function scanAdapters(): void
    {
        // 扫描适配器逻辑
    }
}
```

#### 事件数据

`Weline_Framework_Module::module_upgrade` 事件不传递额外的数据。

#### 使用示例

##### 示例 1：扫描数据库适配器

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Manager\ObjectManager;

class ScanAdapterObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        // 扫描数据库适配器
        $adapterScanner = ObjectManager::getInstance(
            \Weline\Framework\Database\Service\AdapterScanner::class
        );
        $adapterScanner->scan();
    }
}
```

##### 示例 2：刷新模块缓存

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;
use Weline\Framework\Manager\ObjectManager;

class RefreshModuleCacheObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        // 清除模块缓存
        $moduleCache = ObjectManager::getInstance(
            \Weline\Framework\Module\Cache\ModuleCache::class
        );
        $moduleCache->clean();
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **性能考虑**：该事件在模块升级时触发，应避免执行耗时操作
3. **错误处理**：观察者中的异常不会被框架捕获，应自行处理错误
4. **幂等性**：确保观察者的操作是幂等的，可以安全地重复执行

#### 系统行为说明

1. **默认观察者**：
   - `Weline_Framework_Database::module_upgrade_adapter_scan` - 扫描数据库适配器（sort: 100）

2. **执行顺序**：
   - 观察者按照 `sort` 属性从小到大排序执行

## 扩展开发

如果需要扩展模块升级功能，可以：

1. **创建新的观察者**：实现 `ObserverInterface` 接口
2. **注册观察者**：在模块的 `etc/event.xml` 中注册
3. **设置执行顺序**：通过 `sort` 属性控制执行顺序
4. **处理异常**：确保观察者中的异常被正确处理
