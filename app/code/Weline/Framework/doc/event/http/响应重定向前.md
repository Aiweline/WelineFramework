# Weline Framework 模块 - 事件文档

## 概述

本文档详细说明了 Weline Framework 模块提供的 `Framework_Http::response_redirect_before` 事件及其使用方法。该事件在HTTP响应重定向前触发，允许其他模块修改重定向URL。

## 事件列表

### 1. Framework_Http::response_redirect_before - 响应重定向前事件

#### 基本信息

- **事件名称**：`Framework_Http::response_redirect_before`
- **事件类型**：HTTP响应事件
- **触发时机**：在HTTP响应重定向前
- **触发位置**：`app/code/Weline/Framework/Http/Response.php` 第 124 行
- **配置文件**：`app/code/Weline/Framework/etc/event.xml`

#### 功能说明

`Framework_Http::response_redirect_before` 事件在HTTP响应重定向前触发，允许其他模块修改重定向URL。可以修改 `url` 和 `code` 字段来改变重定向行为。

该事件主要用于：
- 修改重定向URL
- 修改重定向状态码
- 重定向日志记录
- 重定向验证

#### 触发时机

```php
// app/code/Weline/Framework/Http/Response.php
$data = new DataObject(['url' => $url, 'code' => $code]);
$this->getEvenManager()->dispatch('Framework_Http::response_redirect_before', $data);
$url = $data->getData('url');
$code = $data->getData('code');
http_response_code($code);
Header("Location:$url");
exit(0);
```

#### 使用场景

- **URL修改**：修改重定向URL
- **状态码修改**：修改重定向状态码
- **日志记录**：记录重定向日志
- **重定向验证**：验证重定向URL

#### 使用方法

##### 基本用法

在模块的 `etc/event.xml` 文件中注册观察者：

```xml
<?xml version="1.0"?>
<config xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
        xs:noNamespaceSchemaLocation="urn:Weline_Framework::Event/etc/xsd/event.xsd"
        xmlns="urn:Weline_Framework::Event/etc/xsd/event.xsd">
    <event name="Framework_Http::response_redirect_before">
        <observer name="Your_Module::modify_redirect"
                  instance="Your\Module\Observer\ModifyRedirectObserver"
                  disabled="false"
                  shared="true"
                  sort="100"/>
    </event>
</config>
```

创建观察者类：

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ModifyRedirectObserver implements ObserverInterface
{
    /**
     * 执行观察者逻辑
     *
     * @param Event $event
     * @return void
     */
    public function execute(Event &$event): void
    {
        $url = $event->getData('url');
        $code = $event->getData('code');
        
        // 修改重定向URL
        $modifiedUrl = $this->modifyRedirectUrl($url);
        if ($modifiedUrl) {
            $event->setData('url', $modifiedUrl);
        }
        
        // 修改重定向状态码（可选）
        // $event->setData('code', 301);
    }
    
    private function modifyRedirectUrl(string $url): ?string
    {
        // URL修改逻辑
        return null;
    }
}
```

#### 事件数据

`Framework_Http::response_redirect_before` 事件传递的数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| url | string | 重定向URL，可以修改此值来改变重定向URL |
| code | int | 重定向状态码，可以修改此值来改变状态码 |

**获取和修改数据**：

```php
$url = $event->getData('url');
$code = $event->getData('code');

// 修改重定向URL
$event->setData('url', $modifiedUrl);

// 修改重定向状态码
$event->setData('code', 301);
```

#### 使用示例

##### 示例 1：修改重定向URL

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class ModifyRedirectUrlObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $url = $event->getData('url');
        
        // 添加查询参数
        $separator = str_contains($url, '?') ? '&' : '?';
        $url .= $separator . 'redirected=1';
        
        $event->setData('url', $url);
    }
}
```

##### 示例 2：记录重定向日志

```php
<?php

namespace Your\Module\Observer;

use Weline\Framework\Event\Event;
use Weline\Framework\Event\ObserverInterface;

class LogRedirectObserver implements ObserverInterface
{
    public function execute(Event &$event): void
    {
        $url = $event->getData('url');
        $code = $event->getData('code');
        
        // 记录重定向日志
        error_log(sprintf(
            'Redirect: %s -> %s (Code: %d)',
            $_SERVER['REQUEST_URI'] ?? '',
            $url,
            $code
        ));
    }
}
```

#### 注意事项

1. **执行顺序**：观察者按照 `sort` 属性排序执行，数值越小越先执行
2. **覆盖行为**：后执行的观察者可以覆盖先执行的观察者的设置
3. **URL格式**：确保修改后的URL格式正确
