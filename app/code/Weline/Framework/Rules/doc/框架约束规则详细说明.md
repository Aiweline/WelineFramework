## 测试类位置规则

### 规则信息

- **规则名称**：`test-class-placement`
- **规则分类**：test
- **规则优先级**：10
- **规则简述**：测试类不应放在业务代码目录下

### 规则描述

测试类（继承自 `PHPUnit\Framework\TestCase` 或 `Weline\Framework\UnitTest\TestCore`，或类名包含 `Test`）不应放在业务代码目录下。测试类应放在专门的测试目录下，以保持代码结构的清晰和可维护性。

### 约束条件

#### 禁止的目录

测试类不应放在以下业务代码目录下：

- `Model` - 模型目录
- `Controller` - 控制器目录
- `Block` - 块目录
- `Helper` - 辅助类目录
- `Observer` - 观察者目录
- `Plugin` - 插件目录
- `Console` - 控制台命令目录
- `View` - 视图目录
- `Taglib` - 标签库目录
- `Api` - API 目录

#### 允许的目录

测试类应放在以下测试目录下：

- `Test` - 测试目录
- `UnitTest` - 单元测试目录
- `Tests` - 测试目录（复数形式）
- `tests` - 测试目录（小写）

### 测试类识别规则

以下情况会被识别为测试类：

1. **继承关系**：类继承自以下任一基类
   - `PHPUnit\Framework\TestCase`
   - `Weline\Framework\UnitTest\TestCore`

2. **类名模式**：类名包含 `Test` 或以 `Test` 结尾
   - ✅ `MenuTest` - 测试类
   - ✅ `ModelTest` - 测试类
   - ✅ `UserControllerTest` - 测试类
   - ❌ `TestModel` - 业务类（以 Test 开头，且不继承测试基类）
   - ❌ `TestController` - 业务类（以 Test 开头，且不继承测试基类）

### 错误示例

#### 示例 1：测试类放在 Model 目录下

```
app/code/Weline/Backend/Model/MenuTest.php
```

**错误信息**：
```

文件位置：E:\WelineFramework\DEV-workspace\app\code\Weline\Backend\Model\MenuTest.php
请将测试文件移动到专门的测试目录（如 Test、UnitTest 等）下。
```

**修复方法**：
将文件移动到：
```
app/code/Weline/Backend/Test/Model/MenuTest.php
```

将文件移动到：
```
app/code/Weline/Backend/Test/Controller/Index/IndexTest.php
```

### 验证时机

该规则在以下时机自动执行：

1. **系统更新时**：在 `php bin/w setup:upgrade` 命令执行时
2. **规则验证阶段**：在模块升级之前执行
3. **阻止升级**：如果发现违反规则，会阻止系统升级继续执行

### 验证流程

1. 扫描所有激活的模块
2. 对每个模块，检查业务代码目录
3. 识别每个目录下的测试类
4. 如果发现测试类在业务代码目录下，记录错误
5. 所有模块检查完成后，如果有错误，抛出异常并报告

### 常见问题

#### Q: 为什么测试类不能放在业务代码目录下？

A: 原因包括：
1. **代码组织**：保持业务代码和测试代码的清晰分离
2. **自动加载**：避免测试类被业务代码自动加载器扫描
3. **性能**：减少不必要的类扫描和加载
4. **维护性**：便于管理和维护测试代码

#### Q: 如果我的测试类名不以 Test 结尾怎么办？

A: 只要测试类继承自 `PHPUnit\Framework\TestCase` 或 `Weline\Framework\UnitTest\TestCore`，就会被识别为测试类，无论类名如何。

#### Q: 业务类名包含 Test 会被误判吗？

A: 不会。如果类名以 `Test` 开头（如 `TestModel`），且不继承测试基类，不会被识别为测试类。
