## 命令系统概述

Weline Framework 的命令系统基于 `CommandInterface` 接口，提供了灵活的命令创建和管理机制。命令系统支持：

- **模块命令**：每个模块可以定义自己的命令
- **框架命令**：框架核心提供的命令
- **命令别名**：为命令提供简短的别名，方便使用
- **自动扫描**：自动扫描并注册所有命令
- **帮助系统**：内置帮助信息格式化工具

### 核心接口和类

- `CommandInterface`：命令接口，所有命令必须实现
- `CommandAbstract`：命令抽象类，提供基础功能
- `CliAbstract`：CLI 抽象类，提供命令行基础功能
- `CommandHelper`：命令帮助信息格式化工具

## 创建新命令

### 命令类结构

创建一个新的命令类，需要实现 `CommandInterface` 接口或继承 `CommandAbstract` 抽象类。

#### 方式一：实现 CommandInterface 接口

```php
<?php

namespace YourModule\Console\Your\Command;

use Weline\Framework\Console\CommandInterface;
use Weline\Framework\Output\Cli\Printing;

class YourCommand implements CommandInterface
{
    public function __construct(
        private Printing $printer
    ) {
    }

    public function execute(array $args = [], array $data = []): void
    {
        // 命令执行逻辑
        $this->printer->success('命令执行成功！');
    }

    public function tip(): string
    {
        return '命令描述';
    }

    public function help(): array|string
    {
        return '命令帮助信息';
    }
}
```

#### 方式二：继承 CommandAbstract 抽象类（推荐）

```php
<?php

namespace YourModule\Console\Your\Command;

use Weline\Framework\Console\CommandAbstract;
use Weline\Framework\Output\Cli\Printing;

class YourCommand extends CommandAbstract
{
    public function __construct(
        Printing $printer
    ) {
        $this->printer = $printer;
    }

    public function execute(array $args = [], array $data = []): void
    {
        // 命令执行逻辑
        $this->printer->success('命令执行成功！');
    }

    public function tip(): string
    {
        return '命令描述';
    }

    public function help(): array|string
    {
        return \Weline\Framework\Console\CommandHelper::formatHelp(
            'your:command',
            '命令描述',
            [
                '-h, --help' => '显示帮助信息',
                '-v, --verbose' => '详细输出',
            ],
            [],
            [
                '基本用法' => 'php bin/w your:command',
                '带参数' => 'php bin/w your:command --verbose',
            ]
        );
    }
}
```

### 命令命名规则

命令的命名遵循以下规则：

1. **类名**：使用大驼峰命名（PascalCase），例如 `YourCommand`
2. **命令名**：根据文件路径自动生成，使用小写字母和冒号分隔
3. **路径映射**：
   - 文件路径：`Console/Your/Command/YourCommand.php`
   - 命令名：`your:command:your-command`
   - 实际使用：`php bin/w your:command:your-command`

#### 命名转换规则

- 目录名转换为命令前缀（小写，用冒号分隔）
- 类名转换为命令名（大写字母前加连字符，全小写）
- 例如：`Console/Server/Start.php` → `server:start`

### 命令文件位置

#### 模块命令

模块命令应放在模块的 `Console` 目录下：

```
app/code/YourModule/
└── Console/
    └── Your/
        └── Command/
            └── YourCommand.php
```

**命名空间**：`YourModule\Console\Your\Command\YourCommand`

#### 框架命令

框架命令放在框架的 `Console` 目录下：

```
app/code/Weline/Framework/Console/
└── Console/
    └── Command/
        └── Upgrade.php
```

**命名空间**：`Weline\Framework\Console\Console\Command\Upgrade`

```php
    // 执行逻辑
    if ($verbose) {

    }
    
    $this->printer->success("处理完成: {$name}");
}
```

返回命令的简短描述，用于命令列表显示。

```php
public function tip(): string

```

## 命令别名功能

### 别名功能概述

命令别名功能允许为命令定义简短的别名，方便快速调用。例如：

- 原命令：`php bin/w server:start`
- 别名：`php bin/w s:start` 或 `php bin/w s:s`

别名功能支持两种定义方式：

1. **aliases() 方法**：动态返回别名数组（推荐）
2. **ALIASES 常量**：静态定义别名数组

### 使用 aliases() 方法

在命令类中实现 `aliases()` 方法，返回别名数组。

```php
<?php

namespace YourModule\Console\Your\Command;

use Weline\Framework\Console\CommandAbstract;

class YourCommand extends CommandAbstract
{
    // ... 其他方法 ...

    /**
     * 返回命令别名数组
     * 
     * @return array 别名数组
     */
    public function aliases(): array
    {
        return [
            'yc',           // 简短别名
            'your-cmd',     // 带连字符的别名
            'y:cmd',        // 带命名空间的别名
        ];
    }
}
```

**特点**：

- ✅ 动态生成，可以根据条件返回不同的别名
- ✅ 支持运行时逻辑判断
- ✅ 推荐使用方式

**示例：根据环境返回不同别名**

```php
public function aliases(): array
{
    $aliases = ['yc'];
    
    // 开发环境添加额外别名
    if (defined('DEV') && DEV) {
        $aliases[] = 'dev:yc';
    }
    
    return $aliases;
}
```

### 使用 ALIASES 常量

在命令类中定义 `ALIASES` 常量，静态定义别名数组。

```php
<?php

namespace YourModule\Console\Your\Command;

use Weline\Framework\Console\CommandAbstract;

class YourCommand extends CommandAbstract
{
    /**
     * 命令别名常量
     */
    public const ALIASES = [
        'yc',
        'your-cmd',
        'y:cmd',
    ];

    // ... 其他方法 ...
}
```

**特点**：

- ✅ 静态定义，性能更好
- ✅ 简单直接，适合固定别名
- ⚠️ 不支持动态逻辑

### 别名注册机制

别名注册在命令扫描和更新时自动完成，流程如下：

1. **命令扫描阶段**：
   - 扫描所有命令类
   - 检查是否定义了 `ALIASES` 常量
   - 如果存在，注册常量中定义的别名

2. **别名注册阶段**（`registerAliases()` 方法）：
   - 遍历所有已注册的命令
   - 检查命令类是否实现了 `aliases()` 方法
   - 如果存在，调用方法获取别名并注册
   - 确保别名不覆盖已有命令

**注册优先级**：

1. 原始命令名（最高优先级，不会被覆盖）
2. `ALIASES` 常量定义的别名
3. `aliases()` 方法返回的别名（如果别名已存在，不会覆盖）

```363:401:app/code/Weline/Framework/Console/Console/Command/Upgrade.php
private function registerAliases(array $commands): array
{
    foreach ($commands as $group => $group_commands) {
        foreach ($group_commands as $command => $command_data) {
            if (!isset($command_data['class'])) {
                continue;
            }
            
            try {
                $command_class = ObjectManager::getInstance($command_data['class']);
                if ($command_class instanceof CommandInterface) {
                    // 检查命令类是否实现了 aliases() 方法
                    if (method_exists($command_class, 'aliases')) {
                        $aliases = $command_class->aliases();
                        if (is_array($aliases) && !empty($aliases)) {
                            // 为每个别名注册命令
                            foreach ($aliases as $alias) {
                                if (is_string($alias) && !empty($alias)) {
                                    // 确保别名不存在，避免覆盖已有命令
                                    if (!isset($commands[$group][$alias])) {
                                        $commands[$group][$alias] = $command_data;
                                    }
                                }
                            }
                        }
                    }
                }
            } catch (\Throwable $exception) {
                // 如果无法实例化命令类，跳过别名注册
                if (DEV && CLI) {
                    $this->printer->warning(__('无法注册命令别名：%{1} - %{2}', [$command_data['class'], $exception->getMessage()]));
                }
                continue;
            }
        }
    }
    
    return $commands;
}
```

### 别名最佳实践

1. **保持简短**：别名应该比原命令更短，方便输入
2. **避免冲突**：确保别名不会与其他命令冲突
3. **语义清晰**：别名应该能够表达命令的含义
4. **统一风格**：在模块内保持别名命名风格一致
5. **优先使用 aliases() 方法**：更灵活，支持动态逻辑

## 命令参数处理

### 参数获取

命令的 `execute()` 方法接收两个参数：

- `$args`：命令行参数数组
- `$data`：额外数据数组

```php
public function execute(array $args = [], array $data = []): void
{
    // 获取短选项
    $verbose = isset($args['v']) || isset($args['verbose']);
    
    // 获取长选项
    $name = $args['name'] ?? $args['n'] ?? 'default';
    
    // 获取位置参数（通过 $data 传递）
    $file = $data['file'] ?? null;
    
    // 处理参数
    if ($verbose) {
        $this->printer->note('详细模式已启用');
    }
    
    $this->printer->success("处理文件: {$file}, 名称: {$name}");
}
```

### 参数验证

```php
public function execute(array $args = [], array $data = []): void
{
    // 必需参数验证
    if (empty($args['name'])) {
        $this->printer->error('错误：必须指定 --name 参数');
        $this->printer->note('使用 --help 查看帮助信息');
        return;
    }
    
    // 参数类型验证
    $port = (int)($args['port'] ?? 8080);
    if ($port < 1 || $port > 65535) {
        $this->printer->error('错误：端口号必须在 1-65535 之间');
        return;
    }
    
    // 执行逻辑
    $this->printer->success("启动服务器，端口: {$port}");
}
```

### 示例：创建一个完整的命令类

```php
<?php

namespace YourModule\Console\Cache;

use Weline\Framework\Console\CommandAbstract;
use Weline\Framework\Console\CommandHelper;
use Weline\Framework\Output\Cli\Printing;

class Clear extends CommandAbstract
{
    public function __construct(
        Printing $printer
    ) {
        $this->printer = $printer;
    }

    /**
     * 命令别名
     */
    public function aliases(): array
    {
        return [
            'cc',           // cache:clear 的简短形式
        ];
    }

    /**
     * 命令描述
     */
    public function tip(): string
    {
        return '清除缓存';
    }

    /**
     * 命令帮助信息
     */
    public function help(): array|string
    {
        return CommandHelper::formatHelp(
            'cache:clear',
            '清除应用程序缓存。可以清除所有缓存或指定类型的缓存。',
            [
                '-h, --help' => '显示帮助信息',
                '--type=<类型>' => '指定要清除的缓存类型（可选值：config, data, view, all）',
                '--all' => '清除所有类型的缓存',
            ],
            [],
            [
                '清除所有缓存' => 'php bin/w cache:clear --all',
                '清除配置缓存' => 'php bin/w cache:clear --type=config',
                '使用别名' => 'php bin/w cc --all',
            ]
        );
    }

    /**
     * 执行命令
     */
    public function execute(array $args = [], array $data = []): void
    {
        // 显示帮助
        if (isset($args['h']) || isset($args['help'])) {
            $this->printer->print($this->help());
            return;
        }

        // 获取缓存类型
        $type = $args['type'] ?? null;
        $clearAll = isset($args['all']);

        if ($clearAll) {
            $this->clearAllCache();
        } elseif ($type) {
            $this->clearCacheByType($type);
        } else {
            $this->printer->warning('请指定要清除的缓存类型，或使用 --all 清除所有缓存');
            $this->printer->note('使用 --help 查看帮助信息');
        }
    }

    /**
     * 清除所有缓存
     */
    private function clearAllCache(): void
    {
        $this->printer->note('正在清除所有缓存...');
        
        // 清除逻辑
        $types = ['config', 'data', 'view'];
        foreach ($types as $type) {
            $this->clearCacheByType($type);
        }
        
        $this->printer->success('所有缓存已清除！');
    }

    /**
     * 按类型清除缓存
     */
    private function clearCacheByType(string $type): void
    {
        $allowedTypes = ['config', 'data', 'view'];
        
        if (!in_array($type, $allowedTypes)) {
            $this->printer->error("不支持的缓存类型: {$type}");
            $this->printer->note('支持的类型: ' . implode(', ', $allowedTypes));
            return;
        }

        $this->printer->note("正在清除 {$type} 缓存...");
        
        // 清除逻辑
        // ...
        
        $this->printer->success("{$type} 缓存已清除！");
    }
}
```

创建或修改命令后，需要更新命令列表：

```bash
php bin/w command:upgrade
```

或者使用别名：

```bash
php bin/w cmd:up
```

### 命令缓存

命令列表会被缓存到 `var/commands.php` 文件中，以提高性能。更新命令后会自动更新缓存。

### 查看所有命令

```bash
php bin/w
```

或

```bash
php bin/w cli
```

## 总结

### 创建命令的步骤

1. ✅ 在模块的 `Console` 目录下创建命令类
2. ✅ 实现 `CommandInterface` 或继承 `CommandAbstract`
3. ✅ 实现 `execute()`, `tip()`, `help()` 方法
4. ✅ （可选）实现 `aliases()` 方法或定义 `ALIASES` 常量
5. ✅ 运行 `php bin/w command:upgrade` 更新命令列表
6. ✅ 测试命令是否正常工作

### 别名功能要点

1. ✅ 支持两种方式：`aliases()` 方法（推荐）和 `ALIASES` 常量
2. ✅ 别名在命令扫描和更新时自动注册
3. ✅ 别名不会覆盖已有命令
4. ✅ 保持别名简短且语义清晰

### 最佳实践

1. ✅ 使用 `CommandAbstract` 作为基类
2. ✅ 使用 `CommandHelper::formatHelp()` 格式化帮助信息
3. ✅ 实现 `aliases()` 方法提供命令别名
4. ✅ 在 `execute()` 方法中验证参数
5. ✅ 提供清晰的错误提示和帮助信息
6. ✅ 更新命令后运行 `command:upgrade`
