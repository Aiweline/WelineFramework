# Taglib 编译管道

## 概述

编译管道是 Taglib v2 的核心优化机制，通过一系列可组合的编译通道对 AST 进行优化。

## 管道架构

```
AST 输入
    ↓
ConstantFoldingPass (常量折叠)
    ↓
DeadCodeEliminationPass (死代码消除)
    ↓
InlineOptimizationPass (内联优化)
    ↓
优化后的 AST
```

## 使用编译管道

### 创建默认管道

```php
use Weline\Framework\View\Taglib\Compiler\CompilePipeline;

$pipeline = CompilePipeline::createDefault();
```

### 手动配置管道

```php
$pipeline = new CompilePipeline();
$pipeline->addPass(new ConstantFoldingPass());
$pipeline->addPass(new DeadCodeEliminationPass());
```

### 执行优化

```php
$optimizedAst = $pipeline->process($ast);
```

## 内置优化通道

### ConstantFoldingPass（常量折叠）

将编译期可确定的表达式求值。

**优化示例：**

```html
<!-- 优化前 -->
<lang>STATIC_KEY</lang>

<!-- 优化后（如果是静态翻译） -->
<?= "静态翻译结果" ?>
```

**实现原理：**

```php
class ConstantFoldingPass implements CompilePassInterface
{
    public function process(ProgramNode $ast): ProgramNode
    {
        // 1. 合并连续的 TextNode
        // 2. 尝试折叠静态表达式
        return $ast->withChildren($this->optimizeNodes($ast->children));
    }
}
```

### DeadCodeEliminationPass（死代码消除）

移除不会产生输出的代码。

**优化示例：**

```html
<!-- 优化前 -->
<if condition="false">
    This will never be shown
</if>

<!-- 优化后 -->
（移除整个 if 块）
```

**实现原理：**

```php
class DeadCodeEliminationPass implements CompilePassInterface
{
    public function process(ProgramNode $ast): ProgramNode
    {
        // 1. 移除空文本节点
        // 2. 移除静态 false 条件的 if 块
        // 3. 移除空标签
        return $ast->withChildren($this->eliminateDeadCode($ast->children));
    }
}
```

## 自定义编译通道

### 实现接口

```php
use Weline\Framework\View\Taglib\Compiler\Pass\CompilePassInterface;
use Weline\Framework\View\Taglib\Ast\ProgramNode;

class MyOptimizationPass implements CompilePassInterface
{
    public function process(ProgramNode $ast): ProgramNode
    {
        // 自定义优化逻辑
        return $ast;
    }
    
    public function getName(): string
    {
        return 'my-optimization';
    }
    
    public function getPriority(): int
    {
        return 50; // 优先级，数字越小越先执行
    }
}
```

### 注册通道

```php
$taglib->addCompilePass(new MyOptimizationPass());
```

## 优先级约定

| 优先级范围 | 用途 |
|-----------|------|
| 0-10 | 预处理（如宏展开） |
| 10-30 | 常量折叠、常量传播 |
| 30-50 | 死代码消除 |
| 50-70 | 内联优化 |
| 70-90 | 代码简化 |
| 90-100 | 后处理 |

## 优化示例

### 文本合并

```php
// 优化前 AST
[TextNode("Hello"), TextNode(" "), TextNode("World")]

// 优化后 AST

```

### 静态 if 消除

```php
// 优化前 AST

```

## 调试

### 查看管道通道

```php
$names = $pipeline->getPassNames();
// ['constant-folding', 'dead-code-elimination']
```

### 移除通道

```php
$pipeline->removePass('dead-code-elimination');
```

## 性能考虑

1. **通道顺序** - 某些优化依赖其他优化的结果
2. **避免多次遍历** - 尽量在一次遍历中完成多个优化
3. **缓存中间结果** - 对于复杂计算使用缓存

## 扩展示例

### 自动 HTML 转义通道

```php
class HtmlEscapePass implements CompilePassInterface
{
    public function process(ProgramNode $ast): ProgramNode
    {
        return $ast->withChildren($this->escapeNodes($ast->children));
    }
    
    private function escapeNodes(array $nodes): array
    {
        return array_map(function(Node $node) {
            if ($node instanceof PhpPlaceholder && $node->isEcho) {
                // 包装 htmlspecialchars
                return new PhpPlaceholder(
                    $node->line,
                    $node->placeholder,
                    "htmlspecialchars({$node->expression})",
                    $node->originalCode,
                    true
                );
            }
            return $node;
        }, $nodes);
    }
    
    public function getName(): string { return 'html-escape'; }
    public function getPriority(): int { return 80; }
}
```
