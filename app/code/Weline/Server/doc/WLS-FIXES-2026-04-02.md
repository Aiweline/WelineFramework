## 🔴 严重问题修复详情

### ✅ C1. IPC 消息类型错误 - ServiceOrchestrator::broadcastRoutingCacheClear()

**问题描述**:  
直接使用 `json_encode()` 构造 IPC 消息，而不是使用标准的 `ControlMessage::encode()`，导致消息格式不一致。

**错误日志**:
```
[2026-04-02 08:28:38] [Master@default] [ERROR] [Orchestrator] 控制操作执行异常 
id=ctrl_op_1 action=routing_cache_clear 
error=Weline\Server\IPC\MasterControlServer::sendTo(): Argument #2 ($message) must be of type string, array given
```

**修复前代码**:
```php
$message = \json_encode([
    'type' => ControlMessage::TYPE_ROUTING_CACHE_CLEAR,
], JSON_UNESCAPED_SLASHES | JSON_UNESCAPED_UNICODE) . "\n";
$this->controlServer->sendTo($instance->ipcClientId, $message);
```

**修复后代码**:
```php
$message = ControlMessage::encode([
    'type' => ControlMessage::TYPE_ROUTING_CACHE_CLEAR,
]);
$this->controlServer->sendTo($instance->ipcClientId, $message);
```

**影响**:
- ✅ 消息格式统一，符合 NDJSON 协议
- ✅ Dispatcher 可以正确解析路由缓存清理命令
- ✅ 避免类型错误异常

**文件**: `app/code/Weline/Server/Service/ServiceOrchestrator.php:7186`

### ✅ C2. 类型错误 - SchedulerSystem::sleep() 参数类型不匹配

**问题描述**:  
`SchedulerSystem::sleep()` 方法签名要求 `int` 类型参数，但某些调用可能传入 `float` 类型。

**错误日志**:
```
[2026-04-02 09:04:02] [WorkerSSL#1:16897@default] [ERROR] 
Weline\Framework\Runtime\SchedulerSystem::sleep(): Argument #1 ($seconds) must be of type int, float given
```

**修复前代码**:
```php
\Weline\Framework\Runtime\SchedulerSystem::sleep($bindRetryDelay);
```

**修复后代码**:
```php
\Weline\Framework\Runtime\SchedulerSystem::sleep((int)$bindRetryDelay);
```

**影响**:
- ✅ 避免 TypeError 异常
- ✅ Worker SSL 预热流程稳定
- ✅ 类型安全保证

**问题描述**:  
Dispatcher 进程无法正常启动或注册到 Master。

**错误日志**:
```
[2026-04-02 08:14:33] [Master@default] [ERROR] [Master自检] Dispatcher 异常或缺失: 期望 1，就绪 0 — 尝试维护/拉起
```

**状态**: ⚠️ 需要在运行时调查

**可能原因**:
1. Dispatcher 启动命令参数错误
2. 端口冲突（443 或其他端口被占用）
3. IPC 连接超时
4. 进程启动后立即崩溃
5. 权限问题（Windows 上绑定特权端口）

**调查步骤**:
1. 启动 WLS 服务：`php bin/w server:start`
2. 检查 Dispatcher 进程是否存在：`tasklist | findstr php` (Windows) 或 `ps aux | grep dispatcher` (Linux)
3. 查看 Dispatcher 日志：`var/log/wls/default/error-*.log`
4. 检查端口占用：`netstat -ano | findstr :443`
5. 尝试手动启动 Dispatcher：
   ```bash
   php app/code/Weline/Server/bin/dispatcher.php 0.0.0.0 443 10443 2 default --control-port=19960
   ```

**临时解决方案**:
- 如果是端口冲突，修改配置使用非特权端口（如 8443）
- 如果是权限问题，以管理员身份运行
- 如果是 IPC 超时，增加 `registerTimeout` 配置

## 🟡 中等问题修复详情

### ✅ M1. 资源泄漏风险 - 连接池未正确清理

**问题描述**:  
在连接池 `acquire()` 方法中，如果 `connect()` 成功但在标记 `busy` 前发生异常，连接会泄漏。

**修复前代码**:
```php
if (\count($this->pool) < $maxSize) {
    $conn = $this->createConnection();
    if ($conn->connect()) {
        $this->pool[] = ['conn' => $conn, 'busy' => true, 'last_used' => \microtime(true)];
        return $conn;
    }
}
```

**修复后代码**:
```php
if (\count($this->pool) < $maxSize) {
    $conn = $this->createConnection();
    try {
        if ($conn->connect()) {
            $this->pool[] = ['conn' => $conn, 'busy' => true, 'last_used' => \microtime(true)];
            return $conn;
        }
    } catch (\Throwable $e) {
        // 连接失败，清理资源
        $conn->close();
        $this->log('Connection failed during acquire: ' . $e->getMessage());
    }
}
```

**影响**:
- ✅ 防止连接泄漏
- ✅ 异常情况下正确清理资源
- ✅ 提高长时间运行稳定性

**文件**: `app/code/Weline/Server/Shared/Connection/ConnectionPoolManager.php:84-94`

### ✅ M2. 错误处理不完整 - MasterControlServer::sendTo()

**问题描述**:  
`sendTo()` 方法在发送失败时只检查 EOF，不区分临时错误和永久错误。

**修复前代码**:
```php
if (!$sent) {
    $this->ipcLog("[IPC-Master] SEND FAILED --> {$tag}: type={$type}");
    if (@\feof($socket)) {
        $this->removeClient($clientId);
    }
}
```

**修复后代码**:
```php
if (!$sent) {
    // 获取错误信息
    $errno = @\socket_last_error($socket);
    $errstr = $errno > 0 ? @\socket_strerror($errno) : 'unknown';
    $this->ipcLog("[IPC-Master] SEND FAILED --> {$tag}: type={$type}, errno={$errno}, error={$errstr}");

    // 区分临时错误和永久错误
    // EPIPE(32), ECONNRESET(104), ETIMEDOUT(110), ECONNABORTED(103)
    $permanentErrors = [32, 103, 104, 110];
    if (@\feof($socket) || \in_array($errno, $permanentErrors, true)) {
        $this->removeClient($clientId);
    }
}
```

**影响**:
- ✅ 更详细的错误日志
- ✅ 区分临时错误和永久错误
- ✅ 避免误判断开连接
- ✅ 提高 IPC 通信可靠性

**文件**: `app/code/Weline/Server/IPC/MasterControlServer.php:600-606`
