# WLS (Weline Server) 架构图

## 概述

WLS 是 Weline Framework 的高性能常驻内存 HTTP/HTTPS 服务器，支持多种运行模式和攻击防护能力。

### 非阻塞架构优势

1. **不阻塞请求**: Dispatcher 只写入信号文件，不等待 CDN 响应
2. **异步处理**: Cron 定时任务独立运行，不影响请求处理
3. **自动恢复**: 无新攻击超过 5 分钟后自动关闭防护模式
4. **去重机制**: 已在攻击模式中不重复通知
5. **历史归档**: 攻击记录自动归档，便于后续分析

## 组件说明

### Master Process (主控进程)

- 管理 Dispatcher 和 Worker 进程的生命周期
- 处理信号 (SIGTERM, SIGHUP 热重载)
- 监控子进程状态

### TCP Passthrough Dispatcher (透传分流器)

- 纯 TCP 层面的流量转发
- 不解密 SSL，只透传加密流量
- 使用 MSG_PEEK 读取 ClientHello 提取 SNI
- 集成攻击探测功能

### AttackDetector (攻击探测器)

| 检测类型 | 说明 | 默认阈值 |
|---------|------|---------|
| rate_limit | 请求频率限制 | 100次/10秒 |
| path_scan | 路径扫描检测 | 20次404/60秒 |
| malicious_pattern | 恶意特征匹配 | SQL注入/XSS/命令注入 |
| bad_user_agent | 恶意UA检测 | 扫描工具/恶意爬虫 |
| protected_path | 保护路径 | .env/.git/wp-admin |
| slowloris | 慢速攻击 | 30秒/请求 |

### RoutingCacheService (路由缓存服务)

- SNI 到 Worker 端口的映射缓存
- IP 到 Worker 端口的映射缓存
- 连接 ID 映射（Keep-Alive 场景）
- 学习模式：从 Worker 响应的 X-Weline-Route-Hint 头学习路由

### Worker (工作进程)

- SSL 握手和解密
- HTTP 请求处理
- 业务逻辑执行
- Route Hint 自报告（添加 X-Weline-Route-Hint 响应头）

## CDN 适配器接口

所有 CDN 适配器必须实现 `AdapterInterface`，攻击防护相关方法：

```php
interface AdapterInterface
{
    // ... 其他方法 ...
    
    /**
     * 开启攻击防护模式
     */
    public function enableAttackMode(string $zoneId, array $credentials, array $attackData = []): array;
    
    /**
     * 关闭攻击防护模式
     */
    public function disableAttackMode(string $zoneId, array $credentials): array;
    
    /**
     * 检查是否支持攻击防护模式
     */
    public function supportsAttackMode(): bool;
}
```

## Cloudflare 攻击防护实现

Cloudflare 适配器实现了以下攻击防护功能：

1. **Under Attack Mode**: 通过 `settings/security_level` API 设置为 `under_attack`
2. **IP 封禁**: 通过 `firewall/access_rules/rules` API 封禁攻击者 IP
3. **自动恢复**: `disableAttackMode` 将安全级别恢复为 `medium`

```php
// Cloudflare API 调用示例
// 开启攻击模式
PATCH /zones/{zone_id}/settings/security_level
{ "value": "under_attack" }

// 封禁 IP
POST /zones/{zone_id}/firewall/access_rules/rules
{
    "mode": "block",
```
